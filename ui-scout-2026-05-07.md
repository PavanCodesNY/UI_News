## 1. ASTRODITHER — https://www.awwwards.com/sites/astrodither

WHAT IT DOES: Full-viewport WebGPU canvas running an audio-reactive Three.js scene with a custom GPU fluid simulation, selective bloom post-processing, and a Bayer-style ordered-dither shader laid over the entire framebuffer. The single-page layout is a fixed canvas with overlaid HTML containing minimal centered serif type and a small bottom-right meta block; user input (audio amplitude / FFT bins, scroll-driven time-warp uniform) drives shader uniforms in real time, and the entire post chain runs in TSL (Three Shader Language) so it is a node-graph compositing the bloom + dither + chromatic aberration passes.

PACKAGES:
- `three@^0.165` (must be a build that exposes WebGPU + TSL — `three/webgpu` and `three/tsl`)
- `@react-three/fiber@^9` (only if you want React; raw Three.js is fine)
- `@react-three/drei@^9.105` for `<OrthographicCamera>`, `<Stats>`, `<Loader>`
- `gsap@^3.12` + `gsap/ScrollTrigger` for the time-warp scroll uniform
- `lenis@^1.1` for smooth scroll feeding ScrollTrigger
- `tone@^15` or the native Web Audio API `AnalyserNode` for FFT
- `next@^15` (App Router) or `vite@^5` for the host
- `tailwindcss@^4` for the HTML overlay
- Fonts: `@fontsource-variable/instrument-serif` for the headline, `@fontsource-variable/jetbrains-mono` for the meta

IMPLEMENTATION:
1. Mount one full-viewport `<canvas>` with `WebGPURenderer` (`renderer = new THREE.WebGPURenderer({ antialias: true, powerPreference: 'high-performance' })`); fall back to `WebGLRenderer` if `navigator.gpu` is missing.
2. Build the scene as an offscreen `RenderTarget` (RGBA16F) at `devicePixelRatio` capped at 1.5 for perf. Inside it: a `MeshBasicMaterial` quad textured with the fluid sim output (a ping-pong pair of float textures updated per frame by a TSL compute node — advection → divergence → pressure (20 jacobi iters) → gradient subtract).
3. Post chain in TSL: `bloom(threshold:0.85, radius:0.6, intensity:1.2) → chromaticAberration(strength:0.0025) → dither(matrix:bayer8x8, levels:6)`. The dither node samples a precomputed 8×8 Bayer threshold map and quantizes each channel to N levels — this is the signature look.
4. Audio: `audioCtx.createMediaElementSource(audioEl) → AnalyserNode(fftSize:512)`. Each frame copy `getByteFrequencyData()` into a `Float32Array` uniform `uAudio`. Drive `uBloomIntensity`, `uFluidForce`, and `uTimeWarp` from low/mid/high bands.
5. Scroll-driven time warp: `ScrollTrigger.create({ scrub: 1.5, onUpdate: self => uniforms.uTimeWarp.value = self.progress * 4 })`.
6. HTML overlay: `position:fixed; inset:0; pointer-events:none;` with two layers — center wordmark (`mix-blend-mode: difference; font-size: clamp(48px, 9vw, 144px); letter-spacing:-0.04em; font-weight:300`) and a bottom-right meta strip (`font-family:JetBrains Mono; font-size:11px; opacity:0.7; uppercase`).
7. Performance: pause RAF when `document.hidden`, use `IntersectionObserver` to suspend when canvas off-screen, and clamp `renderer.setPixelRatio(Math.min(window.devicePixelRatio, 1.5))`. Disable bloom on `(prefers-reduced-motion: reduce)` and on devices with `navigator.deviceMemory < 4`.
8. Mobile: switch to `WebGLRenderer` + half-res render target + skip fluid sim (replace with a static normal-map distortion).

WHY IT WORKS: Ordered dithering is a deliberate downward visual move — quantizing to 6 levels per channel reads as "this is computed, not rendered" which signals technical authorship. The single-focal-point center type with `mix-blend-mode: difference` over a chromatic background creates maximum figure-ground separation while remaining unified. Audio-driven uniforms convert passive looking into a feedback loop, exploiting the same engagement loop as music visualizers.

GOOD: The TSL post chain is genuinely state-of-the-art — most sites still ship raw GLSL through `EffectComposer`. Audio reactivity is novel but optional, so the page degrades gracefully.
BAD: WebGPU coverage is still ~75% — the WebGL fallback drops the dither uniformity. No keyboard skip-to-content; screen readers see only the wordmark. Pulls 18MB on first paint.

CLEAN APPLICATION: Apply this to Clean's landing page hero by replacing the dither + audio with **a deterministic GPU dither pass driven by live MCP-traffic stats**. The hero canvas renders a low-poly mesh of Clean's logo or context-graph; a TSL post-pass dithers it to 5 levels using a Bayer matrix, and the dither density `uDensity` is bound to a `WebSocket` of real proxy calls/sec from a public Clean MCP demo endpoint. The headline `"Less context. More signal."` sits center with `mix-blend-mode: difference`. This converts the marketing surface into live infra telemetry — the page literally is the product.

## 2. Fourmula AI — https://www.awwwards.com/sites/fourmula-ai

WHAT IT DOES: A two-color (#020108 black, #FF6B02 orange) marketing site for a generative-asset AI built in Webflow + GSAP. The layout is a vertical stack of full-bleed sections each with a different scroll-pinned animation: hero with a kinetic typographic mask reveal, a horizontal-scroll preset gallery (translateX bound to scrollY), a sticky comparison slider, and a text-marquee CTA footer. Type-led, image-sparse — every section uses the orange as a single accent against the near-black ground.

PACKAGES:
- `webflow` host (or rebuild as `next@^15` + `tailwindcss@^4`)
- `gsap@^3.12` with `gsap/ScrollTrigger`, `gsap/SplitText`, `gsap/ScrollSmoother`, `gsap/CustomEase`
- `lenis@^1.1` (if non-Webflow)
- `motion@^11` (Framer Motion) for any React islands
- Fonts: a wide grotesk like `@fontsource-variable/space-grotesk` paired with `@fontsource-variable/inter` for body
- `lucide-react@^0.400` for icons
- `lottiefiles/react-lottie-player@^3` for the animated logo loop

IMPLEMENTATION:
1. Global tokens — Tailwind config: `colors: { ink: '#020108', flame: '#FF6B02' }`, `fontFamily: { display: 'Space Grotesk Variable', body: 'Inter Variable' }`. CSS vars: `--accent: #FF6B02; --bg: #020108; --fg: #F5F1EA`.
2. Scroll smoothing: `ScrollSmoother.create({ smooth: 1.4, effects: true, normalizeScroll: true })`. ALL sections get `data-speed` parallax.
3. Hero: `SplitText` the headline into chars; entry tween `gsap.from('.hero-char', { yPercent: 110, stagger: 0.012, duration: 0.9, ease: CustomEase.create('out', '0.16,1,0.3,1') })`. A masked SVG of the wordmark fills with `--flame` via `clip-path: inset(0 100% 0 0)` animated to `inset(0 0 0 0)` over 1.2s.
4. Horizontal preset gallery: outer `section` is `100vh`, inner `track` is `width: 320vw`, pin with `ScrollTrigger.create({ trigger: section, pin: true, scrub: 1, end: '+=300%' })` and tween `gsap.to(track, { x: '-220vw' })`. Each card is `420px × 60vh` with `border: 1px solid rgba(245,241,234,0.08)` and an inner image cover.
5. Sticky comparison slider: a single `<div>` with two stacked images, the top one clipped by `clip-path: inset(0 var(--split) 0 0)`. Bind `--split` to a draggable handle (`Draggable.create`) AND to scroll progress so it animates on enter and is interactive after.
6. Marquee footer: two duplicated rows, `animation: marquee 28s linear infinite`, each glyph uppercase `font-size: clamp(96px, 18vw, 320px); letter-spacing: -0.06em; line-height: 0.85`.
7. Cursor: a 12px circle that scales to 64px on `[data-cursor="hover"]` elements; add `mix-blend-mode: difference` so it inverts over the orange.
8. Responsive: kill ScrollSmoother + horizontal pin under 1024px; collapse the preset gallery to a vertical stack with `scroll-snap-type: y mandatory`.

WHY IT WORKS: Limiting the palette to two colors forces hierarchy through type weight and motion, not chroma — exactly the constraint that makes type-led sites feel premium. The split-text + ease curve `cubic-bezier(0.16, 1, 0.3, 1)` reads as "expensive" because it's the same easing used in Apple keynotes. Horizontal scroll resets the user's spatial expectation, making the gallery feel like a separate room.

GOOD: Two-color discipline. CustomEase + SplitText is the gold standard for marketing-site type reveals.
BAD: Horizontal-pin sections murder mobile UX if not gated by `min-width:1024px`. The whole site is heavy on JS — first contentful paint suffers; no progressive enhancement story.

CLEAN APPLICATION: Apply this to Clean's pricing page by **replacing the horizontal preset gallery with a horizontal-scroll plan comparison** (`Free / Team / Scale / Enterprise` as four 420px cards) where each card is a code-block preview of the matching `clean.config.toml`. Use the two-color discipline — Clean's existing brand color as `--accent` against near-black. Hero headline `"Pricing that scales with your context window."` reveals via SplitText. The marquee footer becomes an infinite ticker of customer logos at `font-size: clamp(64px, 12vw, 200px)` instead of typical small grayscale logo cloud — this signals "we have real customers" louder than a logo bar.

## 3. T11 — https://www.awwwards.com/sites/t11

WHAT IT DOES: A creative-entertainment agency portfolio (Awwwards SOTD May 7, 2026) built on a heavy GSAP timeline. The layout is a sequence of full-viewport "scenes" — hero with a layered video bento, a case-study grid that morphs between mosaic and list views via scroll, a vertically-pinned timeline of past activations, and a contact section with a magnetic CTA. Imagery is bold high-contrast video clips with `mix-blend-mode: lighten` overlays carrying typographic captions.

PACKAGES:
- `next@^15` App Router
- `gsap@^3.12` + `ScrollTrigger`, `Flip`, `MotionPath`, `Observer`, `SplitText`
- `lenis@^1.1` (LenisOptions: `{ duration: 1.2, easing: t => 1 - Math.pow(1 - t, 4) }`)
- `framer-motion@^11` for the magnetic cursor + page transitions
- `@studio-freight/hamo@^0.6` for `useResizeObserver`/`useFrame` hooks
- `next-video@^2` for adaptive HLS/MP4 video tiles
- `tailwindcss@^4`
- Fonts: a modern serif display like `@fontsource-variable/fraunces` + `@fontsource-variable/general-sans`

IMPLEMENTATION:
1. Hero bento: CSS grid `grid-template: repeat(6, 1fr) / repeat(12, 1fr); gap: 8px;` filled with 5 video tiles at varying `grid-area`. Each tile autoplays muted, `object-fit: cover`, with a 1px inner ring `box-shadow: inset 0 0 0 1px rgba(255,255,255,0.06)`.
2. GSAP `Flip` for mosaic→list: capture state with `Flip.getState('.case-tile')`, change classes (re-templating the grid to single-column), then `Flip.from(state, { duration: 1.1, ease: 'power3.inOut', stagger: 0.04 })` driven by an `Observer.create({ type:'wheel,touch', onUp/onDown })`.
3. Vertical pinned timeline: outer section `height: 600vh`, inner sticky panel `height: 100vh`. ScrollTrigger pins it; a horizontal year marker (`2018 → 2026`) translates as `x: -progress * trackWidth`. Each case tile fades+scales in via `ScrollTrigger.batch('.year-card', { start:'top 80%', onEnter: els => gsap.to(els, { y:0, opacity:1, stagger:0.06 }) })`.
4. Magnetic CTA: `onMouseMove` compute distance from button center, translate button by `dx*0.25, dy*0.25` with `gsap.to(btn, { x, y, duration: 0.6, ease: 'power3.out' })`. Inside, a 100% rounded background scales from `0` to `1` on hover via `transform-origin: bottom`.
5. Captions: `position: absolute; bottom: 16px; left: 16px; mix-blend-mode: lighten; text-transform: uppercase; font-family: General Sans; font-size: 12px; letter-spacing: 0.08em; line-height: 1`.
6. Page transitions: a single full-viewport `motion.div` with `clipPath: inset(0 0 100% 0)` → `inset(0 0 0% 0)` on exit then reverse on enter, `duration: 0.7, ease: [0.83, 0, 0.17, 1]`.
7. Performance: lazy-load videos with `IntersectionObserver(rootMargin: '200px')`; preconnect to the video CDN; use `<link rel="modulepreload">` for GSAP plugins on the route.

WHY IT WORKS: The Flip-driven mosaic↔list is a Gestalt continuity demo — the same elements are continuously transformed instead of replaced, which the brain reads as "the same world from a different angle" rather than a page reload. Heavy serif display + tight monospace caption is a long-running editorial pairing that signals "thinking studio" not "template farm."

GOOD: Flip-based layout transitions are still the highest-craft pattern in 2026 web. Magnetic CTA with `power3.out` is calibrated correctly (most sites overshoot to `back.out`).
BAD: 6× viewport-height pinned section is brutal on low-end Android. Video-heavy hero pushes LCP past 4s on 4G.

CLEAN APPLICATION: Apply this to Clean's product-demo section by using a **Flip-driven view toggle between "Raw context" and "Cleaned context"**. Two states of the same content: state A is a chaotic JSON blob with all 200+ tools spilled out, state B is the same elements re-grouped into 6 semantic clusters with the rest filtered. Use `Flip.from(state, { duration: 1.4, ease: 'power3.inOut' })` triggered by a single button. Captions in JetBrains Mono caption style. This is the single most powerful demo we can ship — the user sees the actual product mechanic in pure DOM motion, no video.

## 4. Immersive Garden — https://www.awwwards.com/immersivegarden/

WHAT IT DOES: Studio site (CSSDA SOTD May 5, 2026) anchored on a custom WebGL scroll experience — a single procedurally-generated mesh in the hero (subdivided plane with vertex displacement driven by simplex noise), a horizontally-scrolling case index where each thumbnail is a `<canvas>` running its own RGB-shift hover shader, and an "About" section with 3D-text built via Troika that follows the mouse with a damped spring. Layout discipline is strict 12-column with generous 1.4× line-heights and 2× section padding.

PACKAGES:
- `three@^0.168` + `three/examples/jsm/postprocessing/*`
- `@react-three/fiber@^9`, `@react-three/drei@^9.105`, `@react-three/postprocessing@^2.16`
- `troika-three-text@^0.50` for the 3D type
- `gsap@^3.12` + `ScrollTrigger`
- `next@^15`, `tailwindcss@^4`
- `clsx@^2`, `framer-motion@^11`
- `simplex-noise@^4` for the hero displacement
- Fonts: `@fontsource-variable/migra` (or similar high-contrast serif) + `@fontsource-variable/inter`

IMPLEMENTATION:
1. Hero plane: `PlaneGeometry(8, 4, 256, 128)`, `ShaderMaterial` with a vertex shader that does `pos.z += snoise(vec3(pos.xy * 0.6, uTime * 0.2)) * 0.4`. Fragment paints by world-Y to give a subtle gradient. Camera at `[0, 0, 4]`, fog enabled.
2. Per-tile shader on the case index: each thumbnail is a `<Canvas>` with a single textured quad, fragment shader `sample(uvA + offsetR) → R, sample(uvA + offsetG) → G, sample(uvA + offsetB) → B`, where `uvA` is the base UV and `offset*` ramps from 0 → 0.012 on hover via a `uHover` uniform tweened by `useSpring({ stiffness: 120, damping: 22 })`.
3. Horizontal case index: `flex; overflow-x: scroll; scroll-snap-type: x mandatory`. Each card `min-width: 32rem; aspect-ratio: 4/5`. Bind `scrollLeft` to ScrollTrigger so vertical wheel becomes horizontal scroll above 1024px.
4. 3D follow text: `<Text3D>` from Troika; in `useFrame` lerp `mesh.rotation.y` toward `pointer.x * 0.3` with damping 0.1.
5. Type system: hero `clamp(72px, 11vw, 192px)` migra serif at `letter-spacing: -0.03em; line-height: 0.95`; body `18px / 28px` inter at `letter-spacing: 0.005em`. Section padding `clamp(96px, 14vh, 196px)` top/bottom.
6. Pointer cursor: small dot (`8px`) + outer ring (`32px`) lagging 0.12s; ring scales to `64px` over interactive elements.
7. Performance: `dpr={[1, 1.5]}` on `<Canvas>`, `frameloop="demand"` with manual `invalidate()` on hover, and quantize the noise to a 64×32 grid then bilinear-sample in vertex shader to halve cost.

WHY IT WORKS: The vertex-displacement plane reads as "geological surface" — cognitively safer than full free-form 3D for a B2B audience. RGB shift on hover is restrained (max 0.012 UV) so it whispers rather than shouts. 12-column rigor + serif headline = "studio you'd hire to redesign your bank."

GOOD: Restraint. RGB shift at 0.012 UV is the right magnitude. Frameloop demand keeps idle GPU at 0%.
BAD: Three independent `<Canvas>` instances per page = three WebGL contexts. Some browsers cap at 16 contexts site-wide which becomes a real issue on case-detail pages.

CLEAN APPLICATION: Apply this to Clean's docs landing surface by **using the hero displacement plane to render Clean's context graph as a vertex-displaced 3D field**. Each "node" in the cleaned MCP graph is a vertex; vertex Z displaces by token count. Mouse pan rotates the camera 8° max. Below the hero, a horizontal-scroll docs index (Quickstart, Proxy, TOON, Filters, Routing, SDK) using the same per-tile RGB-shift shader — every doc card looks alive but never moves until pointed at. This is the rare case where 3D is justified: the product literally is a graph.

## 5. Aceternity UI — Hero Section with Shadow and Scales / ASCII Art / Text Flipping Board (March 2026 drops) — https://ui.aceternity.com/components

WHAT IT DOES: Three free components dropped within the last six weeks that compose into a complete startup landing-page hero. The Hero with Shadow and Scales is a centered split-flap-style headline over an animated radial mesh-gradient with a long inner-shadow vignette. The ASCII Art component renders any image as live ASCII via a `<canvas>` ImageData read-back, with a 15-preset charset switcher and per-frame animation. The Text Flipping Board scrambles random characters before settling on the target string — a split-flap simulation. All three are Tailwind + Framer Motion and ship as drop-in TSX.

PACKAGES:
- `next@^15` (App Router, RSC-compatible — these are `"use client"` islands)
- `react@^19`, `react-dom@^19`
- `tailwindcss@^4` (CSS-first config; `@theme` block in `globals.css`)
- `framer-motion@^11.18` (the Aceternity components target the new `motion/react` import)
- `clsx@^2` + `tailwind-merge@^2` (the `cn()` util Aceternity uses)
- `lucide-react@^0.400`
- `@radix-ui/react-tabs@^1` and `@radix-ui/react-tooltip@^1` if pairing with shadcn/ui blocks
- Fonts: `@fontsource-variable/geist` + `@fontsource-variable/geist-mono` (the default Aceternity stack)

IMPLEMENTATION:
1. Install shadcn/ui base: `pnpm dlx shadcn@latest init` then add `tailwindcss-animate` and the `cn` utility. Create `lib/utils.ts` with `export const cn = (...i: ClassValue[]) => twMerge(clsx(i))`.
2. Hero with Shadow and Scales: outer wrapper `relative h-[100svh] flex items-center justify-center overflow-hidden bg-black`. Background = a `<div>` with `background: radial-gradient(60% 50% at 50% 40%, rgba(120,120,255,0.25) 0%, transparent 60%)` + a second layer with `conic-gradient` rotated by a CSS variable updated in a `requestAnimationFrame` loop at `0.05deg/frame`. Inner shadow: `box-shadow: inset 0 -200px 200px -100px rgba(0,0,0,0.8)`. Heading wraps the Text Flipping Board.
3. Text Flipping Board: store `target` string, animate each char index from a random ASCII (0x21–0x7E) to the target at staggered `delay = i * 24ms`, total resolve `~600ms`. Use `useReducedMotion()` to skip scramble.
4. ASCII Art: `<canvas>` 800×600, draw the source image via `drawImage`, `getImageData()` returns a `Uint8ClampedArray`. For each `8×16` block, average luminance, map to charset index `chars[Math.floor((avg / 255) * (chars.length - 1))]`, render with `ctx.fillText(char, x, y)`. Animate by sliding the source image's `sx` by 1px per frame (panning) or by progressively revealing chars left-to-right.
5. Compose: Hero stack = `<HeroShadowScales><FlippingBoard text="Less context. More signal." /><CTAButtons /></HeroShadowScales>` with the ASCII Art component used in a secondary "How it works" section as a logo treatment.
6. Type tokens: headline `font-family: 'Geist', sans-serif; font-size: clamp(56px, 8vw, 128px); font-weight: 600; letter-spacing: -0.04em; line-height: 0.95`. Mono for code/captions: `font-family: 'Geist Mono'; font-size: 13px`.
7. Color tokens (dark by default): `--background: #0A0A0A; --foreground: #F5F5F5; --muted: #1A1A1A; --accent: #6366F1`. Toggle to light only if a system preference flips.
8. Accessibility: every Framer Motion variant gets a `useReducedMotion` guard returning a static state; the ASCII canvas has an `<img alt>` sibling for screen readers.

WHY IT WORKS: Shadow + Scales gives a centered focal point with a long vertical fall-off — the eye reads the inner shadow as a horizon line, anchoring the headline. Split-flap text triggers the "casino payout" attention pattern: fast random motion resolving to a stable answer holds gaze for the entire 600ms resolve. ASCII art reframes whatever image you put in it as "computed" — immediate signal that this is a developer tool.

GOOD: Aceternity's components ship with proper `forwardRef`, prefers-reduced-motion, and CSS-var theming — drop-in, not "demo only."
BAD: The radial + conic gradient combo lights up paint cost; on Safari it can stutter. ASCII Art is a genuinely heavy `getImageData` per frame — fine for a logo, do not run it on a hero background.

CLEAN APPLICATION: Apply this to Clean's landing page hero AS THE PRIMARY BUILD TODAY. Compose: Hero with Shadow and Scales as the wrapper → Text Flipping Board for the headline `"Less context. More signal."` (resolves on mount, then re-scrambles every 8s with a different value: "Tokens saved", "Tools filtered", "Latency cut") → centered subhead in Geist Mono → two CTAs (`Start free` primary, `Read the docs` ghost with `border-input/50`) → below, the ASCII Art component rendering Clean's wordmark live in `Geist Mono` at 11px char size. This pattern is shadcn-compatible so it slots cleanly into any existing Next.js App Router shell.

KEY CONSTRAINTS: Clean is developer infrastructure. The result must read as **fast, deterministic, computed** — not flashy or generic. Specifically AVOID: (1) stock gradient blobs without a reason — every gradient must be tied to a uniform that maps to product data, (2) any cursor-follow 3D or spotlight cursor effects (overdone, signals "AI wrapper site"), (3) auto-playing video, (4) full-viewport horizontal-pin sections (we are not a portfolio site), (5) emoji or rounded illustrative SVG, (6) more than two type families and one accent color, (7) hero LCP > 2.5s on 4G — the WebGPU pass must be lazy and behind `IntersectionObserver`. Performance budget: JS chunk for the marketing route ≤ 180KB gzipped, hero LCP ≤ 1.8s on M2 / WiFi, `prefers-reduced-motion` must produce a fully static page.
