## 21 Hrs On The Moon — https://www.awwwards.com/sites/21-hrs-on-the-moon

WHAT IT DOES: Long-form scroll-driven WebGL narrative documenting the 21 hours humans spent on the lunar surface, built by Studio 28K + Set Snail. Layout is a full-viewport WebGL canvas locked behind a virtual (Lenis-style) scroll, with an editorially typeset foreground layer that fades chapter titles and captions in sync with camera moves through a low-poly moon terrain. Visual hierarchy is a binary duotone (#060B0B on #FFF3EA) with SDF-rendered display type driving all information density — no imagery competes with the 3D scene.

PACKAGES:
- next@14 (app router, static export)
- three@0.165
- @react-three/fiber@8, @react-three/drei@9
- @react-three/postprocessing@2 (Bloom, Vignette, DepthOfField)
- gsap@3.13 with ScrollTrigger + Observer
- lenis@1.1 (smooth virtual scroll driving GSAP)
- tailwindcss@3.4 (foreground only)
- @fontsource-variable/inter-tight for foreground, self-hosted display serif (e.g., PP Editorial New) for chapter markers
- ktx2-loader + BasisTextureLoader for compressed textures
- howler@2 (spatial audio triggers per chapter)

IMPLEMENTATION: Component tree: `<ScrollProvider>` (Lenis) → `<Scene>` (`<Canvas dpr={[1, 1.75]} gl={{ antialias: false, powerPreference: "high-performance" }}>` with a single `<PerspectiveCamera fov={38}>`, `<MoonTerrain>` (KTX2 heightmap → `MeshStandardMaterial` with tri-planar shader), `<DustParticles count={2400}>`, `<PostFX>`), plus a sibling `<ChapterOverlay>` (absolute-positioned HTML). GSAP timeline is scrubbed by Lenis progress (`ScrollTrigger.scrollerProxy` + `lenis.on('scroll', ScrollTrigger.update)`); each chapter is a labeled tween that lerps camera position + look-at target + fog density. Text reveals: `SplitText` chars → `y: 40 → 0`, `opacity: 0 → 1`, `duration: 0.9`, `ease: "expo.out"`, `stagger: 0.012`. Canvas is 100vh sticky; the "scroll length" is a spacer div `height: 800vh`. Perf: cap DPR at 1.75, `frameloop="demand"` with `invalidate()` on scroll, single directional light + hemisphere fill (no shadows), instanced dust. Responsive: below 900px swap terrain for a Lottie-baked prerender fallback and disable PostFX.

WHY IT WORKS: The binary palette eliminates chroma decisions and forces attention onto motion and typography — this is Gestalt figure/ground turned into a story engine. Every scroll input maps to a camera micro-move (motion semantics), so the user feels like they're driving the narrative, not watching it. Editorial numbering ("01 / 21 HRS", "02 / DESCENT") anchors progress without a progress bar, respecting the 8pt vertical rhythm.

GOOD: The scroll↔camera coupling never drops frames on M-series hardware; the fallback path degrades gracefully.
BAD: Mobile support is a resigned Lottie prerender — not the real experience; no reduced-motion path documented.

CLEAN APPLICATION: Apply this to Clean's landing-page hero on tryclean.ai by replacing the moon terrain with a low-poly "data mesh" (nodes = leads, edges = enrichment paths) that Huginn traces as the user scrolls, with three editorial chapters — "01 / SOURCE", "02 / ENRICH", "03 / ACTIVATE" — each timed to a camera pan across the mesh while a live-looking KPI counter tickers in the corner. Use the exact GSAP+Lenis+ScrollTrigger stack above; keep the duotone constraint (Clean off-white on ink-black) so the hero reads as "developer infrastructure," not marketing site.

## Brunello Cucinelli — Callimacus AI E-com — https://www.awwwards.com/sites/brunello-cucinelli-ai-e-com

WHAT IT DOES: An e-commerce experience by makemepulse where there are no static pages — the interface is dynamically assembled per interaction by the Callimacus AI runtime (Groq-hosted, millisecond inference). A single persistent contextual prompt bar sits along the bottom; the entire viewport above it is a component canvas that mutates on intent (browse → grid, "show me linen for summer evenings" → mood board, "compare these two" → side-by-side). The AI is invisible — no chat bubbles, no "AI is thinking" — just a stream of modular components sliding into place.

PACKAGES:
- next@15 (app router, streaming RSC)
- react@19 (use() + Suspense boundaries per component slot)
- framer-motion@11 (layout animations for component swap)
- @tanstack/react-query@5 (streamed intent → component payload)
- radix-ui primitives (Popover, Dialog, ToggleGroup) as unstyled base
- tailwindcss@3.4 with a strict token-only config
- vaul@0.9 (bottom-sheet prompt bar on mobile)
- @groq/sdk (millisecond inference for intent parsing)
- lenis@1.1 for scroll continuity across component swaps
- three@0.165 + @react-three/fiber for the ambient product 3D module (optional slot)

IMPLEMENTATION: The frontend is a two-layer architecture — a persistent `<PromptShell>` (bottom-fixed input, ~72px tall, `backdrop-blur-xl` on `rgba(20,18,16,0.72)`) wrapping a `<ComponentCanvas>` that renders a stream of slot components keyed by intent hash. Intent parsing: on submit, POST to Groq → returns a typed layout schema `{ slots: [{ id, type: "grid" | "hero" | "compare" | "story", props }] }`; hydrate slots as async server components. Component swap uses framer-motion `<LayoutGroup>` + `layoutId` shared across slot types so a product card that was in a grid can morph into the hero of a story panel — `transition: { type: "spring", stiffness: 260, damping: 32 }`. Typography is a display serif at 5.5vw for hero moments, 15px/1.55 humanist sans for body. No navigation bar — a subtle "trail" of past intents lives as a horizontal chip strip above the prompt (`overflow-x-auto`, `scroll-snap-type: x mandatory`). Perf: RSC streams first paint under 800ms; framer-motion `LayoutGroup` uses FLIP so swaps stay at 60fps; images ship as `<Image priority>` with AVIF + LQIP.

WHY IT WORKS: The invisible-AI approach fixes the biggest failure of chat-UX in production: users don't want to talk, they want to be understood. By making the prompt a peripheral utility and the canvas the star, it re-establishes visual hierarchy where content dominates and controls recede — the exact inversion of ChatGPT. Layout continuity via `layoutId` gives motion semantic meaning: things that morph are "the same thing seen differently," which is how humans think.

GOOD: Millisecond intent inference makes the interface feel psychic; RSC streaming means no loading skeletons.
BAD: SEO story is complicated (dynamic assembly), and without an obvious taxonomy, first-time visitors can feel lost — needs an onboarding intent primer.

CLEAN APPLICATION: This is the single most important reference for Clean today. Apply this to Clean's **product demo section** on tryclean.ai (the fold immediately below the hero) by building a live, non-static "Ask Clean" surface: a persistent prompt bar at the bottom of the section, prefilled with rotating example intents ("find 200 Series A CTOs in fintech hiring right now"), and above it a `<ComponentCanvas>` that renders three demo scenes on rotation — leads grid, enrichment card, outreach draft — all sharing `layoutId` so a lead card morphs from list → detail → email draft in one continuous motion. Use the framer-motion + shared layoutId pattern exactly as spec'd. This proves what Clean *is* without a video.

## MONOLOG — https://www.awwwards.com/sites/monolog

WHAT IT DOES: Studio portfolio for MONOLOG (Huy Nguyen) built on Webflow with layered GSAP + Three.js, presented as a monochrome (#080807 / #DDDDD5) editorial with a single wide serif display face doing all the heavy lifting. Layout is a vertical rhythm of over-scaled type blocks interleaved with muted WebGL "spotlight" moments where a shader-warped image floats behind the text. Scroll-driven horizontal case study bands break the vertical flow at deliberate intervals.

PACKAGES:
- Webflow (host)
- gsap@3.13 with ScrollTrigger, ScrollSmoother, SplitText, DrawSVGPlugin
- three@0.165 (loaded per-section, not global)
- lenis@1.1 (or Webflow's native ScrollSmoother if paired with GSAP Club)
- glslify + custom fragment shaders for the "distortion" hover on hero image
- @fontsource-variable/fraunces or similar wide-optical-size serif at 12vw display
- lil-gui only in dev
- rive-react@4 (for the small logotype loop)

IMPLEMENTATION: Sections are stacked full-viewport blocks; SplitText splits the display headline into words, then GSAP staggers `clip-path: inset(100% 0 0 0) → inset(0 0 0 0)`, `duration: 1.1`, `ease: "expo.out"`, `stagger: 0.08`. Each hero image is a WebGL plane (Three.js `PlaneGeometry` sized to CSS bounding rect on resize) with a fragment shader that samples a noise texture as displacement, driven by `uMouse` (lerped) and `uScrollVelocity` (from Lenis). Horizontal case study strip uses `ScrollTrigger` with `pin: true` and `scrub: 1` translating a container `x: 0 → -(container.width - window.innerWidth)`. Type scale system: display 12vw / hero 6vw / eyebrow 0.72rem uppercase tracking 0.14em / body 1.05rem line-height 1.55. Vertical rhythm on 8px grid, section paddings on 12vh multiples. All animations honor `prefers-reduced-motion` by short-circuiting to `duration: 0.01`.

WHY IT WORKS: Restraint. One typeface, two colors, one motion primitive per section — the result reads as authored, not assembled. Over-scaled display type creates a magazine-grade information hierarchy that instantly signals "craft" (contrast ratio between display and body exceeds 20:1 in optical weight). The WebGL layer isn't a stunt — it lives *behind* the copy at low opacity, doing the job of texture without breaking readability.

GOOD: The horizontal-in-vertical break is a masterclass in pacing — a genuine surprise that resets attention.
BAD: 12vw display type gets uncomfortable below 380px viewports; body copy lags on lower-end Android.

CLEAN APPLICATION: Apply this to Clean's **customer stories / case studies** section on tryclean.ai by pinning one horizontal band inside an otherwise vertical scroll — three case study "cards" (Series A fintech, dev tools startup, RevOps agency) slide horizontally as the user scrolls vertically, each with an over-scaled quote in the wide serif and a muted WebGL underlay of the customer's logo mark being deformed by scroll velocity. Use the exact ScrollTrigger `pin + scrub` pattern above. This gives Clean editorial gravitas without stock-photo cringe.

## Depo Luxe — https://www.awwwards.com/sites/depo-luxe

WHAT IT DOES: A luxury film campaign archive presented as a director's reel — a vertical film-strip navigation of case studies (McQueen, Kenzo, Azzaro) with editorial numbering ("N°01 / N°02 / N°03") and one fullscreen looping video per row. Built on 11ty (static SSG) with a tight 2-color palette (#fff / #000). Navigation is a single left-edge index; hovering a row previews the film in the right panel with a masked wipe.

PACKAGES:
- @11ty/eleventy@3 (static site generator)
- @11ty/eleventy-img (AVIF + WebP + LQIP)
- alpine.js@3 or vanilla ES modules (no framework runtime)
- gsap@3.13 with SplitText + CustomEase
- lenis@1.1
- hls.js@1 for adaptive video (or mux-player)
- postcss + open-props for the 2-color token layer
- @fontsource-variable/inter-tight (body) + a wide editorial display (headers)

IMPLEMENTATION: Layout is a two-column grid `grid-template-columns: minmax(320px, 32%) 1fr`. Left column: sticky navigation, one row per case study, each row `padding: 2vh 3.2vw`, `border-bottom: 1px solid rgba(255,255,255,0.06)`. Right column: fullscreen `<video autoplay muted loop playsinline preload="metadata">` at `object-fit: cover`. Hover interaction: on row `mouseenter`, GSAP timeline (1) wipes the current video out via clip-path `polygon(0 0, 100% 0, 100% 100%, 0 100%) → polygon(0 0, 0 0, 0 100%, 0 100%)` (`duration: 0.6`, `ease: CustomEase.create("depoWipe", "0.76, 0, 0.24, 1")`), (2) swaps `<video src>`, (3) wipes new video in with the reverse polygon. Row-index typography: eyebrow `N°` in italic serif, number in tabular sans, size 0.82rem — placed absolute at the row's top-right. Cursor is a custom SVG circle that scales `1 → 2.4` on hoverable elements via `mix-blend-mode: difference`. All videos preloaded to `metadata` only, first frame poster used for LQIP.

WHY IT WORKS: The film-strip metaphor is doing double work — it's a navigation *and* a brand statement. Editorial numbering (Gestalt: continuity + closure) makes 30 items feel curated instead of listy. The masked wipe respects motion semantics: content isn't fading, it's *cutting* — same language as film editing, which reinforces the domain.

GOOD: 11ty means the entire site ships at under 40kb of JS; loads instantly.
BAD: Video-heavy on mobile without a data-saver fallback; hover-only interaction excludes touch fully — needs a tap-to-preview state.

CLEAN APPLICATION: Apply this to Clean's **docs navigation** on docs.tryclean.ai by adopting the split layout — left column is a sticky index of API/SDK sections with editorial `N°01 / DISCOVERY`, `N°02 / ENRICHMENT` numbering; right column shows a live, syntax-highlighted code snippet + result preview for whichever section is hovered/focused. Use the exact clip-path wipe between snippets. This makes docs feel like a product tour, not a text dump — and reinforces "developer infrastructure" positioning.

## shadcn/typeset — https://ui.shadcn.com/docs/typeset

WHAT IT DOES: A single-CSS-file typography system released as part of shadcn/ui's July 2026 drop, built to style HTML *and* streamed markdown (chat, docs, articles) in one rhythm. Exposes three container-scoped controls — size, leading, flow — so a typeset "adapts to its container" without JS. Streaming-aware: new blocks arriving in a chat/docs stream never cause earlier blocks to re-flow their margins, borders, or fonts. Ships alongside Base UI becoming the new shadcn default primitive (over Radix) the same week.

PACKAGES:
- shadcn@4 CLI (`npx shadcn@latest add typeset`)
- Base UI (`@base-ui-components/react`) — new default primitive layer
- tailwindcss@3.4 with `@tailwindcss/postcss`
- @fontsource-variable/inter-tight (default), @fontsource-variable/geist-mono (code)
- next-mdx-remote@5 or @content-collections/mdx for docs surface
- ai@4 (Vercel AI SDK) if streaming into the typeset container

IMPLEMENTATION: Install typeset with `npx shadcn@latest add typeset`, receive a single `typeset.css` file in your project. Wrap streamed markdown / docs prose in `<div className="typeset" data-typeset-size="md" data-typeset-leading="relaxed" data-typeset-flow="narrow">`. The CSS uses container queries (`@container typeset (min-width: 640px)`) to scale — no JS, no ResizeObserver. Streaming stability comes from `contain: layout style` on each block child plus fixed `margin-block` tokens that never depend on siblings. For chat: place typeset inside the shadcn `MessageScroller` (June 2026 release) — messages stream token-by-token without triggering ancestor reflow. Configure three named typesets in the builder (docs, chat, article), export a `preset` class each maps to. For light/dark: `@media (prefers-color-scheme)` + CSS custom properties `--typeset-fg`, `--typeset-fg-muted`, `--typeset-code-bg` — all overridable per typeset. Pair with Base UI's new `Popover`, `Dialog`, `Menu` primitives (drop-in Radix replacements with better accessibility defaults).

WHY IT WORKS: Typography is the one primitive every AI product ships and most get wrong. Container-scoped controls solve the actual problem — the same prose needs different rhythm in a 380px chat bubble vs a 720px doc column vs a 1120px article. By making the container the source of truth (not global CSS), typeset kills the "prose classes fight each other" bug that plagues every AI dashboard.

GOOD: Streaming stability without JS is a genuine engineering win — no more layout jank on token arrival.
BAD: Locked to shadcn/Base UI conventions; teams on unstyled setups need a migration path.

CLEAN APPLICATION: Apply this directly to Clean's **product surface** — every place Clean streams AI-generated content (lead-list summaries in the dashboard, enrichment reports, outreach drafts, docs at docs.tryclean.ai). Install typeset, define three presets: `chat` (leading-relaxed, flow-narrow, 15px), `report` (leading-normal, flow-wide, 16px), `docs` (leading-loose, flow-wide, 17px). Migrate away from Tailwind Typography's `prose` classes — they cause the exact stream reflow bug typeset fixes. This is a same-day install and touches every Clean surface.
