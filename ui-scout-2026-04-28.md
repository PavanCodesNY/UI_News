## 1. Detroit Paris — https://www.detroit.paris/

WHAT IT DOES: A full-bleed editorial homepage for an "AI production house" built around a sticky vertical scroll-narrative. The hero is a fixed split layout: left column holds an oversized serif wordmark on a near-black canvas (#0A0908), right column streams autoplay generative video reels stacked in a 2-column masonry that drift upward at ~40% scroll speed via a `transform: translate3d` parallax. As you scroll, an SVG mask peels open over the wordmark to reveal the next section, and a horizontal text marquee runs at 60px height across section seams. Cursor is a bespoke 12px circle that scales to 64px and shows "VIEW" on any media hover via a `mix-blend-mode: difference` overlay.

PACKAGES:
- next@14.2 (App Router)
- gsap@3.12 (+ ScrollTrigger, Observer)
- lenis@1.1 (smooth scroll, replaces deprecated `@studio-freight/lenis`)
- framer-motion@11
- splitting@1.0.6 (per-character text reveal)
- @gsap/react (useGSAP hook for SSR safety)
- tailwindcss@3.4
- ABC Diatype (paid, fallback: Inter Tight)
- Editorial New (PangramPangram, fallback: PP Editorial New free trial)

IMPLEMENTATION:
1. Root layout wraps children in a `<LenisProvider>` with `lerp: 0.1, duration: 1.2, smoothTouch: false`. Sync `lenis.on('scroll', ScrollTrigger.update)` and call `gsap.ticker.add((t) => lenis.raf(t * 1000))` once.
2. Hero: CSS grid `grid-cols-[1fr_1fr]` with `min-h-svh`. Left cell has `position: sticky; top: 0; height: 100vh`. Right cell holds 6–8 `<video autoplay muted loop playsinline>` tiles in two `flex-col gap-2` columns.
3. Parallax tiles: Each column wrapped in `gsap.to(col, { yPercent: -40, ease: 'none', scrollTrigger: { trigger: hero, start: 'top top', end: 'bottom top', scrub: 1 } })`. Offset right column by `yPercent: -25` start to stagger.
4. Wordmark reveal: SVG `<text>` filled white inside `<mask id="reveal">`. Animate `<rect>` width inside the mask from 0 → 100% with ScrollTrigger `scrub: 0.6`.
5. Marquee seam: GSAP infinite tween `gsap.to('.marquee-track', { xPercent: -50, duration: 30, repeat: -1, ease: 'none' })`. Duplicate content twice in the track to avoid pop.
6. Custom cursor: fixed `pointer-events-none` div, position via `gsap.quickTo(cursor, 'x', { duration: 0.3, ease: 'power3' })`. Add `[data-cursor="view"]` attribute on media; on `mouseenter` scale to 64 and inject text node.
7. Performance: `<video>` tiles use `preload="metadata"`, IntersectionObserver lazy-attaches `src` only when within 200px of viewport; pause when offscreen.
8. Breakpoints: <768px collapses to single column, kills cursor overlay, parallax disabled (set `scrub: false`), marquee duration shortened to 18s.

WHY IT WORKS: The split sticky/scroll layout creates simultaneous hierarchies — typography as identity (left, anchored) vs. portfolio proof (right, kinetic) — so visitors register the brand and the work in one glance instead of two scrolls. The `mix-blend-mode: difference` cursor enforces a single accent color that adapts to any background, eliminating the contrast-debt that custom cursors usually accumulate. Marquees at section seams act as motion-rest beats, training the eye that scrolling will reveal more without adding cognitive load.

GOOD / BAD: Best-in-class video gating (no autoplay until in viewport, no jank at 60fps). Bad: serif wordmark drops below 56px on mobile and loses its presence — the design language doesn't scale down.

CLEAN APPLICATION: Apply this to Clean's landing page hero by building a sticky-left / scrolling-right split where the left column is a serif wordmark "Clean" with the tagline "The context layer for agents," and the right column streams a vertical reel of token-savings comparisons (raw JSON vs. TOON, side by side, cycling through 6 real MCP responses). Use the same Lenis + GSAP scrub model so the wordmark mask peels open to reveal the product CTA at scroll-50%. Replace the marquee with a live-token-count strip ("18.2M tokens saved this hour across the network") fed by a static JSON for now.

## 2. Oryzo AI — https://oryzo.ai/ (Awwwards SOTD Apr 13, by Lusion)

WHAT IT DOES: A single-page cinematic product story that treats a satirical "AI cork coaster" with full luxury-launch production. The page is a chained sequence of GSAP-pinned sections, each running a Three.js scene with the coaster as the hero mesh, and orchestrating camera dolly + mesh rotation + bloom intensity from a single ScrollTrigger timeline. Footer ends with an interactive particle field (~12k GPU points) that follows cursor with a `flowfield` shader. Color system is binary: deep brown (#100904) and high-chroma orange (#FF8539) — no neutrals, no grays.

PACKAGES:
- three@0.166
- @react-three/fiber@8.16
- @react-three/drei@9.108 (PerspectiveCamera, useGLTF, Environment, Float)
- @react-three/postprocessing@2.16 (Bloom, ChromaticAberration, Vignette)
- gsap@3.12 + ScrollTrigger
- leva@0.9 (dev-only tweaking, strip in prod)
- maath@0.10 (random point distributions for particles)
- valtio@1.13 (lightweight scene state shared between R3F and DOM)
- next@14.2

IMPLEMENTATION:
1. Single `<Canvas>` mounted at root with `dpr={[1, 1.75]}`, `gl={{ antialias: false, alpha: true }}`, `frameloop="demand"` to skip frames when nothing is animating.
2. Scenes are pinned `<section>` elements (each `100svh`) with a shared GSAP master timeline: `gsap.timeline({ scrollTrigger: { trigger: '#story', start: 'top top', end: '+=600%', scrub: 1, pin: true } })`. Each scene is a `.to()` tween that mutates a Valtio store: `{ cameraZ, modelRotY, bloomIntensity, fogDensity }`.
3. R3F components subscribe to the store with `useSnapshot` and write transforms via `useFrame`, NOT via GSAP — this keeps the React tree thin and lets R3F's reconciler batch updates.
4. Coaster mesh: 1.2MB Draco-compressed GLB, `useGLTF.preload('/coaster.glb')` in `_app`. Material is `MeshPhysicalMaterial` with `clearcoat: 1, clearcoatRoughness: 0.05, roughness: 0.6`.
5. Particle footer: `<Points>` with 12000 vertices generated by `maath/random.inSphere(new Float32Array(36000), { radius: 1.5 })`. Custom `ShaderMaterial` reads cursor from a uniform `uMouse` (vec2 NDC) and applies `position += normalize(uMouse - position.xy) * 0.04` in the vertex shader.
6. Type animation: Splitting.js per-character split, GSAP `from({ y: 100, opacity: 0, stagger: 0.015, ease: 'power4.out', duration: 1 })` triggered when section enters.
7. Audio: a 4-second whoosh at section transitions via Howler.js, gated behind a "sound on" toggle in the corner; default off.
8. Bundle budget: Three + R3F + drei tree-shaken to ~340KB gzipped using `import { Bloom } from '@react-three/postprocessing'` (no barrel), Draco decoder loaded from CDN.

WHY IT WORKS: The two-color system forces every element to compete on form, not chroma — there's nowhere for visual noise to hide, so weak ideas get cut at the design phase. Pinned-scrub storytelling is the closest the web gets to film editing: the user controls the timeline but the choreography is authored, which combines agency with craft. The particle footer rewards the user for reaching the bottom — a haptic-feeling moment that converts a passive scroll into a remembered experience.

GOOD / BAD: Best-in-class scroll choreography — every micro-frame feels intentional, no dead air. Bad: ~340KB JS + 1.2MB GLB + 8 video posters = 4.5MB initial payload; on 3G the hero is blank for 6+ seconds with no skeleton.

CLEAN APPLICATION: Apply this to Clean's product demo section by building a 4-scene scroll-pinned R3F story that visualizes an MCP request hop: (1) raw JSON cube enters from above, (2) compresses through a TOON "lens" shader, (3) emerges as a smaller geometric form labeled with token count, (4) particle field forms the Clean wordmark on exit. Use Clean's brand color as the only accent against pure black. Cap GLB assets to 400KB total — no luxury-brand bloat — and gate the canvas behind `prefers-reduced-motion`.

## 3. AUDIOLAB (R3F Audio Visualizer) — https://www.webgpu.com/showcase/audiolab-react-three-fiber-audio-visualizer/

WHAT IT DOES: A browser-based "radio" UI with a real-time 3D scene that reacts to FFT data from the currently playing track. The interface is a fixed faceplate (knobs, station readout, VU meter) rendered in DOM/CSS, and a transparent R3F canvas sits behind it rendering an instanced mesh field whose per-instance scale + emissive intensity is driven by 64 frequency bins. Switching tracks crossfades both audio and shader uniforms via a single `gsap.to(uniforms, ...)` call.

PACKAGES:
- three@0.166
- @react-three/fiber@8.16
- @react-three/drei@9.108 (Instances, Instance, useDetectGPU)
- @react-three/postprocessing@2.16 (Bloom only)
- tone@15 (audio analyser node with FFT)
- gsap@3.12
- zustand@4.5 (audio + scene state)
- vite@5

IMPLEMENTATION:
1. Audio graph: `Tone.Player` → `Tone.FFT(64)` → `Tone.Destination`. The FFT node is read every frame inside `useFrame((state) => { fftData = analyser.getValue(); ... })`.
2. Instanced mesh: `<Instances limit={4096}>` with a shared `BoxGeometry(0.05, 1, 0.05)`. Place 4096 instances on a 64x64 grid; on each frame, set per-instance `scale.y = 1 + fftData[i % 64] * 8` and emissive HSL based on bin index.
3. Rendering: `useFrame` writes `instancedMesh.instanceMatrix.setUsage(THREE.DynamicDrawUsage)` once, then mutates the matrix array directly for max throughput.
4. Faceplate UI: pure CSS Grid `grid-template-areas`. Knobs are `<input type="range">` styled with conic-gradient backgrounds, rotation via `transform: rotate(calc(var(--val) * 270deg - 135deg))`.
5. GPU fallback: `useDetectGPU()` from drei; tier ≤ 1 swaps the instanced field for a simple `<Plane>` with a 2D shader bar visualizer.
6. Crossfade: `gsap.to(uMix, { value: 1, duration: 1.2, ease: 'power2.inOut' })` blends two FFT buffers in the shader during track switch — no audio click, no visual jump.
7. State boundary: Zustand store holds `{ track, playing, volume, bpm }`. Both DOM and R3F subscribe; only R3F runs at 60Hz, DOM components use selector subscriptions to avoid re-renders on FFT updates.
8. Performance budget: ~6ms/frame on M1, capped at 60fps via `frameloop="always"` only while playing; switch to `"demand"` when paused.

WHY IT WORKS: The DOM-over-canvas split lets each layer do what it's best at — DOM for crisp UI affordances and a11y, WebGL for high-frame visuals — without the common mistake of trying to render UI inside R3F. Instanced rendering at 4096 elements feels alive without the GPU cost of 4096 draw calls. Crossfading shader uniforms instead of swapping scenes preserves spatial continuity, which is the same trick film editors use to hide hard cuts.

GOOD / BAD: Best-in-class audio↔visual coupling — frequencies map intuitively, the eye believes the sound. Bad: requires user gesture to start audio (browser policy) but the empty canvas before that interaction looks broken — needs an explicit "press play" affordance that the current design buries.

CLEAN APPLICATION: Apply this to Clean's dashboard analytics view by replacing audio-FFT with live MCP-call telemetry. A 32x32 instanced grid where each cell represents a token-bucket per minute; cell height = tokens consumed, cell hue = % savings via TOON. The faceplate UI on the right becomes filter chips (per-tool, per-model, per-tenant) styled identically to the radio knobs. Use the same Zustand-with-selector boundary so the chart updates at 60Hz without re-rendering the chrome.

## 4. Wild Week Athens — https://wild-week-athens.framer.website/

WHAT IT DOES: A festival microsite built end-to-end in Framer in 1.5 weeks using AI-generated imagery. The page uses a chunky modular grid (3-col desktop, 1-col mobile) where each module is its own self-contained interaction: a draggable schedule cube, a sticky sponsor ribbon, a WebGL noise-displacement hero image, and a "shuffle" filter button that re-orders speaker cards via FLIP layout animation. Type system is one face (PP Neue Montreal) at three sizes — no italics, no weights below 500.

PACKAGES:
- framer (no-code, but compiled output uses):
- framer-motion@11 (under the hood)
- @react-three/fiber@8 (WebGL hero only)
- glsl-noise (3D simplex noise displacement)
- date-fns@3 (schedule logic)
- vercel-edge-config (filter state)

IMPLEMENTATION (for a hand-coded port, since Framer's output isn't directly forkable):
1. Grid: CSS `grid-template-columns: repeat(12, 1fr); gap: 8px`. Modules span `col-span-{4,8,12}` with explicit row placement to lock layout integrity. Mobile collapses to `grid-cols-1` with manually re-ordered modules via `order: var(--m-order)`.
2. Hero displacement: `<Plane args={[16, 9]}>` with custom `ShaderMaterial`. Vertex shader displaces along normal: `pos.z += snoise(vec3(uv * 3.0, uTime * 0.3)) * 0.4`. Fragment samples a 1920x1080 AI-generated still texture. `useFrame` increments `uTime`.
3. Draggable schedule cube: `<motion.div drag dragConstraints={ref} dragElastic={0.2} whileDrag={{ scale: 1.05, cursor: 'grabbing' }}>`. Six faces use `transform: rotateY(...) translateZ(150px)`, parent has `transform-style: preserve-3d`.
4. Speaker filter: array of speaker objects in state. On filter change, `<AnimatePresence mode="popLayout">` wrapping a `.map()` of `<motion.div layout layoutId={s.id}>` triggers FLIP — Motion automatically interpolates from old to new grid positions over 400ms `ease: [0.32, 0.72, 0, 1]`.
5. Sponsor ribbon: `position: sticky; top: 0` with a horizontal `display: flex` of logos at `height: 48px`. On scroll past 80% page, ribbon slides up with `translateY(-100%)` and hands off to a footer CTA.
6. Color: 3-color system — bone (#F4F0E6), ink (#0E0E0E), highlight (#E63B17). Used in 70/20/10 ratio across all modules.
7. AI-asset workflow: every hero image generated in Midjourney v7, run through a Photoshop bichrome action to enforce the 3-color palette, then served as AVIF (~80KB each) with WebP fallback.

WHY IT WORKS: Modularity is the design — each block is internally rich enough to be its own page, so the homepage doubles as a sitemap. The single-typeface, single-weight constraint is the discipline that makes the maximalist module density feel intentional instead of chaotic; without that constraint the page would read as cluttered. AI assets are bichromed so they obey the palette — the most common AI-design failure (mismatched generated stills) is solved at the post-processing layer, not the prompt layer.

GOOD / BAD: Best-in-class proof that "vibe-built" sites can win awards if the constraints are enforced. Bad: shipping a draggable cube as a primary nav element is a usability tax — most users won't discover the schedule because the affordance ("drag me") isn't visible until hover, and there's no fallback list view above the fold.

CLEAN APPLICATION: Apply this to Clean's docs navigation by adopting the modular 12-col grid as the docs landing page (replacing the typical sidebar+content layout). Each "topic" (MCP setup, TOON format, telemetry, billing) becomes a span-4 module with its own micro-interaction: a live token counter, a copy-paste install snippet that animates on hover, an embedded schema visualizer, etc. Enforce one typeface (Inter or JetBrains Mono) at three sizes only. Skip the AI-image bichrome workflow — Clean's brand is monochrome anyway.

## 5. Silent House — https://silent-house.com/ (Awwwards SOTD Apr 28)

WHAT IT DOES: A studio site for "three companies under one roof" that uses a tri-pane horizontal scroll on desktop — three vertical panels representing each sub-brand, each scrollable independently. A bottom-fixed scrubber lets you drag a time-marker that synchronizes scroll position across all three panels (so you can read "what each company is doing in 2024" simultaneously). On mobile, panels collapse into a swipeable carousel with hard snap points and a progress dot trio.

PACKAGES:
- astro@4.10 (zero-JS by default, islands for interactive parts)
- gsap@3.12 (+ ScrollTrigger, Draggable, Observer)
- lenis@1.1
- three@0.166 (single fluid background only, not in main bundle)
- @studio-freight/hamo (resize observer, debounced)
- vue@3.4 (interactive islands)
- satori (OG image generation at build)
- view-transitions polyfill

IMPLEMENTATION:
1. Layout: `<main>` is `display: grid; grid-template-columns: repeat(3, 100vw); height: 100vh; overflow: hidden`. Each `<section>` is its own scrollable y-axis container with `overflow-y: scroll; scroll-snap-type: y mandatory`.
2. Sync scrubber: bottom-fixed `<input type="range">` styled to span the viewport. On `input`, dispatch `panel.scrollTo({ top: pct * panel.scrollHeight })` on all three panels in `requestAnimationFrame`.
3. Independent vs. synced mode: a toggle in the corner. When synced, listen on the focused panel's `scroll` and propagate via Observer; when independent, panels scroll alone and the scrubber tracks only the focused one.
4. Background: a single `<canvas>` behind all panels runs a 2D Perlin-noise fluid sim (no Three.js — just `<canvas>` + `imageData` writes at 30fps). Color shifts based on which panel is in focus via `IntersectionObserver`.
5. Mobile: media query `@media (max-width: 900px)` rewrites grid to `grid-template-columns: 100%` and adds `scroll-snap-type: x mandatory` on the parent, with each panel becoming `min-width: 100vw; scroll-snap-align: start`.
6. View transitions: panel switches use the native `document.startViewTransition()` API for ~200ms crossfade; Astro's view-transitions polyfill backfills Firefox/Safari.
7. Type loading: `font-display: swap` on a single 18KB woff2 (Söhne or fallback Inter), preloaded in `<head>` with `as="font" crossorigin`.
8. Performance: Lighthouse mobile 96/100/100/100. Total JS shipped on first load = 24KB gzipped because Astro static-renders 95% of the page.

WHY IT WORKS: A tri-pane sync is the rarest navigation pattern on the web, which means the affordance itself is the brand statement — you instantly understand "three companies, one practice" without reading copy. The independent/synced toggle respects that some users want the comparison, others want focus — that's an explicit accessibility gesture, not just a power-user toggle. Astro + 24KB JS budget is the technical proof that "fast" and "experimental" aren't contradictions.

GOOD / BAD: Best-in-class JS economy — most experimental sites ship 800KB+ to do less. Bad: tri-pane scroll has zero discoverability until you drag the scrubber; first-time visitors often think the page is broken because vertical scroll on the body does nothing.

CLEAN APPLICATION: Apply this to Clean's pricing page by building a tri-pane layout for the three customer segments — "Indie devs," "Startups," "Enterprise" — each with its own independently scrollable feature column and a bottom-fixed comparison scrubber that syncs them when toggled. Use Astro for the entire marketing site to keep first-load JS under 30KB. The single-canvas Perlin background can render Clean's brand pattern (or stay transparent — TBD with design).

## BUILD DIRECTIVE

TARGET: Clean landing page hero + immediate-below product proof section (single continuous scroll, no separate page). The current hero is the highest-leverage surface — first 10 seconds determine bounce.

APPROACH: Build a sticky-left / kinetic-right split hero (Detroit Paris pattern) where the left pane locks a serif Clean wordmark with a one-line value prop ("The context layer for agents") and a single primary CTA, while the right pane scrolls a 2-column masonry of live "before/after" cards — raw MCP responses on the left of each card, TOON-compressed on the right, with token counts and a bright savings badge. As the user scrolls, the wordmark mask peels open (GSAP scrub) to reveal a 4-scene R3F pinned story (Oryzo AI pattern, but with Clean's monochrome palette and a 400KB asset cap) that animates a single MCP request through the Clean proxy: raw JSON → TOON lens → smaller geometric form → particle wordmark on exit. End with a tri-pane (Silent House pattern) "for indie / for startup / for enterprise" comparison block that shares one bottom-fixed scrubber. Smooth scroll via Lenis throughout, keep total first-paint JS under 180KB gzipped, ship a static `prefers-reduced-motion` fallback that swaps every animation for an immediate state.

STACK:
- next@14.2 (App Router) — or astro@4.10 if SEO and JS budget take priority over interactive density
- tailwindcss@3.4
- gsap@3.12 + ScrollTrigger + Observer + @gsap/react
- lenis@1.1
- three@0.166 + @react-three/fiber@8.16 + @react-three/drei@9.108 + @react-three/postprocessing@2.16
- framer-motion@11 (FLIP animations for the masonry cards only)
- splitting@1.0.6
- zustand@4.5 (UI state)
- valtio@1.13 (R3F scene state shared with DOM)
- maath@0.10 (particle distributions)
- Inter Tight (sans) + PP Editorial New trial (display serif)
- next/font for self-hosted font loading
- AVIF posters for any video tile, WebP fallback

REFERENCE URLS:
- https://www.detroit.paris/ — split-sticky hero structure, custom cursor, marquee seam
- https://oryzo.ai/ — pinned R3F scroll story, two-color discipline, particle footer
- https://silent-house.com/ — tri-pane with sync scrubber, JS economy via Astro

KEY CONSTRAINTS:
- Clean is developer infra. Avoid: glassmorphism, generic gradient blobs, Stripe-knockoff section dividers, "AI-feel" iridescent gradients, any motion that doesn't carry information.
- Every animation must encode meaning (a payload compressing, a request hopping, a savings counter ticking). No motion-as-decoration.
- Monochrome base (true black #000 + warm off-white #FAFAF7) with exactly ONE accent color. No secondary palette.
- Type: one display serif (used only in the hero wordmark) + one neutral sans (everything else). Two faces total.
- First paint JS budget: 180KB gzipped. Hero must be interactive within 1.2s on a throttled 4G + Moto G4 simulation.
- The R3F canvas must lazy-mount only after the user begins scrolling past the hero. Before that, the canvas slot holds an SVG poster.
- Accessibility: every scroll-driven animation has a `prefers-reduced-motion` branch that snaps to the end state. Custom cursor disabled at <1024px and behind reduced-motion.
- No autoplay audio. No hijacked back-button. No scroll-jacking longer than 600vh.
- Ship the comparison cards section even if the R3F story slips — it carries the pricing argument and is the highest-converting block.
