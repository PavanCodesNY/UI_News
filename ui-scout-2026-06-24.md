## Sui — https://www.sui.io/

WHAT IT DOES: Layer-1 blockchain marketing site (Awwwards SOTD, June 24, 2026, Developer Award). A full-bleed WebGL hero renders an animated, refractive "object-graph" mesh — a node-graph of glowing primitives connected by bezier light trails — driven by GLSL fragment shaders with simplex-noise displacement and a chromatic-aberration post pass. The page is a vertical scroll-stage where each section binds a normalized scroll progress (0→1) into shader uniforms and into GSAP tweens that morph the camera dolly, swap material palettes, and stage SplitText reveal lines. Layout system: 12-col grid with editorial pull-quotes that break the grid, a sticky top nav that swaps from glass-blur to solid on scroll, and oversized monospaced numerals as the section-counter motif.

PACKAGES:
- next@15 (App Router, RSC for content sections, client components for canvas)
- three@0.169 + @react-three/fiber@9 + @react-three/drei@9.115 + @react-three/postprocessing@2.16 (Bloom + ChromaticAberration + Vignette)
- glslify@7 for shader includes
- gsap@3.13 + @gsap/react@2 (ScrollTrigger, SplitText, Observer)
- lenis@1.1 (smooth scroll, integrated with ScrollTrigger via `lenis.on('scroll', ScrollTrigger.update)`)
- tailwindcss@4 + class-variance-authority@0.7 + clsx@2
- next/font with `JetBrains Mono` (numerals/metadata) + `Inter Tight` (display) + a serif accent (`Fraunces` 144pt for editorial pulls)
- framer-motion@11 (only for nav + modal micro-state; canvas owns the rest)
- lucide-react@0.453

IMPLEMENTATION:
1. Root layout: `<Lenis root />` wrapping `<Canvas eventSource={document.body} eventPrefix="client" />` fixed at `inset-0 z-[-1]`. DOM scrolls normally; the canvas reads `window.scrollY` and exposes a `useScroll()` (drei) `offset`.
2. Hero shader: a `<mesh>` with `IcosahedronGeometry(2, 64)` and a `shaderMaterial` (drei `shaderMaterial` helper). Vertex shader: `position += normal * noise(position * 1.4 + uTime * 0.18) * uDistort` where `uDistort` lerps 0.15 → 0.55 across hero scroll. Fragment shader: a 3-stop gradient (`#0B0F19` → `#5B8DEF` → `#A3D8FF`) sampled by fresnel `pow(1.0 - dot(N, V), 2.0)`, additive light-trails via raymarched SDF lines connecting fixed node positions.
3. Post: `<EffectComposer>` → `Bloom(intensity=0.85, luminanceThreshold=0.6, mipmapBlur=true)` → `ChromaticAberration(offset=[0.0012, 0.0012])` → `Vignette(eskil=false, offset=0.3, darkness=0.6)`. Disable mipmapBlur on `dpr<1.5` mobile.
4. Sections: each `<section>` is `h-[100svh]`. `ScrollTrigger.create({ trigger, start: 'top top', end: 'bottom top', onUpdate: self => uniforms.uProgress.value = self.progress })`. Use `gsap.context()` per section for safe cleanup.
5. Type stage: SplitText into chars, `gsap.from(chars, { yPercent: 110, rotateX: -40, stagger: { each: 0.018, from: 'start' }, duration: 0.9, ease: 'expo.out' })` triggered at `start: 'top 80%'`.
6. Nav swap: `ScrollTrigger.create({ trigger: hero, start: 'bottom top', toggleClass: { targets: nav, className: 'is-solid' } })`. CSS: `.is-solid { backdrop-filter: none; background: rgb(11 15 25 / .92); border-bottom: 1px solid rgb(255 255 255 / .06) }`.
7. Responsive: mobile (<768px) replaces canvas with a static AVIF poster + a CSS `@property --grad` keyframed gradient to preserve perceived motion at 0 GPU cost.
8. Performance: `frameloop="demand"` on `<Canvas>`, manual `invalidate()` on scroll; `dpr={[1, 1.75]}`; geometry merged via `useMemo`; AVIF posters preloaded with `<link rel="preload" as="image" type="image/avif">`.

WHY IT WORKS: The shader-driven object-graph is a literal visualization of Sui's object-centric data model — motion is semantic, not decorative. High contrast between the dark navy field and the cyan rim-light creates a single, unambiguous focal anchor in every scroll frame (Gestalt figure-ground). Monospaced numerals + serif pulls + sans body sets a three-voice typographic hierarchy that reads as "engineering meets editorial."

GOOD / BAD:
- Good: shader uniforms reused across sections — one canvas, zero re-mounts. Mobile fallback is a real image, not a degraded canvas.
- Bad: hero canvas mounts before fonts swap, so the FOIT shows for ~120ms on slow connections; could be solved by gating canvas on `document.fonts.ready`.

CLEAN APPLICATION: Apply this to Clean's **landing page hero** by replacing the marketing copy hero with a single full-bleed R3F canvas rendering a live, animated "context graph" — nodes are MCP servers (GitHub, Linear, Slack, Postgres, etc.) connected by light-trail edges to a central "Clean" node that emits a TOON-format pulse. Scroll-bind two uniforms: `uCompressionRatio` (0 → 1, drives a literal "size" shrink animation of token particles flowing along edges, visualizing TOON's 60% token reduction) and `uActivePill` which highlights one MCP node per scroll section as we explain each integration. Keep nav glass-blur → solid swap. Use JetBrains Mono for all numerics (token counts, ms latency) and Inter Tight for display. No marketing photos anywhere on the page.

## EverSwap (by Lusion) — https://www.awwwards.com/sites/everswap

WHAT IT DOES: Token-swap product landing page (Awwwards SOTD June 22, 2026; 7.58/10). Hero is a real-time WebGL "liquid metal" interactive — two chrome volumes morph and merge under cursor proximity, driven by raymarched SDF smoothmin in a fullscreen fragment shader, with cubemap-based environment reflection. Scroll-jacked horizontal section reveals product features as a filmstrip; each panel uses a video sprite-sheet (encoded as one `webm` with per-frame UV offsets) decoded in a custom shader to avoid `<video>` element cost. Cursor leaves a 4-tap motion-blur trail composited as a post-effect. Layout is single-column, 1440px max width, with aggressive negative space (60vh of breathing room between sections).

PACKAGES:
- vite@6 + react@19 + typescript@5.6
- three@0.169 + @react-three/fiber@9 + @react-three/drei@9.115
- ogl@1.0 (lightweight WebGL for the cursor-trail FBO ping-pong; Lusion's house pattern)
- glslify@7 + glsl-noise@0.0.0 + glsl-easings@1.0.0
- gsap@3.13 (ScrollTrigger, ScrollSmoother, Observer for horizontal scrubbing)
- @studio-freight/hamo@0.6 (resize/raf hooks)
- howler@2.2 (subtle UI sound on swap-cta hover)
- satori@0.11 + sharp@0.33 for OG image generation

IMPLEMENTATION:
1. SDF smoothmin shader: `float smin(float a, float b, float k) { float h = clamp(0.5 + 0.5*(b-a)/k, 0.0, 1.0); return mix(b, a, h) - k*h*(1.0-h); }`. Two spheres with positions lerped toward cursor (mapped to NDC, smoothed via critically damped spring `omega=12, zeta=1.0`, dt-aware).
2. Reflection: `samplerCube uEnvMap` loaded from a 1k HDR via `RGBELoader` + `PMREMGenerator`. Material samples `textureCube(uEnvMap, reflect(-V, N))` and multiplies by a fresnel-modulated tint `mix(#C9D4E5, #E8F0FF, fresnel)`.
3. Cursor-trail: an OGL FBO at 0.5× viewport. Each frame: bind FBO A, draw previous FBO B faded by `0.92`, draw a soft additive circle at cursor, swap A↔B. Final pass samples FBO and adds to scene with `screen` blend.
4. Horizontal scrub: `ScrollSmoother` + a wrapper `position: sticky; height: 100vh` and an inner `display:flex; width: ${panels * 100}vw` translated by `gsap.to(inner, { x: -((panels-1)*100)+'vw', scrollTrigger: { trigger: section, start: 'top top', end: '+=300%', scrub: 1, pin: true } })`.
5. Sprite-sheet video: encode 60-frame loop as a 6×10 atlas webm; in shader, `vec2 uv = uvBase * vec2(1.0/6.0, 1.0/10.0) + cellOffset(currentFrame)`. `currentFrame = floor(uTime * 24.0) % 60.0`.
6. Type: Söhne Mono for tickers, GT Sectra for the editorial hero phrase ("Move value at the speed of light"), Söhne for body. Font sizes follow a perfect-fifth scale (16 / 24 / 36 / 54 / 81).
7. Color: zero saturated colors. Palette is `#0A0A0B`, `#F4F4F1`, `#C9D4E5` (steel), single accent `#9DFF52` reserved only for the swap CTA.

WHY IT WORKS: Direct manipulation feedback — the chrome merges as the cursor moves between the two volumes — is a literal embodiment of "swap." It teaches the product mechanic before any copy is read. The restrained palette + 60vh whitespace + reserved-accent-color rule create a luxury-product reading rhythm; the accent green becomes the visual primitive for "action."

GOOD / BAD:
- Good: cursor-state IS the value prop. Sprite-sheet video bypasses `<video>` decode jank entirely.
- Bad: SDF raymarch at full DPR cooks integrated GPUs; thermal throttles after ~90s. No reduced-motion fallback declared; misses `prefers-reduced-motion`.

CLEAN APPLICATION: Apply this to Clean's **"How it works" product demo section** below the hero by building a single horizontally-pinned scrub stage with 4 panels: (1) raw MCP tool JSON enters left, (2) Clean proxy compresses to TOON in the middle as two volumes merge under cursor, (3) compressed payload exits right, (4) live token-count diff (-62%) animates in monospace. Use the SDF smoothmin merge as a literal metaphor for "MCP request meets Clean." Reserve a single Clean accent color (use the brand green) exclusively for the "Compress" pill — never anywhere else on the page. Keep everything else neutral steel.

## Fauna Robotics — https://www.awwwards.com/sites/fauna-robotics

WHAT IT DOES: Humanoid robotics company landing page (Awwwards SOTD June 16, 2026, Developer Award, 7.39/10). Built in Webflow. Hero is a stack of three short-loop product videos (no autoplay-and-pray; preloaded as `<video preload="auto" muted playsInline>` and orchestrated by IntersectionObserver), framed by a 24px white border that creates an "appliance bezel" reading. Page is single-column, asymmetric — text columns set at 5/12 grid width, video tiles set at 7/12, alternating sides per scroll section. Color palette is exactly 2 colors (#EFEFEF + #D1E3FF); the warmth and approachability come entirely from the photography and motion, not from chromatics.

PACKAGES:
- webflow (host) — or to replicate in code: next@15 + tailwindcss@4
- gsap@3.13 (ScrollTrigger only — no smooth scroll, native scroll preserved for accessibility)
- @vimeo/player or self-hosted HLS via hls.js@1.5 (for 4k tile reels)
- next/font with `Söhne` (display, 48–96pt) + `Söhne Mono` (microcopy / spec sheets)
- lucide-react for chevrons only
- No 3D, no shaders, no canvas

IMPLEMENTATION:
1. Bezel: each video tile is `<div class="p-6 bg-white rounded-[28px] shadow-[0_2px_0_rgba(0,0,0,0.04),0_24px_60px_-30px_rgba(0,0,0,0.12)]">` wrapping `<video class="rounded-[12px] w-full aspect-video object-cover">`. The bezel reads as "consumer appliance" — a deliberate counter-signal to industrial robotics tropes.
2. Video orchestration: `useIntersection(ref, { threshold: 0.4 })` → `video.play()` on enter, `video.pause()` + `video.currentTime = 0` on exit. Loop locally, never network-buffer mid-scroll.
3. Type scale: hero is 96pt Söhne at `tracking-[-0.04em] leading-[0.92]`. Body is 18pt at `leading-[1.55]`. Spec callouts are 12pt Söhne Mono uppercase at `tracking-[0.18em]`.
4. Grid breaks: text columns are `col-span-5 col-start-2` on even sections, `col-span-5 col-start-7` on odd. Video columns mirror. Alternation creates page rhythm without animation.
5. Motion: ONLY two animations — (a) on section enter, the bezel scales from `0.96` to `1.0` over `0.8s ease-out-expo`; (b) text reveals via CSS `clip-path` rolling up `inset(100% 0 0 0)` → `inset(0 0 0 0)` over `0.6s cubic-bezier(0.22, 1, 0.36, 1)`. No SplitText, no parallax.
6. Sound: muted by default. A small "Unmute" toggle in the corner (Söhne Mono 11pt) reveals ambient sound — never auto-plays audio.

WHY IT WORKS: The restraint IS the message. Robotics sites default to neon-on-black "future-tech" — this one looks like a domestic appliance ad from a Japanese homeware magazine. The 24px white bezel is a single design primitive doing huge semiotic work: it codes "safe for the living room." Asymmetric grid + zero parallax keeps the reader's eye on the videos (the actual proof) instead of the chrome.

GOOD / BAD:
- Good: chooses a single, repeated formal device (the bezel) and lets it carry the brand. Type/spacing discipline is exceptional.
- Bad: 4k tile reels on every section pushes mobile data usage past 30MB without a "data saver" mode. Lazy-load thresholds could be more aggressive.

CLEAN APPLICATION: Apply this to Clean's **product showcase section** (below hero, above pricing) by replacing the current generic feature cards with a three-section asymmetric grid where each section pairs a 5/12-column copy block with a 7/12-column "appliance-bezeled" terminal recording — actual asciinema captures of `claude` running with vs without Clean's MCP proxy. Bezel each terminal recording in a 24px white frame with the same shadow profile. Constrain the palette to two colors only (use Clean's existing neutral + a single soft accent for highlighted log lines). Animate ONLY the clip-path text reveal and the bezel scale-in — no parallax, no canvas, no GSAP timeline. The discipline signals product confidence.

## Indigo Laboratory — https://www.awwwards.com/sites/indigo-laboratory

WHAT IT DOES: Music/fashion experimental single-page site (Awwwards SOTD June 18, 2026, Developer Award; 7.33/10). Built on Nuxt 3 + Curtains.js (WebGL plane-based image distortion) + GSAP. Every image on the page is converted into a WebGL plane and distorted by an audio-reactive vertex shader — the page literally pulses to a background track sampled via the WebAudio API's `AnalyserNode.getByteFrequencyData()`. Single hard-cut between two color zones (#6A15FF deep violet → #F4F3EB warm bone) defines the page's visual halves. Navigation is a custom cursor + a 4-letter section sigil that morphs on hover.

PACKAGES:
- nuxt@3.13 + vue@3.5
- curtainsjs@8.1 (WebGL plane rendering with HTML-element sync)
- gsap@3.13 (CustomEase, MorphSVG for the sigil)
- howler@2.2 + standardized-audio-context@25 (cross-browser WebAudio)
- @vueuse/core@11 (useRafFn, useMouse, useEventListener)
- splitting@1.0.6 (poor-man's SplitText alternative, free)
- locomotive-scroll@5 (smooth scroll integrated with Curtains)

IMPLEMENTATION:
1. Audio analyser: `const ctx = new AudioContext(); const analyser = ctx.createAnalyser(); analyser.fftSize = 256; source.connect(analyser).connect(ctx.destination);`. Per frame: `analyser.getByteFrequencyData(dataArray); const bass = avg(dataArray.slice(0, 8)) / 255;`. Pass `bass` as uniform `uAudio` to every plane shader.
2. Curtains plane vertex shader: `position.z += sin(position.x * 4.0 + uTime) * uAudio * 0.4; position.xy += vec2(noise(position.xy * 2.0 + uTime * 0.3)) * uAudio * 0.1;`. Fragment: standard texture sample with a duotone post: `vec3 col = mix(vec3(0.42, 0.08, 1.0), vec3(0.96, 0.95, 0.92), luma(texColor));`.
3. Color hard-cut: page is two stacked sections with no transition between — section A is `bg-[#6A15FF] text-[#F4F3EB]`, section B inverts. The cut happens at exactly 50vh of the second section. Use `mix-blend-mode: difference` on a centered marquee that crosses the cut to make a single element render in both palettes simultaneously.
4. Custom cursor: a `position: fixed` `<div>` with `pointer-events: none`, animated via `gsap.quickTo(el, 'x', { duration: 0.18, ease: 'power3' })`. On hover of any `[data-cursor="play"]`, the cursor swaps to a 64px circle with a centered "PLAY ▶" sigil that morphs from `▶` to `⏸` via MorphSVG.
5. Section sigil: 4 fixed letters in the corner (e.g. `ALBM`) that scramble through random glyphs on hover using a 30ms interval, then settle on the next section name — a tiny tribute to the Ling glitch-text pattern.
6. Type: Editorial Old serif at 240pt for the band name, GT Cinetype Mono for labels. Justified body text at 16pt with `text-wrap: balance`.
7. Audio gating: site loads muted, with a single full-bleed "ENTER" gate (a single button) that user must click to satisfy autoplay policy and start the audio analyser.

WHY IT WORKS: The audio reactivity is not decoration — the entire page IS the audio waveform. Visual-musical synchrony is the highest-value motion semantic possible for a music brand: every pulse you see corresponds to something you can hear, so the page feels alive instead of "animated." The two-color hard cut is a brutalist editorial device that reads as confidence — most sites default to gradients to soften transitions; this one weaponizes the cut.

GOOD / BAD:
- Good: WebAudio analyser hooked into shader uniforms is a real, hard-to-fake craft signal. Single-button audio gate respects autoplay rules.
- Bad: keyboard navigation around the custom cursor is broken on Firefox; SR users get no announce when sections change. Inaccessible by default.

CLEAN APPLICATION: Apply this to Clean's **docs landing / changelog page** (not the marketing home) by using Curtains.js to render code-block screenshots as WebGL planes that "breathe" subtly with a procedural sine wave (no audio — replace `uAudio` with a slow `sin(uTime * 0.4)` modulated by scroll velocity). Keep the two-color hard-cut idea: top half of changelog is Clean's brand-neutral, bottom half (showing what's deprecated/legacy MCP behavior) hard-cuts to a muted purple-grey to visually mark "old world / new world." Reserve the audio-reactivity itself for an Easter egg: if `localStorage.cleanDevMode === '1'`, hook up an analyser to GitHub release-PR webhook event stream (mocked as a steady ping) and let the docs pulse with deploy frequency.

## Huly — https://huly.io

WHAT IT DOES: Open-source all-in-one team workspace landing page. Uses Spline-exported 3D scenes embedded on a marketing page to show app UI floating in mid-air, rotating, and parallax-reacting to scroll and cursor. The 3D scenes are rendered by Spline's runtime (`@splinetool/runtime`) in a single `<canvas>`, not Three.js directly. The 3D is purely product surface — a stylized rotating macOS-window mockup of the actual product UI — so the visitor sees the app, not a metaphor. Single-column scroll with anchored sections; each 3D model is a separate Spline scene loaded on viewport-enter and unloaded on viewport-exit to keep GPU budget bounded.

PACKAGES:
- next@15 + react@19
- @splinetool/runtime@1.9 + @splinetool/react-spline@4.0
- framer-motion@11 (section enter/exit + spring-based parallax)
- tailwindcss@4 + tailwind-merge@2 + clsx@2
- next/font with `Inter` Variable
- usehooks-ts@3 (useIntersectionObserver)
- @vercel/og for OG images
- lottie-react@2.4 for icon flourishes

IMPLEMENTATION:
1. Spline embed: `import Spline from '@splinetool/react-spline'; <Spline scene="https://prod.spline.design/[id]/scene.splinecode" onLoad={onSplineLoad} />`. Wrap each `<Spline>` in `<div className="aspect-[4/3]" />` with a skeleton blur-up while loading.
2. Lazy load: a `<SplineLazy>` wrapper uses `useIntersectionObserver({ threshold: 0.1 })` → only mounts the Spline component when within 50vh of viewport. Unmount and `splineRef.current?.dispose()` when more than 100vh past viewport.
3. Cursor parallax: `splineRef.current.setVariable('mouseX', e.clientX / window.innerWidth - 0.5)`; in the Spline scene, that variable drives a `rotation.y` on the parent group. Same for `mouseY` → `rotation.x`. Apply a critically-damped spring inside the scene's State machine.
4. Scroll-bound rotation: `useScroll({ target: ref, offset: ['start end', 'end start'] })` → `useTransform(scrollYProgress, [0, 1], [-15, 15])` → `splineRef.current.setVariable('scrollRot', value)`.
5. Performance budget: cap at one mounted Spline scene at any time. Use `<canvas>` `preserveDrawingBuffer: false`, `powerPreference: 'low-power'` for mobile.
6. Type/grid: max-w-[1240px] container, 8pt baseline grid, Inter Variable at `font-feature-settings: 'cv11', 'ss01'` for the geometric numeral set.

WHY IT WORKS: The 3D shows the product, not an abstraction. Most "3D landing pages" decorate; Huly's 3D earns its budget by being a literal interactive screenshot. Lazy load + unmount keeps the page below 16ms-per-frame even with multiple Spline scenes total. The single-column, generously spaced layout means scroll cadence is metronomic, which makes the spring-based 3D parallax feel like response rather than randomness.

GOOD / BAD:
- Good: Spline as deployment vehicle for product surface is a smart choice — designers can iterate without engineering involvement. Disciplined unmounting.
- Bad: Spline runtime is ~600kb gz; on a marketing page that loads 4 scenes, that's a Lighthouse Perf hit no matter the lazy loading. Defer-load is a band-aid.

CLEAN APPLICATION: Apply this to Clean's **integrations section** (the "Works with your stack" surface) by building a Spline scene of the Clean proxy as a glass cube with MCP server logos (Notion, Linear, GitHub, Postgres) orbiting it on independent axes. Wire `cursorX/cursorY` to subtly tumble the cube; bind `scrollYProgress` of the section to swap which MCP logo "docks" into the cube face. Lazy-mount-and-dispose per the Huly pattern — one Spline scene live at a time, max. Use this ONLY here — not in the hero — because the hero already carries the shader graph from the Sui pattern. The integrations section is where Spline's "product mockup floating in space" reading earns its bytes.

## BUILD DIRECTIVE

TARGET: Clean's **landing page hero** (tryclean.ai homepage above the fold + first 2 scroll sections).

APPROACH: Build a single full-bleed R3F canvas as the hero background that renders an animated MCP context-graph — a central "Clean" node (small luminous sphere, IcosahedronGeometry, displaced by simplex noise) connected via raymarched SDF light-trails to 6 orbiting MCP-server nodes (GitHub, Linear, Notion, Postgres, Slack, Stripe). On page load, animate token-particles flowing INTO the central node as bulky JSON cubes, then OUT as 60%-smaller TOON tetrahedrons — the literal mechanic of Clean's compression layer rendered in real time. Bind `window.scrollY` via Lenis to a single `uProgress` uniform that (a) drifts the camera dolly inward, (b) cycles which MCP node is "active" (highlighted by fresnel rim-light), and (c) drives a SplitText reveal of headline copy. Constrain the palette to dark navy + Clean's brand accent + one cyan rim-light — no marketing photos, no stock illustration, no emoji. Below the canvas hero, place two stacked sections: section 2 is the horizontal-pinned SDF-smoothmin "compression demo" (EverSwap pattern, four panels showing JSON → TOON merge), section 3 is the Spline-embedded integrations cube (Huly pattern).

STACK:
- next@15 (App Router) + react@19 + typescript@5.6
- three@0.169 + @react-three/fiber@9 + @react-three/drei@9.115 + @react-three/postprocessing@2.16
- glslify@7 + glsl-noise@0.0.0
- gsap@3.13 + @gsap/react@2 (ScrollTrigger, SplitText, ScrollSmoother)
- lenis@1.1
- tailwindcss@4 + class-variance-authority@0.7 + clsx@2 + tailwind-merge@2
- @splinetool/react-spline@4.0 (integrations section only)
- next/font: Inter Tight (display), JetBrains Mono (numerics), Fraunces 144pt (one editorial pull-quote)
- lucide-react@0.453

REFERENCE URLS:
- https://www.sui.io/ — for the shader-driven node-graph hero pattern + nav glass-swap
- https://www.awwwards.com/sites/everswap — for the horizontal-pinned SDF-smoothmin compression-demo section
- https://huly.io — for the disciplined lazy-mount-and-dispose Spline integrations cube

KEY CONSTRAINTS:
- Clean is developer infrastructure, not an AI consumer toy. Avoid: rainbow gradients, glassmorphism for its own sake, hand-drawn illustration, lottie mascots, emoji in body copy, marketing-stock photography, "AI sparkles," shimmer backdrops, autoplaying voiceover, scroll-jacking the home page (only the demo section pins), and any animation that fires faster than 12hz on idle.
- Type voice: dense, technical, monospaced for numerics, editorial-serif for ONE pull-quote per page (not more). No marketing hype words ("revolutionary," "magical," "next-gen"). The copy must be specific to TOON compression ratios and MCP semantics.
- Performance: hero canvas must hold 60fps on M1 Air at full DPR, gracefully degrade to a static AVIF poster on mobile, and respect `prefers-reduced-motion` by freezing all shader uniforms and disabling post-processing.
- Accessibility: every animated headline has a non-animated `<h1>` sibling for screen readers; the canvas is `aria-hidden`; nav reaches all sections via tab order before the canvas mounts.
- Bundle budget: hero route under 180kb gz JS, 90kb gz CSS. Spline scene on integrations section is the only allowed exception (deferred load).
