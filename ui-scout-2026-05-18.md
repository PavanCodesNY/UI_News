## 1. Enerblock — https://enerblock.net/en/

**Source:** Awwwards Site of the Day, May 16, 2026 (also listed May 17 in the SOTD index — page metadata says May 16). SOTD score 7.29/10, DEV award 7.46/10. Built by BANILA Studio + Workoholics.

WHAT IT DOES: Single-page B2B product site for a modular construction system, structured as a vertical scroll narrative of pinned sticky sections. The hero is a full-bleed background video on a near-black canvas (#0C0B11) overlaid with a vector logo that morphs on cursor proximity (translate + scale + path-warp triggered by `mousemove` mapped to a normalized x/y vector). Subsequent sections use ScrollTrigger-pinned panels where text and image elements crossfade and translate as `progress` advances 0→1, plus a marquee-style team grid with hover-isolated portraits. Two-color palette: warning-orange accent #FF5949 and near-black #0C0B11.

PACKAGES:
- `astro@^5` (framework, partial hydration with islands)
- `@astrojs/sanity` + `@sanity/client` (CMS — Sanity Studio)
- `gsap@^3.12` with `ScrollTrigger`, `Observer`, `SplitText` (paid Club GreenSock plugins)
- `lenis@^1` (smooth scroll, pairs with ScrollTrigger via `lenis.on('scroll', ScrollTrigger.update)`)
- `tailwindcss@^4` (utility CSS)
- Self-hosted display sans (Söhne-class) + monospace caption font, served as `font-display: swap` `.woff2`
- `@vercel/og` or static OG generation for socials

IMPLEMENTATION:
1. Astro layout with a `<Lenis>` wrapper island, sectioned page: `<Hero>`, `<ProblemSection>`, `<HowItWorks pinned>`, `<TeamMarquee>`, `<StickyCTA>`, `<Footer>`.
2. Hero logo morph: register one `mousemove` listener on the hero container. Normalize to `(-1..1, -1..1)`. Drive a GSAP timeline that animates SVG `<path>` `d` attribute via MorphSVG (or pre-bake 8 path states and `gsap.to({ morphSVG: paths[idx] }, { duration: 0.6, ease: 'power3.out' })`). Simultaneously translate the logo on a `0.04` lerp factor (`gsap.quickTo`) for inertial cursor follow.
3. Pinned section: `ScrollTrigger.create({ trigger, start: 'top top', end: '+=2000', pin: true, scrub: 0.8 })`. Inside, `tl.fromTo('.copy', { yPercent: 30, opacity: 0 }, { yPercent: 0, opacity: 1, ease: 'power2.out' }, 0)`. Sub-pin three "chapters" per panel with `snap: { snapTo: [0, 0.33, 0.66, 1], duration: 0.4 }`.
4. Team marquee: CSS `animation: marquee 40s linear infinite;` on a duplicated track. On row hover, `gsap.to(siblings, { opacity: 0.25, duration: 0.4 })`, `gsap.to(target, { scale: 1.05, duration: 0.4, ease: 'power3.out' })`. Pause the marquee with `animation-play-state: paused` on `:hover`.
5. Type: display headings clamp(`44px → 8.5vw → 140px`), tracking `-0.04em`, line-height `0.92`. Body: 18px/1.5, weight 400, captions in uppercase mono with `letter-spacing: 0.08em`.
6. Performance: lazy-load section media via Astro `<Image inferSize loading="lazy" />`; preload only the hero video poster; serve hero as `.webm` (VP9) + `.mp4` (H.264) at ≤2 MB, `playsinline muted autoplay`.
7. Breakpoints: 480 / 768 / 1024 / 1440. Mobile collapses pinned panels into stacked viewports (kill ScrollTrigger on `(max-width: 768px)` with `matchMedia`).

WHY IT WORKS: Two-color discipline (one accent + one near-black) forces hierarchy to live in typography and motion, not color noise — the eye reads orange as "where to look next." The pinned scrub model converts scroll distance into a narrative time axis, which is the same pattern a product walkthrough should follow (each scroll tick = one product step). The cursor-reactive logo creates a 200ms feedback loop that flags "this site is alive" before the user has read a single word.

GOOD: Color discipline, scroll choreography, and font pairing are best-in-class — every motion has a job. BAD: Pinned scrolling on trackpad devices feels heavy at default `scrub` values; on mobile the team marquee can starve the main thread if not virtualized.

CLEAN APPLICATION: Apply this to **Clean's landing page hero + "How it works" section** by (a) using a two-color palette of one Clean accent (e.g. neon green or electric blue against `#0A0A0F`), (b) using the cursor-reactive logo morph for the Clean wordmark — but morph between the literal Clean glyph and a stylized MCP-proxy diagram (small node graph), so the brand mark IS the product mental-model, and (c) replacing the pinned narrative panels with three steps: "Connect → Compress (TOON) → Route" where scrub progress advances the visual of a data packet flowing left-to-right through the proxy.

## 2. Fourmula AI — https://fourmula.ai

**Source:** Awwwards Site of the Day, May 2, 2026. SOTD score 7.27/10, DEV 6.96/10. Built by Fourmeta Agency.

WHAT IT DOES: AI generative-asset platform marketing site. Opens with a custom loading screen (counter 0→100 with character-by-character glyph swap, monospace) that cross-fades into a dynamic hero — a black canvas (#020108) with a single orange (#FF6B02) wordmark and a procedurally morphing 2D blob/noise field behind it. Below the fold, sections use "seamless scroll" (Lenis-driven momentum) and a footer that reveals an interactive grid on hover where each cell pulses in a Conway-like wave. Two-color palette only: black + orange.

PACKAGES:
- `next@^15` or pure Webflow export (Awwwards lists Webflow + GSAP; treat as either)
- `gsap@^3.12` (`ScrollTrigger`, `CustomEase`, `TextPlugin`)
- `lenis@^1`
- `motion@^12` (formerly framer-motion) if React route
- `tailwindcss@^4` if React route
- Custom OffscreenCanvas/WebGL fragment shader OR `paper.js@^0.12` for the morphing blob
- Mono display font (e.g. `JetBrains Mono`, `Berkeley Mono`, or `Geist Mono`)
- `lottie-web@^5` for micro-icons

IMPLEMENTATION:
1. Loading screen: 3-second guaranteed minimum. Wrap a counter `<span data-counter>` and increment via `gsap.to({ val: 0 }, { val: 100, duration: 2.6, ease: 'expo.out', onUpdate: e => el.textContent = String(Math.floor(e.targets()[0].val)).padStart(3, '0') })`. Add a SplitText glyph-shuffle: each frame, randomly replace 1–2 displayed digits with a glyph from `01XΩ§∆` for 80ms then snap to true digit.
2. Hero blob: 600×600 `<canvas>` running a 2D noise loop. Use `simplex-noise@^4`. Each frame, sample 64 points around a circle, perturb radius by `noise2D(angle * 1.2, t * 0.0008) * 80`, draw with `ctx.bezierCurveTo`. Fill with radial gradient: `#FF6B02` center → `transparent` edge. Composite mode `screen` over black.
3. Section transitions: `lenis = new Lenis({ duration: 1.2, easing: t => Math.min(1, 1.001 - Math.pow(2, -10 * t)), smoothWheel: true })`. ScrollTrigger.scrollerProxy('body', { scrollTop, getBoundingClientRect }). Use `gsap.timeline({ scrollTrigger: { trigger, start: 'top 70%', end: 'bottom 30%', scrub: 1 } })` per section.
4. Type: hero `clamp(56px, 11vw, 180px)`, weight 500, tracking `-0.05em`. Sub-copy in mono 14px, uppercase, `letter-spacing: 0.12em`.
5. Footer cell wave: 12×8 grid of `<div>` cells. On `mousemove` over the grid, compute distance from cursor to each cell center; opacity = `clamp(0, 1 - dist / 240, 1)`. Add 60ms staggered delay outward from cursor cell using `gsap.to(cells, { opacity, stagger: { amount: 0.3, from: 'center' } })`.
6. Breakpoints: 480 / 768 / 1280. Disable blob canvas on mobile; replace with a static gradient PNG.

WHY IT WORKS: The loader is doing two jobs at once — masking actual asset load AND committing the visitor to a hero reveal they're already invested in (the 0→100 count is a small bet they want to see resolve). The single-color accent (orange against pure black) maps directly onto AI product semantics: heat, activity, generative energy. The morphing blob runs ~6 fps of *visual* change but only ~30 fps of canvas work — it feels alive without burning the GPU.

GOOD: Loader-to-hero choreography is textbook narrative onboarding; type hierarchy is ruthless. BAD: Accessibility score is 6.2/10 (jury vote) — likely missing reduced-motion handling and contrast on small mono copy. Don't replicate the contrast misses.

CLEAN APPLICATION: Apply this to **Clean's landing-page hero loader** by (a) building a 1.8-second loader showing a TOON-format compression demo — left side a JSON blob, right side the equivalent TOON string, with a `tokens: 1,247 → 312` counter ticking down, (b) using mono type throughout the marketing site (Berkeley Mono or Geist Mono) so the brand reads as "developer-credible by default," and (c) replacing the procedural blob with a slow-rotating MCP topology graph (nodes + edges, WebGL lines, 20 nodes max, 30fps target) as the hero ambient element.

## 3. Juan Mora — https://juanmora.co/

**Source:** Awwwards Nominee Apr 27, 2026 (also listed as SOTD May 15 in the SOTD calendar — Awwwards index lags on the canonical page). Built by Juan Mora / Morable.

WHAT IT DOES: A design director's portfolio that reads like an editorial print magazine spread translated to scroll. Hero is a large serif word-mark with a kinetic underline that pulses with cursor proximity. Project entries are full-bleed image cards with a magnetic-hover read-more affordance that warps a circular SVG cursor on entry. The "About" route uses a horizontal page transition where the homepage shears off-screen left while a colored panel (vibrant pink/yellow/teal rotating by visit) slides in from right with stagger-revealed text. Lottie micro-animations punctuate scroll milestones (an arrow that draws itself when each section enters viewport).

PACKAGES:
- Webflow export OR `next@^15` (Awwwards lists Webflow + GSAP + Lottie)
- `gsap@^3.12` (`ScrollTrigger`, `Flip`, `SplitText`)
- `lottie-web@^5` (or `@lottiefiles/dotlottie-web`)
- `lenis@^1`
- `tailwindcss@^4`
- Display serif (e.g. `PP Editorial New`, `Tiempos Headline`, or `Migra`) + neo-grotesk sans (Inter, Söhne)

IMPLEMENTATION:
1. Custom cursor: `<div id="cursor">` absolute-positioned, GPU-composited. Use `gsap.quickTo('#cursor', 'x', { duration: 0.4, ease: 'expo.out' })` and same for `y`. On hover over `[data-magnetic]`, lerp cursor toward target's center with weight 0.35.
2. Underline pulse: cursor's distance to underline `d` controls SVG `<line stroke-width>` 1px→3px and `stroke-dasharray` offset for kinetic shimmer. Easing `power2.out`, duration `0.35s`.
3. Project card hover: GSAP Flip — capture initial state, swap card to expanded layout in DOM, `Flip.from(state, { duration: 0.7, ease: 'expo.inOut', absolute: true })`. Image gets a `clip-path: inset(0 0 0 0)` reveal animated via `gsap.fromTo({ clipPath: 'inset(50% 50% 50% 50%)' }, { clipPath: 'inset(0 0 0 0)', duration: 0.9, ease: 'expo.inOut' })`.
4. About transition: on click, generate a full-viewport `<div class="curtain">` with `transform: translateX(100%)`, then `gsap.to(curtain, { x: 0, duration: 0.8, ease: 'expo.inOut' })`. Simultaneously translate current page `gsap.to(main, { xPercent: -30, duration: 0.8 })` then swap route content under the curtain and reverse.
5. Lottie milestones: `lottie.loadAnimation({ container, renderer: 'svg', loop: false, autoplay: false, path: '/lottie/arrow.json' })`. Trigger via `ScrollTrigger.create({ trigger: el, start: 'top 80%', onEnter: () => anim.play() })`.
6. Type system: H1 serif `clamp(60px, 9vw, 160px)` `font-style: italic` `line-height: 0.95`; body sans 18px/1.55. Mix of italic display + ALL-CAPS sans labels at 11px `letter-spacing: 0.18em` is the editorial signature.
7. Color: rotate accent on visit by `localStorage.setItem('visitColor', pick([pink, yellow, teal, lime]))`.

WHY IT WORKS: Editorial typography (italic serif display + ALL-CAPS micro-labels) is borrowed from print magazines where the eye is trained to follow contrast jumps — it bypasses "SaaS-feeling" and reads as personality. The curtain transition gives an emotional reset between pages, which is exactly what a portfolio needs (each project is its own world). The visit-randomized accent color is a tiny cleverness that makes the third visit feel different from the first without changing layout.

GOOD: Type system and transition choreography are best-in-class. BAD: Magnetic cursor + GSAP Flip Pro is heavy on lower-end Android; needs a `(prefers-reduced-motion)` and `(hover: none)` fallback.

CLEAN APPLICATION: Apply this to **Clean's docs / changelog page** by (a) treating each changelog entry as an editorial card with display-serif version number (`v 0.4.2` in italic Editorial New, 96px) + sans body, (b) using the curtain transition between changelog page → individual entry, and (c) repurposing the magnetic cursor only over interactive code blocks (hover a snippet → cursor turns into a "copy" pill that magnetically snaps to the code top-right).

## 4. ASTRODITHER — https://astrodither.com/ (via https://www.awwwards.com/sites/astrodither)

**Source:** Awwwards SOTD + Developer Award, May 5, 2026. Score 7.35/10. Created by Robert Borghesi.

WHAT IT DOES: Single-page experimental microsite that is essentially a full-screen WebGL canvas running an Astro × Three.js fragment shader that renders a real-time Bayer-matrix dithered gradient field, with two-color output (#FF4FA3 hot pink + #0B0B12 near-black). Text overlays use the dither texture as a `mix-blend-mode` mask so headlines look like they're being printed through the same noise field. Mouse position modulates shader uniforms in real time — `uMouse.x` shifts the dither threshold, `uMouse.y` shifts gradient angle.

PACKAGES:
- `astro@^5` (with the React integration for the canvas island)
- `three@^0.165`
- `@react-three/fiber@^9` + `@react-three/drei@^10` (optional — vanilla Three works too)
- Custom GLSL fragment shader (Bayer 8×8 ordered dither + 2-stop gradient + noise)
- `tailwindcss@^4`
- Mono font for overlays (Geist Mono / IBM Plex Mono)

IMPLEMENTATION:
1. Astro page with `client:only` React island holding a `<Canvas>` from r3f, fixed `position: fixed; inset: 0; z-index: 0`. Content sits at `z-index: 1` with `pointer-events: none` except for interactive nodes.
2. Shader (fragment, GLSL ES 3.0):
   ```glsl
   uniform vec2 uResolution;
   uniform vec2 uMouse;
   uniform float uTime;
   const float bayer8[64] = float[](...); // 8×8 Bayer matrix normalized 0..1
   void main() {
     vec2 uv = gl_FragCoord.xy / uResolution.xy;
     float angle = mix(0.2, 1.4, uMouse.y);
     vec2 dir = vec2(cos(angle), sin(angle));
     float g = dot(uv - 0.5, dir) + 0.5 + sin(uTime * 0.2) * 0.05;
     int ix = int(mod(gl_FragCoord.x, 8.0));
     int iy = int(mod(gl_FragCoord.y, 8.0));
     float threshold = bayer8[iy * 8 + ix];
     float bias = mix(-0.15, 0.15, uMouse.x);
     vec3 hot = vec3(1.0, 0.31, 0.64);   // #FF4FA3
     vec3 ink = vec3(0.043, 0.043, 0.07); // #0B0B12
     vec3 col = (g + bias > threshold) ? hot : ink;
     gl_FragColor = vec4(col, 1.0);
   }
   ```
3. Drive uniforms from `useFrame`: `state.mouse` is already normalized in r3f, lerp into `uMouse` at 0.08. Update `uTime` linearly.
4. Headline mask: use the canvas as a CSS `mask-image: paint(astrodither)` via Houdini, OR — simpler — render text in a second canvas and `globalCompositeOperation = 'source-in'` against the shader output, then position as `<canvas>` over the text.
5. Performance: render at `devicePixelRatio: Math.min(1.5, window.devicePixelRatio)` to avoid 4K oversample. Pause on `document.hidden`. Drop to a 1024×1024 internal buffer + `scale(window/1024)` on mobile.
6. Accessibility: respect `prefers-reduced-motion` → set `uTime` to 0 and freeze `uMouse` at center.

WHY IT WORKS: Dither aesthetics carry a specific visual rhetoric — they evoke 80s-90s computing, early Macintosh, and demoscene culture, which reads instantly as "this person speaks fluent computer." Limiting to two colors enforces silhouette-only readability, which is the strongest test of typographic hierarchy. The cursor-responsive shader is high-impact-per-cycle: a single fragment shader running 60fps replaces what would otherwise be 200KB of motion graphics.

GOOD: Single shader does the entire visual identity — extreme efficiency-to-impact ratio. BAD: Fully shader-driven sites are nearly impossible to crawl/index for SEO; the marketing copy lives in DOM but is masked behind blend modes that can confuse screen readers. Don't replicate without a fallback.

CLEAN APPLICATION: Apply this to **Clean's `/demo` or `/playground` page background** by (a) running the same Bayer-dither shader behind the live MCP-proxy playground, but recolored with Clean's accent + `#0A0A0F`, (b) using `uMouse.x` to drive proxy latency simulation (further right = more requests/sec, shader gets denser), so the background literally visualizes the product's throughput, and (c) keeping it OFF the main marketing landing — it's a "product surface" effect, not a hero effect.

## 5. Aceternity UI — Background Ripple Effect + Dotted Glow Background — https://ui.aceternity.com/components

**Source:** Latest Aceternity UI free-tier component drops (referenced in Aceternity's May 2026 catalog along with Webcam Pixel Grid, Parallax Hero Images, Scales, Sparkles). Best-in-class for AI/dev-tool aesthetics.

WHAT IT DOES: Two stackable background primitives. **Background Ripple Effect** is a grid of `<div>` cells (default 56×40 at 1440px) that animate a radial ripple of opacity outward from a clicked cell — each cell is staggered by `dist * 30ms`, scales `1 → 1.2 → 1`, and dims back over 1.2s. **Dotted Glow Background** is a single CSS `radial-gradient(circle at 50% 30%, var(--accent) 0%, transparent 60%)` plus a `background-image: radial-gradient(currentColor 1px, transparent 1px)` dot pattern at `background-size: 24px 24px`, with the glow tied to cursor position via CSS custom properties.

PACKAGES:
- `next@^15` (or any React 19 host)
- `react@^19`
- `tailwindcss@^4`
- `motion@^12` (framer-motion successor — Aceternity migrated)
- `clsx@^2` + `tailwind-merge@^2` (Aceternity's `cn` util)

IMPLEMENTATION:
1. Background Ripple component skeleton:
   ```tsx
   'use client';
   import { motion } from 'motion/react';
   import { useState } from 'react';
   const ROWS = 40, COLS = 56;
   export function BackgroundRipple() {
     const [origin, setOrigin] = useState<[number, number] | null>(null);
     return (
       <div className="absolute inset-0 grid" style={{ gridTemplateColumns: `repeat(${COLS}, 1fr)` }}>
         {Array.from({ length: ROWS * COLS }).map((_, i) => {
           const r = Math.floor(i / COLS), c = i % COLS;
           const dist = origin ? Math.hypot(c - origin[0], r - origin[1]) : 0;
           return (
             <motion.div key={i}
               onClick={() => setOrigin([c, r])}
               animate={origin ? { opacity: [0, 0.6, 0], scale: [1, 1.15, 1] } : {}}
               transition={{ duration: 1.2, delay: dist * 0.03, ease: 'easeOut' }}
               className="border border-white/[0.04] hover:bg-white/5" />
           );
         })}
       </div>
     );
   }
   ```
2. Dotted Glow Background:
   ```tsx
   export function DottedGlow() {
     return (
       <div className="absolute inset-0 bg-neutral-950
         [background-image:radial-gradient(theme(colors.neutral.700)_1px,transparent_1px)]
         [background-size:24px_24px]"
         onMouseMove={(e) => {
           const r = e.currentTarget.getBoundingClientRect();
           e.currentTarget.style.setProperty('--mx', `${e.clientX - r.left}px`);
           e.currentTarget.style.setProperty('--my', `${e.clientY - r.top}px`);
         }}
         style={{
           backgroundImage: `radial-gradient(circle 320px at var(--mx, 50%) var(--my, 30%), rgba(59,130,246,0.25), transparent 60%), radial-gradient(theme(colors.neutral.700) 1px, transparent 1px)`,
           backgroundSize: 'auto, 24px 24px',
         }} />
     );
   }
   ```
3. Stack with `position: absolute; inset: 0; pointer-events: none` (except ripple, which needs `onClick`). Content above at `relative; z-10`.
4. Mobile: reduce ROWS×COLS to 24×16 on `(max-width: 768px)` to keep DOM under 400 nodes. Throttle `mousemove` to 30Hz with `requestAnimationFrame`.
5. Theme: Aceternity expects CSS vars `--background: 0 0 0`, `--foreground: 255 255 255`, `--accent: 59 130 246` in HSL or RGB triples.

WHY IT WORKS: Both backgrounds use a uniform grid as the visual unit, which is "honest" to a dev-tool brand — the grid IS the metaphor for structured data, infrastructure, and addressable space. The interactivity is *light* — ripple only fires on click, glow only follows cursor — so neither competes with the actual content; they're ambient. Aceternity gets this trade-off right: the components feel "alive" with under 5% perceptual cost.

GOOD: Copy-paste ready, theme-via-CSS-vars, sub-2KB JS per component, no canvas. BAD: 2,240 DOM nodes for default ripple grid bloats Lighthouse "Avoid an excessive DOM size"; large grids will hammer mobile Safari.

CLEAN APPLICATION: Apply this to **Clean's pricing page background AND docs sidebar** by (a) using Dotted Glow Background on `/pricing` with the glow color matching the currently-hovered tier card (CSS var swap on `:hover`), so the page visually responds to plan selection, and (b) using Background Ripple Effect (smaller — 24×16 grid) behind the docs sidebar, where clicking a nav item triggers a ripple from that node — a small treat that signals "this is an interactive, developer-built doc system."

## BUILD DIRECTIVE

**TARGET:** Clean's landing-page hero + immediate "How it works" section (i.e., everything from page load through the user's first 1.5 viewports of scroll on `tryclean.ai`).

**APPROACH:** Build a 1.8-second loader-to-hero transition that doubles as a product demo: the loader shows a JSON-to-TOON token-count counter (`1,247 tokens → 312 tokens`) ticking down in mono type on a near-black canvas, then cross-fades into a hero that combines (a) the cursor-reactive monochromatic minimalism of Fourmula AI (single Clean-accent color against `#0A0A0F`), (b) the Enerblock cursor-reactive wordmark — but morphing between the Clean glyph and a 6-node MCP-proxy topology graph rendered as live SVG, and (c) a Dotted Glow Background from Aceternity behind a 3-step "Connect → Compress → Route" scrollytelling panel that scrubs as the user scrolls the first 1.5 viewports. Display type in mono (Berkeley Mono or Geist Mono) throughout to anchor developer credibility. Replace any decorative imagery with live, deterministic visualizations of the proxy at work — every visual element on the hero must literally depict a real Clean capability, not metaphor.

**STACK:**
- `next@^15` (App Router, RSC where possible, `client:only` islands for canvas/motion)
- `react@^19`
- `typescript@^5`
- `tailwindcss@^4` + `@tailwindcss/typography`
- `motion@^12` (was framer-motion)
- `gsap@^3.12` with `ScrollTrigger` + `Observer`
- `lenis@^1`
- `three@^0.165` + `@react-three/fiber@^9` (for the MCP topology graph node renderer)
- `clsx@^2` + `tailwind-merge@^2`
- `geist@^1` (font) + `Berkeley Mono` (paid, self-hosted woff2)
- shadcn/ui (Sera style) for buttons/cards/inputs
- Aceternity UI primitives copy-pasted: Background Ripple, Dotted Glow, Spotlight New

**REFERENCE URLS:**
1. https://enerblock.net/en/ — for the two-color palette discipline, cursor-reactive logo, and pinned scroll narrative pattern
2. https://fourmula.ai — for the loader-to-hero choreography, monochrome-plus-accent treatment, and mono type system
3. https://ui.aceternity.com/components/dotted-glow-background — for the ambient cursor-follow background primitive

**KEY CONSTRAINTS:**
- Clean is developer infrastructure. The result must feel like an internal Vercel/Linear/Anthropic tool, not an agency portfolio. **Avoid:** glassmorphism, drop shadows above 2px, gradient-on-gradient backgrounds, rounded corners above 12px, "AI-shimmer" effects, marquee customer logos, emoji, > 2 fonts, > 3 colors total.
- Type system: mono display (Berkeley Mono or Geist Mono) + Inter or Geist Sans for body. That is it.
- Color: `#0A0A0F` background, `#FAFAFA` foreground, one Clean accent (suggested: electric green `#A3FF12` or signal cyan `#00E0FF`). No more.
- Motion: every animation must have a `(prefers-reduced-motion)` no-op fallback. All ScrollTriggers must dispose on route change. Hero canvas must run ≤ 30 fps with `devicePixelRatio` capped at 1.5.
- Performance budget: LCP ≤ 1.2s on 4G, total JS ≤ 180KB gzipped on landing route, no third-party scripts above the fold.
- Accessibility: WCAG AA contrast (≥ 4.5:1) on all body copy; the hero canvas must NOT mask any text via blend modes (learn from ASTRODITHER's mistake).
- **Do not** ship anything resembling: floating 3D objects, parallax scroll on hero images, "scroll-triggered storytelling" with stock illustrations, AI-generated hero images, or video backgrounds longer than 8 seconds.
