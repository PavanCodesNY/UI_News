## 1. KVS Studio — https://www.kvs.services/  (Awwwards SOTD + Developer Award, May 18, 2026)

WHAT IT DOES: Single-page studio site driven by a fixed WebGL canvas behind a Vue (Nuxt) DOM layer. The page is split into vertically stacked "story frames" — hero with a parallax-distorted product mockup, case-study tiles that snap-scroll into a 3D plane that bends/skews on intersection, and a quiet text+grid services block. Cursor is tracked globally and drives a custom shader (chromatic-aberration sweep, soft displacement) on the active hero plane. Color system is monochrome black with a single signal hex `#FF5500` used only on hover targets and active state — no other accents anywhere.

PACKAGES:
- `nuxt@3.x`, `vue@3.x`
- `three@0.165+`, `@tresjs/nuxt` (Vue's R3F-equivalent) or raw `three` with a custom renderer
- `gsap@3.12+` with `ScrollTrigger`
- `@studio-freight/lenis@1.x` (smooth scroll)
- `glsl-noise`, custom `.glsl` shader files compiled via `vite-plugin-glsl`
- `splitting@1.x` or `gsap/SplitText` for character-level reveals
- Fonts: a single variable serif/display (looks like Editorial New or Migra) + a mono for labels
- `lottie-web@5.x` (one micro-loop in services block)

IMPLEMENTATION:
1. Mount a `position:fixed; inset:0; z-index:0; pointer-events:none` `<canvas>` for the persistent WebGL scene. DOM content sits at `z-index:1` with `mix-blend-mode: normal` and transparent backgrounds so the canvas reads through dedicated "slot" rectangles.
2. Each "slot" in the DOM is an empty `<div data-gl-slot>` with measured `getBoundingClientRect`. On scroll, sync a textured `THREE.PlaneGeometry` to those rects (convert px to NDC). This is the basement.studio / Lusion pattern.
3. Shader: vertex shader applies a sin-wave distortion driven by a `uScrollVelocity` uniform (lerped: `uScrollVelocity += (target - current) * 0.08`). Fragment shader does an RGB-shift proportional to velocity and a soft dither pattern at `0.04` strength.
4. Cursor: pass `uMouse` (smoothed via `lerp(current, target, 0.12)`) to fragment, use it as the center of a radial chromatic aberration up to `0.012` UV offset.
5. Smooth scroll via Lenis at `duration: 1.2, easing: t => 1 - Math.pow(1 - t, 3)`. Hook Lenis into GSAP ticker.
6. Section reveals: GSAP `ScrollTrigger` with `start: "top 80%"`, character stagger `0.018s`, `y: 24 → 0`, `opacity: 0 → 1`, ease `expo.out`, duration `1.1s`.
7. Hover state on tiles: GSAP timeline scaling underline `scaleX 0 → 1` over `0.5s ease power3.out`, color swap to `#FF5500`, plane distortion amplitude doubles on the matching slot only.
8. Responsive: below `768px` disable WebGL entirely, replace with static `<img>` + CSS `transform` parallax. Saves battery and complexity.
9. Performance: `dpr={[1, 1.5]}` cap, `<canvas>` resizes via debounced `ResizeObserver` (200ms), instanced meshes for any repeating geometry, frustum culling on, antialias off (rely on FXAA pass or MSAA only at high DPR).

WHY IT WORKS: Monochrome with a single signal color is the textbook contrast hierarchy — every interactive element earns attention without competing with content. The fixed canvas + DOM-slot pattern resolves the false choice between "WebGL site" and "real content site": you get the motion language of a 3D site with the typography control of HTML. The scroll-velocity-as-distortion shader is motion-semantic — it makes you feel speed instead of telling you about it.

GOOD / BAD:
- GOOD: Best-in-class restraint. Animation amplitude is always proportional to user input (velocity, cursor distance) so it never plays "at" you.
- BAD: With JS off there's still no fallback for the hero plane — it's a blank rectangle. On a 60Hz integrated GPU laptop the chromatic aberration shader pushes frame time over 16ms during fast scrolls.

CLEAN APPLICATION: Apply this to Clean's landing page hero (`tryclean.ai`) by replacing the static "MCP proxy" illustration with a fixed-position WebGL canvas that renders a wireframe topology of agent → proxy → tool nodes. Tie node-edge "pulse" intensity to scroll velocity so the network appears to "carry more traffic" the faster the user scrolls. Use a single signal hex (Clean's existing accent, applied only to the proxy node and active edges) against a near-black canvas. DOM content (headline, sub, CTA) sits over a transparent slot, and the headline characters split-reveal with the same 18ms stagger.

## 2. Enerblock — https://enerblock.net/en/  (Awwwards SOTD + Developer Award, May 17, 2026, by BANILA Studio × Workoholics)

WHAT IT DOES: A long-form scrollytelling product page for an industrial construction system. The structural conceit is a "scroll = manufacturing process" mapping — each section corresponds to a stage in the production pipeline (design → engineering → fabrication → delivery), and the hero contains a horizontally-pinned, scrubbed video/canvas sequence of a building component being assembled. A persistent thin progress bar at the top of the viewport indexes the stage. Text blocks are typographically heavy (large, almost editorial), with rules and small all-caps labels that look like factory drawings.

PACKAGES:
- `next@14.x` (App Router) or `astro@5.x`
- `gsap@3.12+` with `ScrollTrigger` and `ScrollSmoother` (Premium tier — required for the pinned scrub)
- `@studio-freight/lenis` if not using ScrollSmoother
- `framer-motion@11.x` for the smaller in-section transitions only
- A PNG/WebP sequence (~120 frames) OR an `<canvas>` with a paused `<video muted playsinline preload="auto">` whose `currentTime` is scrubbed (the Apple AirPods technique)
- Variable font: Söhne / Inter Display for body, Editorial New or Tobias for headlines
- `lottie-web` for the progress micro-icons

IMPLEMENTATION:
1. Hero pin block: `ScrollTrigger.create({ trigger: ".hero", pin: true, start: "top top", end: "+=3000", scrub: 0.6 })`. Inside, animate a single timeline that (a) scrubs the video `currentTime` from 0 → duration mapped to scroll progress, (b) cross-fades 4 captions positioned absolutely with `opacity` and `y` ±20px at each quarter-progress point, (c) advances the progress-bar fill `scaleX 0 → 1` with `transform-origin: left`.
2. For the video scrub, listen on RAF (not native `timeupdate`): `video.currentTime = progress * video.duration`. Some browsers (Safari) require the video to be H.264 baseline @ 24fps and `<= 5MB` to scrub smoothly — otherwise convert to a PNG sequence and drive `image.src` from a preloaded `Image[]` pool.
3. Section transitions: each subsequent block reveals with a `clip-path` wipe — `clip-path: inset(0 0 100% 0)` → `inset(0 0 0% 0)` over `1.2s cubic-bezier(0.65, 0, 0.35, 1)` triggered when its top hits 70% viewport.
4. Typographic system: headline `clamp(2.5rem, 6vw, 6.5rem)`, line-height `0.92`, letter-spacing `-0.03em`. Body `1.0625rem / 1.6`. Labels `0.6875rem / 1.2 uppercase, letter-spacing 0.16em`. Vertical rhythm based on an 8px grid, section padding `clamp(96px, 12vh, 160px) 0`.
5. Color: warm-paper background `#F4F1EB`, ink `#1A1A1A`, single accent for diagrams (looks like a deep terracotta `#A6442B`).
6. Performance: lazy-mount each `ScrollTrigger` with `ScrollTrigger.batch` for stagger reveals; mark large media as `loading="lazy" decoding="async"`; ship `<link rel="preload" as="video" type="video/mp4">` for the hero sequence only.
7. Mobile: drop the pinned scrub. Replace with a tap-to-play inline `<video>` and a vertical sequence of static captions.

WHY IT WORKS: The page literally is the product — scrolling reads as fabrication. Apple's product pages popularized this; Enerblock applies it to B2B industrial where it usually never happens, and that incongruity is the whole experience. The editorial typography signals "engineering catalog," reinforcing seriousness while the animation provides the dopamine.

GOOD / BAD:
- GOOD: Information density is high but legibility never breaks; the pinned scrub teaches the product without a single explainer video CTA.
- BAD: The 3000px-equivalent pin means low-traffic, low-time-on-page visitors bounce before the punchline. Mobile fallback feels like a different (lesser) site.

CLEAN APPLICATION: Apply this to Clean's "How it works" / product-demo section by mapping scroll to the lifecycle of a single agent request — request enters → proxy intercepts → context layer compresses tokens (TOON format) → tool call executes → response returns. Pin the section, scrub a 30-frame canvas of the request packet moving through the topology, and cross-fade captions for each stage with the same `clip-path inset` wipes. Reuse Clean's existing type stack but tighten line-height to `0.92` on the section headline and add a thin top progress bar that fills as the scrub advances. This is the section that turns "MCP proxy" from a buzzword into a verb.

## 3. Cursor — https://cursor.com/  (Cursor 3 launch April 2, 2026; homepage actively iterating through May)

WHAT IT DOES: Marketing homepage rebuilt around the "Agents Window" narrative. Hero is a high-fidelity product screenshot in a dark, glass-edged window chrome, animated with a subtle floating tilt (parallax tied to cursor position, max 6° rotateX/rotateY). Below the hero, three feature blocks each pair a paragraph with an isolated, looping product micro-interaction (an agent spinning up, files being edited side-by-side, a design-mode annotation overlay). The whole page is dark by default with a near-white text deck; only the active product UI inside the screenshots carries color. Type system uses Cursor's wordmark font for headlines (geometric sans with sharp terminals) and Inter for body.

PACKAGES:
- `next@15.x` (App Router, RSC)
- `tailwindcss@3.4+` with custom `theme.extend` for the dark palette
- `framer-motion@11.x` for the hero parallax and feature reveals
- `lenis` for smooth scroll
- `@radix-ui/react-*` for any tooltips/menus
- `lucide-react` for icons
- Component primitives from shadcn/ui (likely a custom-themed fork)
- Inter Variable + Cursor's custom wordmark font (self-hosted woff2)
- Lottie or pre-rendered MP4 for the looping product clips (NOT iframes — they record real Cursor sessions, export as WebM, and loop)

IMPLEMENTATION:
1. Page shell: `bg-[#0A0A0A] text-zinc-100`, `max-w-[1280px] mx-auto px-6`, vertical rhythm in `space-y-[120px] md:space-y-[200px]`.
2. Hero parallax: `useMotionValue(x), useMotionValue(y)` updated on `pointermove`, `useTransform` to map to `rotateX: [-6, 6]`, `rotateY: [-6, 6]`, applied with `transform-style: preserve-3d, perspective: 1200px`. Smooth via `useSpring({ stiffness: 100, damping: 20, mass: 0.5 })`.
3. Window chrome on the hero screenshot: outer `<div>` with `rounded-2xl border border-white/10 bg-zinc-950 shadow-[0_50px_100px_-20px_rgba(0,0,0,0.7),0_30px_60px_-30px_rgba(0,0,0,0.6)]`, top bar with three traffic-light dots (`bg-zinc-700`) and a centered breadcrumb. Inside: the actual product screenshot at `2x` resolution.
4. Feature loops: each is a `<video autoPlay muted loop playsInline preload="metadata">` wrapped in the same window chrome. Layout uses CSS grid `lg:grid-cols-[1.1fr_1fr]` with `gap-16`. Text column has eyebrow (`text-xs uppercase tracking-[0.18em] text-zinc-500`), headline (`text-4xl md:text-5xl tracking-tight`), body (`text-zinc-400 leading-relaxed`).
5. Section reveal: `whileInView={{ opacity: 1, y: 0 }}` from `{ opacity: 0, y: 24 }`, `transition={{ duration: 0.7, ease: [0.22, 1, 0.36, 1] }}`, `viewport={{ once: true, margin: "-15% 0px" }}`.
6. CTA: a single high-contrast button `bg-white text-black hover:bg-zinc-200`, `rounded-full px-5 py-2.5 text-sm font-medium`, plus a ghost secondary `border border-white/15`.
7. Navigation: sticky top bar with `backdrop-blur-xl bg-black/40 border-b border-white/5`, transitions to opaque after `scrollY > 32`.
8. Performance: every product video is < 1.5 MB, served as AV1 with WebM fallback; `poster` is the first frame so layout never shifts.

WHY IT WORKS: The hero shows the product, not a metaphor for the product. Dark-on-dark UI with a single bright accent (the active panel inside the screenshot) is hierarchy by negative space — the eye lands on UI exactly where the message lives. The 6° parallax is below the threshold of "this is a 3D effect" and above the threshold of "this is dead" — it sits in the uncanny middle that reads as "alive but serious."

GOOD / BAD:
- GOOD: Highest signal-to-noise of any dev-tool homepage shipping right now — every section makes you understand a feature without watching a video.
- BAD: A real screenshot dates fast — every product update breaks the page. The footer information density (10+ link columns) is the only part that feels like 2017.

CLEAN APPLICATION: Apply this to Clean's landing page hero AND the three feature blocks below it. Replace the abstract proxy diagram with an actual screenshot of the Clean dashboard (or a stylized but real recreation: a panel showing tokens-in vs. tokens-out with the TOON-compression delta highlighted). Use the same 6° pointer parallax, the same window-chrome treatment (`rounded-2xl border border-white/10`, the heavy double-shadow), and the same eyebrow / 4xl-5xl headline / `text-zinc-400` body cadence in the feature trio: (1) MCP Proxy intercept, (2) TOON context compression, (3) Per-agent observability. Each feature gets a real ~5s WebM loop of the dashboard doing exactly that one thing.

## 4. ASTRODITHER — https://www.awwwards.com/sites/astrodither  (Awwwards SOTD + Developer Award, May 6, 2026, by Robert Borghesi)

WHAT IT DOES: Experimental fullscreen WebGL site whose entire visual signature is a real-time dither shader applied to a rotating 3D object (an orb / abstract geometry) over a pure black background with one neon-pink signal color `#FF4FA3`. Mouse moves rotate the object on `y/x` axes; scroll zooms the camera and increases dither density. There is essentially no DOM content — type appears as part of the canvas, post-processed through the same dither.

PACKAGES:
- `astro@5.x` (yes, the site that brands itself "ASTRO-dither" is shipped on Astro — they leaned into the joke)
- `three@0.165+`
- `postprocessing@6.36+` (`EffectComposer`, custom `Effect` for dither)
- `glslify` or `vite-plugin-glsl`
- A custom Bayer-matrix or blue-noise dither shader (8×8 ordered dither is the cheapest acceptable choice; blue-noise texture lookup looks more organic)
- `gsap` for camera-position tweens
- Fonts rendered to canvas via `troika-three-text`

IMPLEMENTATION:
1. Scene: single `Mesh` with custom `ShaderMaterial`. Object can be a `IcosahedronGeometry(1, 32)` with a vertex shader that displaces along normals using 4D simplex noise (3 axes + time): `displacement = snoise(vec4(p * 1.5, uTime * 0.2)) * 0.18`.
2. Color pass: fragment shader computes a 2-tone gradient (`mix(bgColor, accent, fresnel)`) where `fresnel = pow(1.0 - dot(normal, viewDir), 2.5)`.
3. Dither pass: full-screen `Effect` subclass. Load a `bluenoise_64x64.png` texture, sample it tiled across the screen, threshold the input color's luminance against the noise — output is either `bgColor` or `accent`. This is what gives the entire screen that grainy halftone look.
4. Interaction: `pointermove` → smoothed mouse vec2 → rotates the mesh `rotation.y = mouseX * 0.6`, `rotation.x = -mouseY * 0.4`. Scroll → `camera.position.z` lerped from 3.2 → 1.4 across page length; in parallel, the dither shader's `uPixelSize` uniform decreases from `4.0` → `1.5` (zoom in = finer dither, like leaning into a print).
5. Type: `troika-three-text` instances positioned in world space, also routed through the same dither effect so they integrate visually.
6. Pixel ratio capped at `1.0` deliberately — the chunkier the better.
7. Mobile: keep the canvas but reduce `IcosahedronGeometry` detail to 16 and disable post-processing on devices where `navigator.hardwareConcurrency < 6`.

WHY IT WORKS: Dither has a strong cultural memory (early Mac, terminals, demo-scene art) that reads as "engineered" rather than "designed." The two-color constraint is a maximum-contrast play that's impossible to mess up — every pixel is either the background or the accent, so hierarchy is forced. Adding noise to a 3D scene is the inverse of the usual move (clean up artifacts); embracing the artifact creates identity.

GOOD / BAD:
- GOOD: Identity-defining. You will recognize a screenshot of this site in your peripheral vision.
- BAD: Doesn't tell you what it is. It's a portfolio piece, not a product — applied to a SaaS landing page this would tank conversion.

CLEAN APPLICATION: Apply this technique to Clean's docs index or a standalone `/manifesto` or `/why` page. The technical surface that has the highest tolerance for "feel" content is the docs landing — a single `IcosahedronGeometry` rendered with a blue-noise dither in Clean's accent color, sitting behind the docs nav. The dither aesthetic visually argues "low-level / systems work" without needing copy, which is the right register for an infrastructure tool. Do NOT use this on the main landing — it's not a conversion surface.

## 5. Fourmula AI — https://fourmula.ai/  (Awwwards SOTD, May 2, 2026, by Fourmeta)

WHAT IT DOES: Product marketing site for an AI image generator. Hero is an asymmetric split: left 60% holds a left-aligned typographic headline with character-level scroll-in, right 40% holds a stacked deck of product output images that cycle on a soft auto-rotate (every 3.5s) with a `mask-image` reveal transition. Below the hero, a "before / after" interactive slider shows raw product photo → generated studio shot; the dragger is custom (a thin vertical bar with a circular knob). Sections lower on the page are a horizontal-scroll gallery (vertical scroll → translateX) of customer examples, a pricing block, and a final CTA. Color is near-black `#020108` with a warm orange signal `#FF6B02`.

PACKAGES:
- Webflow (this is the actual stack — every animation is Webflow Interactions or Webflow's native Motion, plus GSAP injected via an embed)
- For a code rebuild: `next@15.x`, `tailwindcss`, `framer-motion`, `gsap`, `lenis`
- `embla-carousel-react` or custom for the image deck
- `react-compare-slider@3.x` (or roll your own — 60 lines of code)
- Font: Inter Display for headline, Inter for body

IMPLEMENTATION:
1. Hero grid: `grid grid-cols-12 gap-8`, headline in `col-span-7`, deck in `col-span-5`. Headline uses GSAP SplitText, characters animated `y: 60 → 0, opacity: 0 → 1, stagger: 0.025, ease: power3.out, duration: 0.9`.
2. Image deck: 4–5 absolutely-positioned `<img>`s with `transform: rotate(-3deg) translate(0, 0)` through `rotate(2deg) translate(8px, -4px)` to create a tactile stack. On rotate, the front image animates `clip-path: inset(0 0 0 100%) → inset(0 0 0 0%)` over `0.8s ease-in-out`, while the back-most image jumps to front via z-index swap. Interval `3500ms`.
3. Before/after: a single container with two stacked `<img>` (the "after" image clipped via `clip-path: inset(0 calc(100% - var(--pos)) 0 0)`). A draggable knob updates a CSS custom property `--pos` via `pointermove`. On idle, a `requestAnimationFrame` loop slowly auto-drags from 30% → 70% back and forth so the page never looks static.
4. Horizontal gallery: pinned section with GSAP ScrollTrigger, `xPercent: -100 * (slides - 1)` mapped to vertical scroll over `+=2000`. Slides are `flex-shrink-0 w-[420px]` cards with `aspect-[3/4]`.
5. Typography: hero headline `text-[clamp(3rem,7vw,7.5rem)] tracking-[-0.04em] leading-[0.95] font-medium`. Eyebrow `text-xs uppercase tracking-[0.2em] text-orange-500`. The accent orange is used on three things only — eyebrow, primary button, and underline-on-hover. Everywhere else is monochrome.
6. CTA buttons: `bg-orange-500 text-black rounded-full px-6 py-3 text-sm font-semibold` with a hover state that scales the button to `1.03` and pushes a `box-shadow: 0 0 0 6px rgba(255,107,2,0.18)` glow over `200ms ease-out`.

WHY IT WORKS: The asymmetric 60/40 hero with stacked tactile images is the exact opposite of the "centered headline + product screenshot" pattern that has saturated AI SaaS marketing. The before/after slider does the heavy lifting of "what does this product actually produce" without forcing a video click. The single warm accent on an almost-black background is the highest-contrast color choice possible without being aggressive.

GOOD / BAD:
- GOOD: The before/after slider with auto-idle drift is genuinely clever — it sells the product while idle without feeling like an ad.
- BAD: Webflow's animation runtime adds ~120KB of JS for what could be 30 lines of GSAP, and the horizontal-scroll gallery has no progress indicator, which leaves users uncertain about how long the section is.

CLEAN APPLICATION: Apply this to Clean's "Before / After" section on the landing page, comparing a raw agent transcript (massive token count, full JSON tool schemas) to Clean-processed (TOON-compressed, ~75% smaller). Build a draggable compare slider with the same idle auto-drift behavior. Use Clean's accent only on the eyebrow, the compare knob, and the CTA underneath. This is the cleanest single-screen demonstration of Clean's core value prop — "your context costs less" — and Fourmula's before/after pattern is the template.

## BUILD DIRECTIVE

TARGET: Clean landing page hero + immediately-following "How it works" / product-demo section at `tryclean.ai`.

APPROACH: Rebuild the above-the-fold and the first scroll section to combine three patterns from today's scouting. (a) From Cursor: replace the abstract proxy diagram with a real, high-fidelity Clean dashboard screenshot inside a dark glass-edged window chrome (`rounded-2xl border border-white/10`, double-shadow, 6° pointer-parallax tilt). (b) From KVS Studio: render the agent → proxy → tool topology as a fixed-position WebGL canvas sitting behind the dashboard window — pulse intensity on edges proportional to scroll velocity, single Clean accent color on the proxy node only. (c) From Enerblock: in the section immediately below the hero, build a pinned-scrub sequence that walks the user through one request's lifecycle (request in → MCP intercept → TOON compression → tool call → response), with a thin top progress bar and clip-path-wipe captions per stage. Color system stays near-black `#0A0A0A` with a single Clean accent; type is Inter Variable + a single display serif for the headline at `clamp(2.5rem, 6vw, 6.5rem)` with `tracking-tight, leading-[0.92]`. Mobile drops the WebGL canvas, drops the pinned scrub, and ships a static stacked variant.

STACK:
- `next@15.x` (App Router, RSC)
- `tailwindcss@3.4+`
- `framer-motion@11.x` (hero parallax + section reveals)
- `gsap@3.12+` with `ScrollTrigger` + `ScrollSmoother` (pinned scrub — requires GSAP Business/Premium)
- `@studio-freight/lenis@1.x`
- `three@0.165+` + `vite-plugin-glsl` for the topology canvas
- `@react-three/fiber@8.x` + `@react-three/drei@9.x` if going R3F instead of raw three
- `lucide-react` for iconography
- `shadcn/ui` primitives for any nav/buttons (already in Clean's stack — keep it)
- Inter Variable (self-hosted woff2) + a display serif for the H1 only

REFERENCE URLS:
- https://cursor.com/ — chrome, parallax, screenshot-first hierarchy
- https://www.kvs.services/ — fixed canvas + DOM slot pattern, scroll-velocity-driven shader
- https://enerblock.net/en/ — pinned scrub with stage progress bar and clip-path captions

KEY CONSTRAINTS: Clean is developer infrastructure. The result has to read as "fast, technically credible, low-level" — not "creative agency." Specifically AVOID: (1) cute illustrations, hand-drawn arrows, or gradient-blob backgrounds — Clean is plumbing, not a wellness app. (2) Three.js scenes that are decorative rather than diagrammatic — the topology canvas must literally represent the product, not a metaphor for it. (3) Long copy in the hero — the headline does one job, the screenshot does the second, no third element. (4) Any animation longer than 1.2s on a reveal — devs scroll fast and will leave. (5) Custom cursor effects, magnetic buttons, or 3D-text gimmicks — those signal "site, not product." (6) Webflow runtime — the entire stack is React/Next so animations stay codified and the production bundle stays under control. Anything that doesn't make a developer think "I trust this with my agent's token budget" gets cut.
