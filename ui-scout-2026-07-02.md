## Wembi — https://www.awwwards.com/sites/wembi (live: wembi.com)

WHAT IT DOES: Single-page Nuxt marketing site for a "virtual twin" performance product. The layout is a stack of full-viewport sections where each section pins on scroll and morphs its centered video/canvas element into the next via ScrollTrigger pinning + a masked video-swap technique — the current section's `<video>` element is CSS-clipped to an SVG path that animates its `d` attribute as the user scrolls, and the next section's video fades through the same clip region. Navigation is a persistent 12px monospaced top bar with a magnetized cursor-follower dot that snaps to nav items on hover (transform origin-lerped at ~0.12 ease). The color story is exactly two hues — a bio-green `#beff8b` and a cool-gray `#9dbbc5` — layered on off-white, with all typography set in a wide-set neo-grotesk.

PACKAGES:
- `nuxt@3.13+`
- `vue@3.4+`
- `gsap@3.12` + `@gsap/react` (ScrollTrigger, ScrollSmoother, SplitText)
- `lenis@1.1` for inertial scroll (piped into ScrollTrigger via `scrollerProxy`)
- `three@0.167` + `@react-three/fiber@8.17` (only used for the hero orb — not the whole site)
- `craft-cms` headless backend, consumed via GraphQL
- `sass@1.77`, PostCSS with `postcss-preset-env`
- Typeface: `PP Neue Montreal` (Pangram Pangram) + `JetBrains Mono` for micro-labels
- `sanity-plugin-media` for the video CDN (mux)

IMPLEMENTATION:
1. Root layout: `<AppShell>` → `<NavBar/>` (fixed, `mix-blend-mode: difference` so it inverts over dark hero) → `<main>` → `<Footer/>`.
2. Each section is a `<Section pin={true} height="200vh">` — the doubled height gives ScrollTrigger a scrub range while the sticky child renders one viewport tall.
3. Pin config: `ScrollTrigger.create({ trigger: section, start: "top top", end: "+=100%", pin: true, pinSpacing: false, scrub: 0.8 })`.
4. Video morph: each section holds an `<svg><clipPath id="morphN"><path d="…"/></clipPath></svg>` and the video is `clip-path: url(#morphN)`. Animate the `path`'s `d` attribute via `gsap.to(pathEl, { attr: { d: nextD }, ease: "power2.inOut", duration: 1, scrollTrigger:{scrub:1} })`. Interpolate between paths with `flubber` if morphs differ in vertex count.
5. Cursor follower: `useMouse()` composable + `gsap.quickTo(dot, "x", { duration: 0.35, ease:"power3" })`. On nav-item hover, snap the dot's width to the target's `getBoundingClientRect()` and add `mix-blend-mode: difference`.
6. Colors as CSS custom properties: `--bio:#beff8b; --cool:#9dbbc5; --ink:#0a0a0a; --paper:#f7f7f5;`. Use these only — no gradients, no tints.
7. Typography scale: hero clamp is `clamp(72px, 9vw, 168px)` at `line-height:0.88, letter-spacing:-0.03em, font-weight:500`. Body is `18px/1.5`. Micro-labels: `11px/1 mono, uppercase, letter-spacing:0.12em`.
8. Performance: `<video>` uses `preload="metadata"`, `playsinline`, `muted`, and swaps `src` via IntersectionObserver 200vh before entry. Serve H.265 + AV1 via Mux with `videoBitrate=1400k`.

WHY IT WORKS: Two-color palettes force hierarchy to be earned by typography and motion, not by chromatic noise — the eye reads shape and rhythm. The morphing clipPath creates continuity between narrative beats, so the user never feels a "cut" — one of the strongest tools for turning a marketing site into a demo. And the mono micro-labels signal engineering rigor without shouting; they're the visual equivalent of a service tag on hardware.

GOOD / BAD:
- GOOD: The morph choreography converts scroll-fatigue into a single continuous 40-second story; conversion-optimized without feeling salesy.
- BAD: Pinning + video morphs = ~140MB of buffered video on desktop first-paint. Mobile falls back to static images with a subtle Ken Burns — respect that fallback or you'll ship a 4G-hostile page.

CLEAN APPLICATION: Apply this to Clean's landing page hero-through-features flow. Rebuild the top 4 sections (hero → "the problem" → "the proxy" → "TOON format") as pinned viewports, and morph a single centered SVG (an abstract "context packet" — a rounded rectangle with data-line internals) across all four beats. Section 1: packet is bloated and jittery with a fake token counter incrementing at 40/frame. Section 2: packet compresses along one axis while the counter falls 4x. Section 3: proxy lines route around it. Section 4: packet re-emerges as a taut TOON syntax block. Use `#beff8b` as Clean's semantic-green accent, keep the rest black-on-off-white. Result: the visitor scrolls once and has understood the product.

## NRG | Build Your Data Center — https://business.nrg.com/campaigns/build-your-data-center/

WHAT IT DOES: Configurable-scene landing page built by Rogue Studio. Center of the viewport holds a live 3D isometric data-center rig; a right-rail step-list ("Site → Power → Cooling → Network → Deploy") drives camera position, environment map, and part visibility. Selecting a step animates a `THREE.OrthographicCamera` along a pre-baked spline (`CatmullRomCurve3`), swaps a `<Sky>` HDRI for a step-specific one, and reveals sub-modules with an emissive fade. Score of 7.37 on Awwwards, palette locked to yellow `#FFC20E` on near-black `#261D26` — enterprise-industrial vocabulary rendered as arcade.

PACKAGES:
- `next@14.2` (app router)
- `react@18.3`
- `@react-three/fiber@8.17`, `@react-three/drei@9.108` (`useGLTF`, `Environment`, `MeshTransmissionMaterial`, `Center`, `Bounds`)
- `three@0.167`
- `framer-motion@11.3` for HUD/2D overlays
- `zustand@4.5` for step state
- `leva@0.9` (dev-only for tuning)
- `tailwindcss@3.4`
- Font: `Founders Grotesk X-Condensed` for display, `Söhne Mono` for the step-list

IMPLEMENTATION:
1. Split viewport CSS grid: `grid-template-columns: 1fr 380px` desktop; single column with 3D on top, list under, mobile.
2. `<Canvas orthographic camera={{ zoom: 90, position: [10,10,10] }}>` with `dpr={[1,1.5]}`, `gl={{ antialias:true, powerPreference:'high-performance' }}`.
3. Preload the assembly: one `.glb` (Draco-compressed, ~2.3MB) with 5 named nodes: `site`, `power`, `cooling`, `network`, `compute`. Use `useGLTF.preload('/assets/rig.glb')` in the root.
4. Step state (zustand): `{ step: 0..4, prevStep, transitioning }`. On step change, drive camera via `useFrame`: `camera.position.lerp(cameraPositions[step], 0.06)`; `camera.lookAt(lookAtTargets[step])`. Environment cross-fade: two `<Environment>` components, animate their `intensity` prop with `motion` values.
5. Part visibility: each node has a `material.opacity` and `material.emissiveIntensity`; use `useSpring` from `@react-spring/three` to animate them with `{ tension: 120, friction: 26 }`.
6. HUD: right rail is a `<motion.ol>` where each `<li>` has a `layout` prop; active item expands to reveal a 60-word description with a staggered word reveal (`stagger: 0.02, y: 8 → 0`).
7. Micro-signature: yellow `#FFC20E` is used ONLY on: the active step number, the emissive edge of the active 3D part, and the CTA button. Nothing else is yellow. This is what makes the palette legible.
8. Performance: cap `<Canvas>` to `frameloop='demand'` during idle; call `invalidate()` on every state change. Pixel-ratio lock at 1.5 max on retina.

WHY IT WORKS: The site turns an enterprise sales page into a 90-second tour by binding scroll/click to a physical model — the user builds understanding by building the model. Isometric ortho projection eliminates the "which perspective is real?" cognitive load, and the yellow-on-black restriction gives every emphasized element the same semantic weight ("this is what's important right now").

GOOD / BAD:
- GOOD: Ortho + monochrome-plus-one-accent is the cleanest possible way to render technical hardware. Every part is legible even at 100% zoom.
- BAD: The step-list is keyboard-navigable but not screen-reader-annotated; the 3D scene is `aria-hidden` but has no text-mode alternative. Enterprise buyers with accessibility mandates will bounce.

CLEAN APPLICATION: Apply this to Clean's product demo section — the surface between the hero and the pricing table. Build "Build Your Context Layer" as a right-rail step-list: `Connect → Route → Compress → Serve → Observe`. Left side: an isometric R3F scene of a "context pipeline" — five extruded slabs (Data Sources → MCP Proxy → TOON Encoder → Agent → Telemetry) arranged in a shallow S-curve. Each step lights one slab in Clean's accent green `#beff8b`, animates a packet along the pipe, and reveals a live JSON→TOON diff in an overlay. Camera pans along `CatmullRomCurve3`; env swaps between "cold data center" HDRI and "warm agent runtime" HDRI. This gives the visitor a mental model of what Clean sits between — the whole positioning problem in a single interactive.

## Granyon — https://www.granyon.com/

WHAT IT DOES: Copenhagen/DC studio site, Awwwards Honorable Mention + CSSDA SOTD (2026-07-01). Built entirely in Webflow with hand-written GSAP interactions layered on top. The signature move: every project card is a bento-tile whose thumbnail image is `object-fit: cover` inside a `clip-path: inset(0 round 24px)` that expands to `inset(-2vh -2vw round 0px)` on hover — the image "breaks out" of its tile. Case study opens as a full-page overlay with a shared-element transition on the thumbnail (using Flip plugin) — the tile grows into the hero of the next page without a route change. Typography is mixed: a wide-serif display (`Reckless Neue`) paired with a tight-tracked sans (`Söhne`) — the tension is the identity.

PACKAGES:
- `webflow` (native runtime)
- `gsap@3.12` + `Flip` + `ScrollTrigger` + `SplitText`
- `lenis@1.1`
- `barba.js@2.10` for page transitions (or Webflow's native transitions with a custom script)
- Fonts: `Reckless Neue` (Displaay), `Söhne` (Klim)
- `swiper@11` for the horizontal case-study carousel
- Optionally `matter-js@0.20` for the falling-tag interaction in the About section

IMPLEMENTATION:
1. Bento grid: CSS grid, `grid-template-columns: repeat(12, 1fr)`, tiles span variable columns (`col-span: 4|6|8|12`) at gap `12px`. Each tile is `aspect-ratio: 4/3` or `1/1`.
2. Tile hover: `gsap.to(thumb, { clipPath: "inset(-2vh -2vw round 0px)", duration: 0.7, ease: "expo.out" })` + `gsap.to(tile, { yPercent: -2 })`. On mouseleave, snap back with `ease: "power4.inOut"` at 0.55s.
3. Shared-element transition (Flip): on tile click, `const state = Flip.getState(thumb)`; navigate/render the case-study; then `Flip.from(state, { duration: 0.9, ease: "expo.inOut", absolute: true, scale: true })`. The image morphs from the grid cell to the case-study hero seamlessly.
4. Typography contrast: display uses `Reckless Neue Semibold, font-size: clamp(56px, 7vw, 140px), line-height: 0.95, letter-spacing: -0.02em`. Body uses `Söhne, 16px/1.55, letter-spacing: 0`. Small caps for section labels: `Söhne Kraftig, 11px, tracking: 0.14em, uppercase`.
5. Palette: cream `#f2efe9`, ink `#0b0b0b`, accent `#ff4d1c`. Accent shows up on links, cursor states, and one word per hero — treat it as scarce.
6. Cursor: `mix-blend-mode: difference` circle, `24px` radius default, expands to `72px` with content-label "VIEW" on tile hover.
7. Scroll pacing: Lenis at `duration: 1.4, easing: (t)=> Math.min(1, 1.001 - Math.pow(2, -10*t))` — the exponential curve makes long pages feel less endless.
8. Loader: on first paint, an SVG mask with the logo mark expands from 0 → viewport width over 900ms `power4.inOut`, revealing the hero.

WHY IT WORKS: The bento layout gives content hierarchy without headings — tile size does the work of h1/h2/h3. The Flip-based shared-element transition eliminates the "page change" jolt, so the site feels like a single application. The Reckless/Söhne pairing is the current studio-website meta because the friction between wide serif and tight sans reads as taste — it says "we chose this," not "we defaulted to Inter."

GOOD / BAD:
- GOOD: The Flip transition is the highest ROI motion technique on the web right now — 40 lines of code, transforms your site from "pages" to "spaces."
- BAD: Cursor-hijacked hover states (custom cursor, no browser default) break trackpad users' spatial memory and cause accessibility complaints. Provide a `prefers-reduced-motion` bypass.

CLEAN APPLICATION: Apply this to Clean's docs/showcase surface — a page like `/showcase` that displays real customer setups (or open-source demos while the customer list is small). Build a bento grid of "context recipes" — each tile is a real MCP-proxy config with a live TOON preview thumb. On hover, the tile clips out of its cell; on click, use GSAP Flip to expand into a full-page "recipe" showing the raw config, the token-count savings, and the code snippet to copy. Pair `Reckless Neue` for section headers with `JetBrains Mono` for the config code — the tension gives the docs a magazine feel instead of a wiki feel.

## Shader.se — https://shader.se (Codrops case study: https://tympanus.net/codrops/2026/05/19/80s-business-tech-seamless-scene-transitions-inside-shader-ses-scroll-driven-webgpu-pipeline/)

WHAT IT DOES: A portfolio site built as a single WebGPU-rendered R3F canvas that never unmounts. Scroll drives a scrubbable "camera + scene switch" pipeline: instead of routing between pages, the entire site is one Three.js scene graph with 6 named "chapters," and scrolling advances a global `progress` uniform (0..1) that both moves the camera along a spline and cross-fades chapter meshes via a full-screen post-processing pass. The transitions read as gooey/wet — a fluid distortion shader morphs one chapter into the next using a value-noise displacement on a screen-space quad. The aesthetic is intentionally 1980s-business-tech: rendered chrome text, LCD-orange accent (`#ff7a00`), CRT bloom, and gridded floors.

PACKAGES:
- `three@0.167` with WebGPU renderer (`three/examples/jsm/renderers/webgpu/WebGPURenderer.js`)
- `@react-three/fiber@8.17` (canary/`experimental` for WebGPU)
- `@react-three/drei@9.108`
- `three-stdlib@2.32`
- `tsl` (Three Shading Language, built into three r165+)
- `@react-three/postprocessing@2.16` (bloom, chromatic aberration)
- `leva@0.9` for dev tuning
- `next@14.2` + `react@18.3`
- `lenis@1.1`
- Fonts: `PP Editorial Old` for chrome-serif hero, `Berkeley Mono` for terminal-style labels

IMPLEMENTATION:
1. Single `<Canvas gl={async (props) => { const r = new WebGPURenderer(props); await r.init(); return r; }} frameloop="always" dpr={[1,1.5]}>`. Fallback path: if WebGPU init fails, mount a WebGL renderer and skip the distortion shader.
2. Root state: `useProgress` (Zustand) — one number, updated by Lenis on scroll. Map chapters to progress ranges: chapter 0 = `[0, 0.16]`, chapter 1 = `[0.16, 0.33]`, etc.
3. Camera: `THREE.PerspectiveCamera(28, aspect, 0.1, 200)`. Position and rotation are sampled from `CatmullRomCurve3` and `QuaternionKeyframeTrack` per-frame using the global progress. `useFrame(({clock})=> { camera.position.copy(spline.getPointAt(progress)); camera.quaternion.slerp(rotationAt(progress), 0.15) })`.
4. Post-processing distortion (the "seamless transition"): render the scene to an offscreen `WebGPURenderTarget`, then a full-screen quad samples it with a TSL fragment that displaces UVs using a fBm noise `displacement = fbm(uv * 4.0 + time * 0.1) * transitionActive`. `transitionActive` peaks in the ±0.04 range around each chapter boundary.
5. Chapter meshes: each chapter's group has `visible = false` outside its range and fades in via `material.opacity` driven by a bell curve `Math.max(0, 1 - Math.abs((progress - center) / halfWidth) ** 2)`.
6. Text: SDF-rendered via `troika-three-text@0.49` for the display headline; DOM overlay for body copy (positioned absolute, sync'd to progress via CSS custom properties).
7. Performance: budget of ~2M tris total, all instanced. `renderer.setPixelRatio(Math.min(devicePixelRatio, 1.5))`. Reduce post-pass work by only running the distortion shader when `Math.abs(progress - nearestBoundary) < 0.06`.

WHY IT WORKS: The "one persistent scene" architecture eliminates the mental cost of navigation — the visitor never wonders "did the page load?" because there is no page. WebGPU + fBm noise gives a specific texture (soft, atmospheric) that WebGL noise shaders can approximate but not match cheaply. The 80s-business-tech vocabulary is a deliberate anti-trend move: while every other site chases Swiss minimalism, this one owns nostalgia as a differentiator, and the specificity is what makes it memorable.

GOOD / BAD:
- GOOD: Sets a new bar for "single-scene sites" — 60fps on M-series MacBooks, meaningful visual identity, and legitimately new tech (WebGPU + TSL).
- BAD: Locks out ~25% of users whose browsers/GPUs don't support WebGPU well (older Android, some Linux Chromium builds). Fallback path exists but is dramatically degraded.

CLEAN APPLICATION: Apply this to Clean's landing hero — but don't try to be nostalgic; use the persistent-canvas pattern instead. Build a single R3F canvas that holds an abstract "context flow" — a stream of small illuminated cubes flowing along a spline that arcs across the full viewport. The stream reacts to scroll: near the hero, cubes are chaotic and dense (representing raw context bloat); scrolling into the "TOON format" section causes the cubes to snap into a lattice; scrolling into "Proxy" bends the stream through a rendered gateway node. All one canvas, no page reload, no route change. Use Clean's `#beff8b` for the illuminated cubes on a near-black canvas — the two-color restraint keeps it product-focused, not portfolio-flashy.

## OpenCode.ai — https://opencode.ai

WHAT IT DOES: Marketing site for the open-source AI coding agent (7.5M MAU, 160K+ GitHub stars). The identity is aggressive typographic restraint: the entire page — every headline, body copy, footer legal, button label — is rendered in Berkeley Mono on a warm cream canvas (`#fdfcfc`) with nearly-black ink (`#201d1d`). The single "visual moment" is one hero card that mocks up the TUI itself: black background, ASCII pipes (`│ ├ └`), a block-pixel ASCII wordmark, and monospaced terminal output. All interactive elements are 4px-radius rectangles. Bullets are ASCII `[+]` / `[-]`. The site reads like a well-typeset man page — and that IS the brand.

PACKAGES:
- `astro@4.16` (or `next@14.2` — both viable; Astro leans into the static-doc feel)
- Font: `Berkeley Mono` (US Graphics Company, paid license required — critical)
- `tailwindcss@3.4` with a heavily pruned config (no fluid typography, no rounded utilities above `rounded-sm`)
- `shiki@1.24` for code highlighting on any embedded snippets
- No animation library. No motion library. Intentional.
- `mdx-bundler@10` for docs pages that share the marketing chrome

IMPLEMENTATION:
1. Tailwind config: `fontFamily: { sans: ['"Berkeley Mono"', 'monospace'], mono: ['"Berkeley Mono"', 'monospace'] }` — both are the same. Force it via `!font-mono` where necessary.
2. Palette: `--paper:#fdfcfc; --ink:#201d1d; --hairline:#e5e2dc; --terminal-bg:#0d0d0d; --terminal-ink:#c9c9c9;`. That's it. No accent color. The accent is the ASCII bracket.
3. Layout: single column, `max-width: 720px`, centered, generous vertical rhythm (`space-y-16`). Every section is a hairline-bordered box (`border-t border-hairline`).
4. Typography scale — everything in `Berkeley Mono`:
   - Hero: `text-[38px] leading-[1.2] tracking-[-0.01em] font-medium`
   - Section headers: `text-[24px]`, prefixed with `## ` (literal hash-hash-space)
   - Body: `text-[16px] leading-[1.6]`
   - Meta / footer: `text-[14px] text-ink/70`
5. Bullets: replace the CSS list-marker with `content: "[+] "` for closed items, `[-] ` for open. Never use `•`.
6. Hero terminal card: a `<pre>` with `bg-terminal-bg`, `text-terminal-ink`, `font-mono`, padding `p-6`, `rounded-sm`. Inside, an ASCII block for the wordmark (12 lines × ~40 chars) — hand-crafted, not generated. Below it, a simulated `$ opencode --help` output.
7. Buttons: `border border-ink px-4 py-2 rounded-sm hover:bg-ink hover:text-paper transition-none`. No transition. No motion.
8. Docs pages inherit the same shell — the marketing site literally becomes the docs when you click "Docs."

WHY IT WORKS: Radical typographic constraint forces the reader to focus on content — there is no chrome to notice, so the value prop lands harder. For a developer tool, "the marketing page is a man page" is a deep congruence signal: the product is going to feel exactly like this. Sparseness reads as confidence — no marketing team would ship this if the product were weak, so seeing it live is itself a proof point.

GOOD / BAD:
- GOOD: Zero motion, zero JS on the marketing page, sub-100ms LCP, perfect Lighthouse scores. Every dev bookmarks it.
- BAD: Non-developer audiences (finance/procurement gatekeepers evaluating tooling) can't parse this. If your buyer isn't the same person as the user, you'll lose deals to prettier sites.

CLEAN APPLICATION: Apply this to Clean's docs surface (`docs.tryclean.ai` or `/docs`). The docs, quickstart, and API reference should adopt the "man page" aesthetic wholesale — Berkeley Mono top-to-bottom, `[+]`/`[-]` bullets, hairline dividers, hero card that mocks up an actual `clean proxy start` TUI with ASCII pipes, cream canvas. Do NOT apply this to the main landing page — that page has to sell to buyers, not just engineers. But every developer who lands on `/docs` will see the same aesthetic they see on OpenCode, Terraform, and Zed docs — the pattern is now the "we're serious infra" signal.

## BUILD DIRECTIVE

TARGET: Clean landing page — hero-through-features scroll narrative (top ~4 viewport-heights). This is Clean's single most consequential surface right now, and today's scouting pulled three techniques that stack cleanly on it.

APPROACH: Build a single persistent R3F canvas that spans the top 4 sections of the landing page (Wembi's scroll-pinned narrative + Shader.se's single-scene architecture). In the canvas, render an abstract "context stream" — a spline arcing across the viewport with illuminated cubes flowing along it, representing tokens/context. Section 1 (hero): the stream is chaotic, dense, jittering — the "raw context bloat" state, with a large mono counter showing tokens climbing at 40/frame. Section 2 ("The problem"): the stream tangles; a subtle fBm distortion shader wobbles it. Section 3 ("MCP Proxy"): the stream routes through a rendered gateway node with an animated GSAP Flip transition wiping in a code snippet overlay. Section 4 ("TOON format"): the cubes snap into a taut lattice, the counter drops 4x, TOON syntax renders as SDF text via troika. Everything is scroll-scrubbed via Lenis + ScrollTrigger — no click required. Below section 4, break the R3F canvas cleanly and continue with static Webflow-style bento tiles (Granyon pattern) for use cases. This gives Clean a landing top-half that reads as "serious infrastructure with a story" and a bottom-half that reads as "shippable product with real customers."

STACK:
- `next@14.2` (app router)
- `react@18.3`, `react-dom@18.3`
- `@react-three/fiber@8.17`, `@react-three/drei@9.108`, `@react-three/postprocessing@2.16`
- `three@0.167` (WebGL renderer — skip WebGPU for wider compatibility; the visual budget doesn't need it)
- `gsap@3.12` (`ScrollTrigger`, `Flip`, `SplitText`) + `@gsap/react`
- `lenis@1.1`
- `zustand@4.5` for scroll progress + section state
- `troika-three-text@0.49` for SDF text in the canvas
- `framer-motion@11.3` for HUD overlays only (NOT the canvas)
- `tailwindcss@3.4`
- Fonts: `PP Neue Montreal` (display) + `JetBrains Mono` (mono/labels)
- Palette CSS vars: `--ink:#0a0a0a`, `--paper:#f7f7f5`, `--accent:#beff8b`

REFERENCE URLS:
- https://wembi.com — for the pinned scroll narrative and 2-color discipline
- https://shader.se — for the persistent-canvas + scroll-scrubbed camera pattern
- https://www.granyon.com — for the bento continuation below the fold, Flip transitions on use-case tiles

KEY CONSTRAINTS:
- Clean is developer infrastructure, not a portfolio. Every motion must map to a product statement — no motion for motion's sake. If you can't answer "what does this animation say about our product?" in one sentence, cut it.
- Two colors only above the fold: ink + accent green `#beff8b` on paper. No gradients, no glows, no rainbow. Restraint = credibility.
- No proprietary display fonts on the actual page — PP Neue Montreal is fine, but keep body/labels in `JetBrains Mono` so dev visitors read "engineer" not "startup."
- WebGPU is not required and should be avoided today — WebGL renderer only. The visual budget (spline + ~200 instanced cubes + one shader) runs cleanly at 60fps on 5-year-old hardware.
- Do NOT use custom cursors on the marketing page. Provide `prefers-reduced-motion` bypass that renders a static hero image + text version of all four sections.
- Avoid: chrome text, CRT bloom, retro aesthetics, marquees, glassmorphism, gradient meshes, "AI product = purple gradient" clichés. Clean's advantage is looking like Cloudflare or Vercel circa 2022 — precise, monochromatic, engineered.
