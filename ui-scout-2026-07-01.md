## NRG | Build Your Data Center — https://business.nrg.com/campaigns/build-your-data-center/

WHAT IT DOES: A single-page interactive configurator that lets a prospect assemble a data-center site (site eval → power → cooling → completion) as a horizontally-progressing scroll narrative. Layout is a locked 100vh stage where a WebGL/canvas scene plus foreground SVG lockups translate on scroll-driven timelines; each "chapter" is a section-pinned segment with its own micro-interactions (hover-flip stat cards, isometric hardware rigs, cursor-reactive spotlight). Visual hierarchy is dominated by a saturated amber (#FFC20E) type layer floating over deep aubergine (#261D26), with the CTA persistently pinned in the top-right regardless of scroll depth.

PACKAGES:
- next@15 or astro@5
- gsap@3.13 + @gsap/react + ScrollTrigger + ScrollSmoother
- lenis@1.1 (buttery scroll for the pinned chapters)
- three@0.170 + @react-three/fiber + @react-three/drei (isometric server rack / cooling unit models)
- three-mesh-bvh (cursor raycasts against hardware without frame drops)
- @studio-freight/hamo (rAF orchestration)
- framer-motion@11 (foreground SVG lockups + UI microinteractions)
- tailwindcss@3.4 + tailwindcss-animate
- next/font/local for the display face (custom grotesk cut in the amber lockups)
- lucide-react (nav + toolbar)

IMPLEMENTATION:
1. Root layout: `<Lenis root>` wrapping a `<Scene>` (fixed inset-0 R3F canvas, `dpr={[1,1.75]}`, `gl={{ antialias:true, powerPreference:'high-performance' }}`) and a `<Story>` scroll container with 6 sections × 100vh each pinned via `ScrollTrigger.create({ pin:true, scrub:0.6, anticipatePin:1 })`.
2. Chapters: `Site → Power → Cooling → Network → Security → Handoff`. Each is a `<section data-chapter>` with `data-progress-var` values 0..1; write the scroll progress to a CSS var on `:root` (`--p-power`) and drive both SVG animations (`style="transform: translateX(calc(var(--p-power) * -60vw))"`) and R3F uniforms in one tick.
3. R3F scene: single OrthographicCamera at (30,30,30), lookAt origin. One `<Instances>` of low-poly server racks (~800 tris each), one `<mesh>` for the ground plane with a shader that renders an amber hex-grid whose alpha is driven by chapter progress. Rotate the camera 30° between chapters using `useFrame` + `damp3(cameraTarget)` with `THREE.MathUtils.damp` lambda 4.
4. Amber type layer: SVG lockups with `mask-image: linear-gradient(...)` reveal, `font-feature-settings:"ss01","cv11"`. Sizes: hero display `clamp(4rem,11vw,10.5rem)`, tracking `-0.03em`, line-height 0.9.
5. Cursor spotlight: `mix-blend-mode: overlay` div, size 520px, position via `pointermove` throttled to rAF; opacity ramps to 0 when hovering interactive elements (fallback to a native-cursor state).
6. Palette: `--ink:#261D26`, `--signal:#FFC20E`, `--bone:#F5F1EC`. All text is `--signal` on `--ink` except the last chapter which inverts to build a moment of contrast for the CTA.
7. Motion tokens: `--ease-out-expo: cubic-bezier(0.16,1,0.3,1)`, section entrances 900ms, hover 240ms, cursor lag 120ms. Never use ease-in on entrances.
8. Performance: preload the GLTFs with `useGLTF.preload`; suspend R3F while offscreen (`frameloop='demand'` and invalidate on scroll); reduce DPR on `matchMedia('(max-width:768px)')`.
9. Responsive: <900px collapses horizontal scroll into vertical, disables R3F scene, swaps to a Lottie sequence (`lottie-react`) driven by native scroll.
10. A11y: mirror every scroll-progress interaction with a `<nav>` chapter list of anchor links; respect `prefers-reduced-motion` by killing `scrub` and pinning.

WHY IT WORKS: The two-color palette forces every design decision to earn its place — the amber literally is the CTA everywhere, so the eye never has to hunt. Horizontal scroll-linked narrative maps directly to a mental model of building something in stages, which is Gestalt continuation put to work. The pinned camera on a moving scene creates parallax without the vertigo of full-scroll parallax, and the persistent CTA lockup solves the classic "immersive site with no conversion" problem.

GOOD / BAD: GOOD — the amber+aubergine restraint plus a single moving 3D system is textbook editorial rigor with real conversion intent. BAD — horizontal-on-scroll still murders trackpad-less desktop users and Safari iOS jank is real; the 7.37/10 Awwwards score reflects that.

CLEAN APPLICATION: Apply this to Clean's landing page hero + "how it works" section. Build a 4-chapter pinned scroll narrative — `Connect (MCP servers appear as nodes) → Route (edges light amber-equivalent, TOON tokens ripple through) → Compress (a bar chart of "before/after" tokens animates in beside the routing graph) → Deploy (the whole scene collapses into a single CLI/prompt panel showing latency saved)`. Use a single R3F scene with instanced MCP-node meshes rather than a static hero illustration. Persistent "Get access" CTA pinned top-right. Kill the horizontal scroll — Clean's buyers are developers on trackpads; use vertical pinning + scene translation on x-axis instead.

## Wembi — https://www.awwwards.com/sites/wembi

WHAT IT DOES: Minimal B2B SaaS site (July 1, 2026 Awwwards SOTD, Dev Award, 7.49/10, ET Studio × Sentempo, GSAP + Nuxt + Craft). Layout is a strict 12-col grid with a hero that uses a huge display-serif × mono-numeric combo, a slate-blue field (#9DBBC5) grounded by a single acid-mint (#BEFF8B) accent. Interaction model: GSAP SplitText character reveals synced to a horizontal-scroll case-study rail, sticky section headers that swap on intersection, and a magnetic cursor that snaps to CTAs.

PACKAGES:
- nuxt@3.13 (SSR/prerender for CMS content)
- @nuxt/content or craft-cms REST client
- gsap@3.13 + SplitText + ScrollTrigger + Draggable
- lenis@1.1
- @vueuse/core (intersection helpers)
- three@0.170 (only for the small ambient blob in the hero — optional)
- lil-gui removed in prod
- vue-lucide-icons
- Tailwind or UnoCSS + tailwind-merge

IMPLEMENTATION:
1. Grid: 12-col, 24px gutters, `max-w-[1440px]`, side padding `clamp(20px,4vw,64px)`. Everything vertically snaps to an 8px baseline.
2. Hero: type stack is a serif display (Editorial Old / GT Sectra alt) at `clamp(72px,12vw,220px)` with `line-height:0.86`, `letter-spacing:-0.04em`; secondary label uses `JetBrains Mono` at 14px uppercase, letter-spaced 0.12em. Numerals are tabular.
3. SplitText reveal: `SplitText.create(el, { type:'chars,words', mask:'chars' })`; animate `y:'110%'` → `0` with `duration:1.1, ease:'expo.out', stagger:{ amount:0.6, from:'start' }`, triggered on `ScrollTrigger { start:'top 80%' }`.
4. Case-study rail: horizontal Draggable strip inside a pinned section (900px tall), each card 620×420, `box-shadow: 0 30px 60px -30px rgba(38,29,38,0.35)`; drag inertia via `Draggable.create(el, { type:'x', inertia:true, edgeResistance:0.85 })`.
5. Magnetic cursor: on `pointermove` compute `dx = (mouseX - btnCenterX) * 0.35`, apply `gsap.to(btn, { x:dx, y:dy, duration:0.4, ease:'power3.out' })` when within 120px radius; reset on leave.
6. Section transitions: use `gsap.matchMedia` for breakpoints. Sticky headers with `position:sticky; top:24px` and a color swap via `ScrollTrigger { onEnter/onLeaveBack }` toggling CSS vars `--fg`, `--bg` between the two brand colors.
7. Ambient hero blob (optional): a single `<canvas>` with a fragment shader (SDF blob with two-octave FBM) — `THREE.ShaderMaterial` uniforms `uTime`, `uMouse`, `uScroll`. Keep it 40% opacity so text stays hero.
8. CMS: sections keyed off Craft entries — the site is content-shaped, so hardcode nothing that a marketer will need to change.
9. Performance: `nuxt.config.ts` `experimental: { payloadExtraction:true }`, prerender critical routes, `font-display:swap`, subset the display serif to the visible headline chars.
10. A11y: SplitText reveals include `aria-label` on the parent to preserve screen-reader text; magnetic cursor respects `prefers-reduced-motion`.

WHY IT WORKS: The palette is basically two swatches, so contrast ratios do all the work — mint accent is used exactly at moments of user action (CTAs, key nouns), which is Fitts' law + color-as-affordance. Editorial serif + mono numerics reads "confident but technical" without cliché mono-everywhere dev-tool voice. Horizontal case-rail rewards curiosity without demanding a full immersive commitment.

GOOD / BAD: GOOD — restraint. Two colors, two typefaces, one interaction pattern, world-class execution. BAD — the ambient blob shader is decorative and inflates the JS bundle; killable without cost.

CLEAN APPLICATION: Apply this to Clean's marketing site typography + case-study section. Adopt the serif-display × mono-numeric pattern for headlines (e.g., `Reduce agent token cost by <mono>62%</mono>`). Build the horizontal Draggable rail as a "who's building with Clean" strip surfacing 6 customer logos + one-line quote cards; ship immediately even before Clean has 6 customers by using dev-tool-native brands from Clean's early network. Steal the sticky-section color-flip for the pricing table divider — flips from ink to bone as the user reads down.

## Disco Dungeon — https://discodungeongame.com/

WHAT IT DOES: A promo site for a turn-based puzzle RPG that maps browser-scroll to a game-character exploration through a hand-illustrated dungeon (Framer Site of Month, June 2026, Benjamin Trigalou). Layout is a single vertical scroll surface where the hero is a fixed art frame; scroll progress moves a rigged 2D character along a spline path through parallax-layered rooms, revealing feature callouts as the character enters each. Interaction model: scroll-position = camera + character position + parallax offsets, all one variable.

PACKAGES:
- framer motion@11 (`useScroll`, `useTransform`, `useSpring`) if implementing in React; framer-native primitives if in Framer
- lottie-react OR rive-react (character rig — either is fine; Rive gives interactive state machines)
- gsap@3.13 + MotionPathPlugin (drive the character along an SVG path)
- lenis@1.1
- pixi.js@8 (optional) if scaling to hundreds of parallax sprites — otherwise plain CSS `will-change:transform` works
- svgo (build-time SVG optimization for the dungeon art)
- react-use-measure (responsive canvas sizing)

IMPLEMENTATION:
1. Layout: fixed 100vh art stage (`position:fixed; inset:0`) rendering the dungeon. Below it, a tall `<main>` that provides scroll length only — think `height: 600vh` for six rooms.
2. Character rig: single Rive `.riv` file with a state machine input `pathProgress` (0..1); on scroll, `useTransform(scrollYProgress, [0,1], [0,1])` feeds `rive.setInputByName('pathProgress', v)`. States: idle, walk, jump, celebrate (last room).
3. Path: hand-drawn `<path>` in an invisible SVG matched to the art. Use `MotionPathPlugin.getRelativePosition(path, progress)` to place the character; do NOT let GSAP scrub Rive directly — animate a plain object and push values to Rive in a single `useFrame`.
4. Parallax rooms: 4 depth layers per room, translate on `y` by `progress * -[80,50,30,15]px` respectively. Layers are SVGs, not PNGs — dungeon art is illustrated, SVG scales.
5. Feature callouts: absolutely-positioned cards keyed to `progress > 0.16`, `progress > 0.32`, etc. Fade in via `motion.div initial={{opacity:0,y:16}} whileInView={{opacity:1,y:0}} transition={{ duration:0.6, ease:[0.16,1,0.3,1] }}`.
6. Audio (optional): ambient dungeon loop via `<audio>` with a mute toggle sticky bottom-right; volume ramps with `progress`.
7. Lock scroll to segments: use `ScrollTrigger.create({ snap:{ snapTo:1/6, duration:0.4, ease:'power2.inOut' } })` so users always land in a room, never between.
8. Performance: rasterize distant parallax layers to `<canvas>` on iOS, keep near-field as SVG; preload the Rive file with `<link rel=preload as=fetch>`.
9. Cursor: replace with a small pixel-torch that lights a radial gradient over the fixed stage (`mask-image: radial-gradient(circle 200px at var(--mx) var(--my), black, transparent)`).
10. A11y: `prefers-reduced-motion` disables snap + character animation and turns the site into a vertical comic-strip of the 6 rooms with the same callouts.

WHY IT WORKS: Scroll-as-navigation-of-a-place is one of the strongest narrative devices on the web because it turns marketing copy into stops on a journey — every callout is *earned* by scrolling into its room. Rive state machines let the character *react* to scroll direction, so scrolling back rewinds animation semantically (turn around, walk left) — this is motion-as-language, not motion-as-decoration.

GOOD / BAD: GOOD — the character-rig-driven-by-scroll pattern is a genre-defining trick, works on any brand willing to commit to a mascot. BAD — this pattern lives or dies on illustration quality; without a real illustrator, the same technique produces a "sad Mario walks through PowerPoint" energy.

CLEAN APPLICATION: Apply this to Clean's onboarding flow / interactive product tour page. Replace the dungeon with a stylized "agent-runtime highway" — the character is a small agent icon that scrolls left-to-right along a pipeline (MCP servers as roadside stations, TOON compression as a checkpoint, target LLM as the destination). Each station triggers a feature callout. Use Rive so the agent can *animate* getting compressed at the TOON checkpoint and *animate* the token savings ledger updating. This is a stronger product explainer than any text.

## Melnūdens Coffee — https://melnudens.coffee

WHAT IT DOES: Framer runner-up (June 2026, Henriks Porciks) for a specialty coffee brand. Layout is editorial: full-bleed hero with a single black-serif wordmark oversized to bleed off the top, a warm ochre/cream palette (approx `#E8DFCE` bone, `#2C2620` roast, `#B77038` amber), scroll-triggered image reveals using clip-path animations, and a horizontally-scrolling product shelf. Interaction model is quiet — everything is entrance-animated with generous timing, no interactive gimmicks.

PACKAGES:
- framer (native site) OR Next.js + framer-motion equivalent
- gsap@3.13 + ScrollTrigger (clip-path scrubs)
- lenis@1.1
- @vercel/og or satori for programmatic OG images per product
- next/image with `sizes="(max-width:768px) 100vw, 50vw"`, AVIF
- next/font/google — `Fraunces` variable serif + `Söhne Mono` OR `JetBrains Mono` for weights
- embla-carousel-react (product shelf)
- shopify buy-sdk if commerce

IMPLEMENTATION:
1. Type system: single variable serif (Fraunces) doing everything from `clamp(88px,14vw,320px)` display down to 14px small caps for eyebrows. Numerals set in mono for prices and dosage grams.
2. Hero: wordmark on `translateY(-8%)` so it bleeds off the top; `mix-blend-mode:multiply` on a hero photograph so the type appears to sit *in* the beans. Use a subtle `filter: contrast(1.05)` on the photo for print-editorial feel.
3. Clip-path reveals: `clip-path: inset(100% 0 0 0)` → `inset(0)` scrubbed to scroll via `ScrollTrigger { scrub:0.8, start:'top 85%', end:'top 40%' }`. Every hero-scale image uses this — never fades.
4. Product shelf: Embla with `slidesToScroll:1`, `align:'start'`, `dragFree:true`. Each card is a photograph + tiny label (single line) + tabular price. On hover, price swaps to "Add" with a 200ms crossfade.
5. Baseline grid: everything on a 4pt grid; vertical rhythm is 32px between paragraphs, 96px between sections, 176px between chapters.
6. Palette: warm neutrals only, one saturated accent (`#B77038`) used exclusively on price and CTA hover. This restraint is the whole design.
7. Micro-interactions: image hover shifts saturation via `filter:saturate(1.15)` over 400ms; anchor links have an underline that draws left-to-right via `background-size` animation.
8. Ecom: minimal drawer cart, no full-page checkout redirect; use Shopify buy-sdk to keep the vibe intact.
9. Performance: hero image ≤180KB AVIF, lazy-load below fold, use `content-visibility:auto` on product rows to skip offscreen paint.
10. A11y: warm neutral palette risks contrast; enforce a WCAG-AA lint step in CI and ship a "high-contrast mode" that swaps `--fg` to `#0E0906` and `--bg` to `#F5F1EC`.

WHY IT WORKS: A specialty product needs a specialty aesthetic — this site earns trust by looking like a magazine, not a Shopify template. The one-typeface, three-color, no-effect discipline is the most underused pattern on the web because it demands art direction. Clip-path reveals feel like page turns rather than modal transitions, matching the tactile expectation of print.

GOOD / BAD: GOOD — the discipline. Zero cheap tricks, every choice defensible. BAD — this aesthetic *only* works with photography this good; try to ship it with stock photos and it becomes a stationery brand.

CLEAN APPLICATION: Apply this to Clean's docs landing page and blog. Adopt the single-serif type system for long-form (Fraunces or similar) with mono only for code and numerals. Ship the clip-path scroll-reveal for every hero code screenshot in a blog post. This gives Clean's technical writing a "The Verge × Stripe blog" gravitas that separates it from the animated-gradient-on-marketing-site sameness of every dev tool.

## Scroll-Revealed WebGL Gallery (Codrops) — https://tympanus.net/codrops/2026/02/02/building-a-scroll-revealed-webgl-gallery-with-gsap-three-js-astro-and-barba-js/

WHAT IT DOES: A reference technical pattern (Codrops, Feb 2026) — a multi-page image gallery where each image is rendered as a Three.js plane with a custom shader that runs a reveal-mask driven by scroll position. Barba.js handles route transitions without full reloads so the WebGL canvas persists across pages; GSAP scrubs shader uniforms. Layout is a bento-grid gallery on the index and a full-bleed detail view on click, with the same textures moving between the two.

PACKAGES:
- astro@5 (islands architecture — WebGL is one persistent island)
- three@0.170
- gsap@3.13 + ScrollTrigger + Flip (the reveal + layout morph)
- @barba/core + @barba/prefetch (SPA transitions)
- lenis@1.1
- ogl OR three — Codrops uses three, but ogl is lighter if bundle size matters
- glslify or vite-plugin-glsl (author `.glsl` files, import as strings)
- tailwindcss@3.4 (utility CSS layer)

IMPLEMENTATION:
1. Persistent canvas: single `<canvas>` in Astro layout, marked `client:load`. It stays mounted across Barba transitions. Camera is OrthographicCamera sized to viewport.
2. Grid → texture mapping: each DOM `<img>` in the grid has `data-webgl` and a ref to a THREE.Mesh (`PlaneGeometry` matching its bounding rect in world units). On scroll, sync mesh position to DOM position via `getBoundingClientRect` in a rAF loop.
3. Reveal shader: fragment shader takes `uProgress` (0..1), a noise texture, and does `float mask = step(uProgress, texture(noise, vUv).r); discard if mask==0;` giving an organic reveal. Author in `.glsl`, hot-reload with vite-plugin-glsl.
4. Scroll drive: for each image, `ScrollTrigger.create({ trigger: imgEl, start:'top 80%', end:'top 20%', scrub:0.5, onUpdate: self => mesh.material.uniforms.uProgress.value = self.progress })`.
5. Route transition: on click, Barba pushes new route; GSAP Flip captures grid layout, morphs the clicked mesh to full-bleed via `Flip.from(state, { duration:0.9, ease:'expo.inOut' })` while the shader flips to a full-alpha state.
6. Post-processing: `EffectComposer` + a chromatic-aberration pass driven by scroll velocity (measure velocity in Lenis `on('scroll')` callback, damp toward 0).
7. Performance: only allocate textures for meshes within `document.documentElement.scrollHeight ± 2 * innerHeight` of the viewport; dispose the rest. Use `KTX2Loader` for compressed textures.
8. Responsive: below 768px, drop the shader reveal and use CSS `clip-path` on the DOM `<img>` directly — WebGL cost isn't worth it on mid-range Android.
9. A11y: images ship as real DOM `<img>` with `alt`; WebGL layer is `aria-hidden` and purely visual.
10. Debug: keep a `?debug=1` flag that renders wireframe planes so you can verify DOM↔WebGL sync.

WHY IT WORKS: Persistent-canvas + Barba transitions eliminate the single ugliest UX bug in immersive sites — the FOUC when route changes reload WebGL. Shader-driven reveals feel materially different from CSS transitions because they respect image content (noise mask reveals through the pixels, not around them), which brains read as depth. The whole pattern is a template for "how to make a product feel expensive without a 3D scene."

GOOD / BAD: GOOD — a genuine best-practice pattern for image-heavy portfolios, docs, and product pages. Cleanly separates DOM (semantics) from WebGL (surface). BAD — Barba adds routing complexity that most teams don't need; if you're not doing route transitions, drop it and keep just the persistent canvas.

CLEAN APPLICATION: Apply this to Clean's docs and blog. Each code screenshot in a docs page is rendered as a WebGL plane so we get: (a) smooth CSS-Grid → hero morph on click, (b) shader-based reveal on scroll that makes long docs pages feel alive without gimmicky animations, (c) persistent canvas across doc routes so navigation feels app-like. Skip the chromatic aberration — Clean is developer infra, not a design portfolio.

## BUILD DIRECTIVE

TARGET: Clean's landing page hero + "how it works" section (single build, hero rebuilds and section is new).

APPROACH: Build a two-phase scroll narrative. Phase 1 (hero, top of page): pinned 100vh stage with a single R3F canvas containing an instanced node graph of MCP servers (nodes) with edges that light up amber on scroll and pulse when a token packet traverses them. Big display serif × mono-numeric headline in Wembi's discipline: "Reduce agent token cost by <mono>62%</mono>." Persistent "Get access" CTA pinned top-right. Phase 2 (how it works, immediately below): four pinned chapters — Connect / Route / Compress / Deploy — using the NRG chapter-narrative pattern but vertical, translating the same R3F scene rather than swapping scenes. In each chapter, an agent-icon (small Rive character à la Disco Dungeon) travels along the pipeline path and animates the specific step (getting routed, getting TOON-compressed, arriving at the LLM). Everything driven by `scrollYProgress` written to CSS vars and R3F uniforms in one rAF tick.

STACK:
- next@15 + react@19
- three@0.170 + @react-three/fiber + @react-three/drei + three-mesh-bvh
- gsap@3.13 + @gsap/react + ScrollTrigger + SplitText + MotionPathPlugin
- lenis@1.1
- rive-react (agent character)
- framer-motion@11 (UI microinteractions only)
- tailwindcss@3.4 + tailwindcss-animate
- next/font/local + next/font/google (Fraunces display serif, JetBrains Mono for numerals + code)
- lucide-react

REFERENCE URLS:
- https://business.nrg.com/campaigns/build-your-data-center/ (chapter-narrative pinning, palette discipline, persistent CTA)
- https://www.awwwards.com/sites/wembi (typography system, SplitText reveals, magnetic CTA)
- https://discodungeongame.com/ (Rive character riding a scroll-linked spline through feature callouts)

KEY CONSTRAINTS:
- Clean is developer infrastructure. Feel technically sophisticated and fast. No flashy grain overlays, no chromatic aberration, no glassmorphism, no gradient-on-gradient hero blobs.
- Every animation must have semantic weight: if it doesn't teach the buyer what Clean *does*, cut it.
- No horizontal-scroll; developers use trackpads and hate it.
- Time-to-interactive on 3G Fast must be <2.5s. R3F canvas suspended offscreen, textures KTX2-compressed, Rive files <120KB.
- Two-color palette maximum for the hero surface (ink + one accent). Do not add "supporting" colors during design review — they always leak in and always dilute.
- All motion respects `prefers-reduced-motion` — replace the R3F scene with a static SVG diagram, drop the character rig, keep the copy.
- Ship the hero + how-it-works as a single React Server Component tree; the WebGL is one client island. Do not client-side the whole page.
