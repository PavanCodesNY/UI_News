# UI Scout Report — 2026-07-04

## Bucks Sauce — https://buckssauce.com/

WHAT IT DOES: A one-product-focused e-commerce site whose spine is a horizontally-anchored home product slider that snaps between SKUs with an inertia curve, paired with an interactive 3D-fruit "spread" on the product page (a stylized cherry/garlic mesh that reacts to cursor position with a tilt + specular highlight sweep). Vertical left-rail menu stays pinned; product grid uses staggered opacity+translateY reveals gated by IntersectionObserver; an about-page bottle "stop motion" sequence swaps a numbered image sequence tied 1:1 to scroll progress via GSAP ScrollTrigger. Overall layout is a fixed 12-col grid with a persistent side-nav column, keeping typographic mass on the left and product imagery in the optical center.

PACKAGES:
- `next@14` (App Router, deployed on Vercel)
- `react@18`, `react-dom@18`
- `gsap@3.12` + `gsap/ScrollTrigger` + `gsap/Observer`
- `three@0.16x` + `@react-three/fiber@8` + `@react-three/drei@9` (for the 3D fruit spread)
- `lenis@1` (smooth scroll)
- `tailwindcss@3` for utility layout, custom CSS for the display serif
- Fonts: a bold high-contrast display serif (e.g., `PP Editorial New` or `Reckless Neue`) + a mono for badges (`JetBrains Mono` or `IBM Plex Mono`)
- `sanity` or `shopify-hydrogen` for commerce data (pattern-inferred)

IMPLEMENTATION:
1. App shell: `<LenisProvider>` → `<Header>` (top-left wordmark + top-right cart) → `<SideNav>` (fixed left, 60px wide, vertical letter-spaced menu items in uppercase mono).
2. Home slider: full-viewport `<section>` with a horizontal flex row of slides where each slide is `w-screen`. Advance via `Observer.create({ type: "wheel,touch", tolerance: 20 })` calling `gsap.to(track, { x: -index * window.innerWidth, duration: 1.1, ease: "expo.out" })`. Background color CSS variable animates in parallel to match sauce hue.
3. 3D fruit: `<Canvas camera={{ fov: 30, position: [0, 0, 6] }}>` with `<Environment preset="warehouse" />`, a GLB fruit mesh loaded via `useGLTF`, and `useFrame` mapping mouse (`state.mouse.x/y`) to `mesh.rotation.y/x` through `damp` with `lambda: 4`. Add subtle `<pointLight>` orbiting the mesh for the specular sweep.
4. Product grid: CSS grid `grid-cols-2 md:grid-cols-3 gap-6`; each card wrapped in a `motion.div` with `whileInView={{ opacity: 1, y: 0 }}` and `viewport={{ margin: "-15% 0px" }}`, `stagger: 0.08`.
5. Stop-motion bottle: pre-decoded image sequence (24 frames, WebP, ~120kb each), pinned section `ScrollTrigger.create({ trigger, start: "top top", end: "+=200%", pin: true, scrub: 0.5, onUpdate: self => setFrame(Math.floor(self.progress * 23)) })`, rendered via `<img>` with `object-cover`.
6. Easing dictionary: `expo.out` for slider snaps, `power3.inOut` for section transitions, `power1.out` for hover states. Durations: slider 1.1s, hover 0.4s, section pins 0.8s.
7. Responsive: side-nav collapses to a bottom-drawer trigger below `md`; 3D canvas swaps to a static WebP hero below `sm` (perf).

WHY IT WORKS: The site uses a rigid orthogonal grid (Swiss/editorial) with one violation — the 3D fruit floats free of the grid — creating a single point of visual gravity that anchors attention (Gestalt figure/ground). Color-swapped backgrounds per SKU exploit chromatic priming so each product feels distinct without a redesign per page. The vertical mono nav signals "system, not brand," which is why the food category feels premium rather than novelty.

GOOD / BAD:
- GOOD: The slider inertia curve and background-hue sync are best-in-class — motion carries semantic meaning (this is a new flavor, not just a new slide).
- BAD: The 3D fruit is style over substance for e-commerce — it delays LCP, blocks CTA discoverability on mobile, and adds ~600kb before a "buy" button paints.

CLEAN APPLICATION: Apply the numbered/color-shifted slider metaphor to Clean's landing-page "how it works" section. Build a horizontal snap slider that steps through the 4 stages (Intercept → Parse → Rewrite → Deliver), where each stage swaps the background to a signature Clean gradient and the center stage renders a live-looking `<pre>` block whose lines animate to show the token/context transformation. Use GSAP Observer for the horizontal snap and Lenis for the vertical scroll below.

## Units — https://units.gr/en/homepage/

WHAT IT DOES: A scrollytelling marketing site for premium student housing, structured as a linear WebGL-backed narrative where a Three.js plane textured with the current section's image morphs (via a displacement shader driven by scroll velocity) into the next section's image, with Lottie micro-illustrations layering DOM-level detail on top of the WebGL canvas. Hover states on unit cards trigger a "detail reveal" where a clip-path mask sweeps from bottom-left to top-right exposing a second, denser card face. Cursor is a custom SVG blob that lags behind the pointer with a spring, scaling on interactive elements.

PACKAGES:
- `nuxt@3` (Vue-based, based on studio's build patterns) — or `next@14` if reproducing in React
- `three@0.16x` with custom fragment/vertex shaders (no R3F, direct WebGLRenderer)
- `gsap@3.12` + `ScrollTrigger` + `SplitText`
- `lottie-web@5.12` for illustration overlays
- `lenis@1` for scroll normalization
- Fonts: neo-grotesque geometric (e.g., `Aeonik` or `Neue Haas Grotesk`)
- `contentful` for CMS (inferred from Awwwards tag)

IMPLEMENTATION:
1. Root layout: single-page vertical scroll with `<Canvas>` fixed at `z-index: 0` full-viewport, DOM content in `<main>` at `z-index: 10` with `mix-blend-mode: difference` on select headings.
2. WebGL layer: single fullscreen plane, custom shader with three uniforms — `uTexture1`, `uTexture2`, `uProgress`. Fragment mixes textures with a radial dissolve driven by simplex noise: `mix(t1, t2, smoothstep(progress - 0.1, progress + 0.1, noise + vUv.y))`.
3. Scroll driver: `ScrollTrigger.create({ scrub: 1.2, onUpdate: self => uniforms.uProgress.value = self.progress })`. Velocity sampled each frame and fed into a `uDisplacementStrength` uniform, so fast scrolling bulges the plane.
4. Section reveals: `SplitText` on each headline, chars staggered 0.02s with `y: 100%, opacity: 0 → 0, 1`, ease `power4.out`, duration 0.9s.
5. Hover unit cards: two absolutely-positioned layers, top layer with `clip-path: polygon(0 100%, 0 100%, 0 100%, 0 100%)` on rest, animating to `polygon(0 0, 100% 0, 100% 100%, 0 100%)` on hover, `duration: 0.55, ease: cubic-bezier(0.65, 0, 0.35, 1)`.
6. Custom cursor: fixed `<div>` with `mix-blend-mode: exclusion`, `gsap.quickTo(cursor, "x", { duration: 0.35, ease: "power3" })` pattern for spring lag, scaling to 2.4x on `data-hover` elements.
7. Lottie: rendered via `lottie-web` in-DOM at absolute positions keyed to section anchors; each Lottie's `setSpeed` reacts to scroll velocity.
8. Responsive: shader plane disabled below 768px; static hero images swapped in. Custom cursor disabled on touch.

WHY IT WORKS: The velocity-driven displacement makes scrolling itself a first-class UI action — the user learns that fast scroll = distortion, slow scroll = clarity, which turns pace into information (motion semantics). The mix-blend-mode difference on titles guarantees legibility over any image background without a scrim, preserving whitespace rhythm.

GOOD / BAD:
- GOOD: The image-morph shader is genuinely best-in-class — it collapses page-transitions and hero-swaps into one continuous surface, which is rare.
- BAD: On low-end laptops (Intel HD graphics) the shader stutters below 40fps, and there is no `prefers-reduced-motion` fallback observed — accessibility gap.

CLEAN APPLICATION: Apply this to Clean's docs navigation surface. Instead of Notion-style sidebar-and-main-column docs, build a scroll-driven "concept flow" where each of Clean's core primitives (MCP proxy, TOON format, context layer, agent registry) occupies a full viewport, and scrolling morphs one architecture diagram into the next via a displacement shader on a fixed WebGL canvas. Diagrams live in the DOM above the canvas so text stays selectable and indexable.

## Wembi — https://www.wembi.ai/

WHAT IT DOES: A digital-twin B2B SaaS marketing site organized around numbered chapters (`N°001` through `N°006`), each functioning as a self-contained scene with its own kinetic type animation, video-loop poster, and swap-in of the accent color. Navigation is a chapter counter fixed to the top-right that ticks up as the user scrolls (matching current section). Section transitions are marked by a full-width horizontal ticker of the chapter title in oversized display type, sliding across as the section enters. Uses a duotone palette — a warm mint (`#beff8b`) as accent and a muted slate blue (`#9dbbc5`) as ground.

PACKAGES:
- `nuxt@3` + `vue@3`
- `gsap@3.12` + `ScrollTrigger` + `SplitText`
- `lenis@1`
- `craft-cms` backend (inferred from Awwwards tag)
- Video: raw `.webm` files served from CDN, `<video autoplay muted loop playsinline>`, no player library
- Fonts: an editorial serif for the "N°XXX" markers (`PP Editorial Old` style) + a technical grotesque for body (`Söhne` or `TWK Everett`)
- `iubenda` for cookie compliance

IMPLEMENTATION:
1. Layout: full-viewport sections with `min-height: 100svh`, each section wrapped in a `<section data-chapter="001">`. Chapter counter component reads which section's `[data-chapter]` intersects viewport center via `IntersectionObserver` with `rootMargin: "-50% 0px -50% 0px"`.
2. Section entrance ticker: on ScrollTrigger `onEnter`, animate a horizontal `<div>` containing repeated section title from `translateX(100%)` to `translateX(-100%)` over 3.5s, `ease: none`, then fade out.
3. Video posters: each section has a WebM loop as the visual centerpiece — served at 2 resolutions (`-mobile.webm` @ 720p, desktop @ 1440p), preloaded with `<link rel="preload" as="video">` for section N+1 while N is visible.
4. "How it works" stepper: horizontally scrolling triptych (`Digital Twin` → `Data / Device / Application`), pinned section, three tabs where the active tab drives which video loop plays. Tab underline animates via `gsap.to` with `duration: 0.6, ease: expo.inOut`.
5. Color transitions: CSS custom properties `--accent` and `--ground` on `:root`, animated via `gsap.to("html", { "--accent": nextColor, duration: 0.8 })` when entering each section (requires `gsap.registerPlugin(CSSPlugin)` and object syntax).
6. Menu: overlay menu triggered by top-right button; full-viewport, background is the current section's video looped at reduced opacity, menu items are large display-serif with a hover state that shows the section number in a mono to the left.
7. FAQ: accordion where each open state slides an answer down with `height: auto` measured, `gsap.to({ height: measured, duration: 0.5, ease: power2.out })`.
8. Easing/durations dictionary: hero title reveal 1.2s `power4.out`, ticker slide 3.5s linear, section pins 0.8s `expo.out`, hovers 0.35s `power2.out`.

WHY IT WORKS: The numbered chapter system gives a B2B product a narrative arc that feels editorial rather than corporate — chunking the story into six discrete units respects Miller's Law and lowers cognitive load. The oversized ticker at each section boundary functions as a page-turn signal, exploiting motion semantics to replace what would otherwise be a "table of contents" nav.

GOOD / BAD:
- GOOD: The commitment to a full editorial system (numbered chapters + display serif + accent color logic) makes a technical AI/IoT product feel human — best-in-class narrative design for infrastructure.
- BAD: The site is Italian-only for most body copy with FAQ answers in Italian only; the interaction pattern is strong but the content strategy blocks the international audience the visual system is clearly built for.

CLEAN APPLICATION: Apply this to Clean's landing page as the primary structural system. Restructure the current single-scroll marketing page into six numbered chapters — `N°001 The Problem` (agents burn tokens), `N°002 The Proxy` (MCP proxy topology), `N°003 The Format` (TOON compression), `N°004 The Context` (semantic layer), `N°005 The Registry` (agent discovery), `N°006 Get Started`. Use the same top-right chapter counter, the same horizontal title ticker at section boundaries, and swap accent color per section (green/blue/purple gradient family). This is the highest-fidelity direct analog for Clean today.

## NRG | Build Your Data Center — https://business.nrg.com/campaigns/build-your-data-center/

WHAT IT DOES: A B2B infrastructure marketing site that gamifies "building a data center" as a 4-phase scroll journey with a persistent 2D isometric node-graph visualization at the center — nodes are lit as each phase is reached, edges animate energy-flow from left to right, and phase-specific 3D icon vignettes rotate in beside the graph. Virtual-tour section is a horizontal drag/scroll of hotspots over a stylized site-plan illustration. Palette is a signature safety-yellow (`#FFC20E`) against near-black (`#261D26`), letterforms are heavy geometric sans in all-caps.

PACKAGES:
- `next@14` or vanilla HTML5 + custom framework (Awwwards tags: HTML5, JavaScript, Figma → suggests custom stack)
- `gsap@3.12` + `ScrollTrigger` + `MotionPathPlugin` (for edge-tracing animations)
- `lenis@1`
- `three@0.16x` for the small 3D icon vignettes (rack, transformer, cooling loop), likely rendered offscreen and composited as PNG sequences for perf
- `svg-path-morph` or hand-rolled morph on SVG `d` attributes
- Fonts: display grotesque all-caps (e.g., `Neue Machina` or `PP NeueBit`) + mono for meta

IMPLEMENTATION:
1. Layout: sticky center-anchored graph canvas (`position: sticky; top: 0; height: 100vh`) with phase-content stacked as `<section>`s that scroll past it on the right side (2-column split: 40% graph left, 60% content right on desktop).
2. Node graph: SVG with all nodes rendered at load, opacity 0.15 by default. Each node has `data-phase="1|2|3|4"`. As a phase section enters view, `gsap.to("[data-phase='1']", { opacity: 1, stroke: yellow, stagger: 0.1 })` and the connecting edges animate via `strokeDashoffset` from full length to 0 (`duration: 1.2, ease: power2.inOut`).
3. Energy pulses: circles ride each active edge using `MotionPathPlugin.getPositionOnPath()` in a `gsap.to({ progress: 1, duration: 2, repeat: -1, ease: none })` loop; multiple pulses staggered.
4. Virtual tour: horizontal drag interaction on a wide illustration, `Draggable.create(container, { type: "x", bounds })`. Hotspots are absolutely-positioned buttons that pulse via CSS `@keyframes`.
5. Phase 04 scroll-through: pinned section, ScrollTrigger scrub tied to `useTransform`-style progress that reveals a stack of card-details in sequence (each card gets `yPercent: -100` on exit).
6. Header on-scroll: shrinks from 96px to 64px, background transitions from transparent to `#261D26e0` (backdrop-blur 12px) — `ScrollTrigger.create({ start: 100, toggleClass: "shrunk" })`.
7. Easing dictionary: node highlights `power2.inOut` 0.6s, energy pulses `none` 2s repeating, phase transitions `expo.out` 1s, header shrink `power3.out` 0.4s.

WHY IT WORKS: The persistent center-of-screen graph is a masterclass in "the artifact is the story" — the user is not reading about the data center, they are watching it come alive node by node as they scroll (motion semantics as literal progress indicator). High-contrast yellow-on-black plus a strict all-caps grotesque creates industrial authority without visual noise, hitting the correct emotional register for hyperscaler procurement.

GOOD / BAD:
- GOOD: The mapping of scroll position to graph completion is genuinely best-in-class — every scroll pixel earns visual state, so there is no "dead scroll" — a very rare achievement in a marketing site.
- BAD: The virtual-tour horizontal drag has no keyboard-arrow alternative documented; accessibility jury flagged this at 6.8/10 for a reason.

CLEAN APPLICATION: Apply this to Clean's product-demo section on the landing page. Build a persistent sticky visualization of the MCP proxy topology (client agent → Clean proxy → toolset MCPs) where each scroll phase illuminates a stage: (1) raw request in, (2) TOON compression firing, (3) semantic context injection, (4) response streaming out. Use animated energy pulses on the edges to make the token savings feel visceral rather than a stat. Yellow-on-black is wrong for Clean; use Clean's mint/violet palette instead — copy the mechanic, not the colorway.

## Sketching the Impossible: A 3D Portfolio Built Without a Single 3D Model — https://tympanus.net/codrops/2026/06/11/sketching-the-impossible-a-3d-portfolio-built-without-a-single-3d-model/

WHAT IT DOES: A Codrops case-study + open-source tutorial for a portfolio site built entirely with Three.js primitive geometries (boxes, spheres, planes) composed and shaded via custom GLSL to feel like hand-sketched 3D scenes — no external GLB/GLTF models, no baked textures. Uses a raymarched fog + procedural line-art shader for a "pencil drawing" pass, and scroll drives a camera on a Catmull-Rom spline through a sequence of scenes.

PACKAGES:
- `three@0.16x`
- `@react-three/fiber@8` + `@react-three/drei@9` (for `<CatmullRomLine>`, `<PerspectiveCamera makeDefault>`)
- `@react-three/postprocessing@2` for the sketch/line-art pass
- `leva` for dev-time shader tuning
- `zustand` for scroll state (progress shared across the scene tree)
- `gsap@3.12` for DOM overlays
- `lenis@1`
- Custom GLSL shaders (fragment + vertex, inline in template strings or `.glsl` files loaded via a Vite raw import)

IMPLEMENTATION:
1. Scene composition: everything is `<mesh>` with `<boxGeometry>`, `<sphereGeometry>`, `<planeGeometry>`, `<torusKnotGeometry>` — no `useGLTF`. Complex forms are constructed by CSG-like union of primitives or by displacement in the vertex shader.
2. Sketch shader: post-processing pass that samples the depth+normal buffer, applies a Sobel filter for edges, and modulates output with a procedural crosshatch texture (`step(sin(vUv.x * 800.0) * sin(vUv.y * 800.0), 0.4)`).
3. Camera on spline: `<CatmullRomLine points={[[0,0,0], [3,2,-5], [7,-1,-12], ...]} />`, on each frame `curve.getPointAt(scrollProgress)` sets camera position, `curve.getTangentAt(scrollProgress + 0.001)` orients `camera.lookAt`.
4. Scroll binding: Lenis emits `onScroll` → zustand store's `setProgress(scrollY / maxScroll)` → subscribed `useFrame` in R3F applies to camera and to per-scene reveal uniforms.
5. Per-scene reveals: each mesh listens to a normalized `[0, 1]` progress window (e.g., mesh appears from `0.2` to `0.35`); vertex shader uses this to interpolate scale/position (`mix(startPos, endPos, smoothstep(0.2, 0.35, progress))`).
6. DOM overlays: HTML section titles positioned via `<Html>` from drei, sync-locked to camera via `distanceFactor` prop.
7. Performance: DPR clamped to `Math.min(window.devicePixelRatio, 1.75)`, `frameloop="demand"` toggled when tab is hidden.

WHY IT WORKS: By using only primitives + shaders, the site achieves visual identity through style not asset weight — the sketch aesthetic is intrinsic to the render pipeline, not a texture. The camera-on-spline model makes scroll feel like a directed camera shot rather than a page scroll (motion semantics: "you're inside the story, not scrolling past it").

GOOD / BAD:
- GOOD: The zero-external-model approach is genuinely best-in-class — total shipped 3D asset weight is under 40kb, versus 5–15MB for typical R3F portfolios.
- BAD: The technique caps at low-poly geometry complexity — you cannot make a photoreal product visualization this way, so it's aesthetic-specific, not general purpose.

CLEAN APPLICATION: Apply this to Clean's hero background. Instead of a stock gradient or a flashy GLB, build a raymarched/primitive scene of abstract "context flowing through a proxy" — a series of primitive boxes connected by thin extruded tubes, shaded with a soft-crosshatch pass in Clean's palette. Camera on a short spline that drifts as the user hovers the hero (subtle parallax) and dips deeper into the graph on scroll. Ships under 60kb, no GLB pipeline, no artist bottleneck.

## BUILD DIRECTIVE

TARGET: Clean's landing-page hero + "how it works" section (a single continuous surface from viewport top through the end of the product explainer).

APPROACH: Rebuild the top of tryclean.ai as a numbered-chapter narrative (Wembi's system) driven by a sticky center visualization (NRG's node-graph mechanic), where the visualization itself is a lightweight primitives-only R3F scene of the MCP proxy topology (Codrops "Sketching the Impossible" technique). Chapter 001 is the hero — a static-camera view of the topology graph with the tagline and a mono-styled CTA. Chapters 002–005 correspond to Intercept → Compress (TOON) → Contextualize → Deliver, each snapping camera position on a spline and lighting up the relevant edge of the graph with animated energy pulses (yellow-on-black replaced by Clean's mint/violet). Chapter boundaries mark with an oversized editorial-serif ticker sliding across the width. Body copy stays DOM-selectable via drei `<Html>` overlays. No stock 3D assets, no video files, no heavy libraries in the first 100kb of JS.

STACK:
- `next@14` (App Router)
- `react@18`, `react-dom@18`
- `@react-three/fiber@8`, `@react-three/drei@9`, `@react-three/postprocessing@2`
- `three@0.16x` with custom GLSL for edge-pulse and background-noise shaders
- `gsap@3.12` + `ScrollTrigger` + `SplitText` (chapter ticker + section pins)
- `lenis@1` (smooth scroll)
- `zustand@4` (scroll-progress store shared between DOM + R3F)
- `tailwindcss@3` + a Clean design-token layer for color/spacing custom properties
- Fonts: `PP Editorial New` (chapter numerals + ticker) via Fontshare, `Söhne` or `Inter Display` (body), `JetBrains Mono` (code + meta labels)

REFERENCE URLS:
- Wembi — https://www.wembi.ai/ (chapter numbering system, ticker transitions, accent-color-per-section)
- NRG Build Your Data Center — https://business.nrg.com/campaigns/build-your-data-center/ (sticky node-graph + phase-driven illumination + energy pulses)
- Codrops "Sketching the Impossible" — https://tympanus.net/codrops/2026/06/11/sketching-the-impossible-a-3d-portfolio-built-without-a-single-3d-model/ (primitives + shader-only 3D approach, camera-on-spline)

KEY CONSTRAINTS: Clean is developer infrastructure — the result must feel technically sophisticated and fast, not flashy or generic. Avoid: stock hero videos, glassmorphism cards over blurred gradients, animated blob/gradient meshes as background, Framer Motion "spring bounce" easings on system elements, testimonial carousels above the fold, generic bento grids for feature callouts, or any Aceternity/Magic UI "spotlight card" component (overused, immediately identifies the site as vibe-coded). Body copy must remain DOM-selectable and indexable — no text baked into WebGL. First-Contentful-Paint must land under 1.2s on Fast 3G; defer R3F canvas mount until after LCP. Provide a `prefers-reduced-motion` fallback that swaps the animated graph for a static SVG version.
