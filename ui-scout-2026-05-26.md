## 1. aino.agency — https://aino.agency

WHAT IT DOES: Single-page studio site that renders the entire UI — text, images, video frames, 3D, micro-games — into one monospaced ASCII character grid. The grid is a fixed-pitch matrix (≈ 120 cols × 60 rows on desktop) driven by a `requestAnimationFrame` ticker; each cell stores a glyph + color and is mutated per frame by a 2D rigid-body physics solver (Verlet-style) and a real-time morphing pass that diffs target glyphs against current glyphs and tweens via easing. Total JS payload ships in ~30 kb gzipped, no React, no Three.js — DOM is a single `<pre>` (or `<canvas>` text-blit) with a CSS `font-variant-numeric: tabular-nums` and `letter-spacing: 0`.

PACKAGES:
- `vite` (build)
- `typescript`
- `planck-js` or `matter-js` (2D physics — Verlet/Box2D port)
- Custom ASCII rasterizer (no library; ~200 LOC)
- `lenis` (scroll-driven cell updates)
- Self-hosted monospace font: `JetBrains Mono` or `Berkeley Mono` (woff2, subset to printable ASCII only — ~12 kb)
- No framework. No Tailwind. No Framer Motion.

IMPLEMENTATION:
1. Mount one `<pre id="grid">` filling 100vw/100vh. Compute `cols = floor(window.innerWidth / charWidth)`, `rows = floor(window.innerHeight / lineHeight)`. `charWidth ≈ 8.4px` and `lineHeight ≈ 14px` at `font-size: 14px`.
2. Allocate `Uint8Array(cols * rows)` for glyph indices and a parallel `Uint32Array` for packed RGBA colors.
3. Render loop: `requestAnimationFrame` → physics step (`world.step(1/60)`) → for each body, sample its AABB into grid cells and stamp glyphs (`█ ▓ ▒ ░` for density, `/ \ | _` for edges). Convert grid to a single string and assign to `pre.textContent` (one write per frame, never per cell).
4. Morphing: each cell holds `{ current, target }`. On every frame, advance current toward target using `Math.sign(target - current)` with a per-cell delay derived from `(x + y) * 0.02` (diagonal cascade). Use `power2.out` easing for opacity rather than glyph swap when fading.
5. Images/video → ASCII via offscreen `<canvas>`: draw at `cols × rows`, sample luminance, map to ramp ` .:-=+*#%@`. For video, run the sample on `requestVideoFrameCallback`, not RAF.
6. Color: keep a single accent (electric green `#00FF66` on `#0A0A0A`) and one variant section with a rainbow palette mapped to `(x + frame) % 6` for the vintage mode.
7. Performance: never touch the DOM per cell. Build the full string in a `Uint16Array` of char codes, then `String.fromCharCode.apply(null, buf)` and assign once.
8. Responsive: at `< 768px` drop the grid to 60 × 40 and disable physics (cells become static targets, only morphing runs).

WHY IT WORKS: The grid system is a single, dominant visual rule — Gestalt's law of similarity and proximity carry every element into one coherent texture, so the site reads as a unified instrument rather than a collection of widgets. Constraining everything to one typeface and one cell size produces extreme rhythm and removes 90% of the design decisions a normal site has to make; the remaining motion (physics, morphing) is then unmistakably authored. The 30 kb payload itself is a brand statement — competence as ornament.

GOOD / BAD:
- GOOD: Best-in-class commitment to a single primitive; loads instantly; physics + ASCII is genuinely novel and ships in less JS than a typical hero image.
- BAD: Screen-reader hostile (one `<pre>` blob), keyboard navigation is performative, and on low-DPI laptops the cells alias badly. No real SEO surface for marketing pages.

CLEAN APPLICATION: Apply this to Clean's **landing-page hero (above the fold)** by replacing the current hero illustration with a 100vw × 70vh ASCII grid that morphs between three target states on a 6-second loop: (state A) an MCP server icon graph drawn with `█` and `─`; (state B) the company wordmark "CLEAN" in 12-row block letters; (state C) a live-looking TOON-format token stream scrolling vertically with hex addresses and JSON fragments rendered as glyphs. Use Berkeley Mono at 14px, accent `#00FF66` on `#0A0A0A`, and drive the morph from a `ScrollTrigger` so the hero "compiles itself" as the user lands. Cap to 30 kb of incremental JS — this is the brand signal: developer infrastructure should look like it ships less, not more.

## 2. Cartier Watches & Wonders 2026 — https://www.awwwards.com/sites/cartier-watches-wonders-2026

WHAT IT DOES: Scroll-driven immersive 3D experience by Immersive Garden. Each section is a self-contained "alcove" — a Three.js scene with bespoke GLSL shaders, baked Blender lighting (HDR + AO), and a custom soundscape via Web Audio API. Camera transitions between alcoves are not cuts but continuous tweened camera paths along Bezier curves driven by GSAP timelines bound to Lenis scroll progress. Scoring 7.53/10 on Awwwards (Design 7.73 / Creativity 7.84).

PACKAGES:
- `three` (^0.165)
- `lenis` (smooth scroll, single source of truth)
- `gsap` + `ScrollTrigger`
- `sass` (preprocessor)
- `glslify` (shader composition) and raw `.glsl` files via `vite-plugin-glsl`
- Web Audio API (native — no Howler)
- `gltfjsx` (Blender → React Three Fiber if RTF used; raw Three.js otherwise)
- `tweakpane` (in-dev for art-direction tuning)

IMPLEMENTATION:
1. One persistent `WebGLRenderer` at `pixelRatio = Math.min(devicePixelRatio, 1.75)` with `antialias: false` + post-process FXAA pass via `EffectComposer`.
2. One `Scene` per alcove, swapped on scroll-section enter. Pre-load adjacent scenes on `intersectionObserver`. Dispose textures on exit (`scene.traverse → mesh.geometry.dispose()`).
3. Camera path: define a `CatmullRomCurve3` per section; map Lenis `scroll` (0–1 per section) to `curve.getPointAt(t)` and `curve.getTangentAt(t)` for `lookAt`. Use GSAP `gsap.to({ t: 0 }, { t: 1, ease: "power2.inOut", scrollTrigger: { ... scrub: 0.8 } })`.
4. Lighting: bake HDR + AO in Blender to a single 2K equirect → `PMREMGenerator` → `scene.environment`. No real-time lights except a single rim `DirectionalLight` for the watch metal.
5. Shaders: custom vertex displacement on fabric/material elements with `simplex3d` noise + time uniform, fragment shader does Fresnel + matcap blend for the gold cases.
6. Audio: one `AudioContext`, per-alcove `AudioBufferSourceNode` with a `GainNode` cross-faded over 1.2s on section change. Hold a global mute toggle, default ON for accessibility.
7. Preload gate: show a typographic counter (0–100%) that completes only when the first 2 alcoves' GLBs + KTX2 textures resolve. Use Basis/KTX2 (`KTX2Loader`) — cuts texture weight ~70% vs PNG.
8. Performance: target 60fps on M-series, 30fps floor on integrated GPUs. Use `renderer.setAnimationLoop` only when `document.visibilityState === 'visible'`.

WHY IT WORKS: Chapters-as-rooms is a strong narrative metaphor — each alcove sets a clear figure/ground and lets the watch dominate via spotlit contrast. Continuous camera tweens (vs hard cuts) preserve spatial continuity, which the brain reads as "real place" rather than "marketing site." The bespoke soundscape adds a second sensory channel that closes the immersion loop without ever being literal.

GOOD / BAD:
- GOOD: Reference-class production quality; correct restraint with one strong primitive per alcove; KTX2 + PMREM choices are exactly what shipping high-end WebGL looks like in 2026.
- BAD: Heavy first-paint (>5 MB even with KTX2); mobile experience strips most of the immersion; not accessible to assistive tech; ROI only justified for brand campaigns, not product pages.

CLEAN APPLICATION: Apply this to Clean's **product-demo section on the landing page** by building a single Three.js canvas that represents Clean as a 3D "context router." Three alcoves, scroll-bound via Lenis + GSAP ScrollTrigger: (alcove 1) raw MCP servers as 12 free-floating glass cubes in a noise field; (alcove 2) Clean proxy materializes as a chrome torus that attracts the cubes into a stable orbit; (alcove 3) camera dollies into the torus and the inside surface shows live TOON-formatted JSON streaming as text-as-texture. Use KTX2 for all textures, one DirectionalLight + one HDR env map, GLSL Fresnel on the chrome, GSAP scrubbed at `scrub: 0.8`. Cap total to 1.5 MB; mobile falls back to a static rendered video.

## 3. Cleo AI — https://www.awwwards.com/sites/cleo-ai

WHAT IT DOES: Marketing site for Cleo (AI financial assistant) built in Framer. The site executes the "un-bank-y" brief by combining tight motion choreography, a tactile mascot illustration system, and chat-bubble-as-hero pattern. Page transitions are Framer's native page-transition primitives with shared element morphing; scroll is native + IntersectionObserver-triggered Framer Motion variants on every section. Earned SOTD May 24 + Developer Award.

PACKAGES:
- `framer` (no-code, but generates React) — or the equivalent React stack:
  - `next` (^15)
  - `framer-motion` (^11) — `motion.div`, `variants`, `useScroll`, `useTransform`
  - `tailwindcss` (^4) + `@tailwindcss/typography`
  - `lucide-react`
  - `geist/font` (Geist Sans + Mono) or `Inter`
  - `radix-ui` for accessible primitives (Dialog, Tabs)
- Asset: bespoke 2D illustrations exported as SVG (animated via Framer Motion `pathLength`)

IMPLEMENTATION:
1. Page chrome: sticky nav with `backdrop-filter: blur(20px)` + `background: rgba(255,255,255,0.6)`, height 56px, border-bottom `1px solid rgba(0,0,0,0.06)`.
2. Hero: 2-column at `lg`, 1-column stacked on `md`. Left col is the H1 + sub + 2 CTAs. Right col is a "phone frame" SVG with a chat bubble stack — each bubble enters with `initial: { opacity: 0, y: 12, scale: 0.96 }`, `animate: { opacity: 1, y: 0, scale: 1 }`, `transition: { duration: 0.5, ease: [0.22, 1, 0.36, 1], delay: i * 0.18 }`.
3. Sections: use `whileInView` with `viewport: { once: true, margin: "-100px" }` and a parent `staggerChildren: 0.08` variant. Standard easing throughout: `cubic-bezier(0.22, 1, 0.36, 1)` (silk).
4. Color system: ivory bg `#FAF7F2`, charcoal text `#111111`, single accent `#7C5CFF` (purple), no gradients in body — gradients only in mascot illustrations.
5. Typography: display in `Geist Sans` at `clamp(40px, 6vw, 96px)`, line-height `0.95`, letter-spacing `-0.03em`. Body at 17/26.
6. Cursors: custom cursor on `:has(button)` only — small disc that scales 1.5× on hover with `mix-blend-mode: difference`.
7. Page transitions: wrap routes in `<AnimatePresence mode="wait">`; on exit, slide the current page up `-40px` with `opacity: 0` over 0.35s, new page enters from `+40px`.
8. Performance: every image is `next/image` with `priority` only on hero. Mascot SVGs inlined to avoid extra requests.

WHY IT WORKS: The site translates a fintech category trope (sober blue, charts, lock icons) into a personality-led system (ivory, mascot, conversational H1s), and that single inversion is the entire moat. Silk easing + small movement (12px, not 80px) keeps the page calm enough that the playful copy feels confident rather than gimmicky. The chat-bubble hero is also a free product demo — visitors learn what Cleo does in one scroll.

GOOD / BAD:
- GOOD: Best-in-class category-inversion; motion is tasteful; copy and visual identity are perfectly aligned; trivially fast on mobile.
- BAD: Framer's runtime adds ~120 kb of overhead a hand-built version wouldn't pay; some animations re-trigger on remount; little technical novelty.

CLEAN APPLICATION: Apply this to Clean's **"How it works" section** (currently a static diagram) by rebuilding it as a 3-bubble chat sequence between "Your Agent" and "Clean" that types out a real interaction (request a tool → Clean returns TOON-formatted context → agent acts). Use `framer-motion` `whileInView` with `staggerChildren: 0.18`, silk easing `[0.22, 1, 0.36, 1]`, ivory backdrop `#F8F8F5`, mono font for the JSON payloads, and a single accent `#00FF66` for the Clean bubble. The conversational form makes Clean's value legible in 4 seconds without a video.

## 4. Sidewave — https://www.awwwards.com/sites/sidewave

WHAT IT DOES: Studio/agency site built as "a system that merges structure, motion, and technology." Awarded SOTD + Developer Award on May 23, 2026. The site is a precision GSAP + Lenis composition over a rigid 12-column grid with full-bleed type, scroll-locked horizontal sections, and clip-path wipe transitions between work cases. WebGL is used selectively — only for a single hero shader and on case-study images for grayscale-to-color reveal on scroll.

PACKAGES:
- `vite` + `typescript`
- `gsap` + `ScrollTrigger` + `SplitText` + `Flip`
- `lenis` (^1.1)
- `ogl` (lightweight WebGL — chosen over three.js for ~12 kb vs ~150 kb)
- `barba.js` or native `view-transitions` for page transitions
- `sass` + BEM naming
- `JetBrains Mono` (UI), `Söhne` or `Inter Display` (headlines)

IMPLEMENTATION:
1. Scroll: instantiate Lenis with `{ lerp: 0.1, smoothWheel: true }`. Bind once: `lenis.on('scroll', ScrollTrigger.update); gsap.ticker.add((t) => lenis.raf(t * 1000)); gsap.ticker.lagSmoothing(0);`.
2. Hero: full-viewport headline split via `SplitText('h1', { type: 'chars,words' })`; on load animate `chars` with `y: 100%`, `opacity: 0` → `y: 0`, `opacity: 1`, `stagger: 0.012, ease: 'power3.out'`.
3. Horizontal locked sections: pin the section with `ScrollTrigger.create({ trigger, pin: true, scrub: 1, end: '+=200%' })` and animate inner `x` from `0` to `-(width - viewport)`.
4. Case images: each `<img>` wrapped in a `<canvas>` that runs an OGL shader. Uniform `uProgress` tweens 0→1 via ScrollTrigger scrub; fragment shader does `mix(luma, color, uProgress)` + slight RGB shift on the edges.
5. Page transitions: clip-path wipe — outgoing page collapses with `clip-path: inset(0 0 100% 0)` over 0.6s `power2.inOut`, incoming reveals with `inset(100% 0 0 0)` → `inset(0)`.
6. Microtype: every label uses `JetBrains Mono` at 12px, tracking `0.08em`, uppercase. Body in `Inter` at 16/24.
7. Color: pure white `#FFFFFF`, pure black `#000000`, one accent `#FF4A1A` (signal orange).
8. Performance: ScrollTrigger refresh debounced on resize (300ms); pause OGL shaders when canvas leaves viewport via IntersectionObserver.

WHY IT WORKS: Single grid, single motion language (everything scrubbed, nothing free-running), and a 3-color palette mean every page reads as the same instrument played at different intensities. SplitText hero is now a cliché, but combined with Flip-based case transitions it stops feeling like a template — the motion grammar is genuinely consistent.

GOOD / BAD:
- GOOD: Perfect technical taste — OGL over Three.js, Lenis bound through GSAP ticker (the correct way), Flip for shared-element transitions; reads as a senior team's site.
- BAD: SplitText hero is approaching exhaustion as a trope; horizontal lock is mobile-hostile (falls back to vertical, losing the gag); accessibility for prefers-reduced-motion is partial.

CLEAN APPLICATION: Apply this to Clean's **docs / changelog page** by building a horizontally-locked "release timeline" section: pin the section, scrub a horizontal scroll of the last 6 release cards (each card a screenshot + 3 bullets), with case-image-style OGL grayscale-to-color reveal on each card as it enters the viewport center. Use `SplitText` only on the section heading "Shipping context infrastructure, weekly." JetBrains Mono for labels, Inter for body, accent `#00FF66`. Pair with the existing Lenis instance — do not introduce a second smooth scroller.

## 5. Aceternity UI: Background Ripple Effect + Encrypted Text — https://ui.aceternity.com/components/background-ripple-effect & https://ui.aceternity.com/components/encrypted-text

WHAT IT DOES: Two recent Aceternity drops (May 2026) that read as native developer-tool primitives. Background Ripple Effect is a CSS-grid of cells where each cell has a `cell-ripple` keyframe (opacity 0.4 → 0.8 → 0.4) staggered by `animation-delay` proportional to its Manhattan distance from a click point, producing a radial ripple of glow across a dark grid. Encrypted Text scrambles its target string each frame with random ASCII chars, then progressively locks letters left-to-right over a configurable duration, ending on the real string — classic "decrypting" reveal.

PACKAGES:
- `react` (^18 or ^19)
- `next` (^15) optional
- `tailwindcss` (^4)
- `motion` (formerly `framer-motion`, ^12) — used by Aceternity for state-driven transitions, not required for either of these two
- `clsx`, `tailwind-merge`
- `lucide-react` (icons)
- Self-hosted fonts: `Geist Mono` for the encrypted text

IMPLEMENTATION:

**Background Ripple Effect:**
1. Render a `div` with `display: grid; grid-template-columns: repeat(40, 1fr); grid-template-rows: repeat(20, 1fr);` filling the viewport behind content.
2. Each cell is a `<button>` (for hit-target) with `border: 1px solid rgba(255,255,255,0.04)` and `background: rgba(255,255,255,0)`.
3. On `onClick(row, col)`, set CSS vars `--origin-row` and `--origin-col` on the grid. Each cell computes its delay: `style={{ animationDelay: \`${(Math.abs(row - origin.row) + Math.abs(col - origin.col)) * 40}ms\` }}`.
4. `@keyframes cell-ripple { 0% { background: rgba(255,255,255,0); } 25% { background: rgba(255,255,255,0.08); } 50% { background: rgba(255,255,255,0.04); } 100% { background: rgba(255,255,255,0); } }`; `animation: cell-ripple 600ms ease-out`.
5. Auto-fire ripples every 6s from a random cell when idle (no user click for 3s) to keep the hero "alive."

**Encrypted Text:**
1. Component prop: `text: string`, `duration: number` (default 1200ms), `scrambleChars` (default `"!<>-_\\/[]{}—=+*^?#"`).
2. `useEffect` runs a `requestAnimationFrame` loop. State `displayed: string`. For each frame, compute `revealedCount = Math.floor((elapsed / duration) * text.length)`. Build `displayed = text.slice(0, revealedCount) + [...text.slice(revealedCount)].map(() => scrambleChars[Math.floor(Math.random() * scrambleChars.length)]).join('')`.
3. Render in `<span className="font-mono">{displayed}</span>`; revealed prefix gets `text-foreground`, unrevealed gets `text-foreground/40`.
4. Trigger on `whileInView` with `IntersectionObserver` so it only runs once when the heading lands in view.

IMPLEMENTATION (composition for hero use):
1. Stack: `position: relative` parent. Background = Ripple grid at `inset-0 z-0`. Foreground = headline + sub + CTAs at `z-10`.
2. Headline = `<EncryptedText text="The context layer for AI agents." duration={1400} />` at `text-6xl font-medium tracking-tight`.
3. On scroll past hero, dim ripple to `opacity-30` to recede behind subsequent content.

WHY IT WORKS: A live, low-amplitude background gives the page a heartbeat without competing with type — the grid is rhythmic but never moves, so the eye still rests on the headline (figure/ground done correctly). Encrypted reveal reads as semantically meaningful for an AI/infra product — "decrypting" maps directly to the brand promise — rather than decorative motion. Both components are GPU-cheap, do not block paint, and degrade gracefully with `prefers-reduced-motion`.

GOOD / BAD:
- GOOD: Both components are ~50 LOC each, framework-agnostic, accessible (semantic markup intact), and now an established visual language for AI dev tooling.
- BAD: Decrypt reveal is verging on cliché in the AI/infra category (Vercel, Replicate, Modal all have versions); if used on every section it becomes noise. Ripple grid can ship a lot of DOM nodes — keep cell count ≤ 800.

CLEAN APPLICATION: Apply this to Clean's **landing-page hero** as a hybrid with finalist #1. Background = Aceternity Background Ripple Effect at 40 × 16 cells, cells in `rgba(0,255,102,0.04)` lines on `#0A0A0A`. Headline = Encrypted Text reveal of "Stop drowning your agents in context." over 1.4s on first paint. Subhead in plain `Geist Sans`. Two CTAs: primary `bg-[#00FF66] text-black`, secondary ghost. Click any background cell to fire a ripple from there — the page literally responds to the developer's first input. Ships in <20 kb of incremental JS; no Three.js needed for the hero.

## BUILD DIRECTIVE

TARGET: **Clean landing-page hero** (tryclean.ai above-the-fold).

APPROACH: Build a single hero section that fuses three of today's strongest patterns into one developer-native canvas. Foundation is the Aceternity Background Ripple Effect (finalist #5) — a 40-column × 16-row dark grid on `#0A0A0A` with subtle `rgba(0,255,102,0.04)` cell borders, clickable, with an idle auto-ripple every 6s. Overlay the headline as the Aceternity Encrypted Text component (1.4s decrypt on first paint) reading "Stop drowning your agents in context." On scroll past the fold, the ripple grid fades to 30% opacity and is replaced by a section-2 ASCII canvas (finalist #1) that morphs between three TOON-format payload states — this carries aino.agency's compositional discipline (one primitive, one font, one rhythm) into Clean's product story. Motion vocabulary throughout: silk easing `[0.22, 1, 0.36, 1]` (Cleo), 12px movement amplitudes, no parallax, no Three.js in the fold. Total incremental JS budget: 40 kb gzipped.

STACK:
- `next` (^15) + `react` (^19) + `typescript`
- `tailwindcss` (^4) + `tailwind-merge` + `clsx`
- `motion` (^12, formerly framer-motion)
- `lenis` (^1.1) — single instance, shared across page
- `gsap` (^3.13) + `ScrollTrigger` — bound through Lenis's ticker
- `geist` (Geist Sans + Geist Mono via `geist/font`)
- `lucide-react` (icons)
- No Three.js, no OGL, no Framer Motion 3D for the hero. Reserve WebGL for a later product-demo section.

REFERENCE URLS:
- https://ui.aceternity.com/components/background-ripple-effect (foundation grid + click interaction)
- https://ui.aceternity.com/components/encrypted-text (headline reveal)
- https://aino.agency (section-2 ASCII morph — visual reference only, not the hero)

KEY CONSTRAINTS: Clean is developer infrastructure. The hero must feel like a terminal, not a campaign site. **Avoid**: gradients in the headline, glassmorphism, abstract 3D orbs, mouse-following spotlight follower cursors, parallax of any kind, scroll-jacked video, lottie illustrations, mascot characters, full-bleed photography, hero-section background videos. **Require**: monospaced numerals everywhere, every animation respects `prefers-reduced-motion`, first contentful paint < 1.2s on 4G, headline must remain crawlable plain text in the DOM (encrypted reveal is presentational only — real text in `aria-label`), no element animates without semantic justification (the ripple = "Clean responds to your input"; the decrypt = "Clean decodes context"). One accent color only: `#00FF66` on `#0A0A0A`. If a CTA needs a second color, use white.
