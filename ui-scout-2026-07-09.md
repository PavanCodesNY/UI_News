# UI Scout Report — 2026-07-09

## Depo Luxe — https://www.awwwards.com/sites/depo-luxe

WHAT IT DOES: A single-page vertical scroll navigation of luxury campaign case studies (McQueen, Kenzo, Azzaro) presented as a film-strip index — editorial two-digit numbering (01, 02, 03…) locked to the left margin, campaign titles typeset in a large serif display face on the right, and a fullscreen video preview that swaps and cross-fades as the user scrolls or hovers each index row. Layout is a CSS grid with a sticky left rail (rank + brand name) and a right-column video canvas that pins during scroll while campaign metadata rotates through with clip-path reveals.

PACKAGES:
- next@15 or astro@4
- gsap@3 + gsap/ScrollTrigger
- lenis@1.1 (smooth scroll)
- @studio-freight/hamo (viewport hooks)
- tailwindcss@3.4
- clsx
- next-video or hls.js (for adaptive H.265 previews)
- next/font (self-hosted display serif — GT Sectra or Editorial New)
- @vercel/og (for share cards)

IMPLEMENTATION:
1. Root layout: `<main class="grid grid-cols-12 gap-0 min-h-screen bg-neutral-950 text-neutral-50">`. Left rail = `col-span-4` sticky container with index list. Right video canvas = `col-span-8` fixed viewport, `position: sticky; top: 0; height: 100vh`.
2. Each campaign row: `<article class="border-t border-neutral-800 py-24 flex items-baseline">` — number in `font-mono text-xs tracking-[0.3em] uppercase opacity-60`, title in `font-serif text-[clamp(48px,7vw,120px)] leading-[0.95] tracking-[-0.02em]`.
3. Video swap: preload 5 short H.265 loops at 720p/2.5Mbps; register one ScrollTrigger per row with `onEnter`/`onEnterBack` firing `videoRef.current.play()` and a GSAP timeline that morphs `clip-path` from `inset(100% 0 0 0)` to `inset(0 0 0 0)` over `duration: 0.9, ease: 'expo.out'`.
4. Cursor: replace default cursor with a `mix-blend-difference` circle (24px), scale to 96px on row hover using `gsap.to(cursor, { scale: 4, duration: 0.4, ease: 'power3.out' })`.
5. Lenis config: `{ duration: 1.4, easing: (t)=>Math.min(1,1.001-Math.pow(2,-10*t)), smoothTouch: false }`.
6. Responsive: `<md` collapses to single column, video moves above title, sticky rail disabled.
7. Perf: `content-visibility: auto` on rows below the fold; `<video muted playsinline preload="metadata">`; preconnect to CDN; only one video plays at a time (pause others in the ScrollTrigger `onLeave`).

WHY IT WORKS: Ranked editorial numbering triggers list-completion behavior — users scroll to see #05 because they saw #01. The mix-blend-difference cursor + oversized serif enforces a single focal point per row (Gestalt figure-ground), while the sticky video acts as a persistent visual anchor that reduces cognitive load between transitions. Motion is expo.out (fast start, slow settle), which reads as "expensive" rather than snappy.

GOOD: Zero navigation chrome — the index IS the nav; every row is a self-contained portal.
BAD: Sticky video is GPU-heavy on low-end Android; disables at `<md` which kills the whole gimmick on 50% of traffic.

CLEAN APPLICATION: Apply this to Clean's **product demo / use-cases section on the landing page** by rebuilding it as a numbered index of the 4 core Clean modules (01 Lead Sourcing, 02 Auto-Profiling, 03 Deep Research, 04 Content Engine). Left rail = sticky module list with two-digit index. Right column = sticky 60vh product screencap that cross-fades between actual Clean UI recordings as the user scrolls each row. Use `mix-blend-difference` on module titles so they pop against dark screencaps. Replace serif with Clean's existing sans stack — keep the editorial numbering.

## MONOLOG — https://www.awwwards.com/sites/monolog

WHAT IT DOES: A studio portfolio built as a horizontal-scroll case study reel with kinetic typography intros — each project card is a `100vw × 100vh` panel, transitioned via a GSAP-driven `translateX` on a container while ScrollTrigger converts vertical wheel input into horizontal motion. Each panel has a three-layer stack: (1) a mask-revealed hero image, (2) a foreground SVG kinetic title with per-glyph rotational entry, and (3) a subtitle typewriter animation triggered on panel focus.

PACKAGES:
- react@19 + vite@5
- gsap@3 + gsap/ScrollTrigger + gsap/SplitText (Club plugin) OR splitting.js@1.1
- @studio-freight/lenis
- three@0.170 + @react-three/fiber (subtle background noise shader)
- @react-three/postprocessing (film grain)
- tailwindcss@3.4
- next/font — PP Neue Montreal + PP Editorial New
- lucide-react

IMPLEMENTATION:
1. Container: `<div class="h-screen overflow-hidden"><div ref={track} class="flex" style={{ width: `${panels*100}vw` }}>…`.
2. ScrollTrigger: `ScrollTrigger.create({ trigger: outer, pin: true, start: 'top top', end: () => `+=${track.scrollWidth}`, scrub: 1.2, onUpdate: (self) => gsap.set(track, { x: -self.progress * (track.scrollWidth - window.innerWidth) }) })`.
3. Kinetic title: SplitText into chars, initial `y: '110%', rotate: 12, opacity: 0`; enter timeline `{ y:0, rotate:0, opacity:1, duration:0.9, ease:'expo.out', stagger:{amount:0.5, from:'random'} }` triggered when the panel's `x` reaches viewport center.
4. Background R3F canvas at `position:fixed; inset:0; z:-1`, a single `<Plane>` with a custom fragment shader applying value noise + slow uniform time offset (`uniforms.uTime.value = clock.elapsedTime * 0.05`). Add `<Noise premultiply blendFunction={BlendFunction.SOFT_LIGHT} opacity={0.35} />`.
5. Cursor: SVG dot that magnetically snaps to CTA buttons using `gsap.quickTo(cursor, 'x', {duration:0.3, ease:'expo.out'})`.
6. Responsive: fall back to vertical native scroll below `1024px`; disable pinning and horizontal transform.
7. Perf: throttle `scrub` to `1` on mobile viewports; preload only ±1 adjacent panel images with `<link rel="preload" as="image" fetchpriority="high">`.

WHY IT WORKS: Horizontal scroll subverts scroll expectation, which extends dwell time (users slow down to figure out the mechanic). Kinetic typography with rotational entry uses motion contrast — the eye locks onto rotating letterforms because they violate the resting orientation prior. The R3F grain layer adds physical texture that hides banding in flat brand colors.

GOOD: Pin-and-scrub gives designers pixel-perfect timing without a single scroll library workaround.
BAD: Horizontal scroll on trackpads with inertia frequently overshoots panels; keyboard/screen-reader users get lost.

CLEAN APPLICATION: Apply this to Clean's **"How it works" landing page section** by building a 4-panel horizontal reel — Panel 1: paste ICP prompt. Panel 2: Clean discovers accounts. Panel 3: auto-profiles buyers. Panel 4: writes personalized outreach. Each panel gets the SplitText rotational title ("DISCOVER" → "PROFILE" → "RESEARCH" → "SEND") and a live-looking product screengrab beneath. Use the R3F grain over Clean's brand color to make the flat marketing surface feel deposited-in-material rather than screen-native.

## Hildén & Kaira — https://www.awwwards.com/sites/hilden-kaira

WHAT IT DOES: A Helsinki content-agency portfolio whose signature is a case study slider that renders LIVE performance data (organic views, likes, top-3 videos per client) pulled daily via Airtable/Make no-code automation, rendered as a horizontal card slider with GSAP transitions. The card front shows client logo + big-number KPIs in a `grid-cols-3`; card back reveals the ranked top-3 videos as embedded TikTok/IG previews. Two-color palette (off-white / near-black) throughout to force focus on the numbers.

PACKAGES:
- webflow (production site) — for a code build: next@15 + tailwindcss@3.4
- gsap@3 + gsap/ScrollTrigger + gsap/Draggable
- swiperjs@11 (or embla-carousel-react@8)
- airtable-js@0.12 (server) OR direct Contentful GraphQL
- @tanstack/react-query@5 (client cache + revalidation)
- react-intersection-observer
- next/font — Neue Haas Grotesk + Söhne Mono
- react-countup (KPI odometer)

IMPLEMENTATION:
1. Data layer: Airtable base with one row per client, columns [logo_url, views_30d, likes_30d, top_video_1_url, top_video_2_url, top_video_3_url]. Nightly Make automation refreshes counts via TikTok/IG Graph API.
2. Server route `/api/case-studies` fetches Airtable, caches 24h in Redis, returns JSON.
3. Slider: `<div class="grid grid-flow-col auto-cols-[minmax(480px,60vw)] gap-6 overflow-x-auto snap-x snap-mandatory">`. Each card `<article class="snap-start rounded-3xl bg-neutral-100 aspect-[4/5] p-10 flex flex-col justify-between">`.
4. KPI block: `<CountUp end={views} duration={2.2} separator="," />` inside `text-[clamp(64px,9vw,160px)] font-medium tracking-[-0.03em] tabular-nums`. Label in `text-xs uppercase tracking-widest text-neutral-500`.
5. Card flip on click: `gsap.to(card, { rotationY: 180, duration: 0.8, ease: 'power4.inOut', transformPerspective: 1200 })`. Back face contains 3 stacked video embeds with rank chips (01, 02, 03).
6. ScrollTrigger reveals: cards enter from `x: 60, opacity: 0` with `stagger: 0.08, ease: 'expo.out'`.
7. Responsive: `<md` becomes vertical stack; card flip becomes bottom-sheet.
8. Perf: lazy-mount video iframes only when card is visible via IntersectionObserver `rootMargin: '200px'`.

WHY IT WORKS: Live numbers make the portfolio a proof engine — every visit shows different data, so the site itself demonstrates the agency's core competency (organic growth) without a single testimonial. Two-color restraint uses achromatic contrast to bury decoration and elevate the tabular-nums KPIs. The card-flip metaphor rewards curiosity (interaction-as-content).

GOOD: Portfolio that updates itself daily — competitors can't fake this without the same pipeline.
BAD: Fully dependent on third-party API uptime; a broken data feed shows either stale numbers or zeros, both of which hurt credibility.

CLEAN APPLICATION: Apply this to Clean's **customer proof section on the landing page (or a dedicated /results route)** by building a live-data card slider showing real Clean customer outcomes — meetings booked (last 30d), replies generated, MQL→SQL conversion delta. Wire it to a Clean-internal metrics API (not third-party) so the pipeline is defensible. Use `tabular-nums` for all KPI digits. Reserve the card flip for revealing the actual outreach email Clean generated for that customer (with logos redacted for enterprise privacy). This turns Clean's landing page from a promise into a running dashboard.

## Radian EXR — https://www.rideradian.com

WHAT IT DOES: A product launch page for an electric enduro motorbike built around a scroll-driven 3D model rotation and a bottom-anchored configurator that slides up from `translateY(100%)` on scroll-end. The hero uses a pre-rendered image sequence (280 frames, WebP) driven by scroll position — no live GL model on hero. Below the fold, a `<canvas>` R3F scene loads the actual GLTF for the configurator with real-time material swaps (frame color, wheel type, seat).

PACKAGES:
- next@15
- three@0.170 + @react-three/fiber + @react-three/drei
- @gltf-transform/core (build-time GLTF compression + KTX2 textures)
- gsap@3 + gsap/ScrollTrigger
- zustand@4 (configurator state)
- valtio (alternative)
- framer-motion@11 (UI panel transitions)
- tailwindcss@3.4
- next/font — Söhne + JetBrains Mono
- @vercel/og

IMPLEMENTATION:
1. Frame sequence hero: export 280 WebP frames at 1920×1080 from Blender. Total <9MB using WebP q=70. Preload with `link[rel=preload][as=image]`. Draw current frame to a `<canvas>` sized `100vw × 100vh` using `ctx.drawImage()`.
2. Scroll driver: `ScrollTrigger.create({ trigger: '#hero', start: 'top top', end: '+=3000', pin: true, scrub: 0.6, onUpdate: (self) => { const frame = Math.round(self.progress * 279); if (frame !== last) drawFrame(frame); } })`.
3. Configurator: R3F `<Canvas dpr={[1,2]} shadows>` with `<Suspense fallback={<Placeholder />}>`. Load `.glb` via `useGLTF('/bike.glb')` — pre-compress with meshopt + KTX2.
4. Material picker: swap via `mesh.material.color.set(hex)` in a `useControls` state hook; camera moves via `<CameraControls>` from drei with `smoothTime: 0.35`.
5. Bottom-sheet configurator: Framer Motion `<motion.div initial={{ y:'100%' }} animate={{ y: showConfig ? 0 : '100%' }} transition={{ type:'spring', stiffness:120, damping:22 }}>`.
6. Price live-updates via `useMemo` on config state; typography `font-mono tabular-nums`.
7. Responsive: `<md` drops the R3F canvas to a static hero image and a single-column config list; retains frame sequence at 720p (140 frames).
8. Perf: `frame % 2 === 0` skip every other frame draw on lower-DPR devices; use `OffscreenCanvas` in a worker for image decode.

WHY IT WORKS: Frame sequence gives cinema-grade lighting without runtime GL cost — visually indistinguishable from real-time, dramatically cheaper. The delayed configurator reveal creates commitment ladder (browse → engage → configure → reserve). Monospace pricing signals engineering credibility to a mechanically-literate buyer.

GOOD: Two-tier 3D strategy (frames for hero, real GL for interaction) — best of both worlds.
BAD: 9MB preload on hero blows LCP budget on 3G; no true fallback for reduced-motion beyond a static image.

CLEAN APPLICATION: Apply this to Clean's **landing page hero** by replacing static screenshots with a scroll-scrubbed frame sequence of the Clean product being used — cursor typing an ICP prompt, panels sliding in with discovered accounts, a lead card expanding into a full research report. 220 WebP frames at 1440×900, total <7MB. Below the fold, use the same technique that Radian uses for the configurator to build Clean's "try it live" module — a real interactive form that runs a mini-Clean workflow against a mocked dataset, with results streaming in via SSE. Monospace type for all counts (leads found, emails drafted).

## NRG | Build Your Data Center — https://www.awwwards.com/sites/nrg-build-your-data-center

WHAT IT DOES: A B2B enterprise landing built by Rogue Studio for a hyperscaler/AI-buyer audience, structured as a vertical scroll narrative with fullscreen chapter transitions, a fixed step indicator (1/6, 2/6…) rail on the right, and CSS-mask reveals for section changes. Copy is set in large `clamp()` display, alternating between white and NRG signal-orange on a graphite background. Each chapter has a WebGL-lite ambient effect (an `<canvas>` running a 2D voronoi mesh in a shader-lite JS loop, not R3F).

PACKAGES:
- next@15
- gsap@3 + gsap/ScrollTrigger + gsap/Observer
- @studio-freight/lenis
- ogl@1.0 (lightweight WebGL — smaller than three.js)
- tailwindcss@3.4
- framer-motion@11
- @vercel/og
- next/font — Aeonik Pro + IBM Plex Mono
- lucide-react

IMPLEMENTATION:
1. Grid: `<main class="relative">` with fixed right rail `<aside class="fixed right-8 top-1/2 -translate-y-1/2 flex flex-col gap-3 text-xs font-mono">` rendering active chapter index with a scaling dot indicator.
2. Chapter section: `<section class="h-[120vh] relative flex items-center">`. Text block `text-[clamp(56px,8vw,140px)] leading-[0.9] tracking-[-0.02em] font-medium`. Alternate color runs by wrapping key phrases in `<span class="text-orange-500">`.
3. Chapter transition mask: `gsap.timeline({ scrollTrigger: { trigger: section, start: 'top 80%', end: 'top 20%', scrub: true }}).fromTo(section, { clipPath: 'inset(0 0 100% 0)' }, { clipPath: 'inset(0 0 0% 0)', ease:'none' })`.
4. Ambient canvas: `ogl` `<Program>` with a fragment shader distance-fielding a 24-cell voronoi noise field, uniforms `uTime`, `uMouse`. Blend mode `screen` at `opacity: 0.25` over the section background.
5. Step indicator: Observer plugin `gsap.registerPlugin(Observer); Observer.create({ type: 'scroll', onUp: () => activateChapter(active-1), onDown: () => activateChapter(active+1) })`.
6. CTA: `<button class="group relative overflow-hidden bg-orange-500 text-black px-8 py-4 font-mono uppercase tracking-widest text-sm">` with a `::before` diagonal-sweep hover using `translateX(-101%)` → `translateX(0)` on `group-hover`.
7. Perf: pause ogl canvas via `IntersectionObserver` — only run RAF when section is visible.

WHY IT WORKS: The step indicator gives buyers a sense of narrative progress in what's effectively a 6-chapter sales deck — enterprise decision-makers appreciate structure. Signal-orange used sparingly against graphite hits ~9:1 contrast ratio on CTAs, and the mono/serif combination reads as "we know infrastructure." Voronoi ambient background suggests scale/network without being literal (avoiding the tired server-room hero shot).

GOOD: Nails the enterprise-B2B tone shift — feels like a product, not a brochure.
BAD: Aggressive `h-[120vh]` chapter heights inflate scroll length on mobile (>18 vertical screens); no jump-to-chapter.

CLEAN APPLICATION: Apply this to Clean's **pricing / enterprise page** by structuring it as a 5-chapter narrative — (1) The problem: BDR reps burn on unqualified leads. (2) The math: cost per meeting today vs Clean. (3) The system: modules explained. (4) The proof: real customer metric. (5) The commit: pricing tiers with CTA. Right-rail step indicator, `clamp()` display type, orange-accent CTAs against Clean's dark neutral. Replace voronoi with a subtle grid-warp shader that reads as "data infrastructure" — pause it off-screen for perf. This transforms Clean's pricing page from a comparison table into a sales-cycle argument.

KEY CONSTRAINTS: Clean is developer infrastructure — the page must load fast, feel engineered, and read as "this ships to production." AVOID: (1) any generic Aceternity spotlight/aurora hero — instantly signals "shadcn boilerplate" and undermines the "we build serious infra" narrative; (2) three.js in the hero — the frame sequence gives the same visual for 1/10 the runtime cost; (3) rounded-3xl candy cards — use rounded-lg or rounded-none, tighter tracking, tabular-nums; (4) glassmorphism and neon gradients — Clean is not a consumer app; (5) autoplaying background video with audio; (6) motion that runs on paint (use transforms and opacity only); (7) hero copy longer than 8 words; (8) horizontal scroll on the marketing page (Radian and MONOLOG got away with it because they're portfolios/products, not SaaS pricing decisions); (9) shipping without a `prefers-reduced-motion` static fallback; (10) LCP >2.0s or CLS >0.05 — this is the single biggest tell that a dev-infra company doesn't know what they're doing.
