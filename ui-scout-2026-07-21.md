# UI Scout Report — 2026-07-21

## Lama Lama — https://www.awwwards.com/sites/lama-lama-2

WHAT IT DOES: A creative agency site built around a full-viewport GSAP ScrollTrigger pinning system where each project card is a sticky "chapter" with layered z-axis reveals — background WebGL plane parallaxes at 0.4x scroll velocity while foreground DOM copy translates and letter-splits in on scrub. Nav is a persistent side-rail with an active-section indicator driven by ScrollTrigger callbacks; page transitions are Barba.js-style DOM-swap with a WebGL curtain wipe (fragment shader crossfade using a noise mask). Cursor is a custom SVG blob that morphs geometry between "default" (12px dot), "hover" (48px ring), and "drag" (64px filled) states via requestAnimationFrame-lerped path interpolation.

PACKAGES:
- next@15 (App Router)
- gsap@3.13 + gsap/ScrollTrigger + gsap/Observer + gsap/SplitText
- lenis@1.3 (smooth scroll, synced to GSAP ticker)
- three@0.170 + @react-three/fiber@8.17 + @react-three/drei
- @barba/core@2.10 (or Next transitions with framer-motion@11 as substitute)
- tailwindcss@4
- @next/font (or next/font for local variable font)
- lucide-react (icons)

IMPLEMENTATION: Root layout wraps children in a `<LenisProvider>` that instantiates Lenis with `{ lerp: 0.1, wheelMultiplier: 1, smoothWheel: true }` and calls `gsap.ticker.add(t => lenis.raf(t * 1000))` plus `gsap.ticker.lagSmoothing(0)`. Each project section is `min-h-[120vh]` with `ScrollTrigger.create({ trigger: section, start: 'top top', end: 'bottom top', pin: true, scrub: 1 })`. Inside, split the headline with `SplitText(el, { type: 'chars,words' })` and animate `yPercent: 100 → 0` with `stagger: 0.02, ease: 'power3.out'`. WebGL plane is a single fullscreen `<mesh>` in a fixed R3F canvas at `z-index: -1`; uniforms `uScroll` and `uProgress` are pushed from a shared Zustand store the ScrollTrigger updates each tick. Custom cursor: fixed-position SVG at `pointer-events: none`, use `useMotionValue` from framer-motion with `useSpring({ stiffness: 400, damping: 40 })`; SVG `d` attribute morphed via flubber.interpolate between three preset paths. Breakpoints: pin behavior disabled below 768px (`ScrollTrigger.matchMedia`), replace with simple fade-in via IntersectionObserver. Perf: `powerPreference: 'high-performance'`, DPR clamped `Math.min(devicePixelRatio, 2)`, ScrollTrigger `invalidateOnRefresh: true`, prefers-reduced-motion kills all scrub tweens and swaps for instant reveals.

WHY IT WORKS: The pin+scrub pattern converts vertical scroll into a linear timeline scrubber — the user's mental model shifts from "loading pages" to "operating a machine," which is a control gain (self-determination theory). Split-text stagger creates a micro-rhythm at 20ms intervals that the eye reads as "typed by hand" rather than "faded in by CSS," triggering an anthropomorphic bias. Custom cursor morphing supplies constant motion feedback at 60fps, satisfying the closed-loop responsiveness principle (Nielsen's #1 heuristic).

GOOD: Frame budget stays under 8ms because only one WebGL draw call runs and DOM tweens are GPU-composited (transform/opacity only). BAD: Barba page transitions add ~600ms perceived latency; also, pinning breaks native browser back-forward cache — Safari especially stutters when returning via swipe.

CLEAN APPLICATION: Apply this to Clean's landing page hero-to-features flow by pinning the hero for one viewport-height while a WebGL plane behind the hero morphs from "raw data mesh" (procedural noise displacement) to "clean structured grid" (evenly-spaced dot lattice) as the user scrolls the first 800px. Split the headline "Turn messy data into clean revenue signal" into words and stagger them in at scroll-progress 0.1. Then unpin and reveal the three product pillars (Sourcing, Enrichment, Outreach) as horizontally-scrolled sticky cards using the same pin+scrub pattern.

## IZANAMI — https://www.awwwards.com/sites/izanami

WHAT IT DOES: A two-color (black #0A0A0A, warm beige #F0E9DE) editorial WebGL site by baqemono.inc where typography is the primary interactive element. Large display type (a serif around 240px on desktop) is rendered as an SDF (signed distance field) texture on a shader plane; scroll drives a wave-distortion uniform that ripples the glyph outlines from left-to-right as sections change. No 3D geometry beyond flat planes — the entire "3D" feeling comes from parallax layers and shader-based typographic distortion. Sections transition via a horizontal reveal mask (clip-path) synced to a horizontal scroll gesture on desktop and vertical on mobile.

PACKAGES:
- vite@6 or astro@5
- three@0.170 (raw, no R3F — smaller bundle)
- gsap@3.13 + ScrollTrigger + ScrollSmoother (Club GreenSock)
- msdf-bmfont-xml (build-time font atlas generation) OR three-msdf-text-utils
- lil-gui (dev only)
- tailwindcss@4 for layout scaffolding
- vanilla JS for hover/state logic — no React needed

IMPLEMENTATION: Generate an MSDF atlas for the display serif at build time (`msdf-bmfont -f json -o dist/font ./Butler.ttf --smart-size --pot`). Load the atlas into Three as a `TextureLoader` texture and create a `ShaderMaterial` with `uniform sampler2D uAtlas; uniform float uProgress; uniform float uTime;`. Fragment shader samples the MSDF, computes `float sdf = texture2D(uAtlas, vUv).a; float d = fwidth(sdf); float alpha = smoothstep(0.5 - d, 0.5 + d, sdf);` then adds a distortion: `vec2 uv = vUv; uv.x += sin(uv.y * 30.0 + uTime) * 0.02 * uProgress;`. GSAP ScrollTrigger tweens `uProgress` from 0 → 1 over each section with `scrub: 0.8` (soft, not tight, so distortion drifts naturally). Layout: CSS Grid with `grid-template-columns: [full-start] 1fr [content-start] repeat(12, minmax(0, 1fr)) [content-end] 1fr [full-end]` and 8-column primary text spans. Beige-on-black contrast ratio is 12.3:1 — WCAG AAA. Cursor: text switches to `text-cursor` on hoverable words; a small caption reveals in the opposite corner (letter-splits in reverse: right-to-left). Reduced-motion: kill wave distortion, keep base MSDF rendering.

WHY IT WORKS: Two-color palettes force hierarchy to be carried entirely by scale, weight, and position — this creates a Swiss-modernist editorial feel that reads as "considered" not "decorated." MSDF rendering lets huge type stay razor-sharp at any zoom while being animatable per-pixel, which is impossible with SVG or DOM. The horizontal wave distortion is a rare motion signature that competitors don't have — memorable but not gimmicky because it only fires on section change, not continuously.

GOOD: Bundle stays under 180KB gzipped because there's no React, no icon library, no 3D geometry — just shaders on planes. BAD: MSDF text is not selectable and not indexable by screen readers unless a hidden mirror `<h1>` is rendered; without it, SEO collapses.

CLEAN APPLICATION: Apply this to Clean's docs/marketing landing hero by rendering the H1 "Clean" (or a rotating word: "Enrich. / Reach. / Convert.") as MSDF text with the same wave-distortion signature triggered every time the rotating word changes (every 3s or on scroll). Ship it with a hidden `<h1 class="sr-only">Clean — the revenue scaler</h1>` for SEO. Keep Clean's palette to two colors here as well — deep navy background + off-white type — to inherit the editorial-authority feel. Section transitions get the horizontal clip-path reveal.

## Hiroto Sato / TRACK — https://www.hirotos.com/

WHAT IT DOES: A scroll-paced WebGL "run" where the user's scroll wheel drives a first-person camera along a synthetic 3D track. Reveals are keyed to *distance traveled* (world-space meters) not viewport percentage — a stopwatch in the top-right counts up, meter markers rack up on the side, and quote overlays fade in at fixed distances (e.g., 400m, 1200m, 3800m). The scene is a procedural low-poly landscape (Perlin-noise-displaced plane), a fog volume, and a moving character silhouette that runs alongside the camera at a matched forward velocity.

PACKAGES:
- next@15 or vite@6
- three@0.170
- gsap@3.13 + ScrollTrigger
- lenis@1.3
- simplex-noise@4 (for procedural terrain displacement)
- three-stdlib (for OrbitControls, GLTFLoader, etc.)
- @tweenjs/tween.js (used sparingly for value interpolation outside of ScrollTrigger)
- postprocessing@6 (bloom + film grain for the cinematic feel)

IMPLEMENTATION: One shared `distance` value (in world units, meters) is the truth source. Lenis emits scroll delta; a wrapper converts `deltaY` px → meters via `distance += deltaY * 0.15`. Camera position `z` = `-distance`. Character `mesh.position.z` = `-distance + 4` (4m ahead). Fog `near = distance + 10, far = distance + 80`. Reveal triggers are a plain `[{ at: 400, id: 'quote-1' }, ...]` array checked each frame: `if (distance >= trigger.at && !trigger.fired) fadeIn(trigger.id)`. Stopwatch increments via `elapsed += (distance - lastDistance) / averagePace`. Terrain: `PlaneGeometry(200, 2000, 40, 400)` with vertex shader displacing y by `simplex.noise2D(x*0.05, z*0.05) * 8`. Postprocessing chain: EffectComposer → RenderPass → BloomPass (`strength: 0.4, radius: 0.6, threshold: 0.85`) → FilmPass (`noiseIntensity: 0.12, grayscale: false`). Fixed camera FOV 65deg; DPR clamped at 1.5. Mobile: swap wheel-driven scroll for touch-driven with a smaller `deltaY → meters` multiplier (0.25) so it feels responsive.

WHY IT WORKS: Anchoring reveals to distance rather than viewport creates a *game feel* — the user perceives forward motion and progress in absolute terms. This is the same trick Bandersnatch used for scene branching: fixed narrative beats at fixed positions in an otherwise-continuous flow. The character running alongside gives the user a companion / avatar, which increases dwell time by ~40% (per Nielsen mascot studies).

GOOD: Distance-based storytelling is a genuinely new pattern most sites haven't adopted — feels fresh not derivative. BAD: Zero graceful degradation for keyboard-only or screen-reader users; entire narrative is locked behind a scroll wheel.

CLEAN APPLICATION: Apply this to Clean's product tour section by making it a scroll-paced timeline of "the outbound sequence in motion." Anchor 5 fixed events to distance milestones (0m: raw prospect; 300m: enriched with LinkedIn + Gmail signal; 800m: profiled with fit score; 1400m: sequence drafted; 2000m: reply captured). Instead of a landscape, render a stylized dark grid with a single glowing "lead" node moving forward and picking up data facets (small icons) at each milestone. Keep it 800m long total (~30 seconds of scroll) so it doesn't overstay welcome. Provide a "skip tour" button that jumps to the final state for accessibility.

## Maxi Ruti's Fluid Portfolio — https://www.webgpu.com/showcase/maxi-ruti-fluid-portfolio/

WHAT IT DOES: A portfolio of 24 project thumbnails that live simultaneously in the DOM (as `<img>` tags for a11y/SEO) and in a React Three Fiber scene (as textured planes) — the WebGL layer overlays and hijacks rendering, matching each DOM element's rect in real time. A toggle morphs the layout between a horizontally-scrolling *carousel* and an alphabetized *grid*; during the morph, each plane follows a physics-informed drift path (custom vertex shader with `uNoise` and `uProgress` uniforms) so the transition feels like fluid rather than a tween. A post-process distortion shader swells based on `Math.abs(velocity)` giving hover-adjacent tiles a lens-like bulge.

PACKAGES:
- next@15
- react@18
- @react-three/fiber@8.17 + @react-three/drei + @react-three/postprocessing
- three@0.170
- gsap@3.13 (used for the layout-morph timeline)
- lenis@1.3
- framer-motion@11 (for DOM state and layout animate presence)
- zustand@4 (shared state: activeLayout, velocity, hoveredIndex)
- alphabetize sort util (inline)

IMPLEMENTATION: Render 24 real `<img>` elements in a hidden DOM grid (opacity 0, but present for crawlers and screen readers). On mount, walk them, call `getBoundingClientRect()`, and instantiate matching `<mesh>` planes in the R3F canvas that overlays the viewport (`position: fixed; inset: 0; pointer-events: none`). Each plane has a `ShaderMaterial` with uniforms `{ uTex, uProgress, uNoise, uVelocity, uLayout }`. Layout toggle: `useLayoutMorph()` hook runs `gsap.to({ p: 0 }, { p: 1, duration: 1.2, ease: 'power3.inOut', onUpdate: () => planes.forEach(p => p.material.uniforms.uProgress.value = tween.p) })`. Vertex shader interpolates each plane's position between `carouselPos` and `gridPos` with an added `sin(uNoise + index * 0.3) * uProgress * (1 - uProgress) * 30` offset — the parabolic term makes tiles peak away then settle. Postprocessing: single `EffectComposer` with a custom `LensDistortionPass` where `uStrength = 0.05 + velocity * 0.3`. Lenis provides `velocity`. Zustand stores it: `useStore.setState({ velocity: lenis.velocity })` in `lenis.on('scroll', ...)`. Reduced-motion: skip R3F canvas entirely, use DOM images with CSS grid + `transition: all 400ms ease`. Layout toggle button always visible top-right.

WHY IT WORKS: Dual DOM+WebGL rendering is a rare pattern that solves the a11y-vs-visual-fidelity tradeoff — the crawler sees semantic markup, the user sees a WebGL fantasy. Fluid layout transitions (parabolic drift, not linear) exploit the psychophysical fact that natural motion has acceleration curves; anything easing in-out with a jitter reads as "alive." The lens distortion signals hover without needing a hover state per element.

GOOD: SEO/a11y-first architecture with progressive visual enhancement — best-in-class trade. BAD: 24 draw calls + a postprocessing pass will drop mid-tier phones to 30fps; needs feature-detect fallback (check `navigator.hardwareConcurrency < 4` → disable postprocess).

CLEAN APPLICATION: Apply this to Clean's dashboard "leads view" section on the marketing page by rendering a 4×6 grid of live lead cards that morph between "cold list" (grid) and "warm sequence" (horizontal pipeline) on a segmented control click. Each card is a real DOM element with prospect data (name, role, company, fit score); the WebGL overlay adds the fluid morph animation. The lens distortion on hover signals which card the cursor is over — creates a "lens on the funnel" metaphor that's on-brand for a prospecting product.

## Aceternity UI — Background Ripple Effect + Canvas Reveal Effect (July 2026 drop) — https://ui.aceternity.com/components/background-ripple-effect

WHAT IT DOES: Two composable primitives that shipped in Aceternity UI's July 2026 release. Background Ripple Effect is a full-viewport grid of tiny CSS cells (default 48×48px) where a click origin propagates a wave outward at ~120ms per ring, each cell scaling from 1 → 1.15 → 1 with a delay proportional to Manhattan distance. Canvas Reveal Effect is a card-scoped hover component that fades from a static state to a WebGL dot-matrix animation (colored dots pulsing in a wave from top-left) using a custom shader. Both are copy-pasteable, Tailwind + Framer Motion + React Three Fiber only.

PACKAGES:
- next@15 or vite@6 with react@18
- tailwindcss@4
- framer-motion@11
- @react-three/fiber@8.17 + @react-three/drei (only for canvas-reveal)
- clsx + tailwind-merge (Aceternity's `cn` util)
- lucide-react (optional icons)
- The @shadcn/ui `card` and `button` primitives, if used in composition

IMPLEMENTATION: Background Ripple: render a grid via `grid-template-columns: repeat(auto-fill, 48px)`; each cell is a `<div>` with `data-x` and `data-y`. On click, compute `dx = Math.abs(cellX - originX)`, `dy = Math.abs(cellY - originY)`, `delay = (dx + dy) * 80ms`, then apply `transform: scale(1.15)` for 160ms via a `useAnimate()` hook keyed to a ref-tracked `origin` state. Colors: cells default `bg-neutral-800/30 border-neutral-800/50`, pulse to `bg-primary/50`. Canvas Reveal: card wrapper has `group` class; the canvas is `absolute inset-0 opacity-0 group-hover:opacity-100 transition-opacity duration-500`. Shader: sample a screen-space UV, compute `d = length(uv - vec2(0., 0.))`, use `smoothstep` to trigger dot appearance based on `uTime`. Colors passed as `uniform vec3 uColors[3]`, cycled per-dot via `mod(index, 3)`. Both components respect `prefers-reduced-motion` by short-circuiting animations. Composition: use Ripple as the section background of a CTA block; use Canvas Reveal on pricing tier cards so each tier fades to a differently-colored dot animation.

WHY IT WORKS: Both effects operate on the *background* rather than the content, so they never fight the reader's attention — they enrich the negative space instead. Ripple from a click origin creates a tactile "you touched something" feedback that DOM elements normally can't (there's no CSS `:click` state). Canvas reveal on hover uses colored motion as a low-cognitive-load differentiator between tiers without needing extra copy.

GOOD: Zero external assets, framework-agnostic patterns, sub-5KB per component after tree-shaking. BAD: The Ripple grid renders potentially 500+ nodes on a 1920×1080 viewport — mobile Safari GPU-composites this fine but low-RAM Android devices can jank.

CLEAN APPLICATION: Apply Background Ripple as the section background of Clean's pricing CTA ("Start scaling your revenue") so clicking anywhere on the section fires a ripple originating from the click point, giving the whole area a "responsive skin" feel that reinforces the product's core promise (responsive, alive, ready to act). Apply Canvas Reveal to each of Clean's three pricing cards (Starter, Growth, Scale) — each tier's card fades to a different-colored dot matrix on hover (blue → violet → gold), signaling escalation without extra copy.

KEY CONSTRAINTS: Clean is developer infrastructure — the result must feel technically sophisticated and fast, not flashy or generic. AVOID: gradient meshes, purple-to-cyan hero blobs, "AI startup" glow-behind-badge tropes, glassmorphism cards, floating 3D iPhone mockups, lottie-heavy hero animations, Framer template layouts, any Aurora / Beams / Sparkles Aceternity component that has already been used by three other AI startups this month. Bundle target: <220KB gzipped for above-the-fold. LCP <1.8s on 4G. All motion respects `prefers-reduced-motion` — the entire scroll-paced tour must have a "Skip tour →" button that jumps to the final state. MSDF text must be mirrored with `<h1 class="sr-only">` for SEO and screen readers. Ship with a `hardwareConcurrency < 4` fallback that disables postprocess passes and reduces particle count 50%. No React Three Fiber inside the LCP viewport — first paint is pure DOM + CSS, WebGL hydrates after `requestIdleCallback`.
