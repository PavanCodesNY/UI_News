# UI Scout Report — 2026-05-21

## 1. Aino — https://aino.agency/

WHAT IT DOES: Renders the entire site as a uniform monospace character grid (roughly 80×40 cells of one-sized `ch` units) where every visual element — headings, navigation, illustrations, transitions, even mini-games — is composed by mutating individual ASCII glyphs in that grid. Hover, click, and scroll inputs feed a lightweight 2D rigid-body solver (Verlet-style point/constraint physics) that pushes glyphs around like beads on a sheet, then snaps them back via spring forces. The whole runtime ships in ~30kB of JS with no WebGL — pure DOM/Canvas2D — and the rebuilt site won Awwwards SOTD + Developer Award on 2026-05-20.

PACKAGES:
- `next@14` or `astro@4` (static delivery — the production bundle is plain HTML + a single JS chunk)
- `typescript@5`
- Custom Verlet physics — no library; mirror with `matter-js@0.19` or roll your own (~150 LOC)
- `tailwindcss@3` for typography tokens only (line-height: 1, monospace fonts)
- Font: `JetBrains Mono` (variable) or `IBM Plex Mono` — both free via `@fontsource-variable/jetbrains-mono`
- `gsap@3.12` (optional, for tween-driven glyph swaps) or `motion@11` (Framer Motion successor)
- `lenis@1.1` for inertial scroll
- No icon library — every "icon" is composed from `─│┌┐└┘╱╲▓▒░█` box-drawing characters

IMPLEMENTATION:
1. Build a `<Grid />` component that maintains a 2D array of `{ char, color, dx, dy, vx, vy }` cells. Render either to a Canvas2D context (`ctx.font = '14px JetBrains Mono'`, `ctx.fillText(cell.char, x*charW, y*charH)`) or to one absolutely-positioned `<span>` per cell when interaction precision is needed.
2. Compute `charW` and `charH` once with `ctx.measureText('M')` and `parseFloat(getComputedStyle(el).lineHeight)`. Set the `viewport` cell count to `Math.floor(width / charW)` × `Math.floor(height / charH)` on resize.
3. Physics tick at `requestAnimationFrame`: for each "live" cell apply gravity (`vy += 0.18`), pointer attraction/repulsion (radius 6 cells, force = `1 / d²`), and a spring back to its grid home (`k = 0.06`, `damping = 0.86`). Integrate Verlet: `pos += (pos - prevPos) * damping + accel`. Cap to ~1.5k active cells.
4. Layout sections by writing strings into rectangular sub-regions of the grid. Hero text: ASCII-art a 4× scaled wordmark using `figlet` at build time, persist as a static array.
5. Transitions between sections: dissolve the current sub-region by randomly swapping cells through `[' ', '·', '∙', '•', '█']` over 600ms, then write the next layout. Use a Halton sequence for visually-pleasing random order.
6. Color: one accent (`#FF5500` or `#10B981`) on `#0A0A0A`. Add a CRT scanline overlay via `linear-gradient(transparent 50%, rgba(0,0,0,0.25) 50%)` at `background-size: 100% 3px`.
7. Performance: pause physics when `IntersectionObserver` says the grid is offscreen; throttle to 30fps on devices with `navigator.hardwareConcurrency < 4`; full static fallback (no JS) renders identical layout at lower fidelity.

WHY IT WORKS: The single-grid constraint creates extreme visual coherence — there is exactly one rhythm, one type size, one alignment grid — which makes the surface feel engineered rather than designed. Physics-driven micro-chaos against that rigid grid is pure tension/release Gestalt: the eye locks onto the regularity, and movement reads as alive precisely because the substrate is so still.

GOOD / BAD:
- GOOD: 30kB total JS, zero WebGL, fully accessible fallback, instant TTI even on 3G.
- BAD: Pointer-only physics excludes touch-only nuance, and the monospace grid is hostile to anything containing long-form prose or product screenshots.

CLEAN APPLICATION: Apply this to Clean's landing-page hero by rendering "Clean — the context layer for agents" as an ASCII grid where each glyph represents one token streaming through the MCP proxy. On mouse-near, the glyphs flow toward the cursor with the Verlet solver, simulating tokens being routed. Replace the bottom-of-hero CTA row with a live `tail -f` style scroll of (synthetic) sanitized agent calls printed glyph-by-glyph at 60 chars/sec. Keep the entire hero under 35kB JS and ship a fully-rendered noscript fallback so it appears instant from cold cache.

## 2. Where Worlds Take Shape (Paodao) — https://paodao.fr/

WHAT IT DOES: A WebGL-first portfolio that ships as a third-person 3D environment — the visitor controls a small character with WASD/touch joystick, walks through procedurally-generated grass and stylized terrain, and "discovers" each project by approaching its in-world artifact, which then triggers a UI overlay. Uses parallax occlusion mapping for terrain detail, instanced grass blades (~150k instances), and a custom gameplay loop with collision against a heightmap. Awwwards SOTD + Developer Award, 2026-05-03.

PACKAGES:
- `three@0.170`
- `@react-three/fiber@8`
- `@react-three/drei@9` (`Sky`, `PointerLockControls`, `useGLTF`, `Instances`)
- `@react-three/rapier@1.4` for character-vs-heightmap collision
- `leva@0.9` for dev tuning (strip from prod)
- `gsap@3.12` for camera tweens between "discovery" and "browse" states
- `react@18`, `next@14`
- `nipplejs@0.10` for the mobile joystick
- Custom GLSL chunks for the grass shader (vertex-bend by wind noise)
- `howler@2.2` for ambient and footstep audio (toggle muted by default)

IMPLEMENTATION:
1. Scene root in `<Canvas dpr={[1, 1.5]} gl={{ antialias: false, powerPreference: 'high-performance' }}>`. Force MSAA off; rely on FXAA via `postprocessing@6.36`.
2. Terrain: a 512×512 plane subdivided 256×256, displaced in the vertex shader from a tileable noise texture sampled twice (low + high octave). Same heightmap is exported to a `Float32Array` for the Rapier `Heightfield` collider.
3. Grass: one `<Instances>` mesh of ~150k blades placed via blue-noise scatter. Vertex shader bends each blade with `sin(time*0.6 + worldX*0.15) * (uv.y^2)` and tints with a gradient ramp from base (`#3B5C2E`) to tip (`#C9D67A`).
4. Character: a low-poly GLB capsule with a `KinematicCharacterController` from Rapier. Camera follows on a damped orbit (`damping = 0.08`, max pitch `0.4`).
5. Project markers: place `<Html occlude>` from drei at each artifact's world position. When the character enters a 4m sphere trigger, GSAP tweens camera to the artifact's `lookAt`, fades a fullscreen project sheet in over 700ms (`ease: 'expo.out'`), and pauses input.
6. Performance gates: detect GPU tier with `detect-gpu@5`; on tier ≤ 1 render a static fly-through video (`<video autoplay muted playsinline>`) instead of the live scene. Lazy-load grass shader and Rapier WASM only after first interaction.
7. Audio: footsteps fire on the controller's `onMove` event with pitch jitter ±5%; ambient wind loop at -18dB.

WHY IT WORKS: The portfolio replaces a list with a place. Spatial memory is far stickier than scroll memory, and the small motor commitment of walking creates a sunk-cost stickiness — visitors invest 30 seconds learning the controls and then refuse to leave. The procedural grass plus stylized lighting hides the underlying low-poly geometry so the scene reads as crafted rather than gamey.

GOOD / BAD:
- GOOD: Best-in-class commitment to the "site as world" pattern; mobile joystick is genuinely usable.
- BAD: Two-second cold-start cost (Rapier WASM + assets), and SEO is degraded — Googlebot sees a `<canvas>` and a few alt strings unless you ship an `<noscript>` route.

CLEAN APPLICATION: Apply this to Clean's product-demo section by replacing the static "how it works" diagram with a tiny 3D scene where the agent (a moving capsule) walks between three stations — the Client, Clean's Proxy (a glowing pylon), and a backend MCP server. The user can grab the camera or just watch a looping autopilot. Each station, when "approached" by the auto-driven agent, opens an in-world panel showing the actual request/response (TOON-formatted JSON) being routed. Hard-cap to one r3f scene on the page, lazy-load below the fold, and ship a static SVG fallback for tier-1 GPUs and crawlers.

## 3. KVS Studio — https://www.kvs.services/

WHAT IT DOES: Studio site built as one continuous WebGL stage: project thumbnails are rendered as plane meshes scrolling along a slightly-curved rail, with the active tile inflating in 3D (z-translate + scale), gentle barrel distortion via a postprocess shader, and case-study transitions executed as full-screen WebGL slit-scan wipes. Single accent color `#FF5500` over near-black; all UI chrome is HTML overlaid on the canvas. Awwwards SOTD + Developer Award, 2026-05-18.

PACKAGES:
- `vue@3.4`, `nuxt@3.12`
- `three@0.170` (direct, not r3f — Vue stack)
- `@vueuse/core@10` for scroll and resize utilities
- `gsap@3.12` + `gsap/ScrollTrigger` + `gsap/Observer`
- `lenis@1.1`
- `postprocessing@6.36` (EffectComposer, BloomEffect, CustomEffect for slit-scan)
- `splitting@1.0` or `gsap/SplitText` for headline reveals
- Variable font: `Inter` and `JetBrains Mono`
- `@studio-freight/hamo@0.6` (rAF orchestration)

IMPLEMENTATION:
1. Persistent `<canvas>` at `position: fixed; inset: 0; z-index: 0`. All page content sits at `z-index: 1` with transparent backgrounds. Single `OrthographicCamera` with `frustumSize = 10`.
2. Project tiles: one `PlaneGeometry(1, 1)` per project, instanced or pooled, with a custom `ShaderMaterial` that samples a thumbnail texture and applies a soft RGB-shift (`split = 0.004 * abs(velocity)`) driven by `Lenis.velocity`.
3. Scroll → camera: `ScrollTrigger.create({ scrub: 0.6 })` maps document scroll to a camera-x translation along a quadratic Bezier path so the tile strip reads as a gentle arc.
4. Active tile is whichever plane intersects the camera's forward ray within ±0.5 units. Animate `scale` `1 → 1.12` and `position.z` `0 → 0.4` on enter with `gsap.to(mesh.position, { z: 0.4, duration: 0.6, ease: 'expo.out' })`.
5. Case-study transition: on tile click, GSAP timeline (a) freezes Lenis, (b) ramps a `uProgress` uniform `0 → 1` over 1.1s driving a slit-scan distortion (`offset.x += sin(uv.y * 80.0) * progress * 0.5`), (c) swaps page content, (d) ramps `uProgress` back.
6. Bloom: `BloomEffect({ intensity: 0.35, luminanceThreshold: 0.85 })`. Keep DPR clamped to 1.5 max.
7. Audio cue on tile change: a 60ms `AudioBufferSourceNode` click at -22dB, gated behind a one-time user gesture.

WHY IT WORKS: Continuous canvas means there is never a page reload "flash" — narrative continuity is preserved across the whole site, which is exactly the feeling a studio that builds digital products wants to project. The single high-saturation accent against deep black sets up a 14:1+ contrast that makes the active tile pop without any extra glow effects.

GOOD / BAD:
- GOOD: Slit-scan transitions are buttery; the one-color discipline is rare and high-status.
- BAD: Studio-site cliché territory — without the technical polish this pattern collapses into "agency reel." Without `prefers-reduced-motion` handling it would also be a vestibular hazard.

CLEAN APPLICATION: Apply this to Clean's dashboard analytics view by reusing the curved-rail metaphor for the request-history timeline: render each agent call as a tile on a horizontally-scrolling WebGL rail, with the active call inflating to show its full TOON payload, latency waterfall, and tool tree. Use the slit-scan wipe as the transition between "list view" and "single request" — it gives an enterprise admin surface a moment of designed motion without resorting to confetti. Cap at 200 visible tiles via virtualization; everything older streams in on scroll.

## 4. Fourmula AI — https://fourmula.ai/

WHAT IT DOES: AI product-image generation marketing site built on Webflow + GSAP. The hero is a split-rail layout where the left column scrolls through plain product photos and the right column scroll-scrubs through generated AI renders of the same product; both columns are pinned vertically while the user scrolls and the comparison reveal is driven by a horizontal mask. Awwwards SOTD 2026-05-02.

PACKAGES:
- `webflow` (or replicate in `next@14`)
- `gsap@3.12` + `gsap/ScrollTrigger` + `gsap/SplitText`
- `lenis@1.1`
- `tailwindcss@3` (if not Webflow)
- `framer-motion@11` (alternative animation engine if going code-first)
- Font: `Söhne` or `Inter Display` for headlines, `Inter` for body
- Icons: `lucide-react@0.451` or inline SVG only
- Image delivery: `next/image` with AVIF + responsive `srcset`

IMPLEMENTATION:
1. Pin section: `ScrollTrigger.create({ trigger: '#compare', start: 'top top', end: '+=200%', pin: true, scrub: 0.5 })`. Inside, two stacked `<div>` layers fill the viewport.
2. Reveal mask: bottom layer (AI renders) is full-opacity. Top layer (plain photos) has `clip-path: inset(0 var(--reveal) 0 0)`. Drive `--reveal` from `0%` to `100%` via a GSAP `gsap.to('#compare', { '--reveal': '100%', scrollTrigger: { scrub } })`.
3. Within each side, image stacks change as the user passes thirds of the scroll range — implement with GSAP timeline labels (`tl.addLabel('shot2', 0.33)`).
4. Headline above hero uses `SplitText` per-character with a 14ms stagger and `clip-path: inset(0 0 100% 0)` reveal — total 480ms.
5. Below the fold, a three-up feature grid uses CSS Grid with `grid-template-columns: 1.2fr 1fr 1fr`. Cards lift on hover with `transform: translateY(-6px)` + a 24px soft shadow, transitioned over 220ms `cubic-bezier(0.22, 1, 0.36, 1)`.
6. Mobile: drop pin behavior, switch to a simple swipe-comparison `<input type="range">` synced to `--reveal`.
7. Performance: preload only the first two image pairs; lazy others. Use `loading="lazy"` and `decoding="async"`. Lighthouse perf ≥ 90 is the bar.

WHY IT WORKS: The before/after compare is the single most persuasive visual primitive in product marketing because it does the user's mental math for them. Wrapping it in a pinned scroll converts the decision into a kinesthetic experience — the buyer *swipes* their way to belief. Two-color discipline (warm cream + ink) keeps the eye on the imagery instead of the chrome.

GOOD / BAD:
- GOOD: Pin + clip-path is GPU-cheap and works everywhere; story arc is unmistakable.
- BAD: This is now a Webflow cliché. Without truly differentiated imagery (Fourmula has it; most knockoffs won't), the structure feels off-the-shelf.

CLEAN APPLICATION: Apply this to Clean's pricing/value page by replacing the AI-image compare with a code-payload compare: left rail shows a raw JSON tool-use response (verbose, ~3kB); right rail shows the same response after Clean's TOON compression (~700 bytes). Pin the section, scrub the reveal as the user scrolls, and animate a running "tokens saved" counter at the top from 0 → actual ratio with `gsap.to({ val: 0 }, { val: 76, snap: 'val' })`. End the pinned scroll with a CTA that drops out of the comparison into a `npm i @clean/proxy` install line.

## 5. Aceternity UI — Background Ripple Effect + Spotlight New — https://ui.aceternity.com/components/background-ripple-effect

WHAT IT DOES: Two newest-drop primitives from the Aceternity registry. (a) **Background Ripple** is a CSS-grid lattice of small square cells where a pointer click triggers a Manhattan-distance ripple — cells within a growing radius animate `opacity` and `border-color` outward, then decay. (b) **Spotlight New** is a hero overlay with two conic-gradient SVG light cones (left + right), each animated via Framer Motion's `motion.div` with `filter: blur(30px)` and `mix-blend-mode: lighten`, configurable per-side intensity, position, and color.

PACKAGES:
- `react@18`, `next@14`
- `tailwindcss@3` (with `tailwindcss-animate` plugin)
- `framer-motion@11` (now published as `motion@11`)
- `clsx@2` + `tailwind-merge@2` (via Aceternity's `cn` util)
- `lucide-react@0.451`
- Aceternity registry CLI: `npx shadcn@latest add https://ui.aceternity.com/registry/spotlight-new.json`

IMPLEMENTATION:
1. Ripple grid: render an N×M CSS grid (`grid-template-columns: repeat(40, 1fr)`) of `<div>` cells. Each cell is `aspect-square`, `border: 1px solid hsl(var(--ripple-line))`, `background: transparent`.
2. On `onClick`, compute the clicked cell's `[col, row]`. For every cell, compute `d = max(|col - cx|, |row - cy|)` (Chebyshev) or `|col - cx| + |row - cy|` (Manhattan). Apply `transition-delay: ${d * 30}ms` and toggle a class that sets `background: hsl(var(--ripple-fill))` for 240ms before reverting.
3. Use `useRef` to map cell elements; avoid re-rendering React on click by mutating `element.style.transitionDelay` directly.
4. Spotlight New: two absolutely-positioned `<svg>` elements with `viewBox="0 0 3787 2842"` and a radial gradient feathered from `hsl(210 100% 85% / 0.18)` to transparent. Wrap each in `<motion.div animate={{ x: [0, 40, -10, 0], y: [0, -20, 10, 0] }} transition={{ duration: 8, repeat: Infinity, ease: 'easeInOut' }}>` for organic drift.
5. Composition: place Spotlight inside a `relative overflow-hidden` hero with `bg-black/[0.96]`; overlay headline text with `bg-gradient-to-b from-neutral-50 to-neutral-400 bg-clip-text text-transparent`.
6. Reduced motion: wrap the motion components in `useReducedMotion()` and fall back to static positions; ripple should still fire on click but skip the staggered delay.
7. Mobile: shrink the ripple grid to 24 cols; spotlight cones reduce blur to `blur(16px)` to limit composite cost.

WHY IT WORKS: Both effects exploit the foreground/background contrast principle — the ripple lattice gives the background its own micro-rhythm so it competes a little with foreground content, signaling that the surface is *alive* without yelling. Spotlight New uses additive light, which the eye reads as depth, against a near-black ground; pairing it with a gradient-clipped headline borrows the same "lit-from-behind" optical trick the Apple iPhone pages use.

GOOD / BAD:
- GOOD: Plug-and-play primitives that don't require WebGL; copy-paste, tweak two CSS vars, ship.
- BAD: Wildly overused — every YC vibe-coded startup ships Spotlight. If you don't customize the colors and motion params, your site looks like every other Aceternity-defaulted site.

CLEAN APPLICATION: Apply this to Clean's docs navigation by using Background Ripple as the canvas behind the docs sidebar header — when the user clicks the Clean logo, a ripple fires from the click point across the lattice, then returns. Use Spotlight New on the dev-tools/Quickstart page header with the left cone tinted Clean's brand green `hsl(152 76% 60%)` and the right cone tinted electric blue `hsl(210 100% 70%)`, drift duration 12s, reduced-motion fallback to a single static gradient. Avoid using Spotlight on the marketing landing hero — too generic, every competitor uses it.

## BUILD DIRECTIVE

TARGET: Clean's landing-page hero (https://tryclean.ai)

APPROACH: Build a new hero that combines Aino's ASCII-grid + 2D physics substrate (technique 1) with the pinned scroll-driven payload compare from Fourmula AI (technique 4), and treat the rest of the page as the calm payoff. The hero canvas is a monospace character grid (Canvas2D, ~30kB JS, no WebGL) where the headline "The context layer for agents" composes itself by streaming characters in from offscreen, settling under spring physics. On pointer-move, glyphs in a 6-cell radius push away with `1/d²` falloff and snap back. Below the hero, the user scrolls into a pinned compare section: left rail is the raw JSON of a real `tools/list` MCP response; right rail is the same response after Clean's TOON compression, with a live "bytes saved" counter scrubbing 0 → 76% as the reveal mask sweeps. End the pinned scroll with a one-line install CTA that types itself out (`npm i @clean/proxy`) followed by a single primary button. No Spotlight, no gradient text orbs, no glassmorphism.

STACK:
- `next@14` (App Router, RSC for everything except the hero canvas)
- `typescript@5`
- `tailwindcss@3` + `tailwindcss-animate`
- `gsap@3.12` + `gsap/ScrollTrigger` + `gsap/SplitText`
- `lenis@1.1`
- `@fontsource-variable/jetbrains-mono` and `@fontsource-variable/inter`
- Custom Verlet solver (~150 LOC, no library)
- `lucide-react@0.451` (sparingly — copy/check/arrow only)
- `clsx@2` + `tailwind-merge@2`
- No WebGL, no Three.js, no Spline, no Rive for the hero
- Image delivery via `next/image`, AVIF preferred

REFERENCE URLS:
- https://aino.agency/ — for the ASCII grid + physics hero substrate and the 30kB JS discipline
- https://fourmula.ai/ — for the pinned scroll compare with scrubbed clip-path reveal
- https://www.kvs.services/ — for the one-accent-color discipline and the slit-scan transition between sections (use sparingly, only between hero and compare)

KEY CONSTRAINTS:
- Clean is developer infrastructure. The hero must feel like reading `top` in a terminal, not like a SaaS landing page. Avoid: gradient blobs, glow rings, glass cards, oversized headline orbs, generic "AI sparkle" iconography, Spotlight/Aurora/Beams components, anything that looks default-Aceternity, hero videos, autoplaying 3D mascots, lottie animations of robots, "trusted by" logo carousels above the fold.
- Total hero JS budget: 35kB gzipped. Lighthouse Perf ≥ 95 mobile.
- Must ship a fully-rendered `<noscript>` fallback that displays the headline and CTA — Googlebot and email-scrapers should see real text, not `<canvas>`.
- Respect `prefers-reduced-motion`: physics pauses, ASCII still renders, scroll-compare degrades to a static side-by-side.
- One accent color only: Clean green (`hsl(152 76% 60%)`) on near-black (`#0A0A0B`). No secondary accent. No purple.
- Copy is technical and concrete. Headline names the artifact (the proxy, the format, the routing). No "supercharge," no "unlock," no "AI-native," no rocket emojis.
