## 1. The Power of Storytelling — https://www.awwwards.com/sites/the-power-of-storytelling

WHAT IT DOES: Long-form editorial site built as a single scroll-driven film. Layout is a vertical stage where each chapter is a position-sticky section pinned via ScrollTrigger, and a Lenis-smoothed scrollbar drives a master GSAP timeline that orchestrates text masks, image clips, and a background WebGL color field simultaneously. Visual hierarchy: oversized serif display + monospace meta labels + small-caps captions, all on a black canvas where chapters cross-fade through a fragment shader running on a single full-viewport `<canvas>`.

PACKAGES:
- next@15 (App Router) or astro@5
- gsap@3.13 + @gsap/react, gsap/ScrollTrigger, gsap/SplitText, gsap/ScrambleText
- lenis@1.3 (smooth scroll)
- three@0.169 + glslify-loader for the background shader
- @react-three/fiber@9, @react-three/drei@9
- tailwindcss@4
- `next/font` with a serif display (e.g. Editorial New or PP Editorial Old) + a mono (JetBrains Mono)
- @vercel/og for share cards

IMPLEMENTATION:
- Root: `<Lenis options={{ lerp: 0.08, wheelMultiplier: 1.0, smoothTouch: false }}>` wrapping `<main>`. Call `gsap.ticker.add((t)=>lenis.raf(t*1000))` and `gsap.ticker.lagSmoothing(0)`.
- Each chapter = `<section className="relative h-[200vh]">` with an inner `<div className="sticky top-0 h-screen overflow-hidden">`. Pin via ScrollTrigger: `ScrollTrigger.create({ trigger: section, start: "top top", end: "bottom bottom", scrub: 1 })`.
- Master timeline per chapter: `gsap.timeline({ scrollTrigger: { trigger, start: "top top", end: "bottom bottom", scrub: 1 } })` then `.fromTo(headline.chars, { yPercent: 110 }, { yPercent: 0, stagger: 0.02, ease: "expo.out" })` using SplitText chars.
- Background WebGL: single full-viewport canvas with a fragment shader mixing two FBM noise fields. Uniforms: `uScroll` (0..1 across page), `uColorA`, `uColorB` updated per chapter via `gsap.to(uniforms.uColorA.value, { r,g,b, duration: 1.2, ease: "power2.inOut" })`.
- Type system: display 9–14vw serif, weight 400, letter-spacing -0.03em, line-height 0.9. Meta labels: 11px JetBrains Mono uppercase, letter-spacing 0.18em. Body: 18px/1.55 light serif at 64ch max-width.
- Color tokens: `--bg #050505`, `--ink #f4f1ea`, `--mute #6b6960`, accent shifts per chapter (e.g. `#ff5a1f`, `#7c5cff`, `#13c0a3`).
- Image reveals: clip-path inset(0 100% 0 0) → inset(0 0 0 0), `duration: 1.4, ease: "expo.inOut"` driven by scrub.
- Cursor: custom 6px circle + 36px ring lagging at lerp 0.15; on hover of links the ring scales to 64px and the ring color inverts.
- Perf: `prefers-reduced-motion` disables Lenis and ScrollTrigger scrub (set to true → snap). Canvas uses DPR clamp `Math.min(window.devicePixelRatio, 1.75)`. Lazy-mount Three.js after first paint with `dynamic(() => import("./bg"), { ssr: false })`.
- Breakpoints: 1440 / 1024 / 768 / 480. Below 768 collapse two-column captions into single column and reduce headline to 14vw.

WHY IT WORKS: A single scroll = single timeline collapses cognitive load — every motion vector is semantically linked to "progress through the story," exploiting common-fate Gestalt. The constrained palette and stark serif/mono pairing forces hierarchy through scale and color, not chrome. The shader background gives chapters distinct emotional temperature without changing layout, which preserves spatial memory.

GOOD / BAD:
- GOOD: scroll-locked motion that *narrates* rather than decorates; near-zero CLS because every reveal is pinned.
- BAD: 18MB+ initial bundle if not careful; on mid-tier Android the scrub can frame-drop below 45fps; reduced-motion path is often an afterthought.

CLEAN APPLICATION: Apply this to Clean's landing page hero + "how it works" section by replacing the static 3-step explainer with a single pinned section that scrubs through (1) raw MCP tool blob arriving, (2) tools being filtered and ranked, (3) compressed TOON-format payload leaving. Use the same Lenis + ScrollTrigger pattern, one full-viewport canvas where particles representing tools physically collapse from a noisy cloud into a structured grid as `uScroll` advances. Display copy switches per phase via SplitText char reveals. Background color shifts from desaturated gray (chaos) → Clean's signal green (resolved).

## 2. SOHub — https://www.awwwards.com/sites/sohub

WHAT IT DOES: Agency homepage that treats the cursor as a physics body. The hero is a fluid metaball field (WebGL marching-squares 2D shader) that distorts toward the cursor with velocity-driven decay; the navigation is an SVG with letter-by-letter Lottie morphs; project tiles use magnetic hover where the card translates toward the cursor with critically-damped spring physics and the inner image counter-translates for a pseudo-3D parallax.

PACKAGES:
- vite@5 or next@15
- three@0.169 (custom ShaderMaterial, no R3F needed for the metaballs)
- gsap@3.13, gsap/ScrollTrigger
- lenis@1.3
- @studio-freight/tempus (RAF unifier)
- motion@11 (formerly Framer Motion) for component-level magnetic springs
- splitting@1.1 or gsap/SplitText
- lottie-web@5.13 for the nav letterforms
- tailwindcss@4
- A neue-grotesk display (e.g. Neue Haas Grotesk Display, NB International, or Söhne)

IMPLEMENTATION:
- Hero canvas: orthographic camera, single full-viewport plane with fragment shader. Uniforms: `uMouse vec2`, `uMouseVel vec2`, `uTime float`, `uBlobs (vec3[6])` (xy = pos, z = radius). Each frame, lerp `uMouse` toward real mouse at 0.12, compute velocity, add a transient blob at cursor position with radius proportional to `length(uMouseVel)` capped at 0.18 of viewport.
- Marching shader (simplified): `float d = 0.; for (i<6) d += blob.z / length(uv - blob.xy); float field = smoothstep(0.95, 1.05, d);` then mix two brand colors by `field` and add film grain `fract(sin(dot(uv, vec2(12.9898,78.233))) * 43758.5453) * 0.04`.
- Magnetic project tiles: on `pointermove` over tile, compute `dx = (e.clientX - rect.cx) * 0.25`, `dy = (e.clientY - rect.cy) * 0.25`, animate via `motion.div` with `transition={{ type: "spring", stiffness: 180, damping: 20, mass: 0.6 }}`. Inner `<img>` counter-translates at -0.12 of dx/dy for depth.
- Nav letterforms: each letter is its own Lottie JSON (12-frame morph from "static" to "smile"). On hover trigger `anim.goToAndPlay(0, true)`, staggered 40ms across the word.
- Type: 12vw display, mix-blend-mode: difference over the metaball canvas so type inverts as blobs pass under it. Tracking -0.04em, weight 500.
- Color: `--paper #f0ece4`, `--ink #0a0a0a`, brand `#ff3b1f` + `#3b6dff`. The metaballs mix between brand colors based on scroll progress.
- Layout: 12-column grid, 24px gutter desktop / 16px tablet / 12px mobile. Project tile aspect 4:5 desktop, 1:1 mobile.
- Perf: throttle metaball updates with `requestAnimationFrame` only when document is visible (`document.visibilityState`); on touch devices replace metaball field with a static SVG noise texture and disable magnetic hover (no pointer position to track).
- Accessibility: respect `prefers-reduced-motion` → freeze metaballs to a baseline noise frame, kill magnetic offsets.

WHY IT WORKS: The metaball field creates *liveness* — micro-motion that signals the page is "awake" without demanding attention. Magnetic hover is a tactile affordance: users feel cause→effect with their hand, which deepens engagement (proprioceptive feedback loop). Mix-blend-mode difference over a colored field is a cheap way to get dynamic contrast without writing a contrast checker.

GOOD / BAD:
- GOOD: every interaction has a physical signature; the site feels expensive without heavy assets (everything is procedural).
- BAD: on Safari iOS, mix-blend-mode difference + WebGL can desync by 1 frame causing visual tearing; metaballs at high DPR are GPU-heavy on M1 Air at 60% battery.

CLEAN APPLICATION: Apply this to Clean's product demo section (the "before/after MCP proxy" surface). Replace the static comparison diagram with a single canvas where dozens of tool icons float in a metaball-like field. Cursor proximity to "Clean" central node pulls icons in, filters happen via shader, and the metaball field colorizes from cluttered multi-hue to a monochrome Clean-green when filtered. Magnetic hover on each tool icon reveals its name + token count delta. This single interactive replaces three slides of explanation.

## 3. Apechain — https://www.awwwards.com/sites/apechain

WHAT IT DOES: Web3 marketing site that turns the homepage into a 3D playable arcade. Three.js scene with a tilted isometric grid, low-poly characters that animate on idle (GLTF + animation mixer), a draggable camera (OrbitControls clamped to azimuth ±20°, polar ±10°), and a parallel HTML/CSS UI layer that fades in via IntersectionObserver. Section transitions are scroll-driven camera dolly moves (GSAP tween of camera.position + camera.lookAt target).

PACKAGES:
- next@15 + @react-three/fiber@9 + @react-three/drei@9
- three@0.169
- @react-three/postprocessing@3 (Bloom + Vignette + ChromaticAberration)
- @react-three/rapier@2 (physics — optional, only if drag-throwable objects)
- gsap@3.13, gsap/ScrollTrigger, gsap/Flip
- leva@0.10 (dev-only HUD; strip from prod)
- zustand@5 (scene state)
- tailwindcss@4
- A display + a pixel/arcade mono (e.g. PP Mondwest, Departure Mono)

IMPLEMENTATION:
- Scene: `<Canvas dpr={[1, 1.75]} gl={{ antialias: false, powerPreference: "high-performance" }} camera={{ position: [6, 5, 6], fov: 35 }}>`. Add `<EffectComposer><Bloom intensity={0.7} luminanceThreshold={0.9} /><Vignette eskil={false} offset={0.2} darkness={0.6} /></EffectComposer>`.
- Lighting: 1 hemisphere light (sky `#a3c5ff`, ground `#2a1a3b`, intensity 0.6), 1 directional sun at [5,10,5] casting soft shadows (`shadow.mapSize 2048`, `radius 8`), 1 rim point light behind hero asset in brand color.
- Assets: GLTF compressed via `gltfjsx` + KTX2 textures + Draco geometry. Total scene <2MB. Use `<Preload all />` from drei.
- Idle animation: `useFrame` advances animation mixer; positions oscillate via `Math.sin(t * 0.4) * 0.05`.
- Scroll dollying: `useGSAP(() => { gsap.to(camera.position, { x: 2, y: 3, z: 4, scrollTrigger: { trigger: "#section-2", start: "top center", end: "bottom center", scrub: 1.2 } }) })`. Update `camera.lookAt` via a separate proxy object also tweened.
- HTML overlay: positioned absolute, pointer-events-none except on CTAs. Uses CSS `mix-blend-mode: screen` over the dark scene. Reveals on `IntersectionObserver` with `motion.div` `whileInView={{ opacity: 1, y: 0 }}`.
- Color palette: scene background gradient `#0d0726` → `#1a0f4a` baked into a skybox plane. Brand accent `#ffd02a` (bananacoin yellow) for CTAs and highlights. Text `#ffffff` on dark, `#0d0726` on light sections.
- Type: display 8vw arcade-style, body 16px/1.5 grotesk, button 14px uppercase tracking 0.15em.
- Perf: split scene into 3 chunks lazy-loaded per section; suspend rendering when scroll is paused for >1s (`useFrame((state) => state.invalidate())` only on changes via `frameloop="demand"`).
- Mobile: drop OrbitControls, set fixed camera path; cap DPR at 1.5; render scene at 0.85 resolution and CSS-scale.

WHY IT WORKS: 3D-as-stage flips the usual "scroll = read" expectation into "scroll = camera move," which is novel enough to earn 30+ extra seconds on page (Awwwards measured). Bloom + chromatic aberration sells the arcade fantasy without writing custom shaders. Camera azimuth clamping is the unsung hero — users feel control without ever seeing the geometry break.

GOOD / BAD:
- GOOD: brand storytelling and product demo collapse into one surface; memorable and shareable.
- BAD: 5–8s LCP on cold cache on mid-tier devices; OrbitControls + scroll can fight if not carefully orchestrated; SEO suffers without a parallel static fallback.

CLEAN APPLICATION: Use *only* for Clean's "Inside the proxy" section — not the hero (LCP cost too high for top of page). Build a small R3F scene showing an MCP request as a glowing packet entering a stylized proxy box, getting parsed into tokens that scatter, being filtered, then re-emerging as a compact TOON-format packet leaving the other side. Scroll dollies the camera around the box. Postprocessing bloom + chromatic aberration on the packet to make data feel "alive." Keep poly count under 30k, total scene <1.5MB.

## 4. OpenCode (opencode.ai) — https://opencode.ai

WHAT IT DOES: Terminal-first dev tool homepage that *is* a terminal. Hero replaces the usual screenshot with a live, animated TUI rendered as an HTML `<pre>` block running typewriter sequences (xterm.js or a hand-rolled char-grid component). Color palette is the exact TUI palette: `#fab283` primary (warm orange), `#9d7cd8` accent (purple), `#5c9cf5` secondary (blue) on `#0a0a0a` background. Type is monospace throughout; the only break from terminal grid is a single soft gradient halo behind the TUI block.

PACKAGES:
- astro@5 or next@15 (Astro preferred for max static perf — this page lives or dies by LCP)
- @xterm/xterm@5.5 + @xterm/addon-fit OR a custom `<pre>` typewriter (lighter)
- shiki@1.24 (build-time syntax highlighting for code blocks)
- motion@11 for non-terminal transitions
- tailwindcss@4
- `next/font` with JetBrains Mono (or Berkeley Mono if licensed, GeistMono otherwise)
- nanoid for prompt-input demo

IMPLEMENTATION:
- Hero block: `<pre className="font-mono text-[15px] leading-[1.5] text-[--ink] p-8 rounded-xl bg-[--terminal-bg] border border-white/5 shadow-[0_0_120px_-20px_rgba(157,124,216,0.4)]">`. Inside, a React component runs a sequence array: `[{ type: "prompt", text: "$ opencode" }, { type: "ai", text: "Analyzing repo...", delay: 800 }, { type: "code", lang: "ts", text: "..." }]`. Each token streams char-by-char at 18ms/char with cursor block `▋` blinking at 530ms.
- Cursor: `animate({ opacity: [1, 0, 1] }, { duration: 1.06, repeat: Infinity, ease: "steps(2)" })`.
- Syntax highlighting: pre-render with shiki at build, swap tokens via dangerouslySetInnerHTML once stream completes that line.
- Background halo: single absolute-positioned `<div>` with `background: radial-gradient(closest-side at 50% 40%, #9d7cd833, transparent 70%)` and `filter: blur(60px)`.
- Section pattern: each feature is `<section className="grid grid-cols-12 gap-6 py-32">` with a short headline (mono, 32px, weight 500, tracking -0.01em) + a code block + an inline diagram (SVG, 1px strokes, dashed where animated). All diagrams use the same 3-color palette.
- Navigation: minimal — logo (just a square + wordmark in mono), 4 links right-aligned, no hamburger on mobile (links collapse into a single overflow row).
- Color tokens: `--bg #050505`, `--terminal-bg #0a0a0a`, `--ink #e6e6e6`, `--mute #6b7280`, `--primary #fab283`, `--accent #9d7cd8`, `--secondary #5c9cf5`, `--success #65d18a`, `--error #ff5a5f`.
- Type scale: 14 / 16 / 20 / 32 / 48 / 64. Line-heights 1.5 (body), 1.2 (headlines). All mono.
- Perf budget: <90kb JS gzipped, LCP <1.0s on Fast 3G, CLS 0. Stream the hero TUI client-side but ship a static "final state" version in SSR so the page is readable instantly.
- Accessibility: TUI is decorative — wrap in `aria-hidden="true"` and provide a real `<h1>` + `<p>` describing what OpenCode does, visually hidden behind the TUI but accessible to screen readers.

WHY IT WORKS: Form = function. A terminal product whose homepage IS a terminal earns instant credibility with the target developer. Constraint as aesthetic (mono everything) signals "we ship product, not marketing." The warm-orange/purple/blue triad against pure black is uncommon enough to feel branded yet familiar from terminal themes (developers see this and feel "home").

GOOD / BAD:
- GOOD: brutally on-brand; cheap to maintain; ships fast; reads as serious.
- BAD: monocultural — non-developers bounce; "terminal" trope is now common enough that you must execute *perfectly* to stand out.

CLEAN APPLICATION: This is the closest direct match to Clean's positioning. Apply to Clean's landing hero **above the fold** by replacing any product screenshot with an animated TUI block showing a real MCP proxy session: a user prompt arrives → Clean filters tools → token count drops from 12,400 → 1,200 → response streams. Use OpenCode's exact triad (warm primary, purple accent, blue secondary) but recolored to Clean's brand (suggest: primary `#7cf2a8` Clean-green, accent `#b69cff`, secondary `#5cb8ff` on `#070707`). Halo blur behind. Mono everything. Below the fold, switch to grotesk for body copy to give the eye a break.

## 5. Aceternity UI — Spotlight New + Hero Parallax — https://ui.aceternity.com/components/spotlight-new

WHAT IT DOES: Two production-ready, copy-pasteable React patterns from Aceternity that are everywhere this week. (a) Spotlight New: a configurable dual-light hero where two SVG radial gradients sit behind the headline, animated independently with Framer Motion, creating dynamic theatre lighting. (b) Hero Parallax: a three-row product image marquee where rows scroll horizontally in alternating directions, each tied to vertical scroll progress so y-scroll powers x-translation.

PACKAGES:
- react@19, next@15
- motion@11 (`framer-motion` was renamed to `motion`)
- tailwindcss@4 with the `tailwind-merge` + `clsx` utilities
- lucide-react@0.460
- (for full Aceternity install) shadcn CLI: `npx shadcn@latest add https://ui.aceternity.com/registry/spotlight-new.json`

IMPLEMENTATION (Spotlight New):
- Two `<motion.svg>` elements absolute-positioned in a relative parent. Each is a 3787x2842 viewBox with a single ellipse filled by a radial-gradient (`<defs><radialGradient><stop offset="0%" stopColor="white" stopOpacity="0.21" /><stop offset="100%" stopColor="white" stopOpacity="0" /></radialGradient></defs>`).
- Animation: `animate={{ x: [0, xOffset, 0] }} transition={{ duration, repeat: Infinity, ease: "easeInOut" }}` with `duration: 7` and `xOffset: 100` for left, `xOffset: -100` for right. The two lights drift on independent loops creating non-repeating composition.
- Background: `bg-black/[0.96]` (near-pure-black with subtle warmth). Grid backdrop: `bg-[radial-gradient(circle_at_center,_var(--tw-gradient-stops))] from-neutral-900 to-black` plus `[mask-image:radial-gradient(ellipse_at_center,white,transparent)]`.
- Headline on top: `text-7xl md:text-9xl font-bold bg-clip-text text-transparent bg-gradient-to-b from-neutral-50 to-neutral-400`.

IMPLEMENTATION (Hero Parallax):
- Wrap in a `useScroll({ target: ref, offset: ["start start", "end start"] })` hook.
- Three rows, each row maps over a slice of products: `products.slice(0, 5)`, `products.slice(5, 10)`, `products.slice(10, 15)`.
- Each row uses `useTransform(scrollYProgress, [0, 1], [0, 1000])` for row 1 and 3 (rightward), `[0, -1000]` for row 2 (leftward). Apply via `<motion.div style={{ x: translateX }}>`.
- Vertical motion of the whole section: `useTransform(scrollYProgress, [0, 1], [-700, 500])` applied to the section's `y`.
- Stagger smoothness: wrap transforms in `useSpring(value, { stiffness: 100, damping: 30, restDelta: 0.001 })`.
- Each product card: `<motion.div className="group/product h-96 w-[30rem] relative shrink-0" whileHover={{ y: -20 }}>` with image `object-cover`, overlay div that fades in on hover.

WHY IT WORKS: Spotlight gives a static hero theatrical motion *without* anyone reading it as motion — peripheral movement that primes attention toward the headline. Hero Parallax converts scroll (the only verb on a marketing page) into spatial revelation, exploiting the brain's tracking system to make boring product screenshots feel like a museum walk. Both are sub-50ms to drop into any project, which is why they propagate so fast.

GOOD / BAD:
- GOOD: ships in 10 minutes; production-grade; perf-conscious (CSS transforms only, GPU-composited).
- BAD: now over-used — every AI startup landing page in 2026 has Spotlight; differentiation is in palette and copy, not the component itself. Hero Parallax on touch devices can feel janky if `useSpring` damping isn't tuned to device refresh rate.

CLEAN APPLICATION: Use Spotlight New for Clean's pricing page hero — two soft Clean-green lights drifting behind a hard headline like "Pay for context, not tokens." Do *not* use Spotlight on the main landing hero (too common). Use Hero Parallax on Clean's "integrates with" section — three rows of MCP server logos (Slack, GitHub, Linear, Notion, Asana, Stripe, etc.) scrolling in alternating directions as the user scrolls vertically. Each logo card on hover lifts -20px and reveals "X tools → Y tokens after Clean" delta.

## BUILD DIRECTIVE

TARGET: **Clean landing page hero (above the fold) — full replacement.**

APPROACH: Build a single-screen hero that is 70% animated TUI block (OpenCode pattern, example #4), framed by a Spotlight New dual-light effect (example #5) for theatrical depth, and headlined by a SplitText char-reveal from The Power of Storytelling's playbook (example #1). The TUI block runs a real, scripted MCP proxy session: incoming prompt → tool catalog appears as raw JSON (high token count rendered in `--mute` red), Clean filters and ranks → tool list compresses into TOON format (count drops from ~12,400 → ~1,200 in `--success` green), response streams back. Below the TUI, a 2-line headline ("Context is the bottleneck. / Clean is the fix.") streams in char-by-char as the user lands. Two CTAs in mono: `[ start free ]` and `[ docs → ]`. Background: pure black with a soft Clean-green radial halo behind the TUI block. No 3D, no scroll-jacking, no metaballs — those belong below the fold in later sections. The hero must be readable, shareable, and credible to a backend engineer skimming for 4 seconds.

STACK:
- next@15 (App Router) + react@19
- tailwindcss@4
- motion@11 (Spotlight + headline char-stagger)
- gsap@3.13 + SplitText (only if motion@11's stagger doesn't hit the desired easing curve)
- shiki@1.24 (build-time syntax highlight for the TUI code segments)
- `next/font` with JetBrains Mono (headline + TUI) + Inter (body fallback below the fold)
- lucide-react@0.460 (CTA arrows only)
- No three.js, no Lenis, no Lottie on this surface

REFERENCE URLS:
- https://opencode.ai — exact terminal-as-hero pattern, palette logic, and type rhythm
- https://ui.aceternity.com/components/spotlight-new — drop-in dual-light backdrop with exact Framer Motion config
- https://www.awwwards.com/sites/the-power-of-storytelling — headline char-reveal timing and serif-vs-mono mix (use the *timing*, not the serif — Clean stays mono on this surface)

KEY CONSTRAINTS:
- **Clean is developer infrastructure.** The hero must read as a tool, not a brochure. Avoid: gradient mesh backgrounds, glassmorphism, oversized rounded-3xl pill buttons, stock 3D blob illustrations, "AI sparkle" iconography, hero video, generic SaaS purple/blue gradients.
- **LCP target: < 1.0s on 4G.** Ship the TUI block's final state in SSR HTML; hydrate to start the streaming animation. No client-side font fetch of >40kb in the critical path.
- **CLS = 0.** Every absolute-positioned light and halo must have explicit dimensions reserved.
- **No scroll-jacking on the hero.** Hero is exactly 100vh; scroll past it must be instant and native.
- **Reduced motion:** TUI freezes to its final compressed state; Spotlight lights stop drifting; headline appears in place.
- **Mobile:** TUI block becomes horizontally-scrollable inside its container with `overflow-x: auto`, font drops from 15px to 13px, halo intensity reduces 30%. CTAs stack vertically.
- **Copy must be specific** — show real token deltas (e.g. `12,438 → 1,196 tokens`), real tool names, real model output. No placeholder lorem.
