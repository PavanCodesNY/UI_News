# UI Scout Report — 2026-06-08

## SOHub — https://www.awwwards.com/sites/sohub

WHAT IT DOES: Design-agency portfolio built on a 12-column CSS grid where every project card is a sticky-positioned panel that crossfades through a shared Three.js canvas pinned behind the DOM. As you scroll, the next card's poster image loads into a WebGL plane that occupies the same bounding box as the active card, and GSAP morphs the plane's UV offset + a Perlin-noise distortion uniform from 1.0 → 0.0 over the card's scroll range. Type is split into per-character spans by GSAP SplitText and staggered in on a 0.6s `power3.out` curve. The cursor is a 12px circle that magnetically snaps to interactive elements with a `gsap.quickTo` lerp at 0.18.

PACKAGES:
- next@15.3
- react@19
- @sanity/client@7
- three@0.165
- @react-three/fiber@9
- @react-three/drei@10
- gsap@3.13 (ScrollTrigger, ScrollSmoother, SplitText, Flip)
- lenis@1.1 (smooth scroll, integrated with ScrollTrigger via `lenis.on('scroll', ScrollTrigger.update)`)
- tailwindcss@4
- next/font with `PP Neue Montreal` (display) + `JetBrains Mono` (mono)

IMPLEMENTATION:
Component tree: `<RootLayout><LenisProvider><CanvasLayer/><CursorLayer/><Header/><main>{children}</main></LenisProvider></RootLayout>`. `CanvasLayer` is a single `<Canvas>` from `@react-three/fiber` with `dpr={[1, 1.5]}`, `gl={{ antialias: false, alpha: true, powerPreference: 'high-performance' }}`, positioned `fixed inset-0 -z-10 pointer-events-none`. Each `<ProjectCard>` registers itself with a Zustand store via `useLayoutEffect` (rect, image src, scrollProgress signal). The `<CanvasLayer>` reads the store and renders one `<mesh>` per active card with a custom shader: vertex passes through, fragment samples a `uniform sampler2D uMap`, applies `vec2 distort = vec2(snoise(vUv*4.0 + uTime*0.1)) * uReveal * 0.06`, and outputs `texture2D(uMap, vUv + distort)`. `uReveal` is driven by `gsap.to(material.uniforms.uReveal, { value: 0, ease: 'power2.inOut', scrollTrigger: { trigger: card, start: 'top 80%', end: 'top 20%', scrub: 0.8 } })`. Card layout: `grid grid-cols-12 gap-x-6 px-8`; each card is `col-span-10 col-start-2` with `min-h-[120vh]` and `position: sticky; top: 12vh`. Type stack: H1 `clamp(72px, 9vw, 168px)` `tracking-[-0.04em]` `leading-[0.92]`; body `text-[15px] leading-[1.5] text-neutral-400`. Color tokens: `--bg: #0A0A0A`, `--fg: #F2F2F2`, `--accent: #D7FF3A`. Page transitions: on route change, GSAP Flip captures the active card's canvas plane and animates it to fullscreen over 0.8s `expo.inOut` while Barba.js (or Next.js parallel routes) swaps DOM. Breakpoints: `sm:640 md:768 lg:1024 xl:1280 2xl:1536`; below `md`, disable canvas reveal (fall back to CSS `mask-image` linear gradient) and drop SplitText to whole-line stagger to save perf. Optimizations: `Canvas` is rendered with `frameloop="demand"` and `invalidate()` is called from a `useFrame` hook only while a `ScrollTrigger` is `isActive`.

WHY IT WORKS: Sticky cards create a "spotlight" rhythm — the viewer always has one focal object, exploiting Gestalt's law of focal point. The shared canvas plane crossfading between cards establishes object permanence (the image isn't replaced, it transforms), which is the same trick film editors use for invisible cuts. Lime accent (#D7FF3A) on a near-black ground hits a 14:1 contrast ratio, well above WCAG AAA, so the eye reads CTAs instantly.

GOOD / BAD:
GOOD: Single shared WebGL canvas instead of one per card — keeps draw calls under 6 and GPU memory flat regardless of project count.
BAD: SplitText fallback on iOS Safari 17 stutters during the first scroll because the font hasn't fully painted; needs a `document.fonts.ready` gate before `ScrollTrigger.refresh()`.

CLEAN APPLICATION: Apply this to Clean's landing-page hero + product showcase by replacing the project cards with three sticky "capability panels" — `MCP Proxy`, `TOON Format`, `Context Layer` — each pinned for one viewport-height of scroll. The shared WebGL canvas should render a procedural network graph (nodes + animated edges) whose layout morphs between three preset configurations via the same uniform crossfade. Use `#0A0A0A` background, `#D7FF3A` (or Clean's existing accent) for active nodes, and PP Neue Montreal for the giant `clamp(72px, 9vw, 168px)` capability headline.

## Apechain — https://www.awwwards.com/sites/apechain

WHAT IT DOES: Single-color (#0054FA) brand site for a Web3 chain, built by makemepulse, using a full-bleed WebGL hero where a generative point-cloud forms a 3D ape silhouette that morphs into the chain's wordmark on scroll. The scene is a 200k-particle `THREE.Points` system driven by a GPGPU compute shader (ping-pong FBO) that interpolates between two pre-baked position textures. Below the hero, sections use a typographic ribbon: a horizontally-pinned section where 240vw of editorial copy translates left as the user scrolls vertically, with inline 3D coin renders triggering at specific scroll progress.

PACKAGES:
- next@15
- three@0.165
- @react-three/fiber@9
- @react-three/drei@10 (`shaderMaterial`, `useFBO`)
- gsap@3.13 (ScrollTrigger, Observer)
- lenis@1.1
- @studio-freight/hamo (`useWindowSize`, `useScroll`)
- glsl-noise (snoise3)
- vite-plugin-glsl (or `raw-loader` for `.glsl` imports)
- howler@2.2 (UI sound cues, debounced and toggle-able)

IMPLEMENTATION:
Hero: `<Canvas gl={{ alpha: false }}><PointCloud count={200000} /></Canvas>`. `PointCloud` uses a 512×512 RGBA float FBO holding XYZ + life per particle. Two static `DataTexture`s hold the baked silhouettes (export from Blender via `geometry.attributes.position.array` flattened into RGBA). The compute material lerps position via `mix(texture(tA, uv).xyz, texture(tB, uv).xyz, uProgress) + curlNoise(uv*3.0 + uTime*0.15)*0.04`. A render material draws each particle as a 1.5px additive circle, color `vec3(0.0, 0.329, 0.98)` with `gl_PointSize = 2.0 + 6.0 * (1.0 - distance(uv, vec2(0.5)))`. `uProgress` is bound to a ScrollTrigger from `start: 'top top'` to `end: '+=200%'` with `scrub: 1.2`. Horizontal ribbon section: container is `position: relative; height: 240vh; overflow: hidden`; inner is `position: sticky; top: 0; height: 100vh; width: 240vw; display: flex` with `gsap.to(inner, { x: () => -(inner.scrollWidth - innerWidth), ease: 'none', scrollTrigger: { trigger: container, pin: true, scrub: 1, end: () => `+=${inner.scrollWidth - innerWidth}` } })`. Typography: a custom variable font drawn at `clamp(96px, 14vw, 280px)` with `font-variation-settings: 'wght' 700, 'wdth' 75`. Cursor: a 36px square with a 1px inner border that scales 1.6× on hover via `gsap.quickTo`. Sound: every primary CTA fires a 200ms sine-wave click at 880Hz, mixed at -18dB; mute persists in localStorage. Mobile: replace the GPGPU cloud with a pre-rendered MP4 (H.265, 1.2MB, looped) and disable the horizontal ribbon — stack sections vertically with `scroll-snap-type: y mandatory`.

WHY IT WORKS: Monochrome electric blue strips the page of decorative noise and lets motion be the entire visual hierarchy — every animation reads as signal, not chrome. The particle morph from organic (ape) to typographic (logo) is a literal demonstration of the chain's brand premise (community → infrastructure) — semantic motion, not decorative motion. Horizontal pinning re-uses vertical scroll bandwidth so editorial copy gets premium screen real estate without forcing a new gesture.

GOOD / BAD:
GOOD: GPGPU FBO approach scales to 200k particles at 60fps on a mid-range M-series Mac because particle updates run entirely on GPU.
BAD: Single saturated color is brutal on tritanopia and after 30+ seconds causes screen-burn fatigue; no high-contrast or reduced-motion toggle.

CLEAN APPLICATION: Apply this to Clean's product demo section by replacing Apechain's particle morph with a 50k-particle visualization of an MCP message flow: particles spawn at "user prompt", traverse a curved path through "Clean proxy", scatter to N tool endpoints, and converge back. Bind morph progress to scroll. Use Clean's single brand color on near-black with no other accents above the fold — let the motion carry the visual weight. Skip the sound layer (dev-tool audience finds it gimmicky), keep the horizontal-ribbon pattern for the "what Clean does" editorial section.

## Cantor8 — https://www.cantor8.io/

WHAT IT DOES: Enterprise Web3 infrastructure site (Canton Network) by Dobre Agency, structured as a single dense grid of "modular tiles" — each tile is a feature card on a 4×N CSS Grid that reveals via a CSS `clip-path: inset()` mask on scroll. The product diagram is an animated SVG flow chart with stroke-dashoffset animations triggered by IntersectionObserver. Type is a structured pair of `Söhne` for body and `Söhne Mono` for tickers/labels, with all numerical data rendered in mono — the visual signature of fintech-grade infrastructure.

PACKAGES:
- next@15
- react@19
- tailwindcss@4
- framer-motion@12 (`useScroll`, `useTransform`, `motion.svg`)
- @radix-ui/react-* (Dialog, Tabs, Tooltip — via shadcn/ui new-york)
- shadcn/ui (Rhea preset — compact dense surfaces, released April 2026)
- lucide-react@0.450
- next/font with `Söhne` (or `Inter Tight` as substitute) + `Söhne Mono` (or `JetBrains Mono`)

IMPLEMENTATION:
Layout: `<main class="mx-auto max-w-[1440px] px-8"><HeroBand/><InfrastructureGrid/><FlowDiagram/><MetricsStrip/><Footer/></main>`. `InfrastructureGrid` is `grid grid-cols-12 gap-px bg-neutral-900` (the gap acts as the divider line by exposing background through 1px gutters). Each tile: `bg-neutral-950 p-8 col-span-4 lg:col-span-3` with `border border-neutral-800/0` and hover `border-neutral-700`. Reveal animation per tile:
```jsx
<motion.div
  initial={{ clipPath: 'inset(0 100% 0 0)', opacity: 0.5 }}
  whileInView={{ clipPath: 'inset(0 0% 0 0)', opacity: 1 }}
  viewport={{ once: true, margin: '-10% 0px' }}
  transition={{ duration: 0.8, ease: [0.16, 1, 0.3, 1] }}
/>
```
FlowDiagram: a 1200×600 SVG with `<path>` strands between four nodes (Issuance → Custody → Execution → Settlement). Each path uses `stroke-dasharray: 1` and `pathLength: 1`; on intersect, `motion.path` animates `pathLength` from 0 → 1 over 1.4s with staggered 0.15s delay between strands. A particle (`<circle r="3" />`) is then animated along each path using `motion.circle` with `offsetPath: 'path(...)'` and `offsetDistance` 0% → 100% in 2.4s, looping. Color tokens: `--bg: #07090C`, `--surface: #0E1217`, `--border: #1B2128`, `--fg: #E6E9EE`, `--mono-fg: #8A95A3`, `--accent: #3B82F6`. Type rules: every number renders in `font-mono tabular-nums tracking-tight`; every label is `text-[11px] uppercase tracking-[0.16em] text-neutral-500`. MetricsStrip: a horizontal row of 5 large numbers — `text-7xl font-mono tabular-nums` — each with a 12px label below; numbers tick from 0 to target value on intersect via `useMotionValue` + `animate(value, target, { duration: 1.4, ease: 'easeOut' })`. Mobile: collapse grid to `grid-cols-1`, replace SVG flow with a vertical step-list, keep mono number ticker.

WHY IT WORKS: 1px grid gutters do the work of borders without adding a border-box — the eye reads structural rigor without visual clutter. Mono typography on every numeral is a signaling device — institutional finance audiences read mono digits as "machine-verified," and the entire trust gradient of the page rides on that single typographic choice. The cubic-bezier `[0.16, 1, 0.3, 1]` (a strong overshoot-free ease-out) on clip-path reveals feels mechanically precise rather than playful — appropriate for infrastructure.

GOOD / BAD:
GOOD: Uses CSS clip-path + IntersectionObserver instead of GSAP ScrollTrigger — ships 60KB less JS, and the animation runs entirely on the compositor thread.
BAD: Dense info architecture pushes contact-CTA past the third fold on mobile; no sticky CTA bar means a real lead-gen page would lose 30%+ conversion vs. a sticky-bottom variant.

CLEAN APPLICATION: Apply this to Clean's pricing page and "How Clean works" technical section. Build a 1px-gutter modular grid where each tile is a Clean primitive (Proxy, TOON, Context Cache, Audit Log). Use mono for every metric (latency, tokens saved, requests/sec) — this is the single highest-leverage move for credibility with an engineering buyer. Implement the SVG flow diagram for the "request lifecycle" section: User → Clean Proxy → MCP Server → Tool Endpoint, with the animated particle traveling along the path. Lock the mono-numerals + clip-path-reveal pattern into Clean's design system as the default for any data-heavy section.

## Aceternity ASCII Art Component — https://ui.aceternity.com/components/ascii-art

WHAT IT DOES: A drop-in React component that takes an image source and renders it as live, animatable ASCII characters on an HTML5 canvas. The image is sampled into a 2D luminance grid, each cell is mapped to a character from a configurable charset (15+ presets: standard, blocks, binary, braille, dots, circles, stars), and the canvas re-paints on each frame with optional animation modes — fade, typewriter (sequential character reveal), and matrix rain (top-down vertical stagger with trailing chars).

PACKAGES:
- react@19
- next@15 (optional — works standalone)
- tailwindcss@4 (for layout, not the canvas)
- motion@12 (formerly framer-motion) — for the React wrapper's enter animation
- clsx@2 + tailwind-merge@2

IMPLEMENTATION:
Component signature: `<AsciiArt src="/portrait.jpg" charset="braille" animation="matrix" fontSize={10} cellAspect={0.55} color="#D7FF3A" background="#0A0A0A" fps={30} />`. Internally: load image via `new Image()`, draw to an offscreen canvas at sample resolution `width / fontSize × height / (fontSize * cellAspect)`, read pixels with `getImageData`, compute per-cell luminance as `0.299*r + 0.587*g + 0.114*b`, map to a charset index `Math.floor((1 - lum/255) * (charset.length - 1))`. Render loop: on a foreground canvas at full display resolution, set `ctx.font = '${fontSize}px ui-monospace'`, set `ctx.fillStyle = color`, clear with `ctx.fillStyle = background; ctx.fillRect(...)`, then iterate cells and `ctx.fillText(char, x*fontSize, y*fontSize*cellAspect)`. Animation modes:
- `fade`: per-cell opacity `o = clamp((t - cellDelay)/0.4, 0, 1)` with `cellDelay = hash(x,y) * 1.2`.
- `typewriter`: cells become visible when `t * cellsPerSecond > index`, where `index = y*cols + x` (left-to-right, top-to-bottom).
- `matrix`: each column has an independent head position `head[x] = t * fallSpeed[x] + offset[x]`; cells within `head - trailLength` to `head` render with opacity gradient and a brighter "head" character one row above.
Use `requestAnimationFrame` and throttle to `fps` via a `lastFrame` accumulator. Performance: `ctx.font` is only set once per frame (set/reset is expensive). For >50k cells, switch to a single `OffscreenCanvas` rendered in a Worker and `postMessage`'d back as ImageBitmap.

Charsets (constants):
```ts
export const charsets = {
  standard: ' .:-=+*#%@',
  blocks: ' ░▒▓█',
  binary: ' 01',
  braille: ' ⠁⠃⠇⠧⠷⠿⣿',
  dots: ' ·•●',
  circles: ' ○◔◐◕●',
  stars: ' ⋅·∗*✦✧★'
};
```

WHY IT WORKS: ASCII typography in a marketing surface is a tribal signal — it reads as "we know computers from the metal up" to a developer audience. The animation modes (especially matrix and typewriter) borrow well-established cinematic shorthand, so users decode the motion as "system is alive / computing" with zero cognitive overhead. Canvas-based rendering (vs. one DOM node per character) keeps frame time under 4ms even at 20k cells.

GOOD / BAD:
GOOD: Configurable charset + animation as data, not as a code branch — every variation is one prop change, which means iteration is free during design exploration.
BAD: Pure decoration on most surfaces — applied above-the-fold it can hijack the first 600ms of perceived performance and crowd out actual messaging.

CLEAN APPLICATION: Apply this to Clean's docs landing page or "API reference" hero — render the Clean wordmark in `braille` charset with the `typewriter` animation, fontSize=12, color = Clean accent, on a `#0A0A0A` ground. Use it as the single decorative element on the docs index; everywhere else in the docs, the content density should dominate. Do NOT use ASCII art on the main marketing landing page hero — engineers will read it as cosplay if it's not anchored to a real product reference (like rendering an actual MCP message log in matrix-rain style above the proxy section).

## Codrops — Scroll-Revealed WebGL Gallery — https://tympanus.net/codrops/2026/02/02/building-a-scroll-revealed-webgl-gallery-with-gsap-three-js-astro-and-barba-js/

WHAT IT DOES: Multi-page editorial gallery where DOM `<img>` elements are mirrored 1:1 by Three.js textured planes positioned in screen-space via a unit-perfect orthographic camera. As the user scrolls, a fragment shader reveals each image with a procedural mask (stripe wipe + noise displacement), and on click, Barba.js drives a page transition where the clicked plane animates from its grid position to fullscreen via GSAP Flip before the next route's DOM mounts underneath.

PACKAGES:
- astro@5 (multi-page routing, near-zero JS by default)
- three@0.165
- gsap@3.13 (ScrollSmoother, ScrollTrigger, SplitText, Flip)
- @barba/core@2.10
- @barba/css@2.2 (or hand-rolled GSAP enter/leave)
- lenis is replaced by GSAP's official ScrollSmoother in this build
- vite (built into Astro)

IMPLEMENTATION:
Layout: each gallery item is `<figure class="card"><img data-webgl src="/img.jpg" /></figure>`. After DOM mount, a `GalleryWebGL` class scans `[data-webgl]`, reads each rect via `getBoundingClientRect()`, creates a `THREE.PlaneGeometry(rect.width, rect.height)` with a `ShaderMaterial` whose uniforms include `uMap` (the same image as a `TextureLoader` result), `uReveal` (0 → 1), `uHover` (0 → 1), and `uTime`. The orthographic camera is set to `new THREE.OrthographicCamera(-w/2, w/2, h/2, -h/2, -1000, 1000)` so 1 world unit = 1 CSS pixel. The plane's position is `(rect.left + rect.width/2 - w/2, -(rect.top + rect.height/2 - h/2), 0)`. On every scroll/resize, `lenis`'s `scroll` event updates plane Y. The HTML `<img>` is set to `visibility: hidden` once WebGL is ready (preserving layout). Fragment shader (reveal):
```glsl
float mask = step(uReveal, vUv.y + snoise(vUv*8.0)*0.08);
vec4 base = texture2D(uMap, vUv + (1.0-uReveal)*vec2(0.0, 0.05));
gl_FragColor = vec4(base.rgb, 1.0 - mask);
```
`uReveal` is animated per card by `ScrollTrigger`: `start: 'top 90%', end: 'top 50%', scrub: 0.6`. Page transition (Barba leave hook):
1. Capture state with `Flip.getState(planeWrapper)`.
2. Animate plane to `position: (0, 0, 0)` and `scale: viewport/cardSize` over 0.9s `expo.inOut`.
3. Fade DOM content with `gsap.to('.content', { opacity: 0, duration: 0.4 })`.
4. Barba swaps routes; on enter, the cloned plane stays mounted, the new DOM fades in over the plane, and the plane is then disposed.
Smooth scroll: `ScrollSmoother.create({ smooth: 1.2, effects: true, normalizeScroll: true })`. Resize handler: debounced 150ms, calls `ScrollTrigger.refresh()` and rebuilds plane geometries (cheap — only updates `geometry.attributes.position`).

WHY IT WORKS: Pixel-perfect WebGL/DOM sync removes the "second layer" feeling that breaks immersion in most hybrid sites — the reveal shader feels like it lives inside the image, not over it. ScrollSmoother + scrubbed reveal (vs. binary "in view" toggle) means the animation is always proportional to user intent, which is the single design rule that separates premium scroll experiences from amateur ones. Flip-driven page transitions turn navigation into spatial continuity, which is what Apple Photos and iOS taught users to expect from "looking inside" something.

GOOD / BAD:
GOOD: Astro multi-page + Barba.js means every route is independently crawlable and the JS payload per page stays under 80KB — rare for sites with this level of motion.
BAD: WebGL/DOM sync breaks under iOS Safari's rubber-band overscroll because Lenis/ScrollSmoother can't fully intercept it; planes visibly lag the DOM at the top/bottom edges.

CLEAN APPLICATION: Apply this to Clean's case-studies / showcase page (when one exists) and to the docs page-transition pattern. Build the showcase grid with `[data-webgl]` cards for each integration partner; use the reveal shader (stripe + noise) on scroll. For docs: capture the active sidebar item's bounding box on link click and Flip-animate it to become the next page's heading — a literal "follow the link" spatial cue. Skip the WebGL grid entirely on the homepage above the fold — too heavy for first-paint, and Clean's audience is judging on TTI within the first 1.5 seconds.

## BUILD DIRECTIVE

TARGET: Clean landing page hero + "How Clean works" technical section (one continuous scroll experience, top of `tryclean.ai`).

APPROACH: Build a sticky-card scroll narrative anchored by a single shared `@react-three/fiber` canvas pinned full-bleed behind the DOM. Above the fold: a tight hero with PP-Neue-Montreal-scale headline (`clamp(72px, 9vw, 168px)`) on `#0A0A0A`, with a procedural WebGL network-graph visualization (~50k particles via GPGPU FBO) showing an MCP message flowing from prompt → Clean proxy → N tool endpoints. As the user scrolls into the "How Clean works" section, three sticky-positioned capability panels (`MCP Proxy`, `TOON Format`, `Context Layer`) crossfade through the same canvas — each panel morphs the network-graph layout to a different preset (route, compress, cache). Below that, a `1px-gutter modular grid` (Cantor8 pattern) of feature tiles with `font-mono tabular-nums` for every metric (latency saved, tokens compressed, requests/sec). Reveal every grid tile with a `clip-path: inset(0 100% 0 0) → inset(0 0% 0 0)` on a `[0.16, 1, 0.3, 1]` cubic-bezier. End the section with an animated SVG request-lifecycle diagram (Cantor8's strand + particle pattern). No ASCII art on the marketing hero — reserve that for the docs landing page only.

STACK:
- next@15.3 + react@19 + tailwindcss@4
- @react-three/fiber@9 + @react-three/drei@10 + three@0.165
- gsap@3.13 with ScrollTrigger + ScrollSmoother + SplitText + Flip
- lenis@1.1
- framer-motion@12 (clip-path tile reveals + SVG path stroke animation only)
- shadcn/ui Rhea preset (April 2026) for any form/CTA primitives
- next/font: PP Neue Montreal (display), JetBrains Mono (mono numerals + labels), Inter Tight (body)
- lucide-react@0.450
- vite-plugin-glsl for `.glsl` imports

REFERENCE URLS:
- https://www.awwwards.com/sites/sohub (sticky-card shared-canvas pattern, type scale, magnetic cursor)
- https://www.awwwards.com/sites/apechain (GPGPU particle morph, single-color discipline, horizontal pin section)
- https://www.cantor8.io/ (1px-gutter modular grid, mono-tabular numerals, SVG flow diagram with particles)

KEY CONSTRAINTS:
- TTI under 1.8s on a Moto G Power — Clean's audience profiles sites before they read them. No hero video, no 3MB hero font, no analytics tag-manager before first paint.
- Single shared WebGL canvas. NEVER ship one canvas per section.
- All numerals on the page render in `font-mono tabular-nums tracking-tight`. This is the single most credibility-buying typographic choice for an infra buyer.
- Single accent color above the fold. No gradients, no glassmorphism, no rainbow logo-clouds. The motion is the visual interest.
- Avoid: ASCII gimmickry on the homepage, sound effects, custom cursors that fight macOS pointer behavior, scroll-jacked horizontal sections on mobile, any "AI sparkle" iconography, Lottie loaders, and "Built with" badges.
- Mobile: WebGL canvas falls back to a 1.2MB looped MP4 (H.265); horizontal-pin sections become vertical scroll-snap; SplitText falls back to whole-line stagger; clip-path reveals stay (cheap and compositor-driven).
- Accessibility: `prefers-reduced-motion: reduce` disables the particle morph (static end-state texture), disables SplitText (whole-line fade-in only), and skips the page-transition Flip. All animated SVG strokes must have a non-animated fallback path visible at all times.
