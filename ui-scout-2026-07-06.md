## 1. Radian — https://www.rideradian.com/  (Awwwards SOTD Jul 4, Developer Award)

WHAT IT DOES: A scroll-driven product launch experience for an electric enduro motorbike, built around a single persistent WebGL scene that the viewport locks onto for the entire hero-through-spec section. Camera and lighting are keyframed against ScrollTrigger progress: the bike orbits, dollies, and dissolves between paint-color variants as you scroll, while HTML sections fade in and out on top of the canvas as overlay panels. The final section swaps out of scroll-lock into an interactive configurator where the same GLTF instance stays mounted, and radio-group state drives material/uniform swaps in a single render frame.

PACKAGES:
- next@14 (App Router)
- react@18, react-dom@18
- three@0.164 (r164+)
- @react-three/fiber@8.16
- @react-three/drei@9.108 (useGLTF, Environment, PerspectiveCamera, ContactShadows)
- @react-three/postprocessing@2.16 (Bloom, Vignette, ChromaticAberration for the paint sheen shot)
- gsap@3.12 with ScrollTrigger and Observer plugins
- lenis@1.1 (smooth scroll, drives ScrollTrigger)
- tailwindcss@3.4 for HTML overlay layout
- @next/font/local for a display serif + neo-grotesk pair
- meshopt-decoder + draco3d for compressed GLTF payloads
- zustand@4.5 for configurator state (color, wheelset, battery)

IMPLEMENTATION:
1. Fixed-position `<Canvas dpr={[1, 1.75]} gl={{ antialias: false, powerPreference: 'high-performance' }}>` with `frameloop="demand"` — invalidate only on scroll or config change. Never leave the render loop running when idle.
2. Root `<Suspense>` wraps a single `useGLTF('/bike.glb')`. Compress the GLB with `gltf-transform` + Meshopt to under 6 MB; use KTX2 textures with basis universal.
3. `useLayoutEffect` builds one master `gsap.timeline({ scrollTrigger: { trigger: '#scene', start: 'top top', end: '+=6000', scrub: 0.6, pin: true }})`. Timeline animates a `THREE.Object3D` "cameraTarget" and camera position, plus material uniforms (`bodyPaint.uniforms.uMix.value`).
4. HTML overlay sections are absolutely positioned inside the pinned wrapper with `z-10` above the canvas, opacity/y-transform tweened by the same timeline with `scrollTrigger.snap({ snapTo: 'labels' })`.
5. Configurator swaps: on route to `/configure`, tear down ScrollTrigger, kill Lenis pinning, remount the canvas at `position: relative`, and let OrbitControls take over. Same GLTF, same materials — no reload.
6. Performance: `gl.setPixelRatio(Math.min(devicePixelRatio, 1.75))`, `ContactShadows` instead of real shadowmap, single HDR env at 1k for reflections, and `preserveDrawingBuffer: false`. Cap post FX with `<EffectComposer multisampling={0}>`.
7. Mobile fallback: below 768 px, replace the R3F canvas with a pre-baked 60-frame MP4 (H.265, 1.4 MB) scrubbed via `video.currentTime = progress * duration`. Keeps the story, kills the GPU cost.
8. Type: display headline in a wide serif at `clamp(56px, 8vw, 148px)`, letter-spacing `-0.03em`, line-height `0.9`. Body in an 18px neo-grotesk.

WHY IT WORKS: The pinned single-scene approach creates a strong figure/ground relationship — the product is the constant, copy is what changes, so cognitive load stays on the object. Scrub-linked animation (as opposed to autoplay) hands the user physical authorship of the experience, which is the same trick Apple's product pages have used since 2017 and it still lands because it maps motor input to visual output 1:1. Restrained post FX (subtle bloom, no lens dirt, no chromatic aberration except on the paint reveal) reads as "expensive product photography," not "web3 hero."

GOOD: Single WebGL context across three narrative modes with zero remounts — that's the hard part and they nailed it. BAD: On low-end Android the pin is jittery; scrub value should be higher (0.8+) or they should ship the video fallback below a stricter perf budget.

CLEAN APPLICATION: Apply this to Clean's landing-page hero + product tour by pinning a single WebGL scene of the MCP proxy pipeline — an abstract 3D graph of nodes (agent → proxy → tool servers) with data packets traveling along the edges. Scroll advances camera through: (1) "agent calls tool," (2) "proxy intercepts, applies context layer," (3) "TOON-formatted response returns." Same persistent scene, three HTML overlay panels, ScrollTrigger scrub 0.6, mobile falls back to a pre-baked MP4. This replaces Clean's current static hero with a technical scroll narrative that shows the product in one continuous shot instead of three disconnected feature blocks.

## 2. FS 60P (60fps) — https://www.cssdesignawards.com/sites/60fps/33314  (CSSDA SOTD Jul 6)

WHAT IT DOES: A luxury watch product site whose entire background is a full-viewport WebGL shader that reacts to cursor position, scroll velocity, and section transitions. The shader renders a metallic-gradient noise field (Voronoi + FBM domain warp) that flows around and behind the watch photography, giving the impression of the watch face reflecting the room. Section changes crossfade the shader palette (steel → rose gold → obsidian) via a uniform tween, while the HTML type layer scroll-locks with GSAP.

PACKAGES:
- vite@5 + typescript@5.4 (marketing site, no SSR needed)
- three@0.164 or ogl@1.0.11 (ogl gives a smaller footprint if there's no scene graph — just a fullscreen quad)
- gsap@3.12 with ScrollTrigger
- lenis@1.1
- @studio-freight/hamo (velocity + pointer tracking hooks)
- glslify or vite-plugin-glsl for shader `#include` support
- tailwindcss@3.4

IMPLEMENTATION:
1. One fullscreen `<canvas>` fixed at `inset: 0; z-index: -1; pointer-events: none`. Single `ShaderMaterial` with a plane geometry at NDC extents (skip the projection matrix entirely).
2. Fragment shader inputs: `uTime`, `uMouse` (vec2, eased), `uScrollVelocity`, `uPalette` (three color uniforms), `uMix` (0..1 between palettes). Core is `fbm(domainWarp(uv * 3.0 + uTime * 0.05))` masked by a soft radial gradient centered on `uMouse`.
3. Pointer lerp: `uMouse.x += (target.x - uMouse.x) * 0.06` per frame — the lag is the whole feel.
4. Section palette tween via `ScrollTrigger.create({ trigger: section, onEnter: () => gsap.to(uniforms.uPalette, { r: 0.83, g: 0.68, b: 0.44, duration: 1.2, ease: 'power2.inOut' })})`.
5. Watch photography: high-res PNG-24 with alpha, positioned `object-fit: contain`, `mix-blend-mode: normal`. Do NOT `mix-blend-mode: screen` the watch itself — the shader reads through the transparent surround and does the work.
6. Type: giant thin serif ("times ish") at 200 wt, `clamp(96, 14vw, 280)`, tracked to `-0.05em`. Small caps labels in mono for spec rows.
7. Perf: `renderer.setPixelRatio(Math.min(devicePixelRatio, 1.5))`, resize observer throttled to 150 ms, `willReadFrequently: false`. On `prefers-reduced-motion`, freeze `uTime` and disable pointer lerp.

WHY IT WORKS: The shader acts as a continuous environmental ambient — it's not "an effect," it's the room the product lives in, which is the exact metaphor luxury watch photography has used since the 1970s (dark rooms, single warm light source). Because the noise is procedural and infinite, there's no repeated pattern for the eye to lock onto, so it stays visually alive without demanding attention. Palette shifts are anchored to editorial section boundaries, so motion is semantically meaningful rather than decorative.

GOOD: Zero-scene-graph WebGL — one quad, one shader, one draw call — while looking like a $200k CGI render. BAD: On Intel integrated GPUs the fragment shader dips to 30 fps; needs a lower-precision (`mediump`) fallback with 4 octaves instead of 6.

CLEAN APPLICATION: Apply this to Clean's landing page background by replacing any static gradient or particle field with a single fullscreen fragment shader in Clean's brand palette. The shader should be a slow-drifting metallic teal/black domain-warp field, cursor-lagged at 0.06 lerp. On scroll into the "context layer" section, tween the palette toward a warmer cyan. This gives Clean's site a permanent atmospheric anchor that costs one draw call and makes the pricing/CTA sections feel like they're inside the same lit room as the hero — solving the "landing page feels stitched together" problem without adding sections.

## 3. NRG — Build Your Data Center — https://business.nrg.com/campaigns/build-your-data-center/  (Awwwards SOTD Jul 1, Rogue Studio)

WHAT IT DOES: A B2B enterprise landing that behaves like a configurator: full-viewport horizontal-panel sections snap on scroll, each panel is a step in the "how NRG builds your data center" story, and a persistent bottom-anchored progress bar plus micro-hover states on the sidebar TOC make the whole thing feel like a native app. Palette is aggressively restrained — one saturated yellow (#FFC20E) against near-black (#261D26) — so every accent color IS an interaction cue. No 3D, no video hero; the impact comes from typographic scale and choreographed reveals.

PACKAGES:
- next@14 with static export (Vercel edge)
- gsap@3.12 with ScrollTrigger and Observer
- lenis@1.1
- framer-motion@11 (only for the sidebar TOC hover states — the heavy scroll work stays in GSAP)
- clsx + tailwind-merge
- tailwindcss@3.4
- next/font/local for a display grotesk (Söhne / Neue Haas Grotesk feel) + a mono for status labels
- lucide-react for iconography

IMPLEMENTATION:
1. Layout is a `min-h-screen grid grid-cols-[240px_1fr]` on desktop, single column on mobile. Left sidebar is a fixed `position: sticky; top: 0; h-screen` TOC.
2. Sections use `snap-y snap-mandatory` on the scroll container with `snap-align: start`, backed by GSAP ScrollTrigger for the reveal choreography (`toggleActions: 'play none none reverse'`).
3. Each section headline animates in via a split-word GSAP reveal: `gsap.from(chars, { yPercent: 100, stagger: 0.02, duration: 0.9, ease: 'expo.out' })`. Use `SplitText` (paid) or Splitting.js (free).
4. Sidebar TOC state syncs to `ScrollTrigger.batch()` — active item gets a `w-full h-[2px] bg-yellow-400` underline animated with `layoutId` in framer-motion.
5. Progress bar at bottom: `<motion.div style={{ scaleX: progress }} className="origin-left h-[3px] bg-yellow-400" />` where `progress` is `useScroll().scrollYProgress`.
6. Copy typography: display headlines at `clamp(72, 11vw, 200)`, tracked `-0.04em`, leading `0.92`. Uppercase mono status labels at 12px with `letter-spacing: 0.16em`.
7. All accent states — hover fills, link underlines, focus rings, form field carets — use the single yellow. This is the discipline that makes the whole thing feel intentional.
8. No cursor-follow, no magnetic buttons, no marquee. Restraint is the aesthetic.

WHY IT WORKS: Enterprise buyers respond to legibility and control, not spectacle. Snapping panels with mandatory scroll say "we've thought about your time." A single-accent-color system does the Gestalt work: because yellow only ever means "next action," attention allocation is nearly automatic. Massive typographic scale relative to a strict grid produces the "confident whitespace" that reads as institutional trust — the same rhythm Bloomberg Terminal marketing and Palantir use.

GOOD: Two-color palette weaponized as an interaction system — this is a masterclass in restraint for B2B. BAD: Mandatory scroll snap on trackpad is polarizing; several users will fight it. Ship an `overscroll-behavior: contain` and consider `snap-type: proximity` instead of `mandatory` for scroll wheels.

CLEAN APPLICATION: Apply this to Clean's landing page as a full rebuild of the mid-page product tour section. Replace the current three-column feature grid with four full-viewport snap panels: "MCP proxy," "Context layer," "TOON format," "Prospect discovery." Use Clean's brand accent as the ONLY color across every hover, link, active TOC item, and CTA border — no gradients, no secondary accents. Left sidebar TOC that highlights the current panel, bottom progress bar. Display headlines at 11vw. This positions Clean as institutional developer infra, not a design-y consumer AI tool, which matches the buyer psychology.

## 4. Hildén & Kaira — https://www.awwwards.com/sites/hilden-kaira  (Awwwards SOTD Jul 5, Developer Award)

WHAT IT DOES: A Helsinki social-content agency site whose entire animation strategy is "less than last time." The site is built in Webflow, animated with GSAP + ScrollTrigger, and its restraint is the point: instead of full-page transitions or hijacked scroll, individual editorial blocks reveal on view with tight `y: 40px, opacity: 0 → 1, duration: 0.6, ease: 'power3.out'` staggers, and text is the primary compositional element. Case study cards use a magnetic hover with a 1.02 scale and a subtle shadow lift — no page rewrites, no view transitions.

PACKAGES:
- Webflow (host + CMS) — export via /export or host on Webflow
- gsap@3.12 with ScrollTrigger, SplitText (or Splitting.js)
- lenis@1.1 (paired with `lenis-webflow` embed pattern)
- Barba.js@2 (optional, for page-to-page transitions)
- Custom fonts loaded via `next/font/local` equivalent (Webflow custom fonts panel)

IMPLEMENTATION:
1. Site is Webflow-built; custom JS lives in a `<script>` embed in the page footer, wrapped in `DOMContentLoaded`.
2. `gsap.registerPlugin(ScrollTrigger)`. Loop all `[data-reveal]` elements: `gsap.from(el, { y: 40, opacity: 0, duration: 0.6, ease: 'power3.out', scrollTrigger: { trigger: el, start: 'top 85%', toggleActions: 'play none none none' }})`.
3. Text splits: `SplitText.create(headline, { type: 'words' })`, then stagger `y: '100%', duration: 0.9, stagger: 0.04, ease: 'expo.out'`. Wrap words in a `overflow: hidden` parent so the reveal masks cleanly.
4. Magnetic case cards: on `mousemove`, calculate offset from card center, translate card by `offset * 0.15`. On `mouseleave`, `gsap.to(card, { x: 0, y: 0, duration: 0.6, ease: 'elastic.out(1, 0.5)' })`.
5. Lenis: `new Lenis({ lerp: 0.1, wheelMultiplier: 1 })`, RAF loop, integrate with ScrollTrigger via `lenis.on('scroll', ScrollTrigger.update)`.
6. Color: two colors, period. A warm cream (`#F4EEE4` feel) and a near-black. No gradients, no accent brights.
7. Type: a Swiss display serif for editorial headlines, a neo-grotesk for body. Ratios: H1 8–10vw, H2 4vw, body 20/32.

WHY IT WORKS: This is the "grown-up" version of every over-animated portfolio site. By deliberately dialing motion DOWN from a v1, the studio is signaling that their content is strong enough not to need scaffolding — which is the same message a client is buying. The restrained reveals prevent motion-fatigue on long editorial scrolls and keep the eye on words, which for a content agency is the actual product.

GOOD: Perfect proof that "boring reveals" done at scale beat "flashy hijack" — this site would work identically in 2015 and 2028. BAD: The magnetic hover on cards is inconsistent on touch devices and gets in the way of tap targets on mobile Chrome — needs a `matchMedia('(hover: hover)')` guard.

CLEAN APPLICATION: Apply this to Clean's blog / docs / customer-story pages by killing every scroll-hijack, full-page transition, and marquee. Ship exactly one reveal pattern (`y: 40, opacity: 0 → 1, ease: 'power3.out'`) via a `[data-reveal]` attribute, split-word headlines with `y: 100%` masked stagger, and a strict two-color content palette (Clean's off-white + near-black) that shows the accent color only on links and CTAs. This makes Clean's written content — technical posts about MCP, TOON, context layers — feel like publication-grade editorial, which is the credibility play for a developer-infra brand.

## 5. Wembi — https://www.awwwards.com/sites/wembi  (Awwwards SOTD Jul 2, Developer Award)

WHAT IT DOES: A Nuxt-based corporate site for a European tech consultancy, built on Craft CMS and driven by GSAP. Signature move is a color-forward two-tone palette (`#beff8b` acid green + `#9dbbc5` cool blue-gray) applied as full section backgrounds, with typography scaled aggressively so page-turns feel like editorial spreads. Navigation uses a bespoke full-screen overlay with a video reveal, and cursor interactions swap the pointer for a small bold pill that grows on hover targets.

PACKAGES:
- nuxt@3.12
- vue@3.4
- gsap@3.12 with ScrollTrigger
- lenis@1.1 with `@studio-freight/lenis-vue` wrapper
- Craft CMS (headless) with `@nuxtjs/apollo` or REST fetch
- tailwindcss@3.4 (via `@nuxtjs/tailwindcss`)
- @vueuse/core for `useMouse`, `useElementVisibility`
- Local fonts via `@nuxtjs/fontaine`

IMPLEMENTATION:
1. Root layout ships two Tailwind color tokens: `--color-acid: #beff8b`, `--color-slate: #9dbbc5`. Every section sets `bg-[color:var(--section-bg)]` from CMS data.
2. Custom cursor: fixed `<div>` at pointer position tracked with `useMouse`, `mix-blend-mode: difference`, scaled to 3× on `[data-cursor="cta"]` hover via a Vue directive.
3. Nav overlay: click hamburger → `gsap.to(overlay, { clipPath: 'circle(150% at top right)', duration: 0.9, ease: 'expo.inOut' })`. Inside, a muted autoplay video sits behind menu links at 40% opacity.
4. Scroll-in animations: same masked word split as Hildén & Kaira, but instead of `power3.out` use `expo.out` with `duration: 1.1` — noticeably slower and heavier.
5. Section transitions in-page: `ScrollTrigger.create({ trigger, onEnter: () => document.documentElement.style.setProperty('--section-bg', section.color) })` — the whole page background tweens color as you enter each section (transition via `--section-bg` CSS var with a `transition: background 400ms ease`).
6. Type: an oversized geometric sans for display, near-mono for supporting labels. H1 sizes hit `min(180px, 15vw)`.

WHY IT WORKS: Two saturated colors used as full-section grounds create hard visual chapters — the eye reads scrolling as "turning pages," and the acid-green ↔ slate rhythm is aggressive enough to be memorable without straying into brand-gimmick territory. Custom cursor with `mix-blend-mode: difference` gives every hover state a tactile confirmation loop for free, which raises perceived interactivity without adding actual UI.

GOOD: Sectional color transitions as a navigation cue is a genuinely reusable pattern for content-heavy corporate sites. BAD: Custom cursor is a mobile-tap nightmare and hurts accessibility; must be `hidden md:block` and respect `prefers-reduced-motion`.

CLEAN APPLICATION: Apply this to Clean's pricing + use-case pages by giving each use case (SDR/marketing, RevOps, dev tools) its own bold accent-tinted section background that transitions on scroll-into-view. Bind the accent tint to a CSS custom property so the whole page repaints between sections in 400 ms. Drop a subtle `mix-blend-mode: difference` custom cursor on desktop only for CTAs. This lets Clean show breadth of use cases without adding subpages or cluttered nav — the color rhythm itself becomes the taxonomy.

## BUILD DIRECTIVE

TARGET: Clean landing page — full rebuild of the hero and immediately-following product-tour section (top 3 viewports).

APPROACH: Replace the current static hero with a single pinned WebGL scene (following Radian's pattern) that visualizes the MCP proxy pipeline as an abstract 3D node graph — agent, proxy, tool servers, with animated data packets traveling along edges. Behind the entire viewport, run a slow-drifting fullscreen fragment shader in Clean's brand palette (following the 60fps FS 60P pattern) so the WebGL scene sits inside a continuous atmospheric ground rather than on a flat canvas. GSAP ScrollTrigger scrubs a master timeline that advances camera through three narrative beats — "agent calls tool," "proxy applies context layer," "TOON-formatted response returns" — with HTML overlay panels fading in on top. Below the hero, transition into an NRG-style two-panel snap-scroll section with a single-accent color system (Clean's brand accent applied to every CTA, link, and active TOC state, nothing else) so the tone shifts from "atmospheric" to "institutional infra" as the user moves down. Below 768 px, drop the R3F canvas and swap in a pre-baked MP4 scrubbed by scroll progress.

STACK:
- next@14 (App Router), react@18, typescript@5.4
- three@0.164, @react-three/fiber@8.16, @react-three/drei@9.108, @react-three/postprocessing@2.16
- gsap@3.12 with ScrollTrigger, Observer, SplitText (or Splitting.js as free fallback)
- lenis@1.1 + @studio-freight/hamo
- tailwindcss@3.4 with clsx + tailwind-merge
- framer-motion@11 (only for TOC layout animations)
- vite-plugin-glsl (if separate marketing subroute) OR raw-loader for `.glsl` in Next
- zustand@4.5 for scene state
- meshopt-decoder + draco3d for GLB compression
- next/font/local for a display grotesk + a mono
- lucide-react for iconography

REFERENCE URLS:
- https://www.rideradian.com/ — pinned single WebGL scene + scrub-linked scroll story (primary structural reference)
- https://www.cssdesignawards.com/sites/60fps/33314 — fullscreen shader as continuous atmospheric ground (background layer reference)
- https://business.nrg.com/campaigns/build-your-data-center/ — single-accent B2B color discipline + snap panels + sidebar TOC (mid-page product tour reference)

KEY CONSTRAINTS: Clean is developer infrastructure. Avoid: (1) glassmorphism, blurred noise gradients, and consumer-AI purple/pink palettes; (2) marquees, gradient text, "AI shimmer" outlines, cursor sparkles; (3) autoplay motion that runs on idle — every animation must be user-driven (scroll, hover, click) or extremely subtle drift; (4) any framework other than React/Next — no Svelte, no Astro, no vanilla; (5) shadcn-default look for the top-of-page — save shadcn for the docs and dashboard shell. Positive constraints: single WebGL context reused across all hero states, one accent color used ONLY as an interaction cue, mono type for spec labels and status states, sub-1.5 dpr cap, mobile MP4 fallback under 2 MB. Target: hero LCP under 2.5 s on 4G Moto G4, TBT under 200 ms, WebGL FPS 60 on M1, 45 on mid-range Android.
