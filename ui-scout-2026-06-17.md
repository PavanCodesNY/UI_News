## Fauna Robotics — https://faunarobotics.com (Awwwards SOTD 2026-06-17, Developer Award)

WHAT IT DOES: Webflow-built marketing site for a humanoid-robotics startup. Layout is a vertical scroll narrative composed of full-bleed video hero, an alternating two-column "capabilities" section with sticky media on one side and incremental text panels on the other (driven by IntersectionObserver), and a closing pricing/contact CTA. Visual hierarchy is built almost entirely from a 2-color palette (off-white #EFEFEF + pale blue #D1E3FF), a single accent neon, and silent-loop autoplaying MP4s that act as static-hero replacements.

PACKAGES:
- webflow (CMS + Interactions 2.0) as platform of record
- lenis ^1.1 (smooth scroll, used inside Webflow custom code)
- gsap ^3.13 + ScrollTrigger (anchor pinning, text reveals)
- splittype ^0.3 (per-char text masks)
- @vimeo/player or native `<video muted playsinline loop autoplay>` (no library) for the hero loops
- inter-tight (variable font, 100–900) + jetbrains-mono for labels
- lucide static SVG (no React; raw inlined icons)

IMPLEMENTATION:
1. Page shell is `<main>` with `position: relative` and `overflow-x: hidden`; smooth-scroll wrapper `<div id="lenis-wrap">` hosts a Lenis instance with `lerp: 0.085, smoothWheel: true, syncTouch: false`.
2. Hero: full-viewport (`h: 100svh`), background = `<video>` (1.6 MB H.265, 1920×1080, 30 fps, 8-second loop) covered with `object-fit: cover`. Headline is a single `<h1>` set in Inter Tight Variable @ `clamp(56px, 9vw, 168px)`, `font-weight: 320`, `letter-spacing: -0.04em`, `line-height: 0.92`. SplitType splits into chars, GSAP timeline staggers `yPercent: 100 → 0, opacity: 0 → 1`, `duration: 1.1, ease: "expo.out", stagger: 0.018`.
3. Sticky-side capability section: CSS grid `grid-template-columns: 1fr 1fr; gap: 8vw`. Left column gets `position: sticky; top: 12vh; height: 100vh;` and swaps videos via ScrollTrigger with `scrub: false, toggleActions: "play none none reverse"` and crossfade (`opacity` + `clip-path: inset(0 0 0 0)` from `inset(0 0 100% 0)`).
4. Mouse follower: 12px circle, `mix-blend-mode: difference`, GSAP `quickTo` (`x` and `y` with `duration: 0.35, ease: "power3"`), scales to 64px on `data-cursor="hover"` targets via `gsap.to(cursor, {scale: 6.4, duration: 0.25})`.
5. Section dividers are hairline `border-top: 1px solid rgba(10,10,10,0.08)` plus a 240×1 SVG progress stroke that animates `pathLength: 0 → 1` on enter.
6. Breakpoints: 1440 / 1024 / 768 / 480. At ≤768 the sticky media collapses and stacks above text; cursor follower disabled (`pointer: coarse` MQ).
7. Perf: every hero/cap video preloaded with `<link rel="preload" as="video">`, `Range: bytes=0-262143` first segment; LCP target ≤2.0 s.

WHY IT WORKS: Restraint as signal — only two desaturated colors plus video do all the lifting, so the brand reads as engineering-serious rather than entertainment. The sticky-pair pattern enforces an explicit reading order (Gestalt continuity), and the ultra-tight headline tracking + low font weight evokes industrial-design CAD callouts. Motion is purely confirmatory (no decorative spin) — it earns attention by being quiet.

GOOD: Two-color palette + variable font + muted video → cinema-grade hero on a Webflow budget. BAD: Whole site is video-dependent — on a throttled 4G connection LCP balloons past 4 s and the sticky section becomes a black rectangle until buffering completes.

CLEAN APPLICATION: Apply this to Clean's landing-page hero. Replace the marketing image with a 6-second screen-capture loop of an MCP proxy intercepting a tool call, displayed inside a 16:10 device frame. Use #EFEFEF page, #D1E3FF accent for connector "active" pulses, and Inter Tight @ 320 weight for the hero. Build the sticky two-column "How Clean works" section below with three pinned frames: (1) raw MCP payload, (2) TOON-compressed payload with byte counter ticking down, (3) context window with green "kept" tokens vs red "saved" tokens. Driver: GSAP ScrollTrigger with `scrub: 0.6`.

## Pacôme Pertant Portfolio — https://pacomepertant.com (Awwwards SOTD 2026-06-09, Portfolio Honors, score 7.76)

WHAT IT DOES: A single-page immersive portfolio rendered as an infinite WebGL spiral. Project thumbnails orbit a vertical axis on a Bezier helix; scrolling rotates the helix while a synchronized audio bus crossfades stems per project. Layout escapes the document model entirely — there is no DOM grid; everything is GPU instances of a `THREE.Mesh` (`PlaneGeometry`) instanced 240× and sampled at points along a Catmull-Rom curve.

PACKAGES:
- nuxt ^3.13 (SSR shell only)
- three ^0.166
- @react-three/fiber is *not* used here — raw Three.js inside a `<canvas>` mounted by a Nuxt plugin
- gsap ^3.13 + ScrollTrigger + Observer
- lenis ^1.1 (virtual scroll proxy → Three.js)
- howler ^2.2 (audio bus, stem mixing)
- postprocessing ^6.36 (EffectComposer: bloom 0.18, chromatic aberration 0.0012, noise 0.03)
- @use-gesture/vanilla (pointer drag-to-rotate)
- ppneue-montreal (display) + ibm-plex-mono (UI)

IMPLEMENTATION:
1. Scene: `PerspectiveCamera` fov 35, position `(0, 0, 6)`. A `CatmullRomCurve3` defined by 16 control points laid in a helix (`radius: 2.2, height: 36, turns: 4`).
2. Instanced helix items: `InstancedMesh(PlaneGeometry(1.6, 1.0), shaderMat, 240)`. Each instance receives a UV scroll offset via per-instance attribute (`InstancedBufferAttribute`).
3. Shader: vertex shader bends planes along curve tangent; fragment shader samples project thumbnail texture, applies a parallax `vec2 uv = baseUV + flowDir * uTime * 0.03`, and an edge-falloff mask.
4. Scroll driver: Lenis ScrollProxy → `uProgress` uniform (`0..1`); GSAP `quickTo` smooths progress with `duration: 0.6, ease: "expo.out"`. The helix rotates `rotation.y = uProgress * Math.PI * 8`.
5. Audio: each project has 3 stems (drums, pad, lead); Howler crossfades stems based on which item is closest to camera (`distance < 1.4`). Master bus has a lowpass that opens from 400 Hz → 18 kHz on click-to-enter.
6. Cursor + dial: SVG ring with `stroke-dasharray` animated to project progress; clicked project triggers a `transform: scale(1) → scale(1.35)`, camera `gsap.to(camera.position, {z: 3.2, duration: 1.4, ease: "expo.inOut"})`.
7. Mobile: helix flattens to a 2D vertical reel; postprocessing disabled (`pixelRatio < 2` && `deviceMemory < 4` heuristic).

WHY IT WORKS: Motion semantics are completely coherent — scroll IS rotation IS audio mix IS focus. Because the same input controls every channel, the user feels they're "tuning" the site, not navigating it. The single-axis helix gives a strong figure-ground (Gestalt), and the muted black/white reinforces that color belongs only to the work itself.

GOOD: Best-in-class motion-audio coupling on the open web; framerate holds 60 fps on M-series + recent NVIDIA. BAD: Total JS payload ≈ 3.8 MB gzipped, blocks interaction on low-end Android for 4 s+, and the audio gate is a friction wall for users on speakers in shared spaces.

CLEAN APPLICATION: Apply this to Clean's product-demo section, not the hero. Build a "Tools In, Context Out" canvas: an instanced helix of 60 MCP tool cards (Slack, Gmail, Notion, GitHub, etc.). Scroll rotates the helix; each card pulses when "selected" by Clean's router, and a small inspector panel pinned bottom-right shows the live TOON payload size dropping. Skip audio (B2B context). Use the same GSAP `quickTo` smoothing on `uProgress`.

## Serve Robotics — https://www.serverobotics.com (Awwwards SOTD 2026-06-11)

WHAT IT DOES: Full-CGI product site for an autonomous sidewalk-delivery robot. Hero is a real-time WebGL scene with the robot rendered as a high-poly GLB (~9 MB Draco-compressed) lit by an HDRI environment; scrolling triggers camera dolly and articulated joint rotations on the robot's compartments. Storytelling is staged in five "acts," each with a pinned 3D camera position, captioned spec sheet, and CGI b-roll.

PACKAGES:
- next ^15.0 (App Router, RSC for static frames)
- three ^0.166
- @react-three/fiber ^9.0
- @react-three/drei ^9.117 (Environment, useGLTF, ContactShadows, MeshTransmissionMaterial)
- @react-three/postprocessing ^2.16 (Bloom, DepthOfField, Vignette)
- meshoptimizer ^0.21 + draco3d (asset pipeline)
- framer-motion ^12.0 (HTML overlays only)
- lenis ^1.1
- pp-supply-mono + söhne for typography

IMPLEMENTATION:
1. `<Canvas>` is fixed full-viewport (`position: fixed; inset: 0; z-index: 0`). Scrollable content is `<main>` above it with `pointer-events: auto` and transparent sections so the 3D scene is always visible through.
2. Robot: `useGLTF('/robot.glb')`; assign bone refs to 6 named joints. A `useFrame` hook reads `scrollYProgress` from Framer Motion's `useScroll({ container: lenisRef })` and maps to bone rotations with `MathUtils.damp(currentR, targetR, 4, delta)`.
3. Camera: stored positions in a const array `[ {pos:[3,1.2,4], look:[0,0.6,0]}, ... ]`. `useFrame` interpolates between adjacent indices based on `scrollYProgress * (acts.length - 1)`, using `Vector3.lerp` per frame.
4. Lighting: `<Environment files="/hdri/studio_2k.hdr" background={false} blur={0.4} />` plus a `<directionalLight intensity={2.4} castShadow shadow-mapSize={[2048,2048]} />` keyed off-axis.
5. Materials: aluminum chassis is `MeshStandardMaterial` (`roughness: 0.28, metalness: 0.85, envMapIntensity: 1.2`); window glass uses `MeshTransmissionMaterial` (`thickness: 0.4, transmission: 1, ior: 1.45, chromaticAberration: 0.06`).
6. Postprocessing: Bloom (`mipmapBlur, intensity: 0.42, luminanceThreshold: 0.85`), DOF (`focusDistance: 0.012, focalLength: 0.05, bokehScale: 2.4`), Vignette `0.4`.
7. Overlay copy: each act has `<motion.div style={{opacity: useTransform(scrollYProgress, [a, b], [0,1])}}>` with text in `pp-supply-mono` 13 px and number-tickers using `framer-motion`'s `animate(count, target, {duration: 0.9, ease: [0.16, 1, 0.3, 1]})`.
8. Perf: GLB loaded with `Suspense` + a `<DracoLoader>` decoder hosted on the same CDN; on `prefers-reduced-motion`, swap canvas for a single hero PNG.

WHY IT WORKS: The fixed-canvas + scrolling overlay pattern keeps the hero "alive" while text changes — the eye never has to retarget. The robot rotates on a believable physical pivot, so motion reads as causal rather than decorative. Aluminum + glass material study communicates engineering credibility before a single spec is read.

GOOD: Camera and bone interpolation are physically smooth (no snap), and DOF correctly tracks focus point. BAD: 9 MB GLB plus HDRI = ~13 MB first paint; users on metered connections see a 3-second white flash before suspense resolves.

CLEAN APPLICATION: Apply this to Clean's "How it works" section. Build a fixed-canvas 3D model of Clean's MCP proxy as a translucent glass cube with labeled ports (one per connector) and a glowing "context bus" running through it. Scroll cycles five acts: (1) raw tool calls in, (2) deduplication, (3) TOON compression, (4) routing, (5) context out. Use the exact MeshTransmissionMaterial settings above for the cube. Skip the HDRI — Clean's brand wants flat-lit, not studio-product.

## Elva — https://elva.ai (Awwwards SOTD 2026-06-15, Developer Award)

WHAT IT DOES: Marketing + onboarding site for a voice-first mobile AI video editor. The signature device is a "glass-blob persona" — a metaball/SDF-rendered character with 30+ behavioral states (listening, thinking, speaking, idle, error, scrubbing, etc.). The blob morphs across hero, feature sections, and CTA, anchoring the whole page as a single persistent character. Layout is mobile-first, single column, with the blob occupying a fixed 320×320 SVG/canvas above each act.

PACKAGES:
- astro ^4.16 (multi-page with islands)
- solid-js ^1.9 (interactive islands; lighter than React for the blob shader)
- three ^0.166 + custom raymarched SDF fragment shader
- gsap ^3.13 (HTML state transitions, easing of shader uniforms)
- howler ^2.2 (voice prompts + UI ticks)
- tailwindcss ^4.0
- radix-ui ^2.0 (Dialog, Tabs, ToggleGroup) — note Feb 2026 unified package
- web-audio-api (native) for live mic level → blob amplitude
- general-sans (variable, 200–700) + jetbrains-mono

IMPLEMENTATION:
1. Blob is a single fullscreen `<canvas>` running a raymarched scene with 3 spherical SDFs combined via smooth-min (`k = 0.45`). Color: white `#FFFFFF` on transparent, with a subsurface fake (`vec3(0.86, 0.91, 1.0)` rim term).
2. State machine (XState or hand-rolled reducer): 30 named states, each maps to a uniform set `{uBaseRadius, uWobbleAmp, uWobbleFreq, uBreathRate, uColorTint, uNoiseScale}`. Transitions are GSAP `gsap.to(uniforms, {duration: 0.6, ease: "expo.inOut"})`.
3. Idle wobble: `position += sin(uTime * uBreathRate) * 0.04`; "listening" adds an audio-reactive radius (`uBaseRadius += rms * 0.6`) driven by `AnalyserNode.getByteFrequencyData`.
4. Page sections are pinned with ScrollTrigger; entering a section dispatches `actor.send({type: 'ENTER_FEATURE_TRIM'})` etc. The blob transitions while the section content fades.
5. Section content: ultra-spacious typography stack: section eyebrow (12 px, `letter-spacing: 0.12em`, uppercase, JetBrains Mono), headline (Variable General Sans @ 48–96 px, weight 360), body (18 px @ weight 360 also).
6. Microsounds: each state transition plays a 90 ms sine sweep via Howler at -28 dB; respect `prefers-reduced-motion` → mute and freeze blob at last state.
7. Mobile: blob downsamples to a 240×240 offscreen canvas, then upscales with CSS `image-rendering: smooth`. Raymarch step count drops 64 → 32, max distance 12 → 8.

WHY IT WORKS: A persistent character creates a parasocial bond — every scroll feels like a conversation with the same entity, which is exactly the product premise. State-driven motion (vs. timeline animation) means the blob always feels alive and responsive instead of choreographed. Generous typography and a single visual subject = monastic clarity.

GOOD: Best-in-class character anchoring on a marketing page; mic-reactive demo on the homepage is a brilliant proof of capability. BAD: Mic permission prompt on landing is a hard sell; many users bounce before granting it, and the fallback (canned demo loop) is much weaker.

CLEAN APPLICATION: Apply this to Clean's onboarding flow. Build a persistent "router" character that lives top-right of the onboarding viewport — a small glass token rendered via the same SDF approach. Drive states from the onboarding step machine: `idle`, `scanning_mcps`, `compressing`, `connected`, `error`. No audio, no mic. Replace blob color with Clean's pale blue `#D1E3FF`. The character carries the user through Connect → Configure → First Tool Call, providing continuity that step indicators alone cannot.

## Aceternity UI — Text Flipping Board + Grouped Sidebar drop — https://ui.aceternity.com/changelog (latest free drop, 2026-06)

WHAT IT DOES: A new free component, "Text Flipping Board," renders a split-flap mechanical display (think airport schedule boards) entirely in React + Tailwind + Framer Motion. Each character is an independent `<motion.div>` that rolls through 28 candidate characters (A–Z, space, dot) before settling on the target. The companion "Grouped Sidebar" is a collapsible navigation with expandable groups whose open/closed state persists across reloads via `localStorage`.

PACKAGES:
- react ^18.3 or ^19.0
- tailwindcss ^4.0
- framer-motion ^12.0 (now `motion` package)
- @radix-ui ^2.0 (unified) for Disclosure primitives in the sidebar
- clsx + tailwind-merge for `cn()` helper
- lucide-react ^0.460 for sidebar icons
- next ^15 (Aceternity's recommended host, but framework-agnostic)
- ibm-plex-mono for the flipping board (mechanical aesthetic)

IMPLEMENTATION (Text Flipping Board):
1. Component signature: `<FlippingBoard text="ARRIVING SHORTLY" charset={['A'..'Z',' ','.']} flipDuration={0.42} stagger={0.06} />`.
2. Render N `<CharCell>` components, one per target char. Each cell holds a `useMotionValue(0)` integer index into the charset and animates with `animate(motionValue, targetIdx, {duration, ease: "easeInOut"})` after a delay of `i * stagger`.
3. CharCell DOM: `<div className="relative h-[1.1em] w-[0.65em] overflow-hidden font-mono">` containing a `<motion.div style={{y: useTransform(idx, v => -v * cellHeight)}}>` column of all charset characters.
4. Top/bottom seam (the real split-flap edge): absolutely positioned `1px` line with `mix-blend-mode: overlay` and an inset shadow `box-shadow: inset 0 1px 0 rgba(0,0,0,0.4)`.
5. Subtle audio (optional, mute by default): play a `0.05s` click sample on every integer change, normalized at -32 dB.

IMPLEMENTATION (Grouped Sidebar):
1. Persistence: `useEffect` reads `localStorage.getItem('sidebar:open-groups')` on mount and writes on each toggle. SSR-safe: hydrate via `useSyncExternalStore` against a memory snapshot to avoid flicker.
2. Group header: button row with rotating chevron (`transform: rotate(0deg → 90deg)` over 180 ms `ease-out`). Content uses Radix Collapsible with `data-state="open"` styled via Tailwind `data-[state=open]:animate-collapsible-down`.
3. Active-item indicator: a 2 px vertical bar absolutely positioned at `left: -1px`, animated with Framer Motion's `layoutId="sidebar-active"` for smooth handoff between items.
4. Width: collapsed 64 px, expanded 280 px; resize handle is a 4 px draggable column with `cursor: col-resize`.

WHY IT WORKS: The flipping board taps mechanical-nostalgia + literal motion semantics — the metaphor is "information is updating" and the user reads it instantly. The grouped sidebar's persistence respects user state, which converts a UI pattern into a tool. Both prioritize information density without becoming noisy because each animation is short, deterministic, and reversible.

GOOD: Both components ship as single-file copy-paste; zero black box. BAD: Flipping board can be visually overwhelming if used on >2 spots per page; on mobile the cell width math (`ch` units) misaligns in Safari ≤17.

CLEAN APPLICATION: Apply Text Flipping Board to Clean's dashboard analytics tile that shows live "Tokens saved today" / "Calls compressed" / "Active connectors" — three flipping rows that update via SSE every 4 s. Apply Grouped Sidebar to Clean's docs navigation: groups for "Quickstart", "MCP Proxy", "TOON Format", "Connectors", "Integrations", "API Reference", with per-group localStorage persistence so a docs reader's expanded state survives reloads.

## BUILD DIRECTIVE

TARGET: Clean's landing-page hero + immediately-below "How Clean works" sticky-pair section. (One contiguous above-the-fold-plus-one-scroll experience.)

APPROACH: Build a Fauna-Robotics-style hero (off-white page, pale blue accent, 6-second silent MP4 loop inside a 16:10 device-frame mock showing live MCP routing) with the Pacôme/Elva-grade scroll-coupled animation. Below the hero, build a sticky two-column "How Clean works" section: left column sticky at `top: 12vh; height: 100vh` showing three pinned frames driven by ScrollTrigger (raw payload → TOON-compressed payload with a tickering byte counter → context window with green/red token diff). Right column scrolls three text panels keyed to each frame. Headline set in Inter Tight Variable @ weight 320, `clamp(56px, 9vw, 168px)`, `letter-spacing: -0.04em`. Use SplitType + GSAP `expo.out` stagger for the headline reveal. Cursor follower with `mix-blend-mode: difference`. No 3D in v1 — Three.js is reserved for the later "Tools In, Context Out" helix demo.

STACK:
- next ^15.0 (App Router)
- tailwindcss ^4.0
- gsap ^3.13 + ScrollTrigger
- lenis ^1.1
- splittype ^0.3
- framer-motion ^12.0 (for the byte-counter `animate(count, target)` only)
- inter (variable) + jetbrains-mono via `next/font/local`
- lucide-react ^0.460 (static iconography)
- Native `<video muted playsinline loop autoplay>` for hero loop; H.265, 1920×1080, ≤2 MB

REFERENCE URLS:
- https://faunarobotics.com (palette, restraint, sticky-pair section)
- https://pacomepertant.com (motion-input coupling, GSAP `quickTo` smoothing pattern)
- https://elva.ai (typography rhythm, persistent character optional in v2)

KEY CONSTRAINTS: Clean is developer infrastructure for MCP context routing — it must read as engineered, fast, and quiet. Avoid: gradient backgrounds, glassmorphism cards, neon glows, "AI hero illustrations," generative-art Three.js, marquee rows of logos, particles or sparkles, hand-drawn arrows, anything Vercel-or-Linear-cosplay (subtle grid backgrounds are a cliché now). Mandatory: LCP ≤2.0 s on 4G, hero video must be under 2 MB, no layout shift, full keyboard navigation, `prefers-reduced-motion` collapses ScrollTrigger to instant transitions. Copy must mention exact savings ("avg 73% tokens saved", "p99 routing 11ms") — no vague "supercharge your agents" verbiage.
