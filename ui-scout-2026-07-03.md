## 1. NRG — Build Your Data Center — https://business.nrg.com/campaigns/build-your-data-center/
Awwwards SOTD (Jul 1, 2026) · Rogue Studio · Dev Award

WHAT IT DOES: A B2B interactive "configurator storyline" — the visitor scrolls through a fixed-viewport canvas where an isometric data-center diagram assembles module by module (power, cooling, land, on-site generation). Each ScrollTrigger step pins the canvas, swaps a layered SVG/PNG stack, transitions body copy in the right rail, and animates a running "capacity readout" (MW, acreage, uptime) with counter tweens. Navigation is a persistent left-rail chapter index that highlights the active section via `scrollTo` with scrub.

PACKAGES:
- `next@15`
- `gsap@3.13` + `gsap/ScrollTrigger` + `gsap/Observer`
- `lenis@1.3` (smooth-scroll driver piped into ScrollTrigger via `ScrollTrigger.scrollerProxy`)
- `framer-motion@12` for right-rail copy transitions
- `clsx`, `tailwindcss@4`
- `@vercel/og` for share cards
- Fonts: `Söhne` (Klim, self-hosted) + `JetBrains Mono@2` for numerics

IMPLEMENTATION:
1. Layout: single 100vh section, `position: sticky; top: 0` for the canvas stage; siblings are empty `min-h-[100svh]` scroll spacers that drive ScrollTrigger progress.
2. Canvas stage: absolute-positioned `<img>` layers (base plate, cooling, power, land, gen) with `will-change: transform, opacity`. On scroll progress `0→1`, animate each layer's `y: 40 → 0, opacity: 0 → 1, scale: 0.98 → 1` staggered by 0.08 with `ease: "power3.out"`.
3. Counter readouts: `gsap.to(counterRef, { textContent: targetMW, snap: { textContent: 1 }, duration: 1.2, ease: "power2.out" })` gated per pin.
4. Right rail: `AnimatePresence mode="wait"` with `initial={{ y: 12, opacity: 0 }} animate={{ y: 0, opacity: 1 }} exit={{ y: -12, opacity: 0 }} transition={{ duration: 0.35, ease: [0.22, 1, 0.36, 1] }}`.
5. Chapter nav: fixed left rail with `mix-blend-mode: difference` so it reads over dark/light plates; active state = 2px bar grows from `0 → 24px` width via `gsap.to`.
6. Smooth scroll: `const lenis = new Lenis({ lerp: 0.1, wheelMultiplier: 0.85 }); lenis.on('scroll', ScrollTrigger.update); gsap.ticker.add((t)=>lenis.raf(t*1000)); gsap.ticker.lagSmoothing(0);`.
7. Responsive: below 900px, kill pinning, render steps as vertical stack, disable counter tweens (use static end values), keep copy transitions.
8. Perf: preload the 5 layered PNGs (`<link rel="preload" as="image">`), decode async, use AVIF+PNG picture fallback; skip GSAP if `prefers-reduced-motion: reduce`.

WHY IT WORKS: The pinned-scroll storyline turns an abstract enterprise decision (buy megawatts) into a legible step-by-step diagram — spatial memory anchors each capability. Difference-blended nav preserves focal hierarchy without visual clutter. Numeric counters exploit the mono-space rhythm to imply precision.

GOOD: Storyline is legible in 30 seconds — a hyperscaler procurement lead learns the product without a demo.
BAD: Pinning on trackpad/keyboard is heavy — 4MB of layered PNGs would benefit from a single WebGL sprite atlas.

CLEAN APPLICATION: Apply this to Clean's landing page "How it works" section. Pin the viewport; scroll assembles the MCP proxy diagram layer by layer — (1) raw API response with token bloat, (2) TOON compression, (3) context routing, (4) audit trail, (5) agent-ready payload. Right rail flips the value prop per step ("83% fewer tokens", "typed schema", "SOC2-ready"). Counter tween the token-savings % from 0 → 83. Use `mix-blend-mode: difference` for the chapter index so it reads over the code preview.

## 2. Wembi — https://www.awwwards.com/sites/wembi
Awwwards SOTD (Jul 2, 2026) · PRO + Dev Award

WHAT IT DOES: A single-page studio site with a full-viewport GLSL shader hero — a slow-moving noise-warped gradient plane driven by mouse-position uniforms, over which a giant serif wordmark sits pinned. Scrolling triggers a horizontal case-study rail with `x` translation tied to `scrollY` (Lenis + GSAP), and each card reveals a WebGL image plane with an RGB-split hover effect. Cursor is replaced by a custom SVG dot that grows to 64px and shows the card label on hover.

PACKAGES:
- `three@0.170`, `@react-three/fiber@9`, `@react-three/drei@10`
- `three-custom-shader-material@6` for injecting into MeshStandardMaterial
- `gsap@3.13` + `ScrollTrigger`
- `lenis@1.3`
- `react@19`, `next@15`
- `postprocessing@6` for the RGB shift + bloom
- Fonts: `PP Editorial New` (Pangram Pangram) + `Neue Haas Grotesk`

IMPLEMENTATION:
1. Hero shader: full-screen `<mesh>` with `PlaneGeometry` sized to camera frustum; fragment shader mixes 3-color gradient by `simplex3d(vUv*1.5 + uTime*0.08)`, warped by `uMouse` lerped in `useFrame` (`mouse.lerp(target, 0.06)`).
2. Wordmark: HTML overlay, `font-size: clamp(96px, 18vw, 340px)`, letter-spacing `-0.04em`, `mix-blend-mode: difference` over the shader.
3. Horizontal rail: outer wrapper `height: 400vh`; inner `position: sticky; top:0; height:100vh; display:flex;` — animate `x: 0 → -(scrollWidth - innerWidth)` on scroll progress.
4. Card WebGL plane: `<Image>` from drei, custom shader material adding `uHover` uniform; on hover `gsap.to(mat.uniforms.uHover, { value: 1, duration: 0.6, ease: "expo.out" })`. Fragment offsets R/G/B channels by `uHover * 0.02 * vec2(1,0)` for chromatic aberration.
5. Custom cursor: `<div>` transformed via `requestAnimationFrame`, lerp `0.15`; scale/label swap on `hover` state per element via `data-cursor` attribute pattern.
6. Postprocessing: `<EffectComposer>` with `Bloom` (intensity 0.35, luminanceThreshold 0.85) + `Noise` (opacity 0.04, blendFunction: OVERLAY).
7. Perf: DPR clamped `[1, 1.5]`, `frameloop="demand"` bumped to `"always"` only in hero viewport via IntersectionObserver.

WHY IT WORKS: The editorial serif over a slow shader creates a "gallery + tech" duality — signals both craft and capability. Horizontal rail breaks the vertical monotony of typical studio sites; RGB-split hover is a signature that reads as "advanced technique" without being gratuitous. Difference-blend wordmark guarantees contrast on any shader hue.

GOOD: Cohesive visual language — every hover, scroll, and cursor state moves in the same easing family (`expo.out`, ~0.6s).
BAD: Horizontal scroll fights native browser gestures on some trackpads; wordmark blend-mode hurts legibility for low-vision users.

CLEAN APPLICATION: Use the shader hero for Clean's landing page above-the-fold. Replace the noise plane with a subtle "context stream" — a shader that renders flowing token particles that clump into TOON schema shapes when the mouse hovers, dispersing back to noise when it leaves. Overlay the wordmark "Clean." in `PP Editorial New` at 18vw with `mix-blend-mode: difference`. Adopt the RGB-split hover for the "Compare formats" card in the value-prop grid — hovering shifts the JSON → TOON diff by a few pixels per channel.

## 3. Units — https://www.awwwards.com/sites/units
Awwwards SOTD (Jul 3, 2026)

WHAT IT DOES: Minimalist editorial portfolio built around a 12-column grid where individual "units" (case-study tiles) rearrange between two states: a dense grid on load, and an expanded reading view when clicked. The transition uses shared-element FLIP animations — clicking a tile morphs it to a full-bleed article, preserving image continuity. Typography is a variable-weight display serif that interpolates weight on scroll velocity.

PACKAGES:
- `next@15`, `react@19`
- `motion@12` (Framer Motion successor, using `LayoutGroup` + `motion.div` with `layoutId`)
- `gsap@3.13` for scroll-velocity typography
- `lenis@1.3`
- `next/font/local` for variable font
- `tailwindcss@4`
- Font: `Söhne Variable` or `Migra Variable` (Pangram Pangram)

IMPLEMENTATION:
1. Grid: `display: grid; grid-template-columns: repeat(12, 1fr); gap: 24px;` — each unit spans variable columns (`col-span-3`, `col-span-6`) authored per tile.
2. Shared-element transition: wrap each tile and its expanded article in the same `<LayoutGroup>`; use `motion.img layoutId={id}` and `motion.h2 layoutId={id+'-title'}`. On click, route change but the layout animates via `transition={{ layout: { duration: 0.6, ease: [0.83, 0, 0.17, 1] } }}`.
3. Variable-weight typography: on scroll, track velocity via `lenis.on('scroll', ({velocity}) => setWeight(clamp(400 + velocity*80, 300, 900)))`; apply as CSS custom property `--wght` bound to `font-variation-settings: "wght" var(--wght)`.
4. Grid reveal on load: stagger tiles with `motion.div initial={{ y: 24, opacity: 0 }} animate={{ y: 0, opacity: 1 }} transition={{ delay: i*0.04, duration: 0.5, ease: [0.22, 1, 0.36, 1] }}`.
5. Cursor: no custom cursor — magnifier state uses `filter: brightness(1.05)` on hover only, keeping restraint.
6. Color: single background `#F4F1EA` (warm off-white), text `#1A1A1A`, accent `#B14A26` used only on hover underlines.
7. Perf: pre-generate blur-hash `data:` URIs at build with `plaiceholder`; use `next/image` with `placeholder="blur"` and `sizes` tuned per col-span.

WHY IT WORKS: Restraint is the design — one grid, two color values, one variable font. The layout-shared morph tricks the brain into believing the article "was always the tile," collapsing the perceived load time. Weight-on-velocity is a subliminal signal of your own energy on the page.

GOOD: The FLIP transition genuinely removes a mental context-switch — best-in-class for portfolio navigation.
BAD: Deep-linking to an article shows the fully-expanded state without the morph — the delight only exists for click-through visitors.

CLEAN APPLICATION: Apply the shared-element morph to Clean's docs → example page transition. Each MCP example card on `/examples` uses `layoutId` matched to the expanded example page — clicking morphs the card's syntax-highlighted snippet directly into the full example. Use variable-weight numerics from `JetBrains Mono` on the pricing page so the token-savings figure "breathes" heavier as the visitor scrolls faster.

## 4. Reactive Depth — Scroll-Driven 3D Image Tube — https://tympanus.net/codrops/2026/02/17/reactive-depth-building-a-scroll-driven-3d-image-tube-with-react-three-fiber/
Codrops · Author: Matis · Feb 17, 2026

WHAT IT DOES: A vertical scroll drives an infinitely looping cylindrical arrangement of image planes wrapped around the camera. Scroll delta feeds a shader-side displacement uniform that curls each plane's UVs along a Bezier curve; inertia keeps motion going for ~800ms after scroll ends. DOM overlays (titles, indexes) are z-locked to each plane via projected-position sync.

PACKAGES:
- `three@0.170`, `@react-three/fiber@9`, `@react-three/drei@10`
- `leva@0.10` (only for authoring; strip in prod)
- `gsap@3.13` for the inertia lerp
- `zustand@5` for a shared scroll store
- `glslify` for shader includes

IMPLEMENTATION:
1. Scene: single `<Canvas camera={{ position: [0,0,0], fov: 55 }}>`; a `<group>` holds N (=24) planes arranged via `plane.rotation.y = i * (Math.PI*2/N); plane.position = new Vector3(0,0,-R).applyAxisAngle(Y, i*angle);` with R≈6.
2. Shader material (GLSL fragment/vertex extends `shaderMaterial` from drei): vertex applies `pos.y += sin(uv.x*3.14 + uProgress*2.0) * uCurl`; fragment samples texture with `vec2 uvWarped = vUv + normal.xy * uHover * 0.05;`.
3. Scroll drive: `useEffect(() => window.addEventListener('wheel', e => { targetScroll += e.deltaY * 0.001 }))`; each frame lerp actual → target with factor 0.08; feed as `uProgress` uniform + rotate the group `y` by `progress * Math.PI*2 / VISIBLE_COUNT`.
4. Inertia: on wheel-end (`clearTimeout` pattern), let target continue via `gsap.to({v:velocity},{v:0, duration: 0.8, ease:'power3.out', onUpdate: () => { targetScroll += this.v*0.01 }})`.
5. Infinite loop: modulo the visible index against N so planes wrap; swap textures at the wrap point using a ping-pong array.
6. DOM overlay sync: for each plane, project world position to screen via `camera.project()` each frame, write to a CSS variable on the matching absolutely-positioned `<div>` — `--x`, `--y`.
7. Perf: use `KTX2Loader` for compressed textures; DPR max 1.5; instanced attributes if N > 60.

WHY IT WORKS: A tube collapses "long scroll" into "rotating drum" — you feel infinite content without a scrollbar reaching the bottom. Curl shader adds physicality (things bend under momentum) that pure translate/opacity can't sell. Inertia is where interactions read as "real" vs "software."

GOOD: The scroll → shader → inertia loop is a masterclass in unified motion — every gesture has the same physical model.
BAD: Discoverability is low; first-time users don't know it wraps. Screen readers see nothing. Provide a text-alternate section.

CLEAN APPLICATION: Use this on Clean's `/customers` or logo-cloud section — rotate a tube of customer logos + agent-framework logos (Anthropic, OpenAI, LangChain, Vercel AI SDK, LlamaIndex, MCP spec). Bend the planes gently under scroll. Sync a small `<caption>` overlay showing the framework name + a token-savings % when a plane is centered. On desktop only; on mobile, degrade to a horizontal marquee.

## 5. Shader.se — Scroll-Driven WebGPU Scene Transitions — https://tympanus.net/codrops/2026/05/19/80s-business-tech-seamless-scene-transitions-inside-shader-ses-scroll-driven-webgpu-pipeline/
Codrops case study · May 19, 2026

WHAT IT DOES: A single WebGPU pipeline renders 4 distinct scenes (grid room, corridor, terminal, boardroom) with a shared g-buffer; scroll interpolates a scene-index uniform that dissolves between scenes using a per-pixel shader gate. Aesthetic is 80s CGA business rendering — 16-color palette, ordered dither, thin CRT scanlines. Copy animates in as ASCII-typewriter overlays.

PACKAGES:
- `@webgpu/types` + native WebGPU API (no wrapper library — hand-authored pipeline)
- `wgsl-linker` for shader includes
- `gsap@3.13` for orchestration
- `lenis@1.3`
- `next@15`
- Fonts: `Berkeley Mono` (US Graphics) or `Cartograph CF` for terminal type
- Fallback: `three@0.170` WebGL2 renderer for browsers without WebGPU (feature-detect via `navigator.gpu`)

IMPLEMENTATION:
1. Pipeline: create a single `GPURenderPipeline` with two texture attachments (previous scene, next scene); a fullscreen fragment mixes them via a Bayer-dither gate: `output = step(dither8x8(fragCoord), uProgress) ? nextSample : prevSample;`.
2. Scene index: `uSceneFrom` = floor(scroll * 4), `uSceneTo` = uSceneFrom+1, `uProgress` = fract(scroll*4). Update uniforms per frame from Lenis progress.
3. Color quantization: after scene render, post-pass snaps color to nearest of 16 palette entries stored in a small storage buffer; add `+dither[fragCoord % 8]` before quantize to preserve gradient depth.
4. CRT overlay: separate render pass draws horizontal scanlines (`sin(uv.y*resolution.y*3.14) * 0.06`) and a subtle vignette (`smoothstep(0.9, 0.4, distance(uv,0.5))`).
5. Typewriter text: `<div>` with per-char `<span>`; on scene-enter (`ScrollTrigger.enter`), animate `--i` from 0 → charCount using `gsap.to` with `duration: 0.02*charCount, ease: 'none'`; each char becomes visible when `index < --i` via CSS `:nth-child` sibling combinator.
6. Fallback: if `!navigator.gpu`, render the same 4 scenes as pre-baked JPEGs cross-faded with CSS blend + a static SVG dither overlay — visually 90% identical.
7. Perf: pipeline compiled once, reused across scene switches; texture atlases pre-uploaded on first idle callback.

WHY IT WORKS: The 80s CGA aesthetic reads as "technical, specific, opinionated" — the antithesis of generic SaaS pastel gradients. Dither-based transitions have a mechanical honesty that fades/blurs lack — they read as "computed" not "designed over." Locks into developer nostalgia + status.

GOOD: The scene-crossfade shader gate is genuinely novel and cheap — a single fullscreen quad handles all transition variety.
BAD: WebGPU adoption is still ~65% in July 2026 — the fallback path must be bulletproof or a third of users see nothing.

CLEAN APPLICATION: Use this style — not the exact tech — on Clean's `/manifesto` or `/why-clean` page. Render 4 "before/after" scenes in a WebGPU pipeline: (1) raw REST payload, (2) TOON-compressed, (3) routed via Clean proxy, (4) agent-ready. Dither-gate between them on scroll. Berkeley Mono type. On non-WebGPU browsers, ship the same 4 as pre-rendered PNG stills with a CSS blend cross-fade — the aesthetic survives.

## BUILD DIRECTIVE

TARGET: Clean landing page — full rebuild of the "How it works" section (between hero and pricing).

APPROACH: Combine NRG's pinned-scroll storyline (Ref 1) with Reactive Depth's inertial shader physics (Ref 4) and Wembi's editorial typography restraint (Ref 2). Build a single sticky-viewport section, ~400vh tall, that walks the visitor through 5 stages of an MCP request as it flows through Clean: raw payload → TOON compression → context routing → audit envelope → agent-ready delivery. Each stage pins the canvas, morphs a WebGL "context stream" shader (particles flowing along a slight curve, curl responds to scroll velocity), and swaps right-rail copy + a counter tween for token savings (0% → 83%). Left-rail chapter index in `mix-blend-mode: difference` so it reads over dark/light shader hues. Below 900px, collapse to vertical static stack — no pinning, no shader, just the 5 steps as illustrated cards.

STACK:
- `next@15`, `react@19`, `typescript@5.6`
- `tailwindcss@4` + `clsx`
- `three@0.170`, `@react-three/fiber@9`, `@react-three/drei@10`, `postprocessing@6`
- `gsap@3.13` + `ScrollTrigger` + `Observer`
- `lenis@1.3` (piped into ScrollTrigger via `scrollerProxy`)
- `motion@12` for right-rail copy `AnimatePresence`
- `zustand@5` for shared scroll/step state
- `plaiceholder` for blur placeholders
- Fonts: `PP Editorial New` (heading), `Söhne` (body), `JetBrains Mono` (numerics)
- Feature detection: `@react-three/fiber` DPR clamp `[1, 1.5]`, `prefers-reduced-motion` fallback to static stack

REFERENCE URLS:
- https://business.nrg.com/campaigns/build-your-data-center/ (pinned storyline structure)
- https://tympanus.net/codrops/2026/02/17/reactive-depth-building-a-scroll-driven-3d-image-tube-with-react-three-fiber/ (inertial shader model)
- https://www.awwwards.com/sites/wembi (typography + shader hero restraint)

KEY CONSTRAINTS: Clean is developer infrastructure — the result must feel technically sophisticated and fast, not flashy. AVOID: (1) purple/pink SaaS gradients — use a restrained palette of `#0A0A0A`, `#F4F1EA`, single accent `#B14A26`; (2) generic "AI orb" glow blobs — the shader must visualize the actual data flow (token particles → TOON schema shapes), not decorative energy; (3) custom cursors — restraint over signature; (4) autoplay hero video — WebGL only, or SVG-static fallback; (5) marketing superlatives ("blazing fast", "revolutionary") — use hard numbers with mono numerics (`83% fewer tokens`, `<8ms proxy overhead`, `SOC2 Type II`); (6) any dependency on WebGPU today — R3F/WebGL2 only, since WebGPU coverage still trails ~35% of visitors; (7) horizontal scroll — trackpad gesture conflict.
