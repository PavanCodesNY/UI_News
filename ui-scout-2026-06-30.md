## 1. Meech213 (Blackpepper Studio) — https://www.awwwards.com/sites/meech213 (Awwwards SOTD 2026-06-30)

WHAT IT DOES: Single-page editorial portfolio on a strict 12-column Swiss grid with oversized serif display type as the anchor and small-caps metadata in the gutters. Hero loads with a staggered character-by-character mask reveal on the headline, then a single fullscreen video pin-scrubs frame range as the user scrolls (playhead bound to ScrollTrigger progress, not autoplay). Project tiles below use pinned horizontal scroll that converts vertical wheel into x-translate, with each tile cross-fading a still-to-video transition on hover via clip-path inset.

PACKAGES:
- `next@15` (Clean React port; production site is Webflow)
- `tailwindcss@4`
- `gsap@3.13` + `ScrollTrigger` + `SplitText`
- `@studio-freight/lenis@1.x` (smooth scroll)
- `fontsource/instrument-serif` or self-hosted `PP Editorial New`
- `fontsource/jetbrains-mono` (metadata captions)
- Native `<video preload="auto" muted playsInline>` for scrubbing

IMPLEMENTATION:
1. Wrap the document in `<Lenis options={{ lerp: 0.08, smoothWheel: true }}>`.
2. HeroVideoSection = absolutely positioned `<video>` with `currentTime` driven by `gsap.to(video, { currentTime: video.duration, ease: 'none', scrollTrigger: { trigger: section, start: 'top top', end: '+=2500', scrub: 0.5, pin: true } })`.
3. Overlay headline: `SplitText.create(h1, { type: 'chars' })` then `gsap.from(chars, { yPercent: 110, stagger: 0.025, duration: 1.1, ease: 'expo.out' })`.
4. Horizontal project rail: sticky `100vh` wrapper, child flex track translated by `gsap.to(track, { x: -(track.scrollWidth - innerWidth), scrollTrigger: { trigger: wrapper, scrub: 1, end: () => track.scrollWidth } })`.
5. Type scale: H1 `clamp(72px, 12vw, 220px) / line-height 0.92 / letter-spacing -0.04em`; metadata `11px uppercase tracked +0.18em`.
6. Grid gutter 24px desktop, 12px mobile. Cards reveal via `clip-path: inset(0 0 100% 0)` → `inset(0 0 0 0)` over 0.9s `cubic-bezier(0.65, 0, 0.35, 1)`.
7. Breakpoints: 1440 / 1024 / 768 / 390. Mobile kills horizontal rail; falls back to stacked cards with `loading="lazy"` videos.

WHY IT WORKS: Aggressive serif display against monospaced micro-labels creates a typographic duet — maximum contrast = clear hierarchy. Scroll-scrubbed video gives visitors a sense of authored control over time (motion semantics that say "every frame is intentional"). Whitespace rhythm follows a 4/8/16/32px scale so the eye finds rest points between dense type clusters.

GOOD / BAD: GOOD — scroll-scrubbed video is genuinely best-in-class, no fake loops. BAD — preloading full hero video balloons LCP on slow connections; no `prefers-reduced-motion` fallback.

CLEAN APPLICATION: Apply this to Clean's landing-page hero by replacing the demo video with a scroll-scrubbed terminal recording of the MCP proxy in action — agent issues a request, proxy compresses 4KB of JSON down to 800 bytes of TOON, downstream tool returns. Pin the hero, scrub terminal frames against scroll progress, drop the H1 ("Context that fits.") as a SplitText character reveal at scroll progress 0. Instrument Serif for the display word, JetBrains Mono for the inline metric chip ("-78% tokens").

## 2. Fauna Robotics — https://faunarobotics.com/ (Awwwards SOTD 2026-06-16)

WHAT IT DOES: Product landing for the Sprout humanoid platform. Above-the-fold is a single autoplaying Vimeo demo video (1440p, muted, loop) framed by minimal black serif "Capable, safe, fun." overlay. Below, 3-card audience matrix (developers / enterprises / researchers) with another inline video, then a feature grid with looping micro-clips per capability and a UGC mosaic of 8 vertical Vimeo loops in packed masonry. Built on Webflow; identity carried entirely by motion (no decorative graphics — every visual is a real robot clip). Palette locked to two tones (`#EFEFEF` + `#D1E3FF`) so video content carries all chroma.

PACKAGES:
- `webflow` (production) OR `next@15` + `react@19` (Clean port)
- `tailwindcss@4`
- `@vimeo/player@2.x` (adaptive bitrate inline players)
- `gsap@3.13` + `ScrollTrigger` (in-view reveals)
- `@studio-freight/lenis@1.x`
- `next/font` — PP Right Grotesk or Söhne (custom geometric sans for body, modern grotesk for display)
- `@vercel/og`

IMPLEMENTATION:
1. Hero: `<VimeoPlayer videoId={...} background autoplay muted loop responsive />` at 100vw/100svh with a `1px solid rgba(0,0,0,0.06)` `border-radius: 20px` inset 16px from edges — the signature framing device recurring on every video.
2. Headline overlays bottom-left at `clamp(48px, 6vw, 96px)`, weight 500, `letter-spacing: -0.03em`.
3. Cross-section fades use `ScrollTrigger.create({ trigger: section, start: 'top 60%', onEnter: () => gsap.to(videoA, { opacity: 0, duration: 0.6, ease: 'power2.out' }) })` — opacity-swap only, NEVER scrub `currentTime` on scroll (thermal on iOS).
4. Capability grid: 2×2 desktop / 1-col mobile, cells `aspect-ratio: 4/3`, videos play on IntersectionObserver entry and pause on exit (battery + bandwidth).
5. UGC mosaic: CSS Grid `repeat(4, 1fr)` with `grid-auto-rows: 240px` and selective `grid-row: span 2` for rhythm; GSAP stagger reveals `y: 60, opacity: 0, stagger: 0.06, scrollTrigger: { start: 'top 80%' }`.
6. Color tokens: bg `#FAFAF7`, fg `#0A0A0A`, accent `#0066FF`. NO shadows — depth comes only from inset video cards.
7. Trusted-by strip: `grid-template-columns: repeat(5, 1fr)`, logos at `opacity: 0.7`.

WHY IT WORKS: For product-driven claims, motion video is the only credible proof — this site refuses to fake it with renders or diagrams. Inset-card framing creates a quiet rhythm that prevents heavy video from feeling chaotic (Gestalt common-region grouping). Off-white background with unsaturated palette lets the robot be the only saturated thing on the page.

GOOD / BAD: GOOD — 100% real-footage policy makes every claim verifiable. BAD — multiple Vimeo players initialized concurrently hurt TTI on mid-tier mobile; no `<source media>` fallback for users blocking autoplay.

CLEAN APPLICATION: Apply this to Clean's landing-page below-the-fold capability grid by replacing robot clips with looping 720p screen recordings of MCP proxy behavior — one cell per capability (Compression / Routing / Caching / Observability). Use the exact inset-card framing (border-radius 20px, 16px inset, 1px soft border) for every tile. Off-white `#FAFAF7` background, single accent for hover states. Inline videos IntersectionObserver-gated.

## 3. Sui — https://sui.io/ (Awwwards SOTD 2026-06-24)

WHAT IT DOES: L1 blockchain marketing site by Sui_Foundation_Design × AKKA Studio PRO. Structure is a modular brand-system landing: header animation (mask reveal on load), transition system between routes (full-viewport wipe carrying a WebGL noise texture), menu expands to full-screen with running video background, modular section blocks driven by After-Effects-authored Lottie animations. Palette is single hero blue `#298DFF` on `#000`. Animation timing is dance-choreographed to a shared beat: 240ms micro, 640ms section, 1200ms route.

PACKAGES:
- `next@14` (app router)
- `framer-motion@11` (layout transitions, AnimatePresence)
- `lottie-react@2.4` (After Effects → JSON playback)
- `ogl@1.0` OR `three@0.166` (route-transition WebGL plane)
- `tailwindcss@3.4`
- `@vercel/og` (dynamic OG images matching brand animation frames)
- Fonts: `Inter` display + `JetBrains Mono` for hex/hash strings

IMPLEMENTATION:
1. Route transitions: wrap `<AnimatePresence mode="wait">` around `page.tsx`. Exit animates a top-layer `<canvas>` from `clip-path: inset(0 100% 0 0)` to `inset(0 0 0 0)` over 700ms `cubic-bezier(0.76, 0, 0.24, 1)`; canvas paints WebGL fBm noise in `#298DFF`.
2. Menu: `<Dialog>` full-screen, background muted `<video>`, links use `staggerChildren: 0.06` with `y: 40 → 0` and `opacity`.
3. Section blocks: each is a `<figure>` with a Lottie animation triggered by `useInView({ margin: '-20% 0px' })`; play once, freeze on last frame.
4. Type-in-motion: hero headline uses `SplitText` with per-word `y: 100%, rotateX: -30deg → 0`, 40ms stagger, 900ms `expo.out`.
5. Grid: 12-col, `gap: clamp(16px, 1.5vw, 32px)`, container `max-width: 1440px, padding-inline: clamp(24px, 5vw, 96px)`.
6. Dark mode only: body `background: #000; color: #EAEAEA;` — accent applied only to interactive states and animated accents.
7. Timing tokens: `--dur-micro: 240ms`, `--dur-section: 640ms`, `--dur-route: 1200ms`, `--ease: cubic-bezier(0.76, 0, 0.24, 1)`.

WHY IT WORKS: Single-accent palette + shared animation timing makes the site feel like one continuous piece of software rather than a stack of pages. Route transitions carry brand identity forward (the noise texture IS the brand), raising perceived trust for an infra product. Type kinetics do the persuasion — the copy itself becomes the animation.

GOOD / BAD: GOOD — every animation earns its place through timing consistency; nothing feels random. BAD — WebGL transition adds ~180ms to LCP on mid-tier Android; no reduced-motion fallback.

CLEAN APPLICATION: Apply this to Clean's landing page + `/product` route by adopting the SAME timing scale (240 / 640 / 1200 ms) across every animation. Build a single WebGL noise transition tinted in Clean's brand color, and reuse it as (a) page-route transition, (b) hero background wash, (c) hover state on primary CTA. Bind all motion to `cubic-bezier(0.76, 0, 0.24, 1)` so the whole product feels like one continuous surface.

## 4. shadcn/ui Chat Components — https://ui.shadcn.com/docs/changelog/2026-06-chat-components (shipped June 2026)

WHAT IT DOES: New primitive set for building agent/chat UIs: `MessageScroller`, `Message`, `Bubble`, `Attachment`, `Marker`. `MessageScroller` is the hard part — a scroll container that solves anchored auto-follow (pin to bottom while a message streams), scroll-position preservation across layout shifts (bubbles growing as tokens arrive), prepended history (loading older messages without a jump), jump-to-message, and per-item visibility tracking. Ships as `@shadcn/react/message-scroller` (headless, unstyled) with a shadcn-styled registry wrapper on top. Base UI + Radix variants both supported.

PACKAGES:
- `@shadcn/react@0.1` (new headless primitives package)
- `shadcn` CLI: `npx shadcn@latest add message-scroller message bubble marker attachment`
- Peer: `react@19`, `react-dom@19`
- `tailwindcss@3.4` + shadcn preset
- `@radix-ui/react-primitive` (peer)
- `lucide-react` (icons)
- `ai@3` (Vercel AI SDK — `useChat()` streaming, only for the interactive playground)

IMPLEMENTATION:
1. Compose:
   ```tsx
   <MessageScroller className="h-[600px]">
     <MessageScrollerContent>
       {turns.map(t => (
         <MessageScrollerItem key={t.id}>
           {t.role === 'system'
             ? <Marker>{t.text}</Marker>
             : <Message from={t.role}><Bubble>{t.text}</Bubble></Message>}
         </MessageScrollerItem>
       ))}
     </MessageScrollerContent>
     <MessageScrollerButton to="end" />
   </MessageScroller>
   ```
2. Streaming: pipe token deltas into the last `<Bubble>`; the scroller's anchor logic pins the view to bottom unless the user scrolls up (auto-follow pauses and reveals the "jump to latest" `MessageScrollerButton`).
3. Prepend history: when the user hits the top, fetch older turns and prepend to `turns` — scroller preserves the visible anchor via ResizeObserver so nothing jumps.
4. Style: `Bubble` variants — `variant="user"` right-aligned, filled brand; `variant="assistant"` left-aligned, `bg-muted`, `border`. `max-width: 65ch`.
5. Reactions/inline actions: `<BubbleReactions>` and `<BubbleActions>` slots — no hand-rolled hover rows.
6. Accessibility: scroller sets `role="log" aria-live="polite" aria-atomic="false"`. Do NOT add duplicate live regions.

WHY IT WORKS: These primitives encode the invisible-but-critical UX of streaming chat (anchor drift, history load, jump control) as reusable behaviors, freeing product teams from re-solving them per surface. Visual defaults are conservative — Bubble is intentionally boring — because in agent UIs, chrome fighting content is the failure mode. Headless-then-styled architecture means teams adopt behaviors without adopting the look.

GOOD / BAD: GOOD — first component set that treats "chat" as behaviorally distinct from "list of divs." Solves a category of bugs (auto-follow-during-stream) that every AI product ships with. BAD — coupled to React 19 and shadcn's Tailwind preset; Vue/Solid users are back to hand-rolling.

CLEAN APPLICATION: Apply this to Clean's product-demo section (the interactive "try an agent" widget on the marketing page) by dropping in `MessageScroller` + `Message` + `Bubble` + `Marker`. Wire Bubble content to a canned agent trace: user turn → tool-call Marker ("calling get_page tool via MCP proxy") → assistant Bubble streaming a TOON-formatted response. The marketing page gets the exact same interaction primitives that live in the actual product — the demo IS the product.

## 5. Indigo Laboratory — https://www.awwwards.com/sites/indigo-laboratory (Awwwards SOTD 2026-06-18)

WHAT IT DOES: Music × fashion editorial site by FPTP Studio × Marea Studio. Structure is a chapter-based scroll with a persistent WebGL canvas layer using Curtains.js to attach shader planes to DOM `<img>` elements. Scroll velocity drives a per-plane displacement uniform (waves pass over the image in the direction of scroll). Sound is diegetic — ambient stems fade in per section, mixed via WebAudio. Nuxt.js handles route-level SSR; transitions between chapters cross-fade the audio bed.

PACKAGES:
- `nuxt@3.12`
- `curtainsjs@8.1` (DOM-driven WebGL planes)
- `gsap@3.13` + `ScrollTrigger`
- Native `AudioContext` (WebAudio graph)
- `@studio-freight/lenis@1.x`
- `tailwindcss@3.4`
- `nuxt-image` (AVIF pipeline)
- Fonts: display serif (`PP Editorial New`-style) + `Inter`

IMPLEMENTATION:
1. Curtains setup:
   ```js
   const curtains = new Curtains({
     container: 'canvas-root',
     pixelRatio: Math.min(devicePixelRatio, 1.5)
   })
   document.querySelectorAll('[data-plane]').forEach(el => {
     new Plane(curtains, el, {
       vertexShader, fragmentShader,
       uniforms: { uTime: {value:0}, uVelocity: {value:0} }
     })
   })
   ```
2. Vertex shader displaces vertices along Y by `sin(uv.x * 10.0 + uTime) * uVelocity * 0.05`.
3. Scroll velocity: subscribe to Lenis `scroll` event, low-pass filter `v = v*0.85 + newV*0.15`, push into every plane's `uVelocity` on rAF.
4. Sound layer: `AudioContext` graph with `GainNode` per chapter; `ScrollTrigger` for each `<section>` ramps its gain `0 → 0.6` on enter, `→ 0` on leave. Global unmute button — NEVER autoplay audio.
5. Chapter transitions: page-level `<Transition name="chapter">` with 900ms fade; audio crossfade runs in parallel.
6. Reduced motion / no-audio path: if `matchMedia('(prefers-reduced-motion: reduce)')` or the user hasn't clicked unmute, skip Curtains init and serve plain `<img>` tags — Curtains gracefully degrades because planes were DOM elements to begin with.

WHY IT WORKS: Curtains's DOM-plane model means layout is authored in normal CSS — SEO, accessibility, and responsive behavior come free. Shader effect is subtle enough (max ~3% displacement) to feel like paper breathing, not glitching — the correct emotional register for editorial content. Audio synced to scroll gives a fourth dimension of pacing designers rarely reach.

GOOD / BAD: GOOD — graceful degradation is textbook. Site works without JS, without audio, without WebGL — you just lose progressive layers of atmosphere. BAD — ambient audio ambush (even with a mute button) is disorienting on tabbed workflows; a persistent global audio-off indicator would fix this.

CLEAN APPLICATION: Apply this to Clean's docs / blog surface (a slower reading context) by wrapping architecture diagrams in Curtains planes with a low-amplitude displacement tied to scroll velocity — the diagrams "settle" as the reader stops scrolling. No audio (wrong context for developer docs). Use it only on hero images and diagram cards, not body copy.

## BUILD DIRECTIVE

TARGET: Clean landing-page **hero + Product Demo section** as one continuous above-and-just-below-the-fold experience.

APPROACH: Rebuild Clean's hero as a scroll-pinned scrubbed terminal-recording sequence (Meech213 pattern) with Sui's route-transition WebGL noise wash as the ambient background at 15% opacity. Headline splits character-by-character with GSAP SplitText on load using Sui's timing tokens (`--dur-section: 640ms`, `cubic-bezier(0.76, 0, 0.24, 1)`). As the user scrolls into the pinned section, a 12-second terminal recording scrubs frame-by-frame showing an agent making a request → MCP proxy compressing to TOON → downstream tool responding, with an animated counter overlaying token reduction ("4,128 → 912 tokens"). Directly below the pin, drop the shadcn June 2026 chat primitives (`MessageScroller` + `Message` + `Bubble` + `Marker`) as an interactive agent playground rendering the same trace as a real streamed conversation — the marketing demo becomes indistinguishable from the product. Frame both the terminal video and the chat playground with Fauna Robotics's signature 20px-radius inset card (16px inset from viewport edges, 1px `rgba(0,0,0,0.06)` border, no shadow).

STACK:
- `next@15` + `react@19` + `typescript@5.6`
- `tailwindcss@4` + shadcn preset
- `gsap@3.13` (ScrollTrigger + SplitText + Observer)
- `@studio-freight/lenis@1.x`
- `@shadcn/react@0.1` + registry adds: `message-scroller`, `message`, `bubble`, `marker`, `attachment`
- `ogl@1.0` (single WebGL noise plane — lighter than three.js)
- `ai@3` (Vercel AI SDK, playground streaming simulation only — no real model on marketing)
- `next/font` self-hosting `Instrument Serif` (display) + `JetBrains Mono` (metrics + TOON payload) + `Inter` (body)
- `lucide-react`
- `@vercel/og`

REFERENCE URLS:
- https://www.awwwards.com/sites/meech213 (scroll-scrubbed hero + SplitText character reveal)
- https://faunarobotics.com/ (inset-card framing + real-footage discipline)
- https://ui.shadcn.com/docs/changelog/2026-06-chat-components (agent playground primitives)

KEY CONSTRAINTS: Clean is developer infrastructure — every animation must justify itself with information density. AVOID: gradient blob hero backgrounds, generic glassmorphism, decorative 3D that doesn't represent product behavior, autoplay audio, RGB-split shaders, custom cursors, horizontal-scroll gymnastics on marketing surfaces, Aceternity-style sparkle/beam text effects, more than 1 display typeface, more than 1 accent color, any motion longer than 1.2s without scroll input, any animation lacking a `prefers-reduced-motion` static fallback, hero video files over 1.5 MB, three concurrent WebGL contexts. TOON payloads and tool-call markers MUST be technically accurate (real TOON syntax, real MCP tool-call shape) — an infra audience will screenshot and roast fake JSON. Terminal recording must be a webm under 800 KB with an AVIF poster frame for instant first paint. Every video, including the hero, IntersectionObserver-gated. LCP target: < 2.0s on mid-tier Android. WCAG AA contrast on the noise canvas (test 4.5:1 against body text).
