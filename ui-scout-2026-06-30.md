## Meech213 (Blackpepper Studio / Nkenna Amadi) — https://www.awwwards.com/sites/meech213

WHAT IT DOES: Single-page editorial portfolio built on a strict 12-column Swiss grid with oversized serif display type as the anchor and small-caps metadata in the gutters. Hero loads with a staggered character-by-character mask reveal on the headline, then a single fullscreen video pin-scrubs frame range as the user scrolls (the playhead is bound to ScrollTrigger progress, not autoplay). Project tiles below use a horizontal pinned scroll that converts vertical wheel input into x-translate, with each tile cross-fading a still-to-video transition on hover via clip-path inset.

PACKAGES:
- next@15 OR webflow runtime (this site ships Webflow; for the Clean React port use next)
- tailwindcss@4
- gsap@3.13 with ScrollTrigger and SplitText plugins
- @studio-freight/lenis@1.x for smooth scroll
- framer-motion@11 (alt path if not using GSAP)
- fontsource/instrument-serif or local self-host of "PP Editorial New"
- fontsource/jetbrains-mono for metadata captions
- @vimeo/player or native <video preload="auto"> for the scroll-scrubbed clip

IMPLEMENTATION: Wrap the document in `<Lenis options={{ lerp: 0.08, smoothWheel: true }}>`. Build a top-level `<HeroVideoSection>` that contains an absolutely positioned `<video muted playsInline preload="auto">` with `currentTime` driven by `gsap.to(video, { currentTime: video.duration, ease: 'none', scrollTrigger: { trigger: section, start: 'top top', end: '+=2500', scrub: 0.5, pin: true }})`. Overlay headline uses `SplitText.create(h1, { type: 'chars' })` then `gsap.from(chars, { yPercent: 110, stagger: 0.025, duration: 1.1, ease: 'expo.out' })`. For the horizontal project rail: a sticky `100vh` wrapper with a child `flex` track translated by `gsap.to(track, { x: -(track.scrollWidth - innerWidth), scrollTrigger: { trigger: wrapper, scrub: 1, end: () => track.scrollWidth }})`. Type scale: H1 clamp(72px, 12vw, 220px) / line-height 0.92 / letter-spacing -0.04em; metadata 11px uppercase tracked +0.18em. Grid gutter 24px desktop, 12px mobile. Cards use `clip-path: inset(0 0 100% 0)` revealed to `inset(0 0 0 0)` over 0.9s `cubic-bezier(0.65, 0, 0.35, 1)`. Breakpoints: 1440 / 1024 / 768 / 390. Mobile: kill the horizontal rail, fall back to stacked cards with `loading="lazy"` videos.

WHY IT WORKS: Aggressive serif display type against monospaced micro-labels creates a typographic duet (max contrast = clear hierarchy). The scroll-scrubbed video gives the visitor a sense of authored control over time — motion semantics that say "every frame is intentional." Whitespace rhythm follows a 4/8/16/32px scale so the eye finds rest points between dense type clusters.

GOOD / BAD: GOOD — the scroll-scrubbed video is genuinely best-in-class, no fake loops. BAD — preloading the full hero video balloons LCP on slow connections; site has no `prefers-reduced-motion` fallback.

CLEAN APPLICATION: Apply this to Clean's landing-page hero by replacing the demo video with a scroll-scrubbed terminal recording of the MCP proxy in action — agent issues a request, proxy compresses 4KB of JSON down to 800 bytes of TOON, downstream tool returns. Pin the hero, scrub the terminal frames against scroll progress, and drop the H1 ("Context that fits.") as a SplitText character reveal at scroll progress 0. Use Instrument Serif for the display word, JetBrains Mono for the inline metric chip ("-78% tokens").

## IVRESS — Spin a Tale — https://www.awwwards.com/sites/ivress-spin-a-tale

WHAT IT DOES: Nine-chapter narrative experience rendered as a single fullscreen WebGL canvas. Each scene is a painterly composite (illustration baked into textures) that responds to cursor with a fluid simulation — dragging mixes color across the surface like watercolor on wet paper, and the cursor leaves ripple wakes that decay over ~1.2s. Chapter progression is wheel-driven, no scrollbar; chapter changes morph one scene into the next via a 1.5s shader cross-dissolve with displacement noise.

PACKAGES:
- three@0.170
- @react-three/fiber@9 + @react-three/drei@10
- ogl@1.x (lighter alt if not using R3F)
- shaderfrog or custom GLSL for the fluid sim (reference: PavelDoGreat/WebGL-Fluid-Simulation)
- gsap@3.13 for chapter timeline orchestration
- howler@2.2 for chapter ambient audio
- tailwindcss@4 for the thin UI chrome only
- zustand@5 for chapter state

IMPLEMENTATION: Single `<Canvas dpr={[1, 1.5]} gl={{ antialias: false, powerPreference: 'high-performance' }}>` mounted at root. Build a `<FluidLayer>` component that runs a velocity + dye double-buffered FBO ping-pong: vertex/fragment pair implements Navier-Stokes advection (resolution 256x256, dye res 1024x1024, splat radius 0.0035, dissipation 0.97 velocity / 0.99 dye, curl 30). Pointer events feed splats into the velocity FBO with `force = (current - last) * 6000`. Above the fluid layer, each chapter is a `<ChapterPlane>` (full-screen quad) that samples its baked painterly texture and adds the fluid dye as a multiply blend. Chapter transitions: `gsap.to(uniforms.uProgress, { value: 1, duration: 1.5, ease: 'power2.inOut' })` driving a displacement shader that uses a noise texture to dissolve scene A into scene B. Bind wheel via Lenis with `wheelMultiplier: 0.6`. Audio: each chapter has an `ambient.mp3` faded in/out with Howler `fade(0, 0.6, 1500)`. Reduced motion: bypass fluid sim, present static painterly stills with fade chapter transitions.

WHY IT WORKS: Direct manipulation of the visual substrate (you literally smear the paint) collapses the gap between visitor and content — that's interaction as authorship. Painterly textures break the sterile WebGL aesthetic that pervades 99% of award sites. Dark-on-dark with single warm light source forces eye to the protagonist (figure/ground via luminance contrast).

GOOD / BAD: GOOD — the fluid sim is a real fluid sim, not a fake shader noise pretending to be one. BAD — gates content behind a 9-chapter manual scroll for users who just want to know what IVRESS sells; mobile drops the sim entirely.

CLEAN APPLICATION: Apply this to Clean's product demo section as a "context flow" visualizer — instead of dye, render token streams as colored particles flowing from "agent" node into "MCP proxy" node, where they compress and emerge thinner on the other side. User drags the proxy node, streams reroute live. Don't go nine chapters; one interactive scene below the hero, fixed aspect ratio 16:9, with a "play demo" / "reset" toggle. Pull the painterly + ripple aesthetic only on light hover affordances on the proxy node itself.

## Fauna Robotics — https://faunarobotics.com/

WHAT IT DOES: Product-launch site for the Sprout humanoid platform. Above-the-fold is a single autoplaying Vimeo demo video (1440p, muted, loop) framed by minimal black serif "Capable, safe, fun." overlay. Below, a 3-card audience matrix (developers / enterprises / researchers) with another inline video, then a feature grid with looping micro-clips per capability and a UGC mosaic of 8 vertical Vimeo loops in a packed masonry. Built on Webflow with the site's identity carried entirely by motion (no decorative graphics — every visual is a real robot clip).

PACKAGES:
- webflow (production site) OR next@15 + react@19 for the Clean port
- tailwindcss@4
- @vimeo/player@2.x for adaptive bitrate inline players
- gsap@3.13 with ScrollTrigger for in-view reveals
- @studio-freight/lenis@1.x
- next/font with PP Right Grotesk or Söhne (this site uses a custom geometric sans for body, modern grotesk for display)
- @vercel/og for OG image generation

IMPLEMENTATION: Hero is one `<VimeoPlayer videoId={...} background autoplay muted loop responsive />` at 100vw 100vh with a `1px solid rgba(0,0,0,0.06)` rounded `border-radius: 20px` inset 16px from the edges (this is the signature framing device — a soft inset card on every video). Headline overlays bottom-left at `clamp(48px, 6vw, 96px)` weight 500 with `letter-spacing: -0.03em`. Trusted-by logo strip uses CSS `display: grid; grid-template-columns: repeat(5, 1fr); align-items: center;` with logos at `opacity: 0.7`. Capability grid: 2x2 desktop / 1-col mobile, each cell `aspect-ratio: 4/3`, plays its inline loop on `IntersectionObserver` entry, pauses on exit (battery + bandwidth saver). UGC mosaic: CSS Grid `grid-template-columns: repeat(4, 1fr); grid-auto-rows: 240px;` with selective `grid-row: span 2` to break uniformity, GSAP stagger reveals `y: 60, opacity: 0, stagger: 0.06, scrollTrigger: { trigger: el, start: 'top 80%' }`. Color tokens: bg `#FAFAF7`, fg `#0A0A0A`, accent `#0066FF`. No shadows anywhere — depth comes from the inset video cards alone.

WHY IT WORKS: For physical products, motion video is the only credible proof; this site refuses to fake it with renders or diagrams. The inset-card framing creates a quiet rhythm that prevents the heavy video content from feeling chaotic (Gestalt common region grouping). The off-white background and unsaturated palette let the robot itself be the only saturated thing on the page.

GOOD / BAD: GOOD — 100% real footage policy makes every claim verifiable. BAD — heavy Vimeo embed weight (multiple players initialized at once) hurts TTI on mid-tier mobile; no `<source media>` fallback for users blocking autoplay.

CLEAN APPLICATION: Apply this to Clean's landing-page below-the-fold capability grid by replacing robot clips with looping 720p screen recordings of MCP proxy behavior — one cell per capability (Compression, Routing, Caching, Observability). Use the exact inset-card framing (border-radius 20px, 16px inset, 1px soft border) for every video tile. Off-white `#FAFAF7` background, single accent for hover states. Inline videos must IntersectionObserver-gate to keep bundle responsive.

## Aceternity UI — Background Ripple Effect — https://ui.aceternity.com/components/background-ripple-effect

WHAT IT DOES: A SVG-precision grid of square cells (default ~32x32 grid of 56px cells) rendered as absolute-positioned divs in a CSS Grid. On pointer click/hover, the cell under cursor opacity-pulses, then cascading neighbors pulse outward with a calculated delay = chebyshev_distance(cell, origin) * 25ms. Result feels like physical impact propagation — a single tap sends a measurable wave across a static surface.

PACKAGES:
- react@19 + next@15
- tailwindcss@4
- motion@12 (the new Framer Motion package name)
- clsx@2.x
- shadcn CLI: `npx shadcn@latest add @aceternity/background-ripple-effect`

IMPLEMENTATION: Component is a `<div className="absolute inset-0 grid">` with `gridTemplateColumns: repeat(${cols}, 1fr)` and `gridTemplateRows: repeat(${rows}, 1fr)`. Each cell is a `<div data-r={r} data-c={c} className="border border-neutral-200/40 hover:bg-neutral-300/60" />`. On click, dispatch a custom event with `(originR, originC)`; each cell calculates `delay = Math.max(Math.abs(r - originR), Math.abs(c - originC)) * 25` and triggers a CSS animation by setting `style={{ '--delay': delay + 'ms', '--duration': '420ms' }}` and toggling `data-rippling="true"`. The animation itself is the keyframe block from Aceternity:
```css
@keyframes cell-ripple {
  0% { opacity: 0.4; }
  50% { opacity: 0.95; }
  100% { opacity: 0.4; }
}
[data-rippling="true"] { animation: cell-ripple var(--duration) ease-out var(--delay) 1 forwards; background: var(--accent); }
```
Use `pointer-events: none` on a mask gradient overlaid on the grid edges (`radial-gradient(circle at center, transparent 0%, var(--bg) 75%)`) so the effect fades into the page background. Mobile: reduce grid to 20x14 cells and only trigger on tap, never hover.

WHY IT WORKS: A regular grid is the visual definition of stillness; one click breaking that stillness into propagating motion is dopamine — change blindness defeated. The chebyshev radial delay matches the brain's expectation for how a shockwave travels (motion semantics that obey physics). Restraint in palette (single accent on a near-monochrome grid) ensures the motion reads even at glance.

GOOD / BAD: GOOD — pure CSS + minimal JS, runs at 60fps even on a 32x40 grid. BAD — grid borders on dense layouts can moiré on low-DPI displays; no built-in `prefers-reduced-motion` guard (component ships without it — must be added).

CLEAN APPLICATION: Apply this to Clean's hero background by rendering the ripple grid behind the headline and CTA, with the ripple origin auto-triggered every 6 seconds at a random cell to keep the surface alive without user interaction. Tint cell accent to Clean's signal-green `#00D17C` at 30% opacity. The grid replaces the typical gradient blob hero background; pairs especially well with monospace UI chrome.

## Codrops — Infinite Scroll with GSAP + Lenis — https://tympanus.net/codrops/2026/05/28/the-never-ending-story-building-a-seamless-infinite-scroll-experience-with-gsap-lenis/

WHAT IT DOES: Vertical infinite-loop gallery of editorial cards where scrolling never ends — the DOM contains N cards but virtual scroll position is modulo'd so the user can scroll forever in either direction. Cards parallax at differential speeds (foreground 1.0x, middle 0.7x, background 0.4x) creating depth. Snap-to-card behavior engages after the user stops scrolling for >150ms, easing the nearest card into perfect viewport center.

PACKAGES:
- gsap@3.13 with ScrollTrigger and Observer plugins
- @studio-freight/lenis@1.x (drives the smooth scroll)
- tailwindcss@4 (layout only)
- react@19 OR vanilla — works in both
- intersection-observer polyfill (only if supporting older browsers)

IMPLEMENTATION: Disable native scroll: `body { overflow: hidden; height: 100vh; }`. Initialize Lenis with `{ infinite: true, lerp: 0.1, wheelMultiplier: 0.9 }`. Lay out N cards in a vertical flex column, total height `H = N * cardHeight + (N-1) * gap`. On each Lenis tick, get `progress` and apply: `track.style.transform = translate3d(0, ${(-scroll % H + H) % H * -1}px, 0)`. Because the modulo gives values in `[0, H)`, duplicate the card array twice in the DOM (`[...cards, ...cards]`) so wrap is invisible. Parallax layers: each card has 3 inner div layers (`data-depth="1"`, `"0.7"`, `"0.4"`) whose translateY is recomputed per tick as `(cardCenterY - viewportCenterY) * (1 - depth) * 0.3`. Snap: use GSAP Observer `Observer.create({ target: window, type: 'wheel,touch', onStop: () => { const nearest = findNearestCard(); gsap.to(lenis, { scroll: nearest.absY, duration: 0.8, ease: 'expo.out' }) }, tolerance: 10 })` with `onStop` debounced 150ms. Performance: only render cards within `viewportHeight * 2` of the current scroll center via a virtualization slice.

WHY IT WORKS: Infinite content with snap creates the slot-machine effect — variable reward inside a controlled bounded interaction. Parallax depth defeats the flatness that kills most card layouts (depth = attention). The 150ms snap delay respects user agency: it doesn't yank them, it suggests where to land.

GOOD / BAD: GOOD — the wrap is invisible because of correct modular arithmetic and duplication. BAD — infinite scroll without a defined exit harms SEO and conversion; users with a clear goal can't tell when they've seen everything.

CLEAN APPLICATION: Apply this to Clean's docs navigation sidebar (NOT the marketing page) as a vertical infinite scroll of tool integrations — every MCP server Clean proxies for, looping through the catalog with snap-to-server. Keep parallax subtle (only 2 layers, max 0.85 differential). Marketing page should never use infinite scroll — Clean's hero/features/CTA flow is conversion-driven and needs a defined bottom.
