## Adcker — https://www.awwwards.com/sites/adcker

WHAT IT DOES: A single-route marketing site for an ad-creative agency that pins the hero canvas behind a vertical scroll narrative. The layout is a 12-column CSS grid with a sticky `position: sticky; top: 0; height: 100vh` viewport stage on the left half and scrolling editorial copy on the right; scroll progress drives a WebGL plane that morphs between five client work clips using a fragment shader cross-fade rather than a video swap. Cursor tracking displaces a noise-warped mask on the canvas (mouse delta → uniform `uMouse` lerped at 0.08 per frame).

PACKAGES:
- next@15
- react@19, react-dom@19
- three@0.170, @react-three/fiber@8.17, @react-three/drei@9.114
- gsap@3.12 with `ScrollTrigger`, `Observer`, `SplitText`
- lenis@1.1 (smooth scroll, replaces locomotive-scroll)
- tailwindcss@4
- next/font with `Neue Haas Grotesk Display Pro` (paid) or `Inter Tight` as drop-in
- lucide-react@0.460 (only used for the burger icon)

IMPLEMENTATION:
1. Wrap `app/layout.tsx` in a `<LenisProvider>` (custom hook): `new Lenis({ lerp: 0.1, smoothWheel: true, syncTouch: false })`, drive `requestAnimationFrame(lenis.raf)`, and pipe `lenis.on('scroll', ScrollTrigger.update)`.
2. Hero is a flex row, 50/50 split at `lg:` and stacked at `md:`. Left column: `<Canvas dpr={[1, 1.5]} gl={{ antialias: true, alpha: true }} camera={{ position: [0, 0, 1], fov: 35 }}>`. Right column: 5 stacked `<section>`s, each `min-h-[120vh]` with `data-clip="0..4"`.
3. ShaderPlane: full-screen `<mesh>` with `planeGeometry args={[2,2]}` and a custom `shaderMaterial`. Uniforms: `uProgress` (0→1), `uTextures[5]` (preloaded `useTexture` array), `uMouse` vec2, `uTime`. Fragment uses `texture2D(currentTex, vUv + cnoise(vUv*4.0 + uTime*0.2)*0.04)` blended toward `nextTex` by `smoothstep(uProgress*0.4, uProgress*0.4+0.2, vUv.y)` for a directional reveal.
4. Hook scroll to `uProgress`: `ScrollTrigger.create({ trigger: gridEl, start: 'top top', end: 'bottom bottom', onUpdate: self => material.uniforms.uProgress.value = self.progress * 4 })`. The `*4` maps total scroll to five segments.
5. Type animation: `SplitText` on every `<h2>` with `chars`, then `gsap.from(chars, { yPercent: 110, stagger: 0.018, duration: 0.9, ease: 'expo.out' })` triggered when each section enters at `start: 'top 70%'`.
6. Cursor warp: window mousemove → `gsap.to(material.uniforms.uMouse.value, { x: nx, y: ny, duration: 0.6, ease: 'power3.out' })` where `nx,ny` are normalized to [-1, 1].
7. Performance: clamp DPR via `useThree(state => state.gl.setPixelRatio(Math.min(window.devicePixelRatio, 1.75)))`, gate animation with `IntersectionObserver` on the canvas wrapper, lazy-decode hero textures using `KTX2Loader` from `@react-three/drei`.
8. Breakpoints: `sm:640 md:768 lg:1024 xl:1280`. Below `lg`, swap canvas for a single MP4 `<video autoPlay muted playsInline loop>` to save battery on mobile.
9. Accessibility: respect `prefers-reduced-motion` by setting `uProgress` directly to the section index (no GSAP tween) and disabling cursor warp.

WHY IT WORKS: The split — fixed visual stage vs scrolling text — creates a stable visual anchor while letting copy do the storytelling work. This is the cinematic "shot list" pattern: each scroll segment is a beat, and the shader cross-fade is the cut. Continuous cursor warp adds *liveness* (the canvas is never frozen) without taxing comprehension. Typography uses one weight and one type ramp, which lets the imagery carry the contrast load.

GOOD / BAD:
- GOOD: Sticky-canvas + shader cross-fade is GPU-cheap (one draw call) yet reads as bespoke film direction.
- BAD: Without the mobile video fallback, mid-tier Android phones drop to 12-15fps; many copycats skip this and ship a broken mobile experience.

CLEAN APPLICATION: Apply this to Clean's landing page hero (`tryclean.ai/`) by replacing the agency reel with a left-pinned terminal visualization that morphs between four states as the user scrolls: (1) raw 50k-token tool output dumped from a "before" MCP server, (2) the same payload streaming through Clean's proxy, (3) the TOON-compacted result at ~8k tokens, (4) the cleaned context delivered into Claude. Use the same shader cross-fade pattern, but replace the noise warp with a "scanline + decode" effect (vertical sweep that resolves blur to sharp text). Right column carries the four claims: "Your tools are noisy", "Clean catches every call", "TOON compresses by 6×", "Your agent stays sharp." This makes Clean's value prop *legible at scroll* — exactly what a developer-infra buyer needs in 8 seconds.

## Studio375 — https://www.awwwards.com/sites/studio375-2

WHAT IT DOES: An agency portfolio with a project index that uses an inverted-magnifier hover: list rows render flat, but mouse-over scales the row to 1.04, lifts the project thumbnail into a fixed ~480×320 preview pinned to the cursor with a 200ms `power3.out` follow, and triggers a CSS `mix-blend-mode: difference` color invert on the row text. The footer is a marquee of 28 logos animated with `transform: translate3d(-50%, 0, 0)` looped via GSAP's `xPercent` infinite tween (no JS measurement, pure CSS-friendly perf).

PACKAGES:
- astro@5 (ships zero JS by default — agency is leaning on this)
- @astrojs/react for the interactive island only
- gsap@3.12 with `Flip`, `Draggable`, no ScrollTrigger needed for the index page
- tailwindcss@4
- vite-plugin-glsl (small inline shader on the cursor blob)
- @vercel/og for OG image generation per project
- Suisse Int'l (Swiss Typefaces, paid) — fall back to `Inter` weights 400/500

IMPLEMENTATION:
1. Project list: server-rendered `<ul>` of `<li role="link">` rows, each `flex items-center justify-between py-6 border-b border-white/10`. No client JS for layout.
2. Floating preview: a single `<div class="fixed pointer-events-none w-[480px] h-[320px]">` mounted once at the page root. Hidden by default with `opacity-0 scale-90`.
3. On `mouseenter` of any row: `gsap.to(preview, { opacity: 1, scale: 1, duration: 0.4, ease: 'power3.out' })`, set `preview.querySelector('img').src = row.dataset.thumb`. On `mouseleave`: reverse.
4. On `mousemove` of the row container: `gsap.to(preview, { x: e.clientX + 24, y: e.clientY - 160, duration: 0.5, ease: 'power3.out', overwrite: 'auto' })`.
5. Row text uses `mix-blend-mode: difference` and color `white`, with the preview thumbnail z-indexed below the text but above a white flood that fills the row on hover (`gsap.to(rowBg, { scaleX: 1, transformOrigin: 'left', duration: 0.5, ease: 'expo.out' })`).
6. Marquee footer: two identical `<ul>` inside a `flex` track. `gsap.to(track, { xPercent: -50, duration: 40, ease: 'none', repeat: -1 })`. `will-change: transform` on the track.
7. Page transitions: `Astro.View Transitions` API (`<ClientRouter />`) with named `transition:name="hero"` on each project card → detail page.
8. Custom cursor blob: `radial-gradient(circle, rgba(255,255,255,0.3), transparent 60%)` 60×60 div, position locked to mouse via `gsap.quickTo`, scaled to 2× on hover of any `[data-cursor="grow"]`.
9. Performance: total JS budget ≈ 22kb gzipped (gsap-core + Flip plugin). Lighthouse perf 99 because Astro emits no React runtime for the static pages.

WHY IT WORKS: It treats the *list* as the hero unit instead of a separate hero block. Hover state collapses three signals into one motion — magnification (importance), thumbnail reveal (preview), color invert (state change) — which is Gestalt common-fate at maximum density. The viewer's focus stays on a single row at all times because the rest goes flat-white. This is the textbook "one thing per moment" pattern.

GOOD / BAD:
- GOOD: Astro-first + one GSAP island = ~22kb JS for an interactive agency portfolio. Best-in-class perf budget.
- BAD: `mix-blend-mode: difference` is unpredictable on dark images; relies on every thumbnail being mid-tone. Breaks if a thumbnail goes pure black or pure white.

CLEAN APPLICATION: Apply this to Clean's docs navigation index (`tryclean.ai/docs`) by rendering the docs sidebar as a Studio375-style list: hovering "Quickstart", "MCP Servers", "TOON Encoding", etc., spawns a 360×240 floating preview at the cursor showing a syntax-highlighted snippet from that doc. Replace the agency thumbnails with `react-syntax-highlighter` previews of the page's first code block. This collapses "what is this section" and "what does the code look like" into a single hover — engineers decide whether to click in 200ms instead of 2 seconds.

## Shader Development Studio — https://www.shader.se/

WHAT IT DOES: A studio site whose entire scrolling experience is a single full-bleed WebGL canvas. The "page" is rendered as 3D scenes that the camera dollies through on scroll — there are no DOM scroll sections, only a tall invisible scroll proxy element. Heading text is rendered inside the canvas using MSDF (multi-channel signed distance field) text via `troika-three-text`, allowing crisp type at any zoom level and per-glyph displacement on a noise field.

PACKAGES:
- next@15 (App Router)
- three@0.170, @react-three/fiber@8.17, @react-three/drei@9.114
- @react-three/postprocessing@2.16 (Bloom, ChromaticAberration, Noise)
- troika-three-text@0.50 (canvas text)
- lenis@1.1 (drives `state.clock` and camera Z)
- tailwindcss@4 (only used for the small DOM nav overlay)
- howler@2.2 (subtle audio reactive layer, opt-in mute)

IMPLEMENTATION:
1. Page is `<div style="height: 600vh">` with a fixed `<Canvas>` underneath. Lenis provides a `scroll` value that maps to `state.camera.position.z = -scroll * 0.01`.
2. Five 3D scene "rooms" placed along the Z axis at z = 0, -50, -100, -150, -200. Each room is a `<group>` containing geometry, MSDF text headings, and a unique custom shader material.
3. Postprocessing chain (declarative, via `<EffectComposer>`): `<Bloom intensity={0.6} luminanceThreshold={0.85} mipmapBlur />`, `<ChromaticAberration offset={[0.0008, 0.0012]} />`, `<Noise opacity={0.04} premultiply />`.
4. MSDF text heading (e.g., "We Build Shaders"): `<Text3D font="/fonts/inter-msdf.json" size={2.4} curveSegments={12}>` with a custom `onBeforeCompile` that injects a vertex displacement: `transformed.z += sin(transformed.x*2.0 + uTime*1.3)*0.08;`.
5. Camera: `<PerspectiveCamera makeDefault fov={50} />` with subtle parallax — mouse position drives `camera.rotation.y = THREE.MathUtils.lerp(camera.rotation.y, mouseX*0.05, 0.05)` per frame.
6. Use `useFrame((state, delta) => material.uniforms.uTime.value += delta)` rather than reading absolute time to keep it tab-pause-safe.
7. Overlay nav: tiny fixed `<header>` with `position: fixed; top: 0; mix-blend-mode: difference; color: white; padding: 16px 24px; backdrop-filter: blur(10px) saturate(180%);`.
8. Audio: ambient drone loop, 80% gain, with a `<button aria-label="Toggle sound">` in the corner. Audio analyser feeds `material.uniforms.uAudio.value` (0..1) so visuals pulse to bass.
9. Performance: aggressive frustum culling per room (`<group visible={isInRange}>`), compile shaders ahead with `gl.compile(scene, camera)` in a `useEffect`. Cap pixel ratio at 1.75. Use `react-device-detect` to ship a static image fallback for low-end mobile.

WHY IT WORKS: It collapses the boundary between "content" and "stage" — the headline IS the geometry. Camera-on-rails scrolling resolves the classic "scroll-driven WebGL is jittery" problem by giving the scene a deterministic timeline. Postprocessing (bloom + grain + chroma) ties disparate scenes into a unified visual signature, the way a film LUT does for cinematography.

GOOD / BAD:
- GOOD: Genuinely best-in-class technical execution — MSDF text is the right call (no DOM/canvas overlap bugs), and audio-reactive uniform is a thoughtful detail.
- BAD: Fully WebGL site is a SEO and accessibility wasteland unless you ship a parallel SSR'd HTML version (they don't, fully). Crawlers see an empty body.

CLEAN APPLICATION: Apply this to Clean's product demo section on the landing page (`tryclean.ai/#how-it-works`). Build a single `<Canvas>` "context journey" room: a 3D wireframe globe of 30 floating MCP server nodes, each labeled with MSDF text ("github-mcp", "linear-mcp", "slack-mcp"). On scroll, camera flies through the cloud and a translucent "pipe" — Clean's proxy — assembles around the camera, with raw token-strings (rendered in MSDF) flowing in from servers, getting visually compressed (geometry instances scale from 1.0 to 0.18 over 800ms with `power3.inOut`), and exiting toward a Claude logo. Skip the audio. Ship an HTML twin under `<noscript>` for SEO. This is what "context layer for agents" looks like as motion design.

## NaughtyDuk© — https://www.awwwards.com/sites/naughtyduk-c-1

WHAT IT DOES: A design studio site whose hero is a 3D rubber-duck mascot rigged with physics — the duck wobbles in a viscous gel that responds to drag. Drag-release uses `@react-three/rapier` with a damped spring constraint so the duck always returns to center. The page transition between case studies is a "duck-dive" where the camera plunges below the floor (a planar reflection) and resurfaces in the next scene; a custom GLSL caustics shader handles the underwater shimmer.

PACKAGES:
- next@15
- three@0.170, @react-three/fiber@8.17, @react-three/drei@9.114
- @react-three/rapier@2.1 (physics)
- gsap@3.12 with `ScrollTrigger` and `CustomEase`
- @studio-freight/lenis@1.1
- maath@0.10 (math utilities for damping)
- tailwindcss@4
- Self-hosted variable font: `PP Editorial New` (Pangram Pangram, paid)

IMPLEMENTATION:
1. Physics scene: `<Physics gravity={[0, -2.5, 0]} timeStep="vary">` (light gravity for floaty feel). Duck = `<RigidBody type="dynamic" colliders="hull" linearDamping={1.8} angularDamping={2.2}>` wrapping a `<gltf>` of the duck mesh.
2. Spring tether: every frame, `useBeforePhysicsStep(() => duckRef.current.applyImpulse({ x: -position.x*4, y: 0, z: -position.z*4 }))` keeps it snapping back to origin.
3. Drag handling: `<Drag>` from `@use-gesture/react` reads pointer drag → applies `applyImpulse({ x: dx*40, y: dy*40, z: 0 })` on release.
4. Page transitions: route change → fire `gsap.timeline({ defaults: { ease: 'power3.inOut' } })`, animate camera Y from 2 to -8 over 0.7s, set `uCausticsIntensity` from 0 to 1, swap scene in the dip, then return camera to 2. Use Next.js `router.events` (App Router: `useTransitionRouter` from `next-view-transitions`).
5. Caustics shader (full-screen quad below the camera): `vec3 caustics = vec3(0.5 + 0.5*sin(vUv.x*40.0 + uTime*2.0)*sin(vUv.y*40.0 + uTime*1.7));` mixed at 30% opacity with the underlying scene.
6. Reflective floor: `<MeshReflectorMaterial blur={[400, 100]} resolution={1024} mixBlur={1} mixStrength={50} roughness={1} mirror={0.5}/>`.
7. Sound design: every drag-release triggers a different `splat` sample (8 variants, randomly picked) via Howler with pitch-shift `rate: 0.85 + Math.random()*0.3`.
8. Performance: rapier physics is wasm — preload it with `<Physics>...` mounted inside `<Suspense>` and a 2D placeholder for first paint. Disable physics entirely below 60fps detection (use `stats.js` in dev, ship a 60fps gate in prod).
9. Reduced motion: snap duck to center, disable caustics, transition becomes a 200ms fade.

WHY IT WORKS: The physics duck is *inviting* — visitors WILL drag it because the affordance is obvious (cursor changes to grab) and the response is satisfying (overdamped spring + sound). This is Csikszentmihalyi's flow-loop applied to a hero: action → immediate feedback → curiosity → next action. The "duck-dive" page transition reframes navigation as an environmental beat rather than a content swap.

GOOD / BAD:
- GOOD: Audio-on-interaction is rare and well-executed — pitch-shifted samples never feel repetitive across 30 drags.
- BAD: Total bundle is ~480kb gzipped (rapier wasm + glb) before LCP even paints; first-time visitors on 4G see the blank state for 4-6s.

CLEAN APPLICATION: Apply this to Clean's onboarding flow (`tryclean.ai/onboarding/connect`). Use rapier physics for the "drop your MCP server here" target: render each detected MCP server as a draggable physics body in a soft-walled tray; users drag servers into a "Connected" zone where they snap with a satisfying tactile thud (Howler sample + spring constraint). Replace the duck with simple labeled cards. Skip caustics — Clean is an infra product, not a studio. The point is to make MCP wiring feel *physical and intentional* rather than "click checkbox" — turning setup into the kind of moment users brag about on Twitter.

## Oryzo AI by Lusion — https://oryzo.ai/

WHAT IT DOES: A satirical product page for a fictional cork coaster, executed as a cinematic scroll sequence. The product (a single CG cork disc) is rendered in WebGL and pinned center-screen for the full page; scroll progress drives a single GSAP timeline that orchestrates camera dolly, product rotation, environment lighting changes, and choreographed text reveals. Section transitions use a "morph wipe" — geometry vertices animate between two product configurations in a single GLSL `mix(positionA, positionB, smoothstep(...))`.

PACKAGES:
- next@15
- three@0.170, @react-three/fiber@8.17, @react-three/drei@9.114
- gsap@3.12 with `ScrollTrigger` (Lusion's signature)
- @studio-freight/lenis@1.1
- @react-three/postprocessing@2.16 (DepthOfField, Bloom)
- tailwindcss@4
- Sharp custom display face (commissioned). Free substitute: `Migra` from Pangram Pangram or `Tobias` from Klim.
- howler@2.2 for ambient pad + impact stingers

IMPLEMENTATION:
1. Single master timeline: `gsap.timeline({ scrollTrigger: { trigger: '#stage', start: 'top top', end: '+=600%', scrub: 0.6, pin: true } })`. Scrub of 0.6 = 0.6s "rubber band" between scroll position and animation, the secret sauce that makes Lusion sites feel buttery.
2. Stage = full-viewport `<Canvas>` plus an absolutely-positioned `<div>` layer with stacked text panels (each `position: absolute; inset: 0; pointer-events: none`).
3. Add labeled keyframes to the timeline: `tl.to(camera.position, { x: 0, y: 0, z: 3, duration: 1 }, 'shot1').to('.headline-1', { yPercent: -100, opacity: 0, duration: 0.4 }, 'shot1+=0.6')`, etc. Use ~14 named labels for a clean sequence.
4. Product mesh: imported `coaster.glb`, custom `MeshStandardMaterial` with anisotropy 8, roughness 0.85, plus a procedural cork normal map (FBM noise in fragment).
5. Morph wipe: two coaster geometries (smooth vs etched), a custom shader uniform `uMorph` (0..1), vertex shader does `vec3 pos = mix(positionA, positionB, smoothstep(0.0, 1.0, uMorph));`. Timeline tweens uMorph at the right scroll beat.
6. Environment: `<Environment preset="studio" />` swapped to `<Environment files="/hdr/sunset.hdr" />` mid-timeline via crossfade by interpolating two `<Environment>` `intensity` props.
7. Postprocessing DOF: `<DepthOfField focusDistance={0.018} focalLength={0.05} bokehScale={4} />` — the soft macro-lens look that sells the "premium product" lie.
8. Audio: `Howler` ambient drone always on (-22 LUFS), with five `Howl` impact stingers triggered at GSAP timeline labels via `tl.call(() => stinger.play(), [], 'shot4')`.
9. Lenis paired with `ScrollTrigger.scrollerProxy` + `ScrollTrigger.update` on `lenis.on('scroll', ...)`.
10. Performance: load the GLB via DRACO (`useGLTF.preload('/coaster.glb')`), cap DPR at 1.75, gate effects with `<EffectComposer enabled={!isMobile}>`.

WHY IT WORKS: Pinning the product and orchestrating one continuous timeline turns the page into a product film, not a webpage. The visual rhythm is musical — every shot lasts roughly the same scroll distance, creating a metronome the body learns within two beats. The DOF + HDR lighting + cork normal map together fool the eye into believing this is a $1000 product even though it's $0 (it doesn't exist). This is Pixar-grade narrative pacing applied to scroll.

GOOD / BAD:
- GOOD: Single-timeline architecture is a maintenance dream — the entire site is one declarative animation script with named labels.
- BAD: It's *the* Lusion formula — at this point hundreds of agency sites copy this exact pattern. Originality has decayed; what feels premium today reads as "Lusion clone" by Q4.

CLEAN APPLICATION: Apply this to Clean's pricing page (`tryclean.ai/pricing`) with a twist — pin a single "context budget meter" 3D object (a glass tube filling with token-cubes) center-stage and scroll-orchestrate it across the three tiers. Hobby (10k tokens/mo) → tube fills 1/10. Pro (1M tokens/mo) → tube fills with falling cubes that compress and pack tighter. Team (unlimited) → tube becomes a ring of recirculating cubes. Each shot pairs with 1-2 lines of pricing copy and the CTA. This makes a normally-boring pricing page feel like an architectural product reveal — and it visually telegraphs Clean's core value (token compression) without saying "we save you tokens."

## BUILD DIRECTIVE

TARGET: Clean landing page hero (`tryclean.ai/` — above the fold + first scroll).

APPROACH: Build the Adcker pinned-canvas + scrolling-narrative pattern, with Shader Studio's MSDF in-canvas text for the "before/after" data labels and Oryzo's master-timeline GSAP scrub for buttery scroll. Left half is a sticky Three.js canvas rendering a 3D "context pipe" — raw MCP server output enters from the top as floating glyph clouds (~5,000 instanced characters), passes through Clean's proxy (a glowing torus), and exits as compact TOON-encoded blocks at the bottom. Right half is four scroll beats of editorial copy: (1) "Your agent is drowning in tool noise", (2) "Clean intercepts every MCP call", (3) "TOON encoding compresses 6×", (4) "Your context stays sharp". Total scroll distance: 400vh. Cursor warps the glyph cloud (subtle 0.04 displacement). Mobile fallback: replace canvas with a 6s looping MP4 + the same scrolling copy.

STACK:
- next@15, react@19
- three@0.170, @react-three/fiber@8.17, @react-three/drei@9.114
- @react-three/postprocessing@2.16 (Bloom only — keep it restrained)
- troika-three-text@0.50 (MSDF labels in canvas)
- gsap@3.12 with `ScrollTrigger`, `SplitText`
- lenis@1.1
- tailwindcss@4
- next/font with `Inter Tight` (variable, 400/500/600) + `JetBrains Mono` (400/500) for code-feel labels
- lucide-react@0.460 (only nav icons)

REFERENCE URLS:
1. https://www.awwwards.com/sites/adcker (pinned-canvas + scroll narrative architecture — the spine of the build)
2. https://www.shader.se/ (MSDF in-canvas text + master postprocessing pipeline — borrow the visual signature, not the full-WebGL trap)
3. https://oryzo.ai/ (master-timeline scrub value of `0.6` and the named-label authoring pattern)

KEY CONSTRAINTS (Clean is developer infrastructure — sophisticated and fast, never flashy or generic):
- AVOID: Full-WebGL DOM-replacement (Shader Studio's mistake — kills SEO and a11y). Always ship semantic HTML for crawlers and screen readers. Canvas is decoration, not content.
- AVOID: Cinematic product-reveal aesthetics (Oryzo). Clean sells to engineers, not consumers. No DOF, no bloom-blown highlights, no soundtrack. Bloom intensity capped at 0.4.
- AVOID: Skeuomorphic "premium" textures (cork, leather, glass). Clean's visual language is monochrome + one accent color (Clean's existing brand teal/lime), monospaced labels, and grid lines. Think Linear/Vercel, not agency-portfolio.
- AVOID: Page-load > 1.5s LCP. Hard budget: <140kb JS gzipped at first paint, lazy-load R3F scene under `<Suspense>` with a static SVG of the pipe as the placeholder. Total <2.5s on a throttled 4G connection or it doesn't ship.
- AVOID: Cursor cursors / custom cursor blobs / "magnetic" buttons. Engineers find them gimmicky. Keep system cursor.
- AVOID: Marketing copy fluff. Every right-column line must be an empirical claim a developer can verify (compression ratio, latency overhead, supported MCP count). Numbers, not adjectives.
- REQUIRE: `prefers-reduced-motion` snaps to final state with no animation. `<noscript>` shows the four-claim narrative as plain typography.
- REQUIRE: All four scroll beats' text content must be present in initial HTML for crawlers (not injected by JS).
- REQUIRE: Real performance gate — measure on a Pixel 7 emulated at 4× CPU throttling. If frame budget exceeds 11ms during scroll, drop the glyph instance count from 5,000 to 1,500 and ship.
