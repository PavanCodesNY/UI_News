## 1. XOX by makemepulse — https://xox.makemepulse.com/

WHAT IT DOES: A full-viewport 2D WebGL tic-tac-toe game where each match procedurally reveals one of makemepulse's team members through bespoke 2D character animations. The interaction loop is: choose opponent → animated character intro → grid renders as instanced WebGL quads → click cell triggers a sprite-driven X/O stamp with squash/stretch motion → AI move responds within ~250 ms → win/draw triggers fullscreen celebration shader + name plate. Layout is a single fixed-aspect canvas centered in viewport with a thin top bar (score, restart) and a bottom bar (current opponent card). No scroll. All UI text is HTML overlay positioned via CSS Grid, not painted into the canvas.

PACKAGES:
- `next` (14.x) or vanilla `vite`
- `ogl` (lightweight WebGL renderer; ~25 KB) OR `three` (0.165+) with `OrthographicCamera`
- `gsap` + `gsap/CustomEase` + `gsap/SplitText`
- `tailwindcss` (utility shell for the HTML overlay)
- `zustand` (game state: board, turn, opponent, score)
- `@vercel/og` or static webp sprites for character portraits
- `nipplejs` is NOT needed; touch handled via pointerdown
- Font: `Söhne Mono` (Klim) or `JetBrains Mono` fallback for the UI chrome; `PP Editorial New` for opponent name plates

IMPLEMENTATION:
1. Component tree: `<App>` → `<CanvasStage>` (full-bleed `<canvas>` with `position: fixed; inset: 0; z-index: 0`), `<HUD>` (overlay with `z-index: 10`, CSS Grid: 64px top row, 1fr middle, 96px bottom row), `<ResultModal>` (portal, `z-index: 30`).
2. Init OGL renderer with `dpr = Math.min(window.devicePixelRatio, 2)`, `antialias: true`, transparent background. Use one `Mesh` per cell (9 instances of a unit quad) with a shared shader.
3. Shader: vertex passes UVs; fragment samples an SDF atlas of X and O glyphs and outputs alpha. Cells take uniforms `u_state` (0 empty, 1 X, 2 O), `u_stampProgress` (0→1), `u_hoverIntensity` (0→1).
4. GSAP timelines: stamp animation = `gsap.to(cell.uniforms.u_stampProgress, { value: 1, duration: 0.55, ease: CustomEase.create("xox", "M0,0 C0.2,0 0.1,1 1,1") })` then a settle squish via a second uniform. Hover idle: `gsap.to(cell.uniforms.u_hoverIntensity, { value: 1, duration: 0.18, ease: "power2.out" })`, reverse on mouseleave.
5. Opponent intro: SplitText the name into characters, `stagger: 0.025`, `y: 24, opacity: 0 → 0, 1`, `ease: "expo.out"`, duration 0.8. Portrait card slides from left with `clip-path: inset(0 100% 0 0) → inset(0 0 0 0)`, 0.9 s, `power3.inOut`.
6. AI move: setTimeout 220–320 ms (jittered) before triggering stamp animation, makes the bot feel "thinking" without being slow.
7. Win detection paints all 3 winning cells with an additive blend pulse (`u_winGlow` 0→1→0 in 0.6 s) and triggers a fullscreen shader ribbon sweep (1.2 s `power4.inOut`).
8. Responsive: canvas square sized to `min(90vh, 90vw, 720px)`. Below 480 px, swap top bar for stacked layout (score above title).
9. Performance: cap to 60 fps via rAF; pause render loop when tab hidden (`document.visibilityState`); pre-decode all character portraits via `<link rel="preload" as="image">` in `<head>`.
10. Accessibility: full keyboard control with arrow keys + Enter; cells get `role="gridcell"` and `aria-label="Row 2 Column 3, empty"`. Announce winners via `aria-live="polite"` region.

WHY IT WORKS: It collapses three principles into one surface: progressive disclosure (you do not see the team roster as a list, you discover it through play), state-driven motion semantics (every animation is a function of game state, so motion always carries meaning), and reduction (one mechanic, one layout, infinite charm). The choice to render in 2D WebGL instead of HTML buys frame-perfect timing on stamp animations and lets the shader unify the visual language across cells, hover, and victory states.

GOOD / BAD: Best-in-class because the mechanic is itself the marketing — every play reveals a hire. Bad: no SSR fallback, the entire experience is gated behind WebGL2 (Safari users on old hardware get a blank canvas); also brutal on battery on mobile.

CLEAN APPLICATION: Apply this to Clean's landing-page hero by replacing the standard "video of the product" with a **live agent coordination puzzle**. Render a 3×3 grid of cells, each representing an agent (Cursor, Claude Code, Windsurf, Codex, etc.). Hovering a cell shows what context that agent currently has; clicking "Sync" runs a 1.2 s GSAP timeline that paints all cells with a shared color gradient and outputs "All agents now share context." The mechanic embodies the product. Use the same OGL + GSAP CustomEase + SDF-glyph shader pattern. Place HUD overlay with Clean's wordmark top-left and a "Try in your terminal" CTA bottom-right.

## 2. Studio Namma — https://www.studio-namma.com/

WHAT IT DOES: A vertically scrolled design-agency site organized as a sequence of full-viewport "chapters," each one a different motion vocabulary (clip-path wipe reveal, masked video portal, horizontal infinite marquee, FLIP-based grid morph). Navigation is a custom menu that opens via a 360 ms ease-out clip-path circle from the top-right and dims background to 12% opacity behind a backdrop-filter blur of 24 px. Cursor is replaced with a 12 px disk that grows to 64 px and inverts color on interactive elements via mix-blend-mode: difference.

PACKAGES:
- `webflow` host (or `next` + `tailwindcss` if rebuilding from scratch)
- `gsap` + `gsap/ScrollTrigger` + `gsap/Flip` + `gsap/CustomEase`
- `lenis` (smooth scroll, 0.1 s damping, `lerp: 0.08`)
- `splittype` for line/word/char split
- `vimeo-player` for autoplay project videos (poster-first, swap on intersection)
- Fonts: `Söhne` (body), `GT Sectra` (display)
- No 3D library required

IMPLEMENTATION:
1. Wrap entire body in a Lenis instance with `smoothWheel: true, smoothTouch: false, lerp: 0.08`. Hook into GSAP ticker so ScrollTrigger updates against Lenis scroll.
2. Hero chapter: title is one SVG `<text>` element, drawn with `stroke-dasharray` reveal driven by `gsap.fromTo(path, { strokeDashoffset: length }, { strokeDashoffset: 0, duration: 1.6, ease: "power3.out" })` on mount.
3. Chapter transitions: each section has `data-bg="#0E0E0E"`. On ScrollTrigger enter, animate `body { background-color }` via `gsap.to(document.body, { backgroundColor, duration: 0.6, ease: "power2.inOut" })`. Text colors toggle via a CSS variable `--ink` swapped at the same time.
4. Project showcase grid: 12-column CSS grid, gap 24 px, cards are 6 cols. Hover triggers a Flip animation: card lifts to z-index 5, scales up via transform `scale(1.04)`, inner image scales counter `1/1.04` so the frame grows but the image stays still. Duration 380 ms, `expo.out`.
5. Marquee strip: two duplicated rows of project names in 280 px text, animated via `gsap.to(strip, { xPercent: -50, duration: 30, ease: "none", repeat: -1 })`. Speed multiplied by scroll velocity reported from Lenis (`ScrollTrigger.velocity` mapped to a 1x–3x factor with `gsap.quickTo`).
6. Custom cursor: two divs (dot + ring). `mousemove` writes `--cx`, `--cy` to root; CSS `transform: translate3d(var(--cx), var(--cy), 0)`. Ring lags via `gsap.quickTo(ring, "x", { duration: 0.4, ease: "power3" })`. On interactive hover, ring scales to 5x and applies `mix-blend-mode: difference`.
7. Page transitions: clicking a project triggers a "curtain" overlay split into 12 vertical strips, each animated `scaleY: 0 → 1` with stagger 0.04, duration 0.7, `power4.inOut`. Next page mounts behind, curtain reverses.
8. Responsive: below 1024 px, drop the custom cursor entirely (it harms touch). Marquee text drops to 160 px. Hero stroke reveal still works but duration shortens to 1.1 s.

WHY IT WORKS: It uses each chapter as a distinct motion sentence — no two sections share a transition vocabulary, but they all share the same easing curves and palette, which creates rhythm without monotony. The cursor inversion via mix-blend-mode is a borrowed technique that signals "interactive" without color cues, which keeps the palette pure (two colors total). Gestalt: every chapter is a closed figure, no bleeding section ends.

GOOD / BAD: Best-in-class for motion choreography and chapter pacing. Bad: Lenis smooth scroll fights iOS momentum and can feel sluggish on trackpads with high inertia; the custom cursor breaks copy-paste muscle memory.

CLEAN APPLICATION: Apply this to Clean's docs navigation surface by structuring the docs into "chapters" — Concepts, Quickstart, Adapters, API, Cookbook — each with its own background color swap on scroll and its own motion vocabulary (concepts = SVG diagram reveals, quickstart = terminal typewriter, adapters = logo wall marquee, API = curtain page transitions). Keep the cursor inversion technique on the side-nav only (where contrast against changing backgrounds matters). Lenis + ScrollTrigger powers the entire pipeline.

## 3. Fourmula AI by Fourmeta Agency — https://fourmula.ai/

WHAT IT DOES: An AI-product (image/video generation) marketing site organized as a single-page scroll with a fixed asymmetric color block layout: a tall left rail in dark ink (#020108) holds the wordmark and a sticky CTA, the right 70% of viewport scrolls through full-bleed product demo loops. Brand uses exactly two colors — near-black #020108 and a single accent orange #FF6B02. Type is oversized geometric sans for hero (~12 vw) and small-caps mono for labels. Every demo video is preceded by a 600 ms orange "shutter" wipe and ends with a poster snap.

PACKAGES:
- `webflow` (production host) or `next` 14 + `tailwindcss`
- `gsap` + `gsap/ScrollTrigger` + `gsap/SplitText`
- `lenis`
- `hls.js` if streaming video, otherwise `<video muted autoplay playsInline preload="metadata">`
- Fonts: `PP Neue Montreal` (display), `JetBrains Mono` (small caps labels)
- Icon set: custom 1.5 px stroke SVGs, not lucide

IMPLEMENTATION:
1. Root layout: `display: grid; grid-template-columns: 280px 1fr;` on desktop. Left column is `position: sticky; top: 0; height: 100vh;` with dark fill. Right column is the scrolling content. Below 1024 px, collapse to single column with the sticky bar moving to top (height 64 px).
2. Color tokens (Tailwind config):
   - `ink: #020108`
   - `flame: #FF6B02`
   - `bone: #F4F1EC`
   - No grays. All shadows use `flame` at 12% alpha.
3. Hero: SplitText the headline by word, animate `yPercent: 100 → 0, opacity: 0 → 1, stagger: 0.06, duration: 0.9, ease: "expo.out"`. Background is `bone`. Headline switches color to `flame` on the verb only ("Generate", "Style", "Shoot") via a `<span>` wrapper.
4. Demo reels: each section pins for 200 vh of scroll via ScrollTrigger `pin: true, scrub: 0.5`. The video element's `currentTime` is scrubbed from a GSAP timeline tween: `gsap.to(video, { currentTime: video.duration, ease: "none", scrollTrigger: { trigger: section, start: "top top", end: "bottom top", scrub: 0.4 } })`. This means scroll drives playback, no autoplay.
5. Shutter wipes between sections: a single full-width `flame` div with `transform-origin: top` scales `scaleY: 0 → 1 → 0`, two-step timeline `power4.inOut` 0.6 s each, content underneath swaps at mid-point.
6. Sticky CTA in left rail: `<button>` with `bg-flame text-ink`, 56 px tall, rounded 4 px, copy "Generate a shot". Hover triggers a `translateY: -2px` and a 4 px shadow with flame glow.
7. Image-gen samples grid: 4-column masonry on desktop, intersection-observer fade in with `opacity: 0 → 1, y: 16 → 0`, stagger 0.04 by row.
8. Performance: every video uses `preload="metadata"` and is replaced with a static webp poster until ScrollTrigger reports it has entered. Use `loading="lazy"` on all imagery below the fold.
9. Accessibility: respect `prefers-reduced-motion` by setting all scrub durations to 0 and replacing the shutter wipes with instant fades.

WHY IT WORKS: Two-color discipline forces the eye to use shape and motion to find hierarchy, which is the right move when the product itself is visual content (you do not want the chrome competing with the demos). The scroll-scrubbed video is a strong proof-of-product mechanic: instead of asking the visitor to press play, scroll IS the play button, and every scroll inch is product. The sticky CTA in the left rail means the conversion ask is always one pixel away.

GOOD / BAD: Best-in-class because the layout itself is a brand asset — the left dark rail with sticky CTA is now recognizable as Fourmula's signature. Bad: scroll-scrubbed video is harsh on low-end Android (frame drops, decoder thrash) and the two-color palette has near-zero accessibility margin if the flame is used on the bone background for small text (only 3.4:1 contrast).

CLEAN APPLICATION: Apply this directly to Clean's landing-page hero and demo section. Clean's product is also visual (terminal output, context graphs, agent activity), so this layout pattern fits perfectly. Use Clean's existing palette (ink + a single accent — pick one) with the same 280 px sticky left rail holding wordmark, version badge, and a single "Install in your terminal" CTA. The scroll-scrubbed video becomes a screen recording of Clean's proxy intercepting a multi-agent task and unifying context: scroll position drives playback, every inch of scroll reveals a moment of the demo. Do not use grays. Do not use more than 2 colors.

## 4. Aceternity UI: Text Flipping Board — https://ui.aceternity.com/components/text-flipping-board

WHAT IT DOES: A split-flap (Vestaboard-style) display React component that scrambles through a randomized alphabet character set, flap by flap, until each tile lands on the target glyph. Each "flap" is a 3D-rotated `<div>` driven by Framer Motion, with the rotation interpolating from `rotateX: 0` to `rotateX: -180` while the top and bottom halves cross-fade to reveal the next character. Component takes a string and renders one tile per character; tiles flip independently with staggered start times.

PACKAGES:
- `next` 14 + `react` 18
- `framer-motion` (11.x) or `motion` (post-rename)
- `tailwindcss`
- `clsx` for class merging
- Optional: `react-use-measure` if you need responsive tile sizing
- Fonts: `JetBrains Mono`, `IBM Plex Mono`, or `Berkeley Mono`. Must be monospace.

IMPLEMENTATION:
1. Component signature: `<FlipBoard text="HELLO WORLD" charSet="A-Z0-9 ." duration={1.4} stagger={0.08} />`.
2. Internal state: an array `tiles[]`, each element `{ target: 'H', current: 'A', flipping: true }`. On mount or text-prop change, kick off a per-tile loop.
3. Per-tile loop: for `cycles = 14` iterations, every `duration / 14` ms, set `current` to a random char from charSet. On final cycle, set `current = target`. Use `useEffect` with `setInterval` and clear on cleanup.
4. Tile DOM: a relatively positioned `div` (40 px × 56 px), with two absolutely positioned halves. Top half clips `inset(0 0 50% 0)`, bottom half clips `inset(50% 0 0 0)`. Each half shows the character. On flip, animate the top half: `rotateX: 0 → -90` (Framer Motion `animate`), duration 0.18, ease `[0.4, 0, 0.2, 1]`. At -90, swap the displayed glyph. Then animate -90 → -180 over the same duration.
5. Stagger: tile `i` starts at `i * stagger * 1000` ms via `setTimeout`.
6. Styling tokens:
   - Tile bg: `#0A0A0A`
   - Glyph: `#F5F5F5`
   - Inner shadow: `inset 0 0 0 1px rgba(255,255,255,0.06)`
   - Tile gap: 4 px
   - Border radius: 4 px
   - Mid-tile seam: a 1 px line at 50% height, `rgba(0,0,0,0.6)` to sell the split.
7. Container: `display: flex; flex-wrap: wrap; gap: 4px;` so long strings wrap naturally.
8. Performance: animate via `transform` only (never width/height). Use `will-change: transform` on the half divs during flip, remove after.
9. Accessibility: render a visually hidden `<span aria-live="polite">{text}</span>` so screen readers receive the final string immediately and do not get bombarded with intermediate scrambles.
10. Responsive: tile dimensions scale via `clamp(28px, 4vw, 48px)` width with 1.4× height.

WHY IT WORKS: It exploits the psychological pull of mechanical inevitability — the eye locks onto the scramble waiting for it to "land," which buys the component 1.4 seconds of guaranteed attention per render. Stagger creates a left-to-right read direction that maps onto text-reading muscle memory. The split-flap mechanism is rare enough on the web to feel like a signature, but cheap enough to render that it costs almost no perf.

GOOD / BAD: Best-in-class as a hero-headline replacement or a stat counter (numbers landing one digit at a time). Bad: do not use for paragraph-length copy; the scramble noise becomes overwhelming past ~24 characters. Also fails silently on `prefers-reduced-motion` if you forget the skip path.

CLEAN APPLICATION: Apply this to Clean's landing-page hero stat band. Below the hero headline, render a single line of split-flap tiles cycling through Clean's live metrics: "$7K IN PILOTS", "80+ STARTUPS", "4-5X TOKEN REDUCTION", "100 AGENTS PER PERSON". Pull the strings from a CMS or hardcoded array, swap every 6 seconds with a flip transition. This converts the static "stats grid" most SaaS landing pages have into a single living counter that rewards visitors who linger. Use Clean's mono font (JetBrains Mono or Berkeley Mono). Tile dimensions 44 px × 64 px on desktop.

## 5. Thibault Guignand Portfolio (Codrops case study) — https://tympanus.net/codrops/2026/05/06/from-shader-uniforms-to-clip-path-wipes-how-gsap-drives-my-portfolio/

WHAT IT DOES: A portfolio site where every motion in the page (block snaps, image warps, chromatic aberration offsets, text reveals, page-to-page transitions) is driven by a single `progress` float between 0 and 1 stored in JavaScript, tweened by a GSAP timeline with a custom ease, and copied each frame into a uniform on the GPU via OGL. The shader is stateless. Result: a unified motion grammar where every effect feels like part of the same gesture because they all ride the same curve.

PACKAGES:
- `vite` + vanilla JS (no React in the original; React port is trivial)
- `ogl` (chosen over Three.js for size, ~25 KB)
- `gsap` + `gsap/CustomEase` + `gsap/ScrollTrigger`
- `lenis`
- `splittype` for text splits
- Custom GLSL files, no shader libraries

IMPLEMENTATION:
1. App scaffold: a full-bleed `<canvas>` behind all HTML content (`position: fixed; inset: 0; z-index: -1`). HTML lives in normal flow above it. Canvas renders OGL fullscreen quad.
2. Shared uniform bus: a JS module exports `const uniforms = { uProgress: { value: 0 }, uMouse: { value: [0, 0] }, uTime: { value: 0 }, uResolution: { value: [w, h] } }`. Every shader pass reads from this bus.
3. GSAP timeline per "moment" (page enter, scroll-pinned section, hover state). Each timeline tweens `uniforms.uProgress.value` from 0 → 1 with `CustomEase.create("hero", "M0,0 C0.22,1 0.36,1 1,1")`. Duration 1.2 s.
4. Render loop: `requestAnimationFrame` ticks at 60 fps, copies the JS uniform values into the GPU uniforms, draws. Pause when no animations are active to save battery.
5. Block snap effect: a regular `<div>` is positioned in HTML, but its visual is rendered by the shader. Sample a 1 px texture of the div's current rasterized content (or use `html2canvas` once per state change for fallback). Shader displaces the texture's UVs by `uProgress * direction` then snaps to grid at thresholds 0.4, 0.7, 1.0 via `step()` functions.
6. Chromatic aberration: in fragment shader, sample the texture three times at slightly offset UVs (R, G, B channels separately), with offset magnitude = `uProgress * 0.012`. Wipes out the offset as progress completes.
7. Clip-path wipe (HTML-side, in parallel with shader): `gsap.to(panel, { clipPath: "inset(0 0 0 0)", duration: 1.2, ease: "hero" })` using the same custom ease as the shader. Because both layers share the curve, they read as one motion.
8. Lenis hooked into ScrollTrigger via `ScrollTrigger.scrollerProxy`. Lenis emits scroll, ScrollTrigger pins, GSAP tweens `uProgress`, shader and HTML both ride the same value.
9. Page transitions: route change pauses Lenis, runs a leave timeline (`uProgress 0 → 1` with a different ease, plus HTML clip-path), swaps DOM, runs enter timeline (`uProgress 1 → 0`).
10. Performance: limit DPR to 2; use `precision mediump float` in fragment shader; do not call `html2canvas` more than once per state change.

WHY IT WORKS: This is a single-pattern architecture — once you understand "one progress drives everything," you can predict and trust every motion on the site. That trust is what separates a portfolio that feels "designed" from one that feels assembled. Sharing the easing curve between GPU and DOM is the trick: human eyes are exquisitely sensitive to motion mismatches, and the moment two simultaneous animations use different curves, the whole composition collapses. This site never has that collapse.

GOOD / BAD: Best-in-class for a senior creative-dev portfolio or a product site that wants to signal craft to other developers. Bad: barrier to entry is high (the shader/timeline coupling demands real GLSL chops); also overkill for a B2B SaaS page where the visitor wants to read, not watch.

CLEAN APPLICATION: Apply this technique to Clean's product-demo section ("How Clean works"). Build a single horizontally-pinned scroll section with three "states" — Before Clean (chaos), Connecting (Clean intercepts), After Clean (unified context). One GSAP timeline tweens `uProgress 0 → 1` across the section's scroll length. A WebGL canvas renders a node graph of agents; as progress moves, nodes converge from scattered to a clean lattice. Simultaneously, HTML labels animate via clip-path wipes using the same custom ease. Visitors feel one motion, not five. Use OGL not Three (Clean cares about bundle size).

## BUILD DIRECTIVE

TARGET: Clean's landing-page hero + product-demo section (one continuous surface, rebuilt as a single vertical scroll on tryclean.ai).

APPROACH: Build a two-column fixed-grid layout (Fourmula AI pattern): a 280 px dark left rail with sticky Clean wordmark, "v0.x" version badge, and a single copy-paste install command CTA that stays put through the whole scroll. The right 70% scrolls through three pinned moments. Moment 1 (hero): split-flap stat band (Aceternity Text Flipping Board pattern) cycling through Clean's live numbers under a SplitText headline reveal. Moment 2 (the problem): a scroll-scrubbed product video showing a developer with 5 agents talking past each other, scrubbed by scroll position like Fourmula does its demo reels. Moment 3 (the answer): a WebGL canvas with one `uProgress` uniform driving node-graph convergence (Guignand portfolio pattern), with HTML labels animating via clip-path wipes on the same custom ease. Between moments, use a single full-width accent-color shutter wipe (Fourmula's signature transition). No second accent color. No grays.

STACK:
- `next` 14 (App Router)
- `tailwindcss` 3.4 + custom `flame` accent token
- `gsap` 3.12 + `gsap/ScrollTrigger` + `gsap/CustomEase` + `gsap/SplitText`
- `lenis` 1.1 for smooth scroll
- `ogl` 1.0 for the WebGL canvas (NOT three.js; bundle size matters)
- `framer-motion` 11 only for the split-flap component
- `splittype` for text splits
- Fonts: `Berkeley Mono` (everything chrome), `PP Neue Montreal` (display)
- Icons: hand-rolled 1.5 px stroke SVGs, no icon library

REFERENCE URLS:
- https://fourmula.ai/ (overall layout, sticky left rail, scroll-scrubbed video, two-color discipline)
- https://tympanus.net/codrops/2026/05/06/from-shader-uniforms-to-clip-path-wipes-how-gsap-drives-my-portfolio/ (single-uProgress shader architecture for moment 3)
- https://ui.aceternity.com/components/text-flipping-board (split-flap stat band)

KEY CONSTRAINTS: Clean is developer infrastructure. The result must feel technically sophisticated and fast, not flashy or generic. Avoid: (1) any 3D model, character, or mascot — no Spline, no glb files, no "playful" illustrations; (2) any gradient mesh or "vaporwave" coloring — strictly two colors, dark ink plus one accent; (3) any motion that scrolls slower than 60 fps on a 2019 MacBook Air; (4) any copy that uses the words "MCP," "one MCP," or "$300/seat/month" (these are permanently retired from Clean-facing materials); (5) any em-dashes in copy; (6) total JS bundle for the hero route must stay under 180 KB gzipped (this is why OGL over Three and no GSAP plugins beyond the four listed); (7) every animation must respect `prefers-reduced-motion` — split-flap collapses to instant render, scroll-scrubbed video falls back to static poster, shader timeline plays once and stops.
