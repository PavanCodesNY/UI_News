## Fourmula AI — https://fourmula.ai/  (Awwwards SOTD 2026-05-03, by Fourmeta)

WHAT IT DOES: Single-page generative-AI product site that uses a sticky scroll-pinned hero with a floating 3D mannequin/preset card stack on the right, while the left column cycles section copy synced to scroll progress. Pose presets and outputs swap via crossfade transitions driven by ScrollTrigger; cursor hover on cards triggers a subtle scale + tilt with shadow lift. Layout is a 12-col CSS grid with a persistent left-aligned brand mark, top-right primary CTA, and a horizontal preset reel that scrolls horizontally inside a pinned vertical section.

PACKAGES:
- next@14.2 + react@18.3 (or Webflow if going no-code; Fourmeta shipped this on Webflow)
- gsap@3.12 + @gsap/react (useGSAP)
- gsap/ScrollTrigger, gsap/ScrollSmoother, gsap/SplitText (Club GSAP)
- lenis@1.1.x (smooth scroll if not using ScrollSmoother)
- framer-motion@11 (for card hover/exit choreography if React)
- tailwindcss@3.4 + @tailwindcss/typography
- next/font with `Inter Display` for body and a custom display serif for hero word ("Fourmula" wordmark — heavy grotesk pairing)
- lucide-react@0.383 for chrome icons
- sharp + next/image for AVIF/WebP preset thumbnails

IMPLEMENTATION:
- Component tree: `<Shell>` → `<Nav>` (fixed top, 64px, blurred bg `backdrop-blur-xl bg-black/40`), `<Hero>` (h-[100vh], pinned), `<PresetReel>` (horizontal pin, h-[200vh] outer / 100vh inner), `<HowItWorks>` (3-up bento), `<UseCases>` (split-scroll), `<CTA>`, `<Footer>`.
- Hero grid: `grid-cols-12 gap-x-6 px-10`. Left: cols 1–6 with 96/108/120px display heading, line-height 0.92, letter-spacing -0.04em. Right: cols 7–12 holding a floating card stack (`position: relative`, 3 cards offset by `translate3d(8px,8px,0)` each, shadow `0 20px 40px -10px rgba(0,0,0,0.45)`).
- ScrollTrigger pin: `ScrollTrigger.create({ trigger: '.hero', start: 'top top', end: '+=120%', pin: true, scrub: 0.6 })`. Tween card stack: `gsap.to('.card', { yPercent: -8, rotateX: 6, stagger: 0.05, ease: 'power3.out' })`.
- Horizontal preset reel: outer `h-[300vh]`, inner `sticky top-0 h-screen overflow-hidden`. `gsap.to('.reel-track', { xPercent: -66, ease: 'none', scrollTrigger: { trigger: outer, start: 'top top', end: 'bottom bottom', scrub: 0.5 }})`.
- Card hover: `transform: perspective(800px) rotateX(calc(var(--my)*4deg)) rotateY(calc(var(--mx)*-4deg)) translateZ(0)`. Update `--mx`, `--my` via pointermove listener on each card (no JS animation — just CSS variable handoff to GPU).
- Color tokens: `--bg:#0A0A0B; --fg:#F5F4F1; --muted:#9B9A95; --accent:#E8FF55` (acid yellow-green hero accent for word "AI"). Gradient background under hero: `radial-gradient(60% 40% at 50% 0%, rgba(232,255,85,0.18), transparent 70%)`.
- Type stack: hero display → Editorial Old serif italic for one word + Söhne / Inter Display grotesk for the rest (juxtaposition is the entire move). Body: 17/26, letter-spacing -0.005em.
- Easing: `cubic-bezier(0.22, 1, 0.36, 1)` (expo-out) for entrances; `cubic-bezier(0.65, 0, 0.35, 1)` (in-out-cubic) for pinned scrubs. Durations 600–900ms for one-shots, 0 for scrubbed.
- Breakpoints: 1440 base, 1024 collapse hero to 8/4, 768 stack vertically, drop pinned reel to native horizontal scroll-snap (`snap-x snap-mandatory`).
- Performance: lazy-mount the reel via IntersectionObserver, `content-visibility: auto` on offscreen sections, preload only the first 3 preset images, AVIF + 80q.

WHY IT WORKS: Editorial-grotesk type pairing creates immediate "premium AI" semiotics without leaning on tired neon-cyber clichés. Pinned scroll choreography produces a perceived increase in product depth — the user feels they're "operating" the product as they scroll, not just reading about it. The acid-yellow accent on a near-black canvas hits ~14:1 contrast, anchoring the eye on a single conversion verb per viewport.

GOOD / BAD: Best-in-class — restrained color palette, crisp scroll choreography, visible brand POV. Bad — heavy fonts + 3D card stack push LCP past 3s on mid-tier mobile; horizontal pinned reel breaks Safari iOS scroll momentum if Lenis isn't tuned.

CLEAN APPLICATION: Apply this to Clean's **landing page hero (tryclean.ai)** by replacing the Fourmula preset cards with a stacked rendering of three "context capsules" — a raw JSON blob, a TOON-formatted compaction of the same payload, and a token-count delta badge ("12,400 → 3,180 tokens, -74%"). Use the same pinned-scroll scrub: as the user scrolls 0→100% of hero pin, animate the JSON card sliding behind, the TOON card forward, and the token badge counting down with `gsap.to({val:12400}, {val:3180, snap:{val:1}, scrub:0.6})`. Hero copy left column: "Context for agents" in 120px Söhne with one italic Editorial Old word — "**Compressed.**" — in acid green. Keep the rest of the page editorial-clean — no neon, no glassmorphism.

## Where Worlds Take Shape — https://www.awwwards.com/sites/where-worlds-take-shape  (Awwwards SOTD + Developer Award 2026-05-03)

WHAT IT DOES: Reimagines a portfolio as a playable WebGL world. A persistent Three.js scene fills the viewport; project entries are 3D objects placed in the world — the visitor navigates with WASD/drag/click rather than scrolling, and project detail is revealed via overlapping 2D panels that pin to scene coordinates. Integrated mini-games (per Awwwards inspiration tag) sit inside project entries. Camera transitions use damped lerp with subtle FOV breathing on hover.

PACKAGES:
- three@0.165 + three-stdlib
- @react-three/fiber@8.16, @react-three/drei@9.105 (CameraControls, Html, useGLTF, Environment, MeshTransmissionMaterial)
- @react-three/postprocessing@2.16 (Bloom, ChromaticAberration, N8AO)
- leva@0.9 (dev only, hide in prod)
- gsap@3.12 + ScrollTrigger
- zustand@4.5 for cross-tree game/scene state
- howler@2.2 for spatial audio cues
- next@14.2, react@18.3
- glsl-noise + custom GLSL shaders compiled via vite-plugin-glsl
- meshoptimizer + DRACO loader for sub-2MB gltf budget
- tailwindcss for HUD/2D layer

IMPLEMENTATION:
- Top-level: `<Canvas dpr={[1,1.75]} gl={{antialias:false,powerPreference:'high-performance'}} camera={{fov:42,near:0.1,far:200}}>`. Wrap with `<Suspense fallback={<Loader/>}>`.
- Scene graph: `<Environment preset='night' />`, `<DirectionalLight intensity={0.6} position={[5,8,3]} castShadow />`, `<PointLight />` for accent. Use baked lightmaps (gltf with second UV channel) — avoid realtime shadows on mid-tier hardware.
- World streaming: split world into 3 GLB chunks (~600KB each), load nearest chunk via `useGLTF.preload`. DRACO + meshopt; budget 60k triangles on screen.
- Camera rig: drei `<CameraControls makeDefault smoothTime={0.18} draggingSmoothTime={0.08}>` clamped to a y-bounded box. On hover of a project node, GSAP-tween FOV from 42→38 over 600ms with `ease:'power2.out'`.
- Project nodes: `<mesh>` with `<MeshTransmissionMaterial samples={6} thickness={0.4} chromaticAberration={0.05} anisotropy={0.3} distortion={0.2} />` for a premium glassy look. On click, dispatch zustand `enterProject(id)`; HUD layer fades in a 2D panel positioned via drei `<Html distanceFactor={6} center transform />` anchored to node.
- Postprocessing pipeline: `EffectComposer` → `N8AO(intensity=2,aoRadius=1.2)` → `Bloom(intensity=0.6,luminanceThreshold=0.85,mipmapBlur)` → `ChromaticAberration(offset=[0.0008,0.0008])`. Render at 0.85x resolution and upscale (FXAA).
- Mini-game implementation: each project owns its own subscene factory `(scene, scope) => disposeFn`. Activate on enter, dispose on exit.
- Mobile fallback: detect `navigator.gpu` + DPR + touch; below threshold render a static cinematic loop (h264 720p) instead of live R3F.
- Loading: progressive — show low-poly draft while high-detail streams in. Use `useProgress()` from drei; display a typographic loader (digit counter, no spinner).
- HUD CSS: pointer-events `none` on container, `auto` on actionable elements; HUD typography in JetBrains Mono 14/20.

WHY IT WORKS: Spatial navigation reframes the cognitive task from "consume a list" to "explore a place," which dramatically increases dwell time and per-project attention (Gestalt: figure-ground inversion — the world becomes the figure, content the supporting cast). Using glassy MeshTransmissionMaterial telegraphs technical sophistication without resorting to flashy palettes. FOV-breathe on hover is sub-perceptual but registers as "the world responds to me."

GOOD / BAD: Best-in-class — bold reframing of the portfolio as a place; performance-aware via baked lighting and chunked GLBs. Bad — discoverability cliff (users who don't realize they can drag bounce); WCAG-hostile for keyboard/screen-reader users without an accessible 2D mirror page.

CLEAN APPLICATION: Apply this to Clean's **product demo / "How it works" section** by rendering Clean's MCP proxy topology as a navigable 3D node graph: agents on the left as glassy spheres, the Clean proxy as a central torus, and downstream MCP servers as cubes on the right. Use `<MeshTransmissionMaterial>` exactly as above on the proxy node to telegraph "context layer." On click of an edge, animate a token packet (small instanced mesh) traveling from agent → proxy → server, splitting at the proxy with a real number badge ("12,400 tokens" → "3,180 tokens"). Provide a 2D fallback list under the canvas labeled "View as list" for accessibility — Clean's audience is developers, half of whom will skip the 3D view and want raw spec.

## Studio375 — https://www.awwwards.com/sites/studio375-2  (Awwwards SOTD + Developer Award 2026-04-30)

WHAT IT DOES: Agency portfolio built on Next.js + GSAP with a duotone palette (#cc0597 magenta, #11baff cyan) on near-black. Hero uses an oversized kinetic wordmark that splits and re-forms on cursor proximity using SplitText + per-character GSAP tweens. Project transitions use a flag-style WebGL displacement reveal between routes; mouse cursor is a 12px blended-difference circle that magnetizes to interactive elements.

PACKAGES:
- next@14.2 (App Router) + react@18.3
- gsap@3.12 + @gsap/react + ScrollTrigger + SplitText + Flip + DrawSVG (Club GSAP)
- ogl@1.0.x or three@0.165 for the WebGL transition shader (ogl is lighter — 25KB)
- lenis@1.1
- @studio-freight/tempus for unified RAF
- zustand for cursor + transition state
- tailwindcss@3.4
- next/font: `PP Neue Montreal` (display) + `JetBrains Mono` (numerals)
- gl-matrix for shader uniforms

IMPLEMENTATION:
- Color tokens: `--magenta:#cc0597; --cyan:#11baff; --bg:#070708; --fg:#F2F1ED`. Use HSL variants for hover states. Forbid white text on either accent — both fail AAA; use #070708 on accents and #F2F1ED on bg.
- Custom cursor: fixed 12px disk, `mix-blend-mode: difference`, `transform: translate3d(x,y,0)` updated in a single RAF via `lerp(current, target, 0.18)`. On hover of `[data-magnet]`, target snaps to element bounding-box center; scale to 56px and switch fill to magenta.
- Hero kinetic type: SplitText words → chars. On pointermove within 240px radius, compute per-char `dx,dy` with falloff `1 - (dist/240)^2`, push as `translate3d` via gsap.quickTo. Idle, run a 4s sinusoidal breathe on `letter-spacing`.
- Route transition: on link click, dispatch `transition()` → render a fullscreen WebGL plane sampling the outgoing page (html2canvas snapshot) + incoming page texture, transition with a custom GLSL displacement shader over 900ms `power3.inOut`. Fragment uses a worley-noise mask scrolling along (1,1) direction with a `progress` uniform.
  ```glsl
  float mask = step(progress, noise(uv * 4.0) + uv.x * 0.4);
  vec3 col = mix(textureA.rgb, textureB.rgb, mask);
  // Add chromatic offset at the seam:
  col.r = texture2D(B, uv + vec2(0.005,0)).r * mask + ...;
  ```
- Project grid: 2-col asymmetric, project cards rotate `±2deg` based on column parity, hover lifts via `translateY(-8px)` and scale `1.02` over 400ms `power2.out`.
- Scroll: Lenis with `lerp:0.1, duration:1.2`; hand off to GSAP via Tempus so ScrollTrigger and Lenis share one RAF.
- Section reveals: SplitText lines, mask `clip-path: inset(0 0 100% 0)` → animate to `inset(0 0 0% 0)` with `stagger: 0.04, duration: 1, ease: 'expo.out'`.
- Performance: defer the WebGL transition shader code until first navigation (dynamic import); preload it on idle via `requestIdleCallback`.
- Breakpoints: drop kinetic-on-cursor on `pointer:coarse` (touch) — replace with scroll-driven kinetic. Hero font scales 80px → 132px → 180px at 768/1280/1920.

WHY IT WORKS: The duotone-on-near-black system gives the eye exactly two places to land per viewport — magenta for "act" and cyan for "informational link" — which is a textbook application of pre-attentive attribute encoding (Healey/Ware). Kinetic type on cursor proximity reinforces the brand promise of "we make motion" before the user reads a single project. The flag-style transition shader is the studio's signature — every page-change reaffirms the brand.

GOOD / BAD: Best-in-class — single signature transition repeated everywhere builds brand muscle memory. Bad — magenta + cyan combo is a vibration trap on adjacent edges (avoid; either separate with neutral gutter or downshift saturation 8%).

CLEAN APPLICATION: Apply this to Clean's **pricing page or feature comparison** by adopting the duotone-on-near-black system but recoded to Clean's brand: keep one electric accent (suggest `#7CFFB2` mint or `#E8FF55` acid lime) and pair with a cool steel `#6B8AFF`. Use the kinetic-cursor magnet on plan-tier cards and "Start free" CTA. Skip the WebGL route transition (overkill for a dev tool, hurts perceived speed) — instead use the SplitText line-mask reveal on every section header. Numerals in JetBrains Mono so price/usage tickers look engineered.

## Adcker — https://www.awwwards.com/sites/adcker  (Awwwards SOTD 2026-05-01)

WHAT IT DOES: Editorial-typography-led creative-agency site with a strict 2-color palette (#191919 ink, #efedea cream). Uses GSAP + Barba.js for SPA-style page transitions — content shifts slide in as horizontal columns while a giant masthead word slices vertically. Hover states on project tiles trigger a clip-path image reveal where the project image scales from a tight rectangle out to fill the tile while a serif italic title drops in from above.

PACKAGES:
- WordPress (per Awwwards meta) → equivalent for new builds: astro@4.x + view-transitions or next@14 (App Router with native view transitions)
- gsap@3.12 + ScrollTrigger + SplitText + CustomEase
- barba.js@2.10 (if not using native view-transitions)
- lenis@1.1
- swiper@11 (project carousel)
- self-hosted variable fonts: a serif display (Editorial New / GT Sectra / Söhne Mono pairing) + a grotesk body
- tailwindcss with a custom typography plugin extending font-feature-settings (`'ss01','ss02','liga'`)

IMPLEMENTATION:
- Color tokens: `--ink:#191919; --cream:#efedea; --rule:#191919/12`. No accent color — emphasis is typographic only (italic, scale, weight).
- Typography scale: editorial 12-step scale, base 17px / 1.55 line-height, display steps 32 / 48 / 72 / 108 / 168 / 240. Use `font-feature-settings: 'ss01','liga','onum'` on body for editorial typesetting.
- Layout: 14-col grid with 32px gutters, 80px outer padding. Section rhythm uses `--space-*` scale of `clamp(48px, 6vw, 120px)`.
- Masthead slice transition: on route exit, `gsap.to('.masthead', { clipPath: 'inset(0 0 100% 0)', duration: 0.7, ease: CustomEase.create('outExpo','M0,0 C0.16,1 0.3,1 1,1') })`; route enter mirrors with `inset(100% 0 0 0)` → `0`. Crossfade body content during the seam.
- Project tile hover: tile is `aspect-[4/5]` with image absolutely positioned at `clip-path: inset(20% 20% 20% 20%)`. On hover: `clip-path: inset(0 0 0 0); scale: 1.04` over 700ms `expo.out`. Title shifts from grotesk roman → italic serif via a class swap (CSS animates color/letter-spacing, JS swaps font-family on the same span — 60ms imperceptible).
- Cursor: text label cursor — fixed div that displays "VIEW" or "DRAG" depending on context. Use `mix-blend-mode: difference` over imagery so it works on cream and on top of project photos.
- Smooth scroll: Lenis at default; SplitText lines with `mask` reveal — same recipe as Studio375.
- Carousel: Swiper with `slidesPerView: 'auto'`, `spaceBetween: 24`, free-mode true, `mousewheel.forceToAxis: true`.
- Performance: serif display fonts are heavy — subset to A–Z, 0–9, punctuation only (cuts ~70% file size); use `font-display: swap` and a tuned fallback metric (`size-adjust: 102%`) to avoid CLS.
- Breakpoints: 768 collapses to single col, drops display step from 240 → 96 (avoid horizontal scroll nightmares).

WHY IT WORKS: Two-color editorial systems force every design decision into the typography layer, which is where the most refined craft signals live. Restraint is the message — it telegraphs "we trust the work, not the chrome." Italic-on-hover micro-interaction is a one-bit signal that produces an outsized brand-warmth effect (the system feels human).

GOOD / BAD: Best-in-class — typographic hierarchy is masterclass-level; brand voice unmistakable. Bad — brutally hostile to skim-readers; weak F-pattern affordances mean users without time to read miss the value prop.

CLEAN APPLICATION: Apply this to Clean's **docs site (docs.tryclean.ai) and blog** by adopting the 2-color editorial system 1:1 — `#0A0A0B` ink on `#F4F2EE` cream — and the 12-step typographic scale. Use Editorial Old serif italic for one-word headings (e.g., "**Why.**", "**Install.**") followed by a grotesk subhead. Adopt the SplitText line-mask reveal on docs page entries. Keep a single accent only for code-block syntax tokens. The italic-on-hover swap fits perfectly for inline `code` references → on hover, transform to italic serif glossary with definition tooltip.

## More Than a Portfolio (Codrops tutorial) — https://tympanus.net/codrops/2026/04/28/more-than-a-portfolio-building-a-scroll-driven-3d-world-with-something-to-say/

WHAT IT DOES: Scroll progress drives camera path, object animation, and reveal timing across a hand-built Three.js scene (modeled in Blender). Each scroll viewport corresponds to a "scene" — camera dollies through an architectural underwater-lit interior, with text overlays and 3D objects entering on scroll-mapped timelines. Demonstrates the canonical "scroll-as-timeline" pattern at production quality.

PACKAGES:
- three@0.165
- gsap@3.12 + ScrollTrigger + MotionPathPlugin
- lenis@1.1
- meshoptimizer + DRACO + KTX2 textures (basis-universal)
- vite@5 + vite-plugin-glsl
- Blender (asset pipeline only) → glTF 2.0 export
- HDRI env map (1k for mobile, 2k desktop) hosted as KTX2

IMPLEMENTATION:
- Scene file budget: total < 4MB compressed. Bake all lighting in Blender (Cycles, 256 samples), export to a single texture atlas; ship one mesh + one PBR atlas + one HDRI.
- Camera path: define a `THREE.CatmullRomCurve3` of waypoints in code (or import from Blender as an empty's animation). Sample by scroll progress: `camera.position.copy(curve.getPoint(progress))` and `camera.lookAt(curve.getPoint(progress + 0.01))`.
- Scroll mapping: one master ScrollTrigger driving a single GSAP timeline, all object tweens added with `.to(...)` at proportional offsets. Use `scrub: 0.6` for dampened scrub (snappier than 1, smoother than 0.2).
- Per-section text: position in DOM, animate via SplitText with `clip-path` masks; trigger on `enter` of per-section ScrollTriggers nested inside the master.
- Shader effects: a single procedural caustics shader on water-floor (`gl_FragColor = vec3(causticsNoise(uv*5.0 + time*0.1)) * 0.6 + base.rgb`), godrays via `BloomEffect` only — no volumetric (too expensive on mobile).
- Mobile: detect via `matchMedia('(pointer:coarse)')` + `navigator.deviceMemory < 4`; serve a 720p MP4 fallback that scrubs with scroll (`video.currentTime = scroll * duration`). This is the dirty secret of high-end "WebGL" sites — most ship video on mobile.
- Audio: optional ambient wash via Howler, gated behind a user-initiated unmute button (autoplay policy).
- LCP: hero text must paint independently of WebGL — render text in DOM, mount canvas as background after.
- Rendering: half-res offscreen canvas + FXAA, upscale via CSS — saves 2x fillrate.

WHY IT WORKS: Compressing a 3-month bespoke 3D scene into a single ScrollTrigger timeline gives you cinematic narrative control while remaining 100% deterministic — every frame is a function of scroll progress, which is debuggable and shippable. Pre-baked lighting collapses what looks like an expensive realtime PBR scene into commodity raster work the GPU yawns at.

GOOD / BAD: Best-in-class — disciplined asset pipeline (single mesh + atlas + HDRI), bakes lighting so even mid-tier laptops hit 60fps. Bad — building this honestly takes 2–3 months; no AI-generated 3D will substitute; nothing about the technique addresses keyboard navigation or reduced-motion preferences.

CLEAN APPLICATION: Apply this to Clean's **homepage middle section ("How it works")** as a scroll-pinned 3-stage architectural diagram of the proxy: (1) raw context flowing into the proxy — camera enters a tunnel of streaming JSON cards; (2) compaction at the proxy — camera arrives at a central glassy node where TOON encoding visibly compresses the stream (literal length collapse); (3) emission — camera exits with a thinner, glowing token stream toward the agent. Use the same single-master-timeline ScrollTrigger pattern with `scrub: 0.6`. Bake everything in Blender — Clean's audience is developers and they will judge you on FPS, not on realtime fidelity. Honor `prefers-reduced-motion: reduce` by replacing the scroll-driven camera with a 3-image static stepper.

KEY CONSTRAINTS — Clean is developer infrastructure. The result must feel technically sophisticated and fast, not flashy or generic.
- AVOID: rainbow neon gradients, glassmorphism panels stacked 4-deep, "AI sparkles," generic Aceternity hero spotlights, 3D blob backgrounds, autoplaying full-screen video, parallax-just-because.
- AVOID: any animation that delays first interaction past 1.2s on a mid-tier laptop. LCP < 2.0s. CLS < 0.05.
- REQUIRE: keyboard nav for everything; `prefers-reduced-motion` honored (swap pinned scroll for 3-image stepper); every numeral that ticks must come from a real Clean metric, not a fake.
- REQUIRE: the mid-page Three.js scene degrades to a 720p H.265 video on `pointer: coarse` + `deviceMemory < 4`, and to a 3-image static stepper under `prefers-reduced-motion`.
- REQUIRE: developer-facing copy. Show the actual TOON output. Show the actual token count delta. Link to the GitHub repo and the docs from the hero, not just "Get Started."
- REQUIRE: zero stock photography. Zero illustrations of robots. Zero "boy and laptop" iconography.
- TYPE: one display grotesk + one mono + one Editorial Old italic for accent words. Three families, no more.
- COLOR: max 3 colors. Suggest `#0A0A0B` ink, `#F4F2EE` cream, `#7CFFB2` mint accent. That's the entire palette.
