## 1. Marvell Tile & Stone — https://www.awwwards.com/sites/marvell-tile-stone

WHAT IT DOES: A single-page luxury commerce site built as one continuous WebGL-driven scroll journey. Hero is a full-viewport canvas with a Three.js plane that swaps high-res tile textures based on scroll progress (texture morph via shader uniform `uProgress`), while a sticky pinned section uses Lenis smooth scroll + a horizontal carousel of product collections that reveal on `scroll-snap` thresholds. Type system is large-format display serif (60–144px) layered over the canvas, with a thin uppercase mono caption that tracks the cursor via a `mousemove` listener throttled to RAF.

PACKAGES:
- next@14.2 (App Router) or astro@4 (alt)
- three@0.165
- @react-three/fiber@8.16
- @react-three/drei@9.108
- gsap@3.12 (+ ScrollTrigger, Observer)
- lenis@1.1 (smooth scroll)
- splitting@1.0 or split-type@0.3 (line/word splitting)
- tailwindcss@3.4 (utilities only) or vanilla CSS modules
- @fontsource-variable/instrument-serif + @fontsource/jetbrains-mono
- @studio-freight/hamo (RAF + viewport hooks)

IMPLEMENTATION: Component tree: `<App>` → `<LenisProvider>` → `<CanvasLayer>` (fixed, `pointer-events: none`, z-0) + `<DOMLayer>` (relative, z-10). Inside CanvasLayer mount `<Canvas dpr={[1, 1.5]} gl={{ antialias: true, alpha: true }}>` with one full-screen `<ShaderMaterial>` plane (vertex passthrough; fragment samples between two textures using `mix(texA, texB, smoothstep(0.4, 0.6, uProgress))`). Drive `uProgress` from a GSAP ScrollTrigger timeline scrubbed `1.2`. DOMLayer sections: hero (100vh), pinned horizontal scroller (`pin: true`, `scrub: 1`, x-translate `-100% * (slides - 1)`), product grid (CSS `grid-template-columns: repeat(12, 1fr)` with asymmetric spans `col-span-7 / col-span-5`). Type animations: split words → `gsap.from('.word', { yPercent: 110, stagger: 0.04, duration: 0.9, ease: 'expo.out' })` triggered on `ScrollTrigger.create({ start: 'top 75%' })`. Cursor caption: position absolute, `transform: translate3d(${x+24}px, ${y+24}px, 0)`, lerp toward target each RAF tick at `0.18`. Performance: lazy-init Three only after first paint with `next/dynamic({ ssr: false })`, compress textures to KTX2 via `@gltf-transform/cli`, cap DPR at 1.5, pause RAF on `document.hidden`. Breakpoints: pin disabled `<1024px`, swap horizontal scroller for vertical stacked cards.

WHY IT WORKS: Visual hierarchy is enforced by texture > type > caption (largest-to-smallest, warmest-to-coolest contrast), so the eye lands on product before brand voice. The pinned scrub creates motion semantics that map "scrolling = moving through a showroom," which the user understands without instruction. Whitespace rhythm (96px section padding, 24px baseline grid) reads as expensive — the same Gestalt principle of common region that luxury print uses.

GOOD / BAD: Good — texture-morph on scroll feels physical, not jittery; mono caption is an inspired focal anchor. Bad — 4MB+ texture payload on first load, no `prefers-reduced-motion` fallback, mobile horizontal pin is replaced with a much weaker stacked layout.

CLEAN APPLICATION: Apply this to Clean's landing page hero by replacing the texture-morph with a TOON-format-vs-JSON-payload morph. Build a `<ShaderMaterial>` plane that interpolates between two text-rendered textures: left state shows a 47-line bloated JSON tool schema, right state shows the same data as a 12-line TOON format block. Drive `uProgress` from the first 100vh of scroll so as the user scrolls, the JSON literally compresses into TOON in front of them. Below the canvas, pin a horizontal scroller of three "before/after context size" cards (Cursor, Claude Code, Windsurf) using identical GSAP ScrollTrigger pin. Use `Instrument Serif` only for the headline word "Context"; everything else stays in JetBrains Mono to preserve the developer-tool register.

## 2. Fourmula AI — https://www.awwwards.com/sites/fourmula-ai

WHAT IT DOES: Dark-mode AI product landing (palette: `#020108` base, `#FF6B02` single accent) with a hero that uses a continuously animated SVG noise/grain layer over a CSS conic-gradient orb, pinned scroll-driven feature showcase where each feature tile scales from `0.85 → 1.0` and swaps a video MP4 inline (`<video autoplay muted playsinline preload="metadata">`). Built on Webflow with GSAP ScrollTrigger orchestrating every reveal. Type is a custom geometric sans paired with a mono accent.

PACKAGES:
- gsap@3.12 (ScrollTrigger, ScrollSmoother, SplitText — Club GSAP)
- lenis@1.1 (or ScrollSmoother alone)
- @vercel/og or static MP4 product clips (h.264, 1080p, ~2MB each)
- tailwindcss@3.4 + tailwindcss-animate
- next@14.2 + framer-motion@11.3
- @fontsource/geist + @fontsource/geist-mono OR `Neue Haas Grotesk` self-hosted
- lottie-react@2.4 for the orb (alt to conic-gradient)
- vaul@0.9 for the auth modal pattern

IMPLEMENTATION: Hero = `position: relative; height: 100vh; background: #020108`. Inside: `<div class="orb">` (480px circle, `background: conic-gradient(from 180deg at 50% 50%, #FF6B02, #1a0a02, #FF6B02)`, `filter: blur(48px)`, `animation: spin 18s linear infinite`). Overlay an SVG `<filter id="grain">` with `<feTurbulence baseFrequency="0.9" numOctaves="2"/>` masked at `opacity: 0.08`. Headline split-text reveal: `gsap.from(splitInstance.chars, { yPercent: 120, opacity: 0, stagger: 0.015, duration: 0.7, ease: 'power3.out', delay: 0.2 })`. Feature section: `<section class="features">` with `display: grid; grid-template-columns: repeat(4, 1fr); gap: 8px`. Each card is `aspect-ratio: 4/5; border-radius: 20px; overflow: hidden; background: #0a0408`. ScrollTrigger per card: `{ trigger: card, start: 'top 80%', onEnter: () => { card.querySelector('video').play(); gsap.to(card, { scale: 1, duration: 0.9, ease: 'expo.out' }) } }`. Accent buttons: `bg-[#FF6B02] text-black px-5 py-3 rounded-full font-mono text-sm uppercase tracking-wider`, hover `gsap.to(btn, { scale: 1.04, duration: 0.2 })`. Performance: poster-only on mobile, IntersectionObserver pauses videos when out of view, preconnect to video CDN. Breakpoints: 4-col → 2-col at `md`, single col stacked at `sm`.

WHY IT WORKS: One-accent palette (#FF6B02 against near-black) creates a 13:1 contrast ratio that forces hierarchy without a single weight change. The conic-gradient orb behaves as a single focal heat source — Gestalt closure pulls the eye to it before reading. Auto-playing inline product clips beat static screenshots on engagement because they pre-answer "what does this actually do?" within 2 seconds.

GOOD / BAD: Good — the single accent + mono type combo is the most efficient way to look both serious and AI-native. Bad — four simultaneous autoplay videos in the viewport will throttle on low-end laptops; the SVG grain filter is GPU-expensive on Safari.

CLEAN APPLICATION: Apply this to Clean's product demo section directly below the hero. Build a 4-card bento grid showcasing the four core capabilities — "Context Compression", "Tool Routing", "Token Audit", "Multi-Provider Proxy" — each with an inline 6-second screen-cap MP4 of the actual Clean dashboard demonstrating that capability. Replace the orange accent with Clean's brand accent. Keep the dark base `#0A0A0F` and use the same scale-on-scroll-in animation. Wire each video to play exactly once on first viewport entry, then loop on hover only. This reads as "real product, not vaporware" — the single biggest objection developer-infra companies face from technical buyers.

## 3. ASTRODITHER — https://www.awwwards.com/sites/astrodither

WHAT IT DOES: Single-canvas WebGL experience built around a real-time dithered-bayer post-processing pass. Hero is a `<Canvas>` with a low-poly orbiting sphere/icosahedron, then a custom `EffectComposer` chain runs: `RenderPass → BayerDitherPass (4×4 ordered dither, 2-color palette #FF4FA3 / #0B0B12) → OutputPass`. Cursor controls camera rotation (`camera.rotation.y = lerp(camera.rotation.y, mouse.x * 0.4, 0.06)`). Scroll progresses through three "scenes" by tweening `scene.position.y` and the dither threshold `uniform`.

PACKAGES:
- three@0.165
- @react-three/fiber@8.16
- @react-three/drei@9.108
- @react-three/postprocessing@2.16
- postprocessing@6.35 (custom Effect class for bayer dither)
- leva@0.9 (dev tweaking, drop in prod)
- gsap@3.12 (ScrollTrigger for scene transitions)
- maath@0.10 (easing helpers + lerp)
- next@14.2

IMPLEMENTATION: Custom Effect class extending `postprocessing.Effect` with fragment shader: read scene color, compute bayer matrix index from `gl_FragCoord.xy`, threshold luminance, output `mix(uColorA, uColorB, step(bayer, lum))`. Uniforms: `uColorA = #FF4FA3`, `uColorB = #0B0B12`, `uPixelSize = 2.0`, `uBias = 0.5`. Mount inside `<EffectComposer>` after main render pass. Geometry: `<Icosahedron args={[1.4, 1]} />` with `<MeshNormalMaterial />` for varied luminance input → dither pattern visibly shifts as it rotates. Scroll choreography: three `ScrollTrigger.create` blocks at `start: '0%'`, `start: '33%'`, `start: '66%'`, each tweens `geometryRef.current.position.y` and `effectRef.current.uniforms.get('uBias').value`. Cursor lerp inside `useFrame((_, dt) => { camera.rotation.x = damp(camera.rotation.x, mouse.y * 0.3, 4, dt) })`. DOM overlay: text in `mix-blend-mode: difference` so it inverts against the dithered background. Type: `Px Grotesk` or `Suisse Int'l Mono` at 14px tracking 0.04em. Performance: pixel ratio capped at 1.0 (dither effect already pixelated, anti-alias would defeat the look), `gl={{ powerPreference: 'high-performance' }}`, disable on `prefers-reduced-motion` (swap to static dithered PNG fallback).

WHY IT WORKS: The 2-color palette + bayer dither is a deliberate visual language that signals "early-90s computing aesthetic" (Macintosh, AmiPaint) while running on bleeding-edge WebGL — the contrast between historical reference and technical execution is the whole product. Limited palette enforces extreme contrast hierarchy with zero design decisions left open. Cursor-driven camera rotation creates the "this is alive" feeling without burning attention on overt animation cues.

GOOD / BAD: Good — best-in-class case for using a single technical effect as the entire brand, no decoration needed. Bad — pure dither obliterates accessibility for low-vision users; no fallback for WebGL-disabled browsers; the pink/black palette is locked in and can't be themed.

CLEAN APPLICATION: Apply this to Clean's docs/changelog landing nav as a contained 240×240px hero widget — not the entire page. Render a slowly rotating wireframe dodecahedron (representing the agent context "ball") with the bayer-dither pass in Clean's two-color palette. Place it top-left of the docs nav, replacing the static logo on the docs subdomain only. Keep the docs body itself in clean monochrome typography. The widget visually brands the docs as "this is built by people who actually care about graphics" — a recruiting and credibility signal aimed at the senior engineers who choose infrastructure.

## 4. T11 — https://www.awwwards.com/sites/t11

WHAT IT DOES: Webflow + GSAP-driven studio site by Federico Pian (developer-award winner same day as Marvell). Layout is a stacked editorial scroll — full-bleed image hero, oversized italic display headline that splits across two rows, and a "marquee of capabilities" section that uses an infinite seamless scroll loop (CSS `transform: translateX` driven by GSAP `to(..., { repeat: -1, ease: 'none' })`). Project tiles use a hover-tilt micro-interaction (3D rotateX/rotateY on `mousemove` mapped from element-relative cursor position) and a magnetic CTA button (cursor pulls within a 60px radius, eased lerp).

PACKAGES:
- webflow (no-code) OR next@14.2 + tailwindcss@3.4 to rebuild
- gsap@3.12 (Observer, ScrollTrigger, CustomEase)
- lenis@1.1
- split-type@0.3
- @studio-freight/hamo (RAF utilities)
- locomotive-scroll@5 (alt smooth-scroll)
- @fontsource-variable/instrument-serif (italic display)
- @fontsource/inter (UI text)

IMPLEMENTATION: Headline reveal: split into lines, `gsap.from('.line', { y: 120, rotation: 6, opacity: 0, stagger: 0.08, duration: 1.1, ease: CustomEase.create('cleanOut', 'M0,0 C0.2,0 0.1,1 1,1') })`. Marquee: container `display: flex; width: max-content`, render content twice in sequence, animate `xPercent: -50` (because content is doubled, -50% lands seamlessly at the loop point). Pause on hover with `gsap.to(tween, { timeScale: 0, duration: 0.4 })`. Tilt card: on `mousemove` compute `const rect = card.getBoundingClientRect(); const px = (e.clientX - rect.left) / rect.width - 0.5; const py = (e.clientY - rect.top) / rect.height - 0.5;` then `gsap.to(card, { rotateY: px * 14, rotateX: -py * 10, transformPerspective: 800, duration: 0.6, ease: 'power2.out' })`. Magnetic button: same pattern but translate the inner label by `(px * 0.4, py * 0.4) * radius`. Lenis config: `{ duration: 1.2, easing: t => Math.min(1, 1.001 - 2 ** (-10 * t)) }`. Image grid: CSS `grid-template-columns: repeat(12, 1fr)`, project tiles span `col-span-5 / col-start-2` and `col-span-6 / col-start-7` alternating for asymmetric rhythm.

WHY IT WORKS: The asymmetric 12-col grid with alternating spans creates Gestalt rhythm that reads as intentional editorial design rather than templated portfolio. Magnetic + tilt micro-interactions are tuned just under the threshold of "showy" — they reward mouse exploration without demanding attention. Italic display serif on a tech-flavored monospace body is the most over-used pairing right now precisely because it works: emotional weight in headlines, neutral authority in body.

GOOD / BAD: Good — the `CustomEase` headline reveal (rotation + y + opacity composited) is the cleanest "show, don't shout" entry animation pattern in the showcase right now. Bad — magnetic + tilt simultaneously can feel busy on touchpad scroll; mobile loses 80% of the interaction model.

CLEAN APPLICATION: Apply this to Clean's pricing page. Build the three pricing tier cards (Free / Pro / Enterprise) with the magnetic-button CTA pattern on the "Start free" / "Contact us" buttons — radius 80px, lerp 0.18. Above the tiers, add a single full-bleed italic headline: "Pay for context, not for tokens you didn't need." (split-type line reveal on enter). Below the tiers, add the seamless capability marquee with TOON, MCP, JSON Schema, OpenAPI, Anthropic, OpenAI, Google logos scrolling at 80px/sec. Keep the asymmetric 12-col grid for the FAQ section underneath: question span 5, answer span 6 starting col 7, alternating. This adds editorial sophistication to a page that 95% of dev-tool companies render as a flat 3-column table.

## 5. Immersive Garden (10 Years) — https://www.awwwards.com/sites/immersive-garden-celebrating-10-years

WHAT IT DOES: A studio retrospective built as an interactive WebGL gallery with a curved-scroll camera path. Camera follows a `CatmullRomCurve3` through 3D space, with framed project thumbnails as `<MeshBasicMaterial>` planes positioned along the curve. Scroll position maps to `curve.getPointAt(t)` and `curve.getTangentAt(t)`. Hover on a plane triggers a fluid "wave distortion" via fragment shader `uniform uHover` interpolated 0→1 over 600ms.

PACKAGES:
- three@0.165
- @react-three/fiber@8.16
- @react-three/drei@9.108 (specifically `useCurve`, `MeshDistortMaterial`)
- @react-three/postprocessing@2.16 (for `Bloom` + `ChromaticAberration` passes)
- gsap@3.12
- lenis@1.1
- glsl-noise (perlin/simplex shader includes via `glsl` loader or `glslify`)
- next@14.2 with `@react-three/fiber` SSR-disabled
- @fontsource-variable/jetbrains-mono

IMPLEMENTATION: Build curve: `const curve = new THREE.CatmullRomCurve3([new Vector3(0,0,0), new Vector3(4,1,-6), new Vector3(-3,2,-12), new Vector3(2,-1,-18), new Vector3(0,0,-24)], false, 'catmullrom', 0.5)`. In `useFrame`: `const t = scrollProgress.current; const pos = curve.getPointAt(t); const tan = curve.getTangentAt(t); camera.position.copy(pos); camera.lookAt(pos.clone().add(tan))`. Project planes positioned at `curve.getPointAt(i / N)` rotated to face curve normal. Custom hover shader: vertex displaces `position.z += sin(uTime + position.x * 4.0) * uHover * 0.15`. Postprocessing: `<Bloom luminanceThreshold={0.9} intensity={0.6} />` + `<ChromaticAberration offset={[0.0008, 0.0008]} />` — subtle, do not crank. Scroll → `t` mapping via Lenis raw progress, eased through `gsap.quickSetter(progress, 'value', 'lerp 0.08')`. DOM overlay: project metadata (year, client, role) appears with `opacity` tied to `1 - distance(camera, plane)` so info fades in only when the user is "near" a piece. Type: JetBrains Mono 12px uppercase, `letter-spacing: 0.1em`. Performance: instance the planes if N>20, use `KTX2Loader` for compressed textures, skip postFX on `<lg`.

WHY IT WORKS: A 3D path through a curated archive is the only retrospective format that lets you communicate "we have evolved through space and time" without listing it chronologically — the curve itself is a metaphor. Hover-triggered shader distortion gives weight to the cursor as a "lens" — small motion semantic that signals "you can manipulate this." Subtle bloom + chromatic aberration is the difference between "we shipped a portfolio" and "we shipped a portfolio that looks like a film print."

GOOD / BAD: Good — best execution this month of camera-on-curve, the math + UX intuition are precisely tuned. Bad — discoverability of off-path projects is poor (you can't jump, only scroll); the experience burns ~80MB of memory on Chrome and crashes some iPads.

CLEAN APPLICATION: Apply this to Clean's "How it works" section on the landing page. Build a curved 3D scroll through 5 stations: (1) Agent makes a tool call, (2) Clean intercepts, (3) Schema compressed to TOON, (4) Routed to provider, (5) Response normalized. Each "station" is a textured plane with a 6-line code snippet rendered via `troika-three-text`. Camera path follows a gentle S-curve. Hover on each station triggers the shader-displacement to call attention. Cap at 5 stations (not 20+) so the scene loads in <2MB. This visually answers "what does Clean actually sit between?" — a question every prospect asks in the first 30 seconds.

## BUILD DIRECTIVE

TARGET: Clean landing page hero + "How it works" section (single continuous scroll, replacing both as one unit on tryclean.ai).

APPROACH: Build a dark-mode (`#0A0A0F` base, single-accent palette) one-shot scroll experience that opens with a Three.js shader-driven "JSON-to-TOON compression" animation in the hero — the user's first scroll literally compresses a 47-line JSON tool schema into a 12-line TOON block as a texture morph (Pattern 1: Marvell Tile & Stone). The `uProgress` shader uniform is scrubbed via GSAP ScrollTrigger over the first 100vh. Immediately below, transition into a curved-camera 3D path through 5 stations explaining the proxy lifecycle (Pattern 5: Immersive Garden), each station rendering an actual code snippet via `troika-three-text`. Frame the whole experience in oversized Instrument Serif italic for the headline word "Context" (Pattern 4: T11) and JetBrains Mono for everything else. Add a magnetic CTA button ("Get an API key") with 80px radius lerp (Pattern 4). Keep the entire experience under 4MB initial JS, KTX2-compress all textures, and provide a `prefers-reduced-motion` fallback that swaps the WebGL canvas for a 3-frame static comparison (JSON → TOON → context budget bar).

STACK:
- next@14.2 (App Router) + react@18.3
- three@0.165 + @react-three/fiber@8.16 + @react-three/drei@9.108
- @react-three/postprocessing@2.16 (subtle Bloom only)
- troika-three-text@0.49 (in-canvas code rendering)
- gsap@3.12 + ScrollTrigger + CustomEase (Club GSAP for SplitText)
- lenis@1.1
- split-type@0.3
- tailwindcss@3.4 + tailwindcss-animate
- framer-motion@11.3 (DOM overlay micro-interactions only)
- @fontsource-variable/instrument-serif + @fontsource/jetbrains-mono
- @gltf-transform/cli (build-time KTX2 + Draco compression)
- maath@0.10 (damp/lerp helpers)

REFERENCE URLS:
- https://www.awwwards.com/sites/marvell-tile-stone (texture-morph + ScrollTrigger pinning)
- https://www.awwwards.com/sites/immersive-garden-celebrating-10-years (curved camera path through stations)
- https://www.awwwards.com/sites/fourmula-ai (dark-mode AI product palette + autoplay product clips)

KEY CONSTRAINTS: Clean is developer infrastructure — every visual decision must read as "engineered," not "art-directed." Specifically AVOID: (1) any cursor-trail or particle decoration, (2) the conic-gradient blurred orb pattern (overused this quarter, signals "AI startup that doesn't ship"), (3) bayer-dither or any single-effect-as-brand move (too costume-y for infra), (4) seamless ambient electronic music or sound on scroll, (5) glassmorphism panels or any frosted-blur card (signals 2023 dashboard aesthetic), (6) >200ms scroll lag from over-eager Lenis settings, (7) any animation that does not tie to a real product concept (every motion must communicate something about Clean's actual function — compression, routing, normalization). Hard performance budget: LCP <1.8s on Moto G Power, TBT <150ms, CLS 0. Hard accessibility budget: full keyboard navigation, all WebGL has reduced-motion fallback, headline contrast ≥7:1.
