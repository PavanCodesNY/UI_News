## IZANAMI — https://izanami-official.com/ (Awwwards SOTD July 19, 2026 — baqemono.inc / Tomoyuki Nakata)

WHAT IT DOES: Full-viewport WebGL fluid simulation (curl-noise flow field on a screen-space quad) sits behind sharp serif typography, with GSAP-driven scroll orchestration binding six named beats: `fluid`, `transition`, `parallax`, `navigation`, `scrolling`, `sticky`. Section transitions are handled by a persistent WebGL canvas whose fragment shader morphs between fluid domains as scroll-progress uniforms cross thresholds; DOM sections are absolutely positioned above and driven by `ScrollTrigger.pin` for stickiness with sub-pixel `y` translation. Palette is strictly two colors — #0A0801 near-black and #D9D7D4 warm bone — with typography carrying 100% of the hierarchy.

PACKAGES:
- `three` (0.166+), `gsap` (3.12+), `@gsap/react`, `gsap/ScrollTrigger`, `gsap/SplitText` (Club GreenSock)
- `lenis` (1.1+) for smooth-scroll normalization
- `next` (14 App Router) or Nuxt 3 as shell (Figma-first workflow, either fits)
- Fonts: a Japanese-Latin dual-script serif (looks like a custom Noto Serif JP + Editorial New pairing)
- `postprocessing` (0.6+) for a mild `EffectComposer` chromatic aberration + noise pass
- `simplex-noise` (4.x) as a JS fallback for CPU-side generators

IMPLEMENTATION:
1. Root layout renders a fixed `<canvas>` at z-index -1 covering the viewport. Mount a single `THREE.WebGLRenderer` with `alpha:true`, `powerPreference:'high-performance'`. Use one `OrthographicCamera` (-1..1 both axes) and one `PlaneGeometry(2,2)` covering the frame.
2. Fragment shader: 3-octave curl-noise flow field. Uniforms `uTime`, `uScroll` (0..1), `uMouse` (vec2), `uSection` (0..5 float, GSAP-tweened during section handoffs), `uTint` (vec3 mapping 0..1 between the two palette colors). Advect a base gradient with the flow field; add a subtle grain via `hash21(gl_FragCoord.xy)`.
3. Scroll: `Lenis` with `lerp:0.08`, `smoothWheel:true`, `wheelMultiplier:1`. Feed `lenis.on('scroll', ({progress})=>uScroll.value=progress)`. Wrap in `ScrollTrigger.scrollerProxy`.
4. Sections: each section is a full-viewport `<section>` with `ScrollTrigger.create({pin:true, pinSpacing:true, start:'top top', end:'+=100%', onUpdate:self=>gsap.to(uSection,{value: sectionIndex + self.progress, duration:0.4, ease:'power2.out'})})`.
5. Typography animation: `SplitText` into chars; `gsap.from(chars,{yPercent:110, stagger:0.012, duration:0.9, ease:'expo.out'})` triggered per-section on enter, reverse on leave-back.
6. Cursor: track `pointermove`, tween `uMouse` with `gsap.quickTo` at 0.15s ease, feed into shader for a low-strength radial push on the flow field.
7. Performance: cap DPR at `Math.min(devicePixelRatio, 1.75)`, downgrade to a static grainy gradient PNG when `matchMedia('(prefers-reduced-motion: reduce)')` or when `navigator.hardwareConcurrency < 4`.
8. Breakpoints: kill WebGL below 768px; substitute a looped, muted, `playsinline` H.264 24fps 720p video export of the shader (12–18s loop, <1.2MB).

WHY IT WORKS: The 2-color palette forces every visual decision through typographic hierarchy — the eye has nowhere else to go, so scale/weight/leading carry meaning (contrast law + Gestalt figure-ground). The persistent-canvas approach removes the perceptual break between scenes, so the whole page reads as one continuous artifact (continuity heuristic). Motion is semantic: fluid = brand ethos "Wa/harmony," not decoration.

GOOD: Zero perceptual seams between sections; typography-first hierarchy proves you don't need color to build depth.
BAD: 800ms+ FCP on mid-tier Android because the shader compiles synchronously; JP serif fallback stack causes text-swap flash if `font-display:swap` isn't tuned.

CLEAN APPLICATION: Apply this to Clean's landing page hero (tryclean.ai). Replace the fluid shader with a data-flow shader: a low-contrast grid of moving nodes/edges (representing lead records flowing through Clean's Huginn pipeline), advected by scroll-progress. Keep the 2-color palette (Clean's dark surface + a single accent). Pin the hero for one viewport of scroll, cross-fade a shader "state" uniform from `discover → enrich → route`. This is the anti-generic AI hero: no glowing orbs, no orbiting logos, no cursor-following gradient — a purpose-built shader that literally depicts what the product does.

## Hiroto Sato — https://www.hirotos.com (Awwwards SOTD July 17, 2026, Developer Award)

WHAT IT DOES: R3F portfolio built around three signature interactions: (1) "Sticker Interaction" — hero object is a stack of physicalized 3D stickers (rounded planar meshes with normal-map bump) that peel and re-stack under pointer velocity via a spring-driven Rapier body; (2) "Gallery" — horizontal scroll-hijack of 3D project cards on a curved rail using `CurveModifier`, each card is a plane sampling a video texture, cards distort toward the cursor via vertex-shader displacement; (3) "Page Transition" — full-viewport WebGL mask reveal (radial-gradient alpha threshold sweep, 900ms `power3.inOut`) synchronized with route change. Portfolio artwork is baked in Blender (glTF + KTX2), lit with a single HDRI.

PACKAGES:
- `three` (0.166+), `@react-three/fiber` (8.17+), `@react-three/drei` (9.114+), `@react-three/rapier` (1.5+)
- `three-mesh-bvh` (0.7+) for pointer picking against complex meshes
- `gsap` + `ScrollTrigger` + `Observer` (for touchpad horizontal hijack)
- `motion` (formerly Framer Motion) 11 for DOM overlay transitions
- `ktx2-loader` + `meshoptimizer` for compressed glTF
- Next.js 15 App Router with `next/font` for a Latin+JP display serif pair

IMPLEMENTATION:
1. Global `<Canvas gl={{ antialias:false, powerPreference:'high-performance' }} dpr={[1, 1.75]} shadows={false}>`; wrap in `<Suspense>` with a WebGL skeleton (single flat color, no spinner).
2. Sticker hero: 3–5 rounded `PlaneGeometry(1.5, 2, 32, 32)` instances stacked with 3° yaw jitter. Each uses a `MeshPhysicalMaterial` with `sheen`, `clearcoat:0.4`, and an author-supplied normal map. Attach `RigidBody` (type: `dynamic`, `restitution:0.2`, `linearDamping:2`). On pointer enter+move, apply `applyImpulse` proportional to pointer velocity vector projected onto the sticker plane.
3. Gallery: `CurveModifier` bends a `PlaneBufferGeometry(1.2, 0.75, 40, 40)` along a `CatmullRomCurve3` running left→right. `Observer.create({ target:window, type:'wheel,touch', onChange: e => scrollX.current += e.deltaX * 0.6 })`. Video textures: `THREE.VideoTexture` sourced from `<video>` tags — pause off-screen via IntersectionObserver.
4. Page transitions: full-screen `<ShaderMaterial>` plane at z-index over DOM. Uniform `uProgress` tweened `0→1→0` around Next.js route change (intercept via `router.events`). Fragment: `float mask = smoothstep(uProgress - 0.05, uProgress + 0.05, length(uv - 0.5) * 1.414);` outputs alpha.
5. Assets: run every `.glb` through `gltf-transform optimize --texture-compress ktx2 --simplify 0.6`. Target <2MB per hero mesh, <500KB per gallery card.
6. Motion tokens: `duration: 0.9s`, `ease: [0.16, 1, 0.3, 1]` (custom cubic bezier) — used consistently for every reveal, transition, and pointer-driven spring.

WHY IT WORKS: Physicalized objects trigger the "graspability" heuristic — the brain treats responsive 3D as a real object and forgives visual noise it would reject in flat UI. Consistent motion tokens across every interaction create a "motion language" — the site feels authored, not assembled. Route transitions that fill the viewport hide the FCP of the next page, so navigation feels instant even when it isn't.

GOOD: Motion consistency is textbook — one easing curve, one duration base, everywhere.
BAD: Rapier + BVH + KTX2 loaders push bundle past 1.4MB gzipped; TTI on 3G is >7s.

CLEAN APPLICATION: Apply this to Clean's product-demo section on the landing page. Build a "Sticker Interaction"-style stack of 3D lead-record cards (each card = one enriched prospect: name, title, company logo, signals) that the visitor can grab/toss with the cursor. On flick, the top card animates through a Huginn pipeline (discover → enrich → score → route) with the same 900ms `[0.16,1,0.3,1]` ease. Reuse the page-transition mask for navigating between demo tabs (Growth Scaler, Revenue Scaler, Knowledge Base Engine). This turns "here's what our product does" from a static screenshot into an object the visitor has already touched.

## House of Honey — https://www.houseofhoney.com/ (Awwwards SOTD July 15, 2026, Developer Award — Edoardo Lunardi + Eva Landaluce)

WHAT IT DOES: Editorial architecture portfolio built on Next.js App Router + Sanity, with `Lenis` normalized scroll and Mux-powered video-first storytelling. Grid pages ("Spaces," "Themes") use a masonry-style CSS Grid where each cell holds an autoplaying Mux HLS clip that upgrades in resolution on hover; typography is a display serif with a monospace label pair. Contact page has a full-viewport interactive form where fields expand inline with a magnetic focus ring driven by pointer distance.

PACKAGES:
- `next` (15), `@sanity/client` (6.x), `next-sanity`
- `tailwindcss` (v4), `@tailwindcss/typography`
- `@mux/mux-player-react` or `hls.js` + custom `<video>` orchestrator
- `lenis` (1.1+)
- `motion` (11) for DOM micro-interactions
- Fonts: display serif (Editorial New / Reckless) + mono (JetBrains Mono / Söhne Mono)
- `sharp` for build-time poster generation

IMPLEMENTATION:
1. Layout shell: `<html>` with `Lenis` mounted in a top-level client component. `wrapper` prop wraps content. `wheelMultiplier: 1`, `lerp: 0.1`, `smoothTouch: false` (native inertial scroll on iOS is better than smoothed).
2. Grid: `grid-template-columns: repeat(12, minmax(0,1fr))`, cells span 4/6/8 columns with `grid-auto-rows: minmax(200px, auto)`. Each cell is a `<Link>` wrapping a `<MuxVideo>` with `preload="none"`, `muted`, `loop`, `playsInline`, `autoPlay` — but gated behind an `IntersectionObserver` (only autoplay when >30% visible).
3. Hover state: on `pointerenter`, swap Mux playback quality via `muxPlayer.currentPlaybackQuality = 'high'`; on `pointerleave`, back to `'medium'`. Overlay a caption block that animates in with `motion` — `initial:{y:8, opacity:0}`, `animate:{y:0, opacity:1}`, `transition:{duration:0.35, ease:[0.22,1,0.36,1]}`.
4. Contact form: each `<input>` sits in a wrapper measuring pointer distance via `useMotionValue`. Focus ring is a `motion.div` with `boxShadow` interpolated by `useTransform(distance, [0, 240], ['0 0 0 0 rgba(0,0,0,0)', '0 0 0 2px rgba(black, 0.85)'])`. Labels use a fluid `clamp(14px, 1.4vw, 22px)`.
5. Sanity schema: `space`, `theme`, `journal`, `person` — each with a Portable Text body + a Mux video reference. Use `next-sanity` `sanityFetch` with `revalidateTag` per document.
6. Perf: `next/image` with `sizes` per column span. `dynamic(() => import('./LenisProvider'), { ssr:false })`. Ship no motion library on the server render for the grid page; hydrate only cells in viewport.
7. Deploy on Vercel with `Edge Runtime` for the CMS-driven routes; Mux handles CDN for video.

WHY IT WORKS: Video-as-primary-content forces visual pacing to match content rhythm (each cell is a beat). Serif+mono type pair borrows from print editorial — communicates authorship and taste (semiotic signaling). Magnetic focus ring is subtle enough that most users won't consciously notice it — which is exactly why it registers as "quality" (satisficing threshold for polish).

GOOD: Video-first grid with quality-swap-on-hover is a genuinely under-used pattern; feels premium without gimmick.
BAD: Contact page's magnetic focus ring is invisible on touch — no fallback micro-interaction; keyboard users get nothing.

CLEAN APPLICATION: Apply this to Clean's docs/case-study index page. Turn "Customer Stories" into a Mux-powered grid where each cell autoplays a 6–10s loop of the customer's dashboard being used (recorded via Loom, transcoded to Mux). Hover upgrades to HD, click routes to a full case study built in Sanity. This positions Clean's customers as *characters* in a portfolio rather than logos in a wall — proof-by-motion, not proof-by-logo-graveyard. Reuse Lenis + the exact Motion easing `[0.22, 1, 0.36, 1]` sitewide for consistency.

## CoffeeTech® — https://www.awwwards.com/sites/coffeetech-r (Awwwards SOTD July 16, 2026, Developer Award — Or Halevi / MAKETHECUT®)

WHAT IT DOES: Product-marketing site for a coffee-tech brand built on Webflow + GSAP + custom JS. Hero is a rotating 3D bean-shaped mesh (loaded as glTF) that responds to scroll velocity — spinning faster as the user scrolls, damping with inertia when they stop. Section transitions use color-mask sweeps between the two palette colors (#C8603D burnt-orange and #0D0E13 near-black). Product-spec sections use "sticky column" pattern: image column pins while feature-text column scrolls, with GSAP-driven number counters and SVG line-draw diagrams.

PACKAGES:
- Webflow as CMS/shell
- `gsap` (3.12+), `ScrollTrigger`, `ScrollSmoother` (Club GreenSock)
- `three` (0.166+) loaded via Webflow custom code embed
- `@splinetool/runtime` (optional, if the 3D bean was authored in Spline instead of Blender)
- `lottie-web` (5.12+) for micro-icon animations
- Fonts: geometric sans (Neue Haas Grotesk / Inter Display) + a serif accent for pull quotes

IMPLEMENTATION:
1. Webflow shell handles layout/CMS; drop a global `<script>` embed for the 3D + GSAP layer. Use Webflow's `data-` attributes to hook JS into designer-authored elements (`data-3d="bean"`, `data-count-to="1250"`).
2. 3D hero: init `THREE.WebGLRenderer` into a fixed absolute-positioned canvas. Load `bean.glb` (target <300KB after `gltf-transform optimize`). Animate `mesh.rotation.y += (velocity * 0.001)` where `velocity` is derived from `ScrollTrigger.getVelocity() / 1000`, damped with `gsap.to(velocity, {value:0, duration:1.2, ease:'power2.out'})` on scroll end.
3. Section transitions: full-viewport SVG mask (`<clipPath>` with an animated `d` attribute or a `<rect>` with animated `x`) driven by `ScrollTrigger` `onEnter`. GSAP tweens `clipPath` from `inset(100% 0 0 0)` to `inset(0 0 0 0)` over 700ms `power4.inOut`.
4. Sticky specs: `.spec-row` uses `ScrollTrigger.create({trigger, start:'top top+=80', end:'bottom bottom-=80', pin:'.spec-row__media'})`. Numbers count via `gsap.to({v:0}, {v: dataCountTo, duration:1.2, ease:'power2.out', onUpdate: function(){ el.textContent = Math.round(this.targets()[0].v).toLocaleString() }})`.
5. SVG line-draw diagrams: `<path stroke-dasharray>` equal to `pathLength`, animate `stroke-dashoffset` from `pathLength` to `0` over 900ms `power2.inOut` on `ScrollTrigger` enter.
6. Palette locked to two colors + one neutral; every hover/focus state uses opacity or scale, never a third color.

WHY IT WORKS: Scroll-velocity-driven 3D creates a haptic loop — visitor learns "my input directly moves this object" within the first 500ms, which builds engagement (operant conditioning micro-loop). Pinned specs with counters are the industrial-design equivalent of a magazine spread — reader-controlled pacing, one idea per pin. Two-color palette + scroll-mask transitions makes the whole site feel like one continuous motion graphic.

GOOD: Scroll-velocity → rotation is a masterful low-cost 3D pattern; almost every product site should be doing this.
BAD: Webflow's compiled JS bundle plus custom Three.js + GSAP Club = 400KB+ before content; on flaky wifi the hero canvas paints blank for 1.5–2s.

CLEAN APPLICATION: Apply this to Clean's pricing page + feature-comparison section. Use the scroll-velocity 3D pattern on a Clean product icon (or the Huginn spiral) in the pricing hero — makes the driest page (pricing) feel alive. Use the pinned-column + counter pattern for the "By the numbers" strip: `Leads enriched: 4.2M+`, `Avg. enrichment latency: 380ms`, `CRM writes per second: 240` — counters trigger on scroll, spec media pins. Palette lock to Clean's dark + single accent; kill any third color in hover states.

## shadcn/ui — Rhea style + Chat primitives (July 2026 release — https://ui.shadcn.com/docs/changelog/2026-07-base-ui-default)

WHAT IT DOES: shadcn/ui shipped two structural changes in July 2026: (1) Base UI is now the default primitive layer (Radix still supported via `--base radix`) — meaning components use accessible, unstyled primitives from `@base-ui-components/react` under the hood; (2) new `Rhea` style variant, and a set of chat primitives — `MessageScroller`, `Message`, `Bubble`, `Attachment`, `Marker` — designed for building agentic-AI conversation UIs. Rhea keeps shadcn's rounded/soft aesthetic but adds denser surfaces, ring-based focus states, and subtle press-transform on buttons.

PACKAGES:
- `shadcn` CLI (latest), init with `npx shadcn@latest init --base ui --style rhea`
- `@base-ui-components/react` (primitives, replacing Radix as default)
- `tailwindcss` v4 with the shadcn preset
- `motion` (11) for chat message enter/exit
- `lucide-react` for icons
- `class-variance-authority`, `tailwind-merge`, `clsx`
- Optional: `@ai-sdk/react` (Vercel AI SDK v5) if you're wiring to an LLM stream

IMPLEMENTATION:
1. Bootstrap: `npx shadcn@latest init --style rhea` in an existing Next 15 app. Confirm `--base ui` (Base UI) unless you have a Radix dep you must keep.
2. Install chat primitives: `npx shadcn@latest add message-scroller message bubble attachment marker`.
3. `MessageScroller` composition:
```
<MessageScroller className="h-full">
  <MessageScroller.Anchor />
  {messages.map(m => (
    <Message key={m.id} role={m.role}>
      <Bubble>{m.content}</Bubble>
      {m.attachments?.map(a => <Attachment key={a.id} src={a.src} type={a.mime}/>)}
      {m.marker && <Marker>{m.marker}</Marker>}
    </Message>
  ))}
  <MessageScroller.AutoScroll />
</MessageScroller>
```
4. `Rhea` visual tokens: denser radii (`--radius: 0.625rem`), focus ring is `outline-none ring-2 ring-ring/60 ring-offset-2 ring-offset-background`, buttons get `active:translate-y-[0.5px] active:shadow-inner`.
5. Streaming: wire to `@ai-sdk/react` `useChat({ transport: new DefaultChatTransport({ api: '/api/chat' }) })`. `Bubble` supports incremental text via a `stream` prop that consumes an `AsyncIterable<string>`.
6. Motion: each `<Message>` mounts with `motion.div initial={{opacity:0, y:6}} animate={{opacity:1, y:0}} transition={{duration:0.18, ease:[0.22,1,0.36,1]}}`. Keep enter-animation duration under 200ms — anything longer feels laggy in a chat UI.
7. Marker component: perfect for tool-call badges — render `<Marker variant="tool">web.search</Marker>` inline with a message when the agent invokes a tool.

WHY IT WORKS: Base UI as default fixes shadcn's accessibility ceiling (Radix has known gaps for `Combobox` and `Tree`). Chat primitives baked into the design system means every AI product using shadcn now speaks the same visual language — this is a massive convergence event. Rhea's denser radii + ring focus solves the "shadcn looks like every other AI app" complaint by adding just enough distinctness while preserving muscle memory.

GOOD: First component library to ship dedicated chat primitives — moves the whole ecosystem forward; agentic UIs now have a shared vocabulary.
BAD: Rhea's press-transform is 0.5px — invisible on Retina but jitters on 1x displays; ring-based focus is beautiful but fails WCAG 2.2 non-text contrast at low `--ring/60` opacity on light themes.

CLEAN APPLICATION: Apply this to Clean's in-product agent conversation view (the surface where users chat with Huginn to request leads/enrichments/reports). Rip out any custom chat components and adopt shadcn's `MessageScroller`/`Message`/`Bubble`/`Attachment`/`Marker` verbatim. Use `<Marker variant="tool">` to expose every tool call Huginn makes — `clean.find_prospects`, `clean.enrich_contact`, `clean.add_leads_to_crm` — so users see the agent's reasoning as it happens. Adopt Rhea globally so Clean's docs, dashboard, and marketing site share the same visual token base. Bonus: initialize with `--base ui` so Clean picks up accessibility improvements for free on Combobox-heavy surfaces like the lead-filter builder.

## BUILD DIRECTIVE

TARGET: Clean landing-page hero + immediately-below product-demo section (tryclean.ai).

APPROACH: Rebuild the fold as a single persistent WebGL canvas (IZANAMI's pattern) running a data-flow shader that literally depicts what Clean does — a low-contrast field of moving nodes/edges advected by scroll-progress, cross-fading through `discover → enrich → score → route` states. Below the fold, drop into a Hiroto-style physicalized 3D lead-record card stack that the visitor can toss/flick with the cursor, where a flick animates the top card through the Huginn pipeline with a consistent 900ms `[0.16, 1, 0.3, 1]` ease. Pin the hero for one viewport of scroll (IZANAMI-style ScrollTrigger.pin) so the shader-state transition and the card-stack introduction feel like one continuous handoff. Palette locked to Clean's dark surface + one accent (2 colors, no third). Typography carries the hierarchy — no gradient text, no glowing badges. Cursor drives one uniform (`uMouse`) and one Rapier impulse; that's it.

STACK:
- `next` 15 (App Router), `react` 19
- `three` 0.166+, `@react-three/fiber` 8.17+, `@react-three/drei` 9.114+, `@react-three/rapier` 1.5+
- `gsap` 3.12+ (Club GreenSock: `ScrollTrigger`, `ScrollSmoother`, `SplitText`, `Observer`)
- `lenis` 1.1+
- `motion` 11 (for DOM overlays / route transitions)
- `tailwindcss` v4 with shadcn Rhea preset (adopt sitewide for consistency with the in-product view)
- `@base-ui-components/react` (via shadcn `--base ui`)
- `postprocessing` 0.6+ (light chromatic aberration + noise pass only)
- `three-mesh-bvh` for card pointer picking
- Fonts: pair a technical mono (JetBrains Mono / Söhne Mono) with a neutral display sans (Inter Display / Neue Haas Grotesk) — no serif, this is dev infrastructure not editorial

REFERENCE URLS:
- https://izanami-official.com/ — persistent-canvas + scroll-state shader handoff
- https://www.hirotos.com — physicalized 3D interaction + 900ms `[0.16,1,0.3,1]` motion token
- https://www.awwwards.com/sites/coffeetech-r — scroll-velocity → 3D rotation, pinned counter specs (secondary reference for the "by the numbers" strip below the hero)

KEY CONSTRAINTS (Clean is developer infrastructure — must feel technically sophisticated and fast):
- DO NOT ship glowing orbs, orbiting logos, gradient meshes, or cursor-following blob gradients. Every AI startup ships these. They read as "generic AI landing page" and actively hurt trust with a technical buyer.
- DO NOT ship a third color. Two-color palette is the discipline that separates award-winning from templated.
- DO NOT ship the WebGL canvas without a `<video>` fallback for mobile <768px and `prefers-reduced-motion`. TTI on 3G must stay under 3.5s.
- DO NOT ship marketing-copy hero text ("Supercharge your growth with AI"). The visual is doing the work — the copy should be a single technical sentence naming the product's actual mechanism (e.g., "Continuous prospect discovery + enrichment for your CRM. 380ms average enrichment latency, 4.2M records live.").
- DO NOT introduce a new motion easing curve mid-page. One base ease (`[0.16, 1, 0.3, 1]`), one duration base (900ms with 200ms/450ms/900ms variants), everywhere.
- MUST bundle-split the WebGL/Rapier layer as a dynamic import so the LCP hero paints from server-rendered typography while the shader hydrates below the LCP threshold.
- MUST cap `devicePixelRatio` at 1.75 and disable the shader entirely for `hardwareConcurrency < 4` or `deviceMemory < 4`.
- MUST reuse the same `[0.16, 1, 0.3, 1]` ease and Rhea design tokens in the in-product surface so the landing page and app feel like one product, not two teams.
