## Dragonfly Redux — https://www.awwwards.com/sites/dragonfly-redux

WHAT IT DOES: Long-scroll marketing site for a crypto investment fund built by Studio Freight (SOTD 07-22, jury 7.39). Two-color system (#FA4C14 orange on #000000 black), full-viewport type slabs, a WebGL scene layered behind DOM that stays in sync via Lenis-lerped scroll values driving GSAP ScrollTrigger scrub. Signature move is a `clip-path` "fold" reveal between sections: as you scroll, the leaving section clips inward while the entering section clips outward from the same seam, creating a theatrical page-turn feel without any real page navigation. Sticky positioning holds the WebGL canvas at 100vh while native scroll indexes the timeline.

PACKAGES:
- next@15 (React 19, App Router)
- three@0.169
- @react-three/fiber@8
- @react-three/drei@9
- lenis@1.1 (formerly @studio-freight/lenis)
- gsap@3.12 with ScrollTrigger and CustomEase
- @studio-freight/hamo (React hooks for lifecycle + scroll)
- tempus (rAF loop unification, from Studio Freight)
- vue@3 for the marketing shell (they went Vue; a Next equivalent is fine)
- postcss + a Satoshi/Neue Haas Grotesk pairing (self-hosted via next/font)

IMPLEMENTATION:
1. Root layout mounts a fixed `<Canvas>` at `position: fixed; inset: 0; z-index: 0; pointer-events: none` and a `<main style="position: relative; z-index: 1">` for DOM sections.
2. Wrap in a `<Lenis root>` provider. Configure `lenis = new Lenis({ duration: 1.2, easing: t => Math.min(1, 1.001 - Math.pow(2, -10 * t)), smoothWheel: true, lerp: 0.1 })`. Drive `gsap.ticker.add(t => lenis.raf(t * 1000))` and `gsap.ticker.lagSmoothing(0)`.
3. Register ScrollTrigger with `scrollerProxy(document.documentElement, { scrollTop(v){ return arguments.length ? lenis.scrollTo(v, {immediate: true}) : lenis.scroll } })` then `ScrollTrigger.defaults({ scroller: document.documentElement })`.
4. For each section, create a ScrollTrigger with `scrub: 1.2, start: 'top bottom', end: 'bottom top'` that maps `self.progress` to a `useRef` value read inside R3F's `useFrame` — this is the DOM→WebGL bridge Studio Freight is known for.
5. The fold: two adjacent sections both use `clip-path: polygon(0 0, 100% 0, 100% var(--fold), 0 var(--fold))` and its inverse, animating `--fold` from 100% → 0% on the leaving one and 0% → 100% on the entering one, tied to the same scrub timeline. Use `will-change: clip-path` and hardware-accelerate the containing block.
6. Type: display sizes clamped `font-size: clamp(64px, 12vw, 220px); line-height: 0.9; letter-spacing: -0.03em; font-weight: 500`. Keep line-height tight so the type behaves like a block.
7. Color tokens: `--bg: #000; --fg: #FA4C14; --muted: rgba(250,76,20,0.6)`. Use `mix-blend-mode: difference` on the cursor so it inverts against both bg and type.
8. Performance: `<Canvas dpr={[1, 1.75]} gl={{ antialias: false, powerPreference: 'high-performance' }} frameloop="demand"` and call `invalidate()` from ScrollTrigger's `onUpdate` — do not render 60fps when idle.
9. Respect `prefers-reduced-motion: reduce` by disabling Lenis (`lenis.stop()`), replacing the fold with instant section swaps, and unmounting the Canvas.

WHY IT WORKS: The two-color palette forces every design decision through pure hierarchy — size, spacing, and motion do all the work no color can do. The fold is a motion-semantic trick: it makes each section feel like an intentional act of composition rather than a passive scroll, which sells "we curate carefully" without any copy claiming it. Lenis-synced WebGL removes the classic "js scroll drifts from browser scroll" nausea; the whole page feels like one substance.

GOOD: Two-color discipline is genuinely rare and expensive to pull off; the DOM/WebGL sync is textbook. BAD: Fold effects tank scroll performance on mid-tier Android and break `Cmd+F` find-in-page over hidden text if not scoped carefully.

CLEAN APPLICATION: Apply this to Clean's landing page hero-to-product transition by building a single 200vh section that pins on scroll, cross-folding from the hero headline ("Your entire go-to-market, on autopilot") into the product surface (Huginn agents animating leads across a table). Keep Clean's palette to two tokens for this zone — `--ink: #0A0A0A` and `--signal: #FF5A1F` — and let the fold do the visual heavy lifting instead of adding a screenshot mockup. Route it through Lenis so the CRM demo animation inside can scrub in sync.

## Lacoste — Polo Factory — https://www.awwwards.com/sites/lacoste-polo-factory

WHAT IT DOES: Product-as-experience configurator (SOTD 07-22, Developer Award). Full-screen 3D polo shirt rendered in real time; user picks fabric, collar color, stitching, and monogram, and the model updates instantly with materials swapping via texture atlases. Camera dollies on scroll through six "factory station" chapters (yarn, knit, dye, cut, embroider, ship). Interaction pattern: sticky right rail holds the customization controls; center holds the model; left holds descriptive copy per chapter. No page reloads — one canvas, one story.

PACKAGES:
- three@0.169
- @react-three/fiber@8
- @react-three/drei@9 (useGLTF, useTexture, PresentationControls, MeshTransmissionMaterial, Environment)
- @react-three/postprocessing@2 (subtle Bloom + N8AO ambient occlusion)
- gsap@3.12 + ScrollTrigger for camera keyframes
- leva@0.9 (used in dev; strip for prod)
- meshoptimizer + gltf-transform for asset pipeline (draco + KTX2)
- zustand@4 for the configurator state (color, fabric, monogram)
- react-use-measure@2

IMPLEMENTATION:
1. Ship one `polo.glb` (~1.2MB after Draco + KTX2). Split materials into named slots: `Body`, `Collar`, `Cuff`, `Stitch`, `Croc`, `Monogram`. Textures 1024², BC7/KTX2 compressed.
2. `useGLTF` the model; iterate meshes and inject material refs into a zustand store: `useConfigurator = create(set => ({ collarColor: '#0A5C3E', setCollarColor: c => set({collarColor: c}) }))`.
3. In `useFrame`, `materialRefs.collar.color.set(collarColor)` — direct mutation, no re-render.
4. Camera choreography: build a GSAP timeline with six `to()` steps hitting scroll-mapped labels. Each label sets `cameraRef.position` and `controlsRef.target` with `ease: 'power2.inOut', duration: 1`. Pin the canvas: `ScrollTrigger.create({ trigger: '.factory', start: 'top top', end: '+=600%', pin: true, scrub: 1 })`.
5. Right-rail controls: HTML overlay (not Drei's `<Html>`) at `position: sticky; top: 0; right: 0; width: 360px; height: 100vh`. Each swatch is a `<button aria-pressed>` — accessible, keyboard-navigable, and mirrors to the model via zustand.
6. Monogram: render user text into a 512×512 offscreen `<canvas>` via `CanvasTexture`, then assign to the monogram material's `map`. Debounce input at 120ms.
7. Lighting: `<Environment preset="studio" intensity={0.6} />` plus one directional key light with `shadow-mapSize={2048}` on the polo only. Contact shadows via `<ContactShadows blur={2.5} opacity={0.4} />`.
8. Fallback: detect `!gl` or `matchMedia('(max-width: 768px)')` and serve a static hero image with a "Configure on desktop" CTA. Never crash mobile.

WHY IT WORKS: The single-canvas continuity is the whole trick — because you never leave the scene, every configuration change reads as a consequence of your choice, not a page reload. It converts "browsing" into "making," which is the psychological shift luxury brands charge premiums for. Sticky-rail with center-focal-object is a Gestalt figure-ground move: the product is always the figure, controls are always the frame.

GOOD: Real-time material swap with zero jank is production-grade R3F work. BAD: 1.4MB initial payload is ambitious for 4G; the mobile fallback effectively hides the product from half the audience.

CLEAN APPLICATION: Apply this to Clean's product demo section by building a single pinned canvas that shows the Huginn agent workspace with three "stations" — Prospect → Enrich → Send. As the user scrolls, camera pans across a schematic of the pipeline (not a literal 3D scene — use SVG on a canvas or a WebGL 2D scene with instanced quads for lead cards). Controls on the right let the user toggle "Add ICP filter" or "Enable auto-outreach" and the animation updates in-place. One scene, three chapters, no page reloads.

## Lama Lama — https://www.awwwards.com/sites/lama-lama-2

WHAT IT DOES: Design agency portfolio (SOTD 07-20, Developer Award, 7.64 design score). Cream-on-charcoal palette (#F9F4EB / #1A1C1C), oversized editorial serif for headlines against neo-grotesk for UI. Cursor is a small circle that morphs into a large disc labeled "VIEW" when hovering project cards, driven by React state + spring physics. Project grid is a bento layout where each cell has a WebGL displacement hover effect (image warps toward the cursor, then relaxes on leave).

PACKAGES:
- next@15
- @react-three/fiber@8 + drei@9 (per-card canvases, or one shared canvas with `<View>` from drei)
- three@0.169
- gsap@3.12
- lenis@1.1
- framer-motion@11 (renamed to `motion@11` in some builds — use latest)
- next/font for Playfair Display or Söhne
- clsx + tailwindcss@4

IMPLEMENTATION:
1. Custom cursor: `<div id="cursor">` fixed to viewport, `pointer-events: none`, animated with `motion.div` using `useSpring` on `x, y` with `stiffness: 500, damping: 40, mass: 0.5`. Hover state expands `scale` from 1 → 6 and morphs `borderRadius: '50%' → '20%'` with `background: --fg`. Text inside fades in on hover.
2. Bento grid: CSS `grid-template-columns: repeat(12, 1fr); grid-auto-rows: 8vh; gap: 24px`. Assign `grid-column: span 6/8/4` per card manually — the deliberate irregularity is the aesthetic.
3. Per-card hover displacement: use drei's `<View track={ref}>` so N cards share one canvas. Each `<View>` renders a `<PlaneGeometry>` with a `ShaderMaterial` doing `vec2 offset = mix(vec2(0.), uMouse - vUv, 0.25 * hoverStrength); gl_FragColor = texture(uTex, vUv + offset * 0.1);`. Animate `hoverStrength` 0→1 with GSAP on `mouseenter/leave`, duration 0.6, ease `power3.out`.
4. Type system: `--display: 'Playfair Display', serif; --sans: 'Söhne', system-ui`. Hero: `font-size: clamp(72px, 10vw, 180px); font-family: var(--display); font-style: italic` for the accent word, roman for the rest.
5. Reveal animation on scroll: split headlines with `SplitType` or `split-type@0.3`, stagger `y: 40 → 0, opacity: 0 → 1` per word with `stagger: 0.04, ease: 'expo.out'`.
6. Grain: overlay a 400×400 PNG noise at `mix-blend-mode: overlay; opacity: 0.08; position: fixed; inset: 0; pointer-events: none`. Ties DOM and WebGL together visually.
7. Reduced motion: swap the disc cursor for the native cursor, disable displacement, keep the grid.

WHY IT WORKS: The cursor is a proof-of-craft signal — anyone can build a grid; only agencies that care about detail build a cursor with weight. The serif/sans pairing at extreme size contrast creates an editorial rhythm that positions the studio as considered rather than trendy. Warm cream over near-black is the classiest possible contrast without going pure white, which reads as premium print catalog rather than a website.

GOOD: The displacement shader on a shared `<View>` canvas is the correct performance pattern. BAD: Custom cursors have a real accessibility cost — screen readers can't see the label, and touch users get nothing.

CLEAN APPLICATION: Apply this to Clean's case-studies / customer-logos section by building a bento grid where each cell is a customer with a hover displacement on their logo, plus a persistent "VIEW CASE" morphing cursor. Keep Clean's dark theme (#0A0A0A) but borrow the warm off-white (#F5F1E8) as the surface color for the cards — Clean's landing currently has nothing warm on it, and one cream section would create the exact editorial texture that makes B2B SaaS feel human.

## IZANAMI — https://www.awwwards.com/sites/izanami

WHAT IT DOES: Long-scroll storytelling site by baqemono.inc (SOTD 07-18, PRO + Developer Award). Japanese mythology narrative told through a WebGL scene where translucent ink-like textures flow across the screen, driven by a Perlin-noise displacement shader on a fullscreen plane. Text overlays in vertical Japanese typesetting (`writing-mode: vertical-rl`) with fade-in on scroll. Cursor leaves a trailing particle stream via GPU-side instancing.

PACKAGES:
- three@0.169
- @react-three/fiber@8
- @react-three/drei@9 (shaderMaterial helper)
- glslify + glslify-loader for shader imports
- lygia (shader library — noise, curl, matcap helpers)
- gsap@3.12 + ScrollTrigger
- lenis@1.1
- tone@14 (subtle ambient audio — optional, on user gesture only)
- Google Fonts: Noto Serif JP + a custom Latin display

IMPLEMENTATION:
1. Full-screen `<Canvas>` behind DOM. One `<PlaneGeometry args={[2, 2]}>` filling clip space, `<shaderMaterial>` with custom `vertexShader` (passthrough) and `fragmentShader` doing layered `snoise3D` from lygia at three octaves, mixed with a scroll-driven `uProgress` uniform.
2. Ink look: `vec3 ink = mix(vec3(0.02), vec3(0.85, 0.83, 0.78), smoothstep(0.3, 0.7, noise));` — warm off-white on near-black. Add `gl_FragColor.a = pow(noise, 2.2) * uIntensity` to get the fluid alpha edge.
3. Scroll driver: GSAP timeline updates `material.uniforms.uProgress.value` from 0 → 1 across the page. Add a second uniform `uMouse` and lerp toward pointer position in `useFrame`.
4. Cursor particles: instanced `<InstancedMesh count={2000}>` of tiny quads. In `useFrame`, shift a ring buffer index each frame writing current mouse position + random offset into `instanceMatrix`; older instances fade via `alpha = 1 - (age/lifetime)`. GPU handles the transform.
5. Typography: `writing-mode: vertical-rl; text-orientation: mixed; font-family: 'Noto Serif JP', serif; font-size: clamp(18px, 2vw, 24px); line-height: 2.2; letter-spacing: 0.15em`. Latin headlines stay horizontal for contrast.
6. Section reveals: `gsap.fromTo('.section', { opacity: 0, y: 60 }, { opacity: 1, y: 0, scrollTrigger: { trigger, start: 'top 70%' } })`.
7. Audio: gate a Tone.Player behind a "Sound On" toggle in the corner. Never autoplay.

WHY IT WORKS: Vertical typesetting is a semantic move, not just style — it forces the reader to slow down and adopt a different reading posture, which sells the "sacred narrative" positioning. The ink shader is one shader doing all the work; the restraint is the craft. Off-white over near-black at low contrast (not full contrast) feels like a scroll of aged washi paper, not a website — perfect ethos for the subject.

GOOD: Single-shader everything is a masterclass in performance-through-restraint. BAD: Vertical typesetting is unreadable to most non-Japanese audiences without a hover-to-translate affordance, which the site lacks.

CLEAN APPLICATION: Apply this to Clean's docs landing page ("Learn Clean") by using a fullscreen shader background — but adapt the ink texture to a subtle digital-noise field in Clean's signal-orange on ink-black, at 8% opacity so it never fights the type. Use the vertical-typesetting concept for section markers only (spine labels like "01 — PROSPECT" running vertically down the left gutter of each doc page), keeping horizontal reading intact for the body. The ambient shader plus vertical spine tags will make the docs feel like a designed publication instead of a wiki.

## Aceternity UI — Canvas Reveal Effect (Shader Drop) — https://ui.aceternity.com/components/canvas-reveal-effect

WHAT IT DOES: Component (shipped with the recent 07-2026 Motion for React + Tailwind v4 refresh) that renders a dot grid using a custom GLSL fragment shader inside a React Three Fiber `<Canvas>`. On hover, dots animate from center-outward in a staggered ripple, colors interpolate through a passed-in `colors={[[r,g,b], ...]}` array, and a radial-gradient mask fades the edges. Sits inside any card; hover fills the card with the effect. Inspired by Clerk's card hover.

PACKAGES:
- react@19
- three@0.169
- @react-three/fiber@8
- motion@11 (formerly framer-motion)
- tailwindcss@4
- clsx + tailwind-merge (for `cn` helper)
- next@15 (or Vite — framework agnostic)

IMPLEMENTATION:
1. Install via CLI: `npx shadcn@latest add https://ui.aceternity.com/registry/canvas-reveal-effect.json` (Aceternity uses the shadcn registry format now).
2. Component API: `<CanvasRevealEffect animationSpeed={3} containerClassName="bg-black" colors={[[236, 72, 153], [232, 121, 249]]} dotSize={2} />`. Wraps in `<motion.div>` with `AnimatePresence` so it mounts on `whileHover`.
3. Shader concept: pass `uTime`, `uHover`, `uResolution`, `uColors[6]`. In fragment shader, compute `vec2 grid = floor(gl_FragCoord.xy / uDotSize); float dist = length(grid - uResolution/2. / uDotSize); float wave = smoothstep(0., 1., uTime * uAnimationSpeed - dist * 0.05);`. Draw a circle for each grid cell, colored by `mix(uColors[i], uColors[i+1], wave)`.
4. Container pattern: wrap the shader in a parent card with `radial-gradient(circle at center, transparent, black 80%)` as a `::after` overlay to fade edges into the card bg — this is what makes it feel like the dots are "inside" the card, not stamped on top.
5. Trigger: use Motion's `whileHover` with `initial={{ opacity: 0 }}, animate={{ opacity: 1 }}, exit={{ opacity: 0 }}` and `transition: { duration: 0.4 }`. Don't animate the shader itself in and out — animate the wrapper opacity; the shader's ripple handles the visual "reveal."
6. Performance: `<Canvas dpr={1} gl={{ antialias: false, alpha: true }}>` — dpr=1 is fine for dots. Only mount the Canvas on hover, unmount on leave.
7. Accessibility: `role="presentation"; aria-hidden="true"` on the effect. Never put meaningful content inside — it's decoration.

WHY IT WORKS: The dot grid is a perfect information-density surface: dense enough to feel technical, sparse enough to feel calm. Animating from center outward gives the hover a "you triggered this" causal read, which is dopamine-grade micro-feedback. The radial mask edge trick is what separates amateur (rectangular shader block) from professional (soft-edged luminous field).

GOOD: Ships as a copy-paste component with clean props — five minutes to production. BAD: Every AI/dev-tool startup now uses this exact effect, so it reads as "shipped with Aceternity" more than as brand-specific craft unless customized meaningfully.

CLEAN APPLICATION: Apply this to Clean's pricing page tier cards. Wrap each pricing card (Starter / Growth / Scale) in a hovered CanvasRevealEffect with tier-specific colors — Starter: cool cyan `[[0,180,216],[0,150,199]]`, Growth: signal orange `[[255,90,31],[255,140,0]]`, Scale: violet-magenta `[[139,92,246],[236,72,153]]`. The hover ripple emphasizes each tier without needing extra badges or animations. Combine with a subtle upward scale (1 → 1.02) and border-color shift to sell interactivity.

## BUILD DIRECTIVE

TARGET: Clean landing page hero-to-product section (the first two viewports the visitor sees at tryclean.ai).

APPROACH: Build a single pinned 200vh section that opens with a two-color Dragonfly-style headline zone (Clean's ink-black `#0A0A0A` on signal orange `#FF5A1F`, oversized display type at `clamp(72px, 12vw, 200px)`), then folds via a Studio-Freight `clip-path` transition into a Lacoste-style product surface where Huginn is shown running: leads flowing across a three-station schematic (Prospect → Enrich → Send) rendered as a shared R3F `<View>` canvas so we get one WebGL context handling both the fold's background gradient noise and the pipeline animation. Ambient particle trail follows the cursor across the full section (IZANAMI-style instanced quads at low opacity). The three station cards use Aceternity's Canvas Reveal Effect on hover for tier-specific product surfaces. Everything is scrubbed via Lenis + GSAP ScrollTrigger so the section reads as one continuous motion beat, not two pages stapled together.

STACK:
- next@15, react@19, tailwindcss@4
- three@0.169, @react-three/fiber@8, @react-three/drei@9 (`<View>`, `<Instances>`)
- lenis@1.1
- gsap@3.12 + ScrollTrigger + CustomEase
- motion@11 (for card hover states and cursor spring)
- split-type@0.3 (for headline word-stagger reveal)
- lygia (shader noise helpers)
- self-hosted Neue Haas Grotesk Display + JetBrains Mono via next/font
- clsx + tailwind-merge

REFERENCE URLS:
- https://www.awwwards.com/sites/dragonfly-redux (fold transition + two-color discipline)
- https://www.awwwards.com/sites/lacoste-polo-factory (single-canvas continuity + sticky rail)
- https://ui.aceternity.com/components/canvas-reveal-effect (tier card hover treatment)

KEY CONSTRAINTS: Clean is developer infrastructure, so the result must feel like Linear or Vercel — technically sophisticated, fast, and quietly confident — not like a Web3 agency site. Avoid: gratuitous 3D models of anything (no floating logos, no shirt-configurator literalism), auto-playing audio, blur-heavy glassmorphism, rainbow gradient sweeps, chatbot-style "hi how can I help" popovers, and any effect that pushes TTI past 2.0s on a mid-tier laptop. Every animation must respect `prefers-reduced-motion: reduce` by degrading to static states, not disappearing. Mobile must ship a first-class DOM-only version of this section (no WebGL below 768px) — the fold becomes a simple stagger, the pipeline becomes a scrolled list, the CanvasRevealEffect becomes a solid tier color. LCP element must remain the headline text, not the canvas. Ship the whole thing under 180KB gzipped JS on the initial route.
