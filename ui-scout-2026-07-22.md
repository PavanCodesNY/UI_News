## 1. Lacoste — Polo Factory — https://www.awwwards.com/sites/lacoste-polo-factory

WHAT IT DOES: A cinematic product-configurator single-page experience. A GLTF polo model sits center-stage on a fixed WebGL canvas while the DOM scrolls a narrative track behind it; scroll position drives a GSAP ScrollTrigger timeline that (a) tweens the camera along a spline, (b) morphs bake-swapped materials on the shirt for colorway changes, and (c) fires DOM section pins for the "collar / knit / croc badge" chapters. Cursor movement modulates a subtle Perlin-noise displacement on the fabric shader so the shirt "breathes" even when static.

PACKAGES:
- `next@15` (App Router, RSC off on the canvas route)
- `three@0.169`
- `@react-three/fiber@8.17`
- `@react-three/drei@9.114` (useGLTF, Environment, ContactShadows, PerspectiveCamera)
- `@react-three/postprocessing@2.16` (Bloom, DepthOfField, ChromaticAberration ~0.0004)
- `gsap@3.13` + `gsap/ScrollTrigger`
- `lenis@1.1` (smooth scroll driving ScrollTrigger)
- `tailwindcss@4` with `@theme` tokens for the Lacoste greens (#004526, #A3D65C)
- `next/font` loading a licensed grotesk (fallback: Neue Haas Grotesk Display)
- `zustand@4` for cross-component colorway state

IMPLEMENTATION:
1. Layout: `<CanvasStage />` fixed `inset-0 z-0`, `<ScrollNarrative />` positioned `relative z-10` with a total scroll length of ~600vh split into 6 pinned chapters.
2. Canvas: `<Canvas dpr={[1, 2]} gl={{ antialias: true, powerPreference: 'high-performance' }} camera={{ fov: 32, position: [0, 0.4, 3.2] }}>`. Use `Suspense` + a `<Preload all />` so the model, HDRI, and textures are ready before the first ScrollTrigger fires.
3. Camera rig: keep the camera parented to an `Object3D` that walks a `CatmullRomCurve3`; on each `useFrame` sample `curve.getPoint(scrollProgress)` and `curve.getTangent()` for lookAt.
4. ScrollTrigger: one master timeline, `scrub: 1.2`, `snap: { snapTo: 'labels', duration: 0.6, ease: 'power2.inOut' }`. Labels: `intro`, `fabric`, `collar`, `croc`, `color`, `cta`.
5. Color morph: pre-bake 5 albedo textures. On colorway click use `gsap.to(material.uniforms.uMix, { value: targetIndex, duration: 0.9, ease: 'expo.out' })`, blend inside the fragment shader with `mix(texA, texB, uMix)`.
6. Fabric shader: extend `MeshPhysicalMaterial` via `onBeforeCompile` — inject a vertex displacement `pos += normal * cnoise(pos * 8.0 + uTime * 0.15) * 0.004`.
7. DOM typography enters with `SplitText.chars`, `y: 40 → 0`, `stagger: 0.02`, `ease: 'expo.out'`, `duration: 1.1`, keyed to the same ScrollTrigger label.
8. Performance: `frameloop='demand'` outside of scroll, `invalidate()` inside the scrub handler. Cap DPR at 1.5 below 1280px. Ship `.ktx2` textures via `KTX2Loader` + Basis transcoder.
9. Mobile: swap to a 3-tap horizontal snap carousel using CSS scroll-snap; the canvas becomes a single static hero render (pre-baked PNG poster + tiny fallback GLB).

WHY IT WORKS: The fixed-canvas / scrolling-DOM pattern gives the eye a stable subject (Gestalt figure-ground) while the copy delivers narrative — a classic cinema technique. Motion semantics are honest: color changes accelerate then settle (expo.out), matching how we perceive material change in the real world. The 2-color palette (deep green + off-white) hits ~11:1 contrast on primary text, so accessibility survives the drama.

GOOD / BAD: Good — the scroll-locked chapter model turns a product page into a museum tour without ever hiding the CTA. Bad — 600vh scroll length punishes users on trackpads with acceleration off, and the entire experience requires WebGL2 with no meaningful reduced-motion fallback for vestibular users.

CLEAN APPLICATION: Apply this to Clean's **landing-page product demo section** by pinning a canvas that renders a stylized "lead card" 3D object — a floating slab with the prospect's name, company logo, enrichment fields, and a green health-score ring. Scroll chapters: (1) raw lead → (2) auto-profile pass (fields fade in with staggered SplitText) → (3) knowledge-base match (a beam animates from the card to a stack of doc shapes) → (4) outreach fires (a tiny paper-plane geometry launches on scrub label 4). Use the exact same GSAP `snap: 'labels'` config and Lenis stack. Skip the postprocessing bloom — it reads too "marketing site" for infra tools; keep only subtle DoF.

## 2. IZANAMI — https://www.awwwards.com/sites/izanami

WHAT IT DOES: A brand narrative site for a Japanese school built around the concept of "Wa" (harmony). Full-viewport Three.js scenes crossfade as the user scrolls; each scene is a Poisson-disc particle field that morphs from one kanji character to the next using a GPU-computed target-position texture. Kinetic Japanese typography scrolls horizontally in reverse-parallax while a vertical scroll indicator ticks through chapters. GSAP orchestrates the DOM+WebGL sync.

PACKAGES:
- `three@0.169`
- `@react-three/fiber@8.17`, `@react-three/drei@9.114`
- `@react-three/gpgpu` or custom `THREE.WebGLRenderTarget` ping-pong
- `gsap@3.13` + `ScrollTrigger`, `SplitText`, `Observer`
- `lenis@1.1`
- `next@15`
- `tailwindcss@4`
- Fonts: `Shippori Mincho B1` (Japanese) + `Söhne Mono` (Latin)

IMPLEMENTATION:
1. Particle system: 65,536 particles as `THREE.Points`, positions read from a 256×256 `DataTexture`. A GPGPU shader lerps `currentPos → targetPos` per frame with a spring `k=0.06`, `damping=0.85`.
2. Kanji baking: render each kanji to an offscreen 2D canvas, sample its silhouette to produce N target positions; upload as the next `targetTexture`.
3. Vertex shader: `gl_PointSize = mix(1.5, 4.0, smoothstep(0.0, 1.0, velocity)) * (uDPR)`. Fragment: circular alpha mask with `smoothstep(0.5, 0.48, dist)` for anti-aliasing.
4. Scene transitions: on ScrollTrigger label enter, swap `targetTexture` and animate `uColor` uniform (gsap 1.4s expo.out). Camera dollies from `z=6 → z=4` per section for gentle push-in.
5. Horizontal type ribbon: absolute-positioned strip, `translateX` driven by `scrollProgress * -window.innerWidth * 3` with `will-change: transform` and `contain: paint`.
6. Cursor: custom SVG dot that scales 1 → 2.4 on `Observer.create({ target: '[data-magnetic]', onHover: … })`; radius blends with `mix-blend-mode: difference` over dark scenes.
7. Reduced motion: detect `prefers-reduced-motion`; fall back to static kanji SVGs with crossfade opacity only.
8. Loading: pre-decode all kanji silhouettes on a Web Worker so scroll never blocks the main thread.

WHY IT WORKS: Particle-morph transitions carry cultural meaning (harmony = many parts, one form). The reverse-parallax type creates negative velocity, which reads as calm rather than aggressive — matching the brand thesis. Mono-Latin next to Mincho Japanese produces intentional cultural juxtaposition without exoticism.

GOOD / BAD: Good — the GPGPU particle system is genuinely production-grade and holds 60fps on M-series laptops. Bad — the horizontal ribbon has ~180ms hitch on first paint on lower-end Android, and screen readers get almost nothing (aria-hidden covers the entire canvas without a text alternative).

CLEAN APPLICATION: Apply this to Clean's **"How the Growth Scaler works" section** on the landing page. Replace kanji targets with logos-of-interest → a stylized company graph → an enriched lead card silhouette. Three particle morphs = three product beats: "we find them", "we understand them", "we reach them". Keep the ribbon idea but replace kinetic Japanese with a mono-font stream of real prospect job titles ("VP of Data, Series B FinTech", "Head of RevOps, LATAM PE Fund") scrolling reverse-parallax — it doubles as social proof for the target audience.

## 3. shadcn/ui — Base UI Default + Chat Components + Rhea — https://ui.shadcn.com/docs/changelog/2026-07-base-ui-default

WHAT IT DOES: shadcn's July 2026 release flips the default component base from Radix to Base UI (Radix remains supported, dual-shipped forever), adds React Aria as a third base option, and ships a full chat-interface primitive set: `MessageScroller`, `Message`, `Bubble`, `Attachment`, `Marker`. Pair it with the May-2026 `Rhea` style — a denser, ring-focused variant with softer radii — for the most modern shadcn look currently possible.

PACKAGES:
- `shadcn@2.4+` (CLI)
- `@base-ui-components/react@1.0` (default) OR `@radix-ui/react-*` OR `react-aria-components@1.4`
- `tailwindcss@4`
- `class-variance-authority@0.7`
- `tailwind-merge@2.5`
- `lucide-react@0.454`
- `framer-motion@11` (optional but recommended for message enter/exit)
- `next@15` App Router

IMPLEMENTATION:
1. New project: `npx shadcn@latest init --base base` (defaults to Base UI, Rhea style). For existing repos: `npx shadcn@latest add --base base @message-scroller message bubble attachment marker`.
2. Chat surface layout: a 3-region flex column — `<Header/>` (auto), `<MessageScroller className="flex-1 min-h-0">`, `<Composer/>` (auto). `min-h-0` on the scroller is non-negotiable inside a flex parent or the scroll anchor breaks.
3. Streaming: pass `streaming={true}` on the last `<Bubble>` — component handles auto-scroll suppression when the user has scrolled up (`scroll-anchor: manual` under the hood). Emit tokens via SSE, append into a `useOptimistic` array.
4. Marker patterns: `<Marker variant="separator">Today</Marker>`, `<Marker variant="system">Tool call: search_knowledge()</Marker>`, `<Marker variant="error">Retry succeeded</Marker>`. Use `variant="system"` to expose agent tool-use inline — critical for AI-transparency UX.
5. Attachment: renders file cards with mime-aware icons (lucide `FileText`, `Image`, `Table`). Wrap for a hover state that reveals a small "open in tab" button.
6. Rhea tokens: after init the `tokens.css` layer defines `--radius: 0.5rem`, `--ring-offset: 2px`, denser paddings (`--space-2: 0.375rem`). Adopt them across sidebar and command palette to keep the app consistent.
7. Motion: opt-in Framer wrapper on `<Message>` — `initial={{ opacity: 0, y: 6 }} animate={{ opacity: 1, y: 0 }} transition={{ duration: 0.18, ease: [0.22, 0.61, 0.36, 1] }}`. Nothing more — chat UIs die from over-animation.
8. Command palette pairing: use `<Command>` (already Base UI in this release) with a `data-marker="agent"` group so agent commands render as chips instead of rows.

WHY IT WORKS: Base UI ships with proper focus containment, first-class RSC-friendly components, and better ARIA than Radix's Message primitive was ever going to get. Rhea's ring-based focus states + denser spacing hits the "professional tool, not consumer app" register that AI infra buyers expect. The chat primitives eliminate the four bugs every custom chat UI ships with: broken scroll anchoring, streaming jump, restore-on-mount jitter, and missing status separators.

GOOD / BAD: Good — a real, non-toy chat UI in ~200 lines including agent tool markers. Bad — the migration from Radix to Base UI still requires manual token remapping for teams on custom shadcn themes, and `MessageScroller` still doesn't expose an imperative `scrollToMessage(id)` ref cleanly (workaround: forward a ref and use `element.querySelector([data-message-id])`).

CLEAN APPLICATION: Apply this to Clean's **agent chat / co-pilot sidepanel** — the surface where a user tells Huginn "find me 50 Series-B FinTech CROs in EMEA and enrich them." Use `<Bubble>` for the user turn, `<Message streaming>` for the agent turn, and `<Marker variant="system">` between agent turns to expose each tool call (`search_prospects()`, `enrich_contact()`, `add_to_crm()`). This gives Clean the transparent-agent aesthetic that Cursor / Claude Code have made table-stakes, without writing custom scroll math. Retheme with Clean's tokens by only overriding `--primary` and `--ring` — leave Rhea's density as-is.

## 4. Linear — 2026 UI Refresh + Cinematic Landing — https://linear.app/now/behind-the-latest-design-refresh

WHAT IT DOES: Two surfaces to study together. (a) The **product UI** was refreshed in March 2026: a dimmer sidebar, compact rounded top-tabs, standardized headers across issues/projects/reviews/docs, and a keyboard-first command bar where Cmd+K reaches literally every action. (b) The **marketing site** pairs GPU-rendered gradient meshes with a browser-native product demo that renders real Linear data via a fake API; the copy is dev-shaped ("stop context-switching"), never marketing-shaped ("supercharge productivity").

PACKAGES:
- App UI: `react@19`, `@base-ui-components/react`, `tailwindcss@4`, `cmdk@1.0`, `radix-ui/react-popover` (legacy), `tinykeys@2` (global shortcuts), `sonner@1.5` (toasts), `motion@12` (formerly framer-motion) for micro-interactions
- Marketing: `next@15`, `three@0.169`, `@react-three/fiber@8.17`, `paper-shaders` (or hand-written GLSL mesh gradient), `gsap@3.13`, `Söhne` display font (Inter Display as OSS fallback)
- Data grid: `@tanstack/react-virtual@3` for keyboard-navigable issue rows

IMPLEMENTATION (app UI, the piece to steal for Clean):
1. Layout tokens: sidebar `224px` collapsed, `264px` expanded. Content padding `12px 20px`. Top-tab height `36px`, radius `6px`, icon `14px`, label `12.5px / 500`.
2. Sidebar color: `bg: hsl(var(--surface-1))` at 92% lightness of app bg (i.e., ~4% darker than content), border-right at 1px `--border-subtle`. The point: sidebar recedes.
3. Header architecture: every top-level view uses the exact same `<PageHeader title breadcrumb actions filters view-switcher/>` component. Consistency > cleverness.
4. Command bar: `cmdk` with grouped sections `Navigation / Actions / Create / AI`. Every action registers via a `useCommand({ id, label, shortcut, group, run })` hook that also renders the shortcut hint inline in menus.
5. Keyboard hint chips: on `Alt` hold, `data-hint` elements fade in with `transition: opacity 120ms`. On any letter keypress the matching chip inverts (`bg: --accent`, `color: --accent-foreground`) for 180ms.
6. Density modes: expose `Comfortable / Compact / Spacious` under Preferences by swapping CSS variables `--row-height`, `--pad-y` — never touching component code.
7. Motion budget: nothing tweens over `180ms`. Nothing bounces. Ease-out (`cubic-bezier(0.22, 0.61, 0.36, 1)`) universally. Toast enter `140ms`.
8. Empty states: never illustrations — always a two-line body + primary action button + `⌘ N` hint.

WHY IT WORKS: This is the Bauhaus-of-SaaS treatment — every visual decision earns its keep by reducing time-to-target. The dimmed sidebar isn't a style choice, it's a Fitts's-Law amplifier: your eye anchors on the bright canvas and drops into a task. The universal command bar removes the "where is that setting" tax that kills adoption in feature-dense tools.

GOOD / BAD: Good — the single-component `<PageHeader>` doctrine means new features ship with correct UX on day one. Bad — the marketing site's WebGL gradient mesh eats ~40MB of GPU memory on integrated graphics and Linear still has no aria-live region on toast, so screen-reader users miss "Issue created" confirmations.

CLEAN APPLICATION: Apply this to Clean's **entire app shell** (dashboard, prospect list, knowledge base, chat, settings). Enforce the Linear formula immediately: 240–264px dimmed sidebar, 36px compact top-tabs, a single `<PageHeader>` component reused everywhere, `cmdk`-based command bar with `useCommand` hook and Cmd+K reaching every action including "Run growth scaler", "Add lead", "Open Huginn". Ship density modes on day one — infra buyers evaluate on info density per screen. Match the 180ms motion budget exactly; nothing else in the app moves.

## 5. Glitch&Grit — https://www.awwwards.com/sites/glitch-grit

WHAT IT DOES: A monochrome (#FFFBF7 off-white / #000000 black) design-agency portfolio built in Webflow with a brutalist type stack and cinematic transitions. Full-bleed video reels play muted-autoplay behind massive display type; page transitions use a black "curtain" wipe with a scrambled-glyph mask; hover states run per-letter glitch shuffles on nav items. The layout leans into 12-column asymmetry — copy blocks pushed hard right, video keyed off left.

PACKAGES (equivalent Next.js stack for replication):
- `next@15`, `tailwindcss@4`
- `motion@12` (formerly framer-motion)
- `gsap@3.13` + `SplitText`
- `lenis@1.1`
- Fonts: `PP Neue Machina` (display) + `Söhne Mono` (mono captions) — OSS swaps: `Space Grotesk` + `JetBrains Mono`
- `next-video@2` for optimized bg reels (AV1 + WebM fallback, poster JPG at 40KB)
- No 3D — the whole aesthetic runs on typography and video

IMPLEMENTATION:
1. Grid: `grid grid-cols-12 gap-x-6 px-8 md:px-14`. Copy blocks use `col-start-7 col-span-5`; video posters `col-start-1 col-span-6`. Asymmetry is the design.
2. Display type: `text-[clamp(3.5rem,10vw,10rem)] leading-[0.88] tracking-[-0.03em] font-machina font-medium`. Uppercase for section titles; sentence case for body.
3. Page transition: overlay `<Curtain />` div, `bg-black`, `translate-y-full → 0 → -full` via `motion` `duration: 0.7, ease: [0.76, 0, 0.24, 1]`. During the fully-covered midpoint (~350ms), route change happens. Text "GLITCH&GRIT" fades in on the black at `opacity 0 → 1 → 0` with a `SplitText` chars scramble driven by GSAP `text.each(el => gsap.to(el, {text: {value: getRandomGlyph()}, duration: 0.05, repeat: 3}))`.
4. Hover glitch: on nav hover, split label into chars; each char runs a 5-step glyph shuffle over 140ms then resolves. Randomize glyph pool per char.
5. Video handling: `<video autoplay muted loop playsinline preload="metadata" poster={jpgPoster}>`. Wrap in an `IntersectionObserver` that pauses when off-screen (battery + GPU).
6. Cursor: replace native with a 12px black dot; scale to 44px + text label ("view case") on `[data-cursor="case"]` hover, `motion` spring `stiffness: 220, damping: 22`.
7. Scroll: Lenis with `lerp: 0.09`. All scroll-linked animations use ScrollTrigger `scrub: 0.6`.
8. Type reveal: on section enter, `SplitText.lines` with `y: '100%' → 0`, `stagger: 0.06`, `duration: 1.0`, `ease: 'expo.out'`. Never fade — always mask+translate.

WHY IT WORKS: The 2-color palette forces hierarchy to come from typography weight, scale, and position — which is exactly how print art directors have worked for a century. Video-behind-type creates figure-ground ambiguity that the viewer's eye resolves as depth. Glitch effects are motion-semantically honest: brief, deliberate, and matched to the "grit" brand promise.

GOOD / BAD: Good — proof that "no 3D, no color" can beat overwrought WebGL sites when the type system is disciplined. Bad — the black curtain wipe adds a real 700ms to every navigation (a substantial LCP hit if measured), and the per-letter glitch shuffle triggers text-length announcements on some screen readers.

CLEAN APPLICATION: Apply this to Clean's **case studies / customer stories page**. One customer per screen, full-bleed muted video of the customer's actual product (or a screen-recording of their Clean dashboard), massive Neue Machina headline pulled from a quote, mono caption block on the right with metrics (`+412% qualified leads`, `-68% research time`). Use the curtain wipe between customers only — not on primary navigation. Skip the letter-scramble on nav; keep it exclusively on the "Read case study" CTA where the theatrical flourish earns its cost.

## BUILD DIRECTIVE

TARGET: Clean's **app shell + agent chat sidepanel** (the logged-in product surface, not the marketing site). This is the single highest-leverage build today because (a) Clean is developer/RevOps infrastructure — the app UX ships trust more than the landing does, (b) shadcn shipped Base UI + chat primitives 3 weeks ago that make this cheap to ship correctly, and (c) Linear's 2026 refresh is the definitive template for dense professional tools right now.

APPROACH: Rebuild the shell around a Linear-formula sidebar (240–264px, dimmed one shade below content), a universal `<PageHeader>` component reused across dashboard/prospects/knowledge/settings, and a `cmdk`-powered Cmd+K palette that reaches every action including growth-scaler runs and Huginn queries. Slide out a right-side agent panel (420–520px) built on shadcn's new `MessageScroller / Message / Bubble / Attachment / Marker` primitives with `variant="system"` markers exposing every tool call the agent makes — this is the transparency Cursor and Claude Code have made mandatory. Adopt Rhea tokens (dense paddings, ring-based focus, 0.5rem radii) globally. Enforce a strict 180ms motion budget with `cubic-bezier(0.22, 0.61, 0.36, 1)` on everything, and ship Comfortable/Compact density modes on day one.

STACK:
- `next@15` (App Router)
- `react@19`
- `tailwindcss@4` with Rhea tokens
- `shadcn@2.4+` initialized with `--base base` (Base UI) + Rhea style
- `@base-ui-components/react@1.0`
- `cmdk@1.0` + `tinykeys@2`
- `motion@12` (framer-motion successor) — bounded use only
- `lucide-react@0.454`
- `sonner@1.5` toasts with an `aria-live="polite"` region wrapper (Linear misses this — Clean shouldn't)
- `@tanstack/react-virtual@3` for prospect table
- `class-variance-authority@0.7`, `tailwind-merge@2.5`
- Fonts: `Inter Display` (UI) + `JetBrains Mono` (data columns, tool-call markers)

REFERENCE URLS:
- https://linear.app/now/behind-the-latest-design-refresh (dimmed sidebar, header doctrine, motion budget)
- https://ui.shadcn.com/docs/changelog/2026-07-base-ui-default (Base UI default + chat components)
- https://ui.shadcn.com/docs/changelog/2026-05-rhea (density tokens)

KEY CONSTRAINTS: Clean is developer/RevOps infrastructure. The result must feel like a Bloomberg terminal crossed with Linear — dense, fast, keyboard-first. **AVOID**: (1) any WebGL, particle systems, or 3D in the app shell — save that for the marketing site's product demo, (2) hero-style oversized display type or brutalist asymmetry in-app — the Glitch&Grit playbook is landing-page-only, (3) illustrations in empty states — use two-line copy + primary action + shortcut hint, (4) any motion over 180ms, any bounce ease, any decorative animation without a functional trigger, (5) light-mode-only shipping — dark mode is default for this buyer, light mode is polished but secondary, (6) marketing-voice copy anywhere in the product — "Run scaler" not "Supercharge your pipeline", (7) hiding agent tool calls behind a "show details" affordance — markers must be inline and always visible.
