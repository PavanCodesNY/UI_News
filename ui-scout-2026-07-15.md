## 1. NRG | Build Your Data Center — https://business.nrg.com/campaigns/build-your-data-center/

WHAT IT DOES: Full-screen scroll-driven configurator that walks the user through building a bring-your-own-power AI data center in phases. Layout is a fixed left-side "Phase 0X" progress rail plus a full-bleed center canvas where an interactive node graph (data-center topology) mounts, animates in with staggered opacity + Y-translate, and rewires as the reader scrolls. Each phase is a pinned ScrollTrigger section (~100vh) with three sub-beats: intro copy, node reveal, virtual tour interactive. Palette is 2-color: `#FFC20E` (amber) on `#261D26` (near-black plum) — extreme contrast, one accent, everything else typography.

PACKAGES:
- `next@15` (App Router)
- `tailwindcss@4` (OKLCH tokens)
- `gsap@3.13` + `@gsap/react` (ScrollTrigger, pinning)
- `lenis@1.3` (smooth-scroll driver)
- `framer-motion@12` (micro-interactions on nodes)
- `three@0.170` + `@react-three/fiber@9` + `@react-three/drei@10` (node canvas)
- `clsx`, `tailwind-merge`
- Fonts: `Suisse Int'l Mono` (headings) + `Suisse Int'l` (body) via `next/font/local`

IMPLEMENTATION:
Component tree: `<PhaseRail />` (position: fixed left-6 top-1/2, rotated 90° vertical count "01 / 04"), `<PhaseSection id="phase-0X">` containing `<PhaseCopy />` and `<NodeCanvas />`. Wrap the whole page in a `<LenisProvider>` and register `ScrollTrigger` inside a `useGSAP` hook. Each `<PhaseSection>` gets `ScrollTrigger.create({ trigger, start: "top top", end: "+=200%", pin: true, scrub: 0.6, snap: 1/3 })`. Node reveal is a `three.js` scene with an `InstancedMesh` of 60–120 low-poly boxes (rack nodes) plus `LineSegments` between them; on scroll progress 0→0.5 lerp instance matrices from a scattered start to a grid, on 0.5→1.0 lerp the line opacity from 0 to 1 and add a `bloom` `EffectComposer` pass at strength 0.6. Colors: nodes `#FFC20E` emissive 0.4, lines `#FFC20E` at alpha 0.35. Camera easing `power2.inOut`, 900 ms per beat, spring `{ stiffness: 120, damping: 22 }` for hover. Type: display 96–128 px `clamp(6rem, 10vw, 8rem)`, `line-height: 0.92`, `letter-spacing: -0.03em`, monospace numerals for phase counters. Breakpoints: canvas hidden below `md` and replaced with a poster `<video>` autoplayed muted playsinline. Optimize: `ScrollTrigger.refresh()` on `ResizeObserver`, `three` scene uses `frameloop="demand"` and only re-renders on `ScrollTrigger.onUpdate`.

WHY IT WORKS: The two-color palette forces every ounce of hierarchy through type weight, size, and negative space — a Bauhaus principle called "reduction to essentials." The pinned scroll turns a marketing page into a controlled narrative (Gestalt continuity: nodes literally connect as the story does). The single amber accent is a "cognitive beacon" — the eye returns to whatever glows yellow, so CTAs never fight for attention.

GOOD / BAD:
GOOD: The interactive node graph is the copy — it says "we build infrastructure" without a paragraph.
BAD: Scrub-scrolling on trackpads is janky when the tab isn't focused; mobile fallback video is 8 MB, kills LCP.

CLEAN APPLICATION: Apply this to Clean's landing page hero + product-tour section by replacing the current static hero with a 3-phase pinned scroll: Phase 01 "Discover accounts" (nodes scatter → cluster into ICP), Phase 02 "Enrich" (lines connect nodes to LinkedIn/Gmail data logos), Phase 03 "Outreach" (single node pulses, sends a message that animates into a UI card). Use Clean's off-black + one lime/electric-yellow accent — no gradients, no glassmorphism. Instruction to design agent: "Rebuild Clean's above-fold hero as a Lenis + GSAP ScrollTrigger pinned canvas with a Three.js instanced-node graph that animates through the Discover → Enrich → Outreach story. Kill the current static screenshot. Amber accent only on the active node."

## 2. 21 Hrs On The Moon — https://www.21hrs.space/

WHAT IT DOES: Vertical storytelling site marking the Artemis mission. Structure is chapter-based: a fullscreen WebGL "Interactive Moon Map" hero with drag-to-rotate + click-to-hotspot (each hotspot opens a chapter), a 360° panorama gallery (equirectangular textures in a `SphereGeometry` with inverted normals), audio-snippet cards that play NASA archival audio on hover with an animated waveform, and photo galleries where images cross-fade on scroll velocity. The whole thing is a single-page app with programmatic route pushes for each hotspot.

PACKAGES:
- `three@0.170`
- `@react-three/fiber@9` + `@react-three/drei@10` (`OrbitControls`, `Environment`, `Html`)
- `@react-three/postprocessing@3` (Bloom, ChromaticAberration)
- `gsap@3.13` (timelines for chapter transitions)
- `howler@2.2` (audio with fade-in/out)
- `wavesurfer.js@7` (waveform viz)
- `next@15`
- `tailwindcss@4`
- Font: `NB Architekt` (display) + `Söhne Mono` (labels)

IMPLEMENTATION:
Root `<Canvas camera={{ fov: 45, position: [0, 0, 5] }}>` renders `<Moon />` (a `SphereGeometry(1, 128, 128)` with an 8k NASA `MoonMap.jpg` `MeshStandardMaterial({ map, normalMap, roughness: 0.95 })`). Hotspots are `<Html>` children pinned to `Vector3` positions on the sphere surface; render as 12 px amber dots with a `motion.div` scale pulse `{ scale: [1, 1.6, 1], transition: { duration: 2, repeat: Infinity, ease: "easeInOut" }}`. Panorama gallery: separate `<Canvas>` mounted per chapter with a `SphereGeometry(500, 60, 40)` scaled `[-1, 1, 1]` (invert to render inside) and the pano image as `map`; camera at origin with `OrbitControls` locked to zoom-only + look. Audio waveform: `wavesurfer.js` init with `barWidth: 2, barGap: 1, height: 40, waveColor: "#F5F5F5", progressColor: "#FFC20E"`. Chapter transition: `gsap.timeline().to(camera.position, { z: 2.2, duration: 1.4, ease: "power3.inOut" }).to(".chapter-overlay", { autoAlpha: 1, y: 0, duration: 0.6 }, "-=0.4")`. Bloom pass strength 0.4, luminanceThreshold 0.85. Use `Suspense` + `<Preload all />` from drei; gate the hero with a 0→100 loading counter driven by `useProgress` from drei. Mobile fallback: swap `<Canvas>` for a scroll-linked `<video>` (moon rotation prerendered at 30 fps H.265, ~4 MB).

WHY IT WORKS: The moon-as-navigation IS the mental model — Gestalt principle of "spatial mapping" (the interface matches the topic's geography). Audio + waveform + visual creates cross-modal reinforcement, which memory research shows increases dwell time 3–5×. Palette restraint (near-black + off-white + one amber) keeps the eye on the WebGL, not the chrome.

GOOD / BAD:
GOOD: The interactive moon is genuinely useful — you learn where each experiment happened, not just that it happened.
BAD: First paint is 3.2 s on a fast connection; hero WebGL is 12 MB. No accessible fallback for the moon map at all.

CLEAN APPLICATION: Apply this to Clean's "How Huginn works" explainer surface (currently docs/product-tour) by rebuilding it as an interactive 3D "agent flow" globe: a central sphere = Huginn, orbiting nodes = tools it can call (LinkedIn, Gmail, CRM, Enrichment). User drags to rotate, clicks a tool node to see a short 6-second demo video + a copyable curl snippet. Instruction to design agent: "Build a `/how-it-works` page: R3F canvas with a central Huginn sphere (subsurface-scatter shader, off-black material with amber emissive rim) and 8 orbiting tool nodes on a `SphereGeometry(2)`. Click a node → GSAP camera dolly to that node → an HTML overlay slides up with the demo. No audio."

## 3. House of Honey — https://www.houseofhoney.com/

WHAT IT DOES: Editorial-grade portfolio for an interior design studio. Layout is asymmetric magazine grid: full-bleed hero with two stacked full-width images that reveal on scroll via a `clip-path` inset animation, project index that lists projects as huge display-serif titles with geo-coordinates as monospace subtitles, an editorial "column" section ("Dear Honey") formatted like a print journal. Type-first, image-second. Almost zero UI chrome — no visible nav until scroll, then a minimal top bar fades in. Runs on Next.js + Sanity CMS.

PACKAGES:
- `next@15` (App Router, RSC for Sanity fetches)
- `@sanity/client@6` + `next-sanity@9`
- `tailwindcss@4`
- `gsap@3.13` (clip-path reveals, split text)
- `@studio-freight/lenis@1.3`
- `framer-motion@12` (nav fade)
- `sharp` (image opt via `next/image` `loader: 'sanity'`)
- Fonts: `GT Alpina` (display italic serif) + `Neue Haas Grotesk Display` (body) + `JetBrains Mono` (coordinates)

IMPLEMENTATION:
Grid system: 12-col CSS Grid with `gap: clamp(1rem, 2vw, 2rem)`, project titles span `col-span-12 lg:col-span-10 lg:col-start-2` at `font-size: clamp(3rem, 8vw, 10rem)`, `line-height: 0.9`, `letter-spacing: -0.045em`, `font-variation-settings: "opsz" 96`. Image reveals use GSAP + `clip-path: inset(100% 0 0 0)` → `inset(0% 0 0 0)`, duration 1.2 s, easing `cubic-bezier(0.65, 0, 0.35, 1)` (custom bezier used across all reveals for consistency), staggered 0.08 s. Project list hover: title `color: theme(colors.honey)` transition 300 ms + monospace coordinates translate `x: 8px` on hover. Header: `position: fixed, top: 0`, opacity 0 until `window.scrollY > 100`, then `motion.header` fades to 1 with `y: [-8, 0]` in 400 ms. Cursor: replace default with a 6 px circle that grows to 40 px on hover of any interactive element (`mix-blend-mode: difference`). Sanity content: query with GROQ, ISR every 60 s. Images: Sanity CDN with `?w=2048&q=85&fit=crop&auto=format`, next/image with `sizes="(min-width: 1024px) 80vw, 100vw"`.

WHY IT WORKS: Editorial grids leverage the F-pattern eye track that has 60 years of print research behind it. Serif display type reads as "curated" — luxury brands know this. The clip-path reveals are motion semantics: "an image is being unveiled," not "an image is sliding." Cursor blend-mode difference makes the pointer legible on any image without needing a shape drop-shadow.

GOOD / BAD:
GOOD: Types like a print magazine, works because they earned the right — every image is genuinely well-shot.
BAD: Custom cursor breaks on trackpads with tap-to-click and on Firefox occasionally; disables native accessibility affordances (focus outlines).

CLEAN APPLICATION: Apply this to Clean's `/customers` + `/blog` surfaces (currently generic case-study cards and blog list). Instruction to design agent: "Rebuild `/customers` as an editorial index: a vertical stack of customer names in `GT Alpina Italic` at 128 px, each with a monospace subtitle (`ACV: $180K · Vertical: B2B SaaS · Time to Close: 14 days`). Full-bleed hero image per case study revealed with a clip-path GSAP scroll animation. Kill the card grid. For `/blog`, use the same treatment — post titles as huge serif type, publish date + reading time in mono."

## 4. Fauna Robotics — Sprout Product Page — https://faunarobotics.com/product

WHAT IT DOES: Product page for a $50K developer humanoid. Structure: autoplay hero video with a "Play video" button that triggers a fullscreen Vimeo modal, three capability pillars with icons + short copy, five capability sections (Locomotion, Teleoperation, Autonomy, Social, Safety) each rendered as a synchronized 4-up video grid where hovering any tile pauses the others (or auto-cycles at 3.5 s intervals). Ends with an interactive spec table (tabbed: Overview / Human Interaction / Manipulation / Sensing / Usability / Locomotion / Fun) and a horizontal marquee image gallery of product shots. Built on Webflow, but the pattern is portable. Palette is off-white `#EFEFEF` + pale blue `#D1E3FF`.

PACKAGES (if rebuilt in Next.js):
- `next@15`
- `tailwindcss@4`
- `framer-motion@12` (tab transitions, hover state)
- `@radix-ui/react-tabs@1` (spec tabs, accessible)
- `@vimeo/player@2` (fullscreen video modal)
- `embla-carousel-react@8` (marquee)
- `lucide-react` (icons — or custom SVG)
- `react-intersection-observer@9` (auto-pause videos off-screen)
- Fonts: `Söhne` (display + body) + `Söhne Mono` (specs)

IMPLEMENTATION:
Capability sections use a 4-col grid on desktop, each cell a `<video autoPlay muted loop playsInline preload="metadata">` with an `IntersectionObserver` — pause when `!entry.isIntersecting` (huge WPO win). On hover of any tile, use a shared React context to set `activeTileId`; each `<video>` reads context and calls `.play()` if active, `.pause()` otherwise. Below each tile: a label in `Söhne Mono` uppercase, `letter-spacing: 0.08em`, `font-size: 12px`. Spec tabs: Radix Tabs `orientation="horizontal"` on desktop, `orientation="vertical"` on mobile, `TabsList` with a `motion.div` layoutId "tab-indicator" for underline animation (Framer Motion shared-layout). Marquee: Embla with `loop: true, dragFree: true, containScroll: false`, autoplay plugin `delay: 0, playOnInit: true, stopOnInteraction: false, stopOnMouseEnter: true` at 30 px/s. Hero video: intersection observer + `object-fit: cover`, poster is a WebP first-frame. Modal: Radix Dialog with backdrop `backdrop-filter: blur(24px) saturate(1.2)`.

WHY IT WORKS: The synchronized-video grid is dense information delivery — four demos in the space of one, and the mind naturally reads them as a matrix (Gestalt proximity). Tabbed specs respect that some readers want everything and some want nothing — progressive disclosure done right. The pale palette + one saturation accent = "medical-grade credibility" (used in Apple health, ResMed, Ōura).

GOOD / BAD:
GOOD: Every capability claim is proven with a moving demo — nothing is asserted, everything is shown.
BAD: 27 concurrent muted autoplay videos on the product page is a battery drain; page weight is 42 MB total.

CLEAN APPLICATION: Apply this to Clean's product page (`/product` or the "how it works" section of homepage). Instruction to design agent: "Build a 4-up synchronized-video capability grid for Clean's four core loops: Discover, Enrich, Outreach, Report. Each cell is a 720p 8-second screen recording of the actual product loop, autoplayed muted looped, pause-on-off-screen via IntersectionObserver. Below the grid, a Radix Tabs spec block: Sources / Enrichment fields / Outreach channels / Reporting dimensions. Use Clean's dark palette, keep videos to <2 MB each via H.265."

## 5. shadcn/ui — Base UI Default + Message Scroller (July 2026 drop) — https://ui.shadcn.com/docs/react/message-scroller

WHAT IT DOES: shadcn/ui made Base UI the default primitive layer on July 15, 2026 (replacing Radix as the recommended choice) and shipped a new headless `@shadcn/react` package. The centerpiece is `MessageScroller` — a headless AI-chat transcript primitive that handles turn anchoring, streamed-output following, prepend-preservation (older-message loading without jump), visibility tracking, and a "jump to latest" pill. It's the missing piece for any chat-with-agent UI that streams tokens and loads history. Also shipped: `@shadcn/typeset` (typography reset), `Message`, `Bubble`, `Attachment`, `Marker` primitives (June, still hot).

PACKAGES:
- `@shadcn/react` (headless behaviors — `MessageScroller`, hooks)
- `@base-ui-components/react@1.6` (Base UI primitives, replacing Radix)
- `@ai-sdk/react@2` (`useChat`) or `@tanstack/react-ai` for TanStack shops
- `ai@5` (`DefaultChatTransport`)
- `tailwindcss@4`
- `class-variance-authority`, `clsx`, `tailwind-merge`
- Init: `pnpm dlx shadcn init` (Base UI is now the default; use `-b radix` to opt back)

IMPLEMENTATION:
Wire the transcript with the compound API:
```tsx
<MessageScroller.Provider autoScroll defaultScrollPosition="last-anchor" scrollPreviousItemPeek={64}>
  <MessageScroller.Root className="relative flex flex-1 flex-col overflow-hidden">
    <MessageScroller.Viewport className="flex flex-1 flex-col overflow-y-auto [mask-image:linear-gradient(to_bottom,transparent,black_24px,black_calc(100%-24px),transparent)]">
      <MessageScroller.Content className="flex flex-col gap-4 p-6">
        {messages.map((m, i) => (
          <MessageScroller.Item key={m.id} messageId={`m-${i}`} scrollAnchor={m.role === "user"}>
            <Message role={m.role}><Bubble>{renderParts(m.parts)}</Bubble></Message>
          </MessageScroller.Item>
        ))}
      </MessageScroller.Content>
    </MessageScroller.Viewport>
    <MessageScroller.Button className="absolute bottom-3 left-1/2 -translate-x-1/2 rounded-full border bg-background px-3 py-1 text-xs shadow-sm inert:opacity-0 transition-opacity">
      Jump to latest ↓
    </MessageScroller.Button>
  </MessageScroller.Root>
</MessageScroller.Provider>
```
Key configs: `scrollAnchor={message.role === "user"}` anchors each turn at the user's message so streamed AI replies push down without losing context. `autoScroll` only follows the tail while the reader is already at the edge — wheel/touch immediately releases. `defaultScrollPosition="last-anchor"` opens history at the last turn instead of the absolute bottom. Style with `data-scrollable` attributes: `[data-scrollable~="end"]::after` for a bottom edge fade, hide the jump button when neither edge scrolls. Use `useMessageScrollerVisibility()` to power an outline sidebar of visible turns. Migration from Radix: `pnpm dlx skills add shadcn/ui` then ask your agent "migrate accordion to base-ui" — progressive, one component at a time, produces `.migration/*.md` reports.

WHY IT WORKS: Anchoring on user turns matches how people mentally track a conversation ("I said X → what did it say back?") — a fundamental UX principle called "reference-point stability." The edge-mask + scroll-fade utility create the "elegant terminal" feel that OpenAI, Anthropic, and Vercel all converged on. Headless-plus-styled-wrapper is the only right pattern for AI chat because every product needs its own message shape.

GOOD / BAD:
GOOD: This is the first primitive that correctly handles streaming + history-load + jump-to-latest without any of the classic scroll bugs (jump on prepend, lost-anchor on stream, unwanted follow).
BAD: Requires you to already be on Tailwind v4 + React 19; the `ai@5` dependency is not stable in TanStack Query-heavy apps yet.

CLEAN APPLICATION: Apply this to Clean's Huginn chat surface (the in-product AI agent panel) and the marketing-site "Try Huginn live" demo widget. Instruction to design agent: "Rip out the current transcript div in the Huginn panel and replace with the `MessageScroller` primitive from `@shadcn/react`, wired to `useChat` from `@ai-sdk/react` against the existing `/api/huginn` endpoint. Anchor on user turns (`scrollAnchor={m.role === 'user'}`), set `autoScroll` and `defaultScrollPosition='last-anchor'`. Style with Clean's dark palette + amber accent on the jump-to-latest pill. Add an edge fade top/bottom via the `[data-scrollable~='...']` selectors. Migrate the current shadcn Radix components to Base UI in a follow-up branch using `pnpm dlx skills add shadcn/ui`."

KEY CONSTRAINTS: Clean is developer infrastructure, so the result must read as "precise" not "flashy."
