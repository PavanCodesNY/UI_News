## 1. Linear (2026 Homepage) — https://linear.app

WHAT IT DOES: Single-viewport hero stacks kinetic typography (a sentence whose noun cycles between "teams", "agents", "operators" on a 2.4s loop) over a pixel-perfect, autonomously-animating mock of a Linear issue thread where a Codex AI agent picks up a ticket, comments, and opens a PR. The entire page is scaffolded with paper-like uppercase labels (tiny 11px tracking-wide caps with hairline borders) so it reads as a technical specification document, not marketing collateral. Subsequent sections are a series of self-playing product micro-demos (each ~6s loop) with sticky-scroll horizontal pinning between them via Lenis + native CSS scroll-snap.

PACKAGES:
- `next@15` (App Router, RSC)
- `react@19`
- `tailwindcss@4` (with `@tailwindcss/postcss`)
- `framer-motion@12` (for the issue-thread micro-demo)
- `lenis@1.1` (smooth scroll)
- `geist@1.3` (`GeistSans` + `GeistMono`)
- `clsx@2` / `tailwind-merge@2`
- `lucide-react@0.4xx` for hairline icons
- `vaul@1` for the in-page command-palette overlay

IMPLEMENTATION:
Component tree: `<Hero><WordSwapper /><IssueMockDemo /></Hero>` then `<ScrollSnapStack>` containing 5 `<PinnedDemo>` cards. `WordSwapper` uses Framer Motion `AnimatePresence` mode="popLayout" with `initial={{y:18,opacity:0,filter:"blur(6px)"}}`, `animate={{y:0,opacity:1,filter:"blur(0)"}}`, `exit={{y:-18,opacity:0,filter:"blur(6px)"}}`, transition `{duration:0.42, ease:[0.22,1,0.36,1]}` (cubic-out-expo). Interval `2400ms`. `IssueMockDemo` runs a state machine via `useReducer` — steps: `idle → agent_typing → comment_posted → pr_drafted → pr_opened` with `setTimeout` chain (900/1400/900/1100ms). Each step swaps a row in a vertical timeline rendered as Tailwind grid `grid-cols-[20px_1fr] gap-x-3`. Vertical guide line is a `bg-[oklch(0.94_0.005_220)]` 1px column. Avatars are 20x20 ring-1 ring-black/10. Type stack: body `Geist Sans` weight 400 letter-spacing `-0.011em`, code `Geist Mono` 12px. Background is solid `oklch(0.985 0.003 240)` (warm-cool off-white) — NOT pure white. Hairline labels: `text-[10.5px] tracking-[0.16em] uppercase text-zinc-500 border-t border-zinc-200 pt-2`. Section transitions: `scroll-snap-type: y mandatory` on `<html>`, each `<PinnedDemo>` is `scroll-snap-align: start` `min-h-screen`. Lenis init: `new Lenis({ duration: 1.1, easing: t => Math.min(1, 1.001 - 2 ** (-10 * t)) })`. Breakpoints: hero stacks vertically `<lg`, mock demo hides on `<md` and replaces with a static frame poster. Performance: lazy-mount each PinnedDemo via `useInView({ amount: 0.4 })` to keep CPU under 4% on Apple Silicon idle.

WHY IT WORKS: Visual hierarchy weaponizes whitespace — 6 vertical rhythms (label, headline, eyebrow, demo, caption, action) instead of the standard 3, which makes the page read like a spec sheet. Motion semantics are diegetic: every animation IS the product doing real work, so curiosity converts to comprehension instead of dazzlement. The off-white background lowers the contrast floor, which makes the in-product UI (rendered at higher contrast) read as the focal subject — classic figure-ground inversion.

GOOD: The "product is the demo" pattern eliminates the gap between marketing claim and proof — visitors leave knowing exactly what shipping looks like.
BAD: The autonomous demos burn ~120ms/frame on low-end Android; the mobile poster fallback is necessary but undersells the product to the half of traffic that gets it.

CLEAN APPLICATION: Apply this to Clean's landing page hero by replacing the current static value-prop with a 60/40 split: left column a kinetic headline "Stop shipping tokens. Ship `[context | memory | tools | budget]`" cycling on 2.4s with the same blur-in/out Framer Motion preset, right column a live-rendering mock of an MCP request flowing through Clean's proxy — show a JSON tool-call entering, the TOON-compressed payload below it (token count animating down: `1,847 → 312`), then a "Forwarded to Claude" stamp. Use Geist Mono for the JSON, Geist Sans for everything else, `oklch(0.985 0.003 240)` background. Scaffold every section with the hairline uppercase labels ("01 / CONTEXT LAYER", "02 / TOOL PROXY") to lean into the developer-infrastructure aesthetic.

## 2. Vercel (2026 Marketing Site) — https://vercel.com

WHAT IT DOES: The hero is a full-bleed WebGL mesh-gradient shader (no DOM elements behind it) that drifts continuously and reacts to cursor proximity by warping the gradient field via a mouse-uniform fed into a simplex-noise displacement. Above it: massive 88px Söhne display headline, live product-dashboard screenshot in a hairline-bordered frame floating ~12% below the fold so it peeks at first paint. Below the hero, an interactive code-snippet tabbed component cycles through Next.js/React/Vue/Svelte deploys, and an enterprise logo wall uses a CSS mask to fade the leftmost and rightmost two logos on an infinite marquee.

PACKAGES:
- `next@15` + `react@19`
- `three@0.16x` (NOT @react-three/fiber — Vercel ships raw three.js for the hero to keep bundle <40kb gzip)
- `glsl-noise` (npm) for the `#pragma glslify: snoise = require('glsl-noise/simplex/3d')` import
- `raw-loader` or `vite-plugin-glsl` for `.glsl` imports
- `tailwindcss@4`
- Söhne (commercial, Lineto) — fallback `Inter Display` from `@fontsource-variable/inter`
- `shiki@1` for the code-snippet syntax highlighting (server-rendered)
- `motion@12` (Framer Motion) for the logo marquee

IMPLEMENTATION:
Hero shader: a single `THREE.PlaneGeometry(2,2)` covering the NDC clip space, vertex shader pass-through, fragment shader samples `snoise(vec3(uv*1.4, uTime*0.08))` twice at different scales, mixes between 5 hardcoded oklch color stops via `smoothstep` bands. Mouse uniform `uMouse` is updated in a `requestAnimationFrame` loop with exponential damping `m += (target - m) * 0.06`. Resize: `renderer.setPixelRatio(Math.min(window.devicePixelRatio, 1.5))` — capped at 1.5 to keep mobile under 16ms/frame. Dashboard frame is a static PNG at 2x density inside a `<div className="rounded-2xl ring-1 ring-black/8 shadow-[0_50px_120px_-30px_rgba(0,0,0,0.25)]">`. Code snippet uses Shiki's `bundledLanguages` with `themes: { light: 'github-light-default', dark: 'vercel-dark' }`. Tab switcher uses `framer-motion`'s `layoutId="tab-indicator"` for the underline. Logo marquee: two duplicated `flex` rows in an `overflow-hidden` parent with `mask-image: linear-gradient(90deg, transparent, black 12%, black 88%, transparent)`, `animation: marquee 32s linear infinite`. Breakpoints: hero shader downscales `setSize(w/2, h/2)` and CSS `image-rendering: auto` blurs the upscale on `<md`. Reduce-motion: replace shader with a static gradient PNG poster via `matchMedia('(prefers-reduced-motion: reduce)')`.

WHY IT WORKS: The mesh gradient creates an organic, hand-shaped focal point in a layout that is otherwise rigid grid — this introduces a single point of "life" that draws the eye without competing with the headline. Söhne at 88px with -0.022em tracking lands in a sweet spot where it reads as confidence, not shouting. The peeking dashboard exploits the visual-discovery principle (incomplete forms invite scroll).

GOOD: The shader is genuinely lightweight (~28kb gzipped) and runs at 60fps on a 5-year-old MacBook — most clones use heavier R3F setups that tank Lighthouse.
BAD: The mesh gradient is now so over-copied it's become visual noise across SaaS; without distinctive color choices it reads as "AI startup template."

CLEAN APPLICATION: Apply this to Clean's product-demo section (NOT the hero — the hero needs the Linear treatment from #1). Build a "Watch tokens compress in real time" interactive panel with a WebGL shader background using simplex-noise colored with Clean's brand palette (deep ink + a single accent — avoid the standard purple-to-pink that everyone copies). The shader's `uMouse` uniform should drive a contextual "heat map" effect: as the cursor moves, brighter zones show where tokens were saved in the underlying TOON-compressed payload. Cap pixel ratio at 1.25 on mobile, ship a static poster fallback for reduced-motion.

## 3. Pacôme Pertant Portfolio — https://www.pacomepertant.com (Awwwards SOTD 2026-06-09)

WHAT IT DOES: A single-page Nuxt 3 portfolio where the hero is a Three.js scene of an extruded glass shape (custom geometry, refractive material) that rotates in response to scroll position via GSAP ScrollTrigger. As you scroll, the shape morphs through 4 keyframes while the camera dollies, and project thumbnails fade up into a 12-column grid with stagger. Audio: a low ambient drone (sound design is the differentiator — Pacôme is a sound designer) modulated by scroll velocity via Web Audio gain node.

PACKAGES:
- `nuxt@3.13`
- `three@0.16x`
- `@tresjs/nuxt@4` (Nuxt-friendly R3F equivalent)
- `gsap@3.12` with `gsap/ScrollTrigger`
- `tone@15` for the Web Audio drone
- `splitting@1.1` for character-level text animation
- `@vueuse/core@11` for `useScroll`, `useMouse`, `useMediaQuery`
- `lenis@1.1` (used together with ScrollTrigger via the lenis-gsap adapter)
- Local custom variable font (looks like a wide neo-grotesk — substitute `@fontsource-variable/general-sans`)

IMPLEMENTATION:
ScrollTrigger setup: one master timeline pinned to `<main>` with `scrub: 1.2` (lagged scrub creates the "weighted" feel). Each keyframe is a `tl.to(mesh.rotation, { x: ..., y: ..., duration: 1 })` chunk. Refractive material: `MeshPhysicalMaterial` with `transmission: 1, thickness: 1.6, roughness: 0.05, ior: 1.5, iridescence: 0.4`, lit by 3-point setup (key DirectionalLight at intensity 1.4, fill HemisphereLight 0.6, rim PointLight 0.8). HDR environment from `@pmndrs/assets` (`hdri/lobby_1k.hdr`). Audio: `const drone = new Tone.Oscillator(55, 'sine').connect(new Tone.Filter(220, 'lowpass')).toDestination(); drone.start();` then `useScroll`'s velocity drives gain: `gain.value = Math.min(0.6, Math.abs(velocity) * 0.001)`. Project grid: `grid-template-columns: repeat(12, 1fr)` with each project occupying variable column-spans (3, 5, 4, 6) for asymmetric rhythm. Text reveal: `Splitting({ target: 'h1', by: 'chars' })` then GSAP `stagger: 0.018, y: 100%, ease: 'expo.out', duration: 1.1`. Cursor: a 14px circle that magnetizes to `[data-magnetic]` elements via `gsap.quickTo` with damping. Breakpoints: kill Three.js scene below 768px, replace with looping mp4 poster (h.264 + h.265 fallback, ~800kb each). Reduce-motion: disable audio + ScrollTrigger scrub, snap-jump between keyframes instead.

WHY IT WORKS: Multi-sensory coherence — the audio drone modulating with scroll velocity creates somatic feedback that scroll alone can't, making the page feel "alive" without flashy visuals. The refractive glass material activates pareidolia (we read it as a sculpture, not a div), which transfers craft authority to the work shown. Asymmetric grid spans create reading rhythm — variable column-spans force the eye to re-anchor on each row.

GOOD: The audio-on-scroll is genuinely novel and lifts the portfolio out of the homogeneous "GSAP portfolio" template — restraint (low gain, single oscillator) is what makes it work instead of feeling like a 2010s Flash site.
BAD: Autoplaying audio without an explicit toggle violates Chrome's autoplay policy on first load — the site requires a tap before audio engages, which most visitors miss.

CLEAN APPLICATION: Apply this to Clean's "How it works" docs landing by building a scroll-pinned diagram of the MCP proxy flow rendered in Three.js. Each scroll keyframe advances one stage: (1) raw tool list arrives, (2) Clean intercepts, (3) TOON compression shrinks payload visibly, (4) forwarded to LLM, (5) response returns. Use a refractive `MeshPhysicalMaterial` cube as the visual proxy for "the Clean layer" so it reads as a discrete object the request passes through. Skip the audio (overkill for infra docs), but keep the lagged ScrollTrigger scrub (`scrub: 1.2`) — it makes the diagram feel weighted and considered.

## 4. Fauna Robotics — https://faunarobotics.com (Awwwards SOTD 2026-06-17)

WHAT IT DOES: Hero is a fullscreen autoplay video of a quadruped robot prototype filmed on a paper-grey cyclorama, color-graded to a desaturated palette of `#EFEFEF` and `#D1E3FF`. Video plays inline behind a 1-line headline ("Robots that learn the world by living in it") with the nav inverting blend mode (`mix-blend-mode: difference`) so it stays readable over both light and dark frames. Subsequent sections use a Webflow-native CMS to drive a 3-column research grid with hover-state video previews. The interaction signature is gesture-based: drag-to-explore on the robot capability cards, with momentum scrolling via GSAP Draggable.

PACKAGES:
- Built in Webflow — to replicate in code:
- `next@15` or `astro@5` (Astro is closer to Webflow's static-first model)
- `tailwindcss@4`
- `gsap@3.12` with `gsap/Draggable` and `gsap/InertiaPlugin` (Club GreenSock)
- `framer-motion@12` for the hover video previews
- Inter Tight (variable) — `@fontsource-variable/inter-tight`
- Self-hosted h.265 + h.264 video pair via `<video>` source switching
- `react-intersection-observer@9` for lazy-mount

IMPLEMENTATION:
Hero video: `<video autoPlay muted playsInline loop preload="metadata" poster="/hero-poster.avif">` with two `<source>` tags (h.265 hevc first for Safari, h.264 fallback). Target file size: 2.2MB hevc / 3.8MB h.264 for a 12-second loop at 1920x1080. CSS: `object-fit: cover; filter: brightness(0.92) saturate(0.7);` to match the graded look without re-encoding. Nav: `position: fixed; top: 0; mix-blend-mode: difference; color: white;` — the difference blend mode inverts against the video frame so it auto-adapts. Headline: Inter Tight 600 at `clamp(48px, 5.5vw, 88px)`, `letter-spacing: -0.03em`, `line-height: 1.02`. Drag-to-explore cards: GSAP Draggable on a horizontal track `<div className="flex gap-6 will-change-transform">`, `Draggable.create(track, { type: 'x', inertia: true, bounds: container, edgeResistance: 0.85 })`. Hover video preview: on `whileHover`, mount a muted looping `<video>` over the thumbnail with `framer-motion`'s `initial={{opacity:0}} animate={{opacity:1}} transition={{duration:0.25}}`. CMS: a JSON manifest at build time, rendered via Astro content collections. Breakpoints: drag track becomes vertical stacked on `<md`. Performance: hero video lazy-loads the second source via `media="(min-width: 768px)"` so mobile gets a 4:5 portrait crop at 900px wide (~1.2MB).

WHY IT WORKS: The two-color palette (`#EFEFEF` + `#D1E3FF`) is a deliberate restriction that signals editorial discipline — the visitor reads it as "these people care about every decision." `mix-blend-mode: difference` on the nav is a structural solution to an interaction problem (readability over varying video) instead of a hack like adding a backdrop gradient. Drag-to-explore exploits the affordance principle: anything that suggests physicality (inertia, edge resistance) makes interaction feel earned instead of automated.

GOOD: The desaturated grade gives the robotics-startup category its first genuinely cinematic site — most peers ship over-saturated marketing video that looks like a TV ad.
BAD: Webflow as a CMS choice means every additional CMS item adds DOM weight; the site already ships ~480kb JS, and a third research column expansion will push past Lighthouse mobile budgets.

CLEAN APPLICATION: Apply this to Clean's "Pricing" or "Customers" page by adopting the two-tone restraint (`oklch(0.98 0.005 250)` + a single accent at `oklch(0.83 0.06 240)`) and using drag-to-explore for the customer case-study cards. The mix-blend-mode trick is the highest-leverage steal: apply `mix-blend-mode: difference` to the floating nav across the entire Clean site so it auto-adapts over the heroes, gradients, and code blocks below. Avoid the hero video — Clean is infrastructure, not hardware; a video of "fewer tokens" is unwatchable. Use a TOON-compressed code diff playing in a `<pre>` block instead.

## 5. Gucci: Mystery Unfolds — https://mysteryunfolds.gucci.com (Awwwards SOTD 2026-06-17)

WHAT IT DOES: An interactive narrative microsite for a Gucci capsule — visitor scrolls through 7 chapter-scenes, each a full-viewport pinned section with parallax layers (background plate, midground prop, foreground product, character cutout) driven by GSAP ScrollTrigger. Between chapters, a horizontal "transition reel" plays a 0.8s scrubbed video clip that bridges scenes. Content is fetched via GraphQL from a headless Contentful instance so the same microsite ships in 6 languages. Audio is per-scene ambient, fading between scenes with a 1.2s crossfade.

PACKAGES:
- `next@15` (because GraphQL/i18n is easier than Webflow for this scale)
- `gsap@3.12` with `ScrollTrigger`, `ScrollSmoother` (Club GreenSock)
- `graphql-request@7` + Contentful's Content Delivery API
- `next-intl@3` for the 6-language switcher
- `tone@15` for crossfaded audio
- `framer-motion@12` for the menu/UI overlays
- `lottie-react@2.4` for chapter-marker animations
- `lazysizes@5` for the chapter image plates
- Gucci's custom typeface (substitute `@fontsource/playfair-display` + `@fontsource/work-sans` for development)

IMPLEMENTATION:
Pin each chapter: `ScrollTrigger.create({ trigger: chapterRef, start: 'top top', end: '+=120%', pin: true, scrub: 1 })` with a master timeline that runs `tl.to(bg, { yPercent: -8 }).to(mid, { yPercent: -22 }, 0).to(fg, { yPercent: -38 }, 0).to(char, { yPercent: -55 }, 0)`. The differential yPercent values create parallax depth without a 3D engine. Transition reel: a `<video>` with `currentTime` driven by ScrollTrigger's progress callback — `onUpdate: self => video.currentTime = video.duration * self.progress`. GraphQL: at build time, fetch all chapter content via Next.js `generateStaticParams` so the page is fully static per locale (`/en`, `/it`, `/ja`, etc). Audio crossfade: two `Tone.Player` instances, one fading in (`gain.rampTo(0.5, 1.2)`) while the other fades out (`gain.rampTo(0, 1.2)`). Chapter markers: a fixed left rail with 7 Lottie dots, current chapter scales `transform: scale(1.4)` via `animate-pulse`. Mobile: pin/scrub is replaced with `scroll-snap-type: y mandatory` and the parallax differential is reduced to `-2/-6/-12/-18` for less motion sickness. Reduce-motion: scroll-snap only, no scrub.

WHY IT WORKS: Parallax with strictly increasing yPercent magnitudes encodes depth in the same way our visual system reads it (closer = faster relative motion), which is why the scene reads as a stage even though it's flat divs. The video scrub between chapters bridges the discontinuity that scroll-pinning normally creates (the "snap" between pinned sections) — it makes the entire 7-chapter sequence feel like one continuous shot. GraphQL behind a fully-static page means every chapter loads instantly with no API roundtrip — luxury brands cannot afford a spinner.

GOOD: The video-scrub-between-chapters trick is genuinely state-of-the-art and turns a 7-section scroll into a continuous film.
BAD: At 6 languages × 7 chapter scenes × 4 parallax plates each, the asset payload is ~14MB on first locale visit; pre-cache strategy assumes CDN warm-up that smaller teams can't replicate.

CLEAN APPLICATION: Apply this to Clean's "How it works" page (alternative to the #3 Pacôme treatment — pick one) by structuring it as 5 narrative chapters: (1) MCP request enters, (2) tools are inspected, (3) context is rebuilt, (4) TOON compresses, (5) response forwards. Each chapter is pinned with parallax layers: background gridded blueprint plate (-10% yPercent), midground request payload (-25%), foreground Clean logo waveform (-45%). Between chapters use the ScrollTrigger video-scrub trick on a 0.6s clip showing the request "morphing" — this is the single most copyable Gucci technique for any infra story. Skip the audio and Lottie (overkill for B2B dev), keep the parallax and the video bridge.

KEY CONSTRAINTS:
- Clean is developer infrastructure. The result must feel like reading a well-laid-out technical spec, not a marketing site.
- AVOID: standard purple-pink AI gradient, autoplay hero video, glass-morphism cards, oversized 3D models that don't represent product behavior, hero copy that mentions "AI-powered" or "intelligent" or "supercharge."
- AVOID: Aceternity-style spotlight/sparkle/meteor effects — these are the visual signature of "no real product yet."
- AVOID: bento grids in the hero (overdone; reserve for a deep-link section if needed).
- Bundle budget: <120kb JS gzip on first load. Hero shader must be <30kb.
- Lighthouse mobile performance ≥ 90. The mock demo gets a static poster fallback on mobile (Linear's pattern).
- Type stack is locked: Geist Sans for body + headline, Geist Mono for all code, JSON, and uppercase hairline labels.
- Color: warm off-white `oklch(0.985 0.003 240)`, ink `oklch(0.18 0.01 250)`, single accent at `oklch(0.72 0.13 195)` (a cool teal — claim a color not yet owned by an AI-infra peer). NO secondary accents.
- Every section uses the `NN / SECTION NAME` hairline label from Linear. This is the brand-defining scaffold device for the entire site.
