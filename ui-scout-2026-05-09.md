## Studio Namma — https://studionamma.com

WHAT IT DOES: Single-page studio site built on a 12-column Webflow grid with a fixed-position custom cursor and a full-bleed hero whose headline characters are split-mounted via SplitText and revealed on a per-glyph stagger. Page-to-page navigation is replaced by a full-screen "Transition Page" overlay (a black panel that slides in from the bottom, holds a centered logo for ~700ms while the next route's DOM is mounted, then slides out the top), so the whole site behaves like one continuous reel. The portfolio layout is a horizontal-scroll gallery driven by GSAP ScrollTrigger that pins the section, transforms vertical wheel delta into `x` translateX on a flex track, and triggers per-card image scale-in (1.06 → 1.0) as each card enters viewport.

PACKAGES:
- `webflow` (host)
- `gsap@3.13+` (core)
- `gsap/ScrollTrigger`
- `gsap/SplitText`
- `gsap/ScrollSmoother` (for the Lenis-style inertial scroll)
- `lenis@1.1+` (alternative if not on GSAP Premium)
- Self-hosted variable font (looks like a custom serif display + a neo-grotesque mono pair — substitute `PP Editorial New` + `JetBrains Mono`)
- No 3D, no React — pure DOM + GSAP

IMPLEMENTATION:
1. Root layout: `<main data-scroll-container>` with sections `Hero`, `Manifesto`, `Selected-Work`, `Approach`, `Footer`.
2. Cursor: fixed `<div class="cursor">` 12×12px, mix-blend-mode `difference`, follows pointer with `gsap.quickTo(el, "x", { duration: 0.25, ease: "expo.out" })`. Scale to 3× on `data-cursor="hover"` elements.
3. Hero text: split each `<h1>` line with `SplitText({ type: "lines, words, chars" })`. Animate `chars` from `{ yPercent: 110, rotate: 6 }` → `{ yPercent: 0, rotate: 0 }` with `stagger: 0.012, duration: 0.9, ease: "expo.out"`.
4. Page transition: on link click, `gsap.timeline()` → tween a full-bleed overlay `{ yPercent: 100 } → { yPercent: 0 }` with `ease: "power3.inOut", duration: 0.7`, then `await fetch(nextRoute)`, swap inner HTML, then tween overlay `{ yPercent: -100 } duration: 0.7`.
5. Horizontal portfolio: pin section for `pin: true, scrub: 1, end: () => "+=" + track.scrollWidth`. Map vertical scroll progress to `x: -track.scrollWidth + window.innerWidth`.
6. Footer "magnetic" CTA: 200×80 button, on `mousemove` translate by `(mouseX - centerX) * 0.3`. On leave, return with `elastic.out(1, 0.4)`.
7. Breakpoints: `>=1280px` desktop horizontal scroll, `<1024px` collapse to vertical stack and disable pin. Always provide `prefers-reduced-motion` fallback that nukes all GSAP timelines and shows static layout.
8. Performance: throttle ScrollTrigger to `ticker.lagSmoothing(500, 33)`. Use `will-change: transform` only during active animations, remove on completion.

WHY IT WORKS: The split-char hero exploits Gestalt closure — viewers see whole words even before chars finish stacking, so the animation reads as fast even at 900ms. The transition overlay enforces a cinematic rhythm that makes route loads feel intentional rather than slow. Mono-pair typography (display serif + technical mono) signals "studio that builds, not just designs" — a credibility cue Clean needs.

GOOD / BAD:
- GOOD: Page transitions hide async route latency without ever showing a loading spinner.
- BAD: Horizontal pinned scroll breaks down on touch trackpads with momentum scrolling — overshoots the section and loses ScrollTrigger sync.

CLEAN APPLICATION: Apply this to Clean's docs navigation (`docs.tryclean.ai`) by using the same overlay-transition pattern between docs pages so navigating from "What is Clean?" → "Quickstart" → "MCP setup" feels like a single continuous sidebar-driven reel rather than full page reloads. Use SplitText only on the docs landing H1, not body — keep body copy static for readability.

## Fourmula AI — https://fourmula.ai

WHAT IT DOES: Dark-mode AI product site with an orange-on-near-black palette (`#020108` background, `#FF6B02` accent — exactly two colors, no grays). The hero is a "dynamic hero": a video-driven mask reveals a generative-image grid that shifts which image is in focus as the user scrolls — implemented as a CSS Grid with one `aspect-ratio: 4/5` cell per row that scales `1.0 → 1.18` and lifts to z-index 2 as ScrollTrigger progress crosses its viewport center. The loading screen is a centered orange progress bar with the model name typing in mono characters (typewriter timing 30ms per char), gating page mount until WebGL/video assets resolve. Navigation is a top-anchored thin pill that contracts to a hamburger circle after 200vh scrolled, expanding back into a full overlay menu with a 6-column grid of project thumbnails on click.

PACKAGES:
- `next@14+` or `webflow` (it ships as Webflow)
- `gsap@3.13+` + `gsap/ScrollTrigger`
- `lenis@1.1.x`
- `framer-motion@11+` (if React)
- `@vercel/og` for OG images
- `inter` + `jetbrains-mono` (Google Fonts) — looks like Inter Display + JetBrains Mono
- Loaders: `lottie-react` for the loader bar / `@studio-freight/hamo` for resize observer

IMPLEMENTATION:
1. Color tokens — define exactly two: `--bg: #020108; --accent: #FF6B02;`. Plus `--fg: #FAFAF7` for primary text, `--fg-dim: rgba(250,250,247,0.55)` for secondary. No additional grays.
2. Loader: full-screen `<div role="status">`, centered 240×2px bar, fill `width: 0% → 100%` over `Promise.all([fonts.ready, asset preloads])`. Mono text above ticks chars via `setInterval(30)` until model name "Fourmula v2" finishes.
3. Dynamic hero grid: `display: grid; grid-template-columns: repeat(5, 1fr); gap: 8px;` with 25 image cells. `ScrollTrigger.create({ trigger: heroGrid, start: "top top", end: "bottom top", scrub: 1, onUpdate: self => focusCell(self.progress) })` where `focusCell(p)` finds index `Math.floor(p * 25)` and applies `transform: scale(1.18); z-index: 2; transition: transform 0.6s cubic-bezier(0.22, 1, 0.36, 1)` while scaling all others to `0.95`.
4. Sticky pill nav: `position: fixed; top: 16px; left: 50%; transform: translateX(-50%);` Width animates `clamp(280px, 40vw, 560px) → 56px` on `scrollY > window.innerHeight * 2`, with text fading via `clip-path: inset(0 100% 0 0)` reveal.
5. Section typography: hero H1 is `clamp(56px, 8vw, 144px)` Inter Display 600 with `letter-spacing: -0.04em` and orange-fill `text-decoration: underline; text-decoration-thickness: 2px; text-underline-offset: 0.2em;` on key terms only.
6. 404 page: same dark+orange palette with a single rotating 3D wireframe cube (Three.js) on a transparent background, "404 — out of frame" mono caption beneath. Cube rotates `0.005rad/frame` on x and y.
7. Interactive footer: full-bleed `<section>` 100vh tall, scroll-driven from `letter-spacing: -0.04em → 0.18em` across the brand wordmark, plus a magnetic email subscribe field that snaps cursor to the input center within a 120px radius.
8. Motion system: standardize easing as `cubic-bezier(0.22, 1, 0.36, 1)` for entrances, `cubic-bezier(0.4, 0, 0.2, 1)` for exits. Default duration `0.6s`. Use `prefers-reduced-motion: reduce` to disable scrub-driven transforms.

WHY IT WORKS: The two-color palette is a forcing function — every contrast decision is binary, which produces a screenshot-perfect site with zero design debt. Orange-on-near-black hits a contrast ratio of ~7.2:1 (AAA for normal text). The dynamic hero converts the abstract "AI generates assets" claim into a kinesthetic demo without playing a video — users feel the model "selecting" images. Loader-as-product (typing the model name) primes the user that the product is what's loading, not the site.

GOOD / BAD:
- GOOD: Two-color palette + scroll-driven hero converts technical credibility into immediate visual proof.
- BAD: The 25-cell grid scrub becomes janky on M1 base / Intel iGPU below 60fps; needs a `if (devicePixelRatio < 2 || hardwareConcurrency < 8)` fallback to static grid.

CLEAN APPLICATION: Apply this to Clean's landing page hero (tryclean.ai) by replacing the current hero with a near-identical dark+orange dynamic grid where each cell is a code snippet card (function name, file path, embedding score badge) and scrolling drives a "search" animation that surfaces semantically related cells (highlighting orange) and dims the rest. Use the loader-as-product pattern to type "indexing 14,372 functions across 47 repos…" before the page mounts.

## Event Horizon (fade.run) — https://fade.run

WHAT IT DOES: A 9-chapter scroll-driven WebGL narrative simulating a fall into a black hole using physically-based gravitational lensing, time dilation, and frame dragging via Schwarzschild–Kerr metrics. The entire 3D scene is one `<canvas>` driven by Three.js + custom GLSL shaders, with chapter UI overlaid as absolutely-positioned HTML that snaps in/out via Framer Motion. Scroll position drives camera radial distance from singularity, accretion-disk rotation speed, and a chromatic-aberration shader pass whose strength scales with proximity. Mobile users can scan a QR to "sync" their phone (likely a `WebRTC DataChannel` or `BroadcastChannel`-via-server pattern) so the phone's gyroscope drives camera yaw on desktop.

PACKAGES:
- `three@0.169+`
- `@react-three/fiber@8.x`
- `@react-three/drei@9.x` (for OrbitControls, useGLTF, Stats)
- `@react-three/postprocessing@2.x` (chromatic aberration, bloom, vignette)
- `gsap@3.13+` + `gsap/ScrollTrigger`
- `framer-motion@11+`
- `lenis@1.1+`
- Custom GLSL — written inline as `glsl-literal`
- `qrcode.react` for the phone-sync QR
- `peerjs` or custom WebSocket for cross-device sync
- Math: `mathjs` or hand-rolled tensor math
- Fonts: a thin geometric sans (`Söhne Light` or substitute `Inter Display 200`) + technical mono

IMPLEMENTATION:
1. Scene setup: `<Canvas dpr={[1, 2]} gl={{ antialias: false, powerPreference: "high-performance" }}>`. Use `<Stats>` only in dev. Disable shadows (black hole doesn't need them).
2. Black hole shader: a single fullscreen `<ShaderMaterial>` quad in NDC space. Fragment shader raymarches around a Schwarzschild radius `r_s = 2GM/c²`. For each pixel, compute deflection angle `α ≈ 4GM/(bc²)` and sample the background skybox at the deflected direction. Output: a real gravitational lens, not a billboard texture.
3. Accretion disk: `<TorusGeometry args={[3, 1.2, 64, 256]}>` with a custom shader that takes `time`, computes a radial flow noise (curl noise or Perlin in polar coords), mixes orange `#FF6B02` to white at high intensity. Disk rotates `0.2 rad/s` and dilates time-shift toward observer.
4. Scroll → physics binding: ScrollTrigger creates 9 chapters each `1 * window.innerHeight` tall. `onUpdate` writes to a shared `physicsState` object — `cameraR`, `kerrA`, `chromaticStrength`. The R3F render loop reads this each frame.
5. Chapter overlays: each chapter is `<motion.section>` with `initial={{ opacity: 0, y: 40 }}`, `whileInView={{ opacity: 1, y: 0 }}`, `viewport={{ amount: 0.6 }}`. Body copy in mono, max-width 36ch.
6. Phone sync: scan QR → opens phone-side page on shared room ID, requests `DeviceOrientationEvent` permission, broadcasts `(alpha, beta, gamma)` at 30Hz over WebRTC DataChannel. Desktop reads channel and applies to `camera.rotation` with low-pass filter (0.15 mix).
7. Performance gate: on mount, run a 200ms benchmark (count rAF frames). If <40fps, redirect to a static keyframe scrubber (image sequence) at the same scroll positions.
8. Audio: ambient sub-bass + chapter-triggered tones via Tone.js, cross-faded over 800ms on chapter change. Always behind a user-gesture unmute button.

WHY IT WORKS: This is interaction-as-pedagogy — the user *experiences* time dilation rather than reading about it, which exploits embodied cognition (the most reliable retention mechanism in education research). The chromatic aberration ramp creates rising visual tension that maps directly onto the narrative arc. The phone-sync feature is a credibility flex: it tells viewers "we built infrastructure to make this work."

GOOD / BAD:
- GOOD: Scroll-bound physics is faithful to the actual paper, not a fake — that authenticity is what makes "wow" durable.
- BAD: Hard 60fps requirement on integrated GPUs means the static fallback gets shipped to ~30% of users; the budget spent on the WebGL path doesn't pay back for them.

CLEAN APPLICATION: Apply this to Clean's product demo section (`tryclean.ai/#how-it-works`) by replacing static screenshots with a 5-chapter scroll-driven canvas that shows: (1) a 3D codebase as a force-directed graph of function nodes, (2) tree-sitter parsing animating across nodes, (3) embeddings forming as nodes cluster in semantic space, (4) a query vector ray-casting through clusters and lighting the nearest k-nearest-neighbors orange, (5) call-graph expansion rippling outward across edges. Use R3F for the graph, drei `<Line>` for edges, GSAP scrub for the timeline. Pure pedagogy — make the user *feel* what semantic search means.

## PieterKoopt — https://pieterkoopt.nl

WHAT IT DOES: WebGL portfolio that fuses HTML and 3D using a "portal" technique — DOM elements have their bounding rects tracked every frame and re-projected into WebGL coordinates, so a 3D character can "walk into" what looks like a CSS-laid-out section. Section transitions animate the 3D scene (camera, character pose, environment cube) in lockstep with the new section's HTML mount. Project titles render as WebGL `Text` textures with a per-vertex stretch shader keyed off scroll velocity (`scrollVel * 0.0008` mapped to mesh `scale.y`). Character animations come from Mixamo, retargeted onto a low-poly rig, switching between idle / walk / point clips on section change.

PACKAGES:
- `three@0.169+`
- `@react-three/fiber@8.x`
- `@react-three/drei@9.x` — specifically `MeshPortalMaterial`, `useGLTF`, `useAnimations`, `Text`, `useScroll`
- `@react-three/postprocessing` — `Bloom`, `Vignette`
- `gsap@3.13+` + `ScrollTrigger`
- `lenis@1.1+`
- Mixamo FBX → GLB pipeline (exported from Blender with NLA tracks baked)
- Custom GLSL for the velocity stretch
- `next@14+` (App Router)

IMPLEMENTATION:
1. Layout: `<main>` is two stacked layers — `position: fixed` `<Canvas>` covers viewport, `position: relative` HTML layer scrolls on top with `pointer-events: none` until specific interactive zones.
2. Portal sections: each "scene" section gets a `<div data-portal-id="lab">` with known dimensions. A `useFrame` callback reads `getBoundingClientRect()` for every portal, converts to NDC: `((rect.x + rect.width/2) / window.innerWidth) * 2 - 1`. The portal's `MeshPortalMaterial` plane is positioned at that NDC and sized accordingly.
3. Character rig: load `character.glb` once with `useGLTF`. `useAnimations(animations, character)` returns named clips — `actions.idle.play()` by default. On section change, fade out current and fade in new with `0.4s` crossfade.
4. Velocity stretch on title: vertex shader: `vec3 stretched = position; stretched.y *= 1.0 + abs(uVelocity) * uStretchAmount;`. Update `uVelocity` from a `useEffect` listener on `lenis.on('scroll', ({ velocity }) => uVelocity.current = velocity)`.
5. Camera: a single `PerspectiveCamera` whose position and lookAt are GSAP-tweened on section change with `duration: 1.4, ease: "power3.inOut"`. No OrbitControls — fully scripted.
6. Cube environment: each section has a tiny baked HDRI (`drei` `<Environment files="..."/>`) that crossfades using `intensity` over 0.6s.
7. Performance: limit DPR to `[1, 1.75]`. Use `frameloop="demand"` and call `invalidate()` only when scroll moves or animations are active. Disable postprocessing on `prefers-reduced-motion`.
8. Mobile: the portal trick fails on small screens — fall back to a static character rendered once per section as a mp4 video poster + autoplay loop.

WHY IT WORKS: The portal technique resolves the oldest tension in 3D web design — "do I commit to fullscreen WebGL and lose typography, or commit to HTML and lose presence?" — by giving you both. The velocity-stretched titles add a kinesthetic feedback loop where scroll input has visual output, training users that the page is responsive to them.

GOOD / BAD:
- GOOD: Portals make the 3D feel grounded in the page, not stacked on top — radically more coherent than the usual "3D background + flat hero" sandwich.
- BAD: First contentful paint is gated on character GLB + animation clips; even at 1.8MB, that's 800ms+ on slow connections, and there's no skeleton state.

CLEAN APPLICATION: Apply this to Clean's onboarding flow (`tryclean.ai/onboarding` after signup) by treating each onboarding step as a portal scene where a low-poly "agent" character points at the relevant action — Step 1: agent walks to a GitHub icon and gestures (connect repo). Step 2: agent stands in a swirl of code particles (indexing). Step 3: agent hands a glowing token to the camera (your MCP URL). Use the same portal+character technique to keep onboarding feeling like one continuous physical place rather than 3 disjoint forms.

## Obys — https://obys.agency

WHAT IT DOES: Brutalist-minimal agency site with a strict 3-color palette (`#000` background, `#FFFFFF` text, `#ECD06F` gold accent) and a typographic system built on a single display serif at extreme scales (`8vw`–`24vw`) over a tight 12-column grid. The signature interaction is a "crazy mode" toggle that swaps the entire layout from a calm vertical scroll into a chaotic asymmetric grid where elements are randomly rotated, scaled, and overlapped — implemented as a parallel CSS class (`body.crazy`) with prebuilt transform variants per element. Page transitions use a horizontal wipe with an embedded video preview of the next case study. Everywhere, the cursor is replaced by either a label-tooltip ("View case", "Watch reel") or a small video preview when hovering case-study cards.

PACKAGES:
- `next@14+` (likely; could be Nuxt — output looks SSR React)
- `gsap@3.13+` + `ScrollTrigger` + `SplitText`
- `lenis@1.1+`
- `framer-motion@11+`
- Display serif: looks like `PP Editorial New` or `Romie` (substitute `Instrument Serif`)
- Mono accent: `JetBrains Mono` or `Söhne Mono`
- `react-use-gesture` / `@use-gesture/react` for cursor interactions
- No 3D — pure 2D + CSS grid + GSAP

IMPLEMENTATION:
1. Color tokens: `--bg: #000; --fg: #fff; --accent: #ECD06F;`. That's it. Treat any other color as a bug.
2. Type scale: H1 `clamp(80px, 18vw, 320px)` Editorial-Serif italic, `letter-spacing: -0.04em`, `line-height: 0.85`. Body `16px / 1.5` mono.
3. Grid: `display: grid; grid-template-columns: repeat(12, 1fr); column-gap: 24px; row-gap: 96px;` at desktop. Section padding `clamp(80px, 12vw, 200px) 0`.
4. Crazy mode: every element with a `data-crazy` attribute defines its alt position via inline custom properties (`--crazy-x`, `--crazy-y`, `--crazy-rot`). On toggle, `body.crazy *[data-crazy] { transform: translate(var(--crazy-x), var(--crazy-y)) rotate(var(--crazy-rot)); }` with a `transition: transform 0.9s cubic-bezier(0.85, 0, 0.15, 1)`. Pre-author the chaos — don't randomize at runtime.
5. Cursor system: `<div class="cursor" data-mode="default">`. Modes: `default` (12px dot), `view` (80px gold pill saying "View"), `video` (180×100px MP4 preview that swaps `src` per project). Mode controlled by `data-cursor` on hover targets.
6. Page transitions: 2-layer overlay. Layer A (`bg: #ECD06F`) wipes `clip-path: inset(0 100% 0 0) → inset(0 0 0 0)` over 0.7s. Halfway, an `<video muted autoplay loop>` of the destination project plays inside the overlay for 600ms. Layer A then wipes out from the right.
7. Marquee: a `<div class="marquee">` with `animation: scroll 25s linear infinite; transform: translateX(0) → translateX(-50%);` containing duplicated content. Pause on hover.
8. Accessibility: `body.crazy` toggle is a button with `aria-pressed`. Screen-reader hidden mp4 cursors. `prefers-reduced-motion` disables crazy mode and all transforms.

WHY IT WORKS: The 3-color palette and single-display-serif system act as a visual signature — viewers recognize Obys instantly across 20 projects. The crazy-mode toggle is brand storytelling as interaction: it says "we are calm and we are chaotic, and we choose when" — which is the agency's value prop. Editorial-scale serif italic on black with gold underlines reads like an art-book imprint, signaling craft.

GOOD / BAD:
- GOOD: The crazy mode does in 3 seconds what a 60-second brand video would try to convey, and it doesn't bury the calm baseline.
- BAD: At display sizes >200px, italic serif breaks subpixel hinting in some browsers and ships visibly fuzzy on Windows Chrome — needs explicit `text-rendering: geometricPrecision` and a fallback for low-DPI.

CLEAN APPLICATION: Apply this to Clean's pricing page (`tryclean.ai/pricing`) by trading the SaaS-table cliché for a 3-color editorial layout — black background, off-white text, single accent color (use Clean's existing orange `#FF6B02` instead of gold), one display-serif italic header for each tier name ("Free", "Team", "Enterprise") at `12vw`, mono body for the feature list. Add a "compare modes" toggle that swaps between minimal-view (just price + CTA) and verbose-view (full feature matrix) using the same `body.crazy`-style class swap technique.

Below the loader, the hero is a 5×5 grid of code-snippet cards (each shows function name, file path, language badge, embedding score) where ScrollTrigger drives a faux semantic-search interaction: as the user scrolls the first viewport, a query string types into a fixed pill at top, cells light orange in semantic-distance order, and a call-graph overlay (using R3F + drei `<Line>`) ripples outward connecting matched cells. The next chapter uses an Event-Horizon-style scroll-bound 3D scene to show a force-directed code graph, tree-sitter parsing animating across nodes, embeddings forming clusters, then a query vector ray-casting and lighting nearest-neighbors. KEY CONSTRAINTS:
- Clean is developer infrastructure — every animation must encode product truth, not decoration. If the search demo isn't actually showing semantic search semantics, cut it.
- No 3D character mascots. No skeumorphic chrome. No glassmorphism. No purple-blue gradients. No floating geometric shapes "for vibes."
- Lighthouse Performance ≥ 90 on a throttled MotoG4. Means: lazy-mount the R3F canvas on scroll-into-view, ship a static SVG for the hero grid until the user scrolls, gate WebGL on a 200ms rAF benchmark and fall back to image-sequence scrubber on weak hardware.
- Typography is two faces. Anyone who suggests adding a third gets vetoed. Italic serif belongs on Obys, not Clean.
- Two colors only on the landing — `#020108` and `#FF6B02`. Plus `#FAFAF7` text and a single `rgba(250,250,247,0.55)` dim. That is the entire palette.
- Accessibility: `prefers-reduced-motion: reduce` must produce a fully static, fully readable page. Color contrast ≥ 7:1 for headlines, 4.5:1 for body.
- TTFB ≤ 200ms (Vercel edge), LCP ≤ 1.8s, CLS ≤ 0.05.
- Avoid: cookie banner overlap with sticky pill nav (z-index audit before ship), mobile horizontal scroll bleed (set `overflow-x: clip` on `<html>`), fonts-loading flash (use `font-display: swap` only on body, `optional` on display face — re-snap acceptable).
