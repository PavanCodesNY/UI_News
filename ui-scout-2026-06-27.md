## Ten Years Away — https://ten.375.studio/en (Awwwards SOTD 2026-06-27)

WHAT IT DOES: A scroll-driven interactive graphic novel that pins a fixed WebGL canvas (full-viewport `<canvas>` element, `position: fixed`) and pages through 10 chapter panels by reading `window.scrollY` into a normalized `progress` value (0–1). Each panel composites layered PNG/SVG illustration planes inside Three.js with depth offsets in the z-axis (parallax) and frame-by-frame sprite swaps timed to scroll deltas, with GSAP ScrollTrigger driving section-scoped timelines (`scrub: 1.2`) for camera dolly, opacity, and stroke-draw of inline SVG dialogue bubbles. The layout system is a virtual scroll height (`document.body` set to ~1000vh) with no real DOM flow inside chapters — every "page" is a Three.js scene swap triggered at a scroll offset.

PACKAGES:
- `three@^0.169.0`
- `gsap@^3.12.5` + `gsap/ScrollTrigger`
- `lenis@^1.1.13` (smooth scroll, syncs with ScrollTrigger via `lenis.on('scroll', ScrollTrigger.update)`)
- `vite@^5` + `vanilla TS` (no React on the canvas; React only for chapter index UI)
- `@studio-freight/tempus` for RAF loop sync
- Fonts: `next/font` style locally hosted custom serif display + grotesque body
- Asset pipeline: PNG sprite sheets exported from After Effects via Bodymovin / png-sequence, optimized through `sharp` to WebP

IMPLEMENTATION:
1. Mount a single `<canvas id="stage" class="fixed inset-0 -z-10">` with `WebGLRenderer({ antialias: true, alpha: true, powerPreference: 'high-performance' })` and `OrthographicCamera`.
2. Build a `Chapter` class: takes `{ startProgress, endProgress, layers: [{texture, depth, anchor}] }`. Each layer is a `PlaneGeometry` + `MeshBasicMaterial({ map, transparent: true })` parented to a `Group`.
3. Drive progress via Lenis: `lenis.on('scroll', ({ progress }) => uniformBus.scrollProgress.value = progress)`.
4. Inside `renderer.setAnimationLoop`, for each chapter compute `local = (global - chapter.start) / (chapter.end - chapter.start)`, clamp 0–1, and update group positions with `THREE.MathUtils.lerp(startZ, endZ, easeOutCubic(local))`.
5. Use GSAP for any DOM overlays (text, chapter chrome): `gsap.timeline({ scrollTrigger: { trigger: section, start: 'top top', end: '+=200%', scrub: 1.2, pin: true } })`.
6. Preload textures per chapter with a 2-chapter look-ahead in a `LoadingManager`; show a chunked progress bar.
7. Performance: cap `renderer.setPixelRatio(Math.min(devicePixelRatio, 2))`, throttle non-visible chapters with `frustumCulled`, and disable rendering when tab hidden (`document.visibilitychange`).
8. Responsive: clamp camera frustum to `min(viewportAspect, designAspect)` and scale a master `Group` so illustration framing never crops faces.
9. Mobile fallback: detect `(pointer: coarse)`, swap to a non-WebGL `IntersectionObserver` slideshow of static frames + CSS keyframes.

WHY IT WORKS: Continuous scroll-as-time collapses navigation into reading — there's no "click to advance" cognitive load, so attention stays inside the diegesis. The fixed canvas + virtual scroll height inverts the normal page metaphor; the page becomes a camera, not a container. Parallax depth gives weight to flat illustration without resorting to 3D models, which keeps the brand's hand-drawn warmth intact.

GOOD / BAD:
- GOOD: Smooth-scroll-locked Three.js with per-chapter scenes is the cleanest way to deliver a 10-act narrative without route changes; load times stay manageable.
- BAD: ScrollTrigger pinning breaks deep-linking; share URLs lose context. Heavy CPU on low-end Android even with sprite sheets — fallback is required, not optional.

CLEAN APPLICATION: Apply this to Clean's **landing page "How the proxy works" section** by building a 5-chapter scroll narrative that pins a WebGL canvas showing the MCP request lifecycle: chapter 1 raw agent request → chapter 2 tools redacted → chapter 3 TOON compression in-flight → chapter 4 cached context layer → chapter 5 token savings counter ticking down. Each "chapter" is a Three.js scene composed of typographic planes (the JSON/TOON payloads as live-rendered text) with depth parallax. Cap to 300vh of scroll, not 1000 — keep it scannable. The point is to make the abstract proxy mechanic feel physical.

## Sui — https://sui.io (Awwwards SOTD 2026-06-24, Developer Award)

WHAT IT DOES: A Layer 1 blockchain marketing site that uses a WebGL noise-field hero (Perlin-based fragment shader rendered to a full-bleed plane, animated via uniform time + mouse position) sitting behind sharp monospace typography. Layout is a strict 12-column grid (CSS `grid-template-columns: repeat(12, minmax(0, 1fr))`) with section breaks marked by full-bleed horizontal rules and oversize numerals. Interaction model: hover on nav items triggers a magnetic-cursor effect (cursor position lerped toward link center at `0.15` damping), and "section landing" reveals use IntersectionObserver-triggered character-by-character stagger via SplitText. The visual hierarchy reads as: shader hero → eyebrow + 64–96px display headline → 3-up developer feature grid with icon + monospace KPI → CTA strip.

PACKAGES:
- `next@^15.0.0` (App Router, RSC for static marketing)
- `react@^19`, `react-dom@^19`
- `tailwindcss@^4.0` with custom theme tokens
- `motion@^11` (Framer Motion successor) for layout transitions
- `gsap@^3.12` + `SplitText` plugin (paid) for text reveals
- `three@^0.169` + `@react-three/fiber@^9` + `@react-three/drei@^9` for the shader hero
- `lenis@^1.1` for smooth scroll
- `sanity@^3` as headless CMS for blog/docs
- Fonts: `IBM Plex Mono` (Google Fonts) for KPIs and code, `Inter Display` for headlines

IMPLEMENTATION:
1. Hero shader: `<Canvas dpr={[1, 2]}><ShaderPlane /></Canvas>` where ShaderPlane uses a custom GLSL fragment shader with `uniform float uTime; uniform vec2 uMouse;` and renders Simplex noise warped into vertical bands, tinted with the brand teal `#61E5C0` over deep ink `#0F4233`.
2. Animate `uTime` via `useFrame((state) => material.uniforms.uTime.value = state.clock.elapsedTime * 0.15)`; pass mouse with `useFrame` reading `state.mouse.x/y`.
3. Headline reveal: on mount, run `SplitText.create(headline, { type: 'chars' })` then `gsap.from(splits.chars, { yPercent: 110, opacity: 0, duration: 0.9, ease: 'expo.out', stagger: 0.02 })`.
4. Magnetic nav: each `<a>` registers a `pointermove` listener computing `(e.clientX - rect.center.x) * 0.3` and applies via `gsap.quickTo(el, 'x', { duration: 0.4, ease: 'power3' })`.
5. KPI strip: animated number counter using `motion.span` and `useMotionValue` + `animate(motionValue, target, { duration: 2, ease: 'easeOut' })` triggered when in viewport.
6. Section dividers: full-bleed `<hr>` styled as `border-top: 1px solid var(--ink-15)` with a left-anchored numeral `<span class="font-mono text-[200px] leading-none opacity-10">01</span>`.
7. Color tokens (Tailwind `@theme`): `--ink: #0F4233`, `--mint: #61E5C0`, `--bone: #EFEFEF`, `--accent-warm: #D1E3FF`.
8. Performance: render shader only when hero is in viewport (IntersectionObserver pauses `Canvas` frameloop via `frameloop="demand"` then triggers `invalidate()` per RAF when visible).

WHY IT WORKS: Monospace + display serif/grotesque pairing signals "engineering credibility" while the soft shader noise prevents the page from feeling like a terminal printout — warmth and rigor in the same frame. The 12-column grid + oversized section numerals borrows editorial design's hierarchy, which is rare in dev infra and feels premium. Magnetic micro-interactions reward developer-typical hover-heavy exploration.

GOOD / BAD:
- GOOD: This is the highest-fidelity "developer infrastructure that still has taste" template currently on the open web — directly applicable to Clean.
- BAD: SplitText is paid (GSAP Club). The shader hero is the largest LCP contributor; without `frameloop="demand"` it tanks Core Web Vitals on mid-tier mobile.

CLEAN APPLICATION: Rebuild Clean's **landing page hero + KPI strip** by adopting this exact pattern: a shader noise field (re-tint to Clean's brand palette — keep one cool primary and one warm accent), oversized Inter Display headline ("The context layer for every agent" or current copy) with SplitText char-stagger reveal at `expo.out`, magnetic-cursor nav, and a 3-up KPI strip below the fold showing the proxy's animated counters ("89% tokens saved", "12ms p50 added latency", "1,400+ MCPs supported"). Anchor the whole hero in a 12-column grid with a tiny `01` `02` `03` numeral set per section. Use `IBM Plex Mono` for all KPIs and code snippets, `Inter Display` for headlines, body in `Inter` regular.

## Fauna Robotics — https://faunarobotics.com (Awwwards SOTD 2026-06-17, Developer Award)

WHAT IT DOES: A humanoid-robotics company site built on Webflow with a two-color palette (`#EFEFEF` bone background, `#D1E3FF` cornflower accent) and a character-led storytelling system — the robot mascot "Sprout" appears as a hand-drawn line illustration that animates via SVG stroke-dashoffset across sections. Layout is single-column, generous (vertical rhythm of ~140px between sections), with a hero featuring a slow auto-playing muted product video framed by a 1px border with rounded corners (`border-radius: 24px`). Section transitions use a sticky-pinned reveal: each block scrolls in from `y: 40px, opacity: 0` with a 200ms delay between elements, triggered by IntersectionObserver at `rootMargin: '-15% 0px'`.

PACKAGES:
- Webflow (production CMS — but reimplementable as:)
- `next@^15` + `react@^19` + `tailwindcss@^4`
- `motion@^11` for IntersectionObserver-driven reveals
- `lucide-react@^0.400` for icons
- Fonts: `Söhne` (paid Klim) or close substitute `Inter` for body, `GT Sectra` for headlines
- Self-hosted MP4/WebM video with `<video autoplay muted loop playsinline preload="metadata">`
- SVG illustrations of "Sprout" exported from Figma as inline SVG for stroke animation

IMPLEMENTATION:
1. Page shell: `max-w-[1280px] mx-auto px-6` with sections as `py-32` blocks.
2. Hero: `<section class="relative aspect-[16/9] rounded-3xl overflow-hidden border border-ink/10"><video class="absolute inset-0 w-full h-full object-cover" ... /></section>`.
3. Sprout SVG: each `<path>` gets `stroke-dasharray: <length>; stroke-dashoffset: <length>` initially; on intersection, `motion.path` animates `strokeDashoffset` to 0 with `duration: 1.6, ease: [0.16, 1, 0.3, 1]` (expo out).
4. Section reveal pattern (reusable component):
   ```jsx
   <motion.div initial={{ y: 40, opacity: 0 }} whileInView={{ y: 0, opacity: 1 }} viewport={{ once: true, margin: '-15% 0px' }} transition={{ duration: 0.7, ease: [0.22, 1, 0.36, 1] }}>
   ```
5. Color tokens: `--bone: #EFEFEF`, `--corn: #D1E3FF`, `--ink: #0A0A0A`. No other colors. Buttons are `bg-ink text-bone` or `bg-corn text-ink`.
6. Type scale: 14/16/18 (body), 24/32/40 (subheads), 64/80/96 (display). Letter-spacing `-0.02em` on display, `-0.01em` on subheads.
7. Performance: `loading="lazy"` on all non-hero video/img, video poster image inlined as base64 SVG placeholder.

WHY IT WORKS: A two-color palette forces the team to compose with type, scale, and whitespace — which always reads as more confident than a multi-color system. The character mascot is the emotional anchor that prevents "robotics startup" from feeling cold; Sprout makes the brand approachable without becoming childish. Generous vertical rhythm signals "we have nothing to hide and time to spend with us."

GOOD / BAD:
- GOOD: Two-color discipline is the single highest-ROI design move a small team can make — it makes everything else easier.
- BAD: Webflow is the build tool — performance is bound by Webflow's runtime, and there's no path to interactive product demos beyond embedded iframes.

CLEAN APPLICATION: Rebuild Clean's **product/about page** by collapsing the existing palette to exactly two colors (one neutral background, one signature accent — pick from current brand) plus near-black ink, then introduce a single hand-drawn line character that represents the "MCP proxy" — anthropomorphize the gateway as a friendly courier passing context between agents and tools. Animate its line strokes on scroll. Use this character as the emotional through-line across the page (it appears in 4–5 spots, each time in a different pose). Keep all sections at 140px vertical rhythm and use only two button styles.

## Indigo Laboratory — https://www.indigo-laboratory.com (Awwwards SOTD 2026-06-19)

WHAT IT DOES: A jewelry brand site structured as five chaptered "tales," each with its own ambient audio loop, color theme, and motion personality. Navigation is a fixed left-rail vertical menu showing 5 numerals (01–05); clicking morphs the entire page through a WebGL transition (full-screen displacement shader using a noise texture as the distortion map) while crossfading the audio bed. Each tale is built as a horizontally-scrolling track of full-viewport panels with magnetic cursor-following on every interactive element and audio reactivity (low-pass filter cutoff modulated by scroll velocity).

PACKAGES:
- `three@^0.169` + `@react-three/fiber@^9` + `@react-three/drei@^9` + `postprocessing@^6.36` for the displacement transition
- `next@^15` + `react@^19`
- `gsap@^3.12` + `Observer` plugin for scroll/drag tracking
- `tone.js@^15` for audio scheduling + low-pass filter automation
- `howler@^2.2` as fallback audio loader
- `tailwindcss@^4`
- Fonts: A serif display (e.g. `Tiempos Headline`) + a thin grotesque body

IMPLEMENTATION:
1. Audio: pre-load 5 OGG loops (~200KB each) via Tone.Player; route through a `Tone.Filter({ type: 'lowpass', frequency: 1200 })`. On chapter swap, automate filter cutoff from 200 → 12000 Hz over 1.2s, then crossfade gain.
2. Chapter transition: render a full-screen `<Plane>` with a `ShaderMaterial` that samples two textures (`uTextureFrom`, `uTextureTo`) and a noise displacement map. Driven by a `uProgress` uniform animated from 0 → 1 via `gsap.to(uniforms.uProgress, { value: 1, duration: 1.4, ease: 'power2.inOut' })`. The "textures" are dynamic render-targets capturing each chapter's rendered DOM via `html-to-image` → texture upload.
3. Horizontal scroll: GSAP Observer tracks wheel/touch deltas, drives a `transform: translate3d(x, 0, 0)` on a `<div class="flex h-screen">` track.
4. Magnetic cursor: shared `useMagnetic` hook applies `transform: translate(${dx * 0.2}px, ${dy * 0.2}px)` via `gsap.quickTo`.
5. Accessibility: respect `prefers-reduced-motion` → disable WebGL transition, fall back to opacity crossfade; mute audio by default with explicit play affordance.
6. Routing: each chapter is `/tales/[slug]`, with shared layout that owns the audio context so it persists across route changes (Next.js App Router parallel routes).

WHY IT WORKS: Multisensory (sight + sound + motion) chapters create distinct memory anchors — each tale becomes a place, not a page. The unified WebGL transition is the "stitching" that makes 5 disparate worlds feel like one product. Horizontal scroll for narrative content is risky in general, but here it matches the linearity of a "tale" — the user knows they're on a story track, not browsing a catalog.

GOOD / BAD:
- GOOD: The audio + low-pass automation is the kind of polish that signals atelier-level craft and is wildly underused on the web.
- BAD: Audio autoplay is blocked on most browsers; the entire experience requires user gesture, which most visitors don't grant. Horizontal scroll has discoverability issues on desktop trackpads with horizontal-shift gestures.

CLEAN APPLICATION: Apply this **only to Clean's docs/onboarding flow**, not the marketing page. Build a 5-chapter "Getting Started" track (1: install proxy → 2: connect first MCP → 3: see token savings → 4: configure caching → 5: deploy to prod). Each chapter has its own subtle UI accent color and a very quiet (-20dB) audio cue on chapter entry (NOT continuous loops — single 200ms tones, like macOS system sounds). Use the WebGL displacement transition between chapters. Skip the horizontal scroll — keep vertical. The audio cue + visual transition combo gives onboarding a sense of "completing a quest" that step-list docs cannot.

## IL CAPO PRODUCTION — https://www.ilcapoproduction.com (Awwwards SOTD 2026-06-26)

WHAT IT DOES: A video production studio site that opens with a full-bleed muted hero reel auto-playing under a massive serif wordmark (160–240px desktop) with a CSS `mix-blend-mode: difference` overlay so the type reads white on any video frame. Project grid below is a 3-column asymmetric masonry where each tile is a 16:9 thumbnail that swaps to an inline auto-playing muted preview on hover (after a 300ms intent delay). Click opens a route-level modal that scales the tile up to full viewport (FLIP animation via Motion's `layoutId`) while loading the full-bitrate reel.

PACKAGES:
- `next@^15` + `react@^19` (App Router with route-intercepting modals)
- `motion@^11` for `layoutId`-driven FLIP transitions
- `tailwindcss@^4`
- `lenis@^1.1` for smooth scroll
- `mux-player@^3` or `@mux/mux-node` for adaptive bitrate video
- `gsap@^3.12` for the hero text reveal
- Fonts: An editorial serif (e.g. `GT Super Display`, `Migra`, or `Ogg`)

IMPLEMENTATION:
1. Hero: `<section class="relative h-screen overflow-hidden"><video class="absolute inset-0 w-full h-full object-cover" autoplay muted loop playsinline /><h1 class="absolute inset-0 grid place-items-center text-[clamp(96px,20vw,240px)] font-serif text-white mix-blend-difference">IL CAPO</h1></section>`.
2. Tile hover-preview: on `pointerenter`, set `setTimeout(() => preview.play(), 300)`; on `pointerleave`, `clearTimeout` and `preview.pause(); preview.currentTime = 0`.
3. FLIP modal: `<motion.div layoutId={`project-${slug}`}>` on both the tile and the modal route — Motion handles the shared-layout interpolation. `transition={{ duration: 0.6, ease: [0.32, 0.72, 0, 1] }}`.
4. Masonry: CSS grid with `grid-auto-rows: 1px` + JS computing `grid-row-end: span ${Math.ceil(height/1)}` per tile (or use `columns: 3` for a simpler flow).
5. Performance: muted video previews use a low-bitrate 720p H.264 with `preload="metadata"`; full-quality is Mux HLS loaded only on modal open.
6. Type: serif display with `font-feature-settings: 'liga', 'dlig', 'kern'` and `text-rendering: optimizeLegibility`.

WHY IT WORKS: `mix-blend-mode: difference` solves the hardest problem in video heroes — type contrast on motion — with a single CSS property. The hover-to-preview-then-FLIP-to-detail pattern is best-in-class for "show, don't tell" and reduces decision fatigue: the visitor sees the work before committing a click.

GOOD / BAD:
- GOOD: `mix-blend-mode: difference` over autoplaying video is the most reliable way to make typography work on heroes without resorting to a darkening gradient.
- BAD: Multiple simultaneous autoplaying videos on the grid can saturate mid-tier mobile decoders; needs aggressive throttling to "only the hovered preview plays."

CLEAN APPLICATION: Apply this to Clean's **case studies / customer logos section** by building a 6-tile grid where each tile is a short (8s, looping, muted) screen recording of a customer's agent running through Clean's proxy — instead of static logos. Hover triggers play after 300ms intent delay, click FLIPs to a full case study with the full demo video plus the customer quote and the token-savings KPI. Use the `mix-blend-mode: difference` trick on the section heading ("Built by teams shipping agents in production") layered over a montage of all 6 demos playing in muted slow grid. This converts "logo wall" into "evidence wall."

## BUILD DIRECTIVE

TARGET: Clean's **landing page hero + KPI strip + "How the proxy works" scroll section** (single contiguous build).

APPROACH: Combine three patterns from today's scout into one cohesive landing surface. Steal Sui's monospace-meets-display typography, shader noise hero, magnetic nav, and SplitText char-stagger reveal — this is the credibility foundation. Below the hero, build a 3-up animated KPI strip in IBM Plex Mono with `useMotionValue` counters tied to viewport entry. Then implement Ten Years Away's pinned-canvas chapter scroll pattern (collapsed to 5 chapters and 300vh, not 10 chapters and 1000vh) to walk the visitor through the MCP proxy lifecycle: raw agent request → tool redaction → TOON compression → cached context layer → token savings counter. Each chapter is a Three.js scene composed of typographic planes rendering the live JSON/TOON payload as text textures, with depth parallax. The whole experience must work without WebGL on mobile (fall back to an `IntersectionObserver` 5-step slideshow with CSS keyframes).

STACK:
- `next@^15.0.0` (App Router, RSC)
- `react@^19`, `react-dom@^19`
- `tailwindcss@^4.0` with CSS variables for tokens
- `motion@^11` (Framer Motion successor) for KPI counters + section reveals
- `gsap@^3.12.5` + `gsap/ScrollTrigger` + `SplitText` (GSAP Club) for hero text reveal and pinned chapter timeline
- `three@^0.169.0` + `@react-three/fiber@^9` + `@react-three/drei@^9` for shader hero and chapter canvas
- `lenis@^1.1.13` for smooth scroll, synced with ScrollTrigger
- `IBM Plex Mono` (Google Fonts) for KPIs/code; `Inter Display` for headlines; `Inter` for body
- Custom GLSL fragment shader for hero noise (Simplex noise, two-color tint from brand palette)

REFERENCE URLS:
- https://sui.io — primary typographic/grid/shader reference
- https://ten.375.studio/en — primary scroll-narrative reference for the proxy lifecycle section
- https://faunarobotics.com — palette discipline reference (two-color rule)

KEY CONSTRAINTS:
- Clean is developer infrastructure. Avoid: gradient meshes that feel like generic AI startup landing pages (Stripe-clone gradients, Vercel-clone radial blurs without earned reason), oversized lottie illustrations of robots/clouds/abstract blobs, "AI sparkle" gimmicks, marketing copy that names a number without showing the math behind it.
- Mandatory: monospace for all numeric proof points, real code snippets (not screenshots) using a syntax-highlighted block (`shiki` at build time, not runtime), p50/p99 latency labels with units, an LCP budget of ≤2.0s on a 4G throttled mid-tier Android.
- The hero shader must render only when in viewport (`frameloop="demand"` + IntersectionObserver `invalidate()`).
- All animation must respect `prefers-reduced-motion`: shader pauses to a static frame, chapter scroll collapses to a vertical step list, SplitText reveals collapse to a single fade-in.
- The chapter canvas must not pin for more than 300vh — if the lifecycle doesn't fit in 5 acts × 60vh, cut acts.
- Do not autoplay audio. Do not add a cursor trail. Do not add scroll-jacking that breaks find-in-page or anchor links.
