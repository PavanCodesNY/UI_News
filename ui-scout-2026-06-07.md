## Apechain — https://www.awwwards.com/sites/apechain

WHAT IT DOES: Awwwards SOTD + Developer Award on 2026-06-06. A single-page WebGL hero that renders a real-time procedural "cultural playground" — a tiled isometric world with shader-driven hover states acting as the entry point to an otherwise dense Web3 infrastructure product. Layout system is a fixed full-viewport `<canvas>` underlay with absolutely-positioned HTML overlays for nav, CTA, and stat counters; scroll is hijacked by Lenis and pumped into a `useFrame` clock that animates camera dolly and material uniforms (`uTime`, `uScrollProgress`) rather than translating sections.

PACKAGES:
- next@15 (App Router)
- three@0.169
- @react-three/fiber@9
- @react-three/drei@10
- @react-three/postprocessing@3 (Bloom, Vignette, ChromaticAberration)
- gsap@3.13 + @gsap/react (ScrollTrigger now free post-Webflow acquisition)
- lenis@1.2 (smooth scroll, drives R3F clock)
- tailwindcss@4
- framer-motion@12 (HTML overlay micro-motion)
- satori + next/font for variable display type (Geist / NeueHaasGrotesk)
- glsl shader-loader (raw GLSL imports for tile fragment shader)

IMPLEMENTATION:
1. Root layout: `<Canvas dpr={[1, 1.5]} gl={{ antialias: false, powerPreference: 'high-performance' }}>` fixed inset-0, behind a `<main>` with `pointer-events-none` so HTML overlays own clicks.
2. Sync Lenis to R3F: in a `useEffect`, `lenis.on('scroll', ({ scroll, limit }) => scrollProgress.current = scroll/limit)`. Inside `useFrame`, lerp camera position along a `CatmullRomCurve3` waypoint path using `MathUtils.damp(current, target, 4, delta)`.
3. Tile field: `<instancedMesh args={[null, null, 4096]}>` of a hexagonal `RingGeometry`, with a custom `ShaderMaterial`. Vertex shader displaces y by `sin(uTime*0.4 + position.x*0.3) * 0.15`; fragment shader mixes a 2-color palette via `smoothstep(0.2, 0.8, fract(uScrollProgress*10.0 + vUv.x))`.
4. Hover interaction: raycast on `onPointerMove`, write hovered instance ID into a `Float32BufferAttribute('aHover', n)` updated via `setX(id, 1.0)` then ease back to 0 in `useFrame`.
5. HTML overlay: CSS grid `grid-cols-12 gap-6 px-8`, hero headline uses `text-[clamp(48px,9vw,168px)]` with `font-variation-settings: 'wght' 720, 'opsz' 144`, paired with a `mix-blend-difference` color so it reads over any canvas tone.
6. Counter pills animate with Framer Motion `<motion.span animate={{ y: 0, opacity: 1 }} transition={{ type: 'spring', stiffness: 320, damping: 28 }}>`.
7. Performance: cap DPR at 1.5, throw `<PerformanceMonitor onDecline={() => setDpr(1)}>` from drei, suspend canvas on `prefers-reduced-motion: reduce`, swap to a static poster `<Image>` below `md:` breakpoint.
8. Breakpoints: `sm:` switches canvas to a poster, `lg:` enables full instanced field, `2xl:` raises instance count to 8192.

WHY IT WORKS: Figure-ground hierarchy is inverted — the WebGL field is the figure, type sits on it like signage on architecture, which signals "the product IS the medium." Color contrast is engineered (display text uses `mix-blend-difference` so legibility is guaranteed against any tile color), and motion semantics map scroll progress to a single global uniform so every animated element feels causally linked rather than independently triggered. This is the same trick Apple uses on AirPods scroll pages.

GOOD / BAD:
- GOOD: Single shared `useFrame` clock means 60fps with 4k instances on M-series; no animation library jitter.
- BAD: Mobile falls back to a static poster — the "wow" is desktop-only, and the lighthouse score takes a 14-point hit from the postprocessing chain.

CLEAN APPLICATION: Apply this to Clean's landing page hero by replacing the tile field with a procedural node-graph of MCP servers — each instance is a hex tile representing a tool/MCP, and the shader pulses outward from a center "Clean proxy" node along the same `uScrollProgress`. On scroll, the camera dollies through the graph from a wide topology view down to a single edge labelled "Claude → Clean → 47 MCPs", revealing the product demo HUD as overlay HTML. Use `mix-blend-difference` for the wordmark so it reads across the gradient.

## The Monolith Project — https://www.webgpu.com/showcase/the-monolith-project/

WHAT IT DOES: A 13-scene scroll-driven WebGL short film built with React Three Fiber that transitions seamlessly between hand-drawn 2D illustration and fully-lit 3D scenes via shader-based morph. Layout is a single sticky `<Canvas>` with no DOM page transitions — scenes are swapped inside R3F through a `Composable Materials` system, a `Composable Particle System`, and a custom deferred renderer that lets each scene compose lights, post effects, and outline passes from shared primitives.

PACKAGES:
- three@0.169
- @react-three/fiber@9
- @react-three/drei@10 (useGLTF, Sparkles, shaderMaterial)
- @react-three/postprocessing@3 (custom deferred renderer wrapper)
- gsap@3.13 + ScrollTrigger
- leva@0.10 (dev-only scene tuning)
- maath@0.10 (easing + vec helpers)
- zustand@5 (global scene/transition state store)
- tone@15 or howler@2.2 (synced audio cues per scene)
- vite@6 + vite-plugin-glsl

IMPLEMENTATION:
1. Sticky canvas: `<div className="h-[1300vh]"><div className="sticky top-0 h-screen"><Canvas /></div></div>` — page is 13 viewport-heights tall and the canvas stays pinned.
2. Scene controller: zustand store with `{ activeScene, progress, transitionAlpha }`. A top-level `<ScrollDriver>` uses ScrollTrigger to map scroll to `progress` (0..13), then derives `activeScene = Math.floor(progress)` and `transitionAlpha = progress % 1`.
3. Composable materials: define `useComposableMaterial({ base: 'standard', layers: [outline, hatch, gradientFresnel] })` that returns a `THREE.ShaderMaterial` whose fragment chunks are stitched at build time. Each scene picks layers, no monolithic shader.
4. Deferred renderer: render scene to MRT G-buffer (`albedo`, `normal`, `depth`, `id`), then run a screen-space pass that draws outlines from depth+normal discontinuities (Sobel on `normal`, threshold on `depth` derivative). This is what gives the Moebius-style line work.
5. 2D→3D morph: each scene root holds two children, `<Sketch2D/>` (a textured `<Plane>` with hand-drawn art) and `<Scene3D/>`. Fragment shader on the plane samples both via `mix(sketchTex, sceneRender, smoothstep(0.4, 0.7, uMorph))` while the 3D camera lerps from orthographic (matches sketch FOV) to perspective.
6. Particles: a single instanced `<Points>` of 50k particles with a per-scene `uPattern` uniform; behavior (snow, dust, embers) is selected by `uPattern` rather than separate components.
7. Audio: scene-keyed `Tone.Player` triggers on `activeScene` change, crossfade duration matches the GSAP scrub.
8. Performance: `frameloop="demand"` + `invalidate()` only on scroll change; particles use `gl_PointSize` based on distance not geometry; outline pass runs at 0.75 resolution and is upsampled.
9. Mobile: detect `navigator.deviceMemory < 4`, drop deferred pass, swap to forward + cheap fresnel outline; reduce particles to 8k.

WHY IT WORKS: This is composition over inheritance applied to rendering — each visual effect is a primitive that scenes opt into, so 13 distinct looks share one engine. The 2D-to-3D shader morph is a continuity device: the viewer never registers a "page change," which keeps narrative tension. Gestalt continuity (the line that starts as a sketch becomes a 3D edge) makes the whole 7-minute experience feel authored, not assembled.

GOOD / BAD:
- GOOD: Production-quality scroll-driven storytelling that reads like film — proves that "marketing site" and "interactive narrative" can be the same artifact.
- BAD: Asset weight (~85MB first paint without aggressive code-splitting per scene), and the audio autoplay requires a user gesture so the first scene loses its score until the user scrolls.

CLEAN APPLICATION: Apply this to Clean's product demo section by building a 4-scene scroll-driven narrative ("Before Clean: chaos → Connect: handshake → Route: traffic visualized → After: clean signal"). Use the composable materials pattern — one base `<MCPNodeMaterial>` with opt-in layers (`pulse`, `congestion`, `tokenStream`) so all 4 scenes share geometry but each reads visually distinct. The 2D→3D morph is the metaphor: scene 1 is a literal terminal log (textured plane), scene 4 is a 3D topology graph — same shader morphs between them as scroll progresses.

## Scroll-Revealed WebGL Gallery (Codrops Tutorial) — https://tympanus.net/codrops/2026/02/02/building-a-scroll-revealed-webgl-gallery-with-gsap-three-js-astro-and-barba-js/

WHAT IT DOES: A multi-page WebGL image gallery where DOM `<img>` elements are mirrored 1:1 by Three.js planes; on scroll, a fragment shader reveals each image via a noise-driven mask; on click, the plane Flip-animates into a full-viewport detail view and Barba.js swaps the underlying page without unmounting the WebGL context. Layout is a vertical CSS grid of image cards with `getBoundingClientRect()` measurements piped to plane positions every frame.

PACKAGES:
- astro@5
- three@0.169
- gsap@3.13 + ScrollTrigger + ScrollSmoother + Flip + SplitText (all now free)
- @barba/core@2.10
- vite-plugin-glsl
- lenis@1.2 (alternative to ScrollSmoother)
- sharp (Astro image optimization)

IMPLEMENTATION:
1. Astro layout: persistent `<canvas id="webgl">` outside the Barba container, so it survives route transitions. Pages live inside `<main data-barba="container">`.
2. Plane sync: for each `.gallery-img`, create a `THREE.Mesh(new PlaneGeometry(1,1), galleryMaterial.clone())` and on every `ScrollSmoother` update call `plane.position.set(rect.left + rect.width/2 - vw/2, -(rect.top + rect.height/2 - vh/2), 0)` in pixel-space (camera is orthographic, FOV maps 1 unit = 1 pixel).
3. Reveal shader: fragment samples `uTexture`, then a `noise(vUv * 8.0)` mask is compared to `uProgress` driven by ScrollTrigger `onUpdate`. `float m = smoothstep(uProgress - 0.05, uProgress + 0.05, noise); gl_FragColor = vec4(color.rgb, m);`
4. Hover: GSAP tween on `material.uniforms.uHoverStrength.value` to 1, vertex shader displaces the plane along normal by `uHoverStrength * 0.05 * sin(vUv.x * PI)`.
5. Click → detail: capture initial `Flip.getState(plane)`, mutate plane to full-viewport bounds, `Flip.from(state, { duration: 0.9, ease: 'expo.inOut' })`. Barba.js fires `beforeLeave` to keep the plane alive and `enter` to swap DOM content underneath.
6. Page transitions: Barba `transitions: [{ leave: ({ current }) => gsap.to(current.container, { opacity: 0, duration: 0.4 }), enter: ({ next }) => gsap.from(next.container, { opacity: 0, duration: 0.4 }) }]`. WebGL context never dies.
7. Resize: re-measure all rects, update camera orthographic frustum to viewport, no shader recompile.
8. SplitText hero: `new SplitText('.hero-h', { type: 'chars,lines' })`, then `gsap.from(split.chars, { y: 100, opacity: 0, stagger: 0.02, ease: 'expo.out' })`.

WHY IT WORKS: Spatial coherence — the image in the grid and the image in the detail view are literally the same WebGL plane, so the brain stitches them as a single object moving, not two separate elements. This is Gestalt common-fate operating across a route change, which is rare and feels expensive even when it isn't. Noise-mask reveals replace the cliché fade-in with something that reads as material (paint, fog) rather than CSS.

GOOD / BAD:
- GOOD: One WebGL context across routes is the right architecture — no flash, no reload, accessible because the DOM is real `<img>` and `<a>` underneath.
- BAD: If the user resizes mid-transition the rect math drifts for a frame; debounce isn't quite tight enough in the reference implementation.

CLEAN APPLICATION: Apply this to Clean's docs navigation by treating every code block in the docs as a WebGL plane mirrored to a `<pre>` element. On scroll, blocks reveal with the same noise mask (in Clean's case, the noise is replaced by a horizontal "decode" sweep using ASCII glyphs). On click into a block, Flip animates it to a full-viewport playground where the user can edit and re-run — the underlying Barba page swap loads the playground route without reloading the syntax highlighter or losing the WebGL background.

## Aceternity UI — Canvas Text + Text Generate Typewriter — https://ui.aceternity.com/changelog

WHAT IT DOES: Two new (March 2026) text animation components. `Canvas Text` renders headline copy by rasterizing the type to a hidden canvas, sampling the alpha channel, then drawing thousands of curved bezier paths clipped to the text shape — output looks like type filled with flowing colored ink. `Text Generate Typewriter` types characters one-by-one with synced mechanical keyboard sound samples (key down + up). Both are drop-in React components shipped via the shadcn registry.

PACKAGES:
- react@19
- next@15
- tailwindcss@4
- framer-motion@12 (or motion@12 — the post-rebrand name)
- clsx + tailwind-merge (`cn` helper)
- canvas-confetti (optional supporting effects)
- howler@2.2 (for typewriter sound samples)
- `npx shadcn@latest add https://ui.aceternity.com/registry/canvas-text.json`

IMPLEMENTATION (Canvas Text):
1. Component signature: `<CanvasText text="MCP Proxy" colors={['#7C3AED','#EC4899','#06B6D4']} lineCount={2400} />`.
2. On mount, create an offscreen `OffscreenCanvas(w, h)`, draw text with `ctx.font = '900 240px InterDisplay'`, `ctx.fillStyle = '#fff'`, `ctx.fillText(text, 0, baseline)`.
3. Get `ImageData`, walk pixels with stride 4, push `{x, y}` into `points[]` only where `alpha > 128`.
4. Render to a visible `<canvas>` via `requestAnimationFrame`: for each line, pick 4 random points from `points[]` as bezier control points, stroke with `ctx.strokeStyle = colors[i % colors.length]` and `ctx.globalAlpha = 0.08`.
5. Animate: time-shift control points by `Math.sin(t * 0.001 + i)`, redraw 60fps. Clip is implicit because all points sit inside the text mask.
6. Responsive: re-sample on resize with `ResizeObserver`; cap `lineCount` at 1200 on `(max-width: 768px)`.

IMPLEMENTATION (Text Generate Typewriter):
1. `useEffect` with a `setTimeout` chain (or `framer-motion`'s `animate(useMotionValue(0), text.length, { duration: text.length * 0.06 })`).
2. On each tick, append the next char to displayed state. Play `keyDown.mp3` (Howler) at the same tick, schedule `keyUp.mp3` 80ms later.
3. Variable typing rhythm: `delay = baseDelay * (0.7 + Math.random() * 0.6)` so it sounds human, not metronomic.
4. Cursor: `<span className="inline-block w-[2px] h-[1em] bg-current animate-pulse" />`.

WHY IT WORKS: Canvas Text replaces "gradient text" (overused) with rendered media — the headline becomes evidence of work, which is exactly the signal a dev-infrastructure brand needs to send. Typewriter with audio engages an extra modality (sound), which is uncommon on the web and instantly memorable; the Von Restorff effect makes the headline stick. Both components are subtle when paused and demand attention when in motion, which matches the rhythm of a scroll-driven page.

GOOD / BAD:
- GOOD: Canvas-based rendering means it scales independently of DOM, so 2400 paths cost almost nothing on GPU.
- BAD: Typewriter audio is a hard accessibility hit — must be opt-in or trigger only after a user gesture; otherwise it's hostile on muted-by-default mobile.

CLEAN APPLICATION: Apply Canvas Text to Clean's landing page hero as the `One protocol. Every tool.` headline — sample Clean's palette (deep slate `#0A0E1A`, electric violet `#6E56CF`, signal green `#00D4AA`), so the ink-fill metaphor reads as "many tools, one channel." Use Text Generate Typewriter for the `<code>` snippet directly below ("`curl https://clean.run/connect | sh`"), audio off by default with a small speaker toggle — the typewriter rhythm matches the "developer-on-a-keyboard" identity.

## Vaulk — https://www.awwwards.com/sites/vaulk

WHAT IT DOES: Awwwards SOTD 2026-06-01 by LEOLEO + Keny Zachelin. A defense-industrial product site for modular shelter systems. Visual system is anti-marketing: monospace nav, fixed grid lines drawn over content, sectioned cinematic stills with text-as-stencil overlays, and a horizontal scroll-jacked product configurator where pressing arrow keys swaps modules in real time. Layout is a CSS grid with explicit `gridTemplateColumns: 'repeat(12, 1fr)'` and grid lines rendered as 1px `border-r` on a fixed overlay `<div>`.

PACKAGES:
- nuxt@4 or next@15
- gsap@3.13 + ScrollTrigger + Observer (horizontal scroll hijack)
- lenis@1.2
- tailwindcss@4 with `font-mono: 'JetBrains Mono Variable'` and `font-display: 'NeueHaasGroteskDisplay'`
- framer-motion@12 (HUD micro-motion)
- splitting@1.1 (per-char stencil cuts)
- three@0.169 + @react-three/fiber@9 (only for the configurator viewer)
- @react-three/drei (Bounds, OrbitControls disabled, AccumulativeShadows)

IMPLEMENTATION:
1. Page chrome: position-fixed grid overlay — `<div className="fixed inset-0 pointer-events-none grid grid-cols-12">{Array(11).fill(0).map(i => <div key={i} className="border-r border-white/8" />)}</div>`. This grid stays visible above content the entire scroll.
2. Stencil text: `mix-blend-mode: difference` on display headlines so they cut through hero stills with no fill color of their own.
3. Horizontal scroll section: GSAP Observer captures wheel + touch, animates a horizontal `x: -i * window.innerWidth` on a wrapper, while a top progress bar shows section count `01/06`.
4. Configurator: small `<Canvas>` with a single GLTF model, modules toggled via React state, swap is a 0.6s `gsap.to(mesh.scale, { x: 0, ease: 'expo.in' }).then(() => /* swap mesh */).then(() => gsap.from(newMesh.scale, { x: 0, ease: 'expo.out' }))`.
5. Type rhythm: `text-[clamp(11px, 0.9vw, 13px)]` for the mono labels, `text-[clamp(56px, 8vw, 144px)]` for the display headlines, line-height locked at `0.92`.
6. Color: pure `#0A0A0A` background, `#F4F4F1` text, single accent `#FF4D1A` reserved for active states only.
7. Cursor: custom 1px crosshair + bracket pair that snap to nearest grid cell — implemented as a `<div>` with `transform: translate(snap(x), snap(y))` and `pointer-events-none`.
8. Micro-interactions: every button is `border-[0.5px]` with a hover sweep — `before:` pseudo-element animated `translateX(-100%) → 0` on `group-hover`.

WHY IT WORKS: Restraint is the message. The fixed grid signals system, not page. The single accent color obeys the 60/30/10 rule pushed to 90/9/1, which makes any CTA in the accent color feel decisive. Monospace + display pairing is a long-running signal that the brand belongs to people who read source, not marketing copy. Cursor snapping makes the user feel like an operator, not a visitor.

GOOD / BAD:
- GOOD: Best-in-class type system and grid discipline — this is what "technical product" should look like.
- BAD: Horizontal scroll hijacks accessibility — keyboard tab order desyncs from visual order, and a screen reader gets lost.

CLEAN APPLICATION: Apply this to Clean's pricing page and the developer-docs landing. Adopt the fixed 12-col grid overlay at 6% white opacity so every section feels like part of a single system. Use stencil headlines with `mix-blend-mode: difference` over the gradient backgrounds in pricing tiers. Reserve a single accent (Clean's signal green) for the "Get API key" CTA — every other CTA stays outlined. The crosshair cursor is overkill for the whole site, but use the snap-to-grid behavior on the API playground only.

## BUILD DIRECTIVE

TARGET: Clean landing page hero + immediately-following product-demo scroll section. This is the highest-leverage surface — first impression for every visitor, the place where the "developer infrastructure" identity has to land in under 3 seconds.

APPROACH: Build a single-canvas, scroll-driven hero that combines Apechain's full-bleed WebGL underlay, The Monolith Project's composable-material scene system, and Vaulk's grid + monospace chrome. The canvas renders a procedural MCP topology — a `Clean Proxy` core node at center, ringed by instanced hex tiles representing connected MCPs (Slack, Linear, Notion, Postgres, etc.). On scroll, a Lenis-driven progress uniform `uScroll` does three things: (1) dollies the camera from a wide overhead of the graph down to a single edge labeled `Claude → Clean → 47 tools`; (2) reveals the headline via Aceternity Canvas Text, with the ink colors keyed off Clean's palette; (3) types a `curl https://clean.run/connect | sh` snippet character-by-character below the headline using Text Generate Typewriter (audio off, speaker toggle visible). A fixed 12-col grid overlay at 6% white opacity persists across the whole page. The next section morphs the WebGL view (Monolith-style 2D↔3D shader) from the topology graph into a literal terminal log of MCP calls flowing through Clean — same canvas, same materials, different uniform values.

STACK:
- next@15 (App Router) + react@19
- tailwindcss@4 with `font-display: NeueHaasGroteskDisplay`, `font-mono: JetBrains Mono Variable`, `font-sans: Inter Variable`
- three@0.169 + @react-three/fiber@9 + @react-three/drei@10 + @react-three/postprocessing@3
- gsap@3.13 + ScrollTrigger + Flip (free)
- lenis@1.2
- framer-motion@12
- zustand@5 (scroll progress + scene state)
- shadcn/ui v3 + Aceternity registry (`canvas-text`, `text-generate-typewriter`, `spotlight-new`)
- howler@2.2 (typewriter audio, opt-in)
- vite-plugin-glsl + glsl-noise

REFERENCE URLS:
- https://www.awwwards.com/sites/apechain (WebGL hero pattern, blend-mode headline)
- https://www.webgpu.com/showcase/the-monolith-project/ (composable materials + 2D→3D scene morph)
- https://www.awwwards.com/sites/vaulk (grid overlay, monospace nav, restraint)

KEY CONSTRAINTS: Clean is developer infrastructure — the result must feel like instrumentation, not marketing. Avoid: (a) generic glassmorphism cards, (b) AI-startup gradient blobs with no underlying mechanic, (c) stock 3D meshes (no donuts, no abstract shapes — every visible thing must represent an MCP, a request, or a tool), (d) autoplay audio, (e) any motion that exceeds 800ms without scroll input, (f) hero copy that says "AI-powered" or "the future of." Performance budget: <1.8s LCP on M2 / Fast 3G, canvas falls back to a static SVG topology below `md:`, lighthouse perf score ≥ 88 with the canvas active. Accessibility: every animated element must have a reduced-motion path, the typewriter must respect `prefers-reduced-motion: reduce`, and the canvas must have an `aria-hidden="true"` with all semantic content in the HTML overlay.
