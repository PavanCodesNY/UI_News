# UI Scout Report — 2026-06-07

Five technically dense, visually striking UI examples scouted from Awwwards SOTD/Nominee feed, Aceternity UI's latest drops, and recent dev-infra launches. Each entry is implementation-grade: a frontend or design agent can build directly from it without further research.

## 1. Hashgraph Ventures — https://hashgraphvc.com/

WHAT IT DOES: Two-color (#EEEEEE background, #000209 ink) brutalist editorial layout for an early-stage blockchain/AI VC firm. The hero is a full-viewport Three.js scene that renders a procedural lattice of nodes-and-edges (a "hashgraph" gossip-protocol metaphor) that subtly rotates on a y-axis idle animation and reacts to scroll velocity by accelerating its orbit. Below the fold, the page becomes a CMS-driven (Sanity) editorial grid with serif display type, oversized numerals, monospaced metadata, and section breaks marked by sticky scroll-anchored 3D module zooms. There is no color other than the ink/paper pair — all hierarchy is created through type weight, scale, and negative space.

PACKAGES:
- next or nuxt (Nuxt.js 3 used here; Next.js 15 equivalent fine)
- three
- @tresjs/nuxt or three vanilla (Nuxt module pattern)
- gsap with ScrollTrigger
- lenis (smooth scroll)
- @sanity/client + next-sanity (or @nuxtjs/sanity)
- tailwindcss v4
- a serif display font (e.g. PP Editorial New, Söhne Mono for meta, Inter for UI)

IMPLEMENTATION:
- Layout: 12-col grid with 24px gutter, 1440px max-width inner, asymmetric column spans (e.g. headline takes col 1-9, meta takes col 10-12 with 200% line-height monospace).
- Hero: full viewport Canvas, fixed position. Three.js scene with: an InstancedMesh of ~600 nodes positioned on a 3D Fibonacci sphere (radius 4), connected by LineSegments computed from a k-nearest-neighbors graph (k=3) — feed the connectivity into a BufferGeometry of line vertices. Camera at z=8, perspective fov 35. Idle: rotate scene y-axis at 0.0008 rad/frame. On scroll, multiply rotation speed by `1 + (scrollVelocity * 0.02)` and dampen back via lerp at 0.05.
- Materials: nodes use a MeshBasicMaterial in pure #000209; lines use LineBasicMaterial #000209 at opacity 0.18. No lights, no shadows — pure silhouette.
- Type system: H1 clamp(64px, 9vw, 144px) / line-height 0.92 / letter-spacing -0.04em (serif); body 17px/1.55 sans; eyebrow 11px monospace uppercase letter-spacing 0.16em.
- Transitions: GSAP ScrollTrigger pin sections; for module reveals use `gsap.to(camera.position, { z: 4, duration: 1.2, ease: "expo.inOut", scrollTrigger: { trigger: section, scrub: 1 } })`.
- Lenis init: `new Lenis({ duration: 1.2, easing: t => Math.min(1, 1.001 - Math.pow(2, -10 * t)) })`.
- Performance: throttle DPR to `Math.min(window.devicePixelRatio, 2)`, use `WebGLRenderer({ antialias: true, powerPreference: 'high-performance' })`, freeze tweens when `document.visibilityState === 'hidden'`.
- Responsive: <768px collapse to single column, swap Three.js scene for a static SVG render of the same lattice (export from Three.js once, embed as inline SVG).

WHY IT WORKS: The 2-color constraint forces hierarchy to live entirely in typography and whitespace — there's nowhere to hide weak content, so every word has to earn its place. The procedural 3D lattice is a literal metaphor for the firm's name and thesis, which makes the technical execution feel narrative rather than decorative. The asymmetric editorial grid signals high-taste curatorial confidence — the visual language of a magazine, not a corporate website.

GOOD / BAD:
- GOOD: Restraint as flex — proves you don't need gradients/glassmorphism to look premium. The scroll-velocity-coupled 3D rotation is one of the most elegant uses of scroll motion this year.
- BAD: Heavy WebGL hero blocks LCP on mid-tier mobile; the no-color palette can read as "unfinished" to users outside design-literate audiences.

CLEAN APPLICATION: Apply this to Clean's landing page hero by replacing the gradient/dark hero with a pure 2-color palette (off-white #FAFAFA + near-black #0A0A0A) and a Three.js InstancedMesh visualization of an "MCP context graph" — nodes representing tools/agents, edges representing context flow, with the same scroll-velocity rotation. Wrap the headline in an editorial 12-col grid with monospace eyebrow ("v0.4 · context layer for agents"), serif display H1, and monospaced meta sidebar listing supported protocols. The whole homepage above the fold should commit to ink-on-paper austerity.

## 2. Apechain — https://apechain.com/

WHAT IT DOES: A maximalist, gamified Web3 marketing site by makemepulse (Next.js + WebGL). The structure is a vertical scroll-narrative built around four signature "elements" the studio explicitly calls out: an animated hero header with a 3D Ape character, a spotlight section that uses a radial mask to reveal content under a cursor-tracking light, an "All apps" grid that mosaics ecosystem dApps as draggable physics tiles, and a playful footer with elastic SVG typography and bouncy 3D objects. The entire site runs at sustained 60fps with WebGL composited under DOM, using shared GSAP timelines synced to scroll position.

PACKAGES:
- next@15
- react@19
- three
- @react-three/fiber, @react-three/drei
- gsap (Club GreenSock for SplitText, MorphSVG)
- @studio-freight/lenis (smooth scroll)
- matter-js (2D physics for tile grid)
- howler (audio for micro-interactions)
- tailwindcss v4
- @next/font (custom display font — likely PP Neue Machina or similar geometric)
- lucide-react (icons)

IMPLEMENTATION:
- Layout: full-bleed sections, each ~100-150vh. Persistent WebGL canvas fixed at z-index -1, DOM overlays at z-index 1.
- Hero: R3F Canvas with a GLB-loaded character (Draco-compressed), idle animation mixed from baked clip, head-tracks cursor via lookAt(new THREE.Vector3(mouseX*2, mouseY*1, 5)). Background: shader plane with a custom fragment shader running a 2D noise field tinted in brand colors (blue-purple-orange gradient).
- Spotlight section: a fixed div with CSS mask `mask-image: radial-gradient(circle 200px at var(--mx) var(--my), black 0%, transparent 100%)`. Update --mx/--my from a pointermove listener with rAF batching.
- "All apps" tile grid: Matter.js engine driving 18 rectangular bodies in a fixed bounding box. Each DOM tile reads `body.position` and `body.angle` each frame, applied via `transform: translate3d(x, y, 0) rotate(angle)`. Mouse becomes a kinematic body that pushes tiles. Drop tiles back to grid positions on idle (5s timeout) using `Body.setPosition` with lerp.
- Footer: SVG text with GSAP MorphSVG morphing letterforms on hover. 3D objects are small Three.js instances (~200 verts each) bouncing with simple gravity sim, capped at 12 active.
- Animation orchestration: one master GSAP timeline driven by `ScrollTrigger.scrub: 0.6`. Each section registers labels.
- Audio: Howler-managed sprite sheet, debounced 50ms, only plays after first user gesture.
- Performance: Suspense-wrap R3F scenes, lazy-load matter-js, use `useFrame((state) => { if (state.clock.elapsedTime % 0.033 < 0.016) update() })` to throttle expensive systems to 30fps where acceptable.
- Responsive: <1024px disable Matter.js (static CSS grid), <768px drop hero 3D character for a hand-illustrated SVG.

WHY IT WORKS: Every section uses a different interaction language (3D character, spotlight, physics tiles, morphing type) — variety prevents scroll fatigue, but a consistent color palette and typography binds it. The "spotlight" pattern in particular delivers strong contrast between what's revealed and what's hidden, satisfying Gestalt figure-ground perception. The physics grid creates a "loot box" feel — the user feels they discovered something rather than read a feature list.

GOOD / BAD:
- GOOD: The orchestration discipline — one master timeline, shared canvas — is what keeps a maximalist site from feeling chaotic.
- BAD: 6MB+ JS payload, heavy CPU on mobile Safari, and tile physics get genuinely confusing as a navigation pattern when users want to actually find an app.

CLEAN APPLICATION: Apply the spotlight technique to Clean's product demo section. Build a section titled "See what your agent sees" where the default view shows a wall of obfuscated/redacted tool definitions (gray-on-gray noise), and the user's cursor reveals a 320px radius spotlight showing actual MCP tool schemas — `clean_proxy_call`, `context.recall`, etc. — through the CSS `mask-image: radial-gradient(circle 320px at var(--mx) var(--my), black 0%, transparent 100%)` pattern. Pair with a fixed headline like "Your agent's context, finally visible." This makes Clean's value proposition (context visibility) into a literal interaction.

## 3. Steven.com — https://steven.com/

WHAT IT DOES: An immersive 3D-motion portfolio for Steven Bartlett's creator-economy ecosystem (Diary of a CEO, Flight Group, etc.), built on Webflow + custom Three.js layer. Standout elements: a preloader that doubles as a brand intro (a slow type-in of "STEVEN" with a counter), a menu overlay that opens with a per-letter SplitText slide-in + 3D camera dolly to a "menu world," "unusual" navigation where each nav item is a 3D card floating in space (clicking triggers a portal-style page transition), and a particle-trail mouse animation on subpages (a chain of GPU-instanced sprites following the cursor with verlet-spring physics).

PACKAGES:
- webflow (host) OR next@15 if rebuilt
- three
- gsap + SplitText + CustomEase
- lenis
- splitting (alternative to SplitText if no Club GSAP)
- barba.js (if non-Webflow) for SPA page transitions
- a variable display font (e.g. Mori Variable, PP Right Grotesk)

IMPLEMENTATION:
- Preloader: full-screen overlay, monospace counter incrementing 000→100 over actual asset load time (use `THREE.LoadingManager` for real progress). On 100, GSAP timeline: counter slides up, "STEVEN" wordmark splits letter-by-letter via SplitText with stagger 0.04, ease `power3.out`, then mask wipes top→bottom with `clip-path: inset(0 0 0 0)` over 0.8s.
- Menu overlay: trigger creates a fullscreen `position:fixed` div with backdrop-filter blur(20px). Inside: 6 nav items as SplitText words, each animated in with `{ yPercent: 100, rotateX: 90 }` → `{ yPercent: 0, rotateX: 0, stagger: 0.05, ease: 'expo.out', duration: 0.9 }`. Behind menu, the WebGL camera dollies from `z=8` to `z=2` over 0.6s with `power4.inOut`.
- 3D nav cards: each nav target is a Three.js Mesh (PlaneGeometry 2×3) with a video/texture face. Float idle: `position.y = baseY + Math.sin(elapsed + i*0.5) * 0.05`. Hover: scale to 1.08 with elastic ease.
- Page transitions: portal effect = render current page to a RenderTarget, animate a circular mask shader from 0→1 radius centered on clicked card while loading new page. Shader uniform `uProgress` driven by GSAP.
- Particle trail: array of 24 sprites, each frame: `sprite[i].position.lerp(sprite[i-1].position, 0.2)`, head follows raycasted mouse-to-z=0 plane. Use InstancedMesh + per-instance matrix updates, not 24 individual meshes.
- Smooth scroll via Lenis at `duration: 1.4, easing: t => 1 - Math.pow(1-t, 4)`.
- Performance: cap pixel ratio at 1.5, render-on-demand (call `renderer.render()` only on scroll/pointer events for static sections).

WHY IT WORKS: The preloader sets a premium ceremony — users are primed to slow down and pay attention. The menu's per-letter motion and camera dolly create a sense of moving through space, not just opening a UI panel. The particle trail is purely sensual feedback that says "this site is alive" without being a feature — it's atmosphere. The unifying principle is motion-as-language: every interaction has a physical weight.

GOOD / BAD:
- GOOD: Choreography across separate systems (preloader → menu → 3D nav → page transition) feels like one continuous performance.
- BAD: A 4-5 second preloader is hostile to direct-response traffic; mobile particle trail spikes the GPU and shortens battery noticeably.

CLEAN APPLICATION: Apply Steven.com's preloader + per-letter menu pattern to Clean's onboarding flow. The signup/connect flow opens with a 1.5s "Provisioning context layer..." preloader (real progress from API calls, formatted as monospaced 000→100 counter), then the onboarding steps reveal with SplitText word-stagger animations. For docs navigation, replace the standard sidebar with a 3D-card menu where each top-level doc section (Proxy, Context, Schemas, Auth) is a floating Three.js plane that the user dollies through — using the camera-dolly transition to move between sections instead of a page reload.

## 4. Vaulk — https://vaulk.com/

WHAT IT DOES: A defense/architecture brand by LEOLEO (Vue.js + Nuxt.js + GSAP) for modular shelter systems. The site leans into industrial/military design semantics: blueprint-style 3D wireframe explorations of the product, Rive-animated technical diagrams that play on scroll, and a signature "pixel page transition" — when navigating, the outgoing page resolves into a low-res pixel grid that scrambles, then re-renders as the incoming page (using a canvas-based dither shader). Color palette is military neutral: off-white, charcoal, signal-orange accent.

PACKAGES:
- nuxt@3
- vue@3
- three
- gsap + ScrollTrigger + Flip
- @rive-app/canvas (or @rive-app/webgl)
- ogl (lightweight WebGL — possible alternative to three)
- lenis (or @studio-freight/lenis-vue)
- tailwindcss
- a technical monospace (e.g. JetBrains Mono, Berkeley Mono)

IMPLEMENTATION:
- 3D wireframe: load product GLB, override material with `MeshBasicMaterial({ wireframe: true, color: 0x111111 })`. Add a secondary EdgesGeometry pass for crisper outlines: `new THREE.LineSegments(new THREE.EdgesGeometry(geometry, 15), edgeMaterial)`. Camera orbits on scroll via ScrollTrigger scrub mapping `scrollProgress → camera azimuth angle`.
- Rive animations: export technical diagrams from Rive with timeline state machines. Mount via `<Rive src="/diagram.riv" stateMachines="Default" />`. Trigger inputs from ScrollTrigger callbacks: `stateMachine.input('progress').value = scrollProgress * 100`.
- Pixel page transition: on route change, capture current page via `html2canvas` or render to an offscreen canvas. Pass that texture to a shader plane with a fragment shader that quantizes to a pixel grid: `vec2 uv = floor(vUv * uPixels) / uPixels; gl_FragColor = texture2D(uTexture, uv) * step(rand(uv), uReveal);`. Animate `uPixels` from 200 → 20 → 200 over 0.9s and `uReveal` as a noise threshold dissolve.
- Smooth scroll: Lenis, lock during transitions.
- Layout: 8-col grid with brutalist proportions, oversized monospaced numerals for section markers ("01 / SHELTER", "02 / DEPLOY").
- Typography: serif H1, monospace meta, sans body. Strong type contrast (display 8x body size).
- Color: bg #F2F0EC, ink #111111, signal #FF4F1F (used sparingly — borders, hovers, single CTA).
- Performance: Rive renders to canvas not DOM; lazy-instantiate per viewport via IntersectionObserver.

WHY IT WORKS: The wireframe-and-blueprint visual language is a perfect semantic match for the product (architectural shelter systems). Rive's vector-precise technical diagrams scale to any DPR with zero file-size penalty — the right tool for the job versus heavy raster GIFs. The pixel transition is more than decoration: it semantically reinforces "modularity" — pages dissolve into and reassemble from a grid of units.

GOOD / BAD:
- GOOD: Semantic motion — every animation reinforces the product story rather than just showing off.
- BAD: Pixel transition adds 600-900ms perceived latency between pages; some users will read this as "slow" not "intentional".

CLEAN APPLICATION: Apply the wireframe + Rive approach to Clean's product demo / architecture section. Replace any rendered "how it works" diagram with: (1) a Three.js wireframe rendering of Clean's architecture (Agent → MCP Proxy → Tool Servers) where the camera scroll-orbits the model, exposing each layer as the user scrolls, and (2) Rive-animated request flows showing TOON-format payloads moving through the proxy with per-step state inputs driven by scroll position. Pair with monospace numeric section markers ("01 / INGEST", "02 / NORMALIZE", "03 / DISPATCH"). The technical-blueprint aesthetic is exactly the credibility signal a dev-infra company needs.

## 5. Aceternity Canvas Reveal Effect (Clerk-style hover dot matrix) — https://ui.aceternity.com/components/canvas-reveal-effect

WHAT IT DOES: A copy-paste React component that renders a GLSL dot-matrix on a canvas via a Three.js ShaderMaterial. On hover (or always-on static variant), the dots animate via a per-instance opacity wave whose phase is driven by 2D noise + radial distance from the cursor, producing the "expanding dot field" effect made famous on Clerk's homepage. Each dot is procedurally drawn in a fragment shader — there are no DOM nodes — so it scales to thousands of dots with negligible CPU cost. Configurable props: animationSpeed (default 0.4), opacities (array of 10 stops), colors (RGB triplets), dotSize, showGradient.

PACKAGES:
- next@15 or react@19
- three
- @react-three/fiber
- framer-motion (motion@latest — for the wrapping fade/hover state)
- tailwindcss v4
- shadcn/ui registry (install via `npx shadcn@latest add @aceternity/canvas-reveal-effect`)

IMPLEMENTATION:
- Install: `npx shadcn@latest add @aceternity/canvas-reveal-effect`.
- Structure: a `<CanvasRevealEffect>` wraps a `<DotMatrix>` Three.js plane inside an R3F `<Canvas>`. Plane geometry is full-viewport quad.
- Shader (fragment, simplified): for each fragment, compute `vec2 cellId = floor(vUv * uTotalSize)`; pick an opacity from `uOpacities` indexed by `mod(cellId.x + cellId.y, 10)`. Draw the dot via `smoothstep(uDotSize, uDotSize - 0.02, length(fract(vUv * uTotalSize) - 0.5))`. Modulate brightness by `sin(uTime * uSpeed + cellId.x * 0.3 + cellId.y * 0.3)`.
- Hover variant: wrap the `<CanvasRevealEffect>` in a parent div with `onMouseEnter` setting an `isHovering` state, then crossfade the canvas via Framer Motion: `<motion.div animate={{ opacity: isHovering ? 1 : 0 }} transition={{ duration: 0.4 }} />`.
- Optional gradient overlay: absolute-positioned `bg-gradient-to-t from-background via-transparent to-transparent` mounted over the canvas to fade dots into the card background.
- Props in practice: `animationSpeed={3} containerClassName="bg-emerald-900" colors={[[0,255,255],[0,255,170]]} dotSize={2}`.
- Performance: maxFps cap of 60 inside the ShaderMaterial; pause animation when card is off-screen via IntersectionObserver wrapper.

WHY IT WORKS: It executes the "magical hover surprise" pattern — a card looks flat at rest, but reveals depth and motion on intent (hover). This rewards user exploration without taxing pre-interaction performance. Because it runs entirely on the GPU via shader, it scales independently of dot count — visually rich without CPU cost. And because the colors are configurable per-card, it becomes a system, not a one-off effect.

GOOD / BAD:
- GOOD: GPU-shader-based — zero DOM overhead, scales to any density. Perfect ROI on visual delight per kilobyte.
- BAD: The Clerk-style "expanding dots" has become a 2024-2026 cliché in dev-tool marketing — using it without modification reads as derivative.

CLEAN APPLICATION: Apply this to Clean's pricing page tier cards. Mount a `<CanvasRevealEffect>` inside each pricing tier card with tier-specific colors: Free tier `[[100,116,139]]` (slate), Pro tier `[[59,130,246]]` (Clean's blue), Enterprise tier `[[16,185,129]]` (emerald). Default state is a flat card; on hover the dot matrix reveals with a 0.4s fade, signaling tier "energy." Differentiate from the Clerk cliché by replacing dots with a custom shader rendering monospace hex-character glyphs (e.g. `0x4F`, `0xA3`) that pulse in the same opacity wave — a literal nod to Clean's infrastructure-as-code positioning.
