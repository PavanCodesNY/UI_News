## RISK — https://www.awwwards.com/sites/risk

WHAT IT DOES: Portfolio site for a European post-production studio built as a full-bleed WebGL canvas with a scroll-driven timeline of case-study reels. The layout system is a fixed viewport with an internal virtualized scroller (Lenis) driving a Three.js orthographic camera across a horizontally arranged plane grid; each reel is a plane textured with a compressed video that plays only when its plane enters a 20%-viewport hover zone. Visual hierarchy is a single dominant hero cluster (studio wordmark + hovered reel title in monospace) with peripheral metadata (frame counters, timecode, aspect ratio) chromed in like a video-suite HUD.

PACKAGES:
- `webflow` (host)
- `gsap` ^3.13, `gsap/ScrollTrigger`, `gsap/CustomEase`
- `three` ^0.169
- `@studio-freight/lenis` ^1.1
- `postprocessing` ^6.36 (for chromatic aberration + noise)
- `hls.js` ^1.5 (adaptive video textures)
- Fonts: `PP Neue Montreal` (Pangram Pangram), `JetBrains Mono`
- Icons: custom SVG sprite (no icon lib)

IMPLEMENTATION:
1. Root: `<div id="app">` with `position: fixed; inset: 0; overflow: hidden` and a sibling `<div id="scroll-proxy" style="height: 800vh">` to give Lenis something to scroll.
2. Three.js scene: `OrthographicCamera` with `left/right/top/bottom` mapped to CSS pixels so plane coords match layout. Renderer at `devicePixelRatio` clamped to 1.5, `antialias: false`, `powerPreference: "high-performance"`.
3. Reel grid: N planes of size `viewportW * 0.42 × viewportH * 0.62`, laid out on X at 1.15× width spacing. Each plane uses a `ShaderMaterial` with uniforms `uTexture` (VideoTexture), `uHover` (0→1), `uProgress` (Lenis normalized), `uAspect`. Fragment applies barrel distortion `uv = uv + (uv - 0.5) * pow(length(uv - 0.5), 2.0) * 0.08 * uHover`.
4. Lenis: `duration: 1.2`, `easing: t => 1 - Math.pow(1 - t, 4)`, `wheelMultiplier: 0.9`. On `scroll` event, `camera.position.x = lerp(camera.position.x, lenis.progress * totalWidth, 0.12)`.
5. Hover HUD: on `pointermove`, raycast; if plane hit, tween `uHover` to 1 with `gsap.to(u.uHover, { value: 1, duration: 0.4, ease: "expo.out" })`; simultaneously animate a fixed HTML overlay `[data-hud="title"]` with SplitText character-stagger reveal (0.02s stagger, `expo.out`, 0.5s duration).
6. Post-processing: `EffectComposer` → `RenderPass` → `EffectPass(new ChromaticAberrationEffect({ offset: [0.0006, 0.0006] }))` → `EffectPass(new NoiseEffect({ premultiply: true, blendFunction: BlendFunction.OVERLAY }))` with noise strength `0.08`.
7. Video textures: use HLS with `.needsUpdate = true` in `requestVideoFrameCallback`; pause `.play()` on planes outside frustum + 500px margin.
8. Breakpoints: below 900px, collapse to a native vertical `<ul>` of `<video muted playsinline preload="metadata">` — kill WebGL entirely to save battery.
9. Perf: `renderer.setPixelRatio(Math.min(devicePixelRatio, 1.5))`, textures at `LinearFilter`, `generateMipmaps: false`, video max resolution 1280×720.

WHY IT WORKS: The HUD chrome (monospace timecodes, frame counters) reads as insider-industry vocabulary — visitors decode "this studio speaks video" before parsing a single word. Fitts' law is respected: reels are enormous targets, hover is generous. The chromatic aberration is under 0.1% offset, subliminal not gimmicky — signals "cinema" without shouting.

GOOD / BAD:
- GOOD: Uses HLS + `requestVideoFrameCallback` so 20+ reels play without dropping frames on M-series MacBooks.
- BAD: Screen-reader accessibility is minimal — no ARIA labels on canvas nodes; keyboard users get a fallback list but the fallback is unlabeled.

CLEAN APPLICATION: Apply this to Clean's **customer proof section** on the landing page. Replace the reel grid with a horizontally scrolling grid of customer dashboard captures (Huginn output screenshots, ICP heatmaps, prospect-list-in-motion loom-style clips). Same Three.js plane grid, same monospace HUD ("47,281 accounts refined · $2.4M sourced · run-id 8fa9c2"). Kill the aberration effect (developer audiences read it as "trying too hard"), keep the noise at 0.04.

## NRG | Build Your Data Center — https://www.awwwards.com/sites/nrg-build-your-data-center

WHAT IT DOES: B2B campaign microsite that walks hyperscaler buyers through a "build your own power" data-center configuration flow. Interaction is a pinned-viewport scroll narrative: user scrolls, a Three.js/R3F scene composes a data center piece by piece (site plot → generation stack → grid interconnect → cooling → racks), with each scroll segment locking the camera on a new subsystem and revealing an HTML sidecar of specs, tradeoffs, and MW/CapEx numbers. Layout is a 12-col grid where columns 1–7 host the canvas and 8–12 host contextual copy that swaps via `AnimatePresence`.

PACKAGES:
- `next` ^15 (app router)
- `react` ^19
- `@react-three/fiber` ^9, `@react-three/drei` ^9.115
- `three` ^0.169
- `gsap` ^3.13, `gsap/ScrollTrigger`
- `framer-motion` ^11.11 (sidecar transitions)
- `tailwindcss` ^4.0
- `zustand` ^5 (global scene state)
- Fonts: `Söhne` (Klim), `IBM Plex Mono`
- Textures: KTX2 compressed via `@loaders.gl/textures`

IMPLEMENTATION:
1. Layout: single sticky container `h-[600vh]` with `<Canvas className="fixed top-0 left-0 w-[58vw] h-screen">` and an HTML column `absolute right-0 w-[42vw]` for narrative.
2. Six ScrollTrigger sections, each 100vh. On enter, `zustand` state emits `activeSection: 0..5`, driving both camera position (via `useFrame` lerp) and sidecar swap.
3. Scene composition: base plot is a low-poly terrain (`PlaneGeometry` 200×200 segments, displaced by a noise texture). Each subsystem is a preloaded GLB (`useGLTF.preload('/models/gen-stack.glb')`) and enters via `gsap.from(mesh.scale, { y: 0, duration: 0.9, ease: 'expo.out' })` triggered by section entry.
4. Camera: `PerspectiveCamera` fov 35, keyframed positions per section stored as `[Vector3, Vector3][]` (position + target); `useFrame` interpolates with `damp3(camera.position, target, 0.08, delta)` from drei's math helpers.
5. Lighting: single directional light with `castShadow`, `shadow-mapSize: [2048, 2048]`, plus a warm rim `pointLight` at `[10, 5, -8]` intensity 0.6. Environment via `<Environment preset="city" background={false} />`.
6. Sidecar copy: `<motion.div layout>` with `initial={{ y: 24, opacity: 0 }} animate={{ y: 0, opacity: 1 }} exit={{ y: -24, opacity: 0 }} transition={{ duration: 0.5, ease: [0.22, 1, 0.36, 1] }}`. Numeric tickers (MW, CapEx, $/kW) use `useMotionValue` + `useTransform` with `useSpring({ stiffness: 90, damping: 20 })`.
7. Perf: `preserveDrawingBuffer: false`, dynamic `dpr={[1, 1.5]}` via drei, aggressive frustum culling, all GLBs draco-compressed to <400KB each.
8. Mobile: below 1024px, collapse canvas to a stacked series of static hero renders (baked in Blender, exported as WEBP) with the sidecar becoming vertical prose — no WebGL.

WHY IT WORKS: The pinned canvas + scrolling sidecar is textbook "narrative anchoring" — the visual stays central so users don't lose their place while the copy does the persuasion. Numbers ticker up as sections change, giving dopamine reward for scrolling; each section resolves one buyer objection (power, cost, timeline, reliability, sustainability). Progressive disclosure means an intimidating 10-figure decision feels navigable.

GOOD / BAD:
- GOOD: Genuinely educational — a buyer with zero infrastructure background leaves knowing what "bring your own power" means. Rare for B2B enterprise.
- BAD: Total JS payload is ~2.1 MB before models load; TTI on 4G is well over 6 seconds. No skeleton state during model load — screen just sits.

CLEAN APPLICATION: Apply this to Clean's **product tour / "How Clean works" page**. Build a scrolling narrative that composes the Clean flywheel piece by piece: (section 1) empty CRM → (section 2) ICP builder appears → (section 3) prospects flow in → (section 4) auto-profiling enriches records → (section 5) revenue scaler routes to sequences → (section 6) closed-won dashboard. Use R3F for a stylized isometric board rather than photoreal 3D (Clean isn't heavy infrastructure — it should look like a schematic). Camera pinned left, spec sidecar right with number tickers ("accounts profiled: 0 → 12,847"). This is the pattern that best matches Clean's need to explain a multi-stage product without a demo call.

## House of Honey — https://www.awwwards.com/sites/house-of-honey-1

WHAT IT DOES: Interior design studio portfolio conceived as spatial navigation — the user "moves through a designed space" rather than scrolls a document. Layout is a single-page immersive canvas where project thumbnails are arranged in a 2D plane the user pans (drag + inertia) rather than scrolls. Selecting a project triggers a shared-element transition where the thumbnail expands into a full-viewport case study with vertical scroll returning to the pan-space on close. Visual hierarchy uses generous whitespace and serif typography with each project card carrying its title as an overlay.

PACKAGES:
- `next` ^15
- `sanity` ^3 (CMS)
- `next-sanity` for image URL builder
- `framer-motion` ^11.11 (layout animations, shared element transitions)
- `@studio-freight/lenis` ^1.1
- `gsap` ^3.13 (draggable pan-plane)
- `gsap/Draggable` and `InertiaPlugin`
- `tailwindcss` ^4.0
- Fonts: `GT Sectra` (Grilli Type) display, `GT America` body

IMPLEMENTATION:
1. Root: `<main className="fixed inset-0 overflow-hidden">` with an inner `<div ref={plane} className="absolute" style={{ width: '3600px', height: '2400px' }}>` acting as the pan-space.
2. Register `Draggable.create(plane, { type: 'x,y', inertia: true, bounds: { minX: -1800, maxX: 0, minY: -1200, maxY: 0 }, edgeResistance: 0.85, throwProps: true })`.
3. Project cards: absolute-positioned within the plane at hand-tuned coordinates. Each card is a `<motion.div layoutId={`card-${slug}`}>` with an image and overlaid `<h3>` in serif.
4. Detail view: on click, `router.push(`/projects/${slug}`)` and render `<motion.div layoutId={`card-${slug}`}>` at full viewport. Framer Motion's `LayoutGroup` handles the shared-element morph — no manual FLIP.
5. Cursor: custom SVG cursor (a small "drag" glyph) that scales up on hoverable cards; implemented as a fixed `<div>` following `pointermove` with `transform: translate3d()` and `will-change: transform`.
6. Ambient details: subtle idle drift on the plane when the user hasn't interacted for 4s — `gsap.to(plane, { x: '+=8', y: '+=4', duration: 6, repeat: -1, yoyo: true, ease: 'sine.inOut' })`. Kill on `pointerdown`.
7. Type animations: `SplitText` on project titles, staggered `y: 24 → 0, opacity: 0 → 1` at 30ms per char on `layoutId` completion.
8. Mobile: swap Draggable-plane for a scrollable native grid; layoutIds still animate on tap.

WHY IT WORKS: The metaphor lands — interior design is spatial, so browsing this studio's work spatially feels congruent. Draggable inertia gives the site a tactile "materiality" that scrolling a linear feed can't. Serif type on wide whitespace signals "curated, considered, not mass-market" — reinforcing the studio's price point through form alone.

GOOD / BAD:
- GOOD: The `layoutId` shared-element transition is a masterclass — zero manual choreography, browsers handle the FLIP, and it never desyncs.
- BAD: On first load, the drag affordance is unclear for scroll-native users; some visitors will hit the plane, try to scroll, and bounce.

CLEAN APPLICATION: Apply this to Clean's **customer case studies / logos section** on the landing page. Instead of a static logo wall, build a draggable canvas where 40+ customer logos are scattered across a pannable plane. Clicking a logo triggers a shared-element expansion into a mini case study (industry, use case, "before Clean" pipeline, "after Clean" pipeline, quote). This makes the "we work with everyone" story explorable rather than a wall of ignored logos.

## 21 Hrs On The Moon — https://www.awwwards.com/sites/21-hrs-on-the-moon

WHAT IT DOES: Editorial WebGL storytelling piece anchored to the Apollo 11 lunar surface stay. The entire site is a scroll-driven Three.js scene where camera path, terrain lighting, and typographic beats are choreographed against a normalized scroll value. Two-color palette (#060B0B ink, #FFF3EA paper) creates a print-editorial feel over a technically dense render. Visual hierarchy is chronological: scroll = time, so scroll progress maps directly to elapsed lunar hours displayed as a large monospace clock in the corner.

PACKAGES:
- `three` ^0.169
- `gsap` ^3.13 with `ScrollTrigger`
- `@studio-freight/lenis`
- `troika-three-text` ^0.52 (WebGL text for on-scene typography)
- `postprocessing` ^6.36 (film grain, vignette)
- `astro` ^5 (SSG shell)
- `tone` ^15 (spatial audio cues)
- Fonts: `PP Neue Corp Wide` (Pangram Pangram), `Söhne Mono`

IMPLEMENTATION:
1. Astro island hosts a single `<canvas>`. `ScrollTrigger` normalized to `0..1` across a `1000vh` proxy.
2. Terrain: baked heightmap (single 4096×4096 R16F texture) applied to a `PlaneGeometry(400, 400, 512, 512)` with `MeshStandardMaterial` normal-mapped from a NASA LRO tile.
3. Camera path: an array of 12 `Vector3` waypoints + look-at targets. On scroll, `getPointAt(t)` on a `CatmullRomCurve3` yields camera position; look-at also curve-interpolated. Camera roll animated separately with a sine-based idle jitter of ±0.3° to feel handheld.
4. Lighting: single `DirectionalLight` whose angle tracks scroll — sun elevation shifts from 8° at scroll 0 (dawn) to 62° at scroll 1 (noon on the moon). Shadow bias tuned tight (`-0.0005`) to avoid acne on the low-poly terrain.
5. Type beats: `troika-three-text` labels attached to Vector3s in the scene (crater names, time-of-event captions). Opacity keyframed by proximity to camera + scroll segment. `letter-spacing` scales up as the label fades in — evokes analog news teletype.
6. Audio: `Tone.Player` for radio-chatter loops, spatialized via `Tone.Panner3D` positioned in the same world coords as the LM.
7. Post-processing: `NoiseEffect` at 0.06, `VignetteEffect` darkness 0.5, offset 0.35. No aberration.
8. Fallback: if `navigator.hardwareConcurrency < 4` or `matchMedia('(prefers-reduced-motion: reduce)').matches`, serve a static image gallery + captions instead.

WHY IT WORKS: The two-color rule forces every visual decision into contrast and composition rather than color-noise — the result reads as archival, worthy, museum-grade. Scroll-as-time is a semiotic match: users feel the 21 hours passing rather than being told about them. Spatial audio is the sleeper feature — hearing the LM in stereo makes the visual feel three-dimensional even before the eyes finish parsing.

GOOD / BAD:
- GOOD: The 2-color constraint is discipline — no fallback to "we'll just add a blue accent." Every winner does this.
- BAD: On tablets in portrait, the wide-camera framing composes poorly and important type beats fall out of viewport.

CLEAN APPLICATION: Apply this to Clean's **"origin story" or manifesto page** (or an About section on the landing page). Build a scroll-driven narrative that treats Clean's founding story or GTM philosophy like an editorial spread — heavy monospace numbers ticking through GTM stats ("$47B wasted on cold outbound annually · 2.1% average reply rate · 68 hours per SDR per week on manual work") while a stylized 2-color canvas moves through abstract diagrams. Keep it two colors: Clean's brand ink over paper. This is where Clean earns brand equity rather than performance.

## Longbow — https://www.awwwards.com/sites/longbow

WHAT IT DOES: Marketing site for a British featherweight EV sports car brand. Uses a fixed vertical rhythm of full-viewport sections, each with a dominant hero image + oversized display type + a single metric callout (0-60mph, weight, range). Scroll transitions are cinematic vertical wipes where the outgoing section clips upward while the incoming section slides up from below, both moving at 0.85× scroll velocity via ScrollTrigger — the site feels heavier and more deliberate than default 1:1 scrolling. Type is the load-bearing element: massive display serifs juxtaposed with tight monospace metric tickers.

PACKAGES:
- `next` ^15
- `gsap` ^3.13, `gsap/ScrollTrigger`, `gsap/SplitText`
- `@studio-freight/lenis`
- `framer-motion` ^11.11 (page transitions between routes)
- `tailwindcss` ^4.0
- `@vercel/og` (dynamic OG images per model)
- Fonts: `Editorial New` (Pangram Pangram) display, `Neue Haas Grotesk` body, `JetBrains Mono` metrics
- Images: `next/image` with `priority` on hero, AVIF preferred

IMPLEMENTATION:
1. Section structure: `<section className="h-screen relative overflow-hidden">` × N, each with an inner `<div className="absolute inset-0" data-parallax>` for the image and an overlay grid for type.
2. Lenis at `duration: 1.4, smoothWheel: true, wheelMultiplier: 0.75`.
3. Section transitions: `ScrollTrigger.create({ trigger: section, start: 'top top', end: '+=100%', pin: true, pinSpacing: false, scrub: 0.5 })` combined with a timeline that translates the section's `[data-parallax]` from `y: 0` to `y: -15%` and its `[data-copy]` from `y: 0` to `y: -30%` — parallax at differential rates creates depth.
4. Type entrance: `SplitText(headline, { type: 'lines,words' })`, then `gsap.from(split.lines, { yPercent: 110, duration: 1.1, stagger: 0.06, ease: 'expo.out' })`, triggered when section is 40% in view.
5. Metric tickers: numbers count from 0 to target with `gsap.to({ v: 0 }, { v: target, duration: 1.8, ease: 'expo.out', onUpdate() { el.textContent = Math.round(this.targets()[0].v).toString() } })`.
6. Cursor: on hover over CTAs, cursor scales to a filled disc and inverts nearby text via `mix-blend-mode: difference`.
7. Nav: transparent on hero, transitions to a solid pill on scroll past 100vh — `mix-blend-mode: difference` again so text is always readable regardless of background.
8. Performance: hero videos are `<video autoPlay muted loop playsInline preload="metadata">` at 1080p H.265, poster served immediately via `next/image` blurred placeholder.
9. Mobile: pin animations disabled below 768px; sections become normal scroll-flow with entrance animations only.

WHY IT WORKS: The 0.85× scroll dampening is subtle but critical — it makes every micro-interaction feel more considered, which maps to the "hand-built" brand promise. Display serif + monospace metrics is a proven duo (see Rimac, Koenigsegg) — the serif signals "heritage / craft" and the mono signals "engineering / performance." Together they claim both halves of the luxury-EV mind.

GOOD / BAD:
- GOOD: Type-forward approach means the site works even before hero images finish loading — LCP is under 1.4s on fast 4G.
- BAD: Pin-scrub animations feel great on trackpad but jittery on Windows mouse wheels; no OS-specific tuning.

CLEAN APPLICATION: Apply this to Clean's **landing page hero + top three feature sections**. Adopt the pinned full-viewport section pattern with 0.85× scroll dampening. Use editorial display serif for the hero headline ("Sell to the buyers who want to buy") and JetBrains Mono for the running metric tickers below ("1.4M+ ICP-matched accounts · 47% reply rate · $2.4M sourced last week"). Full-bleed dashboard captures scale up as background with 15% parallax delta. This is the fastest single upgrade to Clean's landing page brand tier.

## BUILD DIRECTIVE

TARGET: Clean's **/how-it-works product tour page** (or an equivalent scroll-narrative section embedded in the landing page below the hero).

APPROACH: Build a pinned-canvas scroll narrative that composes the Clean revenue-scaler flywheel piece by piece — empty CRM → ICP builder → prospect intake → auto-profiling enrichment → sequence routing → closed-won dashboard. Canvas is a stylized isometric R3F scene occupying the left 58% of viewport; a right-side sidecar in the remaining 42% swaps copy per section using Framer Motion `AnimatePresence`, with numeric tickers spring-animating on section change ("accounts profiled: 0 → 12,847", "$0 → $2.4M sourced"). Overlay a fixed monospace HUD in the corner showing normalized progress and a "run-id" indicator that increments — this reads as "you are watching a real system run" rather than a marketing animation. Constrain the palette to two colors (Clean's brand ink + off-white paper) and use `Editorial New` for section headlines against `JetBrains Mono` for all metrics. No chromatic aberration, no bloom, no environment maps — the aesthetic is developer-tool schematic, not consumer sizzle reel.

REFERENCE URLS:

KEY CONSTRAINTS: Clean is developer-oriented GTM infrastructure. The result must feel like a system diagram animating, not a marketing sizzle reel. Avoid: glassmorphism, gradient meshes, floating orb backgrounds, purple-to-pink neon, generic Framer template hero, "AI star field" backdrops, ambient particle systems with no data meaning, unlabeled iconography. Every animated number must reference a real Clean metric (accounts profiled, prospects sourced, reply rate, revenue sourced). Every scene node must map 1:1 to a product surface (Huginn, revenue scaler, knowledge base engine, prospect finder). TTI budget: sub-2s on 4G with progressive canvas hydration — models under 300KB draco-compressed each, texture atlases under 512KB. Reduced-motion fallback must be a static schematic image + prose, not a stripped animation.
