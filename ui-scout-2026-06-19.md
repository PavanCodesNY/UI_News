# UI Scout — 2026-06-19

Scout of today's best UI work. Sources: Awwwards SOTD feed (Jun 12–18, 2026), Vercel/Linear's current marketing surfaces, Aceternity UI's June drops. Five picks selected for technical depth and direct applicability to Clean (developer infrastructure, dark-mode, agent-first).

---

## 1. Indigo Laboratory — https://www.awwwards.com/sites/indigo-laboratory

WHAT IT DOES: A scrollytelling sensory site for a jewelry brand structured as five discrete "tales", each with its own color palette, typography variant, and ambient audio bed that cross-fades on section enter. The layout system is a horizontally-locked vertical scroll: a `position: sticky` viewport snaps each tale to fullscreen while inner content (image grids, kinetic type) reveals on a scroll-progress timeline. The visual hierarchy is editorial-first — oversized serif display, asymmetric image plates, and a persistent fixed-corner chapter index that animates its active state via SVG morph.

PACKAGES:
- `next@14` (App Router, RSC) or `nuxt@3`
- `gsap@3.12` with `ScrollTrigger`, `ScrollSmoother`, `SplitText`
- `lenis@1.1` for inertia scrolling
- `@studio-freight/hamo` for resize/raf hooks
- `howler@2.2` for the cross-fading ambient audio
- `framer-motion@11` for in-view reveals on non-pinned sections
- `tailwindcss@3.4` + a custom theme per tale (CSS variables swapped on chapter)
- Fonts: `Editorial New` (PangramPangram) for display, `Söhne Mono` for chapter index, `Inter` body
- `lucide-react` icons

IMPLEMENTATION:
- Component tree: `<Shell>` → `<AudioBed/>` (Howler manager, fades 800ms cubic-in-out) → `<ChapterIndex/>` (fixed bottom-right) → `<TaleSection id="tale-1..5"/>` × 5 → `<Footer/>`.
- Each `<TaleSection>` is `min-height: 100vh; position: sticky; top: 0;` inside a parent that is `height: 500vh` (one viewport per tale). Inner content is animated via `ScrollTrigger.create({ trigger: section, start: 'top top', end: '+=100%', scrub: 1 })`.
- Display type uses `SplitText` per word; reveal is `gsap.fromTo(words, { yPercent: 110, rotate: 6 }, { yPercent: 0, rotate: 0, stagger: 0.04, ease: 'expo.out', duration: 1.1 })`.
- Image plates use clip-path mask reveal: `clip-path: inset(100% 0 0 0)` → `inset(0 0 0 0)`, 1.4s `cubic-bezier(0.16, 1, 0.3, 1)`.
- Color theme: per-tale CSS variables (`--bg`, `--fg`, `--accent`) swap via `gsap.to(document.documentElement, { '--bg': '#0d0a08', duration: 0.8 })` at `onEnter`.
- Audio: preload all 5 stems, cross-fade with `howl.fade(0, targetVol, 800)`. Gate behind a one-time mute-toggle for autoplay policy.
- Breakpoints: 1440 (design), 1024 (condense plate grids 2→1), 640 (stack chapter index to top bar, kill smooth scroll, fall back to native scroll).
- Perf: lazy-mount tales 2–5 with IntersectionObserver (rootMargin: 200%), use `loading="eager"` for tale 1, `eager`+`fetchpriority="high"` on hero image, `next/image` AVIF, blur placeholder.

WHY IT WORKS: Editorial typography + scroll-locked chapter pacing creates the "magazine you read with your scroll wheel" effect — high contrast between sections forces the eye to reset, which is Gestalt common-region grouping at chapter scale. The audio layer adds a second sensory channel that increases dwell time (multi-modal encoding). Negative space between plates lets the product be the only thing competing for attention.

GOOD / BAD:
- GOOD: Per-chapter theme swap via CSS vars is the most performant way to do "feels-like-a-different-site" without unmounting.
- BAD: Audio autoplay is hostile to muted-by-default browsers; the chapter scrub becomes a finger-trap on iOS Safari without a "skip chapter" affordance.

CLEAN APPLICATION: Apply this to Clean's **landing page hero + "how it works" section** by treating each architectural layer (Client → Clean Proxy → MCP Server → Model) as a "tale". Pin a single 100vh viewport, scrub between four states: bare request, then TOON-compressed payload overlays, then routing/policy, then model response. Each state swaps `--accent` and crossfades a subtle ambient UI sound (typed-key clack, success chime — all under 200ms, mutable). Keep the chapter index as a vertical dotted spine on the left at `position: fixed; top: 50%`.

## 2. Fauna Robotics — https://www.awwwards.com/sites/fauna-robotics

WHAT IT DOES: A humanoid-robotics startup site that humanizes its mascot ("Sprout") via warm asymmetric shape language and close-up product cinematography. The layout is a Webflow-built single-page narrative with 60fps loop video heroes, a magnetic cursor that snaps to interactive elements, and a horizontal product-detail carousel driven by mouse wheel + drag. Visual hierarchy is character-first — the robot is always the protagonist, copy is supporting.

PACKAGES:
- `webflow` (production) or `next@14` + `tailwindcss` for re-implementation
- `gsap@3.12` + `Observer` plugin (wheel/drag normalization)
- `lenis@1.1`
- `mux-player` or `<video>` with HLS for hero loops
- `framer-motion@11` for hover spring states
- `lottie-react@2.4` for accent micro-illustrations
- Fonts: `PP Neue Montreal` (display), `Söhne` (body)
- Color tokens: warm off-white `#F2EEE8`, ink `#1A1612`, sage accent `#7A8A6A`

IMPLEMENTATION:
- Magnetic cursor: 24px filled circle, follows mouse with `gsap.quickTo(el, 'x', { duration: 0.4, ease: 'power3.out' })`. On `mouseenter` of `[data-magnetic]`, scale to 64px and snap `x/y` toward the target's bbox center with a 0.35 strength factor.
- Hero video: 1920×1080 H.265 fallback to H.264, looped, `autoplay muted playsinline`, `poster` is a single AVIF frame. Overlay a 12% noise SVG at 0.04 opacity to kill banding.
- Product carousel: a `flex` row at `width: max-content`, translated with `gsap.to(track, { x: -progress * scrollWidth, ease: 'none' })` driven by `Observer.create({ type: 'wheel,touch,pointer' })`. Decay friction `0.92` per frame.
- Type setting: display set at `clamp(56px, 7.5vw, 128px)`, line-height `0.92`, tracking `-0.03em`. Body at 17/26 with measure 64ch.
- Section pacing: alternating 100vh hero/72vh product strips, gutters at 64px desktop / 24px mobile.
- Breakpoints: 1440 / 1024 / 768 / 480. Mobile kills magnetic cursor entirely (battery, touch model).
- Perf: aggressive `content-visibility: auto` on below-fold sections, preconnect to video CDN, `<link rel="preload" as="video">` on the hero source.

WHY IT WORKS: Asymmetric shapes + warm palette reads as "approachable AI", which is the entire positioning bet. Magnetic cursor + character-led photography creates an emotional contract before any feature copy is read — psychology of trust transferred from the mascot to the brand. Whitespace rhythm (always 1.5× the previous gap) creates a calm reading cadence.

GOOD / BAD:
- GOOD: Character-first hierarchy is a master class in B2B humanization without going cartoony.
- BAD: Magnetic cursor + heavy hero video is a 6+ MB above-the-fold payload; CLS and LCP suffer on mid-tier mobile.

CLEAN APPLICATION: Apply this to Clean's **product personality / "meet your proxy" section** on the landing page. Replace the robot mascot with an abstract animated "Clean proxy" glyph (rounded square with a subtle pulse + connection lines that orbit it on hover, driven by `gsap.to` with `Observer`). Use the magnetic cursor pattern only on the primary CTA + nav items — `data-magnetic` attribute, 0.3 strength. Keep the warm/calm palette idea but invert it for Clean's dark theme: ink `#0B0B0F` background, off-white text `#EDEAE3`, single accent (Clean's brand green).

## 3. Elva (voice-first AI video editor) — https://www.awwwards.com/sites/elva

WHAT IT DOES: Marketing site for a mobile AI editor whose hero is a real-time WebGL "glass blob" with 30+ behavioral states (idle, listening, thinking, speaking, error). The blob is a single shader-driven mesh — vertex displacement keyed to mic input or scripted state — wrapped in a UI shell that documents agentic camera capabilities. Layout is single-page, near-monochrome (`#FFFFFF` / `#151515`), with a heavy reliance on motion to differentiate sections rather than color.

PACKAGES:
- `next@14`
- `three@0.165` + `@react-three/fiber@8.16` + `@react-three/drei@9.108`
- `gsap@3.12` + `ScrollTrigger`
- `leva@0.9` (state inspector during dev only)
- `glslify` + custom shaders for the blob
- `framer-motion@11` for UI shell transitions
- `tailwindcss@3.4`
- Fonts: `Inter Tight` (display), `Inter` (body), `JetBrains Mono` (captions)

IMPLEMENTATION:
- Blob: `<Canvas>` containing an `<IcosahedronGeometry args={[1, 64]} />` with a custom `ShaderMaterial`. Vertex shader displaces position by `noise(p * freq + time) * amp`. Each "state" is a `{ freq, amp, color, distortion, mixSpeed }` object lerped over 600ms when state changes.
- States are managed by a finite state machine (`xstate@5` or a hand-rolled reducer). Trigger transitions on scroll position (`ScrollTrigger` calls `send('SCROLL_PAST_LISTENING')`).
- Material: `MeshTransmissionMaterial` from drei with `transmission: 1`, `thickness: 0.4`, `roughness: 0.08`, `chromaticAberration: 0.05`, `distortionScale: 0.6`. Behind it, an animated gradient plane (`<mesh>` with a 2-color GLSL gradient) provides the refraction content.
- Scroll choreography: pin the canvas for `~400vh`, drive blob `freq`/`amp` via `ScrollTrigger.scrub`. Each "chapter" of copy slides in from the right at `x: 60 → 0`, opacity `0 → 1`, 600ms `power2.out`.
- UI captions: monospace, set in `JetBrains Mono`, typed-on with a cursor blink at `1s steps(2, end) infinite`.
- Breakpoints: 1280 (design), 768 (reduce blob segments from 64 to 24, disable transmission, fall back to `MeshStandardMaterial` with `metalness: 0.8`).
- Perf: `dpr={[1, 1.5]}` on Canvas, `frameloop="demand"` when off-screen, render-on-demand via `invalidate()`.

WHY IT WORKS: A single living object that mutates by state IS the product narrative — the agent's behavioral states become the navigation. Two-color palette forces motion + form to do all the differentiation work, which feels intentional rather than decorative (motion semantics). Monospace captions signal "developer-adjacent" without being cold.

GOOD / BAD:
- GOOD: Stateful WebGL hero that is also the product demo is the most efficient use of a fold ever — explains, demos, and brands simultaneously.
- BAD: TransmissionMaterial + 64-segment icosahedron is brutal on integrated GPUs; the mobile fallback is a noticeable downgrade.

CLEAN APPLICATION: Apply this directly to Clean's **landing page hero** — replace Elva's glass blob with a **shader-driven "context object"**: a translucent rounded-cube mesh whose interior reveals a flowing TOON-format token stream (animated GLSL noise interpreted as text glyphs). State transitions: `idle` → `compressing` → `routing` → `responding`, each triggered by scroll. Use `MeshTransmissionMaterial` with `chromaticAberration: 0.04` so the green Clean accent refracts through. The same hero teaches three things at once: what TOON is, what the proxy does, and what the response feels like.

## 4. Vercel "AI Cloud" homepage — https://vercel.com

WHAT IT DOES: A near-black background (`#0A0A0A`) hero with continuous shader-based gradient animation (radial noise warped over time), behind a tight type stack and a live product mega-menu. Information architecture splits the product line into AI Cloud / Core Platform / Security columns; each sub-product (v0, AI Gateway, Vercel Agent, Sandbox) is keyed to a distinct accent hue but stays monochrome-by-default. Below the hero, every section uses a subtle 1px grid pattern as a structural backdrop, and product blocks are bento-style cards with hover-triggered micro-animations.

PACKAGES:
- `next@14` (RSC, edge runtime where possible)
- `geist@1.3` (font + UI components — Vercel's own)
- `tailwindcss@3.4`
- A custom WebGL shader hero (raw `WebGL2RenderingContext` or a thin `regl@2.1` wrapper — no three.js needed)
- `framer-motion@11` for hover/exit transitions on bento cards
- `lucide-react` icons
- `radix-ui/react-navigation-menu` for the mega-menu
- `@vercel/analytics` + `@vercel/speed-insights`

IMPLEMENTATION:
- Shader hero: fullscreen `<canvas>` with a fragment shader: `vec3 col = mix(c1, c2, noise(uv * 2.0 + time * 0.05));` plus a domain-warp pass. Render at `0.6 * devicePixelRatio` and `image-rendering: auto` upscale to save 60–70% GPU cost. Composite with `mix-blend-mode: screen` over the `#0A0A0A` background.
- Type stack: Geist Sans at `clamp(48px, 6vw, 80px)`, weight 500, tracking `-0.04em`, line-height `0.95`. Eyebrow label at 13px uppercase, tracking `0.12em`, opacity `0.6`.
- Grid pattern: `background-image: linear-gradient(rgba(255,255,255,0.04) 1px, transparent 1px), linear-gradient(90deg, rgba(255,255,255,0.04) 1px, transparent 1px); background-size: 64px 64px;` with a radial mask to fade edges.
- Bento cards: `rounded-2xl`, `border: 1px solid rgba(255,255,255,0.08)`, `bg: rgba(255,255,255,0.02)`. On hover: border lerps to `rgba(255,255,255,0.18)` over 200ms, internal glyph animates (e.g., the v0 card runs a Lottie of cursor → code).
- Mega-menu: Radix NavigationMenu, opens with `framer-motion` `height: auto` + `opacity` over 240ms `[0.16, 1, 0.3, 1]`. Each column is a 3-column grid of product links with 32px gutters, icon + name + 1-line description per item.
- Accent system: each product card has `--accent` (cyan for v0, magenta for Gateway, lime for Agent, amber for Sandbox). Default state shows accent at 0.4 opacity in a corner glow; hover ramps to 0.9.
- Breakpoints: 1440 / 1024 / 768. Mobile collapses mega-menu to a stacked drawer with a single accordion.
- Perf: shader runs at `requestVideoFrameCallback`-pegged 30fps when reduced-motion is preferred; static AVIF fallback when `prefers-reduced-motion: reduce`.

WHY IT WORKS: The dark-mode + grid + shader gradient combo signals "infrastructure" without any literal server iconography — it's the developer aesthetic encoded purely in materials. Monochrome-by-default with accent-on-hover creates a clean default state but lets each product own its identity on interaction (Gestalt similarity for the cluster, differentiation per item). Bento grid solves the "we have many products" problem without feeling like a feature list.

GOOD / BAD:
- GOOD: The "speed as aesthetic" delivery — near-perfect LCP/CLS despite the WebGL hero — proves to developers via the medium itself.
- BAD: The aesthetic is now so cloned (every dev-tool launches with `#0A0A0A` + grid + shader gradient) that it risks reading as table-stakes rather than distinctive.

CLEAN APPLICATION: Apply this to Clean's **landing page chrome (background, nav, bento product section)**. Adopt the `#0A0A0A` base + 64px grid pattern with a radial fade mask as the universal background — this is the Clean visual substrate. Build the mega-menu now: split Clean's offering into Proxy / Context / Observability columns, even if some columns are "coming soon" stubs. Bento section directly below hero: 6 cards (TOON compression, MCP routing, agent context, policy/auth, observability, SDK) on a 3×2 grid, each with one accent hue and a hover-only Lottie. Skip the shader gradient in the hero — replace with the live "context object" from Pick 3 (Elva). Take their grid + bento + mega-menu, leave their gradient.

## 5. Linear "product is the demo" homepage — https://linear.app

WHAT IT DOES: A marketing site where the hero is a fully-interactive miniature of the actual Linear UI — real keyboard shortcuts work, the issue list scrolls, AI agents type live in the sidebar. Below the fold, each section pairs a scroll-pinned product surface (Cycles board, Triage view, Insights dashboard) with copy that explains it; the surface mutates as you scroll (issues move, charts redraw). The whole site uses a calm, near-monochrome palette with a single signature purple accent (`#5E6AD2`), and motion is restrained — everything is a 200ms `ease` spring, nothing larger.

PACKAGES:
- `next@14` (App Router)
- `tailwindcss@3.4`
- `framer-motion@11` (heavy — `useScroll`, `useTransform`, `LayoutGroup`)
- `radix-ui` primitives for menus/dialogs in the mini-UI
- `cmdk@1.0` for the command-palette demo
- `lucide-react` icons
- `geist` or `inter` font
- A custom keyboard-shortcut hook (`useHotkeys` from `react-hotkeys-hook@4.5`)

IMPLEMENTATION:
- Hero mini-UI: render a real Linear-shaped component (`<MiniApp>`) at scale `0.92` with `pointer-events: auto`. Hook up `Cmd+K` to a real `cmdk` palette, `J/K` to navigate list items, `E` to "complete" an issue (animates strikethrough + slide-out).
- Live AI agent: a typewriter effect in the sidebar that types a fake-but-plausible agent action every 8s on loop. Built with `framer-motion` `animate` on a child text node, character by character at 28ms per char, cursor `▍` blink at 0.6s.
- Scroll-pinned product surfaces: each section is `sticky top-20 h-screen` containing a product surface; copy column (right side, 40% width on desktop) scrolls past it. Surface mutations are driven by `useScroll({ target: section, offset: ['start end', 'end start'] })` then `useTransform` mapping progress to specific states (issue count, chart values, view tabs).
- Motion vocabulary: ALL transitions are `{ type: 'spring', stiffness: 380, damping: 36 }` OR `{ duration: 0.2, ease: [0.16, 1, 0.3, 1] }`. Nothing longer than 280ms. This is the discipline that makes it feel "fast".
- Type: Inter (or Geist), display at `clamp(40px, 5vw, 64px)`, weight 600, tracking `-0.025em`. Eyebrow 12px uppercase tracking `0.14em`.
- Color: `--bg: #08090A`, `--fg: #F7F8F8`, `--muted: #8A8F98`, `--accent: #5E6AD2`. Accent appears only on links, the AI agent's cursor, and primary CTAs.
- Surfaces use `border: 1px solid rgba(255,255,255,0.06)` and `background: linear-gradient(180deg, rgba(255,255,255,0.02), transparent)`.
- Breakpoints: 1440 (full mini-UI) / 1024 (mini-UI at 0.8 scale) / 768 (replace mini-UI with a tap-through video).
- Perf: virtualize the issue list in the mini-UI (only render 20 rows), debounce `useScroll`-driven `useTransform` to 60fps via `motion`'s own throttling.

WHY IT WORKS: An interactive product miniature inverts the funnel — instead of "sign up to see it", visitors are already using a constrained version. This collapses time-to-value to zero. The relentless 200ms motion budget conditions the visitor's nervous system to associate "Linear" with "fast" before they've read a single feature claim. Restricted accent palette makes the single purple feel like a brand signature rather than a color.

GOOD / BAD:
- GOOD: Real keyboard shortcuts in a marketing-page demo is best-in-class — it teaches the product's value proposition (speed/keyboard-native) through the medium.
- BAD: The mini-UI is heavy JS (200+ KB above-fold even with code-splitting); on slow connections the hero shows a skeleton for a second too long.

CLEAN APPLICATION: Apply this to Clean's **landing page hero (or a dedicated `/playground` section)**. Build a **live, scaled-down Clean dashboard** in the hero: a real terminal pane with a fake `claude-code` session, a real proxy panel showing TOON-compressed payloads streaming in, a real policy editor where visitors can toggle a rule and see the request shape change. Wire up actual keyboard shortcuts (`Cmd+K` opens a command palette listing Clean's MCP tools). Enforce the 200ms motion budget across the entire site — Clean is for developers, so speed-as-aesthetic is the most resonant signal. Use Linear's purple as a structural lesson, not literally — pick one accent (Clean green) and use it ONLY for active states, the streaming cursor, and the primary CTA.

## BUILD DIRECTIVE

TARGET: **Clean's landing page hero + immediately following bento product section** (tryclean.ai homepage above and just below the fold).

APPROACH: Build a hero that is a **live, interactive product miniature** of the Clean proxy in action — a scaled-down dashboard pane on the left showing a real `claude-code` session streaming requests, and on the right a "context object" (shader-driven translucent rounded-cube WebGL mesh) that mutates state as the request flows: idle → compressing (TOON tokens fly inward) → routing → responding. Wire real keyboard shortcuts (`Cmd+K` opens a palette listing MCP tools; `R` resends the last request) so visitors are using the product, not watching it. Below the fold, drop a 3×2 bento grid of Clean's six capabilities (TOON compression, MCP routing, agent context, policy/auth, observability, SDK) on a `#0A0A0A` base with a 64px grid backdrop and radial fade mask. Enforce a hard 200ms motion budget across every transition — Clean's pitch is speed and developer-native ergonomics; the site itself must demonstrate both.

STACK:
- `next@14` (App Router, RSC, edge)
- `tailwindcss@3.4` + custom theme tokens (`--bg: #0A0A0A`, `--fg: #EDEAE3`, `--accent: #00D67A` — Clean green; adjust to brand exact)
- `framer-motion@11` (`useScroll`, `useTransform`, `LayoutGroup`, springs `{ stiffness: 380, damping: 36 }`)
- `three@0.165` + `@react-three/fiber@8.16` + `@react-three/drei@9.108` (for `MeshTransmissionMaterial` on the context object)
- `gsap@3.12` + `ScrollTrigger` (state transitions on the context object)
- `cmdk@1.0` (command palette)
- `react-hotkeys-hook@4.5` (`Cmd+K`, `R`)
- `radix-ui/react-navigation-menu` (mega-menu)
- `lucide-react` icons
- Fonts: `Geist Sans` + `Geist Mono` (or `Inter Tight` + `JetBrains Mono`)
- `@vercel/analytics`, `@vercel/speed-insights`
- `lenis@1.1` optional (only if scroll feel needs polish; can be skipped for budget)

REFERENCE URLS:
- https://linear.app — for the interactive mini-UI pattern and the 200ms motion discipline
- https://vercel.com — for the dark + grid + bento + mega-menu chrome
- https://www.awwwards.com/sites/elva — for the WebGL "stateful object" hero pattern (link through to live site from Awwwards)

KEY CONSTRAINTS:
- Clean is **developer infrastructure** — the site must feel like a debugger, not a brochure. Avoid: stock illustrations, scrolling-text-over-blurry-photo heroes, "Trusted by" logo carousels above the fold, generic AI iridescent gradients, marketing-y headlines like "Reimagine your workflow".
- **Performance is the message.** LCP < 1.5s on a Moto G4. CLS < 0.05. JS budget for the hero ≤ 180 KB compressed. If the WebGL context object pushes this over, ship a static AVIF poster on first paint and hydrate the canvas on idle.
- **No motion over 280ms.** Spring stiffness ≥ 340. Reduce-motion fallback for every animation.
- **Monospace where it counts** — request payloads, code, tool names. Sans for the marketing voice. Never mix them in the same block.
- **One accent color.** Clean green only on the streaming cursor, active state, primary CTA, and the inside of the context object's refraction. Everywhere else is `#0A0A0A` / `#EDEAE3` / `#8A8F98`.
- **No autoplay audio. No magnetic cursor.** Both are visitor-hostile on the infrastructure persona.
- **Mega-menu now** — even with "coming soon" stubs in the Observability column — because it sets the IA ambition for the whole product surface.
