## Hildén & Kaira — https://www.awwwards.com/sites/hilden-kaira

WHAT IT DOES: Case-study-driven agency site for a Helsinki social content studio. Layout is a single vertically-scrolled column of full-bleed case study modules, each pairing a locked-position headline block with a horizontally auto-marquee row of vertical video posters and — critically — a live-data strip (views, watch-time, CTR) piped from Airtable + Make. Interactions: (1) sticky "chapter" headline that swaps on IntersectionObserver enter/leave with a GSAP `y: 100 → 0` + opacity crossfade, (2) inertial marquee on the video row using `gsap.to({ x: '-=100%' }, { repeat: -1, ease: 'none' })`, (3) hover-driven video preview that pauses the marquee and unmutes on `pointerenter`. Palette is two colors only: bone (`#EAE9E6`) and acid lime (`#ECFDAD`). Typography is one heavy grotesque headline face + one mono for metrics.

PACKAGES:
- `gsap` + `@gsap/react` + GSAP `ScrollTrigger`, `Observer`
- Webflow (host + CMS); if rebuilding in React: `next` 15, `next/font/local`
- `airtable` (Node SDK) for the live metrics pipe; polled server-side and revalidated via `next.revalidate = 3600`
- `lenis` for smoothscroll (optional but expected)
- `class-variance-authority` + `tailwindcss` if you rebuild
- Fonts: a heavy grotesque (e.g. `PP Neue Montreal Bold`) + a mono (`Berkeley Mono` or `JetBrains Mono`)

IMPLEMENTATION: Component tree — `<Page>` → `<HeroChapter>` → `[<CaseStudySection>]` → each section is `<StickyHeadline>` + `<Marquee videos={...}>` + `<LiveMetricsStrip data={...}>`. StickyHeadline: `position: sticky; top: 16vh;` inside a section that is `min-height: 200vh` so the headline "reads" while the marquee scrolls behind. Marquee: absolute-positioned track, `will-change: transform`, duplicate the child set 2× for seamless loop, `gsap.to(track, { xPercent: -50, duration: 40, ease: 'none', repeat: -1 })`. Video hover: on `pointerenter`, `gsap.getById(marqueeId).timeScale(0)`, `video.muted = false; video.play()`; on `pointerleave` restore `timeScale(1)`. Metrics strip: fetch server-side, hydrate a `<motion.span>` with `useSpring` on the numeric — animate `0 → value` over 1.2s `power2.out` when in view. Colors as CSS vars `--bg: #EAE9E6; --accent: #ECFDAD; --ink: #121212`. Breakpoints: 640 / 1024 / 1440; kill horizontal marquee under 640 and switch to swipeable snap-scroll `scroll-snap-type: x mandatory`.

WHY IT WORKS: Two-color palette forces contrast to do all the work — the acid lime is a semantic accent (active/live/metric) not a decoration, so every occurrence carries meaning. The sticky-headline / moving-content pattern gives figure-ground separation (Gestalt) without any card chrome. Live numbers create a truth signal: static case studies feel like brochures; auto-updating metrics feel like a product. Motion semantics — slow marquee = calm; number counter = energy — pace the reader through each case.

GOOD: Live data feed from Airtable is the killer detail — most agency sites lie with screenshots, this one won't. BAD: With JS off, the marquee freezes and the live numbers disappear; there's no static fallback. Accessibility on the auto-playing videos is thin (no captions surfaced in the reel view).

CLEAN APPLICATION: Apply this to Clean's **landing-page "proof" section** by replacing generic testimonial cards with live-metric case study modules. Build 3 sticky-headline sections — "Cursor cut context spend 61%", "Vercel agent latency dropped 340ms", "Linear team ships 2× more agent flows" — each with (a) a sticky chapter title, (b) a horizontal marquee of screenshots or agent-trace snippets, (c) a live metric strip that pulls from a Clean-hosted Airtable of aggregate customer KPIs (tokens saved, p95 latency, requests routed). Use the same acid-accent-on-neutral color logic but map to Clean's palette: neutral off-white + a single Clean brand accent for the live metric. Refresh daily.

## Radian — https://www.awwwards.com/sites/radian

WHAT IT DOES: Product launch site for an electric enduro motorcycle. Structure is a full-viewport hero (bike hero shot, name mark, spec ticker), followed by scroll-scrubbed 3D-ish product reveal (image sequence, ~80–120 frames), then a chapter of spec panels, ending in a live configurator (color, wheels, seat, add-ons) with real-time price. Interactions: BARBA.js page transitions between marketing pages and configurator (a full-viewport yellow curtain wipes in on route change), GSAP ScrollTrigger scrubs the image sequence tied to `scrollY`, and hover-based magnetic buttons snap toward the cursor. Palette is two colors: hazard yellow (`#FDEB07`) + near-black (`#121714`). Type is a display grotesque set very large (18–22vw) with generous negative tracking.

PACKAGES:
- `gsap` + `ScrollTrigger` + `Draggable` + `Flip`
- `@barba/core` for SPA-style page transitions
- Webflow (host); if rebuilding: `next` 15 + `framer-motion` for micro-interactions
- `lenis` for smooth scroll
- `three` + `@react-three/fiber` + `@react-three/drei` only if using real 3D; the reference uses a scrubbed image sequence (cheaper, sharper on mid-range devices)
- Fonts: a heavy display grotesque (e.g. `PP Editorial New Ultrabold` or `PP Right Grotesk Spatial Black`)

IMPLEMENTATION: Component tree — `<Layout>` → `<Barba>` → `<Hero>` + `<ScrubReveal>` + `<SpecGrid>` + `<Configurator>`. Hero: yellow flat background, black bike PNG at 80% viewport width, `mix-blend-mode: multiply` on grain overlay. ScrubReveal: preload frames 000–120 via `Promise.all(images.map(preload))`; render one `<canvas>` and blit the current frame; drive index with ScrollTrigger `scrub: 1, start: 'top top', end: '+=250%'`. SpecGrid: `display: grid; grid-template-columns: repeat(12, 1fr)` with headline row spanning 12 and metric cells spanning 3, using CSS `container-type: inline-size` for responsive re-layout. Configurator: option state in Zustand, price recompute in a `useMemo`, swap bike-view with GSAP `Flip.from(state)` for morphed transitions. BARBA page wipe: on `beforeLeave`, `gsap.to('.wipe', { yPercent: 100, duration: 0.6, ease: 'expo.inOut' })`, then swap DOM, then reverse wipe. Magnetic buttons: on `pointermove`, translate button by `(cursor - center) * 0.25` with `gsap.quickTo`.

WHY IT WORKS: Yellow-on-black is a hazard signal — it borrows industrial/moto semiotics and skips any explanation. Oversized display type creates instant visual hierarchy: only one thing is loud, so the eye has zero ambiguity about what to read first. Scrub-reveal converts a product photoshoot into a spec disclosure — you scroll and the machine explains itself. BARBA curtain transitions preserve pace (no reload flash) and give the configurator a cinematic entrance.

GOOD: Scrub-with-images (not real 3D) is a masterclass in "look expensive, load light" — 120 WebP frames beat a 40MB glTF on cold cache. BAD: Configurator layer is heavy on desktop-first assumptions; mobile config UI collapses to a modal that hides the bike (defeats the point of a visual configurator).

CLEAN APPLICATION: Apply this to Clean's **product demo section on the landing page** by building a scroll-scrubbed reveal of the MCP proxy in action. Render a `<canvas>`-based image sequence (60–80 frames) showing: (1) raw agent request hitting the proxy, (2) TOON format compression visualized as tokens collapsing, (3) response returning smaller/faster. Scrub tied to a 200vh section using GSAP ScrollTrigger. Underneath the scrub, mount a mini "config-your-own-proxy" mock — pick model, pick tools, pick output format — with a live token-count and price readout that recomputes on change (like Radian's configurator but for tokens saved). Keep Radian's contrast philosophy: one bold Clean accent, everything else black or off-white; skip the yellow.

## Wembi — https://www.awwwards.com/sites/wembi

WHAT IT DOES: A minimalist B2B/startup site (2-color palette: mint `#beff8b` and dusty blue `#9dbbc5`) using a Nuxt + Craft CMS backbone with GSAP-heavy scroll motion. Layout is single-column, generous whitespace, one hero statement per viewport. Signature interactions: (1) headline text splits per-character and animates in on scroll via `gsap.utils.splitText` + stagger, (2) shape-morph transitions between sections — SVG paths tween between blob shapes via `MorphSVGPlugin`, (3) sticky-scroll pinning where a shape "grows" from a 40px dot to a full-viewport panel as the section scrolls. Cursor is a custom blended cursor that inverts under text.

PACKAGES:
- `gsap` + `ScrollTrigger` + `SplitText` (paid Club GreenSock plugin) + `MorphSVGPlugin` (paid)
- `nuxt` 3, `@nuxt/content` or Craft CMS
- `lenis` (or `@studio-freight/lenis`) for smoothscroll
- `pnpm` if scaffolding new
- Fonts: neo-grotesk pairing (e.g. `PP Neue Machina` + `PP Editorial New` for accents)

IMPLEMENTATION: Component tree — `<App>` → `<Header>` (sticky, transparent) → `[<Section>]` → each section is `<Shape>` + `<HeadlineSplit>` + `<Body>`. HeadlineSplit: `const split = new SplitText(el, { type: 'chars,words' }); gsap.from(split.chars, { yPercent: 100, opacity: 0, stagger: 0.02, ease: 'expo.out', scrollTrigger: { trigger: el, start: 'top 80%' } })`. Shape-morph: two SVG `<path>` elements swapped via `gsap.to(pathA, { morphSVG: pathB, duration: 1.2, ease: 'power3.inOut' })`, triggered on scroll enter. Section pinning: `ScrollTrigger.create({ trigger: section, start: 'top top', end: '+=100%', pin: true })` with a `<Shape scale={progress}>` where `progress` is bound to `scrollTrigger.progress`. Colors as CSS vars `--mint`, `--blue`, `--ink`. Cursor: fixed `mix-blend-mode: difference` div that tracks pointer with `gsap.quickTo(cursor, 'x', { duration: 0.2 })`.

WHY IT WORKS: Two soft pastels + heavy motion is unusual — most designers pair loud motion with loud color. The restraint puts weight on typography and pacing, which reads as confidence. Split-text stagger on entry gives every headline a micro-narrative — the reader "hears" the sentence assemble. Shape-morph between sections is Gestalt continuity: your eye follows the shape instead of losing place at a section break.

GOOD: The morphSVG shape-continuity between sections is the best transition pattern of the year — nothing jumps, everything flows. BAD: SplitText + MorphSVGPlugin are GreenSock Club paid plugins; a free-tier replica needs custom SVG interpolation and manual character wrapping (non-trivial engineering).

CLEAN APPLICATION: Apply this to Clean's **onboarding flow** by using shape-continuity to walk a new user through the mental model: "your agent" → "your proxy" → "your model provider" — three panels where a single SVG blob morphs from a chat bubble (agent) to a hexagon (proxy) to a cloud (provider), each with a `SplitText` headline. Use Clean's palette (whatever the current brand pair is — likely a cool neutral + one accent). Pin each panel for 100vh scroll before the morph triggers. This turns onboarding docs into a scrolly story.

## NRG | Build Your Data Center — https://www.awwwards.com/sites/nrg-build-your-data-center

WHAT IT DOES: B2B corporate campaign site targeting hyperscalers and AI companies looking to secure power for data-center builds. Structure: hero with a scroll-triggered isometric data-center diagram (three.js), 4 disclosure panels (Site Evaluation → Power → Construction → Handoff), an interactive US map with clickable regions showing available capacity in MW, and a lead-capture form gated by a "size your build" calculator (rack count → estimated MW → estimated timeline). Interactions: three.js scene with camera dolly + scroll-linked orbit, D3-driven map with data tooltips, form with progressive disclosure (each answer unlocks the next field with a `height: auto` GSAP tween). Palette is high-corporate: deep navy + electric blue accent + white.

PACKAGES:
- `three` + `@react-three/fiber` + `@react-three/drei` (for `<Environment>`, `<useGLTF>`, `<OrbitControls>`)
- `gsap` + `ScrollTrigger`
- `d3-geo` + `topojson-client` for the US map + `us-atlas` for topology
- `react-hook-form` + `zod` for the calculator/lead form
- `framer-motion` for form field transitions
- `next` 15 (app router)

IMPLEMENTATION: Component tree — `<Page>` → `<Hero3D>` → `<PillarStack>` (4 panels) → `<CapacityMap>` → `<SizingCalculator>` → `<LeadForm>`. Hero3D: R3F `<Canvas dpr={[1,2]}>` with a glTF isometric data-center model, camera at `[8, 6, 8]`, `<OrbitControls enableZoom={false}>` bound to scroll — use `useFrame` and read a shared `scrollY` from `useScroll()` (drei) to drive `camera.position.lerp(targetForProgress(scrollY))`. CapacityMap: `d3.geoAlbersUsa()` projection, states as `<path>` from `us-atlas/states-10m.json`; on hover show a `<Popover>` with MW available (dummy data acceptable — real data comes from CMS). SizingCalculator: 3 steps managed with `useState`, each step reveals via `<motion.div initial={{ height: 0, opacity: 0 }} animate={{ height: 'auto', opacity: 1 }}>`. Result panel computes `MW = racks * 12kW / 1000 * 1.4` (PUE fudge) and displays with a spring-animated counter. LeadForm: react-hook-form + zod validation, honeypot field, submit to `/api/lead` which forwards to HubSpot.

WHY IT WORKS: This is the template for "sell to AI companies without pandering." No neon gradients, no rocket ships — just a technical calculator, a real map, and confident type. The scroll-linked isometric 3D reads as "we have the physical assets," which is the whole pitch. Progressive form disclosure reduces perceived form length by 60%+ (documented conversion pattern). The calculator is the hook: prospects self-qualify before hitting sales.

GOOD: Sizing calculator is the exact right conversion tool for infrastructure — it turns a lead form into a self-serve estimate. BAD: The isometric 3D scene tops 4MB on cold load, and the map has no keyboard nav (arrow-key state selection is missing).

CLEAN APPLICATION: Apply this to Clean's **pricing / "sizing" page** by building an interactive calculator: pick your monthly agent request volume, average tokens per request, and models in use → Clean's proxy returns estimated tokens saved, dollars saved, and p95 latency reduction, all recomputed live. Below the calculator, mount a scroll-linked 3D visual of the Clean architecture: agent → proxy → provider, with the proxy node "glowing" more as savings scale. Use three.js/R3F, keep the load under 1.5MB (no gLTF meshes, use primitives + shaders). Progressive form disclosure to capture the lead only after the user sees their estimated savings.

## Aceternity UI — Shaders Blocks — https://ui.aceternity.com/blocks/shaders

WHAT IT DOES: A newly-shipped block library from Aceternity offering ~15 GLSL fragment-shader backgrounds delivered as drop-in React components (dither, aurora, plasma, liquid, mesh-gradient, noise-warp, halftone, glitch). Each block wraps a `<canvas>` with a compiled fragment shader and exposes props for color palette, speed, scale, and mouse-reactivity. Use pattern: import one block, plug in your brand colors, drop it behind a hero — instant "expensive-looking" background that renders at 60fps on integrated GPUs.

PACKAGES:
- `three` (or vanilla WebGL for lightest builds)
- `@react-three/fiber` if consuming as R3F component
- `tailwindcss` + `clsx` + `tailwind-merge`
- `motion` (framer-motion successor) for any layered DOM animation
- `next` 15 with `next/font`
- Aceternity CLI: `npx aceternity-ui@latest add shader-background-aurora`

IMPLEMENTATION: Component tree — `<Hero>` → `<ShaderBackground variant="aurora" colors={[...]} speed={0.6} mouseReactive />` → `<HeadlineStack>`. Shader mount: full-viewport `<canvas>` positioned absolute, `pointer-events: none`, `z-index: -1`. Fragment shader consumes `uniform float u_time`, `uniform vec2 u_resolution`, `uniform vec2 u_mouse`, and three color uniforms; you multiply time-scrolled Perlin noise fields to get the aurora flow. `useFrame((s) => (uniforms.u_time.value = s.clock.elapsedTime * speed))`. On resize, debounce `u_resolution` update. Overlay: a `backdrop-blur-sm` + `bg-black/40` layer between shader and text to guarantee WCAG contrast on any shader state. Kill switch: `prefers-reduced-motion: reduce` → replace canvas with a static screenshot fallback.

WHY IT WORKS: Shader backgrounds are the 2026 answer to gradient walls — they carry motion without asking the user to interact, so they register as "alive" before scroll happens. Because they're fragment-shader-based (not video), they're 2–5KB compiled and render on GPU with near-zero CPU. Semantically, motion in the background paired with a still headline creates figure-ground reversal that makes the copy pop.

GOOD: Best-in-class ratio of visual impact to bundle size — a 3KB shader replaces a 4MB looping video. BAD: Without a `prefers-reduced-motion` fallback (some variants ship without it), the aurora shader can trigger vestibular discomfort. Also — on low-end Android GPUs a couple variants drop to 20fps.

CLEAN APPLICATION: Apply this to Clean's **landing page hero** by replacing whatever background is there with the "aurora" or "mesh-gradient" shader tinted to Clean's brand colors. Set `speed={0.4}` (calm, not busy), `mouseReactive={true}` (subtle pull toward cursor), overlay a `bg-black/30 backdrop-blur-[2px]` layer beneath the headline for contrast. Add a `prefers-reduced-motion` fallback that swaps to a static screenshot of one shader frame. Keep the shader confined to the hero — don't use it on multiple sections or the site will feel like a screensaver.

## BUILD DIRECTIVE

TARGET: Clean landing-page **hero + product-demo section** (rebuild both, as one continuous scroll experience).

APPROACH: Ship a landing page that opens on a calm shader-gradient hero (Aceternity aurora shader, Clean-branded palette, mouse-reactive) with a single oversized grotesque headline stating Clean's core value prop and one CTA. On scroll, transition into a Radian-style scrub-reveal product demo: a 60–80 frame `<canvas>` image sequence tied to ScrollTrigger scrubs through the MCP proxy in action — raw request → TOON compression → smaller/faster response. Below the scrub, land in a Hildén & Kaira-style live-metric strip: three sticky-headline modules showing aggregate customer proof (tokens saved, latency reduced, dollars saved) with numbers auto-refreshed from a Clean-hosted metrics endpoint (real or synthesized daily). Close with a NRG-style sizing calculator that captures the lead only after the prospect sees their own estimated savings.

STACK:
- `next` 15 (app router)
- `tailwindcss` + `class-variance-authority` + `tailwind-merge`
- `gsap` + `@gsap/react` + `ScrollTrigger` + `SplitText`
- `lenis` for smoothscroll
- `three` + `@react-three/fiber` (only for the shader hero — no meshes)
- `motion` (framer-motion) for micro-interactions and form disclosure
- `react-hook-form` + `zod` for the calculator/lead form
- Aceternity: `npx aceternity-ui@latest add shader-background-aurora`
- Fonts: one heavy grotesque display (`PP Neue Montreal Bold` or equivalent) + one mono for metrics (`Berkeley Mono` / `JetBrains Mono`)

REFERENCE URLS:
- https://www.awwwards.com/sites/radian — scroll-scrub product reveal pattern
- https://www.awwwards.com/sites/hilden-kaira — live-metric case-study module
- https://ui.aceternity.com/blocks/shaders — hero shader background

KEY CONSTRAINTS: Clean is developer infrastructure — the page must read as fast, precise, and technically credible. Avoid: (1) neon AI gradients as decoration; (2) generic "AI startup" hero videos of a laptop with a chatbot; (3) rotating 3D logos; (4) any shader with strobing / high-frequency flicker; (5) more than one shader region on the page; (6) fake numbers — live-metric strip must reflect actual aggregated customer telemetry or be labeled as illustrative. Cold-load target: LCP < 1.6s on 4G, JS bundle < 180KB gzipped for the hero route. Every animated element gets a `prefers-reduced-motion: reduce` fallback. Every metric readout gets a hover/tap explanation of what it means and how it's measured.
