## Indigo Laboratory — https://www.awwwards.com/sites/indigo-laboratory

WHAT IT DOES: Single-page narrative for a jewelry brand built as five sequential "tales" — each scene is a full-bleed canvas where text, image, and audio cross-fade under a fixed scroll-locked progression. Curtains.js renders WebGL displacement shaders on imagery (RGB split, fluid warp) keyed to scroll velocity, while GSAP ScrollTrigger pins each chapter, advances a global progress timeline, and dispatches per-chapter audio cues against Howler-style sound objects. Layout is a vertical "story-snap" container with a fixed nav showing tale indices; horizontal panel reveals occur via clip-path masks on chapter pivots.

PACKAGES:
- nuxt (Nuxt 3, Vue 3)
- gsap, gsap/ScrollTrigger, gsap/ScrollSmoother
- curtainjs (Curtains.js — vanilla WebGL plane renderer)
- lenis (smooth scroll, paired with ScrollTrigger.normalizeScroll)
- @studio-freight/tempus (rAF loop coordinator)
- howler (audio sprite playback)
- splitting-js (character-level text reveals)
- @vueuse/core (intersection + window observers)
- postprocessing (selective bloom on hero plane)
- vite, typescript, sass

IMPLEMENTATION:
1. Root layout = `<SmoothScroll>` (Lenis + ScrollSmoother in normalizeScroll: true mode) wrapping `<ChapterStack>` of 5 `<Chapter>` components, each 100vh tall, position: sticky with top: 0 on the WebGL plane and absolute text overlay.
2. Single Curtains.js instance mounted on a fixed `<canvas>` behind the DOM. Each chapter declares `<Plane vertexShader fragmentShader uniforms={ uProgress, uVelocity, uTexture, uTextureNext }>`. Shader pattern: `mix(texCurrent, texNext, smoothstep(0.0, 1.0, uProgress)) + rgbShift(uv, uVelocity * 0.04)`.
3. GSAP timeline per chapter pinned by ScrollTrigger with `pin: true, scrub: 1.2, anticipatePin: 1, end: '+=120%'`. The master timeline drives `uProgress` (0→1) and triggers `audio.play(chapterId)` on `onEnter`.
4. Text reveal: `gsap.from('.line', { yPercent: 100, stagger: { each: 0.04, from: 'random' }, ease: 'expo.out', duration: 0.9 })` after SplitText line wrap.
5. Cursor: custom 12px circle that scales to 64px with mix-blend-mode: difference on hoverable nodes (transition 280ms cubic-bezier(.2,.8,.2,1)).
6. Color palette: `--ink: #6A15FF; --paper: #F4F3EB;`. Typography: a serif display (~Editorial New / GT Sectra) at clamp(72px, 14vw, 220px) for chapter numerals, monospace caption.
7. Responsive: at <768px disable Curtains plane, replace with static `<img loading="eager">` plus CSS-only animations; preserve scroll-snap-type: y mandatory on the chapter container.
8. Perf: preload next chapter texture on `onEnter` of previous; cap DPR at min(devicePixelRatio, 1.5); pause Curtains rAF when document.visibilityState === 'hidden'.

WHY IT WORKS: Two-color palette forces hierarchy onto type and motion alone — every transition reads as content, not decoration. Sound-on-scroll creates a multi-sensory commitment device (the user invests attention because leaving means losing the soundtrack). The displacement shader keyed to scroll velocity makes the interface feel reactive to the user's body, not their input — that's the trick behind "premium" feel.

GOOD / BAD:
- GOOD: Sub-1s LCP despite the WebGL chrome; audio is opt-in via gesture, accessibility-respecting.
- BAD: ScrollSmoother + ScrollTrigger pin combo can drift on iOS Safari rubberband; no skip-chapter affordance for return visitors.

CLEAN APPLICATION: Apply this to Clean's landing-page product story by replacing the current static "How it works" section with a 4-chapter scroll-pinned sequence: (1) Raw MCP chaos — colored particles colliding; (2) Clean intercepts — particles snap to a clean lattice; (3) TOON encoding — text glyphs reflow into compact tokens with character count counter ticking down; (4) Agents stay in sync — three avatar nodes pulse in unison. Use Curtains.js or react-three-fiber planes with the same uProgress pattern, scrub-pinned at 120vh per chapter. Keep Clean's existing palette but lean into one accent color for the active chapter glyph.

## Pacôme Pertant Portfolio — https://www.pacomepertant.com/

WHAT IT DOES: Motion-designer portfolio that pairs a Three.js hero canvas (an extruded text mesh with bevel + matcap, slowly drifting on a Perlin field) with a GSAP-orchestrated horizontal-then-vertical scroll narrative. The project case studies are presented as full-bleed video reels that mask-reveal as the user scrolls, with custom audio scrubbing tied to scroll velocity — the music slows and pitches down when the user pauses scroll. Awwwards Developer Award winner with a 7.76 score, highest in the June batch.

PACKAGES:
- nuxt (Nuxt 3)
- three, @types/three
- gsap, gsap/ScrollTrigger, gsap/CustomEase, gsap/Observer
- lenis
- tone (Tone.js for pitch/playback-rate audio on scroll)
- splitting-js or gsap/SplitText
- @studio-freight/hamo (rAF + viewport hooks)
- pixi.js (used for case-study video planes with displacement)
- vite, typescript

IMPLEMENTATION:
1. Hero: `<Canvas>` (vanilla Three.js, OrthographicCamera, alpha: true). TextGeometry from a loaded `.json` font (use troika-three-text for crispness), MeshMatcapMaterial with a metallic-noir matcap PNG. Animate position via a 3D noise: `mesh.position.set(noise(t,0,0)*0.4, noise(0,t,0)*0.2, 0)` at ~30fps.
2. Scroll choreography: a master GSAP timeline scrubs with `ScrollTrigger({ trigger: '#stage', scrub: 1.5, pin: true, end: '+=600%' })`. Sub-tweens move the camera along a `CatmullRomCurve3`, snap to keyframes at case-study sections.
3. Case-study reveal: each project is a `<video muted loop playsinline>` clipped by `clip-path: inset(50% 50% round 24px)` and expanded via GSAP to `inset(0% 0% round 0px)` over 1.2s with `CustomEase.create('reveal', 'M0,0 C0.2,0.9 0.1,1 1,1')`.
4. Audio: Tone.Player loads a single mp3; `playbackRate` is bound to `scrollVelocity` clamped [0.6, 1.4] and smoothed with a one-pole filter (`y += (target - y) * 0.08`).
5. Cursor: custom canvas-rendered cursor (not DOM) so it can render under blend modes and at 120Hz.
6. Type: variable-weight grotesque (Söhne Buch / Inter Tight) at `font-variation-settings: 'wght' var(--w)`; `--w` animates 400→800 on hover with a 0.4s ease.
7. Responsive: at <900px drop the 3D hero to a 2D SVG marquee (still animated); collapse horizontal scroll to vertical with the same scrubbed reveals.
8. Perf: powerPreference: 'high-performance', antialias: false + msaa via THREE.WebGLRenderer with `setPixelRatio(Math.min(devicePixelRatio, 1.75))`; pause the audio AudioContext on tab blur.

WHY IT WORKS: The hero teaches you the language — slow drift, soft matcap, no UI noise — so when the case studies snap and reveal, the contrast lands. Pitch-shifting audio on scroll is rare and creates an unconscious "I am driving this thing" feedback loop. Hierarchy is sequential rather than spatial: you can't skim, you can only progress.

GOOD / BAD:
- GOOD: 60fps on a base M1 MacBook Air; works without JS-disabled fallback (static portfolio links remain accessible).
- BAD: Six-screen pinned section is hostile to keyboard navigation; no `prefers-reduced-motion` fallback for the audio pitch shift.

CLEAN APPLICATION: Apply this to Clean's product demo section (replacing the static "see it in action" video). Build a single horizontal-scrubbed pinned section that walks through: prompt → MCP proxy intercepts → TOON compression visualized as text shrinking → agent receives clean context. Use a small Three.js scene of a compact rotating token cube whose facets contain live code snippets via `troika-three-text`. Bind a subtle electronic loop's playback rate to scroll velocity so the demo feels physical. Keep total pinned distance to 400vh max to respect mobile.

## Serve Robotics — https://www.awwwards.com/sites/serve-robotics

WHAT IT DOES: Marketing site for the sidewalk-delivery robot company built around a 3D hero model (compressed glTF, ~2.4MB Draco-encoded) that scroll-rotates and zooms across five product feature sections. Information density is high — multi-column data layouts, animated number counters, and a route-map SVG with stroke-dasharray draw-on. Stack is React + Tailwind + Sanity for CMS-driven case studies. Awarded Site of the Day June 11 with Developer Award.

PACKAGES:
- next (Next.js 15 app router)
- react, react-dom
- three, @react-three/fiber, @react-three/drei, @react-three/postprocessing
- gsap, gsap/ScrollTrigger
- framer-motion (page transitions + UI micro)
- lenis, @darkroom.engineering/hamo
- tailwindcss, tailwind-merge, clsx
- @sanity/client, next-sanity (CMS)
- @next/font (variable Söhne or local Geist)
- sharp (Sanity image pipeline)
- countup.js (number animation)

IMPLEMENTATION:
1. Hero: `<Canvas dpr={[1, 1.75]} gl={{ antialias: true, alpha: true }}>` with `<Suspense>`-wrapped `<RobotModel>` using `useGLTF('/robot.glb')`. Draco-compress via `gltfjsx --draco`. Three lights: directional key (intensity 1.4, shadow.mapSize 1024), hemi (sky #61E5C0, ground #0F4233), and rim spot.
2. Scroll-bound transforms: wrap the model with `<ScrollControls pages={5} damping={0.15}>` from drei, then a custom hook `useScrollChoreo()` interpolates rotation, position, and material emissive intensity per page via `useFrame`.
3. Sections: each `<Section>` is a CSS grid `grid-template-columns: repeat(12, 1fr); gap: 24px;` with type at `clamp(48px, 6vw, 96px)` for headings. Use `text-balance` for headline wrap.
4. Data viz: route-map SVG paths animated with `gsap.fromTo('.route', { strokeDashoffset: 1 }, { strokeDashoffset: 0, scrollTrigger: { trigger: ..., start: 'top 70%', scrub: true } })` (paths use `pathLength="1"`).
5. Number counters use `CountUp` triggered by `IntersectionObserver` at threshold 0.4, duration 1.8s, ease-out-quart.
6. Palette tokens (Tailwind config): `forest: '#0F4233', mint: '#61E5C0', cream: '#F5F1E8'`. Buttons use `bg-mint text-forest` with `transition-transform duration-300 hover:scale-[1.02] active:scale-[0.98]` plus a 1px border that animates on hover via pseudo-element.
7. Page transitions: framer-motion `<AnimatePresence mode="wait">` with a top-down clip-path wipe in mint, 600ms expo.out.
8. Perf: `next/dynamic` import the Canvas with `ssr: false`; preload model with `<link rel="preload" as="fetch" href="/robot.glb" crossOrigin="anonymous">`; quality tier check (`gpuInfo.fps < 45 → drop shadows, switch to MeshBasicMaterial`).

WHY IT WORKS: The robot is the protagonist on every screen, so context never resets — that's continuity-of-character, the same trick Pixar uses for emotional throughline. Mint-on-forest is high contrast (~9:1) but feels soft because both sit in the same hue family. Data feels credible because the counters animate slowly enough to be read, not skimmed.

GOOD / BAD:
- GOOD: Fully SSR'd content layer; SEO and accessibility don't suffer for the 3D layer.
- BAD: 2.4MB glb plus ~400KB shaders means mobile cold-load is ~3.1s on a throttled 4G; no obvious fallback poster image during shader compile.

CLEAN APPLICATION: Apply this to Clean's landing-page hero by replacing the static product screenshot with a small 3D "context graph" — a low-poly node-and-edge mesh where nodes are agents (Claude, Cursor, Cline) and edges pulse when Clean syncs them. Scroll-rotate this graph across the page so it's the protagonist on every section. Use the same `<ScrollControls>` + `useScrollChoreo` pattern. Keep the model under 800KB Draco-compressed. Critical: build an SSR-rendered text version of every section so the 3D never blocks first paint.

## Fauna Robotics — https://www.awwwards.com/sites/fauna-robotics

WHAT IT DOES: Brand site for a humanoid-robotics startup (Sprout) that deliberately rejects the cold-industrial trope — uses pastel #EFEFEF / #D1E3FF, hand-drawn illustration accents, asymmetrical card layouts, and a hero video of the robot performing comedic "personality" gestures. Built on Webflow with custom Lottie + Rive embeds; navigation is sticky pill with on-scroll color inversion. Awarded SOTD June 16 with Developer Award.

PACKAGES (if rebuilt in code rather than Webflow):
- next, react
- @lottiefiles/dotlottie-react (Lottie playback)
- @rive-app/react-canvas (Rive state machines for the robot mascot)
- framer-motion
- tailwindcss, @tailwindcss/typography
- gsap (lightweight — just ScrollTrigger for nav inversion)
- swiper (asymmetric carousel)
- @vercel/og (dynamic OG images per page)
- next-mdx-remote (case study content)
- iconoir-react (line icons matching the illustrative style)

IMPLEMENTATION:
1. Hero is a `<video autoplay muted loop playsinline poster="hero-poster.webp">` with `object-fit: cover; mix-blend-mode: multiply;` over a pastel gradient (`bg-gradient-to-br from-[#EFEFEF] to-[#D1E3FF]`).
2. Mascot in lower right: Rive state machine with inputs `{ hover: bool, scrollVelocity: number }` — eyes track cursor via `inputs.lookX = clamp(mouseX / vw, -1, 1)`.
3. Sticky pill nav: `position: sticky; top: 16px; backdrop-filter: blur(20px) saturate(140%); background: rgba(255,255,255,0.6); border-radius: 999px;`. On scroll past hero, GSAP toggles a class that inverts to `bg-[#0A0A0A]/70 text-white` over 280ms.
4. Card layout: CSS grid with explicit non-uniform tracks `grid-template-columns: 1.4fr 1fr 1.2fr; grid-auto-rows: minmax(280px, auto);` — cards span 1–2 cells unpredictably, creating Bauhaus-style rhythm.
5. Microinteractions: every button is a `<motion.button whileHover={{ y: -2, rotate: -1 }} whileTap={{ scale: 0.97 }} transition={{ type: 'spring', stiffness: 320, damping: 22 }}>`.
6. Type: pairs a humanist sans for body (Inter / GT America) with a chunky display (Söhne Breit or General Sans Semibold) at headlines; line-height 0.95 on display, 1.5 on body.
7. Mobile: switch grid to single column with `aspect-ratio: 4/5` cards; replace Rive mascot with a single autoplaying Lottie for battery.
8. Perf: `next/image` with `priority` on hero poster, lazy on everything else; Rive runtime is ~150KB gzipped — acceptable for the payoff.

WHY IT WORKS: Robotics companies all look the same (black, gray, technical). Choosing warm pastels and a character-led mascot is positioning by aesthetic — the page tells you "we are the friendly one" before you read a word. Asymmetric grids resist scanning, which slows the user just enough to actually absorb copy. Spring micro-interactions match the brand's "alive" claim.

GOOD / BAD:
- GOOD: Personality-first; immediate emotional read; differentiates in a crowded category.
- BAD: Webflow build means custom logic is hacked into embeds; Rive on background can cause jank on low-end Android.

CLEAN APPLICATION: Apply this to Clean's docs/onboarding flow surface (currently the weakest, most generic surface). Replace the standard docs sidebar with a Rive mascot — a small "Clean cube" character that reacts to which doc section the user is reading (rotates to face the current heading, blinks when the page is idle, pulses when a code block is copied). Use asymmetric card grids on the "Get started" page so the three onboarding paths (Claude Code, Cursor, custom MCP client) feel distinct, not parallel. Lean into one accent color per path.

## Shader.se — https://tympanus.net/codrops/2026/05/19/80s-business-tech-seamless-scene-transitions-inside-shader-ses-scroll-driven-webgpu-pipeline/

WHAT IT DOES: Sweden-based creative studio's portfolio rendered as an 80s-corporate-tape aesthetic running entirely on WebGPU via Three.js Shading Language (TSL). Scenes are pre-composed FBOs that bleed into each other through shader-based transitions tied to scroll position — no DOM cuts, no page navigation, the entire site is one continuous WebGPU canvas with `@pmndrs/uikit` rendering "DOM-like" layout inside the 3D context. This is the most technically forward pattern in the 48h window — the live frontier of post-WebGL rendering.

PACKAGES:
- three (>= 0.170 with WebGPURenderer)
- three/tsl (Three.js Shading Language node materials)
- @react-three/fiber (v9 with WebGPU support)
- @react-three/drei
- @pmndrs/uikit (forked/patched for WebGPU)
- lenis (patched to snap between hero and "Our work" sections)
- next or vite (Vite preferred — easier WGSL shader HMR)
- zustand (cross-scene state)
- howler or tone (CRT scanline audio loops)
- @webgpu/types (TypeScript definitions)

IMPLEMENTATION:
1. Boot: `import { WebGPURenderer } from 'three/webgpu'; const renderer = new WebGPURenderer({ antialias: true }); await renderer.init();`. Fall back to WebGLRenderer if `!navigator.gpu`.
2. Scene composition: each "scene" is a React component that exposes a render function via `useImperativeHandle(ref, () => ({ render: (state, nextTex?) => { renderer.setRenderTarget(fbo); renderer.render(scene, camera); return fbo.texture; } }))`.
3. Parent `<Pipeline>` calls each scene's render() in order each frame and composites results through a TSL transition material: `output = mix(currentTex.sample(uv), nextTex.sample(uv), smoothstep(0, 1, scrollT) * vignette(uv))`. Add a CRT-mask TSL function (`step(0.5, fract(uv.y * 480.0))`) for the scanline pass.
4. UI: replace HTML for headings and buttons with `@pmndrs/uikit` `<Container flexDirection="column" gap={16}> <Text fontSize={48}>...</Text> </Container>` so layout lives inside the WebGPU pipeline and inherits the post-processing.
5. Smooth scroll: Lenis with `wheelMultiplier: 0.8, smoothTouch: false, syncTouch: true`. Patch Lenis to support `snap({ targets: [0, hero.height], duration: 0.8 })` between major scenes.
6. Type stack: an 80s monospace (IBM Plex Mono / Berkeley Mono) plus a serif label (GT Sectra) — both loaded as MSDF atlases via troika-three-text for crisp WebGPU rendering at any zoom.
7. Performance: WebGPU command buffers batched per frame; use `device.queue.writeBuffer` for uniform updates instead of recreating bind groups; cap to one offscreen FBO per scene (1024×576 or device-pixel-ratio scaled).
8. Fallback path: if WebGPU unavailable, render scenes as DOM video posters + a CSS-only scroll narrative — same content, different render surface.

WHY IT WORKS: This is post-WebGL — the entire interface is one rendered output, so transitions can affect lighting, geometry, and UI simultaneously. There are no seams between "page" and "canvas" because there is no DOM. The retro 80s framing is design-as-thesis: it says "we know rendering history, we've moved past it." Hierarchy emerges from shader passes, not z-index.

GOOD / BAD:
- GOOD: Genuinely the rendering frontier — if you can build this you signal technical seriousness immediately.
- BAD: WebGPU adoption isn't universal — ~25% of mobile users still won't see it; fallback complexity doubles dev cost; the aesthetic is style-heavy and risks being illegible if applied without restraint.

CLEAN APPLICATION: Apply this to a single high-impact Clean surface — the `/mcp` product page hero. Build a small WebGPU TSL scene where the Clean logo dissolves into a stream of TOON-encoded tokens that flow through a "context pipe" (a tube geometry with refractive shader) and emerge as clean structured data into agent avatars. Use scroll-driven uProgress to morph the scene from "chaos in" to "structure out." Use it as a single hero element — do NOT build the whole site this way. Pair with traditional DOM content below the fold. Add explicit "this requires WebGPU" graceful copy as a non-blocking note.

## BUILD DIRECTIVE

TARGET: Clean's landing-page hero + the immediately-below "How it works" section on https://www.tryclean.ai/ (the highest-leverage surface — every visitor sees it, and it currently does not communicate Clean's technical sophistication relative to the competitive set scouted today).

APPROACH: Replace the current static hero with a small Three.js + react-three-fiber scene featuring a low-poly "context graph" — agent nodes (Claude, Cursor, Cline, custom) connected by edges that pulse when Clean syncs them. The graph rotates and rearranges scroll-bound across four pinned narrative chapters: (1) Raw MCP chaos — particles colliding; (2) Clean intercepts — particles snap to lattice; (3) TOON compression — a token-count number counter ticks down from a huge value to a small one as glyphs reflow; (4) Agents sync — the four nodes pulse in unison and resolve to a steady-state ambient pulse. Drive everything from one GSAP master timeline scrubbed via Lenis-smoothed ScrollTrigger; render the scene model at the bottom of every section as a continuous protagonist (Serve Robotics pattern). Keep the DOM SSR'd in full so SEO and first-paint are unaffected. Borrow the asymmetric card rhythm and the Rive-mascot-reacts-to-content pattern from Fauna Robotics for the "Get started" section directly below.

STACK:
- next@15 (app router, RSC)
- react@19, react-dom@19
- three@^0.170, @react-three/fiber@^9, @react-three/drei@^10
- gsap@^3.13, gsap/ScrollTrigger, gsap/CustomEase
- lenis@^1.3
- framer-motion@^12 (page-level transitions + small UI)
- tailwindcss@^4
- @rive-app/react-canvas (mascot)
- countup.js (token-count number animation)
- troika-three-text (crisp 3D text on token glyphs)
- @vercel/og (dynamic OG)
- typescript@^5.6

REFERENCE URLS:
- https://www.awwwards.com/sites/serve-robotics — protagonist 3D continuity + Tailwind + Next stack
- https://www.pacomepertant.com/ — scroll-scrubbed pinned narrative + audio-reactive feel (do NOT replicate audio pitch shift — implement as visual-only on Clean)
- https://tympanus.net/codrops/2026/05/19/80s-business-tech-seamless-scene-transitions-inside-shader-ses-scroll-driven-webgpu-pipeline/ — TSL transition pattern for the chapter dissolves (use WebGL fallback for now, structure code so WebGPU is a swap)

KEY CONSTRAINTS:
- Clean is developer infrastructure — the result must feel like Linear or Vercel, not Gucci. Avoid: hand-drawn illustration, pastel gradients, brand mascots in the hero (mascot is OK in onboarding only), Lottie-heavy decoration, anything labeled "playful."
- Sub-1.2s LCP on a base M1 / mid-tier Android — that means the 3D scene must NOT block render. SSR the text, defer-load `<Canvas>` with `next/dynamic({ ssr: false })`, and preload the model with `<link rel="preload">`.
- Total 3D payload < 500KB Draco-compressed. The "context graph" should be procedural geometry where possible, not an imported model.
- Respect `prefers-reduced-motion`: disable the scroll-pin chapters and replace with static section transitions.
- Token-count number must be real (or believably real) — pull from a `/api/stats` endpoint that returns a rolling 24h tokens-saved figure. If you can't, fake it convincingly with a deterministic seed so the number is the same per session.
- One accent color only on the active chapter — do not multi-color the graph. Keep brand palette consistent with current Clean site (whatever current accent is, preserve it).
- Every chapter must work without JS executed (fallback: a stacked vertical column of the 4 captions + 4 static images).
- No audio. No autoplay sound under any condition.
