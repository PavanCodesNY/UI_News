## Hiroto Sato — https://www.awwwards.com/sites/hiroto-sato

WHAT IT DOES: A creative-developer portfolio built as a single WebGL canvas that fills the viewport, with DOM overlay text pinned to the same scroll timeline as a Three.js scene. As the user scrolls, the camera dollies through a Blender-authored environment while GSAP ScrollTrigger drives per-section object reveals, material swaps, and text mask reveals — the DOM and canvas share a Lenis-smoothed scroll so the two never desync. Cursor position feeds a custom raymarched fragment shader that offsets a displacement field around a hero mesh, producing a chromatic "liquid metal" hover state.

PACKAGES:
- next@15 (App Router, RSC off for the canvas route)
- react@19, react-dom@19
- three@0.170.x
- @react-three/fiber@8.17.x
- @react-three/drei@9.114.x
- @react-three/postprocessing@2.16.x (Bloom, ChromaticAberration, N8AO)
- postprocessing@6.35.x
- gsap@3.13.x + gsap/ScrollTrigger, gsap/SplitText
- @studio-freight/lenis@1.1.x
- tailwindcss@4, @tailwindcss/postcss@4
- motion@11.x (Framer Motion successor for DOM text)
- Fonts: PP Neue Montreal (display), JetBrains Mono (labels)

IMPLEMENTATION:
1. Root layout: `<Lenis>` provider (`duration: 1.2`, `easing: (t) => Math.min(1, 1.001 - Math.pow(2, -10*t))`), `<Canvas gl={{ antialias: false, powerPreference: 'high-performance' }} dpr={[1, 1.75]}>`. Fix canvas `position: fixed; inset: 0; z-index: -1;`.
2. Scene tree: `<PerspectiveCamera makeDefault fov={35} position={[0, 0, 6]} />`, `<Environment preset="studio" />`, `<Suspense fallback={null}>` wrapping a `<HeroMesh />` (GLB exported from Blender at 30k tris max, Draco compressed).
3. HeroMesh material: MeshPhysicalMaterial with `metalness: 1.0, roughness: 0.12, iridescence: 1, iridescenceIOR: 1.3`. Override the frag shader via `onBeforeCompile` — inject a `uCursor` vec2 uniform and a `uNoise` sampler (16-bit blue noise 256×256) to modulate the normal by `0.08 * noise(uv + uCursor * 0.5)`.
4. Scroll timeline: single `gsap.timeline({ scrollTrigger: { trigger: '#scroll-track', start: 'top top', end: 'bottom bottom', scrub: 0.6 } })`. Tween camera position across 6 keyframes, `mesh.rotation.y` from 0 to Math.PI * 2, and `bloom.intensity` from 0.2 → 1.4 → 0.2.
5. Text overlay: each section is `position: sticky; top: 0; height: 100vh;`. SplitText each headline into chars, animate `y: 60 → 0, opacity: 0 → 1, filter: blur(12px) → blur(0px)` with `stagger: 0.02, ease: 'expo.out', duration: 1.1`.
6. Cursor tracking: `useFrame((state) => { mesh.material.userData.uCursor.value.lerp(state.pointer, 0.08) })`.
7. Perf: `frameloop="demand"` + `invalidate()` on pointer/scroll; disable post-processing on `prefers-reduced-motion`; ship a static poster PNG as `<noscript>` fallback.
8. Breakpoints: below 768px, replace the canvas with a compressed MP4 (H.265, 1.2MB) autoplaying muted; kill ScrollTrigger, keep DOM text animations only.

WHY IT WORKS: Locked-frame parallax between DOM and canvas creates the illusion of a physical object being rotated by the reader — a strong figure-ground contract from Gestalt's common-fate principle. The chromatic hover reads as "live material," which primes trust in craft. Whitespace rhythm (12-col grid, 96px vertical gutters) keeps the eye anchored despite the maximalist center canvas.

GOOD / BAD:
- GOOD: The scroll-scene lock is bulletproof — no drift, no jank at 120Hz. Best-in-class DOM/WebGL choreography of the year.
- BAD: 6.4MB initial payload, LCP ~2.9s on 4G. Mobile fallback is a video, which is a cop-out for a "developer" award winner.

CLEAN APPLICATION: Apply this to Clean's landing page hero by replacing the current static screenshot with a `<Canvas>` that renders a slowly rotating, low-poly 3D representation of the Huginn agent graph — nodes as glass spheres, edges as animated dashed shaders. As the user scrolls the hero, the graph rearranges from a chaotic cluster into a clean pipeline (lead → enriched → scored → in CRM), synced via GSAP ScrollTrigger + Lenis. This turns "we clean and enrich your pipeline" from a claim into a physical demonstration in the first 5 seconds.

## CoffeeTech® — https://www.awwwards.com/sites/coffeetech-r

WHAT IT DOES: An industrial B2B site for commercial coffee roasters that uses a full-viewport Three.js scene of a photoreal machine as its navigation surface — clicking on a valve or drum triggers a camera fly-through to that sub-system's spec panel. Scroll-driven parallax cross-fades between four machine variants using a single shared skinned mesh with morph targets, avoiding model swaps. Data tables (temp curves, throughput charts) are rendered as MDX and animated in with a horizontal reveal mask.

PACKAGES:
- next@15, react@19
- three@0.170.x, @react-three/fiber, @react-three/drei (`useGLTF`, `MeshTransmissionMaterial`, `Html`, `PerformanceMonitor`)
- @react-three/rapier@1.5.x (constraints for door/hatch interactions)
- gsap@3.13.x + ScrollTrigger + Flip
- lenis@1.1.x
- shadcn/ui (Base UI base) + Tailwind v4
- recharts@2.15.x (curves), or visx if custom
- Framer Motion (motion@11)
- MDX via @next/mdx@15
- Draco + KTX2 texture pipeline

IMPLEMENTATION:
1. Preload the roaster GLB at route enter via `useGLTF.preload('/models/roaster.glb')`. Compress: Draco geometry, KTX2 BasisU textures (2K PBR maps → ~1.8MB total).
2. Camera rig: `<CameraControls makeDefault />` from drei. Named waypoints as `{ [region: string]: { position: Vector3, target: Vector3 } }`. On hotspot click: `controls.setLookAt(...waypoint.position.toArray(), ...waypoint.target.toArray(), true)`, duration 1.4s, ease `easeInOutCubic`.
3. Hotspots: `<Html center distanceFactor={8} occlude>` badges with a 12px translucent circle + label. Pulse via CSS `@keyframes` (scale 1 → 1.35, opacity 1 → 0, 1.8s infinite).
4. Morph-target variant switching: single mesh with 4 morphs (drum, hopper, cooler variants). Cross-fade with `gsap.to(mesh.morphTargetInfluences, { [i]: 1, duration: 0.9, ease: 'power2.inOut' })` — never unmount/mount geometry.
5. Data section: sticky left column (spec text), right column horizontal reveal mask (`clipPath: inset(0 100% 0 0)` → `inset(0 0 0 0)`, duration 1.1s, staggered per row).
6. Charts: Recharts LineChart with `animationDuration: 1400`, `animationEasing: 'ease-out'`. Axis labels IBM Plex Mono 11px, grid `stroke='rgba(255,255,255,0.06)'`.
7. Color system: `--bg: #0A0908; --surface: #141210; --accent: #E27D3A` (roast orange), `--text: #F5EFE6`. Set `color-scheme: dark` on `<html>`.
8. Performance: `<PerformanceMonitor onDecline={() => setDpr(1)} onIncline={() => setDpr(1.5)} />`. Freeze the canvas (`frameloop="never"` + explicit `invalidate()`) when the user is reading a spec panel.

WHY IT WORKS: The machine IS the nav — that collapses the "learn about product" and "browse product" flows into a single spatial gesture, which is a Fitts's-law win for a specialist B2B audience. The single-mesh morph strategy hides load between variants, so the site feels like one continuous object rather than a marketing catalog. Contrast ratio 12.8:1 on body copy against #0A0908 gives it a technical, precision-instrument feel.

GOOD / BAD:
- GOOD: The hotspot-to-camera-waypoint pattern is a masterclass in giving a technical audience agency without dumping specs on them upfront.
- BAD: The occluded HTML labels flicker on Safari 17 when the camera passes behind geometry — a known drei quirk. No keyboard nav for hotspots yet.

CLEAN APPLICATION: Apply this to Clean's product demo section (the /product page) by rendering an interactive 3D dashboard scene where the user clicks on pipeline stages (Discover, Enrich, Score, Sync) and the camera flies to reveal a real UI panel for that stage. Use morph targets to cross-fade between "before Clean" (messy CSV) and "after Clean" (clean CRM view) so the transformation is felt, not read.

## Longbow — https://www.awwwards.com/sites/longbow

WHAT IT DOES: A luxury automotive marketing site for a British EV sports car with a hero video that is actually a `<canvas>` playback of an image sequence (KTX2 frames, 60fps) so scroll can scrub through it. Typography is oversized (14vw display) with kinetic layout shifts on scroll — the headline breaks apart into two lines, the second line slides horizontally across three sections while sticky, then reassembles. Palette is limited to two colors: bone white and racing green, with heavy negative space.

PACKAGES:
- next@15, react@19
- gsap@3.13.x + ScrollTrigger + Observer
- lenis@1.1.x
- motion@11 for micro-interactions
- @studio-freight/hamo@0.6.x (raf/resize hooks)
- tailwindcss@4
- Fonts: NB Akademie Std (display), Söhne (body), or free alt: PP Editorial New + Inter
- KTX2 image sequence decoder (basisu wasm) OR `three@0.170` `VideoTexture` if using MP4

IMPLEMENTATION:
1. Hero: preload 120 frames as KTX2 (~4KB each = ~480KB), decode into a single `OffscreenCanvas`. `ScrollTrigger.create({ trigger: '#hero', start: 'top top', end: '+=200%', pin: true, scrub: 0.4, onUpdate: (self) => drawFrame(Math.floor(self.progress * 119)) })`.
2. Kinetic type: SplitText into words, then `gsap.timeline().to(secondLine, { x: '-40vw', duration: 3, ease: 'none' }, 0)` scoped to a pinned section. Font weight animates from 300 to 700 via `variable font wght` axis for a stress build.
3. Grid: 12-column, 8px baseline. Body copy tops at row 6 (never higher) — deliberate "sit-and-breathe" negative space. Margin blocks: 240px section padding on desktop, 96px mobile.
4. Palette: `--bone: #F4F0E6; --racing: #0B3D2E; --ink: #12110E`. Only three colors on the entire site. No shadows anywhere; hierarchy comes from size and position.
5. Micro-interactions: buttons expand horizontally on hover (`width` from `auto` to `+32px`) with GSAP `power2.out` 0.35s, arrow icon translates 8px right. No scale, no lift.
6. Section transitions: horizontal wipe using a `<div>` overlay `transform: translateX(100%)` → `0` → `-100%`, duration 0.9s, `expo.inOut`. Content behind swaps at 50% progress.
7. Sound: optional muted hover-cue at 32ms attack via `Tone.Synth` — a single 880Hz sine blip on primary buttons. Ship muted by default with a toggle in the nav.
8. Perf: preload sequence with `<link rel="preload" as="image">` only for frames 0-30; lazy-load the rest during idle time via `requestIdleCallback`.

WHY IT WORKS: The two-color palette + 14vw type is confidence — it says "you don't need visual noise to hold your attention." That's the same psychological signal as a Savile Row suit. Scroll-scrubbed image sequences give a physical, filmic response that mp4 scrubbing cannot (no keyframe hunting), which reads as "made-by-hand" to a discerning viewer.

GOOD / BAD:
- GOOD: The horizontal-word scroll pin, done with restraint, sets a new bar for "kinetic type without gimmick" — it earns the interaction because the copy is worth reading.
- BAD: Image sequence is 480KB best case, ~2MB typical; Lighthouse Perf drops to 74 on 4G. No fallback for reduced-motion beyond a static frame — should be a poster + text-only story.

CLEAN APPLICATION: Apply this to Clean's pricing page by killing the three-column card comparison and rebuilding as a single vertical scroll story: 14vw headline "Every plan cleans your pipeline. Only the volume changes." pinned while a two-column layout (Free / Pro / Growth) horizontally scrubs into view. Use bone + racing green + charcoal on white. Zero shadows. Force the eye to land on outcomes, not price tags.

## PP Neue Montreal — https://www.awwwards.com/sites/pp-neue-montreal

WHAT IT DOES: A Framer-built interactive type specimen for Pangram Pangram's flagship grotesque, structured as a travel guide through Montreal — each city district becomes a chapter demonstrating a weight, an OpenType feature, or a language script. Draggable weight sliders morph a variable-font headline in real time. A "ligature museum" section is a horizontal-scroll gallery of glyph pairs with hover magnification. All interactions run on Framer's native motion primitives — no Three.js.

PACKAGES:
- Framer (hosted) — or if rebuilding in code:
  - next@15, react@19
  - motion@11 (framer-motion successor)
  - @use-gesture/react@10.3.x
  - tailwindcss@4
  - Fonts: PP Neue Montreal Variable (via Pangram Pangram license, or fontshare.com fallback)
  - use-sound@4 for optional micro-audio

IMPLEMENTATION:
1. Layout: single-page vertical, 8 chapters. Each chapter is `100vh min-height` sticky-scroll. Use `motion.div` with `useScroll` + `useTransform` to fade chapter labels in a 20% viewport window.
2. Variable font sliders: `<input type="range" min="100" max="900">` bound to `document.documentElement.style.setProperty('--wght', v)` and the H1 uses `font-variation-settings: 'wght' var(--wght)`. Drag with `@use-gesture` for a custom slider handle (60px circle, magnetic snap at 100-step increments).
3. Ligature gallery: horizontal scroll with `overflow-x: scroll; scroll-snap-type: x mandatory;`. Each card is `min-width: 40vw; scroll-snap-align: center`. On hover, `transform: scale(1.06); z-index: 10;` with `spring: { stiffness: 260, damping: 22 }`.
4. Language demos: tabs for Cyrillic / Vietnamese / Arabic — each with a sample paragraph that live-swaps on tab click. Use `<AnimatePresence mode="wait">` with a 200ms crossfade.
5. Color: `--paper: #F2EEE7; --ink: #111111; --highlight: #FF4B27` (a single accent used only for interactive states). Zero images below the hero — all visuals are type.
6. Micro-typography: `hyphens: auto; text-wrap: pretty; font-feature-settings: 'ss01', 'ss02', 'liga', 'dlig'`. Headline `letter-spacing: -0.04em`, body `-0.01em`.
7. Motion signature: every entrance uses the same spring `{ type: 'spring', stiffness: 200, damping: 30, mass: 0.9 }` — a consistent physical feel across the site.
8. Buy CTA: floating pill in bottom-right, `position: fixed; bottom: 24px; right: 24px;` that grows +8px on hover, background swaps paper ↔ ink.

WHY IT WORKS: When the product is a typeface, showing it working (variable slider, live language demos) beats screenshotting it. The travel-guide framing gives editorial permission to use very long-form typography, which is itself the product demo. One accent color creates a Von Restorff spike wherever interactivity lives — the user learns "orange = clickable" in seconds.

GOOD / BAD:
- GOOD: The variable-font slider bound to a real `font-variation-settings` axis is the only honest way to demo a type family online — most specimens fake it with static images.
- BAD: Framer's runtime injects ~180KB of JS for what is essentially a static-first typography site. Would be 40KB hand-coded.

CLEAN APPLICATION: Apply this to Clean's docs/knowledge-base entry page by rebuilding it as an editorial "field guide" — chapters like "How to find your ICP", "Enrichment 101", "Signal scoring" — with oversized display type, one accent color for interactive elements (Clean's teal), and inline live-widgets (a small search box, a live signal-count) that let visitors try Clean concepts as they read. Feels like a book you want to finish.

## shadcn/ui — Base UI Default + Typeset Style — https://ui.shadcn.com/docs/changelog/2026-07-base-ui-default

WHAT IT DOES: Not a single site — the July 2026 shadcn/ui release swapped Radix for Base UI as the default component primitives, added React Aria as a first-class alternative, and shipped two new styles: "Rhea" (compact) and "Typeset" (editorial, minimalist, uppercase headings, underline controls, print-inspired). Any product built on shadcn this month reads as sharper, denser, and typographically-led. Vercel, Resend, and a wave of AI-infra startups have already refactored their marketing chrome onto Typeset.

PACKAGES:
- shadcn@latest CLI (2026.07+)
- @base-ui-components/react@1.0.x (default primitives)
- react-aria-components@1.4.x (optional base)
- tailwindcss@4, @tailwindcss/vite@4 or postcss
- lucide-react@0.475.x
- Fonts: Typeset uses a serif display (recommend Söhne Breit or free alt: PP Editorial New / Fraunces) + a mono (JetBrains Mono / Geist Mono)
- next@15, react@19

IMPLEMENTATION:
1. Init: `npx shadcn@latest init --style=typeset --base=base-ui`. Confirms Base UI as primitive base and Typeset as visual style.
2. Typography scale (Typeset):
   - Display: `text-[clamp(3.5rem,7vw,6.5rem)] leading-[0.95] tracking-[-0.03em] font-serif`
   - H2: `text-4xl leading-[1.05] tracking-[-0.02em]`
   - Eyebrow: `text-xs uppercase tracking-[0.14em] font-mono text-muted-foreground`
   - Body: `text-[15px] leading-[1.65] font-sans`
3. Controls: buttons use `border-b-[1.5px] border-current bg-transparent hover:bg-foreground hover:text-background transition-colors duration-150` — no fill, no shadow, only underline. Focus ring is `outline: 1px solid var(--foreground); outline-offset: 4px`.
4. Inputs: `border-0 border-b border-border rounded-none px-0 h-11 focus:border-foreground` — pure editorial form fields.
5. Cards: no border-radius, `border-t border-border` only, `pt-8 pb-12`. Content-first — the border is a rule, not a container.
6. Color: `--background: 0 0% 100%; --foreground: 0 0% 8%; --muted: 0 0% 96%; --accent: 12 90% 55%` (single warm accent used sparingly). Dark mode inverts cleanly.
7. Motion: transitions on `color, background-color, border-color` at `duration-150 ease-out`. No transforms in the base style — reserve motion for content, not chrome.
8. Layout: 12-col grid, `container mx-auto max-w-[1280px] px-6`. Row gutters 96px desktop / 48px mobile. Rule dividers between sections instead of background fills.
9. Rhea overlay for product surfaces: same tokens, but `--spacing-unit: 0.9` scale for dense dashboards.
10. Accessibility win: Base UI ships better keyboard/screen-reader semantics than Radix in v1 — Command menu, Combobox, Select, and Dialog all pass axe-core clean out of the box.

WHY IT WORKS: The editorial + rule-based hierarchy is a direct reaction to the "glass cards on gradient" fatigue that dominated 2024–2025 SaaS design. Typeset trades ornament for typographic authority — it signals "we care about words," which is a Zeigarnik-effect trust win for developer tools that require reading docs. Base UI's a11y wins are invisible-but-material — the whole thing feels quieter, which is the point.

GOOD / BAD:
- GOOD: One migration flips a whole product from "startup template" to "publication." Zero visual novelty, maximum craft.
- BAD: Typeset requires real editorial copy — thin sales-y one-liners look worse in it than in the default style. Style-first teams will bounce off it hard.

CLEAN APPLICATION: Migrate Clean's entire marketing site (tryclean.ai) to shadcn Typeset style + Base UI primitives this week. Rebuild the hero as a full-width display headline ("Clean is the operating system for your pipeline."), eyebrow in mono uppercase, underlined-button-only CTA, single teal accent for links. Kill every gradient, every rounded card, every drop shadow. Replace with rules, editorial spacing, and mono-labeled data. The result: Clean stops looking like every other Y Combinator company and starts looking like Bloomberg Terminal for GTM.

## BUILD DIRECTIVE

TARGET: Clean's landing page (tryclean.ai/) — full rebuild, hero through footer.

APPROACH: Migrate the entire marketing site to shadcn/ui Typeset style on Base UI primitives (from PP Neue Montreal and shadcn/ui July release), then layer a single hero-scene WebGL element (from Hiroto Sato) that renders a live 3D pipeline graph of Huginn agents — nodes as glass spheres, edges as animated shaders, scroll-scrubbed to rearrange from "raw leads" chaos into a clean CRM pipeline. Use Longbow's two-color-plus-one-accent palette discipline (bone white + graphite + Clean teal, zero gradients) and its scroll-pinned kinetic headline treatment for the "Every signal, cleaned" section. Below the hero, adopt CoffeeTech's hotspot-driven product tour — click a stage (Discover / Enrich / Score / Sync) and the camera flies to reveal that product surface as a real UI panel. Type-first, dense, editorial. Feels like a Bloomberg Terminal that happens to know how to render 3D.

STACK:
- next@15 (App Router), react@19, react-dom@19
- shadcn@latest with `--style=typeset --base=base-ui`
- @base-ui-components/react@1.0.x, react-aria-components@1.4.x (fallback)
- tailwindcss@4, @tailwindcss/postcss@4
- three@0.170.x, @react-three/fiber@8.17.x, @react-three/drei@9.114.x, @react-three/postprocessing@2.16.x
- gsap@3.13.x (ScrollTrigger, SplitText, Flip), @studio-freight/lenis@1.1.x
- motion@11 for DOM micro-interactions
- lucide-react@0.475.x
- Fonts: PP Editorial New (display serif), Inter Variable (body), JetBrains Mono (labels/eyebrows)
- Perf: Draco + KTX2 pipeline, `<PerformanceMonitor>` DPR scaling, `frameloop="demand"`

REFERENCE URLS:
1. https://www.awwwards.com/sites/hiroto-sato — for the DOM/WebGL scroll-lock hero
2. https://ui.shadcn.com/docs/changelog/2026-07-base-ui-default — for the Typeset visual system and Base UI primitives
3. https://www.awwwards.com/sites/longbow — for the two-color discipline and kinetic typography restraint

KEY CONSTRAINTS: Clean is developer infrastructure — the result must feel like a precision instrument, not a marketing site. Specifically avoid: (1) gradient backgrounds of any kind — flat surfaces only, hierarchy via rules and space; (2) rounded cards with drop shadows — replace with `border-t` rules and generous vertical rhythm; (3) marketing-y micro-copy (no "AI-powered", no "supercharge", no "10x your"); (4) animated icons in Lucide — icons stay static, only content moves; (5) SaaS-standard three-column pricing — kill it, use the Longbow vertical scroll story; (6) hero video autoplay of a screencast — the WebGL scene IS the demo; (7) any use of Framer for the production site — it inflates JS; (8) more than one accent color (Clean teal only); (9) shipping the WebGL hero without a static poster fallback for `prefers-reduced-motion` and mobile below 768px; (10) any element that doesn't earn its pixels — every section must survive the "would Bloomberg Terminal ship this?" test.
