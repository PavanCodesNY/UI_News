## digitalists — https://www.awwwards.com/sites/digitalists-1

WHAT IT DOES: Today's Awwwards SOTD (June 22, 2026) + Developer Award. Austrian agency site built around a custom WebGL canvas hero with a noise/distortion shader laid over typographic mark; the page uses a sticky scroll-driver where DOM sections are pinned while the canvas runs a single continuous scene that morphs between projects. Layout is a 12-col CSS grid with absolute-positioned floating labels that animate position on scroll (lerp toward target offsets). Transitions between routes are barba-style cross-fades with a shader wipe instead of opacity.

PACKAGES:
- next@15 / react@19
- gsap@3 + @gsap/react (useGSAP) + ScrollTrigger + ScrollSmoother
- three@0.170 + ogl@1 (lighter than three for the hero plane)
- lenis@1.1 (smooth scroll)
- @studio-freight/tempus (rAF unification)
- barba.js@2 or next-view-transitions for route morphs
- satori-ish custom display face + Inter via next/font

IMPLEMENTATION:
Component tree: `<Lenis><SmoothScroller><HeroCanvas /><Sections /><Footer /></SmoothScroller></Lenis>`. HeroCanvas mounts a single full-bleed `<canvas position:fixed inset:0 z:-1>` with a fragment shader sampling: (a) FBM noise modulated by `uTime`, (b) a flow field driven by normalized mouse + scrollProgress, (c) a chromatic aberration pass via offset RGB samples. Drive `uScroll` (0-1) from ScrollTrigger so the shader's color ramp and distortion intensity slide between sections without re-mounting. Pin sections with `ScrollTrigger.create({ trigger, pin:true, scrub:0.6 })`. Floating labels use `transform: translate3d(x,y,0)` updated each tick via `gsap.quickTo` (duration:0.6, ease:'expo.out'). Type scale: display 96/112/clamp(56px, 9vw, 144px), letter-spacing -0.04em, line-height 0.92. Color: bg #0A0A0A, fg #F5F5F0, accent #5B6CFF. Reduced-motion: kill ScrollSmoother, swap canvas for a static gradient PNG, disable shader uniforms. Performance: cap DPR to `Math.min(devicePixelRatio, 1.75)`, pause RAF on `IntersectionObserver` leave.

WHY IT WORKS: One canvas + one type system + one accent color = severe visual hierarchy. The scroll-locked shader gives the page a continuous "physical surface" feel (Gestalt continuity), so every section feels like a camera move not a page jump. Motion is semantic — distortion intensifies as scroll velocity rises, which trains the user that the medium responds.

GOOD / BAD: Best-in-class: shader-driven continuity across an entire scroll instead of section-by-section animation. Bad: heavy initial paint (~1.2MB JS), no LCP image, can stutter on mid-tier Android.

CLEAN APPLICATION: Apply this to Clean's landing page hero (tryclean.ai/). Build a fixed full-bleed WebGL canvas behind the headline running a flow-field shader tinted toward Clean's deep-indigo + electric-teal accents, with `uContextDensity` uniform that swells as the user scrolls through sections (Problem → Wedge → Product → Traction). The headline "Unified context intelligence platform for agent teams" sits in display weight at 9vw, with floating one-line labels ("4-5x token reduction", "100 agents/team", "$7K in pilots") that lerp to scroll-anchored positions. No section transitions — one continuous scene.

## Shader.se — https://tympanus.net/codrops/2026/05/19/80s-business-tech-seamless-scene-transitions-inside-shader-ses-scroll-driven-webgpu-pipeline/

WHAT IT DOES: Studio site styled as a 1987 corporate training tape, rendered in React Three Fiber on the WebGPU path with TSL (Three.js Shading Language) doing node-based materials that compile to both WGSL and WebGL fallback. The signature move: scroll-driven scene transitions where geometry, lighting, and post all morph together because the current scene samples the next scene's FBO mid-transition. Pages declare a "render offset" extending their active range so the next FBO is hot before it's visible.

PACKAGES:
- @react-three/fiber@9 (WebGPU canvas)
- @react-three/drei@10
- three@0.171 with `WebGPURenderer` + TSL imports from `three/tsl`
- @react-three/postprocessing@3 (custom EffectComposer for WebGPU)
- gsap@3 ScrollTrigger
- zustand for cross-scene scroll state
- Inter Display + Berkeley Mono

IMPLEMENTATION:
Top-level: `<Canvas gl={async (canvas)=>{const r=new WebGPURenderer({canvas,antialias:true}); await r.init(); return r;}} dpr={[1,1.5]}>`. Build a `<ScenePages>` graph where each page is an `<group>` wrapping its geometry and a screen-space quad shader. State lives in zustand: `{ scrollY, activePage, transitionT }`. ScrollTrigger updates these as raw values; useFrame interpolates. For transitions, each page's material reads `uNextFBO` (the next page's render target) and blends via screen-space UV inside its TSL graph — when transitionT in [0..1] the fragment is `mix(currentColor, sample(uNextFBO, screenUV), curve(t))`. Curve is `smoothstep(0.0, 1.0, t)` for soft, or a polar-mask for "iris" wipes. Render offsets: each page exposes `{ enter: -0.2, exit: 0.2 }` to keep its FBO valid across transition zones. Post stack: bloom 0.4 / threshold 0.85, film grain 0.06, chromatic aberration 0.0015. Type: 80s corporate — IBM Plex Mono for UI labels, custom display face for headlines, all-caps tracking +0.08em.

WHY IT WORKS: Scroll becomes a camera through a coherent universe instead of swapping slides. Sampling the next scene mid-transition means lighting changes feel like the room is lit by what's coming, not by a cut — a profoundly different perceptual model. The 80s aesthetic is opinionated, which makes the engineering feel intentional rather than "demo".

GOOD / BAD: Best-in-class: scene-graph transitions that solve the "page swap feels janky" problem at its root. Bad: WebGPU still only ~78% support globally, fallback to WebGL doubles shader maintenance, and FBO chaining costs ~12MB VRAM per active scene pair.

CLEAN APPLICATION: Apply this to Clean's product demo section (the part between hero and pricing). Build 4 scenes: (1) "100 agents flying blind" — particles scattered, no connections; (2) "Clean ingests context" — a central proxy node draws lines; (3) "Unified state" — all agents pulse in sync; (4) "Foundational model" — particles condense into a single glowing brain. Scroll-drive transitions where scene 1's FBO is sampled by scene 2's blur kernel, so the chaos visually melts into order. WebGL-only fallback is fine; we don't need WebGPU on day one.

## Maxi Ruti Fluid Portfolio — https://www.maxiruti.com/

WHAT IT DOES: Portfolio where thumbnails behave as soft-body particles in a 3D scene rendered with React Three Fiber, while a shadow DOM layout sits underneath holding accessible links and metadata. Toggling layout (carousel ↔ alphabetized grid) triggers a physics-informed transition: every thumbnail interpolates its target position with a damped spring while a post-process distortion swells then settles. Scroll causes the cluster to "slosh" — a small but legible feedback that the medium is alive.

PACKAGES:
- @react-three/fiber@9
- @react-three/drei@10 (Html, useTexture, PerspectiveCamera)
- three@0.171
- @react-three/postprocessing (DepthOfField, ChromaticAberration)
- @use-gesture/react@10 (drag + wheel)
- framer-motion@12 (DOM shadow layout transitions)
- leva for tuning (dev only)
- maath@0.10 (lerp, easings)

IMPLEMENTATION:
Build two parallel structures: a) 3D `<Instances>` with N planes each textured with project thumbnails, b) a Framer Motion `<motion.ul>` of `<motion.li>` shadowing them with layoutId. Each frame, useFrame computes target positions (`carouselTargets` or `gridTargets`) from a layout descriptor, then `pos.lerp(target, 1-Math.exp(-k*dt))` per plane. k=6 for snappy, 3 for fluid. On scroll velocity > threshold, add a sinusoidal offset on Y proportional to velocity (this is the "slosh"). Post: a CustomEffect that displaces UVs by `noise(uv*8+uTime*0.1) * uDistortion`, where uDistortion is driven by transitionT and scroll velocity. Scroll wired via Lenis. DOM shadow exists for SEO + a11y (real `<a href>` tags). The 3D layer has `pointer-events:none` everywhere except the focused plane.

WHY IT WORKS: Two-system architecture (3D for feel, DOM for semantics) gets you ambition without sacrificing fundamentals — screen readers and SEO see a clean list. Spring physics with velocity feedback nails motion semantics: the page "knows" you're scrolling fast. Distortion-on-transition is the optical cue that makes layout switches feel like a state change in physical material.

GOOD / BAD: Best-in-class: shadow-DOM pattern for accessible 3D layouts. Bad: the post-process distortion is gorgeous on M-series Macs and gross on a 2019 ThinkPad — you must gate it on `navigator.gpu` + `hardwareConcurrency`.

CLEAN APPLICATION: Apply this to Clean's dashboard/analytics preview section. Render the user's connected agents as soft-body nodes in a 3D field, with the DOM shadow holding accessible labels (agent name, last context fetch, token savings). Add a toggle: "By agent" (cluster by agent ID) ↔ "By context type" (cluster by source — code, calendar, docs). Layout transition uses the same spring + distortion swell. The thesis "Clean unifies context" lands viscerally when the user sees disparate agents physically merging.

## Fauna Robotics — https://www.awwwards.com/sites/fauna-robotics

WHAT IT DOES: SOTD June 16, 2026 + Developer Award. Marketing site for a humanoid robotics startup (recently acquired by Amazon). Two-color palette (#EFEFEF bg, #D1E3FF accent), all warm. Built in Webflow with custom JS for the hero animation — a character-driven introduction where their robot "Sprout" is staged in oversized close-ups with playful asymmetric layouts. Heavy on horizontal scroll for product walkthroughs and lottie/rive for micro-character moments.

PACKAGES:
- Webflow runtime
- gsap@3 + ScrollTrigger + Observer
- @rive-app/canvas@2.21 (the character animations)
- lottie-web@5.12 (lower-cost loops)
- lenis@1.1
- Suisse Int'l + Suisse Mono

IMPLEMENTATION:
Hero: a single Rive scene of Sprout with state machines wired to `mouseX/mouseY` for eye-track and to `scrollProgress` for body pose. Section layout uses a 6-col asymmetric grid with deliberate negative space — left column 60% width, right column 30%, gutter pushed off-center. Horizontal product walkthrough is a horizontal `scrub:true` ScrollTrigger pinning the section while translating an inner track: `gsap.to(track, { x: -track.scrollWidth + window.innerWidth, scrollTrigger:{ pin:true, scrub:1, end:()=>'+='+track.scrollWidth } })`. Color palette is two-tone with NO black — body text is #2A2A2A on #EFEFEF for warmth. Type: display 120px / -0.03em / line-height 0.94, body 18px / 1.55. Rive files are streamed: load the hero Rive first, defer subsequent ones until `IntersectionObserver` fires.

WHY IT WORKS: Character-driven marketing softens the "robots are scary" instinct (humanization principle). Two-color palette enforces severe visual rest, which lets the character carry attention. The Rive state machine reacting to cursor is a feedback loop that converts visitors into participants — they linger because something is watching.

GOOD / BAD: Best-in-class: Rive state machines instead of canned Lottie loops — the character feels alive. Bad: zero focus on accessibility (low contrast accents, eye-track has no `prefers-reduced-motion` opt-out in the published version).

CLEAN APPLICATION: Apply this to Clean's "what an agent sees with vs. without Clean" comparison section. Two side-by-side Rive characters: left, an agent labeled "without Clean" stumbling around in fog; right, an agent labeled "with Clean" pulling glowing context strands from a central proxy. Both react to cursor (eye-track + slight body lean), which makes the visitor the demo. Use Clean's two-tone palette (deep-indigo #0E0E2C bg, electric-teal #6EF0D1 accent) — same severe restraint.

## Aceternity UI Split-Flap + Magnetic Button + Unicorn Studio Hero — https://ui.aceternity.com/ + https://www.unicorn.studio/

WHAT IT DOES: Two component patterns shipped in Aceternity UI's recent drop plus the Unicorn Studio runtime (29kb gzipped, layer-based WebGL editor) for hero shaders. (1) Split-flap display component animates characters with Vestaboard-style flip transitions — perfect for live metrics. (2) Magnetic button that drifts toward the cursor on hover with a spring-based return. (3) Unicorn Studio runtime lets you embed any GPU shader scene as a single `<div data-us-project="...">` with auto-pause on scroll-out.

PACKAGES:
- next@15 / react@19
- framer-motion@12 (springs for magnetic button)
- @aceternity/ui (copy-paste, no npm install — components are just .tsx)
- tailwindcss@4
- class-variance-authority + clsx + tailwind-merge
- @unicornstudio/runtime@1.4 (or `<script src="https://cdn.unicorn.studio/v1.4.x/unicornStudio.umd.js">`)
- lucide-react@0.460

IMPLEMENTATION:
Split-flap: array of `<motion.span>` per character, each driven by `useAnimation()` flipping rotateX from 0 → -90 → -180 with a stagger. Stagger: `transition: { delay: i*0.04, duration: 0.45, ease: [0.7, 0, 0.3, 1] }`. Use a monospace face (Berkeley Mono or Geist Mono) so each cell is identical width. Magnetic button: track cursor inside a `useMotionValue(x,y)`, compute distance from center, apply `useSpring(x*0.35, { stiffness: 150, damping: 12 })` to a translate. On leave, snap back via the spring naturally. Unicorn Studio hero: design the scene in unicorn.studio editor (layer canvas, drop in noise + fluid + gradient layers), export project ID, drop `<div data-us-project="<id>" data-us-scale="1" data-us-dpi="1.5" />` then call `UnicornStudio.init()` post-mount. The runtime auto-pauses on `IntersectionObserver` exit. For accessibility: wrap with `prefers-reduced-motion: reduce` media query that swaps the live shader for a static export PNG.

WHY IT WORKS: Split-flap is the highest-signal way to display a "number that changed" — humans evolved to track flipping things, so a metric counter that flips reads as alive in a way that a CSS transition never will. Magnetic button shifts a button from "object" to "agent" — it sees the cursor, so the user feels seen. Unicorn Studio's layer model means a designer (not an engineer) can iterate on the hero shader in 10 minutes, which collapses the design-to-ship loop that usually kills WebGL hero ideas.

GOOD / BAD: Best-in-class: ship velocity. A team can wire all three in a single afternoon and the result looks like a 3-month custom build. Bad: every Aceternity site looks subtly like every other Aceternity site — you must commit to heavily customized typography and palette to escape the template feel.

CLEAN APPLICATION: Apply this directly to Clean's hero + traction strip on tryclean.ai/. Unicorn Studio scene as the hero background (subtle flowfield in deep-indigo + teal). Magnetic "Get a demo" CTA. Split-flap counters in the sub-hero: "4.7x" (token reduction), "80+" (startups in pilot), "$7K" (pilot revenue). Counters flip every 8 seconds between values pulled from a static JSON for now, wireable to live API later. Berkeley Mono for the counter face, Inter Display for the headline.

## BUILD DIRECTIVE

TARGET: Clean landing page hero + immediate sub-hero (tryclean.ai/). One surface, one build, ship today.

APPROACH: Build a fixed full-bleed WebGL hero using the Unicorn Studio runtime (29kb, drop-in) with a custom flow-field shader scene tuned to Clean's palette (#0E0E2C bg, #6EF0D1 accent, #F5F5F0 fg, secondary #5B6CFF). On top: a single display-weight headline "Unified context intelligence platform for agent teams" at clamp(56px, 9vw, 144px), letter-spacing -0.04em. Sub-hero immediately below with three split-flap counters from Aceternity (4.7x token reduction, 80+ startups, $7K pilots) in Berkeley Mono, flipping every 8 seconds. Single magnetic "Get a demo" CTA, framer-motion spring (stiffness:150, damping:12). Scroll-link the shader's flow intensity to scroll progress via ScrollTrigger so the hero "tightens" as the user scrolls (motion semantic: chaos resolving into clarity, matching Clean's thesis). Reduced-motion: swap the canvas for a static PNG export of the shader. No section transitions — one continuous hero surface.

STACK:
- next@15 / react@19 / typescript@5.5
- tailwindcss@4
- framer-motion@12
- gsap@3 + ScrollTrigger
- lenis@1.1
- @unicornstudio/runtime@1.4 (CDN script)
- next/font for Inter Display + Berkeley Mono (self-hosted)
- clsx + tailwind-merge
- lucide-react@0.460 (icons)

REFERENCE URLS:
1. https://www.awwwards.com/sites/digitalists-1 — for shader-driven scroll continuity and severe typographic hierarchy
2. https://ui.aceternity.com/ — for split-flap counters and magnetic CTA implementation

KEY CONSTRAINTS: Clean is developer infrastructure for engineers building agent teams. The page must feel technically sophisticated and fast, not flashy or generic. AVOID: emoji decoration, 3D character mascots, lottie loops, gradient buttons with shine animations, generic "Start free" CTAs, glassmorphism cards, animated checkmarks in feature lists, "trusted by" logo strips with marquee scroll, or any pattern that reads as "AI startup template". The hero must load under 1.5s on a mid-tier Android (Pixel 6a baseline), DPR capped at 1.5, total JS budget 180kb gzipped excluding the shader runtime. No splash screen. No cursor follower with trail. The shader scene must read as instrument-grade — a tool engineers want to look at — not as decoration.
