## 1. Ten Years Away — https://www.awwwards.com/sites/ten-years-away

WHAT IT DOES: A scroll-driven WebGL narrative about climate change. The page is a single full-bleed `<canvas>` running a Three.js scene; GSAP ScrollTrigger drives a normalized scroll progress value (0–1) that timeline-controls camera position, fog density, post-processing exposure, and material uniforms across thirteen "chapters." HTML text blocks sit absolutely-positioned over the canvas with `pointer-events: none` on the canvas layer and `mix-blend-mode: difference` on captions so type stays readable against the shifting palette (#0d1429 → #cce8eb). Chapter transitions are handled by a single GSAP timeline rather than per-section ScrollTriggers — yields one source of truth for scrub.

PACKAGES:
- `three@0.166` — core renderer
- `@react-three/fiber@8.16` — React reconciler for Three
- `@react-three/drei@9.108` — Environment, useGLTF, ScrollControls helpers
- `@react-three/postprocessing@2.16` — Bloom, ChromaticAberration, Vignette
- `gsap@3.12` + `gsap/ScrollTrigger` — master scroll timeline
- `lenis@1.1` — smooth scroll, hands rAF to GSAP via `ScrollTrigger.update`
- `next@14` (app router) + `react@18`
- `tailwindcss@3.4` for HTML overlay layout
- `vaul` or none for sheet UI (n/a here)
- `@vercel/og` for share images
- Font: ABC Diatype (or `next/font/local` with Söhne Mono as a free near-equivalent)

IMPLEMENTATION:
1. App shell: `<Canvas dpr={[1, 1.75]} gl={{ antialias: false, powerPreference: 'high-performance' }} flat>` — flat tonemapping avoids the milky look Three's default gives. Set `frameloop="demand"` and call `invalidate()` from a Lenis `on('scroll')` listener so the scene only renders when scroll position changes (~40% GPU savings on idle).
2. Scene tree: single `<group ref={sceneRef}>` with thirteen `<group>` children, each pre-positioned along the Z axis (z = -i * 14). Animate the parent `position.z` from 0 → -168 across the timeline. Per-chapter materials use `THREE.ShaderMaterial` with `uniforms = { uProgress: { value: 0 }, uColorA, uColorB }` — wire `uProgress` to the GSAP timeline label position via `onUpdate`.
3. Scroll plumbing: `const lenis = new Lenis({ lerp: 0.08, smoothWheel: true })`. In a `useEffect`, `gsap.ticker.add((time) => lenis.raf(time * 1000))` and `gsap.ticker.lagSmoothing(0)`. Build one master timeline: `const tl = gsap.timeline({ scrollTrigger: { trigger: '#scroll-spacer', start: 'top top', end: 'bottom bottom', scrub: 0.6 } })`. Append chapter tweens with `.addLabel('ch1', 0).to(camera.position, { y: 4, duration: 1 }, 'ch1')`.
4. HTML overlay: a sibling `<div className="fixed inset-0 pointer-events-none">` with thirteen full-viewport `<section>` blocks, each `data-chapter="N"`. Use IntersectionObserver to toggle `data-active` and a CSS transition `opacity 600ms cubic-bezier(0.22, 1, 0.36, 1)` for text fade.
5. Postprocessing: `<EffectComposer multisampling={0} disableNormalPass>` with `<Bloom intensity={0.7} luminanceThreshold={0.85} mipmapBlur />` and `<Vignette eskil={false} offset={0.2} darkness={0.85} />`. Animate `bloom.intensity` per chapter via a ref.
6. Mobile: gate WebGL with `useReducedMotion()` and screen width ≥ 768; on mobile fall back to a stacked image-sequence (45 WebP frames, 1280px wide, loaded with `<link rel="preload" as="image">`).
7. Loader: a top-of-page text counter "00 → 100" that polls `THREE.DefaultLoadingManager.onProgress`. Hide canvas behind it until ≥ 95%.
8. Perf budget: total GLTF ≤ 4.5 MB Draco-compressed, single HDR environment (RGBE, 1k), texture max 1024². LCP target ≤ 2.4s on 4G.

WHY IT WORKS: A single continuous scroll metaphor + a monochrome 2-color palette enforces narrative cohesion — every transition feels like the same world deepening, not thirteen different pages. The `mix-blend-mode: difference` caption choice means typography contrast is mathematically guaranteed regardless of background, removing the usual "white-on-white" failure mode of scroll-jacked sites. Motion is *semantic* (forward = deeper into the world), not decorative.

GOOD: Best-in-class scroll-narrative pacing — chapter beats land on screen-height multiples, so there's never a "where am I" moment. Lenis-driven scrub at 0.6s feels weighty, not laggy.
BAD: No keyboard nav, no skip-to-end. Total payload is ~7 MB before assets compress — a hard "no" on metered mobile without the fallback path.

CLEAN APPLICATION: Apply this to Clean's landing-page hero + "How it works" section. Replace the climate chapters with five chapters of the MCP-proxy data flow: (1) raw model request floods in, (2) Clean intercepts at the proxy boundary, (3) context is compressed via TOON, (4) response streams back through, (5) telemetry emits. Use a single Three.js scene of a stylized data pipe — animated particles flowing through a translucent cylinder, with the camera tracking from "outside the agent" (chapter 1) to "inside the proxy" (chapter 3) to "downstream of the response" (chapter 5). Build the chapter timeline with GSAP scrub. Direct instruction to the design agent: build `app/(marketing)/page.tsx` with `<HeroPipeline />` as a 600vh section using R3F + Lenis exactly as specified above, plus a `<FallbackSequence />` for `prefers-reduced-motion` users that swaps to a 45-frame image strip.

## 2. IL CAPO PRODUCTION — https://www.awwwards.com/sites/il-capo-production

WHAT IT DOES: A film-production studio portfolio that treats the entire viewport as a continuous "reel." The grid is a CSS-grid catalog of project tiles where each tile is a `<video muted loop playsinline preload="metadata">` with a poster frame. Hovering a tile triggers `video.play()` and a clip-path reveal of a meta overlay (title, year, role). Page transitions are handled by a fixed full-screen black SVG mask whose `d` attribute morphs via GSAP between an open viewport and a closed slit, giving a "shutter" wipe. Type uses an oversized serif paired with a mono caption — pure Italian-cinema editorial.

PACKAGES:
- `next@14` + `react@18`
- `gsap@3.12` + `gsap/ScrollTrigger` + `gsap/Flip`
- `@studio-freight/lenis@1.1`
- `framer-motion@11` for layout shift handoff on routed views
- `tailwindcss@3.4` with custom `@theme` for the serif + mono pair
- Fonts: `next/font/google` Apoc Revelations (or `IBM Plex Serif` as a free swap) + `JetBrains Mono`
- `sharp` for build-time poster generation from MP4
- Optional: `hls.js@1.5` if streams instead of MP4 files

IMPLEMENTATION:
1. Layout: `grid-template-columns: repeat(12, 1fr); gap: 8px;` with tiles spanning `col-span-{4,6,8}` in a curated asymmetric pattern. Each tile is `aspect-ratio: 16/9` with `overflow: hidden`.
2. Tile component: `<article onMouseEnter={() => videoRef.current?.play()} onMouseLeave={() => { videoRef.current?.pause(); videoRef.current!.currentTime = 0 }}>`. The overlay is a sibling `<div>` with `clip-path: inset(100% 0 0 0)` at rest and `inset(0)` on hover, transitioned `clip-path 720ms cubic-bezier(0.83, 0, 0.17, 1)`.
3. Cursor: a custom `<div id="cursor">` that lerps toward `pointer` events at 0.18 — `gsap.quickTo(el, 'x', { duration: 0.4, ease: 'power3' })`. On tile hover, scale to 64px and swap label to "PLAY".
4. Shutter transition: a top-level `<svg class="fixed inset-0 z-[60] pointer-events-none">` containing a `<path>` whose two states are pre-computed (open, closed). Wrap router navigation: `await gsap.to(path, { attr: { d: closed }, duration: 0.55, ease: 'expo.inOut' }); router.push(href); await gsap.to(path, { attr: { d: open }, duration: 0.55, ease: 'expo.out' })`.
5. Type rhythm: hero h1 `clamp(72px, 12vw, 220px) / 0.86 line-height`, letter-spacing `-0.03em`. Body `16px / 1.5`. Caption mono `12px / uppercase / tracking-[0.16em]`.
6. Color: pure `#0A0A0A` ground, `#F5F1E8` cream type. No accent. Single hot color only on CTAs (`#FF3D00`).
7. Performance: `loading="lazy"` on tiles below the fold; `IntersectionObserver` swaps the `<video src>` in only when within 200px of viewport. Posters are `<img src=".../poster.avif" decoding="async" />` underneath.
8. Accessibility: every tile has `<a>` wrapping it with descriptive text; the video is purely decorative (`aria-hidden`).

WHY IT WORKS: Restraint. A single accent color, two typefaces, and a single transition primitive (the shutter) read as an authored point of view — editorial sites die from accumulating effects. The clip-path overlay obeys "reveal the meaning, hide the chrome": the project title only appears at the moment of intent (hover), which is a near-perfect application of progressive disclosure.

GOOD: The shutter wipe doubles as a route-transition loading state — zero perceived latency between pages.
BAD: Hover-only meta means touch users get no preview of what each tile is. No prefers-reduced-motion handling on the cursor.

CLEAN APPLICATION: Apply this to Clean's customer/use-case section ("Built by teams shipping agents"). Build a 12-column asymmetric grid of customer-story tiles where each tile is a 6-second screen-recording loop of an agent flow inside the product. On hover: video plays + overlay reveals "Company · Use case · ROI metric." Use the SVG shutter as the route transition into the full case study. Direct instruction to the design agent: build `app/(marketing)/customers/page.tsx` with a `<ReelGrid />` component using the exact clip-path overlay + GSAP shutter route transition specified above. Replace the serif with `Söhne Breit` for headlines and `JetBrains Mono` for captions to stay on Clean's developer-brand voice — not film-editorial.

## 3. Fauna Robotics — https://www.awwwards.com/sites/fauna-robotics

WHAT IT DOES: A consumer-facing robotics product site that deliberately rejects industrial sci-fi tropes. Built on Webflow with custom JS. Layout is a soft asymmetric scroll with the humanoid robot "Sprout" rendered as a sequence of high-res PNGs (not a 3D model) driven by a scrubbed canvas image-sequence — same technique Apple's iPad Pro page uses. Backgrounds are pastel gradients (#EFEFEF / #D1E3FF). Microinteractions: SVG eyes that follow the cursor, a button that "blinks" via `mask-image` animation, and section transitions that use a clip-path circle expansion from the previously-clicked CTA.

PACKAGES:
- Webflow as host (alt: `next@14` + `react@18` if rebuilding)
- `gsap@3.12` + `gsap/ScrollTrigger`
- `lenis@1.1`
- `lottie-web@5.12` or `@lottiefiles/dotlottie-react@0.7` for character expressions
- `tailwindcss@3.4` + `@tailwindcss/typography`
- Fonts: `Söhne` for headings, `Inter` for body
- `sharp` + a build script to pre-generate the 90-frame WebP sequence at 1440×1620
- `vanilla-extract` if rebuilding outside Webflow

IMPLEMENTATION:
1. Image sequence: pre-render 90 frames of Sprout from Blender or whatever DCC, export as WebP at 1440×1620 (≤ 32 KB each post-compression, ~2.8 MB total). Use `<canvas width="1440" height="1620">` and an `images[]` array preloaded on idle (`requestIdleCallback`).
2. Scrub: `ScrollTrigger.create({ trigger: '#sprout-section', start: 'top top', end: '+=3000', pin: true, scrub: 0.4, onUpdate: (self) => { const i = Math.min(89, Math.floor(self.progress * 89)); ctx.drawImage(images[i], 0, 0); } })`.
3. Eyes: two `<svg>` circles inside a parent `<div>` tracking `mousemove`. Map cursor delta from element center to a max ±6px translate via `Math.atan2`. Add an idle blink: `setInterval(() => triggerBlink(), 4200 + Math.random() * 1800)` where `triggerBlink()` animates `transform: scaleY(0.1)` for 120ms.
4. CTA: rounded-full button with `mask-image: radial-gradient(circle at var(--mx) var(--my), black 0%, black 60%, transparent 70%)`. On `mousemove` set `--mx` / `--my` from local coords — gives a "highlight follows cursor" effect inside the pill.
5. Circle wipe transition: on CTA click, `gsap.to('#wipe', { clipPath: 'circle(150% at var(--cx) var(--cy))', duration: 0.7, ease: 'expo.inOut' })` where `--cx` / `--cy` are set to the click coords on the source button.
6. Color system: `--bg: #EFEFEF; --accent: #D1E3FF; --ink: #1A1F2B`. All sections use exactly these three.
7. Type rhythm: `h1` `clamp(56px, 8vw, 144px)` weight 500, `letter-spacing: -0.02em`. Bias toward sentence-case, never ALL CAPS — that's the "approachable" tell.
8. Mobile: drop the scrubbed sequence to 30 frames at 640×720; on `prefers-reduced-motion`, render a single hero frame and disable pinning.

WHY IT WORKS: Image-sequence scrubbing is the cheapest "premium product" lever in 2026 — it reads as 3D without the cost. The combination of warm pastel grounds + a character with eyes that follow you triggers parasocial attention; users instinctively pause. Sentence-case + soft radius (16px+ everywhere) signals "consumer-friendly tech," which is rare in robotics — most competitors look militarized.

GOOD: The character-eyes detail is the single highest-ROI microinteraction on the page — it's the moment that converts skim into engagement.
BAD: Webflow-hosted means JS is fighting the runtime; total page weight is ~9 MB on first load. Pinning sections + Lenis on Safari iOS still occasionally drops frames.

CLEAN APPLICATION: Apply this to Clean's product-explainer section ("What Clean does"). Replace Sprout with a stylized 3D rendering of the Clean MCP proxy — a translucent obelisk with internal flowing data lines. Build the image sequence: 60 frames showing the proxy "thinking" — light pulses traveling through the obelisk, periodic shifts in internal geometry. Pin and scrub on a 200vh section. Keep the warm pastel background but tune to Clean's palette (off-white `#F7F5F1`, accent `#6366F1`). Direct instruction to the design agent: build `<ProxyExplainer />` as a pinned ScrollTrigger section between hero and pricing, using the 60-frame WebP scrub technique above — render the frames in Blender from a single rotating-pulse animation, export at 1280×1440. Add a static "data line" SVG overlay with `stroke-dasharray` animation timed to the scroll progress so the data flow always reads, even at low scroll velocities.

## 4. Indigo Laboratory — https://www.awwwards.com/sites/indigo-laboratory

WHAT IT DOES: A single-page experimental fashion/music site that runs a Curtains.js WebGL plane behind every text block, applying a custom GLSL fragment shader that distorts the background image based on mouse position and scroll velocity. Two-color palette (electric `#6A15FF` indigo + bone `#F4F3EB`). Type is set in a single weight of a contemporary grotesk, sized aggressively (hero h1 fills ~70% of viewport width). Background audio loops at -16 LUFS, with a mute toggle in the corner that animates a tiny EQ-bar SVG.

PACKAGES:
- `nuxt@3.12` + `vue@3.4`
- `curtainsjs@8.1` — WebGL plane manager
- `gsap@3.12` + `gsap/ScrollTrigger`
- `lenis@1.1`
- `howler.js@2.2` for ambient audio
- `tailwindcss@3.4`
- Fonts: `PP Neue Montreal` (paid) or `Inter Display` as a free swap

IMPLEMENTATION:
1. Mount Curtains: `const curtains = new Curtains({ container: '#canvas', pixelRatio: Math.min(1.5, window.devicePixelRatio), watchScroll: false })`. Each `<img data-plane>` in the DOM becomes a WebGL plane via `new Plane(curtains, htmlEl, { vertexShader, fragmentShader, uniforms })`.
2. Fragment shader (core):
   ```glsl
   uniform vec2 uMouse; uniform float uVelocity; uniform sampler2D uTexture;
   varying vec2 vTextureCoord;
   void main() {
     vec2 uv = vTextureCoord;
     float d = distance(uv, uMouse);
     vec2 disp = (uv - uMouse) * (0.06 / (d + 0.1)) * uVelocity;
     vec4 r = texture2D(uTexture, uv - disp * 1.05);
     vec4 g = texture2D(uTexture, uv - disp * 1.00);
     vec4 b = texture2D(uTexture, uv - disp * 0.95);
     gl_FragColor = vec4(r.r, g.g, b.b, 1.0);
   }
   ```
   The 1.05/1.00/0.95 split gives a chromatic-aberration ripple on mouse move.
3. Mouse + velocity: track `mouseMove` and compute `velocity = clamp(distance(prev, curr) / dt, 0, 60)`. Lerp the uniform toward target at 0.08 per rAF.
4. Type: hero is a single `<h1>` with `font-size: 22vw; line-height: 0.82; letter-spacing: -0.04em; mix-blend-mode: difference;`. The mix-blend keeps the type readable as the shader distorts behind it.
5. Audio: `const sound = new Howl({ src: ['/ambient.webm', '/ambient.mp3'], loop: true, volume: 0.18 })`. Gate playback behind a user gesture (Safari requirement). Mute toggle is a 24px `<svg>` of three vertical bars whose `<rect>` heights animate via `gsap.to(rect, { attr: { height: () => gsap.utils.random(2, 16) }, duration: 0.3, repeat: -1, yoyo: true })`.
6. Scroll: Lenis at `lerp: 0.1`, no anchors snap. Sections are 100vh each; type fades on `ScrollTrigger.batch` with `opacity 0 → 1` over a 30% travel.
7. Color: only two values. `:root { --ink: #F4F3EB; --bg: #6A15FF }`. No grays — any "muted" effect comes from blend-mode or shader output.
8. Mobile: drop the shader (Curtains plane becomes a fallback `<img>`), disable audio autoprompt, keep the type scale (it actually works on mobile because of `vw`-based sizing).

WHY IT WORKS: One shader + one color combo + one typeface = a complete sensory signature. The shader's RGB-split intensity scales with scroll/mouse velocity, so motion is *responsive*, not canned — the page feels alive specifically because the user is causing the effect. Sound at -16 LUFS is below conscious threshold but raises perceived "production value" significantly.

GOOD: The velocity-driven chromatic aberration is a generalizable primitive — works on any image-heavy site, costs almost nothing.
BAD: WCAG contrast on `#F4F3EB` over `#6A15FF` is ~5.1:1 — passes AA for large text but fails AAA. Auto-prompting audio (even muted) is borderline; the toggle should default to off.

CLEAN APPLICATION: Apply this to Clean's docs landing / changelog page hero. Use the chromatic-aberration shader on a single hero plane that holds a still of a code editor with Clean's MCP config visible. Mouse velocity warps the editor subtly — gives "live code" feel without burning a real Monaco instance on first paint. Use Clean's palette: bg `#0B0B0F`, accent `#6366F1`, ink `#F5F5F7`. Direct instruction to the design agent: rebuild `app/docs/page.tsx` hero as a Curtains.js plane wrapping a 1920×1080 PNG of the marketing config file, with the fragment shader above. Strip the audio. Keep type at a single weight (`Geist Mono` or `JetBrains Mono`), `font-size: 14vw`, `mix-blend-mode: difference` over the warped image.

## 5. Aceternity UI Latest Drops (Bento Grid + Card Spotlight + Background Beams) — https://ui.aceternity.com/components

WHAT IT DOES: A maintained library of copy-paste React components built on Tailwind + Motion (formerly Framer Motion). The three highest-impact recent components: (a) `BentoGrid` — asymmetric responsive grid where one cell can house an embedded animation (Apple-style); (b) `CardSpotlight` — a card whose background renders a radial gradient that tracks the cursor via CSS variables; (c) `BackgroundBeams` — full-section SVG of animated path strokes that wave like search beams using `motion.path` with offset-path animation. All three drop in as `<BentoGrid>`, `<CardSpotlight>`, `<BackgroundBeams />` after a single `npx shadcn add` command via Aceternity's CLI.

PACKAGES:
- `next@14` or `vite@5` + `react@18`
- `tailwindcss@3.4` + `tailwind-merge@2.3` + `clsx@2.1`
- `motion@11.5` (the package formerly known as `framer-motion`)
- `@radix-ui/react-*` for primitives (Tooltip, Dialog, etc.)
- `lucide-react@0.453` for icons
- `class-variance-authority@0.7`
- Optional: `simplex-noise@4.0` for richer beam paths

IMPLEMENTATION:
1. CLI install: `npx shadcn@latest add https://ui.aceternity.com/registry/bento-grid.json` — pulls source straight into `components/ui/`. No package install, you own the code.
2. `BentoGrid` markup:
   ```tsx
   <div className="grid auto-rows-[18rem] grid-cols-3 gap-4 max-w-7xl mx-auto">
     <BentoCard className="md:col-span-2" header={<AnimatedHeader />} title="…" />
     {/* 5 more cells, varying col-span */}
   </div>
   ```
   `BentoCard` is a `motion.div` with `whileHover={{ y: -2 }}` and `transition={{ type: 'spring', stiffness: 320, damping: 26 }}`. Border is `border border-white/[0.08]` over `bg-neutral-950`.
3. `CardSpotlight`: an `onMouseMove` writes `--x` and `--y` via `e.currentTarget.style.setProperty`. Background is `background: radial-gradient(600px circle at var(--x) var(--y), rgba(99,102,241,0.18), transparent 40%)`. No JS animation loop — pure CSS variable update; ~free performance.
4. `BackgroundBeams`: an SVG with 50 generated `<path>` elements. Each path is a smooth bezier from off-screen left to off-screen right. Animate with `<motion.path strokeDasharray="..." animate={{ strokeDashoffset: [1000, -1000] }} transition={{ duration: 5 + Math.random()*3, repeat: Infinity, ease: 'linear', delay: Math.random()*-5 }} />`. Strokes use `stroke="url(#beamGradient)"` where the gradient is `from transparent → indigo-400 → transparent`.
5. Theming: define `tailwind.config.ts` with `colors: { brand: { 50: '...', 500: '#6366F1', 950: '#0B0B0F' } }`. All three components read from `text-brand-500` / `bg-brand-950`.
6. Reduced motion: wrap motion components with `<LazyMotion features={domAnimation}>` and gate `animate` props behind `useReducedMotion()`.
7. Perf: the BackgroundBeams component is the most expensive — limit to one per page and `pointer-events: none` always. Use `will-change: transform` only on the BentoCard hover state, never on the beams.

WHY IT WORKS: Copy-paste source ownership is the right tradeoff for fast-moving dev tools — you get production-grade animation without versioning a dependency that might break your design later. The three components together cover the three jobs every SaaS landing page needs: (a) feature grid that doesn't look like a feature grid (Bento), (b) card whose interactivity invites further reading (Spotlight), (c) atmosphere that doesn't compete with content (Beams). Each is restrained alone; together they read as a system.

GOOD: The CSS-variable spotlight technique is the most cost-efficient hover effect on the modern web — zero JS animation loop, perfect 120Hz on a phone.
BAD: Background Beams shows up on 60% of dev-tool landing pages launched in 2026. It's now a tell, not a differentiator — use sparingly. The library leans heavy on `motion`'s spring defaults, which can feel "bouncy startup" if not retuned to a tighter stiffness/damping ratio.

CLEAN APPLICATION: Apply this to Clean's "Features" section of the landing page (the section between hero and pricing). Build a 3-column / 6-cell BentoGrid where each cell is a feature of the Clean product: (1) MCP proxy live request counter, (2) TOON compression ratio animation, (3) context cache hit rate graph, (4) policy guardrails toggle list, (5) latency histogram, (6) integrations logo wall. Wrap each cell in `CardSpotlight` so hover surfaces a tinted indigo wash. SKIP BackgroundBeams — it's over-used and dilutes Clean's "serious infra" tone. Direct instruction to the design agent: scaffold `<FeatureGrid />` at `components/marketing/feature-grid.tsx` using Aceternity's BentoGrid + CardSpotlight only. Retune `motion`'s spring to `{ stiffness: 380, damping: 32 }` for a snappier (less bouncy) feel that matches developer-tool taste. Use brand color `#6366F1` for spotlight tint, kept under 18% opacity.

## BUILD DIRECTIVE

TARGET: **Clean's landing-page hero + features section** (`app/(marketing)/page.tsx`)

APPROACH: Combine the strongest patterns from today's pool into a single coherent hero-to-features funnel. The hero is a pinned, scroll-scrubbed Three.js scene (Ten Years Away + Fauna Robotics image-sequence techniques merged) showing a stylized MCP proxy with data flowing through it across five chapters of the request lifecycle. Below the hero, a BentoGrid (Aceternity) of six feature cells with CardSpotlight hover, retuned for snappier motion. Type set in a single weight of `Geist` for headings + `Geist Mono` for code, two-color palette (`#0B0B0F` ground, `#6366F1` accent, `#F5F5F7` ink). Lenis drives a single GSAP master timeline. Mobile gracefully degrades the WebGL scene to a 30-frame image sequence; `prefers-reduced-motion` users get a static hero with no scrub at all.

STACK:
- `next@14` (app router) + `react@18` + `typescript@5.5`
- `@react-three/fiber@8.16` + `@react-three/drei@9.108` + `@react-three/postprocessing@2.16` + `three@0.166`
- `gsap@3.12` (+ `ScrollTrigger`, `Flip`) + `lenis@1.1`
- `motion@11.5` (Aceternity components use it)
- `tailwindcss@3.4` + `tailwind-merge@2.3` + `clsx@2.1` + `class-variance-authority@0.7`
- `lucide-react@0.453` + `@radix-ui/react-tooltip`
- Aceternity components installed via `npx shadcn add` (BentoGrid, CardSpotlight)
- Fonts: `next/font` Geist + Geist Mono
- `sharp` for build-time WebP sequence generation
- `@vercel/analytics` for funnel instrumentation

REFERENCE URLS:
- https://www.awwwards.com/sites/ten-years-away (scroll-scrub WebGL narrative — primary hero reference)
- https://ui.aceternity.com/components/bento-grid + https://ui.aceternity.com/components/card-spotlight (features section)

KEY CONSTRAINTS:
- Clean is developer infrastructure. Result must feel like Linear/Vercel/Stripe, not like a fashion site. No serif headings. No ambient audio. No oversized hover scales. No emoji.
- AVOID: BackgroundBeams (oversaturated), full-screen route-shutter wipes (too editorial for dev-tools), parasocial character mascots (Fauna's "eyes-follow-cursor" trick belongs in consumer, not infra), three-color gradients (stay 2-color).
- Perf budget: LCP ≤ 2.0s, total JS ≤ 220 KB gzipped, hero GLTF ≤ 4 MB Draco-compressed. Above-the-fold must be readable with JS disabled (text content fallback in the same DOM).
- Accessibility: keyboard-skippable hero (`Skip to features` link in DOM order before canvas), `prefers-reduced-motion` honored at the scene level, all interactive elements ≥ 44px touch target, 4.5:1 minimum contrast on every text/background pair.
- Motion semantics: scroll = lifecycle progression (request → proxy → response → telemetry). No motion exists that doesn't reinforce that mental model.
