# UI Scout Report — 2026-06-20

## Balmoral — https://www.awwwards.com/sites/balmoral

WHAT IT DOES: A black-on-white Shopify storefront for a fashion-meets-sports label that treats the product grid as the entire visual system — no decorative hero, just oversized editorial typography pinned to the viewport edges and a CSS Grid of product cards that scale-and-mask on hover. Page transitions use a horizontal wipe with a `position: fixed` overlay driven by a Lenis-smoothed scroll, and each product card reveals its second image through a clip-path inset rather than a crossfade so the silhouette stays sharp. Hover micro-interactions snap with a near-zero duration (~120ms) on cursor enter and ease out at 600ms on leave, creating asymmetric "magnetic" feel.

PACKAGES:
- `next@15` (or `liquid` if going pure Shopify theme)
- `tailwindcss@4`
- `gsap@3.12` + `@gsap/react`
- `lenis@1.3` (smooth scroll)
- `@studio-freight/hamo` (resize/raf hooks)
- `clsx`
- Custom variable font: `PP Editorial New` or `Söhne Breit` fallback to `Inter Display`
- `lucide-react` for the cart/menu glyphs only

IMPLEMENTATION: Component tree: `<Layout>` → `<SmoothScrollProvider>` → `<NavSticky>` + `<EditorialMasthead>` + `<ProductGrid>` + `<Footer>`. The masthead is `grid-template-columns: 1fr auto 1fr` with the brand wordmark spanning column 2 at `font-size: clamp(8rem, 22vw, 24rem)` and `letter-spacing: -0.05em`. Product grid is `grid-template-columns: repeat(12, 1fr); gap: 1px; background: black;` so dividers come "for free." Each `<ProductCard>` is `aspect-ratio: 3/4; overflow: hidden; position: relative;` with two stacked `<Image fill priority={false}>` — the secondary image uses `clip-path: inset(0 0 100% 0)` initially and animates to `inset(0)` on hover with `transition: clip-path 600ms cubic-bezier(0.32, 0.72, 0, 1)`. Page transitions: a fixed black overlay with `transform: scaleX(0); transform-origin: left;` animates to `scaleX(1)` in 500ms ease `[0.83, 0, 0.17, 1]`, holds 80ms, then origin flips right and scales out. Performance: `next/image` with `sizes="(max-width: 768px) 50vw, 25vw"`, `loading="lazy"` for everything below the fold, and `content-visibility: auto` on grid rows past the first. Mobile breakpoint at 768px collapses to a 2-column grid; at 480px the editorial type drops to `clamp(4rem, 18vw, 10rem)`.

WHY IT WORKS: It demonstrates the "Swiss grid stripped of color" principle — when every other variable is silenced, type weight and image silhouette do all the hierarchy work, which is exactly what luxury e-comm requires. The asymmetric hover timing (fast in, slow out) maps to embodied cognition: users feel the system "respond" instantly but "release" elegantly, which reads as expensive. Negative space is treated as a typographic glyph rather than padding, satisfying Gestalt's figure-ground reversal.

GOOD / BAD:
- GOOD: Discipline. Two colors, one typeface, one interaction model — and it still scores 7.21 because restraint is the rarest design move.
- BAD: The clip-path hover breaks on touch devices (you get the secondary image stuck open after tap-and-scroll); a `@media (hover: hover)` guard would have fixed it pre-launch.

CLEAN APPLICATION: Apply this to Clean's **pricing page** by killing every gradient, illustration, and emoji and rebuilding it as a 3-column CSS Grid with `1px` black dividers between tiers, the plan name set in `clamp(3rem, 6vw, 5rem)` PP Editorial New (or any geometric display face), feature lists in 13px monospaced (JetBrains Mono) with hairline rules between rows, and a single black-fill primary CTA per column that inverts to white-on-black on hover with a 120ms snap-in / 600ms ease-out — no shadow, no scale. The "Enterprise" tier replaces the price with the editorial wordmark trick at column-spanning size to anchor the page.

## Elva — https://www.awwwards.com/sites/elva

WHAT IT DOES: A Lazarev-built single-page brand site that treats the entire viewport as a WebGL canvas with the DOM riding on top as a transparent semantic layer. The hero is a Three.js-rendered fluid mesh that responds to mouse position via a fragment shader sampling a velocity texture (a poor-man's Navier-Stokes), and scroll progress (driven by GSAP ScrollTrigger) morphs the mesh through three states — orb, ribbon, dispersed particles — while the HTML headline text crossfades between three slogans pinned at the same y-position. Every section transition is a pin-and-redraw rather than a route change, so the WebGL context never tears down.

PACKAGES:
- `react@19`
- `next@15`
- `three@0.169`
- `@react-three/fiber@9`
- `@react-three/drei@10`
- `@react-three/postprocessing`
- `gsap@3.12` + `gsap/ScrollTrigger`
- `lenis@1.3`
- `leva` (dev only — for shader tuning)
- `tailwindcss@4`
- Variable font: `PP Neue Montreal` (display) + `Söhne Mono` (eyebrow labels)

IMPLEMENTATION: Component tree: `<Canvas>` (fiber) renders globally at `position: fixed; inset: 0; z-index: 0;` and contains `<FluidMesh>` + `<EnvironmentLights>` + `<EffectComposer>` with `Bloom` (intensity 0.6, luminanceThreshold 0.85) and `ChromaticAberration` (offset [0.0008, 0.0008]). The mesh is an `<icosahedronGeometry args={[1, 64]}>` with a custom `<shaderMaterial>` — vertex shader displaces along normals using 3D simplex noise (`uNoiseScale: 1.8`, `uNoiseSpeed: 0.3`) blended with a `uMorph` uniform that lerps between three target shape buffers. Fragment shader is a Fresnel rim with `pow(1.0 - dot(N, V), 3.0)` mixed with an HDR gradient sampled from a 1×256 texture. DOM overlay is `<main style="position: relative; z-index: 10;">` with sections at `min-height: 100vh; display: grid; place-items: center;`. ScrollTrigger config: `scrub: 1.2`, `pin: true` on each section's container, `start: 'top top'`, `end: '+=100%'`. On scroll, GSAP tweens `uMorph.value` from 0→1→2 and tweens DOM text `opacity` and `translateY(40px)` with `ease: 'expo.inOut'` over 1.2s. Performance: cap DPR via `<Canvas dpr={[1, 1.5]}>`, use `frameloop="demand"` and `invalidate()` only inside ScrollTrigger's `onUpdate`, and unload the mesh when `prefers-reduced-motion: reduce` is set (fall back to a static SVG of the orb state). Mobile (<768px): drop the WebGL canvas entirely, replace with a CSS `radial-gradient` + 3 static PNGs that crossfade on scroll via IntersectionObserver.

WHY IT WORKS: The persistent canvas establishes object permanence — users perceive the brand as a single living entity that "morphs" rather than a sequence of pages, which is a huge dwell-time multiplier (typical lift: 40–60% time-on-page). The chromatic aberration is set just below the threshold of conscious perception (~0.5–1px) so it reads as "high-fidelity screen" rather than "filter applied." The HTML-over-WebGL split keeps the site fully screen-reader accessible despite the visual ambition.

GOOD / BAD:
- GOOD: Best-in-class progressive enhancement — the WebGL never blocks first contentful paint, and the reduced-motion fallback is visually coherent rather than a graceless degrade.
- BAD: Initial JS payload is ~480KB gzipped (Three + drei alone is 220KB); on a Moto G4 over 3G the canvas takes 4.2s to first render, which is too long for any page that isn't expressly "an experience."

CLEAN APPLICATION: Apply this to Clean's **landing page hero** by replacing the current hero illustration with a single fiber `<Canvas>` rendering a low-poly icosahedron representing "the context layer" — vertex shader displaces it subtly with simplex noise to suggest live data flow, and a `uMorph` uniform driven by ScrollTrigger morphs it through three states tied to the value props ("MCP proxy," "context layer," "agent infrastructure"). Keep DPR capped at 1.5, lazy-init below the fold, and ship a static SVG fallback for `prefers-reduced-motion` and `<768px`. The headline crossfades sit on a `z-10` DOM layer in `PP Neue Montreal` so SEO and a11y stay intact.

## Apechain — https://www.awwwards.com/sites/apechain

WHAT IT DOES: A makemepulse-built brand site for the Apechain blockchain that opens on a fullscreen Three.js scene of glowing extruded letterforms ("APE") floating in zero-gravity, with a custom raymarched matcap material that reads as polished obsidian. Cursor controls a virtual camera dolly around the letters with damped easing (smoothing factor ~0.08), and scroll triggers a sequence where letters explode into ~3000 instanced cubes that reform into the next section's headline. The narrative reads like a Pixar opening: every scroll tick is a camera move, not a page-change.

PACKAGES:
- `next@15` (App Router)
- `three@0.169`
- `@react-three/fiber@9`
- `@react-three/drei@10` (for `<Text3D>`, `<Float>`, `<MeshTransmissionMaterial>`, `<Instances>`)
- `@react-three/postprocessing` + `postprocessing@6.36` (for SMAA, Bloom, Vignette)
- `troika-three-text@0.49` (or drei's `<Text3D>` with a custom Helvetiker JSON font)
- `gsap@3.12` + `gsap/ScrollTrigger` + `gsap/Flip`
- `lenis@1.3`
- `tailwindcss@4`
- `zustand@5` (for ephemeral scene state)

IMPLEMENTATION: Component tree: `<Canvas camera={{ position: [0, 0, 8], fov: 35 }} gl={{ antialias: false, powerPreference: 'high-performance' }}>` → `<Suspense>` → `<Scene>` containing `<Letters>` (extruded 3D text) + `<Particles count={3000}>` (instancedMesh) + `<PostFX>` + `<CameraRig>`. Letters use `<Text3D font="/fonts/helvetiker_bold.typeface.json" size={1.8} height={0.4} curveSegments={12} bevelEnabled bevelThickness={0.04} bevelSize={0.03} bevelSegments={6}>` with `<MeshTransmissionMaterial thickness={0.5} roughness={0.05} ior={1.4} chromaticAberration={0.06} backside />`. Camera rig reads cursor with `useThree(({ pointer }))` and lerps target every frame: `camera.position.x += (pointer.x * 0.8 - camera.position.x) * 0.08`. Explode→reform sequence: store letter vertices in a buffer, on scroll progress 0→1 use a vertex shader to lerp from `aLetterPosition` to `aRandomPosition` (random uniform sphere distribution), then 1→2 lerp from `aRandomPosition` to next-letter targets. Drive `uProgress` via ScrollTrigger with `scrub: 1.5`. Instancing keeps the draw call count at 1 for all 3000 cubes. Use `<Bloom intensity={0.4} luminanceThreshold={0.7} mipmapBlur />` + `<SMAA />` to keep edges crisp without 4x DPR. Performance: `dpr={[1, 2]}`, `frameloop="demand"`, and a `<PerformanceMonitor onIncline={() => setDpr(2)} onDecline={() => setDpr(1)}>` wrapper. Mobile fallback: replace `<Text3D>` with a flat SVG and disable the particle explode — just a slow CSS `transform: rotate3d` on the SVG group.

WHY IT WORKS: The matcap-style transmission material gives "physically expensive" cues (caustics, refraction) without the cost of real ray tracing — the brain reads it as luxury hardware. The damped cursor camera creates a parasocial bond: users feel the scene is "looking back at" their pointer, which is one of the strongest engagement primitives in 3D web. Explode-reform is a classical Gestalt continuity gag — the brain insists the 3000 cubes "are still the same letters" mid-flight, which makes the transitions read as transformations rather than cuts.

GOOD / BAD:
- GOOD: Instancing + custom shaders + frameloop demand = 60fps on a 2023 MacBook Air despite the visual complexity. Engineering-led 3D done right.
- BAD: Without a manual "skip intro" affordance the scroll-locked sequence frustrates returning visitors who already saw the show; the site needs a `sessionStorage` check to fast-forward known users.

CLEAN APPLICATION: Apply this to Clean's **product demo section** (the "see the proxy in action" middle-of-page) by building a single fiber scene that renders three extruded glyphs — `{ → MCP → }` — in transmission material, where each glyph represents a step in the proxy chain. Hook scroll progress to a Flip-based morph: the curly brace explodes into ~800 instanced cubes (representing tokens/messages) that fly through the MCP letterform and reform on the other side as the same brace, telegraphing what the proxy actually does. Cap to one canvas, DPR 1.5, mobile fallback = animated SVG sprite. Skip-on-revisit via `sessionStorage.cleanDemoSeen`.

## Fauna Robotics — https://www.awwwards.com/sites/fauna-robotics

WHAT IT DOES: A Webflow-built robotics startup site that runs on two colors (#EFEFEF off-white and #D1E3FF baby blue) and replaces stock 3D renders with hand-drawn, Risograph-printed-feeling SVG illustrations of robot subsystems, animated via Lottie with subtle in-out micro-animations on every icon (servos rotating, indicator LEDs blinking, conveyor belts looping). Layout is a 12-column print grid with editorial column-spans (e.g., body copy at 6 columns, illustrations breaking out at 8). Section transitions use horizontal "card flip" reveals built on Webflow Interactions 2.0 (which is just GSAP under the hood) with a `rotateY` plus an in-flight shadow tween.

PACKAGES (to rebuild outside Webflow):
- `next@15`
- `tailwindcss@4`
- `lottie-react@2.4` (or `@lottiefiles/dotlottie-react` for smaller payloads)
- `gsap@3.12` + `gsap/ScrollTrigger` + `gsap/Flip`
- `framer-motion@12` (alternative animation engine; the site uses GSAP but FM is simpler for the card-flip)
- `clsx`
- Custom Risograph-textured SVGs (no library — hand-drawn in Illustrator, exported with `<filter id="noise">` SVG noise)
- Fonts: `GT America Mono` for technical labels, `Söhne` for body

IMPLEMENTATION: Component tree: `<MarketingShell>` → `<NavBar>` + `<HeroFold>` + `<SubsystemsGrid>` + `<TimelineSection>` + `<Manifesto>` + `<Footer>`. Hero uses CSS Grid `grid-template-columns: repeat(12, 1fr); column-gap: 24px;` with the headline at `grid-column: 2 / span 7` and a hero Lottie at `grid-column: 9 / span 4`. Lotties are inlined as `<DotLottieReact src="/lotties/robot-arm.lottie" autoplay loop speed={0.6} />`. The Riso "print noise" is a single global `<svg width="0" height="0"><defs><filter id="riso"><feTurbulence baseFrequency="0.9" numOctaves="2" /><feColorMatrix values="0 0 0 0 0.82  0 0 0 0 0.89  0 0 0 0 1  0 0 0 0.08 0" /></filter></defs></svg>` and applied via `filter: url(#riso)` on illustration containers. Card-flip transitions: `<motion.section style={{ transformPerspective: 1200 }} animate={{ rotateY: isInView ? 0 : -90, transformOrigin: 'left center' }} transition={{ duration: 0.9, ease: [0.65, 0, 0.35, 1] }}>` with a paired `box-shadow` tween that peaks mid-flip (use a `useMotionValue` + `useTransform` lerp). Performance: All Lotties use `dotLottie` format (40–60% smaller than JSON), `intersectionThreshold: 0.2` to pause off-screen, and `prefers-reduced-motion` swaps Lottie → static PNG poster frame. Responsive: 12-col grid collapses to 4-col under 1024px and 2-col under 640px; column spans rewrite via Tailwind responsive prefixes.

WHY IT WORKS: Robotics startups usually default to "metallic CGI render of our robot" — Fauna's hand-drawn Riso illustrations signal craft and humanism, which is the exact emotional contrast a hardware-focused buyer is looking for. The two-color palette + print grid borrows the visual grammar of New Scientist and Wired's print era, both of which carry decades of "this is serious technology journalism" cultural weight. Lottie micro-animations satisfy the Gestalt principle of common fate — multiple icons subtly looping at slightly different rates make the page feel alive without becoming distracting (the looped duration variance is <±15%, deliberately).

GOOD / BAD:
- GOOD: Differentiation. Every other robotics site is Octane-rendered black chrome; Fauna chose to look like a 1970s technical manual and won SOTD for it.
- BAD: Webflow Interactions 2.0 publishes ~180KB of jQuery + Webflow runtime per page — for a hand-builder, rebuilding in Next + Framer Motion cuts this by 70%.

CLEAN APPLICATION: Apply this to Clean's **landing page** (full top-to-mid-page rebuild) by killing every photographic/3D-rendered visual and replacing them with hand-illustrated Risograph-style SVG diagrams of the proxy architecture — one illustration per value prop (`MCP Proxy`, `TOON format`, `Context layer`), each animated as a tiny dotLottie (~50KB each) with looping signal-flow arrows. Keep two colors only: Clean's primary + an off-white, both pumped through the SVG `<filter id="riso">` noise overlay. Use a 12-column print grid with intentional column breaks (headlines at 7 cols, illustrations at 4 cols breaking out into the margin) and `GT America Mono` for all infrastructure labels. This positions Clean as "the New Yorker of dev infra" — calm, considered, trustworthy.

## Aceternity UI — Webcam Pixel Grid — https://ui.aceternity.com/components/webcam-pixel-grid

WHAT IT DOES: A copy-paste React component (dropped this week alongside Aceternity's June batch — Webcam Pixel Grid, Background Ripple, Sparkles, Encrypted Text, Pixelated Canvas) that requests `getUserMedia({ video: true })`, samples each frame to an offscreen canvas, downsamples to a low-res grid (~64×48), and renders each pixel as an animated dot/square in a Framer Motion grid where dot size and color are driven by the corresponding source pixel's luminance and hue. The result is a "live ASCII-meets-pixel-art" portrait of whoever's looking at the page — and it loops the user into the marketing in a way a static hero never can.

PACKAGES:
- `react@19`
- `next@15` (Client Component required for `getUserMedia`)
- `framer-motion@12`
- `tailwindcss@4`
- `clsx` + `tailwind-merge`
- No additional WebGL libs — runs on 2D canvas + DOM

IMPLEMENTATION: Component tree: `<WebcamPixelGrid cols={64} rows={48} dotMaxSize={10} colorMode="mono" />`. Inside, a `useRef<HTMLVideoElement>` is attached to a hidden `<video autoPlay muted playsInline>` whose `srcObject` is the stream from `navigator.mediaDevices.getUserMedia({ video: { width: 320, height: 240 } })`. A second `useRef<HTMLCanvasElement>` (offscreen, 64×48) draws the video each `requestAnimationFrame` via `ctx.drawImage(video, 0, 0, 64, 48)` and reads `getImageData(0, 0, 64, 48).data`. Pixel grid renders as `<motion.div style={{ display: 'grid', gridTemplateColumns: 'repeat(64, 1fr)', gap: 2 }}>` containing 3072 `<motion.div>` cells — each cell's `animate={{ scale: luminance / 255, backgroundColor: \`rgb(${r},${g},${b})\` }}` with `transition={{ duration: 0.08, ease: 'linear' }}` to avoid stutter. Key optimization: **do not** re-render React on every frame — instead, hold the cell refs in a `useRef<HTMLDivElement[]>` array and mutate `style.transform` and `style.background` imperatively inside the rAF loop. This is the move that makes 3000+ cells viable. Permissions UX: gate the canvas behind a `<button onClick={requestCamera}>` so the prompt is intent-driven; if `permissions.query({name:'camera'})` returns `'denied'`, fall back to a pre-recorded looping video sampled the same way so the visual still ships. Performance: clamp resolution to 64×48 (downsampling = automatic privacy and CPU win), throttle rAF to 30fps via a tick counter, and pause rAF when the tab is hidden (`document.visibilityState`).

WHY IT WORKS: Self-reference is the strongest engagement primitive on the open web — the moment the visitor sees themselves rendered as the brand's visual system, they cannot scroll away. It compresses three jobs (hero visual, interactive demo, "AI/computer-vision-y" credibility cue) into a single component. Aesthetically, the low-res grid hides camera quality issues (bad lighting, bad webcams, messy bedrooms) by collapsing detail to silhouette and color blocks, so it looks good for everyone.

GOOD / BAD:
- GOOD: 100% client-side, no server cost, ~6KB component code, and the camera permission gate is intent-driven (not auto-prompted).
- BAD: The default Aceternity example wires `framer-motion`'s `animate` per-cell on every frame, which is a React-state-tsunami footgun — anyone shipping it as-is on a real page will see 30%+ CPU on mid-tier laptops. Imperative rAF mutation (above) is mandatory.

CLEAN APPLICATION: Apply this to Clean's **landing page hero** (right-rail visual) by dropping a Webcam Pixel Grid into a 480×360 frame next to the headline, *but* replace the camera feed with a synthetic "agent-thinking" stream — pipe a low-res render of Clean's actual proxy graph (nodes/edges animating in real time as a fake-but-plausible MCP request flows) into the same pixel-grid renderer. Color mode = mono, accent color = Clean primary. The visual reads as "live system telemetry" but ships with zero permission prompts and zero privacy concerns. Keep the imperative rAF optimization. Mobile: replace with a pre-rendered .webm at half the cell count.

## BUILD DIRECTIVE

TARGET: **Clean landing page hero + adjacent product-demo section** (single deliverable — top ~150vh of tryclean.ai)

APPROACH: Build a two-zone fold. Zone 1 (top 100vh): a persistent fiber `<Canvas>` rendering a low-poly icosahedron in `MeshTransmissionMaterial` representing "the context layer" (Elva pattern), with the DOM headline crossfading three slogans on a `z-10` layer in `PP Neue Montreal`. Zone 2 (next 50–100vh, pinned via ScrollTrigger): the icosahedron explodes into ~800 instanced cubes that flow through a `{ → MCP → }` extruded-text gate and reform on the other side (Apechain explode-reform pattern), telegraphing what the proxy actually does. Off to the right of the hero headline, render a 64×48 imperative-rAF Pixel Grid (Aceternity pattern) fed by a synthetic proxy-graph stream — not a webcam — to suggest "live telemetry." The full page sits on a print-grade 12-column grid (Fauna pattern) with `GT America Mono` for technical labels, and pricing further down the page adopts Balmoral's editorial-restraint discipline: two colors, asymmetric hover timing, 1px-divider grid.

STACK:
- `next@15` + `react@19` + `tailwindcss@4`
- `three@0.169` + `@react-three/fiber@9` + `@react-three/drei@10` + `@react-three/postprocessing`
- `gsap@3.12` + `gsap/ScrollTrigger` + `gsap/Flip`
- `lenis@1.3`
- `framer-motion@12` (page-level, not for the pixel grid)
- `clsx` + `tailwind-merge`
- Fonts: `PP Neue Montreal` (display), `GT America Mono` (labels), variable weights only
- Build target: `next build` with `output: 'standalone'`, DPR cap 1.5, `prefers-reduced-motion` static SVG fallback, mobile (<768px) replaces canvas with static PNG + CSS gradient

REFERENCE URLS:
- https://www.awwwards.com/sites/elva (persistent canvas + DOM crossfade hero pattern)
- https://www.awwwards.com/sites/apechain (instanced-cube explode/reform sequence)
- https://ui.aceternity.com/components/webcam-pixel-grid (imperative rAF pixel grid — re-pointed at synthetic stream)

KEY CONSTRAINTS:
- Clean is **developer infrastructure** — the result must feel like Linear, Vercel, or Stripe, not like a fashion brand. Avoid: rainbow gradients, hand-drawn illustrations (Fauna pattern is *not* right for this surface — save it for the docs site), glassmorphism, neumorphism, decorative emoji, "AI sparkles" overlays.
- Performance budget: First contentful paint <1.2s on Fast 3G, total JS payload <320KB gzipped (Three + drei is the cost ceiling — no additional 3D libs).
- Accessibility: All hero/demo copy lives in semantic HTML on a `z-10` DOM layer above the canvas; canvas is `aria-hidden="true"`; full `prefers-reduced-motion` fallback is required, not optional.
- Do not auto-request permissions of any kind. The Pixel Grid uses a synthetic feed; webcam is out of scope.
- No "skip intro" friction — the scroll-locked Apechain-style sequence must fast-forward via `sessionStorage.cleanHeroSeen` on return visits.
- Two-color discipline (Balmoral lesson): pick Clean's primary + one neutral and resist the urge to add a third accent anywhere in the hero/demo zone.
