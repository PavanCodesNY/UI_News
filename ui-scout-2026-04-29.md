## Silent House — https://silent-house.com/ (Awwwards SOTD, Apr 29 2026)

WHAT IT DOES: Full-bleed hero is a single 16:9 hero reel autoplaying a muted MP4/HEVC at ~24fps, with a 6×8 typographic grid overlaid using `mix-blend-mode: difference` so the wordmark inverts against the footage in real time. Below the fold, a horizontal scroll-jacked rail (translate3d on a 7000px-wide flex row driven by ScrollTrigger pinSpacing) cycles 12 client cards; each card has a clip-path `inset(0 round 24px)` reveal tied to scroll progress. A persistent diagonal cursor (a 96px SVG arrow) replaces the OS cursor and rotates 45deg toward whatever it's hovering, animated with a `gsap.quickTo` setter on `mousemove` (lerp 0.18). Page transitions are FLIP — the active card animates from its grid position into a full-screen detail view in 0.7s.

PACKAGES:
- next@15 (App Router)
- react@19, react-dom@19
- gsap@3.13 (incl. ScrollTrigger, Flip, Observer, ScrollSmoother — Club GreenSock plugins)
- @gsap/react@2 (useGSAP hook)
- lenis@1.1 (smooth scroll, hooked via `lenis.on('scroll', ScrollTrigger.update)`)
- @studio-freight/hamo (rAF utilities)
- next-video or mux-player-react@2 for the hero reel
- styled-components@6 OR vanilla-extract@1.16
- @next/font with self-hosted Söhne / Neue Haas Grotesk Display fallback
- lucide-react@0.383 for utility icons

IMPLEMENTATION:
1. Layout shell: `<html data-theme="dark">` → `<body class="font-sans bg-[#0A0A0A] text-[#F5F5F5]">`. Lenis bootstrapped in a `useEffect` mounted before any GSAP timelines; `gsap.ticker.add((time) => lenis.raf(time * 1000))` and disable native ticker lag smoothing.
2. Hero: a `<section>` with `height: 100svh`, video as `<video autoPlay muted playsInline preload="auto" poster=...>` set to `object-fit: cover`. Wordmark is an `<h1 style={{ mixBlendMode: 'difference', color: '#FFFFFF', fontFamily: 'Söhne Breit', fontWeight: 800, fontSize: 'clamp(64px, 14vw, 280px)', lineHeight: 0.86, letterSpacing: '-0.04em' }}>`. Fade in with `gsap.from(chars, { yPercent: 100, stagger: 0.04, duration: 1.1, ease: 'expo.out' })` after a SplitText split.
3. Horizontal rail: a pinned ScrollTrigger panel.
   ```js
   gsap.to(rail, {
     xPercent: -100 * (cards.length - 1) / cards.length,
     ease: 'none',
     scrollTrigger: {
       trigger: section,
       start: 'top top',
       end: () => `+=${rail.scrollWidth - innerWidth}`,
       scrub: 0.6,
       pin: true,
       anticipatePin: 1,
       invalidateOnRefresh: true,
     }
   })
   ```
   Each card is `width: 78vw; aspect-ratio: 4/5; border-radius: 24px; overflow: hidden`; image inside has `scale: 1.12` and is scaled back to 1 across the card's progress with a per-card ScrollTrigger.
4. Custom cursor: a fixed `<div>` 96×96, pointer-events none, transform updated via `gsap.quickTo(el, 'x', { duration: 0.4, ease: 'expo.out' })` (and same for y). On hover of `[data-cursor="enter"]` swap inner SVG to "ENTER" pill.
5. FLIP card → detail: `const state = Flip.getState(card); router.push(...); useGSAP(() => Flip.from(state, { duration: 0.7, ease: 'power3.inOut', absolute: true })`.
6. Breakpoints: lg ≥1280, md 768–1279 (rail switches to native horizontal `overflow-x: auto; scroll-snap-type: x mandatory`), sm <768 disables ScrollSmoother and the custom cursor.
7. Performance: `will-change: transform` only on the rail and active card during scroll; remove via `onScrollEnd`. Convert hero MP4 to HEVC (Safari) + AV1 (Chrome) with `<source type>` ladder; cap bitrate at 6Mbps.

WHY IT WORKS: The mix-blend-mode wordmark over live footage creates instant visual hierarchy without needing color contrast — the eye locks on the typographic mass first because it perpetually self-corrects against whatever is behind it. The pinned horizontal rail abuses Hick's Law: 12 case studies become a single linear decision instead of a paginated grid. The custom cursor is a Fitts's Law cheat — the giant arrow makes target acquisition feel weightless even though hit areas are unchanged.

GOOD: The blend-mode hero and FLIP card transitions are best-in-class, and the entire site holds 60fps on mid-range MacBooks. BAD: Horizontal scroll-jacking is hostile on trackpads with momentum scrolling and unusable for keyboard/screen-reader users — there is no skip-to-content for the rail, and `prefers-reduced-motion` is not honored.

CLEAN APPLICATION: Apply this to Clean's landing page hero by replacing the static "MCP proxy + context layer" tagline with a full-bleed dark canvas of agent traces (live JSON streaming through the proxy, rendered as a typographic waterfall) and overlaying the wordmark "CLEAN" in Söhne Breit at 14vw with `mix-blend-mode: difference`. Below the fold, replicate the pinned horizontal rail as a "How it works" sequence — 5 cards (Connect → Proxy → Compress → Route → Audit), each pinned for ~80vh of scroll, FLIP-animating into a detail panel that shows actual TOON-format payload diff (before/after token count). Use the diagonal cursor only on the rail; default cursor everywhere else for accessibility.

## Detroit Paris — https://detroit.paris/ (Awwwards SOTD + Developer Award, Apr 28 2026)

WHAT IT DOES: An AI production house's portfolio rendered as an editorial, monochrome (#000000 / #FFFFFF) reading experience. The hero is a marquee of 14 client logos auto-scrolling left at ~40px/sec via a duplicated track and `transform: translate3d` (no GSAP needed). Project tiles use `clip-path: polygon(...)` reveals on enter, with images that slow-zoom from `scale(1.18)` to `scale(1)` across 1.6s `cubic-bezier(0.16, 1, 0.3, 1)`. The signature move is a draggable, infinite-loop vertical reel of campaign films — built on GSAP Observer + a modular wrap function so dragging past the last clip teleports back to the first with no visible seam. Cursor follows hover targets with magnetic attraction (radius 80px, lerp 0.25).

PACKAGES:
- webflow (host)
- gsap@3.13 with ScrollTrigger, Observer, Draggable, InertiaPlugin, SplitText (Club)
- swiper@11 (free-mode + virtual slides)
- lenis@1.1
- splide-extension-intersection (optional, used as fallback)
- locomotive-scroll@5 (legacy, replaced by Lenis on mobile)
- pace-js (loader)
- self-hosted ABC Diatype + ABC Diatype Mono via @font-face

IMPLEMENTATION:
1. Webflow shell with custom `<script>` in Site Settings; all GSAP timelines live in `Footer › Before </body>` to ensure CMS items are in DOM.
2. Logo marquee:
   ```css
   .marquee-track { display: flex; gap: 96px; animation: scroll 38s linear infinite; }
   @keyframes scroll { to { transform: translateX(-50%); } }
   ```
   Track contains the logo list rendered twice. Pause on `:hover` via `animation-play-state: paused`.
3. Tile reveal:
   ```js
   gsap.utils.toArray('[data-reveal]').forEach((el) => {
     gsap.fromTo(el,
       { clipPath: 'inset(100% 0 0 0)' },
       { clipPath: 'inset(0% 0 0 0)', duration: 1.6, ease: 'expo.out',
         scrollTrigger: { trigger: el, start: 'top 85%', once: true } });
   });
   ```
4. Infinite vertical reel: build with `gsap.utils.wrap(0, items.length)` + Observer.
   ```js
   const wrap = gsap.utils.wrap(-items.length, items.length);
   Observer.create({
     target: reel, type: 'wheel,touch,pointer', wheelSpeed: -1,
     onChangeY: (self) => snap(wrap(currentIndex + Math.sign(self.deltaY))),
   });
   ```
   Each item is `position: absolute` with `transform: translateY(...)` driven by an animated index variable and `gsap.to` (ease `power3.inOut`, duration 0.9).
5. Magnetic cursor: track mouse globally; on hover of `[data-magnet]`, lerp the element's translate toward `(mouse - elCenter) * 0.35` with `gsap.quickTo` (duration 0.6, ease `expo.out`); reset on mouseleave.
6. Typography rule: H1 = ABC Diatype 900, `clamp(56px, 9vw, 180px)`, `letter-spacing: -0.05em`, `line-height: 0.92`. Body = ABC Diatype Mono 400, 14px, `line-height: 1.5`, uppercase tracking 0.04em for labels.
7. Performance: lazy-load `<video preload="metadata" muted loop>` with IntersectionObserver, switching `src` only when within 200px of viewport.

WHY IT WORKS: Pure black/white palette forces the eye to read motion and typography as the only signals — no color noise. The infinite reel exploits the Zeigarnik effect (incomplete loops keep attention) while the magnetic cursor creates a subtle "you found something" reward each hover. Editorial mono type signals craft over flash, which matches the AI-production positioning (the work, not the hype).

GOOD: Typography hierarchy and the seamless reel are textbook. BAD: Pure 1.0 contrast all-caps body copy fatigues at >2 paragraphs; mobile drag inertia overshoots and feels slippery on iOS Safari due to passive listener mismatch.

CLEAN APPLICATION: Apply this to Clean's "Customers" / "Built for" social-proof section by ditching the standard logo grid and instead building a paused-on-hover marquee of customer/partner logos in solid white on Clean's near-black (#0A0A0A), with logos rendered as inline SVG at 40px height. Then on the homepage's "What developers ship with Clean" section, deploy the infinite vertical reel with 8 mini case studies (each one a 30-second muted MP4 of a real agent run through the proxy). Type system: switch heavy display to ABC Diatype 900 at clamp(56px, 9vw, 180px) and standardize all mono labels (token counts, latency stats, route names) to a single weight at 14px / 0.04em.

## Oryzo AI — https://oryzo.ai/ (Awwwards SOTD by Lusion, Apr 22 2026 — Codrops feature Apr 13)

WHAT IT DOES: A satirical AI-era product launch for a fictional cork coaster, executed at fashion-campaign production quality. The hero is a fixed Three.js canvas with the coaster as a baked-PBR mesh lit by an HDRI, sitting on a procedurally-displaced cork-grain plane. Scroll progress drives a single GSAP master timeline that orchestrates camera dollies (FOV 35→22), mesh rotations, post-processing exposure ramps (UnrealBloom 0.4→1.1), and DOM word-stagger reveals — synchronized via ScrollTrigger scrub. A WebGL-to-DOM transition swaps the baked mesh for a 2D photograph at the section boundary, masked by a grow-from-center radial clip-path so it looks like one continuous shot. Footer has interactive GPU particles repelled by the cursor (10k instanced points on a custom shader).

PACKAGES:
- three@0.166
- @react-three/fiber@9 (with WebGPU async gl prop)
- @react-three/drei@10 (Environment, useGLTF, MeshTransmissionMaterial, useScroll)
- @react-three/postprocessing@3 (Bloom, ChromaticAberration, Vignette)
- three/examples/jsm/postprocessing or `postprocessing@6.36` (UnrealBloomPass)
- gsap@3.13 + ScrollTrigger, Observer, SplitText, CustomEase (Club)
- next@15 + react@19
- lenis@1.1
- leva@0.10 (dev-only HDRI/exposure tuning)
- meshline@3.3 (footer particle trails)
- glslify-loader (custom shader chunks)
- ABC Whyte Inktrap + ABC Whyte Mono

IMPLEMENTATION:
1. R3F Canvas: `<Canvas dpr={[1, 2]} gl={async (canvas) => { const r = new WebGPURenderer({ canvas, antialias: true }); await r.init(); return r; }} camera={{ fov: 35, position: [0, 0.6, 4] }}>`. Wrap with `<Suspense fallback={<Loader />}>`.
2. Cork coaster: load a 0.8MB Draco-compressed GLB; assign a custom NodeMaterial built in TSL with `texture(corkAlbedo)`, `oneMinus(roughness).mul(0.4)`, and a `noise.fbm()` displacement on the side bevel (0.003 amplitude). Bake AO at 1024². HDRI = Poly Haven `studio_small_03_1k.hdr` (rotation `[0, Math.PI / 4, 0]`, intensity 1.2).
3. Master timeline:
   ```js
   const tl = gsap.timeline({ scrollTrigger: { trigger: '#scene', start: 'top top', end: '+=4000', scrub: 0.8, pin: true } });
   tl.to(camera.position, { z: 2.6, y: 0.2 }, 0)
     .to(camera, { fov: 22, onUpdate: () => camera.updateProjectionMatrix() }, 0)
     .to(coaster.rotation, { y: Math.PI * 1.4, ease: CustomEase.create('coaster', 'M0,0 C0.4,0 0.2,1 1,1') }, 0)
     .to(bloomPass, { strength: 1.1 }, 0.3)
     .to('.headline-line', { yPercent: -100, stagger: 0.08, ease: 'expo.inOut' }, 0.1);
   ```
4. WebGL→DOM section bridge: at the section boundary, snapshot the framebuffer to a `<canvas>` element matched to the DOM image's bounding box, fade WebGL canvas opacity to 0 over 0.3s while the DOM photo's `clip-path: circle(0% at 50% 50%)` animates to `circle(140% at 50% 50%)` with `cubic-bezier(0.85, 0, 0.15, 1)`.
5. Footer particles: GPGPU compute shader (TSL `Fn`) — 10,000 particles in two FBOs ping-ponging position/velocity. Force = `attractionToHome - repulseFromCursor` where repulseFromCursor uses `smoothstep(0.0, 0.4, dist) * -2.5`. Render as `instancedMesh` of 0.008-radius spheres with vertex color from velocity magnitude.
6. Audio: a 12-second Web Audio crossfade swelling on the hero pin, ducking on scroll-out (gain 0.0→0.6→0.0). Muted by default; unlock on first user interaction.
7. Typography: hero = ABC Whyte Inktrap 900, clamp(72px, 12vw, 220px); meta labels = ABC Whyte Mono 500, 12px, uppercase, `letter-spacing: 0.12em`.
8. Performance: cap DPR at 2, `powerPreference: 'high-performance'`, dispose meshes/materials on route change, gate post-processing chain behind `prefers-reduced-motion` (also disable scrub pin).

WHY IT WORKS: Treating an absurd object with full cinematic-launch grammar weaponizes incongruity — the user lingers because the gap between "cork coaster" and "Apple-keynote treatment" is itself the joke. The single scroll-driven master timeline gives the entire page one breathing rhythm (pacing principle: one tempo per surface) and the WebGL-to-DOM masked transition removes the seam between "tech demo" and "marketing site" that breaks immersion on most R3F sites.

GOOD: Best-in-class WebGL/DOM choreography and a master timeline that genuinely feels like cinematography. BAD: ~8MB initial JS payload (R3F + postprocessing + GLB + HDRI), TBT >900ms on mid-range Android — the bloom pass alone tanks Lighthouse, and there is no fallback for devices without WebGL2.

CLEAN APPLICATION: Apply this to Clean's product-demo section by treating the MCP proxy itself as the "hero object" — a Three.js scene where Clean's logo geometry sits at center and incoming agent requests appear as glowing particles flowing in from the left, getting "compressed" (visualized as the particle cloud collapsing into a smaller volume) at the proxy node, and exiting right as fewer, denser particles into model endpoints. Drive everything from a single ScrollTrigger master timeline (camera dolly + particle density ramp + DOM stat counters going from "12,400 tokens" → "2,140 tokens"). Use the WebGL-to-DOM masked transition at the section boundary to swap from the 3D scene to a real dashboard screenshot showing the same compression metric. Gate behind `prefers-reduced-motion` and ship a static SVG fallback for the under-WebGL2 long tail.

## Shader Development Studio — https://www.shader.se/ (Awwwards SOTD + Developer Award, Apr 21 2026)

WHAT IT DOES: A studio site framed as a forgotten 1987 corporate training tape, but every "scene" is rendered live in WebGPU via Three.js Shading Language (TSL). Sections cross-dissolve through full-screen render-target blends with VHS-style chromatic aberration, scanlines, and bloom. The grid system is a 12-col `display: grid` overlay anchored to viewport units, with monospace labels positioned at exact `grid-area` coordinates so the layout reads like a CAD drawing. Navigation is a left-side rail of section markers that pulse a 1-frame strobe on click; the body content is bullet-point manifesto copy in IBM Plex Mono with hard `line-height: 1.4`.

PACKAGES:
- three@0.166 (WebGPURenderer build)
- three/tsl (Three Shading Language)
- @react-three/fiber@9 (async gl init for WebGPURenderer)
- @react-three/postprocessing@3
- next@15 + react@19
- gsap@3.13 + ScrollTrigger, CustomEase
- lenis@1.1
- @ibm/plex (self-hosted) + Berthold Akzidenz-Grotesk Pro
- vercel/og (dynamic OG card)
- maath@0.10 (math utils for shader uniforms)

IMPLEMENTATION:
1. Renderer bootstrap (R3F):
   ```js
   <Canvas
     gl={async (canvas) => {
       const r = new WebGPURenderer({ canvas, antialias: true, forceWebGL: false });
       await r.init();
       return r;
     }}
     dpr={[1, 1.5]}
   />
   ```
   Detect WebGPU support; fall back to WebGL2 silently using `navigator.gpu ? WebGPURenderer : WebGLRenderer`.
2. TSL post-processing chain (renderer-agnostic):
   ```js
   import { pass, mrt, output, emissive, uv, time } from 'three/tsl';
   const scenePass = pass(scene, camera);
   const scanlines = scenePass.add( sin(uv().y.mul(800).add(time.mul(2))).mul(0.06) );
   const chroma = vec3(scenePass.r.shift(0.002, 0), scenePass.g, scenePass.b.shift(-0.002, 0));
   ```
   Apply `bloom(0.6, 0.3, 0.85)` from `three/addons/tsl/display/BloomNode`.
3. Section transitions: each "scene" is a separate `Group` that fades in via `materialOpacity` and a render-target swap. Cross-dissolve = blend two render targets with `mix(prev, current, t)` driven by a uniform set from a GSAP timeline.
4. Grid overlay (CSS):
   ```css
   .grid { position: fixed; inset: 0; display: grid; grid-template-columns: repeat(12, 1fr); gap: 16px; padding: 16px; pointer-events: none; }
   .grid .marker { font: 500 11px/1 'IBM Plex Mono'; color: rgba(255,255,255,0.55); text-transform: uppercase; letter-spacing: 0.08em; }
   ```
   Place markers by `grid-column` and absolute positioning within their column for "CAD" readouts (e.g., `00.142s`, `RT-09`, `CH/02`).
5. Manifesto copy: `<ul style={{ listStyle: 'none' }}>` with each `<li>` prefixed by a 1ch "▶" glyph rendered in `color: #FF5215` (the studio's accent orange). Body is `font: 400 14px/1.4 'IBM Plex Mono'`, max-width 56ch.
6. Strobe nav click:
   ```js
   gsap.fromTo(navItem, { backgroundColor: '#FFFFFF' }, { backgroundColor: 'transparent', duration: 0.18, ease: 'steps(2)' });
   ```
7. Performance: WebGPU compute shaders for any particle/grain pass; 1.5x DPR cap; switch to a noise-only static SVG hero on mobile (where WebGPU coverage is thin as of Apr 2026 — Safari iOS still flagged).

WHY IT WORKS: The retro CAD/training-video aesthetic communicates technical depth without resorting to the SaaS gradient cliché — it tells engineers "we know the metal under the abstraction." TSL renderer-agnosticism lets the same shader logic ship to the WebGL2 fallback, so the brand consistency holds across the long tail. The CAD-grid markers anchor the eye to deliberate coordinates instead of a fluid `flex` layout, enforcing a precision-engineering read.

GOOD: TSL pipeline + WebGPU is genuinely state-of-the-art for an agency site, and the grid system is unusually disciplined. BAD: WebGPU still has ~62% browser support globally (caniuse Apr 2026); the WebGL2 fallback path drops several post-passes, so "what you see" varies a lot by device — and the no-JS fallback is an empty page.

CLEAN APPLICATION: Apply this to Clean's developer documentation home (`docs.tryclean.ai`) and to the marketing-site's "Architecture" deep-link section. Replicate the 12-col CAD grid with monospace coordinate markers labeling each architectural component (proxy node, context layer, MCP registry, audit log). Build a small WebGPU/TSL hero animation visualizing TOON-format compression as a packed-bytes diagram — same TSL shader, both renderers — and use the strobe-on-click steps(2) easing for all docs nav transitions. This signals to the developer audience that Clean's craft level matches the infrastructure they're being asked to depend on.

## Aceternity UI — Login Form with Gradient (WebGL + Motion for React) — https://ui.aceternity.com/components/login-form-with-gradient (Free drop, Apr 2026)

WHAT IT DOES: A drop-in auth component where the form sits on a translucent card layered over a full-bleed WebGL gradient mesh. The mesh is a single fragment shader on a fullscreen quad — three radial gaussian blobs that animate via uniform `time` (0.0008 speed), summed and tone-mapped, producing a slow oil-on-water effect. The form itself uses `motion/react` (formerly framer-motion@12) for staggered field entrance and a button hover that morphs background-position on a 3-stop linear gradient. Inputs use Aceternity's "moving border" — an SVG-stroke animation around the input on focus.

PACKAGES:
- next@15 + react@19
- tailwindcss@3.4 (with `tailwind-merge` and `clsx`)
- motion@12 (the renamed framer-motion)
- ogl@1.0.11 OR three@0.166 (Aceternity uses a hand-rolled OGL setup for this one)
- @radix-ui/react-label, @radix-ui/react-checkbox
- lucide-react@0.383
- shadcn/ui (`button`, `input`) — installed via `npx shadcn@latest add button input`
- geist@1.3 (font)

IMPLEMENTATION:
1. Install: `npx aceternity-ui@latest add login-form-with-gradient`. This drops a `<LoginForm />` into `components/ui/` plus a `<GradientCanvas />`.
2. Gradient shader (fragment, GLSL ES 3.0):
   ```glsl
   uniform float uTime; uniform vec2 uResolution;
   void main(){
     vec2 uv = gl_FragCoord.xy / uResolution.xy;
     float t = uTime * 0.0008;
     vec2 c1 = vec2(0.5 + 0.3*sin(t),       0.5 + 0.3*cos(t*0.7));
     vec2 c2 = vec2(0.5 + 0.4*cos(t*1.3),   0.5 + 0.2*sin(t*0.9));
     vec2 c3 = vec2(0.5 + 0.25*sin(t*1.7),  0.5 + 0.35*cos(t*0.5));
     float b1 = exp(-12.0 * distance(uv, c1));
     float b2 = exp(-10.0 * distance(uv, c2));
     float b3 = exp(-14.0 * distance(uv, c3));
     vec3 col = b1 * vec3(0.45, 0.18, 0.95) + b2 * vec3(0.12, 0.6, 1.0) + b3 * vec3(0.95, 0.32, 0.62);
     gl_FragColor = vec4(pow(col, vec3(0.7)), 1.0);
   }
   ```
   Render to a fixed full-bleed `<canvas>` at `dpr: 1` (gradient doesn't need higher) with `mix-blend-mode: normal`, behind everything at `z-index: 0`.
3. Card: `<div class="relative z-10 max-w-md w-full p-8 rounded-2xl bg-white/8 backdrop-blur-xl border border-white/15 shadow-[0_8px_40px_rgba(0,0,0,0.45)]">`.
4. Field entrance:
   ```jsx
   <motion.div initial={{ opacity: 0, y: 12 }} animate={{ opacity: 1, y: 0 }}
               transition={{ delay: i * 0.08, duration: 0.45, ease: [0.16, 1, 0.3, 1] }}>
   ```
5. Moving-border input on focus: an SVG `<rect>` with `pathLength: 1`, animated via `motion.path` `strokeDashoffset` from 1 to 0 over 1.2s `easeInOut`, color stops orchestrated with `useMotionTemplate`.
6. CTA button: `bg-[linear-gradient(110deg,_#7C3AED_0%,_#3B82F6_50%,_#EC4899_100%)] bg-[length:200%_200%] bg-[position:0%_50%] hover:bg-[position:100%_50%] transition-[background-position] duration-[800ms] ease-[cubic-bezier(0.16,1,0.3,1)]`.
7. Reduced-motion: if `useReducedMotion()`, skip all field stagger and freeze the shader uniform `uTime` at 0.

WHY IT WORKS: A fragment-shader gradient gives the surface depth without literal 3D — the eye reads it as "this app has motion DNA" before the user touches anything. The translucent card on a saturated backdrop establishes Z-axis hierarchy via blur and contrast, not borders. Motion's [0.16, 1, 0.3, 1] (Aceternity's house easing) front-loads movement so fields settle quickly — the perceptual snap reduces login friction.

GOOD: Best-in-class as a reusable, ship-tonight auth template; works headless in any Tailwind app. BAD: Multiple gaussian-blob shaders running on a phone GPU at 60fps will burn battery — there's no `IntersectionObserver` to pause when off-screen — and saturated backgrounds at 0.85 contrast sometimes fail WCAG AA on the input placeholder text.

CLEAN APPLICATION: Apply this to Clean's signup/login flow at `app.tryclean.ai/login` and to the homepage's "Get an API key" CTA card. Use the same fragment-shader gradient but reskin the color triad to Clean's palette (deep teal #0C2E2E, electric mint #57F0B6, ink #0A0A0A) so the gradient reads as branded rather than the default Aceternity violet/blue/pink. Replace the moving-border SVG color with mint, drop the email input + magic-link CTA into the translucent card, and add a small mono `[ enter your email ]` placeholder in IBM Plex Mono so the auth surface visually matches the developer-tool typography of the rest of the site. Pause the shader uniform when the page is hidden (`document.visibilityState`) to spare laptop batteries.
