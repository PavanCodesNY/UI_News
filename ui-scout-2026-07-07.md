## MONOLOG — https://www.awwwards.com/sites/monolog

WHAT IT DOES: Studio marketing site for a brand agency targeting founder-led companies. Layout is a full-viewport slab-serif type stage with GSAP-driven scroll pinning that swaps hero "chapters" without navigating (each scroll section pins for ~120vh, cross-fades the typographic content, and advances a chapter counter in the corner). Cursor is a custom SVG blob that reacts to hover targets by inverting its fill and scaling to a larger radius; case-study thumbnails play a WebGL displacement transition on hover (`gl-transition`-style RGB shift + noise mask).

PACKAGES:
- next@15, react@19
- gsap@3.13 + `gsap/ScrollTrigger` + `gsap/Observer`
- @studio-freight/lenis@1.1 (smooth scroll)
- @studio-freight/hamo (viewport hooks)
- ogl@1.0 (WebGL image transitions; lighter than three.js for 2D)
- framer-motion@11 (page transitions only)
- clsx@2, tailwindcss@3.4
- fonts: `Söhne` (Klim) via Fontshare fallback `Neue Haas Grotesk Display`; slab display `Ohno Blazeface`

IMPLEMENTATION:
1. Root: `<Lenis root>` with `duration: 1.2, easing: t => Math.min(1, 1.001 - Math.pow(2, -10*t))`, wheelMultiplier 1, smoothWheel true.
2. Hero: `<section className="h-[220vh] relative">` containing a sticky `<div className="h-screen sticky top-0 flex items-end p-8">`. Inside sticky, absolute-positioned chapter divs stacked with `will-change: transform, opacity`.
3. Register ScrollTrigger with `pin: true, scrub: 0.6, start: "top top", end: "bottom bottom"`. Timeline steps: `.to(ch1, {opacity:0, y:-40, duration:0.5}).from(ch2, {opacity:0, y:40, duration:0.5}, "<")` and repeat for each chapter.
4. Chapter counter: `useTransform`-style via `ScrollTrigger.callback` writing `01 / 05` into a `tabular-nums` monospace slot at top-right (`Berkeley Mono` or `JetBrains Mono`).
5. Custom cursor: single `<div>` fixed at 0,0, translated with `gsap.quickTo(el, "x", {duration:0.35, ease:"power3"})` on `mousemove`. Hover state: `data-cursor="hover"` on links triggers `gsap.to(cursor, {scale:3, backgroundColor:"var(--fg)", mixBlendMode:"difference"})`.
6. Case cards: `<canvas>` per card sized to natural image dims. On hover, run OGL fragment shader that samples `uTexture` and a noise `uDisp`, mixed by `uProgress` animated 0→1 in 700ms with `power2.out`. Return to 0 on mouseleave.
7. Type: Display uses `font-variation-settings: "opsz" 96, "wght" 700` at 12–20vw with `line-height: 0.85` and `letter-spacing: -0.04em`. Body 15/22 in Söhne 400.
8. Palette: `--bg:#0E0E0E` `--fg:#F4F1EA` `--accent:#FF5B1F`. Never full white; always slightly warm off-white.
9. Responsive: below 900px kill the WebGL displacement, replace with static `<img>` + CSS `filter: contrast(1.05)`. Below 600px reduce chapter pin to 150vh and drop custom cursor entirely.
10. Perf: hydrate GSAP timelines inside `requestIdleCallback`; use `content-visibility: auto` on out-of-view sections.

WHY IT WORKS: Editorial slab-serif typography acts as the primary compositional element (Massimo Vignelli's "type as landscape"), forcing the reader's eye across a wide horizontal reading path rather than a stacked vertical one, which triggers a "gallery" mental model — visitors evaluate the studio the way they'd evaluate a portfolio book. The scroll-pinned chapter swap replaces navigation with progressive disclosure, so the studio controls the argument order (positioning → proof → invitation) instead of letting the visitor jump around. Warm off-white on near-black paired with a single hot accent keeps contrast ratios above WCAG AAA on body copy while giving one focal color for CTAs — the eye can't miss the ask.

GOOD: Type-first structure means the site loads perceived-fast and reads well even before JS boots. Custom cursor is subtle, not disorienting.
BAD: Scroll-pinned chapters break "cmd-F" and screen readers. Mobile experience is 40% of what desktop delivers.

CLEAN APPLICATION: Apply this to Clean's landing page hero by replacing the current hero with a 220vh scroll-pinned "chapter stack" that reveals Clean's three positioning claims in sequence: (1) "Your agents talk to too many tools" — slab type, subtle animated diagram of tangled MCP arrows in background; (2) "Clean is the layer between them" — same slab type, arrows collapse into a single vertical spine labeled `clean://`; (3) "Ship agents that scale" — CTA appears. Use the same chapter-counter in top-right showing `01 / 03`. Keep everything monochrome except a single Clean-accent orange for the CTA.

## Hildén & Kaira — https://www.awwwards.com/sites/hilden-kaira

WHAT IT DOES: Social content agency site with a live-data case study loop. Hero is a two-color composition (deep navy + optic-white) using oversized Neue Haas Unica set at ~16vw. The differentiator: portfolio thumbnails display real-time performance stats (views, saves, shares) pulled from Airtable via Make.com automation, updated daily. Scroll model is standard GSAP + ScrollTrigger with a marquee row and reveal-on-scroll masks; the trick is the data ticker.

PACKAGES:
- webflow (native build; also portable to next.js)
- gsap@3.13, `gsap/ScrollTrigger`, `gsap/SplitText`
- @studio-freight/lenis
- airtable@0.12 (browser SDK) OR proxied via edge function
- swr@2.2 for cache/revalidate on client
- fonts: `Neue Haas Unica` (Linotype) + `PP Editorial Old` italic accent

IMPLEMENTATION:
1. Hero split-text reveal: `SplitText` breaks headline into words; `gsap.from(words, {yPercent:110, stagger:0.05, duration:0.9, ease:"expo.out"})`. Words sit inside `overflow:hidden` wrappers.
2. Marquee: single flex row duplicated, animated `x: -50%` with `gsap.to(row, {x:"-50%", duration:40, ease:"none", repeat:-1})`. Pause on hover with `paused` state via mouseenter.
3. Case card data ticker: each card fetches `GET /api/case/:id/metrics` (edge function that reads Airtable). Render numbers with `gsap.to({v:0}, {v: target, duration:2, onUpdate: () => el.textContent = Math.round(this.v).toLocaleString()})`. Trigger on ScrollTrigger `onEnter`.
4. Airtable → Make → website: cron in Make.com fires every 6h, hits each social API (Instagram, TikTok, YouTube), writes back to Airtable. Website reads Airtable through cached edge function (`revalidate: 3600`).
5. Palette: `--ink:#0A1B2E` `--paper:#F5F2EC`. NO greys, NO tints. Every UI element uses one of these two.
6. Reveal masks: sections wrapped in `<div style="clip-path: inset(0 0 100% 0)">`, animated to `inset(0 0 0 0)` over 1.1s `expo.out` on ScrollTrigger enter.
7. Nav: sticky top strip that inverts colors (ink↔paper) when it crosses a section with the opposite bg — use `IntersectionObserver` watching `data-theme="light|dark"` on sections.
8. Cursor: none. Site trusts native cursor.

WHY IT WORKS: Two-color constraint forces every design decision to earn its place — there's no way to hide behind a gradient or a photo. The live data ticker is the actual message: "our work performs, and we're so confident we'll show you the numbers as of today." That converts the portfolio from a static claim into ongoing proof, which is exactly what a content agency needs. Type-scale contrast (16vw hero next to 14px caption) creates strong hierarchy without adding colors.

GOOD: Live data is a moat — competitors can copy the layout but not the trust signal. Two-color palette ages well.
BAD: Airtable + Make is a fragile pipeline; a rate limit break silently shows stale numbers with no fallback UI.

CLEAN APPLICATION: Apply this to Clean's product demo section by replacing the static "how it works" diagram with a live counter surface that pulls actual anonymized aggregate metrics from Clean's own proxy: "1.2M tool calls routed today," "43ms p50 latency," "312 agents connected." Update every 5 minutes via edge function reading from Clean's own analytics. Two-color the entire section — `--ink` for text/lines, `--paper` for bg — and put a single Clean orange only on the counter digits. This turns Clean's landing page from "we claim to be infrastructure" into "we ARE infrastructure — watch it work."

## Radian — https://www.rideradian.com

WHAT IT DOES: Product launch site for an electric enduro motorbike with an interactive configurator. Hero is a full-bleed video loop with a fixed CTA that transitions to a scroll-driven 3D-ish orbit of the bike (actually a pre-rendered image sequence, not real WebGL — 120 frames scrubbed by scroll). Configurator is a state machine over color × wheel × luggage variants that live-swaps compositor layers (base bike PNG + tinted color mask PNG + accessory overlay PNGs), preloaded via a service worker. Uses BARBA.js for SPA-style route transitions between marketing pages.

PACKAGES:
- webflow
- gsap@3.13, `gsap/ScrollTrigger`
- @barba/core@2.10 + @barba/prefetch
- lenis
- lil-gui@0.19 (dev only)
- workbox-sw@7 for image sequence pre-cache
- fonts: custom industrial sans (feels like `PP Neue Machina` or `Söhne Mono` for HUD numerics)

IMPLEMENTATION:
1. Image sequence scrub: pre-render 120 JPEG frames at 1600×900, ~40KB each after mozjpeg (~5MB total). Load into an off-screen canvas via `Promise.all(imgs.map(load))`. On scroll, `ScrollTrigger` maps progress 0–1 to frame index; draw to `<canvas>` full-viewport.
2. Frame draw: use `ctx.drawImage(images[idx], 0,0, canvas.width, canvas.height)`. Debounce with `requestAnimationFrame`.
3. Configurator state: `{color: 'jet', wheels: 'enduro', luggage: 'none'}`. Each option is a `<button>` toggling state. Render is a stack of `<img>` layers with `mix-blend-mode: multiply` for the color tint mask.
4. Preload strategy: on `pointerenter` of any config option, preload the corresponding image layer with `<link rel="prefetch">`. Full config permutation set (~30 images) is precached by service worker on first visit.
5. BARBA transitions: intercept `<a>` clicks, run `gsap.to(container, {opacity:0, y:20, duration:0.4})`, fetch next page HTML, replace `<main>`, run `gsap.from(new, {opacity:0, y:-20, duration:0.5})`. Preserve nav.
6. HUD numerics: spec numbers (torque, range) animate on view with `CountUp` — `gsap.to({v:0}, {v:70, duration:1.6, snap:{v:1}, onUpdate: () => el.textContent = this.targets()[0].v + 'HP'})`.
7. Video loop hero: `<video autoplay muted loop playsinline poster>` with `object-fit: cover`, muted for autoplay policy compliance, poster is the first frame of the image sequence.
8. Responsive: image sequence downgrades to a single hero video on `matchMedia('(max-width: 900px)')`.

WHY IT WORKS: Scroll-scrubbed image sequence delivers the perceptual payoff of 3D (I control the object with my scroll) at ~1% of the WebGL/GLB cost, works on iPhones with a dead battery, and never breaks on Safari. The configurator's state machine over composited PNGs is deliberately old-school — instantaneous variant swap without a WebGL shader compile — which respects the "I want to pick a color and see it now" impatience of a buyer. HUD numerics reinforce the "spec-first, engineered product" positioning without needing hero copy to say it.

GOOD: Extremely performant. Configurator interaction is sub-16ms per swap. Print-quality product photography does more heavy lifting than any 3D would.
BAD: Image sequence total download is real bandwidth (~5MB) and there's no meaningful low-res tier for slow networks — the site just delays.

CLEAN APPLICATION: Apply this to Clean's product demo section by building a scroll-scrubbed image sequence showing an MCP request travelling through Clean's proxy — 60 frames rendered from Figma showing the request packet entering, being TOON-encoded, routed, and responded to. Below the sequence, a state-machine configurator lets developers toggle: transport (SSE / stdio / HTTP), format (JSON / TOON), auth (Bearer / OAuth / API key) — and the diagram overlay updates instantly via composited SVG layers. No WebGL, no runtime cost. Pin the section for ~150vh and let the scroll drive the whole thing.

## Wembi — https://www.awwwards.com/sites/wembi

WHAT IT DOES: B2B SaaS marketing site with an aggressively minimal two-color palette (mint-lime #beff8b + slate-blue #9dbbc5) and hard geometric grid. Nuxt 3 SSR, Craft CMS backend, GSAP for scroll reveals. Layout is a 12-col CSS grid with intentional "gaps" — sections don't fill horizontally, leaving asymmetric whitespace that creates a "notebook margin" reading rhythm. Zero photography; every visual is a diagram, chart, or type composition.

PACKAGES:
- nuxt@3.13, vue@3.4
- gsap@3.13 with `ScrollTrigger`, `SplitText`
- @vueuse/motion@2 (declarative motion for Vue)
- tailwindcss@3.4 with custom `screens` and `spacing` scale
- craft CMS backend via GraphQL (`@nuxtjs/apollo`)
- fonts: `NB International Pro` + `Söhne Mono` for spec numbers
- lucide-vue-next for iconography

IMPLEMENTATION:
1. Grid: `<div class="grid grid-cols-12 gap-x-6">` at 1280px+, sections use `col-start-2 col-span-8` or `col-start-3 col-span-6` — never full-width text.
2. Palette hard-lock: Tailwind config has ONLY `mint` (#beff8b), `slate` (#9dbbc5), `ink` (#0A0A0A), `paper` (#FAFAFA). No greys. Buttons use `bg-ink text-mint`.
3. Diagrams: pure SVG, stroke-only, `stroke-width: 1.25` at all sizes. Animate on enter with `stroke-dasharray` trick: set `stroke-dasharray: pathLength; stroke-dashoffset: pathLength;` then animate offset to 0 over 1.5s.
4. Reveal pattern: SplitText breaks paragraphs into lines; `gsap.from(lines, {yPercent:100, stagger:0.04, duration:0.8, ease:"quart.out"})` inside `overflow:hidden` wrappers. Same easing everywhere for coherence.
5. Section headers: comically large numeric labels (`§ 01 — Overview`) in Söhne Mono at 11px next to display headline at 96px. Rhythm comes from that 8x scale gap.
6. No cursor customization. No page transitions. The site is loud on typography, quiet on motion.
7. Performance: Nuxt SSR + `<NuxtImg>` with AVIF, defer GSAP to `onMounted` with `nextTick`.
8. Accessibility: reveals respect `prefers-reduced-motion: reduce` — set `gsap.set` to end state directly, skip tweens.

WHY IT WORKS: The mint-lime × slate-blue combo is unusual in B2B SaaS (which defaults to blue-and-white or purple-and-white), so the palette itself does brand differentiation before a single word is read. The margin whitespace triggers a "curated / considered" perception — nothing is jammed edge-to-edge, so the eye reads the site as thoughtful. Stroke-only SVG diagrams look native to a technical audience without the corporate-illustration cringe. The rhythm of tiny-mono-label + huge-display-headline uses scale contrast (Tufte) to establish hierarchy instead of color, which stays legible in dark mode, print, and screenshots.

GOOD: Palette is memorable, ownable, and hard for competitors to copy without looking derivative.
BAD: Two colors mean tone-of-voice has to carry warmth; the site can read as clinical if the copy isn't excellent.

CLEAN APPLICATION: Apply this to Clean's docs navigation and pricing page by adopting a 2-color lock: pick one Clean-accent (orange or electric-blue) + ink + paper only. Build the pricing tiers on a 12-col grid, each tier in `col-span-4` with intentional gap columns between them. Number every section with `§ 01`, `§ 02`, `§ 03` in Söhne Mono at 11px above the tier name at 48px. Replace all product illustrations with stroke-only SVG diagrams (`stroke-width: 1.25`) animated with the `stroke-dasharray` reveal on scroll enter. This makes the pricing page feel like technical documentation, not a marketing page — which is what developers trust.

## Aceternity UI — Background Ripple Effect — https://ui.aceternity.com/components/background-ripple-effect

WHAT IT DOES: Full-viewport background composed of a CSS grid of interactive cells (~40×24 grid of `<button>` divs). Each cell has a subtle border and a keyframed opacity animation. On click at any cell, cells within a radius fire a staggered opacity/scale animation outward — creating a visible ripple through the grid — with cells further from the origin animating later and with lower peak intensity. Also drops a persistent low-opacity trail so the ripple leaves a soft afterglow.

PACKAGES:
- next@15 or react@19
- tailwindcss@3.4
- framer-motion@11 (only for the ripple choreography; the base grid is CSS)
- clsx@2
- @radix-ui primitives (optional, only if wrapping into a card)

IMPLEMENTATION:
1. Container: `<div className="relative h-screen w-screen">`.
2. Grid: `<div className="absolute inset-0 grid" style={{gridTemplateColumns:'repeat(40, 1fr)', gridTemplateRows:'repeat(24, 1fr)'}}>`.
3. Cells: map `Array.from({length: 40*24})`. Each cell: `<div key={i} onClick={()=>ripple(i)} className="border-[0.5px] border-neutral-800/50 hover:bg-neutral-800/50 transition-colors" data-idx={i} />`.
4. Ripple math: on click, compute cell's `row, col`. For every cell, compute Chebyshev distance = `Math.max(Math.abs(row-r0), Math.abs(col-c0))`. Fire animation with `delay: dist * 40ms`, `duration: 400ms`, keyframe `opacity: [0.15, 1, 0.15]`.
5. Use `framer-motion`'s `animate()` imperative API to trigger per-cell:
   ```js
   animate(cellRef.current, {opacity: [0.15, 1, 0.15]}, {delay: dist*0.04, duration: 0.4, ease: [0.2,0.8,0.2,1]})
   ```
6. Afterglow: after ripple, set the persistent opacity to `0.15 + Math.max(0, 0.4 - dist*0.02)` so nearby cells stay slightly brighter. Decay over 5s back to baseline.
7. Baseline breathing: entire grid animates `opacity: 0.9 → 1 → 0.9` on a 6s loop via CSS `@keyframes`.
8. Content sits above grid: `<div className="relative z-10 pointer-events-none">` with `pointer-events-auto` on interactive children so ripple stays clickable through the transparent hero.
9. Perf: cap to ~1000 cells max. Use `will-change: opacity` on all cells. Debounce clicks to 150ms.

WHY IT WORKS: The grid is a physical metaphor for infrastructure — a control surface with discrete cells that respond to touch. Ripple animation converts a passive visual into a play surface, which measurably increases time-on-page (people click). The pattern signals "responsive system" without needing to say it. The staggered decay uses a natural power-law falloff (Chebyshev distance × constant delay) that reads as physically plausible, not scripted.

GOOD: 100% CSS + JS, zero images. Renders identically dark/light. Interaction is intrinsically motivating.
BAD: On low-end mobile GPUs, 1000 concurrent opacity tweens drop frames; the effect breaks perceived quality on the exact devices where perceived quality matters most.

CLEAN APPLICATION: Apply this to Clean's landing page hero background by using the ripple grid as a live signal-representation of MCP traffic. Wire the ripple origins to actual events from Clean's demo endpoint — every time a demo call routes through Clean, it fires a ripple at a deterministic cell derived from `hash(toolName)`. The hero copy sits on top; the background is literally showing that Clean is alive and routing. Constrain grid to 32×18 for perf, cap live ripples to 3/sec, degrade to static breathing grid on `matchMedia('(hover: none)')`. This gives the landing page a heartbeat.

## BUILD DIRECTIVE

TARGET: Clean's landing page hero (tryclean.ai/) — full redesign of above-the-fold.

APPROACH: Build a single-section scroll-pinned hero that combines three of today's patterns into one coherent story. Use MONOLOG's chapter-stack scroll pinning (220vh section with sticky viewport, three chapters cross-faded by ScrollTrigger) to sequence Clean's three positioning claims. Behind the type, use the Aceternity Background Ripple Effect as a live signal grid wired to actual (throttled) events from Clean's demo endpoint — every routed call fires a small ripple. Adopt Wembi's palette and type discipline: two-color hard-lock (ink `#0A0A0A` + paper `#F4F1EA`) plus a single Clean-orange accent for the CTA, oversized display type at 12vw, `Söhne Mono` for spec-numeric labels (`§ 01 — Proxy`, `p50 latency`, `tool count`). Use Hildén & Kaira's live-data ticker pattern to surface real aggregate metrics in the third chapter ("1.2M calls routed today, 43ms p50, 312 agents") so the visitor's last impression before scrolling past the fold is that Clean is real and running. No 3D, no video, no custom cursor.

STACK:
- next@15, react@19, typescript@5.6
- tailwindcss@3.4 with palette hard-lock (`ink`, `paper`, `accent`)
- gsap@3.13 + `gsap/ScrollTrigger` + `gsap/SplitText`
- @studio-freight/lenis@1.1
- framer-motion@11 (imperative `animate()` for ripple cells only)
- swr@2.2 (for live metric fetch)
- edge function: aggregate metrics from Clean's proxy, cached 60s
- fonts: `Söhne` (display + body), `Söhne Mono` (numerics), served self-hosted with `font-display: block`
- perf budget: <120KB JS above the fold, <5KB CSS, LCP <1.2s on 4G

REFERENCE URLS:
- https://www.awwwards.com/sites/monolog (chapter-stack scroll structure)
- https://www.awwwards.com/sites/wembi (palette + type + grid discipline)
- https://ui.aceternity.com/components/background-ripple-effect (ripple grid pattern)

KEY CONSTRAINTS: Clean sells developer infrastructure. Avoid: gradient meshes, 3D renders of abstract shapes, "AI" purple/pink, hero videos of stock footage, cursor blobs (already overexposed in dev-infra sites), Lottie animations, marketing-illustration-style SVG mascots. The result must feel closer to a technical spec sheet than a SaaS landing page — as if Stripe's docs and Vercel's ship page had a colder, more editorial child. Every animation must survive `prefers-reduced-motion: reduce` gracefully. Every metric shown must be real; no fake counters. If the ripple grid drops below 55fps on iPhone 12, disable it and ship a static breathing grid instead.
