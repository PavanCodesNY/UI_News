# UI Scout Report — 2026-07-10

## Brunello Cucinelli — AI E-com — https://www.awwwards.com/sites/brunello-cucinelli-ai-e-com

WHAT IT DOES: A luxury e-commerce experience where the entire chrome (nav, filters, product cards, modals) is rendered as frosted-glass panels layered over a full-bleed editorial media stage (hero video + still photography carousel). An embedded AI concierge (the Callimacus platform) sits as a floating pill in the bottom corner; when invoked it expands into a right-side drawer where natural-language queries return curated product grids that animate in with staggered opacity + Y-translate. Layout uses a 12-col CSS grid with generous top/bottom padding (>10vh) and typography-first hierarchy — serif display face at ~120px for hero, sans body at 15px. Awwwards SOTD 2026-07-09, Developer Award, built by makemepulse.

PACKAGES:
- next@^15 (App Router)
- react@^19
- tailwindcss@^4
- motion@^12 (formerly framer-motion) for panel transitions
- gsap@^3.13 + @gsap/react for hero timeline
- lenis@^1.3 for smooth scroll
- @radix-ui/react-dialog for the AI drawer primitive
- @vercel/ai SDK for concierge streaming
- next/font loading a licensed serif (e.g. Fraunces or GT Sectra) + neutral sans (Inter or Söhne)
- sharp for image optimization

IMPLEMENTATION:
Component tree: `<Shell>` → `<HeroStage>` (fixed 100vh, `position: sticky`) → `<ProductRail>` (horizontal scroll) → `<ConciergeDrawer>` (portaled). Glass surfaces: `backdrop-filter: blur(24px) saturate(140%)`, `background: color-mix(in oklch, white 8%, transparent)`, `border: 1px solid color-mix(in oklch, white 12%, transparent)`, `border-radius: 20px`. Hero video: `<video autoplay muted playsinline loop>` at 1440x900 with AV1 + h.264 sources, `object-fit: cover`, poster preloaded. GSAP hero timeline: `duration: 1.8, ease: "expo.out"`, staggers text lines by 0.08s using SplitText. Concierge drawer: Radix Dialog with `data-state=open` transitioning `transform: translateX(0)` from `100%`, `transition: cubic-bezier(0.22, 1, 0.36, 1) 520ms`. Product grid enter: `stagger: { each: 0.05, from: "start" }`, `y: 24 → 0`, `opacity: 0 → 1`. Responsive: below 768px collapse to single-column, disable backdrop-blur on Safari iOS (perf), reduce hero to 70vh. Perf: `content-visibility: auto` on off-screen rows, lazy-load AV1 with `<source>` chain, hydrate concierge only on first user interaction.

WHY IT WORKS: Frosted panels obey figure-ground separation — chrome recedes so photography leads, satisfying luxury's "product-as-hero" principle. Whitespace rhythm at 8/16/32/64px scale creates editorial cadence that feels magazine-authored, not templated. The AI pill's small footprint respects Fitts' Law without hijacking attention, matching the expectation that AI serves rather than sells.

GOOD: Backdrop-blur + oklch color-mix is the correct 2026 pattern — replaces the tired dark-mode-with-glow trope with a genuinely material feel.
BAD: Backdrop filters kill mobile Safari framerates below iPhone 13; gracefully degrades poorly on mid-range Android.

CLEAN APPLICATION: Apply this to Clean's landing page hero by replacing the current density with a fixed 100vh sticky stage that plays a looping product screencapture (Huginn auto-profiling a lead in real time) at 60% opacity behind a single frosted-glass card containing the H1 + primary CTA. The AI concierge pattern maps directly to a floating "Try Huginn" pill bottom-right that expands into a right-drawer sandbox where the visitor types a real ICP query and watches Clean stream a preview of matched accounts — the product demo becomes the hero without a scroll.

## Julien Calot ("FLOT NOIR PRO") — https://www.awwwards.com/sites/julien-calot

WHAT IT DOES: A WebGL-native portfolio built in Webflow + custom shaders. Homepage is a single infinite-scroll ticker where portfolio thumbnails render on a curved 3D plane that responds to cursor + wheel velocity — the plane deforms along the y-axis using a vertex shader tied to `scrollVelocity`. Thumbnails on hover expand into fullscreen case-study reels with a "warp" transition (RGB channel split + radial displacement). Custom 404 renders a distorted grid that the mouse pushes around like fluid. Awwwards SOTD 2026-07-08.

PACKAGES:
- webflow (host) + three@^0.170 loaded via Webflow custom code
- ogl@^1 (lightweight alternative to Three, likely what they use given payload size)
- gsap@^3.13 + ScrollTrigger + Observer
- lenis@^1.3
- glsl-noise for shader noise
- shader files as raw strings (or vite-plugin-glsl if bundled)
- @studio-freight/hamo for useMouse / velocity hooks

IMPLEMENTATION:
Root: `<canvas>` fixed at `z-index: 1` covering viewport. Three.js scene with orthographic camera, a single `PlaneGeometry(2, 2, 32, 32)` instanced N times (once per portfolio item). Vertex shader: displaces `position.y` by `sin(uv.x * PI + uTime) * uScrollVel * 0.4`. Fragment shader: samples image texture with RGB offset `vec2(uRgbShift * 0.01, 0.0)` scaled by scroll velocity. JS loop: `useFrame` updates `uScrollVel = lerp(uScrollVel, lenis.velocity, 0.08)`. Hover transition: GSAP tweens `uHoverProgress: 0 → 1` over `1.1s` with `ease: "expo.inOut"`, fragment shader mixes between thumbnail and video texture. Case study reels: `HTMLVideoElement` used as `VideoTexture`, played only when `uHoverProgress > 0.5`. Infinite scroll: Lenis with `infinite: true`, positions calculated `mod(y, totalHeight)`. Text overlay: DOM `<h2>` positioned absolutely with `mix-blend-mode: difference` so it inverts against any thumbnail. Responsive: below 900px kill WebGL, fall back to CSS `scroll-snap-y` with `object-fit: cover` videos.

WHY IT WORKS: Motion semantics — scroll velocity mapping to deformation makes the page feel physically inertial, which reads as "crafted" to visitors who've seen a thousand static grids. RGB split on hover is a Barnum-signal for "high production value" borrowed from film titles. Difference blend text solves contrast without a dark scrim, preserving image integrity.

GOOD: The scroll-velocity → shader displacement is the single most transferable trick — cheap in shader ops, huge in perceived quality.
BAD: 100% inaccessible — no keyboard nav, no `prefers-reduced-motion` fallback, thumbnails have no alt text in the accessibility tree because they live on canvas.

CLEAN APPLICATION: Apply this to Clean's product demo section (mid-page, below hero) by rendering the "how it works" visualization as a horizontal WebGL rail where cards representing each Huginn step (Discover → Enrich → Profile → Outreach) are 3D planes that deform on scroll velocity. Use a stripped-down shader — no RGB split (too flashy for infra) — but keep the velocity-driven Y displacement. On hover, planes expand into a live mini-demo of that step. Fallback to static cards on mobile and with `prefers-reduced-motion`.

## MONOLOG — https://www.awwwards.com/sites/monolog

WHAT IT DOES: A studio site positioned for "founder-led brands." The interaction model is restraint — a fixed left-side vertical nav with monospace section labels, a right pane that swaps content via GSAP timeline crossfades (opacity + 8px Y translate, 480ms), and hover states that trigger a single 200ms underline weight change with no color shift. Typography is a variable serif at three weights (200/500/800) with tight tracking (-0.02em) on display. Case studies use 12-col grids with `grid-column-gap: 2vw`, images cropped to golden ratio. Awwwards SOTD 2026-07-07, Developer Award.

PACKAGES:
- next@^15 + react@^19
- tailwindcss@^4
- gsap@^3.13 + @gsap/react (core + ScrollTrigger + SplitText)
- lenis@^1.3
- next/font — variable serif (PP Editorial New or Söhne Breit) + monospace (Berkeley Mono or JetBrains Mono)
- No motion library beyond GSAP
- @vercel/og for OG image generation

IMPLEMENTATION:
Layout: `grid-template-columns: 240px 1fr` desktop, single-column mobile. Left nav is `position: sticky, top: 0, height: 100vh`, items are `text-transform: uppercase, letter-spacing: 0.08em, font-size: 12px, font-family: mono`. Right pane uses GSAP for section transitions triggered on nav click: `gsap.timeline().to(current, {opacity: 0, y: -8, duration: 0.32, ease: "power2.in"}).to(next, {opacity: 1, y: 0, duration: 0.48, ease: "power2.out"}, "-=0.1")`. Underline hover: pure CSS `text-decoration-thickness: 1px → 2px`, `transition: text-decoration-thickness 200ms cubic-bezier(0.4, 0, 0.2, 1)`. Scroll: Lenis `duration: 1.2, easing: (t) => 1 - Math.pow(1 - t, 3)`. Grid: `display: grid, grid-template-columns: repeat(12, 1fr), gap: 2vw` with images `aspect-ratio: 1.618 / 1`. Cursor: custom `mix-blend-mode: difference` circle 24px, `transform: translate(x, y)` with `requestAnimationFrame` lerp at 0.14. Responsive: kill sticky nav below 900px, convert to top-slide-in drawer triggered by hamburger.

WHY IT WORKS: This is the "quiet luxury" of web design — every interaction is a single degree of freedom (only opacity, only underline weight, only one color). That restraint is the brand signal. Gestalt similarity groups the monospace nav labels into a single visual unit that reads as "system," building credibility for a studio selling founder brands.

GOOD: The mono-serif pairing at extreme weight contrast is a currently-trending, still-effective device that costs nothing to ship.
BAD: Restraint reads as "empty" if content isn't dense — junior teams copy the aesthetic and end up with beautiful nothing.

CLEAN APPLICATION: Apply this to Clean's docs navigation and pricing page by adopting the sticky left-mono-nav pattern — replace the current top-bar-heavy structure with a `240px` sticky nav using monospace labels for sections (Product, Pricing, Docs, Changelog), and use GSAP crossfade transitions between docs pages instead of page reloads. This gives Clean an infrastructure-grade "IDE-adjacent" feel that developer buyers subconsciously trust.

## Radian — https://www.awwwards.com/sites/radian

WHAT IT DOES: A product launch site for an electric off-road motorbike. Above the fold: a fullscreen GLTF model of the bike rotating on a slow 12s loop, ambient occlusion baked in, three-point studio lighting. Scroll triggers ScrollTrigger-driven camera moves that reframe the bike (front → 3/4 → detail on shock absorber → detail on battery), synced to typography that scrubs through spec numbers. Bottom section: a custom configurator where color/wheel/spec choices update the model in real time via uniform buffers. Awwwards SOTD 2026-07-05, Developer Award.

PACKAGES:
- three@^0.170
- @react-three/fiber@^8.17
- @react-three/drei@^9 (useGLTF, Environment, ContactShadows, PerformanceMonitor)
- @react-three/postprocessing (Bloom, SSAO, Vignette)
- gsap@^3.13 + ScrollTrigger
- leva (dev-only, stripped in prod)
- zustand for configurator state
- next@^15 + react@^19
- draco3d + meshopt-decoder for GLTF compression
- tailwindcss@^4

IMPLEMENTATION:
Canvas: `<Canvas dpr={[1, 1.5]} gl={{ antialias: true, alpha: true, powerPreference: "high-performance" }}>`. Load bike via `useGLTF('/bike-draco.glb')` — model must be Draco+Meshopt compressed to <4MB. Environment: `<Environment preset="studio" />` + one directional light at `[5, 8, 5]` intensity 1.2 casting `shadowMapSize=[2048, 2048]`. Post: `<EffectComposer><Bloom luminanceThreshold={0.9} intensity={0.4} /><Vignette eskil={false} offset={0.3} darkness={0.5} /></EffectComposer>`. Camera control: GSAP timeline attached to `ScrollTrigger` with `scrub: 1` (smooth follow scroll); each keyframe updates `camera.position` + `camera.lookAt` with `ease: "power2.inOut"`, `duration: 1`, snapping between 4 positions. Spec numbers: DOM overlay `position: fixed` with GSAP `CountUp` scrubbed through same timeline. Configurator: zustand store dispatches color to a material uniform `uColor` — no re-render, direct GPU update. Performance: `PerformanceMonitor` from drei drops `dpr` and disables post if fps < 45. Responsive: below 900px use a pre-rendered 60-frame image sequence instead of live WebGL, scrub the sequence with scroll.

WHY IT WORKS: Scroll-scrubbed camera choreography turns product marketing into cinematography — the user isn't reading specs, they're being shown the object as if in a controlled studio shoot. This satisfies the emotional purchase framework luxury goods rely on. Configurator delivers agency at the end (control loop closed), the classic "look → touch → own" retail journey compressed into one scroll.

GOOD: The image-sequence fallback for mobile is the correct engineering trade — 90% of the visual impact at 10% of the runtime cost.
BAD: 12MB total JS payload (Three + drei + postprocessing) means TTI on 4G is >4s; discovery-stage buyers on transit will bounce.

CLEAN APPLICATION: Apply this to Clean's product demo section by using scroll-scrubbed camera choreography over a 3D "data pipeline" visualization — camera moves through nodes representing Lead Discovery → Auto Profiling → Knowledge Base → Outreach, with spec numbers (accuracy %, latency ms, sources scanned) scrubbing in the DOM overlay. Do NOT use a bike; use abstract geometry (spheres, lines, particle flows). Use the mobile image-sequence fallback trick — pre-render a 90-frame WebM at 720p, scrub with scroll, ships in <2MB.

## NRG — Build Your Data Center — https://www.awwwards.com/sites/nrg-build-your-data-center

WHAT IT DOES: A B2B campaign site targeting hyperscalers. Structure is a linear "configure your infrastructure" scroll — each viewport-height section is a stateful step (Power → Land → Grid Access → Timeline) with a persistent right-side sticky summary card that updates as the user scrolls, showing running totals (MW capacity, acres, months to online). Data-viz sections use animated SVG charts (D3) that draw in on scroll enter. Palette is corporate but modern — a single accent color (electric cyan) against neutral grays, no gradients. Awwwards SOTD 2026-07-01, Developer Award.

PACKAGES:
- next@^15 + react@^19
- tailwindcss@^4
- d3@^7 (specifically d3-scale, d3-shape, d3-selection)
- gsap@^3.13 + ScrollTrigger
- motion@^12 for the sticky summary card micro-transitions
- zustand for scroll-step state
- @vercel/og for OG images
- mapbox-gl@^3 for the land-availability map section

IMPLEMENTATION:
Layout: `grid-template-columns: 1fr 380px` desktop; right column is `position: sticky, top: 5vh, height: 90vh`, contains a card with `background: color-mix(in oklch, white 4%, #0a0f14)`, `border: 1px solid #1c2733`. Each scroll section: `min-height: 100vh`, `display: flex, align-items: center`; ScrollTrigger fires `onEnter → zustand.setState({ step: N })`. Summary card values animate with motion `<motion.span>` using `spring: { stiffness: 120, damping: 20 }`. SVG charts: D3 `line()` generator, path `stroke-dasharray: length, length`, `stroke-dashoffset: length → 0` on scroll enter, `transition: stroke-dashoffset 1.6s cubic-bezier(0.22, 1, 0.36, 1)`. Mapbox: `style: 'mapbox://styles/mapbox/dark-v11'`, custom layer for available parcels rendered as GeoJSON polygons with `fill-opacity: 0.4`. Typography: Inter variable, headings `clamp(2.5rem, 5vw, 4.5rem)`, body 17px. Accent cyan `#00e5ff` used only for CTA + active step indicator. Responsive: sticky card collapses into a bottom-fixed bar showing single most-relevant metric.

WHY IT WORKS: The persistent summary card is a working-memory scaffold — the buyer doesn't have to hold running totals in their head, so they stay in exploratory mode longer, which correlates with intent capture. Restricting to a single accent color forces hierarchy through position and weight, giving the site the "data instrument" aesthetic that infrastructure buyers trust more than agency-lookbook designs.

GOOD: The sticky-configurator-with-live-summary pattern is criminally underused in B2B; it's the single biggest conversion mechanic for high-consideration purchases.
BAD: Mapbox adds 800KB and requires an API key ceiling — for a campaign site this is overkill; a static SVG of the US grid would deliver 95% of the value.

CLEAN APPLICATION: Apply this pattern EXACTLY to Clean's pricing page — replace the current three-column card layout with a linear scroll configurator where each viewport step asks a question (Team size? ICP breadth? Enrichment depth? Outreach volume?), and a sticky right-side card shows a running estimate (records/mo, $/mo, ROI multiple). Use the electric cyan accent for the primary CTA. This turns pricing from a comparison exercise into a self-guided sale.
