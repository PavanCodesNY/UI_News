## Wembi â€” https://www.awwwards.com/sites/wembi

WHAT IT DOES: Product marketing site for a virtual-twin performance platform, launched with a Nuxt.js + Craft CMS stack and Awwwards Developer Award (SOTD 2026-07-02). Layout is a vertical stack of full-viewport hero blocks where inline autoplay looping video assets act as both background and interactive hitboxes â€” the cursor drives a fluid mouse-follow blob (SDF-style) that reveals video hover states. Section transitions are GSAP-driven with pinned ScrollTrigger scenes; text runs use SplitText char-by-char stagger with a clip-path reveal from bottom.

PACKAGES:
- nuxt@^3.13
- vue@^3.4
- gsap@^3.12 (+ ScrollTrigger, SplitText, Observer)
- @studio-freight/lenis@^1.1 (smooth scroll)
- three@^0.166 (used only for the cursor blob shader, not full-scene)
- @vueuse/core
- tailwindcss@^3.4 with @tailwindcss/typography
- satori/next-font equivalent via @nuxtjs/google-fonts (Inter Tight + a mono display)
- video.js NOT used â€” raw `<video autoplay muted playsinline loop preload="metadata">` for lightness

IMPLEMENTATION:
- Root layout: `main[data-lenis-prevent-scroll]` wrapping a series of `section.h-screen.snap-start`. Lenis(`duration: 1.4, easing: t => Math.min(1, 1.001 - Math.pow(2, -10 * t))`).
- Each section is a `<HeroBlock>` component: `position: relative; overflow: hidden`. Background = `<video>` at `object-fit: cover; filter: brightness(0.7) contrast(1.05)`.
- Headlines: SplitText with `types: 'chars,words'`. On enter, `gsap.to(chars, { yPercent: 0, opacity: 1, stagger: 0.012, duration: 0.9, ease: 'expo.out' })`. Starting state `yPercent: 110, opacity: 0`.
- Section pin: `ScrollTrigger.create({ trigger: section, start: 'top top', end: '+=100%', pin: true, scrub: 0.5 })` â€” video timeline is tied to `progress` via `video.currentTime = progress * video.duration` for scrubbed frame-perfect storytelling.
- Cursor blob: full-screen `<canvas>` at `mix-blend-mode: difference`, single fragment shader draws a metaball at mouse XY with 40px falloff, lerped at `0.12` per frame for viscosity.
- Typography scale: display 96px/0.9 tracking -0.04em, body 18px/1.5. Palette: near-black `#0A0A0A`, off-white `#F4F1EA`, accent `#DDFF3A`.
- Breakpoints: 640/768/1024/1280/1536. Below 1024 the cursor blob unmounts; video scrubbing becomes autoplay loop.
- Perf: `will-change: transform` on pinned sections only during their trigger; `IntersectionObserver` pauses out-of-view videos; preload the next section's poster only.

WHY IT WORKS: Full-bleed video-per-section forces attention to a single message at a time â€” Miller's chunking taken to the extreme (one idea per screen). SplitText stagger creates motion semantics that read as "assembly," reinforcing the virtual-twin metaphor. Difference-blend cursor establishes brand presence in negative space without adding chrome.

GOOD / BAD: Best-in-class scroll-video sync feels engineered, not gimmicky. Bad: pinned-scrub sections stack ~40MB of video assets â€” a real accessibility and mobile-data smell without `prefers-reduced-motion` fallback.

CLEAN APPLICATION: Apply this to Clean's landing page hero section by replacing the current static hero with a single pinned-scrub section that plays a 6-second render of Huginn (the profiling agent) crawling a company graph. Bind scroll progress to `video.currentTime` so the user "drives" the crawl. Follow with two more pinned sections: (1) auto-profiling result cards materializing via SplitText, (2) lead export dashboard with number counters tied to scroll. Total hero-to-CTA scroll distance: 300vh.

## NRG | Build Your Data Center â€” https://www.awwwards.com/sites/nrg-build-your-data-center

WHAT IT DOES: Enterprise B2B campaign site (built by Rogue Studio) for NRG's AI data center offering. SOTD + Developer Award 2026-07-01. Central feature is an interactive isometric datacenter builder rendered in Three.js â€” user scrolls through phases (site â†’ power â†’ cooling â†’ connectivity) and modular building blocks fly into place. IMPLEMENTATION:
- App shell: `<Canvas camera={{ position: [8, 8, 8], fov: 30 }} shadows dpr={[1, 1.75]}>` fixed to viewport. Below it, a normal DOM scroll container drives R3F state via a Zustand store (`useScrollPhase`).
- Scene: base plinth (rounded box, subtle bevel), grid overlay (custom shader for isometric grid lines fading at distance), and phase modules as instanced meshes. Each module is a low-poly GLTF loaded via `useGLTF.preload`.
- Phase animation: `useScrollTrigger` maps 4 pin sections to a single ScrollTrigger with `snap: 1/3`. On phase change, module transforms interpolate with `gsap.to(mesh.position, { y: 0, duration: 1.2, ease: 'power4.inOut' })` starting from `y: 8` (raining from above) with `stagger: { each: 0.06, from: 'random' }`.
- Camera: `CameraControls` locked (no user input), but scroll drives `truck/dolly` micro-moves â€” Â±0.5 units to feel handheld.
- Lighting: HDRI environment (`Environment preset='studio'` intensity 0.4) + directional light `[10, 20, 5]` casting soft PCF shadows, contact-shadow disk at ground.
- Materials: physical, `roughness: 0.35, metalness: 0.6, envMapIntensity: 0.6`. Accent panels use emissive `#5CE1E6` at `0.4` for the cooling-loop reveal.
- Copy overlay: `<Html transform occlude>` labels attached to each module, appearing at phase-relevant scroll positions with `opacity` tween.
- Fallback: `prefers-reduced-motion` swaps the canvas for a 4-panel static PNG storyboard.
- Perf: `frameloop='demand'` in R3F, `invalidate()` only on scroll delta > 0.5px. Total draw calls kept under 30 via `<Instances>`.

WHY IT WORKS: Isometric axonometric projection (30Â° camera) is culturally coded as "technical/blueprint" â€” instantly signals engineering credibility for an infrastructure sale. GOOD / BAD: Best-in-class enterprise-B2B storytelling â€” makes an abstract capability (colocation) feel concrete. Bad: 12MB GLTF payload with no `<Suspense>` progress UI creates a 3-4s blank canvas on cold load.

CLEAN APPLICATION: Apply this to Clean's product demo section (below the hero) by building an isometric R3F scene showing the Clean pipeline: Raw Signals â†’ Huginn Profiler â†’ Enrichment â†’ CRM Sync. Each stage is a modular block that assembles as the user scrolls a pinned 400vh container. Use JetBrains Mono for stage labels overlaid via `drei/Html`. ## Radian â€” https://www.awwwards.com/sites/radian

WHAT IT DOES: Product-launch experience for an electric enduro motorcycle. SOTD + Developer Award 2026-07-05. Combines cinematic scroll storytelling with a real-time WebGL configurator (colorways, wheel packages, battery options). Uses a shared Three.js scene across marketing sections and configurator â€” camera rigs and pose the bike differently per section but never unmounts. Deep use of DracoLoader + KTX2 textures.

PACKAGES:
- next@^15
- @react-three/fiber@^8.17
- @react-three/drei@^9.114
- three@^0.168 with `three/examples/jsm/loaders/DRACOLoader` and `KTX2Loader`
- three-mesh-bvh (raycast optimization on the configurator hotspots)
- gsap@^3.12 (+ ScrollTrigger, MotionPathPlugin)
- lenis@^1.1
- valtio@^1.13 (configurator state â€” simpler than Zustand for shallow reactive objects)
- framer-motion@^11 (for DOM UI panels only)
- tailwindcss + a custom variable for the accent (matte-orange `#F04E23`)
- @fontsource-variable/inter-tight, @fontsource/space-grotesk (display)

IMPLEMENTATION:
- Single persistent `<Canvas>` mounted at root, `gl={{ antialias: true, powerPreference: 'high-performance' }}`, `dpr={[1, 2]}`.
- Bike model: Draco-compressed GLB (~4MB), KTX2 basisU-compressed textures (~1.2MB total). Preload with `useGLTF.preload('/bike.glb')`.
- Section rigs: an array of `{cameraPosition, cameraTarget, bikeRotationY, sceneEnv}` presets. On ScrollTrigger progress between sections, `gsap.to(camera.position, {...preset, duration: 1.6, ease: 'power3.inOut'})` and same for `controls.target`.
- Wheel rotation tied to scroll velocity: `bike.wheels.rotation.z += lenis.velocity * 0.02`.
- Configurator: variant swaps via `valtio` proxy. Material color change animates through a `useSpring`-driven MeshStandardMaterial `color.lerp(target, 0.15)` per frame until convergence.
- Lighting: 3-point studio rig â€” key `[5, 4, 3]` intensity 3.5, fill `[-3, 2, 4]` intensity 1.2, rim `[0, 3, -5]` intensity 2. `Environment preset='warehouse'` at 0.3 intensity for reflections.
- Post-processing: `@react-three/postprocessing` â€” Bloom `intensity: 0.4, luminanceThreshold: 0.9` on the accent-color emissives; SMAA for edges.
- Section pin/text: same GSAP+SplitText pattern as Wembi; headlines use Space Grotesk 120px display, body Inter Tight 17px.
- Configurator UI: right-rail drawer using framer-motion `AnimatePresence`, sliding in at `duration: 0.5, ease: [0.19, 1, 0.22, 1]` (quintic-out).
- Mobile: 3D scene keeps ~30fps by dropping to `dpr: 1`, disabling post-processing, using low-detail LOD.

WHY IT WORKS: Persistent 3D subject is a classic cinema-language move â€” the viewer's attention stays anchored on the "hero" while context (camera angle, environment) shifts around them. Reduces cognitive load: no re-orientation between sections. Configurator lives inside the same visual world as marketing, so buying feels continuous with exploration.

GOOD / BAD: Best-in-class continuity â€” no jarring context switch to a "product page." Bad: bike-first framing buries specs; a hurried buyer who wants range/weight/price scrolls past a lot of theater.

CLEAN APPLICATION: Apply this to Clean's pricing â†’ onboarding flow by keeping the Huginn agent "avatar" (small isometric assembly of Clean's data-flow nodes) persistent across three views: (1) Pricing tier selection tilts the node graph toward the tier's included features, (2) Workspace setup rotates to show the CRM integration slot, (3) First-run empty state animates seed profiles falling into the graph. Same `<Canvas>` throughout â€” never unmount.

## MONOLOG â€” https://www.awwwards.com/sites/monolog

WHAT IT DOES: Studio portfolio (Huy Nguyen). SOTD + Developer Award 2026-07-07. - three@^0.168 (single shader plane for wordmark, no scene graph)

IMPLEMENTATION:
- Root: Lenis `duration: 1.2, easing: cubic bezier equivalent`. GSAP ticker synced: `Lenis.on('scroll', ScrollTrigger.update); gsap.ticker.add(t => lenis.raf(t*1000))`.
- Hero wordmark: fullwidth `<canvas>` mounted behind a plain `<h1>` (the h1 is `sr-only` for a11y). Shader is a raymarched SDF of the letters, with a `uProgress` uniform driven by scroll. As progress 0â†’1, letters ripple horizontally (sin displacement) and split-tone (color gradient tealâ†’coral).
- Marquee bands: `.marquee { display: flex; will-change: transform; }` with GSAP `gsap.to('.marquee-track', { xPercent: -50, duration: 30, ease: 'none', repeat: -1 })`. Direction alternates per band.
- Project cards: grid uses `grid-template-columns: repeat(12, 1fr)`, cards span 5/6/7 cols in a rhythm. Each card has a `<div class="reveal">` overlay with a mask that reveals the image on hover via `clip-path: inset(0 0 0 0)` from `100%`.
- Cursor: custom cursor circle 40px, `mix-blend-mode: difference`, follows mouse with `lerp 0.16`. On card hover, scales to 80px with text "VIEW" inside.
- Page transitions: GSAP Flip snapshots the clicked card, then `Flip.from(state, { duration: 0.9, ease: 'expo.inOut' })` morphs it into the case-study hero position on the next route.
- Typography: display 180px/0.85 tracking -0.05em, uppercase, PP Neue Montreal Mono. Body 16px, PP Neue Montreal, tracking -0.01em.
- Palette: black `#0B0B0B`, cream `#EFEBE1`, accent coral `#FF5A3B`, accent teal `#00C2A1`.
- Perf: shader canvas only renders when in viewport (IntersectionObserver toggles `frameloop`). Marquees pause on tab hidden.

WHY IT WORKS: Type-as-hero is the most direct way to establish a studio's design voice â€” no imagery required, the letterforms are the pitch. Shader deformation on scroll gives the user physical agency over brand identity, which is memorable and defensible (competitors can copy an image; recreating this feels like theft).

GOOD / BAD: Best-in-class craft signal â€” engineering-forward studios read the shader and know these people can build. Bad: the wordmark shader shipped with no `prefers-reduced-motion` alternative, and low-end devices get a stuttery first paint.

CLEAN APPLICATION: Apply this to Clean's docs navigation and page-header treatment. Use a subdued shader displacement on the `Clean` wordmark that reacts to page-load progress (a technical audience will notice and appreciate). For docs, adopt the 12-col asymmetric grid for the landing sections of each doc category â€” "Guides," "API," "Playbooks" as cards spanning 5/7/12 with hover reveals.
