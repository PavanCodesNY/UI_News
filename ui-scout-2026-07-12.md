## Vectr — https://www.vectrfl.com/ (Awwwards SOTD, 2026-07-12)

WHAT IT DOES: An "AI-driven precision staffing for critical outages" marketing site organized as a fixed-viewport, scroll-driven diorama. The hero locks a monospaced ticker of live outage events under a giant `clamp()` wordmark; scrolling doesn't move the page, it advances a state machine that swaps SVG "radius maps" (concentric city rings that redraw with `stroke-dashoffset` interpolation) while KPI counters roll via `requestAnimationFrame`. Layout is a 12-column CSS grid with a permanently sticky right rail that houses the CTA and a mini radar loop.

PACKAGES:
- `next@15` (app router, RSC)
- `tailwindcss@4` with `@tailwindcss/postcss`
- `gsap@3.13` + `ScrollTrigger` + `Observer` (for the state machine)
- `lenis@1.3` (`import Lenis from 'lenis'`; sync to `gsap.ticker`)
- `motion@11` (Framer Motion successor) for element enter/exit
- `@number-flow/react` for the odometer KPI counters
- Fonts: `geist` (sans + mono) via `next/font`
- `lucide-react` for chevrons/arrow icons
- `clsx` + `tailwind-merge`

IMPLEMENTATION: Root layout wraps `<LenisProvider options={{ lerp: 0.1, wheelMultiplier: 1, smoothTouch: false }}>` and registers `gsap.ticker.add((t) => lenis.raf(t*1000))` with `gsap.ticker.lagSmoothing(0)`. Build the hero as `position: sticky; top: 0; height: 100svh` inside a 500vh spacer; use `ScrollTrigger.create({ trigger: spacer, start: 'top top', end: 'bottom bottom', scrub: 0.6, onUpdate: (self) => setStage(Math.floor(self.progress * 4)) })`. Radius map = SVG with 5 concentric `<circle>` elements, each `pathLength=1`, animate `strokeDashoffset` from 1 → 0 with `ease: 'expo.out'`, `duration: 1.2`. KPI counter uses `<NumberFlow value={n} format={{ notation: 'compact' }} transformTiming={{ duration: 800, easing: 'cubic-bezier(0.4, 0, 0.2, 1)' }} />`. Type ramp: `clamp(3rem, 8vw, 8.5rem)` for wordmark, `text-[11px] tracking-[0.18em] uppercase` for eyebrow labels. Palette: `#0A0A0A` bg, `#F5F5F0` fg, `#FF4A1C` accent for the pulsing outage dot (CSS `@keyframes` scale 1→1.4, `filter: blur(0) → blur(6px)`, 1.4s infinite). Breakpoints: below 768px, collapse the sticky rail into a bottom sheet and disable the state machine (`useReducedMotion` OR `matchMedia('(max-width: 767px)')` → fall back to vertical stack).

WHY IT WORKS: The sticky-hero + scrubbed SVG creates a "single dense screen" that reads as a product dashboard, not a marketing site — visual hierarchy funnels the eye from the monospaced ticker (evidence) to the wordmark (claim) to the CTA (action). Monospace + one hot accent color on grayscale is the Bloomberg Terminal move: signals seriousness and real-time-ness without any decoration. Gestalt continuity in the concentric rings mirrors the mental model of "radius search," so the metaphor is self-explanatory.

GOOD: State-machine-scroll turns a static hero into a demo of the product's core loop. Zero decorative motion — every animation carries information.
BAD: 500vh sticky hero is disorienting on trackpads with high wheel-inertia; no obvious "skip" affordance. Scrub-based animations tank INP on mid-tier Android.

CLEAN APPLICATION: Apply this to Clean's landing page hero (tryclean.ai) by rebuilding it as a sticky-viewport diorama that shows the "outbound loop" as a live state machine. Frame 1: ICP definition (monospace terminal typing "founder, seed, dev tools"). Frame 2: Prospect radius map (concentric rings emanating from an SF pin, dots populating). Frame 3: Enrichment cards flying in with `stroke-dashoffset` LinkedIn/website icons drawing. Frame 4: Sequenced email preview composing itself character-by-character. Frame 5: Reply appears, CTA "Book demo" resolves. Keep the palette to `#0A0A0A / #F5F5F0` plus one Clean accent; use Geist Mono for all ticker/label text.

## 21 Hrs On The Moon — https://www.21hrs.space/ (Awwwards SOTD, 2026-07-10)

WHAT IT DOES: A scroll-driven documentary about Apollo 11's 21 hours on the lunar surface. The whole experience is a single WebGL canvas: a photogrammetry lunar terrain rendered with baked shadows, over which HTML/DOM text panels are pinned. Scroll drives a `THREE.CatmullRomCurve3` camera path across the moon surface, and section panels fade in with a slight y-parallax while ambient audio (spatialized) crossfades between chapters. Two-tone palette (`#060B0B` charcoal, `#FFF3EA` bone) means every UI element defers to the imagery.

PACKAGES:
- `three@0.170` + `@react-three/fiber@9` + `@react-three/drei@9`
- `@theatre/core` + `@theatre/studio` for the camera path animation
- `gsap@3.13` + `ScrollTrigger` (for panel pinning)
- `lenis@1.3`
- `howler@2.2` for spatialized ambient audio + narration ducking
- `postprocessing@6` (bloom + film grain + vignette)
- `zustand@5` for chapter state
- Fonts: `PP Editorial New` (or `Editorial Old`) for display, `NB International Mono` for captions
- `next@15` + `tailwindcss@4`

IMPLEMENTATION: Load the moon terrain as a Draco-compressed GLB (target < 6MB) with a 4K baked lightmap for realtime-free shadows. R3F `<Canvas dpr={[1, 1.5]} gl={{ antialias: false, powerPreference: 'high-performance' }} shadows={false}>`. Build the camera track in Theatre.js as a 21-keyframe sequence over `duration: 100` — expose the sequence position as a Lenis-driven scroll value: `sequence.position = lenis.scroll / (document.body.scrollHeight - innerHeight) * 100`. Postprocessing chain: `EffectComposer` → `RenderPass` → `BloomEffect({ intensity: 0.4, luminanceThreshold: 0.85 })` → `NoiseEffect({ premultiply: true, blendFunction: BlendFunction.SOFT_LIGHT, opacity: 0.12 })` → `VignetteEffect({ offset: 0.35, darkness: 0.75 })`. HTML panels are absolutely positioned with `mix-blend-mode: difference` for the captions so they invert against dark/light lunar regions. Audio: preload 21 x 12s ambient clips at 128kbps mp3; use `Howl({ src, html5: false, volume: 0, loop: true })` and `howl.fade(0, 0.6, 800)` on ScrollTrigger enter, `fade(0.6, 0, 800)` on leave. Add `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">` and use `100dvh` for full-bleed panels.

WHY IT WORKS: The two-color palette and single visual object (the moon) enforce absolute focus — no navigation, no widgets, just narrative. Scroll-as-timeline collapses the reader's agency into a single dimension, which is exactly what documentary storytelling wants. Editorial serif + mono caption is a magazine convention (headline vs. dek vs. sidebar), and it borrows credibility from print journalism.

GOOD: Baked lightmap + dpr cap keeps this at 60fps on M1 Airs. Audio ducking is done tastefully.
BAD: 6MB+ WebGL payload with no skeleton on 3G, LCP is over 4s. Fully inaccessible to screen readers — no linear text alternative.

CLEAN APPLICATION: Apply this to Clean's product demo section on the landing page by building a scroll-driven "day in the life of a Clean-powered outbound loop" as one continuous canvas: a stylized topological map of the internet (LinkedIn nodes, company websites, Gmail nodes) with the camera flying between them as scroll advances. Each chapter beat corresponds to one Clean primitive (Sourcing, Enrichment, Sequencing, Reply-handling). Use the two-tone palette against Clean brand accent as the only source of color. Keep the whole scene under 5MB Draco + reduced postprocessing (drop bloom, keep grain) since Clean visitors are on corporate networks.

## Brunello Cucinelli AI E-Com — https://www.brunellocucinelli.com/ (Awwwards SOTD 2026-07-10, FWA SOTD)

WHAT IT DOES: A "pageless" e-commerce experience built on Solomei AI's Callimacus agentic platform. There are no product listing pages or category pages — the interface is a single canvas with a persistent bottom prompt bar and a modular component system that dynamically assembles cards, editorial imagery, and product carousels in response to user intent. Visual language is frosted glass (`backdrop-filter: blur(24px) saturate(1.4)`), soft blur transitions between states, and hand-drawn SVG sketch details layered over photography.

PACKAGES:
- `next@15` (app router, streaming SSR)
- `ai@3` (Vercel AI SDK) + Callimacus/Solomei backend
- `framer-motion@11` (or `motion@11`) — the LayoutGroup + AnimatePresence pattern is central
- `tailwindcss@4` with custom `backdrop-blur-3xl` extensions
- `radix-ui/react-dialog` + `radix-ui/react-tooltip` for accessible modals
- `zustand@5` for agent state
- `three@0.170` (lightweight — only for the ambient hero shader)
- `use-gesture@10` for drag/gesture on product cards
- Fonts: `Suisse Serif` (or `PP Editorial`) for editorial, `Suisse Int'l` for UI
- `svg-path-morph` for the hand-drawn sketch reveal

IMPLEMENTATION: Layout is `position: fixed` full-viewport with an always-visible bottom bar (`h-[72px] rounded-full mx-auto max-w-2xl backdrop-blur-2xl bg-white/40 border border-white/60`). Content zone is a single `<AnimatePresence mode="popLayout">` that swaps between "receptors" (Discovery, Product, Editorial, Look). Each receptor is a Framer Motion `<motion.div layoutId="stage" transition={{ type: 'spring', stiffness: 180, damping: 24, mass: 0.9 }}>` — the shared layout gives that liquid morph feeling. Frosted glass: every card is `bg-[rgba(250,247,240,0.55)] backdrop-blur-[18px] backdrop-saturate-[140%] border border-white/40 shadow-[0_20px_60px_-20px_rgba(60,50,40,0.25)]`. Hand-drawn details: SVG paths with `stroke-dasharray: 1000; stroke-dashoffset: 1000` animated to 0 on mount with `duration: 1.6s; ease-out`. Prompt bar streams responses via AI SDK `useChat({ streamProtocol: 'data' })` and dispatches tool calls that return a receptor + props; a `Renderer` component switches on receptor name and mounts the corresponding component. Palette: `#FAF7F0` bg, `#3B3229` fg, `#B08D5C` accent (muted camel).

WHY IT WORKS: Frosted glass + editorial serif + hand-drawn ink is the visual equivalent of a boutique's material palette (cashmere, linen, ink on paper) — the interface *feels* like the brand rather than just displaying it. Shared-element layout animations create the illusion that content is *one thing* being reshaped by the user, not many pages being navigated. This is the AI-native e-commerce paradigm: prompt → response → morph.

GOOD: Sets the new bar for AI-native commerce UX — no PLP tyranny. Motion is expressive but rooted in physics (spring), not scripted.
BAD: Zero URLs = zero SEO for individual products, zero sharing, zero back-button. Screen readers get one long region. Battery-hungry on mobile due to constant backdrop-filter.

CLEAN APPLICATION: Apply this to Clean's onboarding flow (first-run experience after signup) by rebuilding it as a single-canvas conversational setup: bottom prompt bar asks "What's your ICP?" → morphs into a live prospect preview → user says "add title filters" → the same canvas morphs to a filter panel with the results updating in place. Use frosted glass cards over a subtle Clean-branded gradient. Every step should feel like reshaping one artifact, not clicking through a wizard. Keep the shared `layoutId` pattern so the ICP definition literally becomes the campaign card.

## MONOLOG — https://bymonolog.com/ (Awwwards SOTD, 2026-07-07)

WHAT IT DOES: A design studio portfolio built in Webflow with GSAP + Three.js accents. The whole site is a typography-first grid experiment: KH Teka display sizes at `clamp(6rem, 18vw, 22rem)` sit above a 12-column grid, and case-study previews reveal with a signature "scale-in-from-92%-with-slight-clip-path-reveal" transition. A subtle Three.js particle field sits behind the hero, and cursor-follow magnetic effects on all clickable elements make the whole page feel physically responsive. Two-color palette (`#080807`, `#DDDDD5`) forces contrast to do all the work.

PACKAGES:
- Webflow (host) — for a from-scratch build: `astro@5` + `react` islands
- `gsap@3.13` + `ScrollTrigger` + `SplitText` + `Flip`
- `three@0.170` (bare, no R3F) for the particle field
- `lenis@1.3`
- Fonts: `KH Teka` (Kometa), `Suisse Int'l Mono` — loaded via `@font-face` with `font-display: swap` and `size-adjust`
- Custom cursor: hand-rolled with `requestAnimationFrame` + `transform: translate3d`

IMPLEMENTATION: Text reveal: use `SplitText.create(el, { type: 'lines,words' })` then `gsap.from(split.words, { yPercent: 110, duration: 0.9, ease: 'expo.out', stagger: 0.02 })` triggered by `ScrollTrigger.create({ trigger: el, start: 'top 85%' })`. Case-study hover: on `mouseenter`, animate `clipPath` from `inset(50% 0)` to `inset(0 0)` with `duration: 0.6, ease: 'power3.out'` while scaling the image `from: { scale: 1.12 } to: { scale: 1 }`. Magnetic buttons: on `mousemove`, translate the child by `(mouseX - centerX) * 0.25, (mouseY - centerY) * 0.25` with `gsap.quickTo(el, 'x', { duration: 0.4, ease: 'power3.out' })`. Three.js particles: 2000 THREE.Points on a plane, vertex shader displaces by cursor world position (raycaster) with a falloff `smoothstep(0.3, 0.0, distance)`. Grid: `grid-template-columns: repeat(12, 1fr); gap: clamp(0.5rem, 1vw, 1.25rem)`. Use `mix-blend-mode: difference` on the custom cursor so it inverts against any background.

WHY IT WORKS: Massive display typography sets a confident editorial tone; the two-color palette makes every hover state (accent-colored underline, image reveal) feel like a signal, not decoration. Magnetic cursor + Flip transitions create the perception that the page is a physical material being touched — this is high-agency motion design. Gestalt proximity in the 12-column grid does the visual hierarchy work without needing dividers.

GOOD: Zero-decoration aesthetic wears well beyond a launch cycle. Type-first hierarchy makes the site legible without images.
BAD: Custom cursor breaks touch UX; SplitText + Three.js particles are ~180KB extra JS for what a static grid could achieve.

CLEAN APPLICATION: Apply this to Clean's docs and homepage navigation by adopting the KH Teka-style display type at `clamp(4rem, 12vw, 14rem)` for section headers ("Sourcing," "Enrichment," "Sequences," "Replies") with the SplitText word-flip reveal on scroll-into-view. Use the magnetic hover on primary CTAs only (Book demo, Start free). Skip the Three.js particle field — Clean should feel like infrastructure, not portfolio-craft — but keep the Flip-based case-study transitions for customer stories.

## Iventions — https://iventions.com/ (CSS DA Website of the Month Oct '25, Awwwards SOTD + Developer Award)

WHAT IT DOES: A studio site for a spatial-experiences agency. The signature move is a Three.js "spotlight installation" treatment: each project is rendered as a 3D object in a dark chamber with volumetric lighting; GSAP paces the scroll such that projects fade up under the spotlight one at a time. Between sections, the camera "walks" through a virtual gallery, and portfolio images are treated as physical prints on 3D walls with realistic lighting.

PACKAGES:
- `three@0.170` + `@react-three/fiber@9` + `@react-three/drei@9` (`SpotLight`, `useHelper`, `AccumulativeShadows`)
- `postprocessing@6` (`SelectiveBloomEffect`, `GodRaysEffect`)
- `gsap@3.13` + `ScrollTrigger`
- `lenis@1.3`
- `leva@0.9` (dev-only, for tuning lights)
- `zustand@5`
- `next@15` + `tailwindcss@4`
- Fonts: `Neue Haas Grotesk Display Pro` + `NB International Mono`

IMPLEMENTATION: Single R3F canvas across the site. Set `<Canvas gl={{ toneMapping: THREE.ACESFilmicToneMapping, toneMappingExposure: 0.8 }} shadows>`. The scene is a dark room (`<mesh>` planes, `MeshStandardMaterial roughness={0.9}`). Each project is a `<Suspense>` group with a `<SpotLight distance={8} angle={0.35} attenuation={2.5} anglePower={4} intensity={0}>` above it. ScrollTrigger onEnter: `gsap.to(spotlight, { intensity: 12, duration: 1.2, ease: 'expo.out' })`, onLeave: `gsap.to(spotlight, { intensity: 0, duration: 0.8 })`. Postprocessing: `SelectiveBloomEffect({ selection: [projectMesh], luminanceThreshold: 0.6, intensity: 1.8 })` combined with `GodRaysEffect({ lightSource: spotlight, density: 0.94, decay: 0.94, weight: 0.4, exposure: 0.55, samples: 60 })`. Camera walk: `CatmullRomCurve3` with 8 control points; scroll drives `curve.getPointAt(progress)` for position and `curve.getTangentAt(progress)` for lookAt. Layout is HTML overlay: `position: fixed; top: 0; pointer-events: none;` with per-section unlock via `pointer-events: auto`. Palette: `#0A0A0A`, `#F2EEE6`, `#D4A85A` (warm spotlight tone).

WHY IT WORKS: Lighting as narrative — the spotlight *is* the reading order. Users don't need any hierarchy cues because photons literally point at what to look at next. Volumetric god-rays add cinematic weight without decoration. The physical-space metaphor (walking through a gallery) is intuitive because it borrows from a real-world experience everyone knows.

GOOD: Uses 3D to solve a real hierarchy problem, not for spectacle. GodRays + selective bloom is the best-in-class postprocessing combo for this look.
BAD: GodRays at 60 samples is expensive; on 60Hz displays without a discrete GPU it drops to 40fps. No graceful degradation shown for `prefers-reduced-motion`.

CLEAN APPLICATION: Apply this to Clean's customer stories / case studies page by rendering each case study as a spotlit "installation" in a dark canvas — the customer's logo and headline pill fade up under a warm spotlight as the user scrolls, with the camera dollying between them. Use it sparingly (only the stories page), keep the god-rays low-sample (24), and provide a `prefers-reduced-motion` fallback that renders each case as a static dark card with a CSS `radial-gradient` fake spotlight.

## BUILD DIRECTIVE

TARGET: **Clean landing page hero rebuild** (tryclean.ai — hero + first-fold demo section)

APPROACH: Rebuild the hero as a Vectr-style sticky-viewport diorama that runs a scroll-driven state machine through the four Clean primitives (Source → Enrich → Sequence → Reply). Use a Bloomberg-Terminal-grade monospace + one accent palette (`#0A0A0A / #F5F5F0 / #FF4A1C` or Clean's accent). Layer Iventions' spotlight-as-hierarchy principle: when each frame is active, dim the others and put a subtle radial glow behind the live frame. Adopt MONOLOG's SplitText word-flip reveal for the H1 and the frame captions. Skip Brunello's frosted-glass — save it for the onboarding flow. The whole hero should be one continuous scroll experience that *demonstrates the product* rather than describing it.

STACK:
- `next@15` (app router, RSC)
- `tailwindcss@4`
- `gsap@3.13` + `ScrollTrigger` + `Observer` + `SplitText`
- `lenis@1.3` (synced to `gsap.ticker`, `smoothTouch: false`)
- `motion@11` for element enter/exit + `<AnimatePresence>` between frames
- `@number-flow/react` for KPI counters (reply rate, meetings booked, etc.)
- `lucide-react` for iconography
- Fonts: `geist` (sans + mono) via `next/font`
- `clsx` + `tailwind-merge`
- No Three.js — infrastructure companies should not ship 6MB WebGL payloads.

REFERENCE URLS:
- https://www.vectrfl.com/ (the sticky-viewport state machine — primary structural reference)
- https://iventions.com/ (spotlight-as-hierarchy — secondary reference for radial-glow treatment)
- https://bymonolog.com/ (typography scale and reveal choreography — tertiary)

KEY CONSTRAINTS:
- Clean is developer infrastructure. Avoid: gratuitous WebGL, 3D characters, marquee "tech" gradients, glassmorphism on the landing page, cursor trails, particle fields, custom cursors, anything that reads as "portfolio site."
- LCP must be under 2.0s on 4G — no hero canvas, no heavy fonts (Geist is subsetted and cached).
- All scroll-driven motion must have `prefers-reduced-motion: reduce` fallback that renders the four frames as a static 4-column grid.
- Below 768px, disable the state machine entirely and render a vertical stack of the four frames with static illustrations.
- Every animation must carry information about the product loop. Zero decorative motion. If you can't answer "what does this animation teach the user about Clean?", cut it.
- Use monospace only for data/labels (timestamps, IDs, counters). Sans for headlines and body.
- Accent color usage: reserve for one thing per frame (the live/active state indicator). Overuse kills the signal.
