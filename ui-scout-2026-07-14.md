## PP Neue Montreal — https://neuemontreal.com/

WHAT IT DOES: A single-page editorial "travel guide" for Pangram Pangram's flagship grotesque. The layout is a stacked vertical narrative — full-bleed hero video → sticky-scroll typography specimens → horizontal-scroll "attractions" gallery → interactive weight chart → font variant grid. Motion is scroll-driven throughout: text mask reveals, image parallax at 0.3–0.6 scroll ratio, and a shared-layout transition where clicked words morph into headings. Every duplicated line ("Refined, Reworked, Reintroduced" appears twice) is a marquee that scrolls at opposite velocities.

PACKAGES: Built entirely on Framer (Framer 17c1b93, generator meta tag). To rebuild in React: `next@15`, `motion@12` (successor to framer-motion), `@studio-freight/lenis@1.3`, `gsap@3.13` + `ScrollTrigger`, `split-type@0.3`, `@radix-ui/react-scroll-area`, `sharp` for `.webp` at 2x, self-hosted `PP Neue Montreal` (Display + Text families, both italic).

IMPLEMENTATION: 
- Root: `<Lenis lerp={0.1} smoothWheel duration={1.2}>` wraps entire app.
- Hero: `<section class="h-screen relative overflow-hidden">` with autoplayed muted `<video>` (Framer's `XFVX6jaxToykdjjsmM3uRB9CnPw.mp4`), overlaid `<h1 class="text-[clamp(4rem,12vw,18rem)] leading-[0.85] tracking-[-0.04em]">` split by SplitType per-word, staggered `y: 40 → 0, opacity: 0 → 1` with `stagger: 0.04, duration: 0.8, ease: [0.16, 1, 0.3, 1]`.
- Duplicate headings: two identical `<div>`s with `translateX` keyframed via GSAP `ScrollTrigger` — top one goes `0 → -20%`, bottom one `-20% → 0` across the pinned section (`pin: true, scrub: 1`).
- Attractions carousel: `<section class="h-[300vh]">` container, inner `<div class="sticky top-0 h-screen flex">` holds a horizontally-translated wrapper — `x: 0 → calc(-100% + 100vw)` on scroll progress.
- Weights chart: SVG timeline (2018 → 2026) with `<circle>` nodes; hover triggers a portal-rendered spec card at `position: fixed, top: mouseY, left: mouseX` with `spring: {stiffness: 260, damping: 20}`.
- Font tester: `<textarea contentEditable style={{fontFeatureSettings: '"ss01","ss02","liga"', fontVariationSettings: '"wght" ${weight}, "opsz" 32'}}` — weight bound to a range slider.
- Ligature demo: `<span class="inline-block hover:scale-110 transition-transform duration-500 ease-[cubic-bezier(0.34,1.56,0.64,1)]">` per glyph pair.
- Perf: `<img loading="lazy" decoding="async" fetchPriority="low">` except hero; use `content-visibility: auto` on off-screen sections.
- Breakpoints: mobile stacks all horizontal-scroll sections into vertical `flex-col`; disable Lenis under 768px (`isTouchDevice() ? null : <Lenis>`).

WHY IT WORKS: The site treats the typeface as a place — Montreal — and structures itself like a guidebook, giving abstract type discussion a spatial narrative that keeps a 30-second attention span engaged for four minutes. Duplicated marquee headlines create visual rhythm (the "musical" repetition principle from editorial poster design) and reinforce hierarchy without adding ornament. The single font, single color scheme forces contrast to be built from size, weight, and whitespace — proving the typeface's range live.

GOOD / BAD: GOOD — the specimen-as-experience pattern earns every scroll and never breaks its own rule of "one font, one story." BAD — no keyboard skip links, video autoplay hits 40MB on load, and the horizontal-scroll section traps screen readers in an inaccessible loop.

CLEAN APPLICATION: Apply this to Clean's landing page hero + "how it works" section by treating the product's four scaler modules (Knowledge Base, Content, Lead Sourcing, Auto-Profiling) as four "attractions" in a horizontal-scroll travel guide. Pin the section for 4× viewport height, translate the module cards horizontally, and use SplitType word-reveals for each module title. Anchor the whole thing with a single serif/mono pairing (JetBrains Mono for code snippets, Neue Haas Grotesk Display for headings) — no illustrations, no gradients. The typography IS the demo.

## Longbow — https://db-longbow.webflow.io/

WHAT IT DOES: Marketing site for Longbow Motors' British EV sports car. Full-viewport dark hero with a hero video of the car pinned centered, tightly-tracked serif wordmark, and a "scroll-hijack" section that transitions from vehicle exterior → interior → engineering diagrams as the user scrolls. Product-page (`/speedster`) is a spec sheet using columnar typography — no product renders, all information density.

PACKAGES: Built on Webflow. To reproduce in a stack: `next@15`, `tailwindcss@4`, `motion@12`, `@studio-freight/lenis@1.3`, `gsap@3.13` + `ScrollTrigger`, `three@0.170` + `@react-three/fiber@8.17` + `@react-three/drei@9.114` (for the GLB car model rotation), `next/font` for local variable font, `plaiceholder` for LQIP.

IMPLEMENTATION:
- Palette: bg `#0A0A0A`, fg `#F2F2F0`, accent `#C8A45A` (single warm metallic).
- Typography: display uses tight tracking `letter-spacing: -0.045em`, line-height `0.92`; body is `text-sm uppercase tracking-[0.15em] font-medium`.
- Hero: `<section class="h-svh relative">` — background `<video autoplay muted loop playsInline>`, foreground large `<h1>` with mix-blend-mode `difference` so text stays legible over any frame.
- Vision block: sticky pinned for `3 * 100vh`, GSAP timeline crossfades three layered `<div>`s of `background-image` at `progress: 0.0, 0.33, 0.66`.
- Car rotation: React Three Fiber canvas with a low-poly GLB, `<PresentationControls global azimuth={[-Math.PI/4, Math.PI/4]}>`, no autoRotate to keep it feeling considered.
- Footer: full-viewport `<footer>` with the wordmark set at `font-size: clamp(6rem, 18vw, 22rem)`, absolute-positioned corner metadata blocks — a common Webflow "brutalist minimal" footer pattern.
- Cursor: custom cursor is a 6px black ring, scales to 40px with `mix-blend-mode: difference` on link hover; use `usePointerCursor()` hook that listens to `mousemove` with `requestAnimationFrame` throttling.
- Transitions: `next-view-transitions` for page changes, `startViewTransition` with `.vt-name-hero` on the car image so it morphs between pages.

WHY IT WORKS: Extreme restraint — one color accent, one photo, one moment of motion per section. The luxury automotive playbook (Aston Martin, Bentley) taught web designers that empty space signals premium; Longbow executes it without falling into stock "expensive minimalism" because the type is aggressively tight. `mix-blend-mode: difference` on the hero title solves the legibility-over-video problem elegantly.

GOOD / BAD: GOOD — the wordmark-as-footer treatment turns the site's outro into brand memory, and the interior/exterior scroll transition is genuinely novel for automotive. BAD — Webflow's default lazy-loading behavior causes a visible pop-in on the vision block; no reduced-motion fallback for the scroll hijack.

CLEAN APPLICATION: Apply this to Clean's pricing page by rebuilding it as a single dark canvas with three columns (Growth Scaler, Revenue Scaler, Enterprise), no shadows, no borders, no gradient card backgrounds. Each price sits at `clamp(4rem, 8vw, 9rem)` in the display font, with the plan features below in `text-xs uppercase tracking-[0.2em]`. Use `mix-blend-mode: difference` on a slim horizontal ticker beneath the prices that shows real-time metrics ("14,382 leads sourced this hour"). Footer becomes a giant "CLEAN" wordmark at 22vw, corner-anchored contact/social. Zero gradients, zero glassmorphism.

## Vectr — https://vectrfl.com/

WHAT IT DOES: Homepage for an AI-driven staffing platform for critical outages (nuclear, industrial). Layout is a vertical-scroll numbered process (01–04) that pins each step and transitions with a scaled photograph swap. Below the fold: 2×2 feature grid with SVG line-icons, then a full-bleed industrial photo with overlay title, then FAQ accordion. Built to feel "operational" — industrial typography, technical labeling, zero decorative flourish.

PACKAGES: Built with `astro@5` (canonical is `.astro`). Stack to replicate: `astro@5` + `@astrojs/tailwind`, `motion@12` for micro-interactions, `@studio-freight/lenis@1.3`, `three@0.170` + `@react-three/fiber@8.17` for the page-transition mesh distortion, `astro-icon@1` with local SVGs (already served from `/icons/features/*.svg`), `sharp` for image optimization, no client-side JS on the base render — hydrate `client:visible` per island.

IMPLEMENTATION:
- Numbered process: `<section class="min-h-[400vh] relative">` container with a `<div class="sticky top-0 h-screen">` inside. GSAP ScrollTrigger with 4 labels; each label triggers `gsap.to('.step-image', {clipPath: 'inset(0 0 0 0)', duration: 0.6, ease: 'power2.out'})` on the incoming image and reverse on outgoing.
- Numbered labels use `font-mono`, `text-6xl`, appear as a two-digit ordinal (`01`, `02`) on the left, description on the right — grid: `grid-cols-[max-content_1fr] gap-24`.
- Feature grid: `<div class="grid grid-cols-2 gap-px bg-neutral-200">` with feature cells `bg-white p-12` — the 1px gap creates hairline dividers without borders.
- Page transitions: intercept anchor clicks, run a `<canvas>` GLSL shader (React Three Fiber `<ShaderMaterial>`) that samples the current viewport and distorts with `uProgress` uniform tweened `0 → 1` over 800ms with `easeInOutCubic`, then swap route.
- Scroll animations (from Awwwards element demo): text lines use `<span class="block overflow-hidden"><span class="block translate-y-full">…</span></span>` with `translateY(0)` triggered by `IntersectionObserver` at `threshold: 0.4` and `transition: transform 0.7s cubic-bezier(0.65, 0, 0.35, 1)`.
- FAQ: native `<details><summary>` styled — no JS. Rotate chevron with `[&[open]>summary_img]:rotate-180 transition-transform duration-300`.
- Icons: single-color SVGs at `24px`, stroke-width `1.5`, no fills — matches the industrial "diagram" feel.
- Palette: bg `#FFFFFF`, foreground `#0A0A0A`, muted `#8A8A8A`, accent `#0057FF` (single blue), safety-adjacent `#F5F1E8` for photo-adjacent panels.

WHY IT WORKS: Every design choice reinforces the "we operate at nuclear-grade standards" claim — mono numerals, technical grid alignment, no decorative motion. The pinned 4-step process is the same pattern Linear and Vercel use for landing pages because it forces the user through a linear narrative while showing depth (each step has its own image). Blue as the only accent color is deliberately industrial-signage-inspired.

GOOD / BAD: GOOD — the Astro islands architecture keeps the initial payload under 60KB while still enabling GSAP scroll pins; the numbered-step pattern is copy-paste transferable to any technical B2B product. BAD — the shader page transition is visually beautiful but adds 200ms of perceived latency and provides no accessible fallback.

CLEAN APPLICATION: This is the closest peer to Clean and should be the primary reference. Rebuild Clean's "How it works" section as a 4-step pinned scroll (`01 Ingest → 02 Enrich → 03 Score → 04 Route`) where each step swaps a screenshot of the actual dashboard. Use Vectr's exact grid-cols-[max-content_1fr] layout with mono numerals. Steal the 1px-gap-on-neutral-bg trick for the "why Clean" feature grid so features feel like cells in a spec sheet, not marketing cards. Keep the single-blue accent — Clean should look operational, not consumer.

## Radian — https://www.awwwards.com/sites/radian

WHAT IT DOES: Product launch site for an electric off-road motorbike. Combines a WebGL configurator (rotate the bike, swap colorways, toggle components) with scroll-driven product storytelling — as you scroll, the bike disassembles into exploded-view engineering diagrams, then reassembles at the CTA. Fixed side rail exposes chapter navigation like a table of contents.

PACKAGES: `next@15`, `react-three-fiber@8.17`, `@react-three/drei@9.114`, `@react-three/postprocessing@2.16` (for bloom + chromatic aberration on the color-swap moment), `leva@0.10` (dev-only, for tuning), `gsap@3.13` + `ScrollTrigger`, `@studio-freight/lenis@1.3`, `zustand@5` for global configurator state, `sharp` for 3x AVIF hero renders, `next-view-transitions` for chapter routing.

IMPLEMENTATION:
- Model: single GLB (~4MB draco-compressed) preloaded on route entry via `useGLTF.preload('/bike.glb')`.
- Configurator canvas: `<Canvas shadows dpr={[1,2]} camera={{position:[4,1.5,6], fov: 35}}>` with `<Environment preset="warehouse">`, `<ContactShadows opacity={0.4} blur={2}>`, one `<directionalLight castShadow intensity={0.8}>`.
- Color swap: material `<meshPhysicalMaterial color={pickerColor} metalness={0.9} roughness={0.35} clearcoat={1} clearcoatRoughness={0.1}>` — animate color transitions with `useSpring` from `@react-spring/three` (`config: {mass:1, tension:180, friction:26}`).
- Explode animation: each mesh has a target `position.add(directionAway * scrollProgress)` computed once at mount; ScrollTrigger drives a single `progress` uniform 0→1 that all meshes read.
- Chapter rail: `<nav class="fixed top-1/2 -translate-y-1/2 right-8">` — each `<a>` tracks the currently pinned section via `useInView` from `motion`, active state gets `w-8` bar (default `w-2`), transition `duration-300 ease-out`.
- Cursor magnet: interactive elements use `useMouseFollow(mouseX, mouseY, {stiffness: 150, damping: 15})` to translate 8px toward cursor when within 100px.
- Loader: three.js `LoadingManager` reports progress → animate a `<div class="fixed inset-0 bg-black text-white">` counter `0 → 100%` with a bottom-progress bar; on complete, `clipPath: 'inset(100% 0 0 0)'` reveals the site over 700ms.
- Fallback: if `navigator.gpu` unavailable or `prefers-reduced-motion`, replace canvas with a static PNG hero.

WHY IT WORKS: The configurator gives the user agency in the first 5 seconds, which is the single highest-leverage engagement move on any product site (proven from Apple's iPhone configurator to Rivian's ordering flow). Pairing agency with a passive scroll narrative means both explorers and skimmers get a rewarding path. Bloom + chromatic aberration on color swap is a strong dopamine cue that reinforces the "premium object" positioning without feeling like a demo reel.

GOOD / BAD: GOOD — configurator state persists into the reserve/buy CTA, closing the loop from exploration to conversion. BAD — 4MB GLB + shader postprocessing means first-load TTI on mid-tier Android crosses 6 seconds; no server-side render of chapter content means SEO is entirely dependent on the loader completing.

CLEAN APPLICATION: Apply this to Clean's product demo section on the landing page by building a mini-configurator for Huginn (the Knowledge Base Engine): let users pick an industry (SaaS, Fintech, DevTools), watch the KB assemble a live company profile on the right, then let them scroll to see the profile "explode" into its structured signal fields (intent scores, hiring signals, tech stack). Use the same chapter-rail navigation pattern to jump between product surfaces. No 3D — the "3D moment" here is the animated graph of enriched fields. Keep the loader-on-progress pattern for real API-latency masking during the live enrichment demo.

## Aceternity UI — Background Ripple Effect — https://ui.aceternity.com/components/background-ripple-effect

WHAT IT DOES: A hero-section background component that renders a 20×20 grid of thin-bordered cells. On click anywhere, ripples cascade outward from the click point with staggered opacity + scale, decaying by radial distance. Cells default to `border-white/10` and gain `bg-neutral-800/50` during the ripple pass. Zero JS on paint — animations are pure CSS `@keyframes` triggered by JS-set custom properties per cell.

PACKAGES: `react@19`, `next@15`, `tailwindcss@4`, `motion@12` (only for occasional pointer callbacks — the ripple itself is CSS), `class-variance-authority@0.7`, `clsx@2.1`, `tailwind-merge@2.5`. Aceternity components ship as copy-paste — no runtime package needed.

IMPLEMENTATION:
- Grid: `<div class="grid" style={{gridTemplateColumns: 'repeat(30, 1fr)', gridTemplateRows: 'repeat(20, 1fr)'}}>` — 600 `<div>` cells.
- Cell base: `class="border border-neutral-800/50 transition-[background] duration-500 ease-out"` with `style={{'--delay': `${Math.hypot(row - clickRow, col - clickCol) * 40}ms`}}`.
- Ripple keyframe: `@keyframes ripple { 0% {background: rgba(23,23,23,0); transform: scale(1)} 50% {background: rgba(23,23,23,0.6); transform: scale(0.98)} 100% {background: rgba(23,23,23,0); transform: scale(1)} }` — applied on click via `animation: ripple 700ms ease-out var(--delay) both`.
- Click handler (single listener on grid parent, not per-cell — huge perf win): compute `rect.getBoundingClientRect()`, derive row/col from click, dispatch class via `parent.dataset.rippleFrom = "${row},${col}"`. Cells subscribe via CSS attribute selector `[data-ripple-from] .cell:nth-child(...)`.
- Cell count optimization: use `content-visibility: auto` on the grid parent; cap cell count to `Math.min(viewportWidth/40, 30) × Math.min(viewportHeight/40, 20)`.
- Overlay: content sits in a `<div class="absolute inset-0 flex items-center justify-center pointer-events-none">` — text is `pointer-events-none` so clicks pass through to the grid.
- Palette: bg `#000`, cells `#171717/50`, ripple `#262626/60`, hero text pure `#FFF` for maximum contrast.

WHY IT WORKS: The grid pattern (Gestalt: continuity + closure) reads as "infinite plane," which frames content sitting on top as anchored and important. Ripples from cursor position create a subconscious feedback loop — the site is "responding to you" before you've done anything meaningful. Because the animation is triggered by user input rather than autoplay, it never feels like decoration; it feels like the interface is alive.

GOOD / BAD: GOOD — single-listener architecture means it holds 60fps even with 600 cells; the effect is copy-paste and works with any Tailwind project immediately. BAD — the entire pattern is aesthetic-only — no accessibility benefit, and screen-reader users get no context that a background even exists.

CLEAN APPLICATION: Apply this to Clean's landing page hero (`tryclean.ai/` first viewport) as the background layer beneath the primary H1 "Growth Infrastructure for GTM Teams." Set grid to 30×20 on desktop, 15×10 on mobile. Change palette: cells `border-emerald-500/8`, ripple state `bg-emerald-500/12` (matches Clean's operational-green brand accent). Bind ripple trigger not only to click but to `mousemove` (throttled to every 400ms with 6-cell radius) so the hero feels alive during passive hover. Text overlay: `<h1 class="font-serif text-6xl md:text-8xl tracking-tight text-white">` centered, with a mono subtitle `text-xs uppercase tracking-[0.3em] text-emerald-500` above ("v2.0 — now in early access").
