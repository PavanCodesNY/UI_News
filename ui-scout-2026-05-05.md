## 1. Where Worlds Take Shape — https://www.awwwards.com/sites/where-worlds-take-shape

WHAT IT DOES: A scroll-and-drag explorable WebGL "world" that replaces the conventional portfolio grid. Project case studies are placed as discoverable 3D objects in a single contiguous scene; the camera follows a Catmull-Rom spline driven by scroll progress while pointer position adds a damped offset (mouse-look) using lerp. Hover targets raycast against invisible bounding boxes that swap a screen-space label and crossfade a video texture onto a plane facing the camera. Real-time post-processing (bloom + chromatic aberration + film grain) is layered over PBR materials on baked GLTF geometry.

PACKAGES:
- next@14 (App Router) or vite@5 + react@18
- three@0.160
- @react-three/fiber@8
- @react-three/drei@9 (PerspectiveCamera, Environment, useGLTF, useTexture, Html, MeshTransmissionMaterial, ScrollControls)
- @react-three/postprocessing@2 + postprocessing@6 (Bloom, ChromaticAberration, Noise, Vignette)
- gsap@3.12 (ScrollTrigger, Observer)
- lenis@1 (smooth scroll)
- maath@0.10 (easing utils, lerp)
- leva@0.9 (dev-only HUD)
- tailwindcss@3.4 + clsx
- next/font with `Inter Display` + `Suisse Int'l Mono` (or `JetBrains Mono`)

IMPLEMENTATION:
1. Component tree: `<Lenis>` → `<Canvas dpr={[1,1.5]} gl={{antialias:true,alpha:false}}>` containing `<ScrollControls pages={6} damping={0.18}>` → `<World />` (group of `<ProjectIsland />` instances) → `<EffectComposer><Bloom intensity={0.6} luminanceThreshold={0.85}/><ChromaticAberration offset={[0.0009,0.0006]}/><Noise opacity={0.08}/></EffectComposer>`. DOM overlay: fixed `<header>` and `<HoverLabel>` rendered with `<Html portal={overlayRef}>`.
2. Camera path: build a `THREE.CatmullRomCurve3` from 8–12 hand-tuned points. Each frame, in `useFrame`, set `camera.position.copy(curve.getPointAt(scroll.offset))` and `camera.lookAt(curve.getPointAt(scroll.offset+0.001))`. Add pointer-driven offset: `camera.position.x += (mouse.x*0.4 - state.x)*0.06; camera.position.y += (mouse.y*0.25 - state.y)*0.06;`.
3. Project islands: instanced `<group>` per case study, each with a baked AO/lightmap GLTF (export from Blender with Cycles bake → Draco-compressed). Use `MeshStandardMaterial` with `aoMap`, `roughnessMap`, `envMapIntensity={0.7}`. Mount a `PlaneGeometry` for video texture (`useVideoTexture`) that lazy-loads on intersect.
4. Hover/raycast: `useCursor` from drei plus `onPointerOver`/`onPointerOut` to drive a `useSpring` on emissive intensity (0 → 0.4) and label opacity (gsap `to` with `duration: 0.45, ease: "power3.out"`).
5. Transitions between project detail and world: route change uses `gsap.timeline({defaults:{ease:"expo.inOut",duration:1.2}})` to dolly camera +z, fade overlay (background `rgba(8,8,10,0.92)`), then mount detail page.
6. Performance: `frameloop="demand"` with `invalidate()` driven by Lenis; KTX2-compressed textures via `useKTX2`; `<Preload all/>`; cap dpr at 1.5; `gl.outputColorSpace = THREE.SRGBColorSpace`; `gl.toneMapping = THREE.ACESFilmicToneMapping; gl.toneMappingExposure = 1.05`.
7. Responsive: below 900px, swap WebGL canvas for a static poster image + native scroll list of projects (feature-detect with `matchMedia` and `navigator.gpu`).
8. Accessibility: render full project list inside a visually-hidden `<nav>` for SR; `prefers-reduced-motion` short-circuits ScrollControls and renders the hidden nav as the default view.

WHY IT WORKS: The site collapses navigation, content, and brand expression into a single spatial metaphor — visitors learn the studio's aesthetic *by piloting it*. Long camera dwell on each island enforces a deliberate rhythm (motion semantics: slow = considered), while the bloom + grain combo earns a cinematic register without resorting to flat gradients. Discoverability is preserved because every island is already on a guided spline — exploration is bounded, never lost.

GOOD / BAD:
- GOOD: Single canvas, Draco+KTX2, frameloop on demand → 60fps on mid-tier laptops; navigation maps 1:1 to a hidden DOM list, so it's crawlable.
- BAD: Time-to-interactive on cold cache pushes 4s+ on 4G; the mobile fallback is a downgrade — visitors on phones never see the brand's actual idea.

CLEAN APPLICATION: Apply this to Clean's landing-page hero by replacing the typical "headline + CTA + product screenshot" stack with a slow-orbiting WebGL scene of Clean's architecture: a stylized wireframe MCP "trunk" with branches representing tool calls, payloads visibly compressing into TOON-format ribbons as they travel. Mouse position parallaxes camera; scroll dollys forward through three stops — Headline ("Context-engineered for agents"), Stat ("84% fewer tokens"), CTA ("Get Clean"). Use `@react-three/fiber` + `drei`'s `MeshTransmissionMaterial` on the trunk for that liquid-glass developer-tool aesthetic. Keep below-the-fold sections in standard DOM. Mobile fallback: a static SVG of the same scene with the headline overlaid.

## 2. Fourmula AI — https://fourmula.ai/

WHAT IT DOES: A vertical-scroll AI product page where the hero "product" (a generated image of a physical object) morphs through a sequence of poses/lighting presets as the user scrolls. The morph is driven by ScrollTrigger pinning a section, then crossfading between pre-rendered WebP sequences using opacity + scale tweens. Surrounding UI (caption, prompt-bubble, control sliders) animates in with staggered word-by-word splits. Sticky horizontal scroll showcases preset gallery with `scrub: true` and a custom cursor that snaps to interactive zones.

PACKAGES:
- webflow (host) — but for code rebuild: next@14 + tailwindcss@3.4
- gsap@3.12 with ScrollTrigger, ScrollSmoother, SplitText (Club GreenSock for SplitText, or community `split-type@0.3` as drop-in)
- barba.js@2 (page transitions) — or `next/navigation` `useTransition` with framer-motion `LayoutGroup`
- lenis@1
- swiper@11 (preset carousel)
- next/image with `loader: "custom"` to pull from R2 / Cloudflare Images
- font: `PP Neue Montreal` (or `Geist`) headline + `JetBrains Mono` for prompt UI
- icons: lucide-react@0.460

IMPLEMENTATION:
1. Hero pin: `ScrollTrigger.create({ trigger:'.hero', start:'top top', end:'+=2400', pin:true, scrub:0.6, snap:{ snapTo:1/N, duration:0.4, ease:'power3.inOut'}})`. Inside the timeline, advance an integer `frameIndex` 0→N and use `requestAnimationFrame` to swap `<picture>` `srcset` (preload all frames as `<link rel="preload" as="image">`).
2. Word-stagger headlines: `SplitText(el,{type:'words,chars'})` then `gsap.from(split.words,{ yPercent:120, opacity:0, duration:0.9, stagger:0.04, ease:'expo.out' })`. Trigger on enter with `ScrollTrigger.batch`.
3. Prompt bubble: a Tailwind `rounded-2xl bg-zinc-900/70 backdrop-blur-xl ring-1 ring-white/10 p-4` with a typewriter effect that writes the prompt one char per 22ms, then a subtle "shimmer" sweep using `bg-[linear-gradient(110deg,transparent,rgba(255,255,255,0.12),transparent)]` + `animate-[shimmer_2.4s_linear_infinite]`.
4. Custom cursor: `position:fixed`, `mix-blend-mode:difference`, drives via `gsap.quickTo(el,'x',{duration:0.35,ease:'power3'})`. Adds `data-cursor="zoom"` zones that scale cursor 4x with a spring (`stiffness:120, damping:14`).
5. Color system: near-black `#0B0B0E` background, fog gradient `radial-gradient(60% 50% at 50% 0%, rgba(255,90,30,0.18), transparent 70%)`, accent `#FF5A1E`, text white at 92% opacity, mono UI in `#A1A1AA`.
6. Page transitions: barba `enter` clones the clicked card into a fixed wrapper, `gsap.to(clone,{ inset:0, borderRadius:0, duration:0.9, ease:'expo.inOut' })`, then mount next route.

WHY IT WORKS: The pinned hero converts the product's central capability (generating consistent product imagery) into the *interaction itself* — scrolling literally generates the next render. Visual hierarchy is enforced by the dark canvas + single-color accent (#FF5A1E), which keeps the orange CTA the only saturated element on the page. Word-level type-in animation primes attention without becoming decorative because each word arrives in service of reading rhythm.

GOOD / BAD:
- GOOD: Concept-content alignment is exact (the page demos the product); preloaded WebP frame sequence is cheap and 60fps everywhere.
- BAD: 14MB total preload kills first-paint on slow networks; SplitText without `prefers-reduced-motion` guard violates WCAG 2.3.3.

CLEAN APPLICATION: Apply this to Clean's product demo section ("How it works") on tryclean.ai. Pin the section and scrub through three states of a single agent request: (1) raw, bloated tool response (long JSON, ~28K tokens labeled), (2) Clean's MCP proxy intercepting and compressing, (3) TOON-format payload (~4.5K tokens). Use a frame-sequence WebP morph between states with the same scrub config (`scrub:0.6`, snap 1/3) so the user controls the compression. Type the prompt-bubble with the user's actual mental model: "Find unread Slack messages from this week." Replace #FF5A1E with Clean's brand accent.

## 3. Studio375 — https://www.awwwards.com/sites/studio375-2

WHAT IT DOES: A two-color (#cc0597 magenta / #11baff cyan) typography-driven agency site built on Next.js + GSAP. Headlines use enormous variable-font weight that animates `font-variation-settings: "wght" 200→900` on scroll-in. Hero features a "marquee" track of project names with overlapping color-blend bands (mix-blend-mode: difference) that streak across as cursor moves. Section transitions cross-wipe with full-bleed colored panels that slide in opposite directions to reveal next content.

PACKAGES:
- next@14
- tailwindcss@3.4 + tailwind-merge + tailwindcss-animate
- gsap@3.12 + ScrollTrigger + Flip + SplitText (or `split-type`)
- framer-motion@11 (for layout-aware transitions)
- lenis@1
- @studio-freight/hamo@0.6 (utility hooks)
- font: `Söhne Breit` or `Migra` variable headline; `Söhne Mono` for captions
- next/font/local

IMPLEMENTATION:
1. Variable-weight reveal: on each heading, `gsap.fromTo(el,{ '--wght': 200, opacity:0.4 },{ '--wght':900, opacity:1, duration:1.4, ease:'power3.out', scrollTrigger:{ trigger:el, start:'top 80%' }})` and CSS `font-variation-settings: 'wght' var(--wght);`.
2. Marquee with blend: two stacked `<div>`s with `animation: marquee 40s linear infinite` (one normal, one reversed). Apply `mix-blend-mode: difference` on the upper layer; both layers contain the same text but tinted cyan and magenta. Set parent background white so blend produces the brand split.
3. Cursor streak: a 24×24 magenta square follows pointer with `gsap.quickTo`, plus a trailing trail of 8 squares each lerping at successively lower factors (0.18, 0.15, 0.12 …). Set `mix-blend-mode: exclusion` on each.
4. Section wipe: on link click, render two fixed full-bleed `<div>`s — top half cyan, bottom half magenta. Animate `clipPath: inset(0 0 100% 0)` → `inset(0 0 0% 0)` for top and inverse for bottom over 0.7s `ease:'expo.inOut'`, then unmount once next route mounts.
5. Layout: 12-col grid, `gap-x-6 gap-y-24`, container max-width `1440px`, side-padding `clamp(16px, 4vw, 96px)`. Headlines `text-[clamp(3rem,8vw,9rem)] leading-[0.92] tracking-[-0.04em]`.
6. Performance: `font-display:swap`, preload only the variable font's primary subset, ship `woff2` with `unicode-range` Latin-only (~22KB).

WHY IT WORKS: The two-color system is a brand-identity machine — every section reads as Studio375 within 0.3s because the palette is permanent and the typography is the content. The mix-blend-mode marquee enforces Gestalt figure-ground reversal, which makes static type feel kinetic without GPU-heavy WebGL. Variable-weight scroll animation is "motion semantics done right": the page literally gets bolder as you commit to it.

GOOD / BAD:
- GOOD: Sub-1s LCP on the homepage; entire visual identity ships in two CSS variables; high contrast = high accessibility.
- BAD: Pure-typography design has zero affordance for showing the agency's actual work above the fold; some users will bounce thinking the site is a teaser.

CLEAN APPLICATION: Apply this to Clean's pricing page. Use a two-color split (Clean's primary + a complementary accent) with each pricing tier as a column whose heading scales `font-variation-settings: 'wght'` from 300→800 as the user scrolls into it, signaling tier weight viscerally. Replace generic "pricing tier" cards with full-column color blocks; on hover, the column fills with the accent and the body copy reveals via `mix-blend-mode: difference`. CTA button uses the same cursor-streak treatment so the user's pointer literally trails into "Start free."

## 4. How has the climate evolved? — https://www.cssdesignawards.com/sites/how-has-the-climate-evolved (CSSDA SOTD May 4, 2026)

WHAT IT DOES: A single-page WebGL data app that renders a 3D landscape per location (search box) and animates 30 years of ERA5 weather data over its surface. The terrain is a heightmap displaced plane, with vertex colors encoding temperature anomaly. A timeline scrubber moves an animated decade window; a particle system overlays precipitation events. Camera orbits via OrbitControls with damped inertia; UI is a fixed sidebar with sliders, location autocomplete, and an SVG line chart that updates in lockstep with the 3D scene.

PACKAGES:
- vite@5 + react@18
- three@0.160 + @react-three/fiber@8 + @react-three/drei@9 (OrbitControls, useTexture, shaderMaterial, Stats)
- @react-three/postprocessing@2 (Bloom on cold-anomaly highlights only via mask)
- d3@7 (scaleLinear, scaleSequential, interpolateRdBu, geoMercator, line)
- d3-geo@3 + topojson-client@3 (world basemap)
- zustand@4 (store for time index, location, dataset)
- @tanstack/react-query@5 (Open-Meteo ERA5 fetch)
- maplibre-gl@4 (mini-locator map, optional)
- date-fns@3
- tailwindcss@3.4 + radix-ui/react-slider + cmdk@1 (location autocomplete)

IMPLEMENTATION:
1. Terrain: 256×256 PlaneGeometry, `position.array` displaced via DEM PNG decoded into a Float32Array (R+G*256 → meters). Vertex color attribute updated each frame from a `Float32Array(width*height*3)` mapped through `d3.interpolateRdBu(t)` where `t = (anomaly - min)/(max - min)`.
2. Custom ShaderMaterial: vertex shader passes `vColor`; fragment outputs `vec4(vColor, 1.0)` plus a Fresnel rim `pow(1.0 - dot(N, V), 2.0) * 0.35`. Add fog `gl_FragColor.rgb = mix(fogColor, gl_FragColor.rgb, exp(-fogDensity*depth))`.
3. Timeline: a `<Slider>` (Radix) bound to a zustand `timeIndex` (0..360 months over 30y). On change, recompute the vertex color buffer (web-worker via Comlink to keep main thread <16ms). Auto-play uses `useFrame` advancing index by `delta * 6` with wrap.
4. Precipitation particles: `THREE.Points` with 5,000 vertices, custom shader that animates `position.y -= time*velocity` and resets above scene; alpha shaped by `smoothstep` so streaks fade near terrain.
5. Side chart: a 320×120 SVG line chart, generated with `d3.line()`, that shows the same temperature series; a `<rect>` cursor mirrors the 3D timeIndex for cross-cue.
6. Performance: instanced terrain only built once per location; vertex color update happens in a Worker every 4th frame; `frameloop="demand"` triggered by store subscribers; `dpr=[1, 1.5]`; all Open-Meteo responses cached in IndexedDB via `idb-keyval`.

WHY IT WORKS: It earns trust because every visual claim is grounded in a public dataset (ERA5) the user can re-query. The dual encoding — vertex color *and* line chart — is a Gestalt redundancy gain: viewers form a stable mental model from the chart and validate it spatially on the terrain. Camera inertia and slow particle settle rate set a contemplative pace appropriate for climate data without crossing into doom-aesthetic.

GOOD / BAD:
- GOOD: WebGPU-friendly architecture, sub-150KB initial JS (deferred Three.js), fully keyboard-navigable timeline.
- BAD: First terrain build for a new location stalls 600–900ms on mid-range Android; no API rate-limit UI when Open-Meteo throttles.

CLEAN APPLICATION: Apply this to Clean's dashboard analytics view (the post-login surface where developers see token-usage trends across MCP servers). Build a 3D "token landscape" where x = MCP server, y = day, z = tokens compressed. Color encodes compression ratio via `d3.interpolateViridis`. The timeline scrubber animates the last 30 days, and a secondary 2D line chart in the side panel shows total savings. Use the same zustand + worker architecture so live data via Clean's own API never blocks the main thread. This makes Clean's value visceral: developers literally see a mountain range of saved tokens.

## 5. Aceternity UI — Background Ripple Effect (new drop) — https://ui.aceternity.com/components/background-ripple-effect

WHAT IT DOES: A full-viewport CSS grid of square cells (e.g., 40×20) where clicking any cell propagates a damped wave to neighbors based on Manhattan distance, briefly inverting their background color and scaling them ~1.06. The hero text sits above the grid with a `mask-image: radial-gradient(ellipse, black 40%, transparent 70%)` revealing only the cells behind the focal copy. Hover state runs a constant low-amplitude shimmer (`animation-delay: calc(var(--i) * 60ms)`).

PACKAGES:
- next@14 (or vite + react@18)
- tailwindcss@3.4 + tailwind-merge + clsx
- framer-motion@11 (for ripple state)
- lucide-react@0.460
- shadcn/ui@latest (Button) — `pnpm dlx shadcn@latest add button`
- font: `Geist Sans` + `Geist Mono` via next/font

IMPLEMENTATION:
1. Grid: `<div className="grid" style={{gridTemplateColumns:'repeat(40,1fr)',gridTemplateRows:'repeat(20,1fr)'}}>`. Each cell is a `<motion.div>` with base classes `border border-white/[0.03] bg-transparent`.
2. Click handler computes Manhattan distance from clicked `(r,c)`: `const d = Math.abs(r-cr)+Math.abs(c-cc)`. Use `framer-motion`'s `animate(controls,{ scale:[1,1.06,1], backgroundColor:['rgba(255,255,255,0)','rgba(255,255,255,0.18)','rgba(255,255,255,0)'] }, { duration:0.6, delay: d * 0.045, ease:[0.22,1,0.36,1] })`.
3. Mask the grid behind the hero copy: wrap grid in `<div className="absolute inset-0" style={{maskImage:'radial-gradient(ellipse 60% 45% at 50% 45%, black 35%, transparent 75%)', WebkitMaskImage: same}}>`.
4. Hero copy: centered `text-balance` headline `text-5xl md:text-7xl font-semibold tracking-[-0.04em]`, sub `text-zinc-400 text-lg max-w-xl`, CTA shadcn `<Button size="lg" className="rounded-full">`.
5. Theme: bg `#09090B`, cell border `rgba(255,255,255,0.04)`, ripple `rgba(255,255,255,0.18)`, accent `#A78BFA` for CTA glow ring.
6. Performance: pointer-event handler on the grid wrapper (event delegation) — *not* per cell. Cap simultaneous ripples to 3 with a queue. Pause animations under `prefers-reduced-motion`.

WHY IT WORKS: A static grid converts blank canvas into a "you can touch this" affordance instantly — every cell is a Fitts'-targetable zone. The mask gradient enforces a clean focal hierarchy (text > grid > everything else) without resorting to a low-contrast hero overlay. The ripple's wave-front speed (~22 cells/s) is fast enough to feel responsive but slow enough to be readable as a propagation, which is exactly the perceptual sweet spot for "system reacting to me."

GOOD / BAD:
- GOOD: 5KB of code, zero dependencies beyond framer-motion, infinitely brand-able by tweaking two CSS variables; plays well with shadcn theming.
- BAD: 800 DOM nodes for the grid hurts JS-disabled environments and slow CPUs; without the `prefers-reduced-motion` guard the constant shimmer is a vestibular trigger.

CLEAN APPLICATION: Apply this to Clean's landing page hero (tryclean.ai) as the *primary* hero background. Replace generic gradient mesh with a 40×20 cell grid where each cell represents a "context token." When a visitor clicks anywhere, the ripple reads as Clean's compression signal — visualizing the product metaphor on first impression. Use Clean's brand color for the ripple (instead of white) and pair with a single H1 ("Less context. More agent.") and a single CTA ("Get a Clean key"). Keep the entire hero under 8KB JS so first-paint stays sub-1s — critical for a developer-tool audience that bounces on slow first impressions.

## BUILD DIRECTIVE

TARGET: tryclean.ai landing-page hero (above-the-fold).

APPROACH: Build a single hero section that fuses (a) the Aceternity ripple grid as the live, click-reactive background, (b) Studio375's variable-weight typographic reveal for the H1, and (c) Where-Worlds-Take-Shape's slow camera-parallax pointer offset applied to the H1+CTA group via a non-canvas CSS transform (no Three.js needed at this tier). Cell click triggers a damped Manhattan-distance ripple in Clean brand color; H1 ("Less context. More agent.") animates `font-variation-settings: 'wght' 300→900` on first paint over 1.4s `ease:'expo.out'`; pointer position drives a `translate3d` on the headline group with `gsap.quickTo` damping at 0.18 — gives the depth of WebGL parallax without the bundle cost. Sub-headline = one sentence on TOON compression. Single CTA. No carousel, no logo wall, no scrolljack. Below-the-fold remains current Clean structure.

STACK:
- next@14 (App Router) + react@18 + typescript@5
- tailwindcss@3.4 + tailwind-merge + clsx
- framer-motion@11 (ripple cell animations)
- gsap@3.12 (`quickTo` for pointer parallax, variable-weight tween)
- shadcn/ui (Button only)
- lucide-react@0.460
- next/font: `Geist Sans` (variable, weight 100–900) + `Geist Mono`
- No Three.js / R3F at this tier — keep hero JS budget under 12KB gzipped.

REFERENCE URLS:
1. https://ui.aceternity.com/components/background-ripple-effect (grid + ripple mechanic — copy directly)
2. https://www.awwwards.com/sites/studio375-2 (variable-weight typography reveal — adapt to brand fonts)
3. https://www.awwwards.com/sites/where-worlds-take-shape (pointer-driven parallax feel — translate to CSS)

KEY CONSTRAINTS: Clean is developer infrastructure. Avoid: gradient meshes, glassmorphism for its own sake, marketing-y stock illustrations, hero videos, scrolljack on first load, animated gradients on text, "AI sparkle" emoji, multi-CTA "Sign up / Watch demo / Read docs" hero rows. Required: monospace numeric copy ("84% fewer tokens" must be `Geist Mono tabular-nums`), every animation gated behind `prefers-reduced-motion`, hero TTI under 1.2s on 4G, hero accessible without JS (server-render H1/CTA, hydrate ripple after). Result must read as `Linear × Vercel × Stripe` — quiet, technical, fast — not as a flashy Awwwards portfolio.
