## 1. ASTRODITHER — https://www.awwwards.com/sites/astrodither

WHAT IT DOES: A fullscreen single-page WebGL canvas runs a GPU dither pass over a procedurally morphing 3D blob (Three.js scene with one mesh, post-process pipeline). The hero is the canvas — copy is overlaid in monospace, anchored to the four corners of the viewport (top-left brand, top-right nav, bottom-left meta, bottom-right CTA). Mouse position drives a uniform feeding the noise function in the fragment shader, so the dither pattern follows the cursor with a 60–80ms lag (lerped uMouse). A scroll-jacked transition on first scroll triggers a chromatic-aberration ramp (uAberration 0 → 0.04 over 800ms) and pushes the camera back on the Z axis from z=2.4 to z=4.2.

PACKAGES:
- `three@^0.165.0`
- `@react-three/fiber@^8.17.0`
- `@react-three/drei@^9.108.0`
- `@react-three/postprocessing@^2.16.0`
- `postprocessing@^6.35.0` (peer)
- `gsap@^3.12.5` (timeline + ScrollTrigger)
- `lenis@^1.3.0`
- `next@^15.0.0` (App Router)
- `tailwindcss@^3.4.0`
- Font: `geist` (Vercel) for monospace anchor copy, `Editorial New` (PangramPangram) for the centered display word

IMPLEMENTATION:
1. Component tree: `<HeroCanvas/>` (R3F `<Canvas dpr={[1, 1.5]} gl={{ antialias: false, powerPreference: 'high-performance' }}>`) → `<DitherBlob/>` (an `<Icosahedron args={[1, 64]}>` with a custom `shaderMaterial`) wrapped in `<EffectComposer multisampling={0}>` containing a `<DitherEffect/>` custom postprocessing pass.
2. Vertex shader: classic noise-driven displacement — `pos += normal * snoise(vec4(pos*1.6, uTime*0.25)) * 0.18;`. Fragment shader of the mesh outputs a velocity-based color `vec3(0.95, 0.92, 0.86)` lit by a single hemisphere light (sky `#fffaf0`, ground `#1a1a18`, intensity 0.8).
3. Dither post pass (Bayer 8×8 ordered dither): in the effect's `fragmentShader`, sample `inputBuffer`, convert to luma `dot(rgb, vec3(0.299,0.587,0.114))`, threshold against `bayer[(int(gl_FragCoord.x)%8)*8 + int(gl_FragCoord.y)%8] / 64.0`, output two-tone `mix(uColorDark, uColorLight, step(threshold, luma))`. Defaults: `uColorDark=#0a0a0a`, `uColorLight=#f5efe6`.
4. Mouse uniform: in `useFrame`, `mesh.material.uniforms.uMouse.value.lerp(mouseVec2, 0.08)`. Bind cursor in a `usePointer` hook reading `pointermove` with passive listener.
5. Layout: 100vh hero with `grid-template-rows: auto 1fr auto; grid-template-columns: 1fr auto 1fr;`. Corner copy is `position: absolute` with 32px insets, font-size 12/16px monospace, letter-spacing 0.06em, uppercase. Centered display word uses `clamp(72px, 14vw, 220px)`, line-height 0.9, mix-blend-mode: difference against canvas.
6. Scroll: Lenis `{ duration: 1.4, easing: t => Math.min(1, 1.001 - Math.pow(2, -10*t)) }`. ScrollTrigger pinned for first 100vh scrub: timeline tweens `uAberration` and `camera.position.z` with `ease: 'power2.inOut'`. Sync via `lenis.on('scroll', ScrollTrigger.update); gsap.ticker.add(t => lenis.raf(t*1000)); gsap.ticker.lagSmoothing(0);`.
7. Performance: `dpr` capped at 1.5, `antialias: false` (dither hides the aliasing anyway), `multisampling: 0`, frameloop `'demand'` falls back to `'always'` while `uMouse` is being lerped. `prefers-reduced-motion` swaps the canvas for a static rendered PNG.

WHY IT WORKS: Two-tone dither eliminates color-noise as a variable, forcing the eye onto silhouette and motion — pure figure/ground Gestalt. The corner-anchored grid creates aggressive negative space in the middle, which the morphing blob owns; this is the rule of thirds replaced with rule of corners. Mouse-following motion gives proprioceptive feedback in <100ms, which is below the conscious threshold and reads as the page being "alive" rather than reactive.

GOOD: GPU dither at 60fps even on integrated graphics; the aesthetic is unforgeable in CSS. BAD: Two-tone hides product UI screenshots (any texture mid-grays gets crushed), and it's pure mood — no information density on first paint.

CLEAN APPLICATION: Apply this to Clean's landing-page hero (tryclean.ai homepage, 0–100vh). Replace the morphing blob with a 3D wireframe representing an MCP request graph — tiny nodes (MCP servers) connected by Bezier edges (token flows), all rendered into the same dither pass. The cursor lerp drives a "selection" highlight that picks the nearest node and brightens its uColorLight band. Corner copy: top-left `clean.ai`, top-right `[product] [docs] [pricing]`, bottom-left `mcp proxy · context layer · TOON encoding`, bottom-right CTA `→ get a demo`. Centered display word: `Compress the context.` This makes the hero unmistakably about agent infrastructure, not a generic blob.

## 2. Where Worlds Take Shape — https://www.awwwards.com/sites/where-worlds-take-shape

WHAT IT DOES: A horizontally scrolled WebGL portfolio where each "case study" is a self-contained mini-game played on a stylized 3D diorama (palette HEX `#1F3D2B` deep green, `#F2A65A` warm sand). Scroll moves the camera along a Catmull-Rom spline through a single continuous world (no scene swaps, no router transitions). Each project zone has a 2–10 second interactive moment — drag a slingshot, route a wire, push a block — that gates progression: the next section unlocks only after the micro-interaction completes, then a soft transition pans the camera to the next diorama.

PACKAGES:
- `three@^0.165.0`
- `@react-three/fiber@^8.17.0`, `@react-three/drei@^9.108.0` (CameraControls, useGLTF, Float, MeshTransmissionMaterial)
- `@react-three/rapier@^1.5.0` (physics for the slingshot/draggables)
- `gsap@^3.12.5` + `gsap/CustomEase`
- `lenis@^1.3.0` (with `orientation: 'horizontal'`)
- `leva@^0.9.35` (dev only)
- GLSL chunks via `glslify@^7.1.1`
- Fonts: `Migra` (Pangram Pangram) display, `Inter` (variable) UI

IMPLEMENTATION:
1. Single `<Canvas>` mounted once. Camera is a `PerspectiveCamera` with `fov=35`. A `useScroll` hook (drei) returns scroll progress `[0,1]`; map progress through `CatmullRomCurve3.getPointAt(p)` and `getTangentAt(p)` to position camera and target.
2. World layout: 6 dioramas placed at fixed world coords along the curve, each ~12 units long, ~8 units wide. Use `<Detailed>` LODs; far dioramas are shown as low-poly proxies (250 tris) and swap to full mesh (8–15k tris) when their bbox enters camera frustum.
3. Mini-game pattern: each diorama exposes `onActivate(progress)`. When camera spline param enters its range, `progress` ramps 0→1 over 600ms with `ease: CustomEase.create('soft', 'M0,0 C0.34,0.01 0.27,1 1,1')`. Game logic enables only at progress > 0.85. Completion sets a Zustand flag; uncompleted games soft-block scroll by clamping Lenis target.
4. Lighting: a single warm directional `#F2A65A` at intensity 1.6, sky-blue `#7CA9C8` ambient at 0.35. Soft shadows via `accumulativeShadows` (8 frames, opacity 0.7).
5. Materials: ground uses `MeshToonMaterial` with a 4-step gradient texture; foliage uses a custom `shaderMaterial` doing wind sway `pos.x += sin(uTime*1.2 + worldPos.z*0.3) * 0.04`. Water sections use `MeshTransmissionMaterial` (drei) with `thickness=0.4`, `roughness=0.05`, `chromaticAberration=0.06`.
6. UI overlay: a fixed-position 13px monospace HUD top-left shows current zone name + completion bar (CSS gradient masked by `width: var(--p)`). Bottom-right shows scroll progress as ASCII `[█▓▒░  ]`.
7. Audio: tiny one-shots on game completion via `tone.js` (sub-50KB), gated behind explicit user gesture so iOS lets it autoplay after first scroll.
8. Mobile: switch to vertical scroll, replace draggable mini-games with single-tap activation, halve `dpr` to 1.0, and disable `MeshTransmissionMaterial` (replace with flat colored material).

WHY IT WORKS: Embedded interaction makes scroll an active rather than passive verb — the visitor *commits* a small physical gesture, which is the same dopamine hook used in idle games (variable-ratio reward + completion satisfaction). The single continuous world (no page transitions) hides loading and removes the "what's next" anxiety; everything is geographically present. The two-color palette acts as brand glue across wildly different scenes — color consistency is the cheapest way to fake aesthetic coherence.

GOOD: Mini-game gating turns time-on-page from a metric into a guaranteed outcome. BAD: Inaccessible (no keyboard/SR equivalent for slingshot), heavy first paint (~6MB), and on iPhone 13 the transmission material drops to ~38fps.

CLEAN APPLICATION: Apply this to Clean's product-demo section (the second screen of tryclean.ai, "How it works"). Build three sequential mini-interactions on one continuous WebGL stage: (1) drag-and-drop an MCP tool list into a "Clean proxy" cube and watch the cube compress them visually (TOON encoding shown as bytes shrinking); (2) drag a slider that scales the agent's context window — a stack of cards collapses into a denser column as TOON kicks in; (3) push a button that fans out parallel agent calls as glowing arrows leaving the cube. Camera is on a fixed spline that pans between the three. Completion of each gates scroll. This converts "what does Clean do" from a paragraph into a 30-second guided demo.

## 3. Fourmula AI — https://fourmula.ai/

WHAT IT DOES: A product landing for an AI image-generation tool, structured as a vertical narrative with five pinned scroll-scrubbed sequences. The hero is a single oversized character set in a sharp display serif with a chromatic-aberration ScrollTrigger reveal — letters split into RGB channels for the first 40px of scroll, then converge. Below, each feature section uses a "stacking cards" pattern where the prior card scales down from 1 → 0.94 and translates Y -40px, gaining a backdrop-blur, while the new card fades in from below. Built on Webflow with a heavy GSAP layer.

PACKAGES (if rebuilt outside Webflow):
- `next@^15.0.0`
- `gsap@^3.12.5` with `ScrollTrigger`, `SplitText`, `CustomEase`
- `lenis@^1.3.0`
- `tailwindcss@^3.4.0`
- `framer-motion@^11.3.0` (for the stacking-card gesture variant)
- `@vercel/og` (dynamic OG images)
- Font: `Migra` display, `JetBrains Mono` for code samples
- `lottie-react@^2.4.0` for icon micro-loops

IMPLEMENTATION:
1. Hero: a single `<h1>` with `font-size: clamp(96px, 18vw, 320px); line-height: 0.85; letter-spacing: -0.04em;`. Wrap in three identical layers absolutely positioned, each tinted via `mix-blend-mode: screen` with text colors `#FF003C`, `#00FFB2`, `#1A6BFF`. ScrollTrigger pins for 80vh, scrubs `transform: translateX(±12px)` on the R/G/B layers from -12/0/+12 to 0/0/0 over the first 0.4 of progress.
2. Stacking cards: use a `<section>` with `position: sticky; top: 80px;` for each card (height 80vh). GSAP timeline pinned across all cards: `gsap.to('.card-' + i, { scale: 0.94, y: -40, filter: 'blur(8px)', ease: 'power2.out' }, scrollProgress[i])`. Each card has a 60/40 split — copy left, product still right. Product still is a video poster + autoplay loop swapping in on `IntersectionObserver`.
3. Type system: display = Migra Extrabold; body = Inter 16/24, weight 400 with weight 600 for inline emphasis. Caps copy uses `font-feature-settings: 'ss01', 'cv11'` for the display — turning on the alternate `g` and stylistic set 1.
4. Motion semantics: every reveal uses one of three eases — `CustomEase.create('soft','M0,0 C0.4,0 0,1 1,1')` for entrances, `power2.inOut` for crossfades, `expo.out` for one-shot reveals. No bounce, ever.
5. Color system: background gradient `linear-gradient(180deg, #0B0B0E 0%, #14141A 100%)`. Accent `#FF6633`. Card backgrounds `rgba(255,255,255,0.04)` over the gradient with `backdrop-filter: blur(20px) saturate(1.4)` and a 1px inner border `rgba(255,255,255,0.08)`.
6. Performance: hero text is plain DOM (not canvas), so SEO and a11y are intact. Videos are 1.2MB H.265 with 720p MP4 fallback, `playsinline preload=metadata`. Total JS first-load ~140KB gzipped.

WHY IT WORKS: Oversized typography pre-commits to a strong opinion before the visitor reads anything — confidence is a brand signal that survives any amount of skepticism. The chromatic-aberration reveal is *just* expensive enough to feel premium without crossing into "we're showing off." Stacking cards solve the "feature dump" problem by making each feature occupy the full viewport in turn — every feature gets equal weight, no horizontal hierarchy to argue about.

GOOD: Webflow + GSAP combo means non-engineers can iterate copy daily without a deploy. BAD: 5 pinned ScrollTriggers compound; on a 60Hz iPad with 5×CPU throttle the stack visibly judders. SplitText breaks copy-paste accessibility unless `aria-label` is doubled.

CLEAN APPLICATION: Apply this to Clean's pricing + features page. Replace the marketing-card stack with a 4-card "what TOON saves you" sequence: (1) "5× fewer tokens" with a live counter animation 100,000 → 19,800; (2) "10× less latency" with a bar chart that shrinks; (3) "1 line to integrate" with a code block typewriter; (4) "$ saved monthly" with a calculator that scales by team size. Each card pins for 100vh with the prior card retreating behind blur. Use Migra for the display number, JetBrains Mono for the code, Inter for body. This turns Clean's value prop from a list into a cinematic.

## 4. Studio375 — https://www.awwwards.com/sites/studio375-2

WHAT IT DOES: An agency portfolio on Next.js with WebGL hover effects on the project grid (palette `#cc0597` magenta, `#11baff` cyan, charcoal background). Hovering a thumbnail triggers a per-project shader transition that displaces pixels along a gradient mask — RGB-split distortion on enter, smooth resolve on leave. Page transitions use a fullscreen color wipe with a clipped SVG path, hiding any next-page network latency under ~700ms of choreographed motion.

PACKAGES:
- `next@^15.0.0` (App Router with `loading.tsx` for streaming)
- `three@^0.165.0`, `@react-three/fiber@^8.17.0`, `@react-three/drei@^9.108.0` (specifically `<Image>` + custom shader override)
- `gsap@^3.12.5` + `Flip` plugin (for the FLIP-style thumbnail expansion)
- `lenis@^1.3.0`
- `tailwindcss@^3.4.0` + `tailwindcss-animate`
- `barba.js@^2.10.0` OR Next.js `useTransitionRouter` (Next 15+) for the page-transition hook
- Font: `PP Neue Montreal` (Pangram Pangram), `Söhne Mono` for captions

IMPLEMENTATION:
1. Project grid: 3-column CSS grid on desktop (`grid-template-columns: repeat(3, 1fr); gap: 24px`), each cell wraps a `<Link>` plus an absolutely positioned `<Canvas>` containing one `<Mesh>` with a `<planeGeometry args={[1,1]}>` matched to the cell's bbox via `useResize`. (Production-correct pattern: one shared canvas with multiple instanced meshes — see GOOD/BAD below.)
2. Hover shader: vertex shader is a passthrough; fragment does `vec2 dir = vUv - 0.5; float d = length(dir); vec2 offset = dir * uHover * 0.08; vec3 col; col.r = texture(uTex, vUv + offset).r; col.g = texture(uTex, vUv).g; col.b = texture(uTex, vUv - offset).b; col = mix(col, vec3(luma(col)), uHover * 0.3);`. Animate `uHover` 0→1 with `gsap.to({u: 0}, {u: 1, duration: 0.6, ease: 'expo.out', onUpdate})`.
3. FLIP transition on click: capture the thumbnail's `getBoundingClientRect()`, navigate to project page, on the project page mount the same image at the source bbox and animate to the new bbox using `Flip.from(state, { duration: 0.9, ease: 'power3.inOut', absolute: true })`. Background fills with the project's accent color simultaneously.
4. Page-transition wipe: a fullscreen SVG `<path>` clipped via `clip-path: path(...)`. On route change, animate the path's `d` from a flat line to a covering wave to a flat line on the other side — a 1200ms 3-keyframe GSAP timeline. Color is the destination page's accent token.
5. Cursor: custom `div.cursor` follows the pointer with `transform: translate3d()` lerped at 0.18, scale 1 → 1.4 on link hover, blend mode `difference` so it inverts over images and copy.
6. Type sizes: project titles `clamp(48px, 6vw, 96px)` with negative `letter-spacing: -0.03em`. Captions Söhne Mono 11px, uppercase, letter-spacing 0.08em.
7. Performance: `frameloop="demand"` and only invalidate while `uHover > 0.001`. `<Image>` decodes off the main thread (`decoding="async"`). Total ~110KB JS first paint thanks to dynamic `import()` of the canvas chunk after `requestIdleCallback`.

WHY IT WORKS: The hover shader gives every project tile a fractional-second moment of "this thing is alive" — a kinetic confirmation that the click will be acknowledged, which lowers click hesitation. Clip-path wipes hide perceived latency: 700ms of animated transition feels faster than a 300ms hard nav because the eye has somewhere to go. The two-color brand palette + grayscale photography means every project looks like it belongs to the same studio even when the underlying client work is wildly different.

GOOD: FLIP transitions across routes feel native, and the per-tile shaders quantize beautifully (no aliasing). BAD: One canvas per tile burns GPU; a single shared canvas with multiple instanced planes is the production-correct pattern. Cursor blend-mode breaks on Safari < 17.

CLEAN APPLICATION: Apply this to Clean's "Customers" or "Use cases" gallery (a future page on tryclean.ai/customers). For each customer logo / case study tile, run the same RGB-split shader on hover — but tint the displacement using *their* TOON savings number as a hue: 4× saving = blue, 8× = teal, 12× = green. On click, FLIP-animate the tile to a fullscreen detail showing their MCP integration architecture. Use `PP Neue Montreal` for the company name, `JetBrains Mono` for the savings stats. The palette becomes a data-viz signal, not just decoration.

## 5. Aceternity UI — Background Ripple + Canvas Reveal — https://ui.aceternity.com/components/background-ripple-effect

WHAT IT DOES: A drop-in React component (recently added to the Aceternity registry alongside `Canvas Reveal Effect`, `Dotted Glow Background`, `Spotlight New`, `Encrypted Text`, `Layout Text Flip`, `Pixelated Canvas`) that renders a CSS-grid of cells; clicking any cell triggers a radial wave of opacity + scale through neighboring cells, with timing scaled by Manhattan distance from the click. Pairs with `Canvas Reveal Effect` — a WebGL particle field that "reveals" on hover, sweeping particles into a CTA target. Both are hero-section background utilities meant to live behind copy without competing for attention.

PACKAGES:
- `react@^18.3.0`, `next@^15.0.0`
- `tailwindcss@^3.4.0`
- `framer-motion@^11.3.0` (used by Aceternity for variants/animate)
- `clsx@^2.1.0`, `tailwind-merge@^2.5.0` (Aceternity's `cn` helper)
- `motion@^11.3.0` (the lighter-weight `motion/react` if migrating)
- For Canvas Reveal: `@react-three/fiber@^8.17.0`, `@react-three/drei@^9.108.0`, `three@^0.165.0`
- Optional: `lucide-react@^0.400.0` for Aceternity's stock icons

IMPLEMENTATION (Background Ripple):
1. Render a `<div>` with `display: grid; grid-template-columns: repeat(57, minmax(0, 1fr)); inset: 0; position: absolute;` and inside it `57 * 36 = 2052` `<div>` cells. Cell base style: `border: 1px solid rgba(255,255,255,0.06); aspect-ratio: 1;`.
2. On click, capture `(rowIdx, colIdx)`. For each cell `(r, c)`, compute `delay = Math.hypot(r - rowIdx, c - colIdx) * 18` ms. Use Framer Motion `animate(cellRef, { backgroundColor: ['transparent', 'rgba(56,189,248,0.45)', 'transparent'], scale: [1, 1.08, 1] }, { duration: 0.6, delay: delay/1000, ease: [0.22, 1, 0.36, 1] })`.
3. Cap simultaneous animations at ~600 by short-circuiting cells with `delay > 1100ms`. Use `will-change: background-color, transform` on a single ancestor only — never on the 2k cells.
4. Add a soft radial mask over the grid: `mask-image: radial-gradient(circle at center, black 0%, black 30%, transparent 70%)` to fade the grid into the page background without a hard edge.
5. Compose with copy: place the ripple grid at `z-index: 0` inside the hero, copy at `z-index: 10` with `pointer-events: none` for the headline so clicks fall through to the grid; CTA buttons sit at `z-index: 20` and absorb their own clicks.

IMPLEMENTATION (Canvas Reveal):
1. Single R3F canvas (`dpr=[1,2]`) with a custom shader plane covering the parent. Particles are a `<points>` mesh with 8000 instances arranged on a noise-modulated 2D plane.
2. Uniforms: `uTime`, `uHover` (0→1 lerped at 0.12), `uColor` (Vec3), `uReveal` (Vec2 cursor in clip space).
3. Vertex shader displaces particles along the normalized direction from the cursor: `pos.xy += normalize(pos.xy - uReveal) * uHover * 0.4`. Fragment renders disc with smoothstep edge `alpha = 1.0 - smoothstep(0.4, 0.5, length(gl_PointCoord - 0.5))`.
4. Wrap as `<CanvasRevealEffect color="#00d4ff" containerClassName="absolute inset-0">{children}</CanvasRevealEffect>`. The `containerClassName` slots into a Tailwind absolute layer behind copy.

WHY IT WORKS: Both components weaponize cursor-correlated motion — the user moves the mouse, something in the world responds within one animation frame, and the brain registers "responsive system" before reading any copy. The radial-mask soft edge avoids the #1 mistake of background animations (visible bounding boxes) and lets copy own the visual hierarchy. Manhattan-distance staggering matches how humans expect ripples to propagate (fastest light → deepest shadow falloff) — violating that expectation feels uncanny.

GOOD: Drop-in copy-paste, Tailwind-native, ~6KB extra JS for the ripple. BAD: 2k DOM nodes on a single grid is an accessibility-tree disaster; screen readers will read it as a 2052-cell unlabeled grid unless you set `aria-hidden="true"` on the wrapper. On low-end Android, click-to-paint exceeds 100ms.

CLEAN APPLICATION: Apply this to Clean's docs landing page (tryclean.ai/docs) hero. Use Background Ripple as the backdrop with cells that visually represent MCP tool calls — each cell labeled by an invisible data-attribute mapping to a tool name. When the visitor lands, programmatically trigger a ripple from the geometric center every 4 seconds using a low-amplitude variant (scale 1.02, opacity 0.2) so the page feels "live" without being noisy. On hover over a docs nav link, fire a ripple originating from the nav item's screen position — this binds the doc IA to the visual field. Replace the cyan accent with Clean's brand color (deep teal `#0F766E` recommended; otherwise use whatever `--brand` token lives in Clean's design system).

## BUILD DIRECTIVE

TARGET: tryclean.ai landing-page hero (the first 100vh of the homepage).

APPROACH: Build a single fullscreen WebGL hero that composes the strongest patterns from today's scout. The canvas runs a Three.js scene with one mesh — a low-poly wireframe MCP request graph (12–20 nodes, edges as Bezier curves) — under an ordered-dither postprocess pass (ASTRODITHER pattern). The cursor lerps a "selection" uniform that brightens the nearest node and fades the rest. Above the canvas, four corner-anchored monospace blocks anchor information density (top-left brand, top-right nav, bottom-left product line, bottom-right CTA), and a single oversized centered display word ("Compress.") uses `mix-blend-mode: difference` against the canvas (Fourmula AI pattern). On first scroll, ScrollTrigger scrubs a chromatic-aberration ramp + camera Z push, then transitions into a stacked-card explainer for the rest of the page. Aceternity's Background Ripple lives behind the docs page (separate surface, secondary install), not the hero.

STACK:
- `next@^15.0.0` (App Router)
- `react@^18.3.0`, `react-dom@^18.3.0`
- `three@^0.165.0`, `@react-three/fiber@^8.17.0`, `@react-three/drei@^9.108.0`
- `@react-three/postprocessing@^2.16.0` + `postprocessing@^6.35.0`
- `gsap@^3.12.5` (CustomEase, ScrollTrigger, SplitText)
- `lenis@^1.3.0`
- `tailwindcss@^3.4.0`, `clsx@^2.1.0`, `tailwind-merge@^2.5.0`
- `motion@^11.3.0` (for non-3D micro-interactions)
- `geist` (Vercel mono + sans), `Migra` display via `next/font/local`

REFERENCE URLS:
- https://www.awwwards.com/sites/astrodither (dither shader + corner-grid layout)
- https://fourmula.ai/ (oversized type, chromatic aberration, stacked-card scroll)
- https://www.awwwards.com/sites/where-worlds-take-shape (single continuous WebGL scene, scroll-bound camera spline) — reference for the section-2 product demo, not the hero

KEY CONSTRAINTS: Clean is developer infrastructure; the page must look like Vercel/Linear/Anthropic's serious cousin, not an art-school portfolio.
- No bouncy springs anywhere. All eases are `power2.inOut`, `expo.out`, or a single `CustomEase.create('clean','M0,0 C0.32,0.01 0.18,1 1,1')`.
- No autoplaying audio. No mouse-trail particles. No floating animated emoji.
- No more than 1 hero animation system; Lenis + ScrollTrigger + R3F is the entire motion stack.
- Total first-load JS budget: 180KB gzipped. Total transferred bytes including the WebGL scene: <600KB.
- LCP under 2.0s on Slow 4G simulation; Three.js chunk is `dynamic()` imported with `ssr: false` so the LCP candidate is the H1, not the canvas.
- 100% keyboard-navigable: tab order skips the canvas (`role="img" aria-label="..."`), all corner copy + CTA reachable in 4 tabs.
- `prefers-reduced-motion` swaps the dithered scene for a static rendered PNG and disables ScrollTrigger scrubs (use snap-points instead).
- No purple/pink gradients. No glassmorphism cards in the hero. No "AI-generated" looking abstract shapes — the geometry must be specifically the MCP request graph.
